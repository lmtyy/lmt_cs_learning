**LFSR (Linear Feedback Shift Register，线性反馈移位寄存器)** 是流密码（Stream Cipher）中最基础、最重要的构建模块。

我们在之前提到的 CSS（DVD加密）、A5/1（GSM手机加密）以及早期的军事密码机中，核心动力源全都是它。

简单来说，LFSR 是一个能产生**“看起来很随机，但实际上是有规律”**的比特流的硬件机制。

------

### 1. 名字拆解 (What is it?)

我们可以把它拆成三个词来理解：

1. **移位寄存器 (Shift Register)**：
   - 它本质上是一排存比特的盒子（比如 4 个格子）。
   - 每个时钟周期，所有比特向右移动一格。
   - 最右边的比特“掉出来”，成为输出（Output）。
2. **反馈 (Feedback)**：
   - 最左边的格子空出来了，填什么进去呢？
   - 不能瞎填，要根据寄存器里现有的某些特定位置的比特算出来，填回去。
3. **线性 (Linear)**：
   - 这个“算出来”的规则非常简单，只允许使用 **异或 (XOR)** 运算（在数学上，异或就是模2加法，是线性的）。

------

### 2. 工作流程演示 (How it works)

让我们看一个最简单的 **4位 LFSR** 例子。

- **状态**：假设寄存器有 4 位 $[b_3, b_2, b_1, b_0]$。
- **抽头 (Taps)**：我们要选几个位置来做反馈。假设选 $b_1$ 和 $b_0$。
- **反馈函数**：新输入的比特 $New = b_1 \oplus b_0$。

**初始状态（种子）：** `1 0 0 1` (即 $b_3=1, b_2=0, b_1=0, b_0=1$)

**第一步 (Clock 1):**

1. **输出**：最右边的 `1` 输出了。
2. **计算反馈**：看 $b_1$ (0) 和 $b_0$ (1)。计算 $0 \oplus 1 = 1$。
3. **移位**：整体右移。`1 0 0` 变成了 `_ 1 0 0`。
4. **填入**：把算的 `1` 填到最左边。
5. **新状态**：`1 1 0 0`。

**第二步 (Clock 2):**

1. **输出**：最右边的 `0`。
2. **计算反馈**：看现在的 $b_1$ (0) 和 $b_0$ (0)。计算 $0 \oplus 0 = 0$。
3. **移位并填入**：变成 `0 1 1 0`。

如此循环往复，不断吐出 0 和 1，形成密钥流。

------

### 3. 核心性质：周期与多项式

LFSR 有几个非常酷的数学性质，这也是它被用于密码学的原因：

#### A. 它是周期的 (Periodic)

因为只有 $n$ 个比特，状态总数是有限的（$2^n$ 种）。所以，它最终一定会回到之前的某个状态，然后开始无限循环。

- **最大周期：** 如果设计得当（选对了抽头），LFSR 可以遍历除了“全0”以外的所有状态。
- **周期长度：** $2^n - 1$。
- *例子：* 一个 16 位的 LFSR，周期是 65535。一个 64 位的 LFSR，周期是 $1.8 \times 10^{19}$（非常长）。

#### B. 它是确定的 (Deterministic)

只要初始状态（种子/密钥）一样，生成的流永远是一样的。这就是流密码解密的基础。

#### C. 特征多项式 (Primitive Polynomial)

你选择哪几位进行 XOR（抽头位置），可以用一个多项式表示。

- 比如上面的例子，$b_{new} = b_1 \oplus b_0$ 且是4位，数学上可能对应多项式 $P(x) = x^4 + x + 1$。
- 只有当这个多项式是 **本原多项式 (Primitive Polynomial)** 时，LFSR 才能达到最大周期 $2^n - 1$。

------

### 4. 为什么要在密码学里用它？

1. **硬件极快**：异或门和移位寄存器是数字电路里最快、最省电的元件。在 CPU 很弱的设备（如早期的手机 SIM 卡、门禁卡、DVD 机）里，LFSR 是唯一跑得动的加密算法。
2. **统计特性好**：一个最大周期的 LFSR 输出的比特流，0 和 1 的分布非常均匀，通过很多随机性测试（看起来很像真随机）。

------

### 5. 致命弱点：Berlekamp-Massey 算法

**警告：单个 LFSR 是绝对不安全的！**

虽然它看起来很随机，但它本质上是线性的。

- **攻击：** 假设你有 $n$ 位的 LFSR。攻击者只需要截获 **$2n$** 位的明文-密文对（从而算出 $2n$ 位的密钥流）。
- **破解：** 利用 **Berlekamp-Massey 算法**，攻击者可以在多项式时间内（极快）解出你的抽头位置和初始状态。

**比喻：** LFSR 就像斐波那契数列（1, 1, 2, 3, 5...），虽然数字在变大，但我只要知道前两个数，就能算出后面所有的数。

------

### 6. 现代用法：非线性组合

既然单个 LFSR 不安全，我们怎么用？

**答案：把好几个 LFSR 并联起来，用非线性的方式混合它们。**

- **CSS (DVD)：** 用加法器混合两个 LFSR。
- **A5/1 (GSM)：** 用三个 LFSR，并引入“钟控机制”（多数表决），让它们走走停停，破坏线性关系。
- **Grain / Trivium：** 现代轻量级流密码，依然使用 LFSR 作为核心组件，但在输出端加了极其复杂的非线性函数。

### 总结

LFSR 是密码学的**“发动机”**。 它提供源源不断的动力（长周期的伪随机流），但你必须给它装上**“变速箱”**（非线性混合函数），这辆车（加密算法）才能安全上路，否则分分钟被黑客拆解。
