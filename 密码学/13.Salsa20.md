这张 PPT 展示了 **Salsa20** 算法最核心的内部构造和工作流程。它非常详细地描绘了数据是如何从密钥变成密钥流的。

我们可以把这张图分为三个部分来解读：**函数定义（顶部）**、**状态矩阵结构（左侧）** 和 **核心运算流程（右侧）**。

### 1. 函数签名 (顶部数学公式)

PPT 最上面一行定义了 Salsa20 的输入和输出：

$$\text{Salsa20}: \{0,1\}^{128 \text{ or } 256} \times \{0,1\}^{64} \to \{0,1\}^n$$

- **输入：**
  - **Key ($k$):** 128位 或 256位（图中绿色圈出的部分）。这是你的核心机密。
  - **Nonce ($r$):** 64位（图中红色下划线）。每次加密必须不同。
- **输出：**
  - **Stream:** $n$ 位的二进制流（最大 $2^{73}$ 位，基本上是无限长）。

**公式解读：**

$$\text{Salsa20}(k; r) := H(k, (r, 0)) \ || \ H(k, (r, 1)) \ || \ \dots$$

- **$||$**: 代表**拼接 (Concatenation)**。Salsa20 产生的数据是一块一块拼起来的。
- **$(r, 0), (r, 1)$**: 这里展示了 **计数器模式 (Counter Mode)**。
  - 第 0 块输入是：密钥 + Nonce + **计数器0**。
  - 第 1 块输入是：密钥 + Nonce + **计数器1**。
  - 这证明了 Salsa20 可以**并行计算**（同时算第0块和第100块），也可以**随机访问**。

------

### 2. 初始状态矩阵 (左侧长条图)

图左侧那个标着 "64 bytes" 的长条形，展示了 Salsa20 的 **4x4 矩阵状态** 是如何填充的。这 64 个字节包含了生成密钥流所需的所有原材料：

- **$\tau_0, \tau_1, \tau_2, \tau_3$ (Constants):** 4个 4字节的常数。
  - 通常是字符串 "expand 32-byte k" 的 ASCII 码。它们像建筑的钢筋骨架，防止全零状态，并破坏对称性。
- **$K$ (Key):** 你的密钥被切分放在了不同的位置（混在常数中间）。
- **$r$ (Nonce):** 8 字节的随机数。
- **$i$ (Index/Counter):** 8 字节的计数器。这是区分不同块的唯一标识。

**结构特点：** 注意 $K$（密钥）被夹在常数 $\tau$ 中间。这种“三明治”结构是为了让密钥在后续的搅拌中能更快地扩散到整个状态里。

------

### 3. 核心运算流程 (中间到右侧)

这是最精彩的部分，展示了 **$H$ 函数** 到底在做什么。

1. **输入状态 (Input State):** 左侧那个填满了 Key, Nonce, Counter 的 64 字节矩阵。
2. **搅拌函数 $h$ (The Mixer):**
   - 箭头指向一个方框 $h$。
   - 这里的 $h$ 指的是多轮的 **ARX 运算**（Add, Rotate, XOR）。
   - **注意底部的注释：** *"h: invertible function" (h 是可逆函数)*。这意味着，如果你知道输出，在数学上你是可以推算出输入的。
   - 仅仅做完 $h$ 是**不安全**的，因为攻击者可以倒推回密钥。
3. **前馈加法 (The Feed-Forward):**
   - **看最右边的 $\boxplus$ 符号**。
   - 流程图把**原始的输入状态**拉了一根线过来，与**经过 $h$ 搅拌后的状态**进行了**相加 (Word-wise Addition)**。
   - $$Output = h(Input) + Input$$
4. **最终输出 (Output Block):**
   - 这个相加操作是**毁灭性**的（对于想倒推的人来说）。
   - 它把“加密前的样子”和“加密后的样子”混在了一起。这使得整个操作变得**不可逆 (One-way)**。
   - 生成的这 64 字节就是最终的密钥流块，用来异或明文。

------

### 4. 底部注释：SSE2

PPT 底部写着：`designed to be fast on x86 (SSE2)`

- **SSE2** 是 Intel CPU 的指令集，允许“单指令多数据”(SIMD)。
- 因为 Salsa20 的状态是 4x4 的矩阵，而且操作都是 32 位的，它正好可以完美利用 SSE2 指令，**一次性处理 4 个数字**。
- 这就是为什么 Salsa20 在软件上能跑得飞快，甚至比硬件加速的 AES 还要快的原因。

### 总结

这张图形象地解释了 **Salsa20 的核心哈希函数 $H$**：

$$H(Input) = \text{Mix}(Input) + Input$$

它利用**计数器模式**生成序列，利用**ARX结构**进行搅拌，最后利用**前馈加法**保证不可逆性，从而生成安全的密钥流。
