## 习题 1

> 以 Figure 2-1 为模型，展示对数组 A = ⟨13, 19, 9, 5, 12, 8, 7, 4, 11, 2, 6, 21⟩ 执行 PARTITION 的操作过程。

### 解答

pivot = 13（第一个元素），下标 1-12。

| 步骤 | 数组状态 | i | j | 操作 |
|------|---------|---|---|------|
| 初始 | **13** 19 9 5 12 8 7 4 11 2 6 21 | 1 | 13(r+1) | — |
| 扫描 | 13 **19** 9 5 12 8 7 4 11 2 6 **21** | 2 | 11 | i停在19(≥13)，j停在6(≤13) |
| 交换1 | 13 **6** 9 5 12 8 7 4 11 2 **19** 21 | 3 | 10 | swap(19,6) |
| 扫描 | 13 6 9 5 12 8 7 4 11 2 19 21 | — | — | i继续右移找≥13的... |
| | | 10 | 10 | i停在19(下标10,≥13)，j停在2(下标10,≤13) |

让我更仔细地逐步执行：

```
初始：[13, 19, 9, 5, 12, 8, 7, 4, 11, 2, 6, 21]
       pivot=13, i=0, j=12(r+1)

第1轮：
  i右移：++i=1, a[1]=19 ≥ 13 → 停
  j左移：--j=11, a[11]=21 > 13 → 继续
         --j=10, a[10]=6 ≤ 13 → 停
  i=1 < j=10 → swap(a[1], a[10])
  数组：[13, 6, 9, 5, 12, 8, 7, 4, 11, 2, 19, 21]

第2轮：
  i右移：++i=2, a[2]=9 < 13 → 继续
         ++i=3, a[3]=5 < 13 → 继续
         ++i=4, a[4]=12 < 13 → 继续
         ++i=5, a[5]=8 < 13 → 继续
         ++i=6, a[6]=7 < 13 → 继续
         ++i=7, a[7]=4 < 13 → 继续
         ++i=8, a[8]=11 < 13 → 继续
         ++i=9, a[9]=2 < 13 → 继续
         ++i=10, a[10]=19 ≥ 13 → 停
  j左移：--j=9, a[9]=2 ≤ 13 → 停
  i=10 > j=9 → break！

pivot归位：swap(a[0], a[9])
数组：[2, 6, 9, 5, 12, 8, 7, 4, 11, 13, 19, 21]
                                      ↑ pivot归位在下标9
```

验证：
- 左边 [2, 6, 9, 5, 12, 8, 7, 4, 11] — 全部 ≤ 13 ✓
- 右边 [19, 21] — 全部 > 13 ✓

## 习题 2

> 简要论证 PARTITION 在大小为 n 的子数组上的运行时间是 Θ(n)。

### 解答

Partition 中有两个指针 i 和 j：
- i 从左端向右扫描，j 从右端向左扫描
- 两个指针**合计**恰好覆盖整个子数组的所有 n 个元素（i 向右走过的 + j 向左走过的 = n）
- 每个元素最多被比较一次，最多被交换一次

因此：
- 比较次数：Θ(n)
- 交换次数：≤ n/2
- 总时间：**Θ(n)**

## 习题 3

> 如何修改 QUICKSORT 使其按非递增顺序排序？

### 解答

只需修改 Partition 中的比较方向：

```cpp
// 原版（非递减排序）：
while (a[++i] < x);    // i 找 ≥ pivot 的
while (a[--j] > x);    // j 找 ≤ pivot 的

// 修改为（非递增排序）：
while (a[++i] > x);    // i 找 ≤ pivot 的
while (a[--j] < x);    // j 找 ≥ pivot 的
```

修改后的效果：
- 划分后左边全部 ≥ pivot，右边全部 ≤ pivot
- 递归排序左右两半后，整个数组就是非递增序

QuickSort 主体代码不需要任何改动，只改 Partition 的两个 while 条件即可。

## 习题 4

> 证明快速排序可以做到最坏情况 O(n lg n)。

### 解答

**方法**：用线性时间选择算法（BFPRT）在每次 Partition 前找到数组的**中位数**，将其作为 pivot。

**步骤**：
1. 调用 BFPRT 的 Select 算法，在 O(n) 时间内找到当前子数组的中位数
2. 以中位数为 pivot 进行 Partition
3. 由于 pivot 是中位数，划分后左右两半的大小都恰好是 n/2

**递推关系**：

```
T(n) = 2T(n/2) + O(n)
```

- `2T(n/2)`：两个规模为 n/2 的子问题
- `O(n)`：Select 找中位数 O(n) + Partition O(n) = O(n)

**解**：T(n) = O(n log n)

> 这样即使在最坏情况下，QuickSort 也能保证 O(n log n)。代价是常数因子变大（因为每次都要做线性时间选择），实际中很少这样做。

## 习题 5：油管问题（Professor Olay）

> 一家石油公司计划铺设一条东西方向的主管道穿过有 n 口油井的油田。从每口井连接一条支线沿最短路径（南北方向）接到主管道。给定各油井的 (x, y) 坐标，如何选择主管道位置使支线总长度最小？证明最优位置可在线性时间内确定。

### 解答

**分析**：
- 主管道是一条水平线 y = c
- 每口井到主管道的距离 = |y_i - c|（纯南北方向）
- 目标：选择 c 使得 Σ|y_i - c| 最小

**结论**：最优的 c = 所有油井 y 坐标的**中位数**。

**证明中位数最优**：

设 y₁ ≤ y₂ ≤ … ≤ yₙ 是所有油井 y 坐标的排序。

- 如果 c < median，将 c 上移一点点 δ：上方的井距离各减 δ，下方的井距离各增 δ。因为 median 上方的井数 < 下方的井数，所以总距离减少。
- 同理 c > median 时下移更优。
- 因此 c = median 时总距离最小。

**线性时间确定**：

使用 RandomizedSelect 或 BFPRT 算法，在 O(n) 时间内找到 n 个 y 坐标的中位数。

```
算法步骤：
1. 提取所有油井的 y 坐标：y₁, y₂, …, yₙ     → O(n)
2. 用线性时间选择找中位数：Select(y, n/2)      → O(n)
3. 主管道位置 = 中位数                          → O(1)
总时间：O(n)
```
