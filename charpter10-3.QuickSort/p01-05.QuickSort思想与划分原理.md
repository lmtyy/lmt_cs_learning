# 快速排序 QuickSort — 思想与划分原理（第1-5页）

由霍尔(C.A.R.Hoare)提出的快速排序，是实践中最快的排序算法之一。核心在于"划分"(Partition)操作。

---

## 第1页：QuickSort 引入

### 与 MergeSort 的关键区别

| | MergeSort | QuickSort |
|---|-----------|-----------|
| 划分方式 | 简单等分（按位置） | 按值划分（按大小关系） |
| 合并步骤 | 需要 Merge 操作 | **不需要合并**！ |
| 子问题关系 | 子问题独立但需合并 | 划分后子问题天然有序 |

- 由著名计算机科学家**霍尔(C.A.R.Hoare)**给出
- 把原序列分成两个子问题，在被分成的两个子问题以后**不再需要归并**
- 被分成的两个子问题必须满足：一子问题中的所有元素都**小于或等于**另一子问题的任何一个元素

> 类比：MergeSort 是"先拆开再拼回去"，QuickSort 是"拆的时候就放对位置了，不用再拼"。

---

## 第2页：QuickSort 算法步骤

1. 选取一个 **pivot**（划分元素/基准元素）
2. **重排**列表，使得：
   - pivot 之前的所有元素 ≤ pivot
   - pivot 之后的所有元素 > pivot
3. 将 pivot 与 ≤ 子表的最后一个元素**交换** → pivot 到达最终位置
4. 对两个子表分别**递归排序**

> ✅ 关键洞察：pivot 经过一次划分后就**归位**了（在最终排序结果中的正确位置），不会再被移动。

---

## 第3页：基本策略

### 形式化描述

将数组 A[1:n] 分解成两个子数组 B[1:p] 和 B[p+1:n]，使得：
- B[1:p] 中的元素均**不大于** B[p+1:n] 中的元素

然后分别对这里两个数组中的元素进行排序（非降的），最后再把两个排好序的数组接起来即可。

### 划分后的结构

```
┌───────────────────────┬───────────────────────┐
│        A[i] ≤ p       │        A[i] > p       │
│   B[1:p] — 左子数组    │  B[p+1:n] — 右子数组  │
└───────────────────────┴───────────────────────┘
                         ↑
                      pivot 的位置
```

> 注意：因为划分保证了左边全 ≤ 右边，所以排完两边后**直接拼接就是有序的**，无需 Merge！

## 第4页：快速排序动画

PPT 通过柱状图动画展示了一次划分操作：

- 蓝色水平线 = pivot 的值
- 柱子高度 = 元素大小
- 红色箭头 = 正在进行交换的元素
- 红色柱子（右侧）= 当前选定的 pivot

**动画展示的效果**：
- 比 pivot 小的元素被移到左边（柱子在水平线以下）
- 比 pivot 大的元素被移到右边（柱子在水平线以上）
- 一次划分完成后，pivot 归位，左右两侧分别递归

## 第5页：划分操作详解

### 操作描述

选取 A 的某个元素 **t = A(s)** 作为划分元素，然后将其他元素重新排列，使 A[1:n] 中：
- 所有在 t **以前**出现的元素都**小于或等于** t
- 而在 t **之后**出现的元素都**大于或等于** t

### 划分前后对比

```
划分前：
┌──────┬──────┬─────┬────────┬──────┬──────────┬─────┬──────┐
│ A(1) │ A(2) │ … │ A(s-1) │ A(s) │ A(s+1)  │ …  │ A(n) │
└──────┴──────┴─────┴────────┴──────┴──────────┴─────┴──────┘
                                ↑
                          t = A(s)（划分元素）

经过一次划分后：
┌──────┬──────┬─────┬──────┬───┬──────┬─────┬──────┐
│ A(1) │ A(2) │ … │ A(j) │ t │ A(k) │ …  │ A(n) │
└──────┴──────┴─────┴──────┴───┴──────┴─────┴──────┘
  ←── 每个元素 ≤ t ──→   ↑   ←── 每个元素 ≥ t ──→
                      pivot归位
```

### 划分的结果

| 区域 | 范围 | 性质 |
|------|------|------|
| 左子数组 | A[1] … A[j] | 每个元素都 **≤ t** |
| pivot | t | 已在最终位置 |
| 右子数组 | A[k] … A[n] | 每个元素都 **≥ t** |

> ✅ 划分完成后，t 就在它排好序后应该在的位置了，永远不用再动。

---

## 总结

```
QuickSort 与 MergeSort 的本质区别：

MergeSort：分（简单）→ 治（递归）→ 合（复杂，O(n)）
QuickSort：分（复杂，O(n) 划分）→ 治（递归）→ 合（无需合并！）

QuickSort 分治结构：
  ├── Partition：选 pivot，将数组分成 ≤pivot 和 >pivot 两部分
  ├── 递归排序左子数组
  └── 递归排序右子数组
  （不需要 Merge 步骤）

核心：划分操作 Partition
  → pivot 一次归位
  → 左边全 ≤ pivot ≤ 右边全
  → 子问题天然有序关系
```
