太好了！既然明确了你的主战场是 **MySQL**，那我们就可以完全抛开 SQL Server 或 Oracle 的那些不同设定的干扰，专心把 MySQL 里的变量体系扒得清清楚楚。

在 MySQL 的世界里，变量有着极其严格的阶级和地盘划分。为了帮你更好地理解它们的作用域， 可以直观地展示这三种变量的“势力范围”。

总的来说，MySQL 中的变量按核心机制分为三大类，咱们逐一攻破：

------

### 1. 局部变量 (Local Variables) —— “只能活在屋子里的守规矩的人”

这是最严谨的一种变量。它有严格的数据类型，而且必须“先打报告（声明）再使用”。它的势力范围最小，连个大门都出不去。

- **长相特征：** 名字前面**没有任何符号**（比如 `v_age`、`total_count`）。
- **生存范围 (作用域)：** 仅限于包含它的 `BEGIN ... END` 代码块内。这就意味着，你只能在**存储过程 (Stored Procedure)**、**自定义函数 (Function)** 或 **触发器 (Trigger)** 里看到它的身影。一旦代码执行跳出了这个 `END`，它就直接死亡被销毁了。
- **如何定义和赋值：**
  - 必须使用 `DECLARE` 关键字，并且必须指定数据类型（比如 `INT`, `VARCHAR(50)`）。
  - 用 `SET` 赋值，或者用 `SELECT ... INTO` 把查询结果塞给它。

**实战代码示例 (在存储过程中计算总价)：**

SQL

```sql
DELIMITER //

CREATE PROCEDURE CalculateOrderTotal(IN p_order_id INT)
BEGIN
    -- 1. 声明局部变量：必须在 BEGIN 之后的第一时间声明，指定类型
    DECLARE v_quantity INT DEFAULT 0;
    DECLARE v_price DECIMAL(10,2) DEFAULT 0.00;
    DECLARE v_total DECIMAL(10,2); 

    -- 2. 赋值：从表中查出数据，塞进局部变量 (SELECT ... INTO)
    SELECT quantity, price INTO v_quantity, v_price 
    FROM order_items 
    WHERE order_id = p_order_id;
    
    -- 3. 计算并用 SET 赋值
    SET v_total = v_quantity * v_price;
    
    -- 4. 输出结果
    SELECT v_total AS 'OrderTotal';
    
END //
-- 一旦到了这里，v_quantity, v_price, v_total 就统统消失了，外界无法访问。
DELIMITER ;
```

------

### 2. 用户自定义变量 (User-Defined Variables) —— “整个会话里到处溜达的自由人”

这种变量主打一个随性，不用提前声明，也没有固定的数据类型约束，你随时随地都能掏出来用。

- **长相特征：** 名字前面带有**单个 `@` 符号**（比如 `@my_user`、`@counter`）。
- **生存范围 (作用域)：** 整个**当前数据库连接 (Session)**。只要你打开了数据库客户端连上 MySQL，在你不关掉这个窗口/连接之前，你在哪里定义了它，后面紧接着执行的其他 SQL 语句都能认出它。但是，如果你把客户端关了重新连，或者你的同事在另一台电脑上连数据库，你们的 `@` 变量是互不相通的。
- **如何定义和赋值：**
  - 不需要 `DECLARE`。
  - 直接用 `SET @变量名 = 值` 赋值。
  - **MySQL 特色用法：** 可以在普通的 `SELECT` 查询语句中用 `:=` 给它赋值（注意这里一定要用冒号加等号，如果在 SELECT 里只用 `=` 会被当成判断相等的条件）。

**实战代码示例 (跨语句传递参数 & 模拟行号)：**

SQL

```sql
-- 场景 1：在这个连接里临时存个值，方便下面多条语句复用
SET @target_date = '2023-10-01';
SELECT * FROM orders WHERE create_time >= @target_date;
SELECT COUNT(*) FROM logs WHERE log_time >= @target_date;

-- 场景 2：MySQL 经典老手操作，在查询中直接用变量生成自增行号
SET @row_num = 0;
SELECT 
    (@row_num := @row_num + 1) AS row_number, 
    username, 
    email 
FROM users 
LIMIT 5;
```

### 3. 系统变量 (System Variables) —— “掌管数据库命运的神明”

这类变量不是用来存你的业务数据的，而是 MySQL 数据库**自带**的，用来控制数据库的运行状态和各种规则。

- **长相特征：** 名字前面带有**双 `@@` 符号**（比如 `@@max_connections`）。
- **生存范围 (作用域)：** 分为**全局 (GLOBAL)** 和 **会话 (SESSION)** 两个级别。
  - **全局：** 影响整个 MySQL 服务器，改了之后所有连上来的用户都受影响（通常需要管理员权限）。
  - **会话：** 只影响你当前的这一次连接操作。
- **如何使用：**
  - 查看变量：`SELECT @@global.变量名;` 或 `SELECT @@session.变量名;` (如果省略级别，默认查 session 级别)。
  - 修改变量：`SET GLOBAL 变量名 = 值;` 或 `SET SESSION 变量名 = 值;`。

**实战代码示例：**

SQL

```sql
-- 查看当前数据库的最大允许连接数（全局配置）
SELECT @@global.max_connections;

-- 我只想让我自己这个连接的查询严格匹配大小写（修改当前会话配置）
SET SESSION sql_mode = 'STRICT_TRANS_TABLES';
```

### 💡 核心梳理总结表 (MySQL 专属版)

| **维度**              | **局部变量 (Local Variable)**              | **用户自定义变量 (User-Defined Variable)**                   | **系统变量 (System Variable)**                     |
| --------------------- | ------------------------------------------ | ------------------------------------------------------------ | -------------------------------------------------- |
| **外观标志**          | 无前缀 (如 `v_count`)                      | 单 `@` 前缀 (如 `@counter`)                                  | 双 `@@` 前缀 (如 `@@version`)                      |
| **生老病死 (作用域)** | 仅存活于 `BEGIN...END`代码块               | 存活于整个当前数据库连接 (Session)                           | 伴随服务器运行 (Global) 或 当前连接 (Session)      |
| **是否需要显式声明**  | **必须**使用 `DECLARE`                     | **不需要**，直接拿来就赋                                     | MySQL 自带，无需也无法声明新建                     |
| **数据类型约束**      | **强类型** (声明时必须定死是INT/VARCHAR等) | **弱类型** (跟着你赋的值变)                                  | 数据库底层设定，不可随意更改类型                   |
| **常见应用场景**      | 编写复杂的存储过程、触发器内部的逻辑流转。 | 简单的 SQL 脚本传参、临时存储查询结果以便下一条语句使用、编写行号等游标替代方案。 | 调优数据库性能、修改服务器配置（如字符集、时区）。 |
