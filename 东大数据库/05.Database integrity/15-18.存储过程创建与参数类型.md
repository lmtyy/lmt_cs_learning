## 第一页PPT：存储过程和函数概述

### 1. 定义
- **存储过程和函数** 都是事先经过编译并存储在数据库中的一段SQL语句的集合。
- 它们可以对一段代码进行封装，以便日后重复调用，提高代码的复用性和管理效率。

### 2. 创建与调用
- **存储过程**：使用 `CREATE PROCEDURE` 语句创建，通过 `CALL` 语句加存储过程名来调用。
- **函数**：使用 `CREATE FUNCTION` 语句创建，通过函数名直接调用（通常作为表达式的一部分）。

### 3. 存储过程的优势
- **降低网络开销**：存储过程直接运行在数据库服务器上，只需传递调用命令和参数，减少了客户端与服务器之间的频繁交互。
- **安全性高**：可以给用户仅授予执行存储过程的权限，而不直接暴露底层表，增强了数据访问的安全性。
- **性能高**：存储过程在创建时被编译和优化，之后每次调用都直接执行编译后的代码，减少了重复编译的开销。

### 4. 存储过程的劣势
- **开发调试难**：存储过程的编写和调试通常不如高级语言方便，缺乏成熟的IDE支持。
- **可移植性差**：不同数据库管理系统的存储过程语法存在差异，迁移时可能需要重写。
- **难以支持分布式数据库**：在分布式环境中，存储过程的逻辑可能难以拆分和扩展，影响系统伸缩性。

## 第二页PPT：创建和调用存储过程的语法

### 1. 创建存储过程的语法格式
```sql
CREATE PROCEDURE <存储过程名>([参数, ..., 参数])
BEGIN
    <SQL语句块>
END <终止符>
```
- **参数列表**：括号内可定义零个或多个参数，即使没有参数也必须保留括号。
- **BEGIN...END**：包含该存储过程要执行的SQL语句块，可以是一条或多条语句。
- **终止符**：默认情况下，SQL语句以分号（`;`）结束。但在存储过程体中可能会包含多个分号，为避免混淆，需要临时修改终止符（通常用 `DELIMITER` 命令）。

### 2. 参数的定义格式
```
[IN | OUT | INOUT] 参数名 参数数据类型
```
- **IN**：输入参数，调用时传入值，过程内部可以读取但不能修改该参数在调用方的原始值。
- **OUT**：输出参数，过程内部可以赋值，并将值返回给调用者。
- **INOUT**：既可作为输入也可作为输出，调用时传入值，过程内部可以修改并返回。
- **参数数据类型**：可以是任何有效的SQL数据类型，如 `INT`、`FLOAT`、`VARCHAR(n)`、`DATE` 等。

### 3. 调用存储过程的语法
```sql
CALL <存储过程名>(参数列表);
```
- 使用 `CALL` 关键字后跟存储过程名和括号，括号内传入实参（如果有）。

## 第三页PPT：示例一——输出某门课程的学生成绩

### 1. 创建存储过程
```sql
DELIMITER //
CREATE PROCEDURE PROC_COURSE_GRADE (IN course_id CHAR(4))
BEGIN
    SELECT * FROM SC WHERE Cno = course_id;
END //
DELIMITER ;
```
- **DELIMITER 的作用**：因为存储过程体中包含分号（`;`），而MySQL默认以分号作为语句结束符。如果直接执行创建语句，MySQL遇到分号就会认为语句结束，导致创建过程不完整。因此先用 `DELIMITER //` 将语句结束符临时改为 `//`，这样整个存储过程体被视为一条完整的语句，直到遇到 `END //` 才结束。创建完成后，再用 `DELIMITER ;` 恢复默认结束符。
- **参数**：`IN course_id CHAR(4)` 表示输入一个课程编号，类型为定长字符。
- **BEGIN...END** 块中是一条查询语句：从 `SC` 表中选取指定课程的所有记录。

### 2. 调用存储过程
```sql
CALL PROC_COURSE_GRADE('1');
```
- 调用时传入课程编号 `'1'`，返回该课程的所有学生成绩记录。
- **示例输出**：
  | Sno        | Cno  | Grade |
  | ---------- | ---- | ----- |
  | 2021310722 | 1    | 87    |
  | 2021310723 | 1    | 92    |

## 第四页PPT：示例二——计算某门课程的平均成绩并存入变量

### 1. 创建存储过程
```sql
DELIMITER //
CREATE PROCEDURE PROC_COURSE_AVG_GRADE (IN course_id CHAR(4), OUT avg_grade FLOAT)
BEGIN
    SELECT AVG(grade) INTO avg_grade FROM SC WHERE Cno = course_id;
END //
DELIMITER ;
```
- **参数**：`IN course_id CHAR(4)` 为输入参数，`OUT avg_grade FLOAT` 为输出参数，用于返回平均分。
- **SELECT ... INTO 语句**：将查询结果（平均分）直接赋值给输出变量 `avg_grade`。这是存储过程中将值传递给输出参数的常用方式。

### 2. 调用存储过程并获取输出值
```sql
CALL PROC_COURSE_AVG_GRADE('1', @avg_grade);
```
- **@avg_grade**：这是一个**用户定义变量**（以 `@` 开头），用于接收存储过程的输出参数值。
- 调用后，平均分被存储到变量 `@avg_grade` 中。

### 3. 查看变量值
```sql
SELECT @avg_grade;
```
- 结果为 `89.5`，即1号课程的平均成绩。

## 补充：SQL中的变量知识点

在MySQL（以及其他一些关系型数据库）中，变量主要分为以下几类：

### 1. 用户变量
- **定义**：以 `@` 开头，由用户定义，作用域为当前会话（连接）。会话结束或连接关闭时变量自动释放。
- **赋值**：
  - 使用 `SET` 语句：`SET @var_name = value;` 或 `SET @var_name := value;`
  - 使用 `SELECT ... INTO` 语句：`SELECT column INTO @var_name FROM table WHERE condition;`
  - 在 `SELECT` 语句中直接赋值：`SELECT @var_name := value;` （注意必须用 `:=`，以免与比较运算符混淆）
- **示例**：
  ```sql
  SET @name = '张三';
  SELECT @age := 25;
  SELECT COUNT(*) INTO @total FROM students;
  ```
- **使用**：可以在后续的SQL语句中直接引用，如 `WHERE name = @name`，或在 `CALL` 语句中作为参数传递。

### 2. 局部变量
- **定义**：在存储过程、函数或触发器的 `BEGIN...END` 块中声明，使用 `DECLARE` 语句。作用域仅限于该块内部。
- **语法**：`DECLARE var_name data_type [DEFAULT default_value];`
- **赋值**：
  - 使用 `SET` 语句：`SET var_name = value;`
  - 使用 `SELECT ... INTO` 语句：`SELECT column INTO var_name FROM ...;`
- **示例**（在存储过程中）：
  ```sql
  CREATE PROCEDURE demo()
  BEGIN
      DECLARE student_name VARCHAR(50);
      SET student_name = '李四';
      SELECT * FROM students WHERE name = student_name;
  END//
  ```

### 3. 系统变量
- **全局变量**：影响整个数据库服务器的操作，如 `max_connections`。设置时需加 `GLOBAL` 关键字：`SET GLOBAL var_name = value;`
- **会话变量**：只影响当前会话，如 `autocommit`。设置时可用 `SESSION` 或省略：`SET SESSION var_name = value;` 或 `SET var_name = value;`
- **查看**：`SHOW VARIABLES;` 或 `SELECT @@var_name;`（`@@` 表示系统变量）

### 4. 参数变量
- 在存储过程或函数中定义的参数（如 `IN`, `OUT`, `INOUT`）本质上也是变量，其作用域在过程体内。输入参数在调用时被初始化为传入值，输出参数用于返回结果。
