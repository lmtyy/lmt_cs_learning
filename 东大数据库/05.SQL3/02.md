## 一、数据操作语言（DML）

### 1. 什么是DML？
**数据操作语言（Data Manipulation Language）** 是SQL的一个子集，用于对数据库中的数据进行**增、删、改、查**操作。它允许用户或应用程序与数据库中的数据进行交互，是日常数据库操作中最常用的部分。

### 2. DML的主要操作
DML通常包括四种基本操作，常被简称为 **CRUD**：

| 操作 | 关键字   | 说明                                             |
| ---- | -------- | ------------------------------------------------ |
| 插入 | `INSERT` | 向表中添加新的数据行                             |
| 查询 | `SELECT` | 从表中检索数据（有时也被视为DQL，但通常归入DML） |
| 更新 | `UPDATE` | 修改表中已有的数据                               |
| 删除 | `DELETE` | 从表中移除数据行                                 |

#### 2.1 插入数据（INSERT）
- **基本语法**：
  ```sql
  INSERT INTO 表名 (列1, 列2, ...) VALUES (值1, 值2, ...);
  ```
- **示例**：
  ```sql
  INSERT INTO student (sno, sname, sage) VALUES ('2024001', '张三', 20);
  ```
- 可以一次插入多行，也可以从另一个查询结果中插入（`INSERT INTO ... SELECT ...`）。

#### 2.2 查询数据（SELECT）
- 是DML中最复杂、最常用的操作，用于从数据库中检索数据。
- 基本结构：
  ```sql
  SELECT 列1, 列2, ...
  FROM 表名
  WHERE 条件
  GROUP BY 分组列
  HAVING 分组条件
  ORDER BY 排序列;
  ```
- 支持连接、子查询、聚合、排序、分页等。

#### 2.3 更新数据（UPDATE）
- **基本语法**：
  ```sql
  UPDATE 表名 SET 列1 = 新值1, 列2 = 新值2 WHERE 条件;
  ```
- **示例**：
  ```sql
  UPDATE student SET sage = 21 WHERE sno = '2024001';
  ```
- 如果不加`WHERE`条件，会更新表中所有行，需谨慎。

#### 2.4 删除数据（DELETE）
- **基本语法**：
  ```sql
  DELETE FROM 表名 WHERE 条件;
  ```
- **示例**：
  ```sql
  DELETE FROM student WHERE sno = '2024001';
  ```
- 同样，省略`WHERE`会删除所有行（但表结构保留）。要彻底删除表，使用`DROP TABLE`。

### 3. 事务控制与DML
DML操作通常受到事务控制的影响。在支持事务的数据库（如InnoDB）中，DML语句可以被提交（`COMMIT`）或回滚（`ROLLBACK`），确保数据的一致性和完整性。

## 二、视图（View）

### 1. 什么是视图？
**视图**是一个**虚拟表**，其内容由查询定义。它不实际存储数据，而是保存了一条SQL查询语句，当用户查询视图时，数据库会动态执行该查询并返回结果。视图可以看作是从一个或多个基表（或其他视图）中导出的“窗口”。

### 2. 视图的作用
- **简化查询**：将复杂的多表连接或子查询封装成视图，用户可以直接查询视图，如同查询普通表。
- **数据安全性**：可以隐藏基表中的敏感列（如密码、工资），只向用户暴露需要的列。
- **逻辑数据独立性**：当基表结构发生变化时，可以通过修改视图定义来保持对外接口不变，减少对应用程序的影响。
- **数据一致性**：视图总是反映基表的最新数据，因为每次查询都会重新计算。

### 3. 创建视图
**语法**：
```sql
CREATE VIEW 视图名 [(列别名列表)] AS
子查询
[WITH CHECK OPTION];
```
- **示例**：创建一个包含学生姓名和所选课程名称的视图。
  ```sql
  CREATE VIEW student_course AS
  SELECT s.sname, c.cname
  FROM student s
  JOIN sc ON s.sno = sc.sno
  JOIN course c ON sc.cno = c.cno;
  ```
- `WITH CHECK OPTION` 用于可更新视图，确保对视图的修改符合视图定义的条件。

### 4. 查询视图
视图的使用方式与普通表相同：
```sql
SELECT * FROM student_course WHERE cname = '数据库';
```

### 5. 更新视图
- **可更新视图**：在一定条件下，可以对视图执行`INSERT`、`UPDATE`、`DELETE`操作，这些操作会传递到基表。
- **可更新视图的条件**（通常需要）：
  - 视图基于单个基表（或可更新的连接）。
  - 视图的列直接对应基表的列（没有聚合、表达式、`DISTINCT`等）。
  - 视图中包含了基表的主键或所有非空列（对于插入而言）。
- 如果视图包含`JOIN`、聚合函数、`GROUP BY`、`DISTINCT`等，通常不可更新。

### 6. 删除视图
```sql
DROP VIEW 视图名;
```

### 7. 物化视图（Materialized View）
- 某些数据库（如Oracle、PostgreSQL）支持**物化视图**，它实际存储数据，可以定期刷新，用于提高查询性能。
- 物化视图不是标准SQL的视图，而是一种特殊的数据库对象。

### 8. 视图与基表的关系
- 视图不占用物理存储空间（除物化视图外）。
- 基表数据变化时，视图查询结果会自动更新。
- 视图可以基于其他视图创建，形成视图链。

---

## 三、DML与视图的关系
- 对视图的查询属于DML的`SELECT`操作。
- 对视图的更新（如果允许）也属于DML的`INSERT`、`UPDATE`、`DELETE`操作。

---

## 四、总结

| 概念     | 作用                             | 关键点                                               |
| -------- | -------------------------------- | ---------------------------------------------------- |
| **DML**  | 操作数据库中的数据               | 包括INSERT、SELECT、UPDATE、DELETE；受事务控制       |
| **视图** | 提供虚拟表，简化查询、增强安全性 | 基于查询定义，不存数据，可更新（有限制），可多层嵌套 |

掌握DML和视图是数据库应用开发的基础，合理使用视图能提升系统的可维护性和安全性。
