这是一个非常经典的**银行数据库**案例，融合了 **INSERT ... SELECT**、多表查询和业务逻辑。我来帮你把题目和课件中给出的 SQL 拆解清楚。

---

## 一、业务需求理解

> 给 **Perryridge 分行** 的所有**贷款客户**每人赠送一个 **200 美元的储蓄账户**，新储蓄账户的账号就用该客户的**贷款编号**。

### 涉及的表及其作用

| 表名 | 字段 | 作用 |
|------|------|------|
| `account` | account_number, branch_name, balance | 储蓄账户信息 |
| `loan` | loan_number, branch_name, amount | 贷款账户信息 |
| `depositor` | customer_name, account_number | 用户与其储蓄账户的对应关系 |
| `borrower` | customer_name, loan_number | 用户与其贷款账户的对应关系 |

### 需要完成的操作

1. 在 `account` 表中为每个符合条件的客户**新增储蓄账户记录**
2. 在 `depositor` 表中**建立客户与这个新储蓄账户的关联**

---

## 二、SQL 语句逐步解析

### 第一步：插入账户信息到 `account` 表

```sql
INSERT INTO account (account_number, branch_name, balance)
SELECT loan_number, branch_name, 200
FROM loan
WHERE branch_name = 'Perryridge';
```

**解释：**
- 从 `loan` 表中选出 **Perryridge 分行** 的所有贷款记录
- 对于每一行：
  - `loan_number` → 作为新储蓄账户的账号 (`account_number`)
  - `branch_name` → 沿用原分行名 (`branch_name`)
  - `200` → 赠送给客户的金额 (`balance`)
- 一次性为所有符合条件的客户创建储蓄账户

**示例：**  
假设 `loan` 表中有一条记录：`('L-001', 'Perryridge', 5000)`  
则会在 `account` 表中插入：`('L-001', 'Perryridge', 200)`

---

### 第二步：插入客户–账户关系到 `depositor` 表

```sql
INSERT INTO depositor (customer_name, account_number)
SELECT borrower.customer_name, loan.loan_number
FROM loan, borrower
WHERE loan.branch_name = 'Perryridge'
  AND loan.loan_number = borrower.loan_number;
```

**注意：课件原 SQL 最后两行有笔误，我帮你修正了**

**解释：**
- 同时从 `loan` 和 `borrower` 两张表中获取数据
- 连接条件：`loan.loan_number = borrower.loan_number`（贷款编号一致）
- 筛选条件：`loan.branch_name = 'Perryridge'`
- 为每个符合条件的客户：
  - 取 `borrower.customer_name` 作为客户名
  - 取 `loan.loan_number` 作为新储蓄账户账号

**示例：**  
假设有：
- `loan`: `('L-001', 'Perryridge', 5000)`
- `borrower`: `('张三', 'L-001')`

则会在 `depositor` 中插入：`('张三', 'L-001')`

## 四、关键知识点总结

| 知识点 | 在本例中的体现 |
|--------|----------------|
| `INSERT ... SELECT` | 一次性插入多条记录 |
| 子查询 / 表连接 | 第二步中 JOIN 了 loan 和 borrower |
| 外键一致性 | `depositor.account_number` 必须能在 `account` 中找到 |
| 业务原子性 | 两个 INSERT 应放在一个事务中执行，避免数据不一致 |

## 五、可优化的写法（了解即可）

在支持多表插入的数据库中（如 PostgreSQL），也可以通过一条 CTE 完成：

```sql
WITH new_accounts AS (
    INSERT INTO account (account_number, branch_name, balance)
    SELECT loan_number, branch_name, 200
    FROM loan
    WHERE branch_name = 'Perryridge'
    RETURNING account_number
)
INSERT INTO depositor (customer_name, account_number)
SELECT borrower.customer_name, loan.loan_number
FROM loan, borrower
WHERE loan.branch_name = 'Perryridge'
  AND loan.loan_number = borrower.loan_number;
```
