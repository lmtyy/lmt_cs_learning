这节课讲的是 **UPDATE（更新/修改数据）**，包括语法、三种修改类型，以及带子查询的更新。我帮你系统梳理一遍。

---

## 一、UPDATE 基本语法

```sql
UPDATE 表名
SET 列1 = 表达式1, 列2 = 表达式2, ...
WHERE 条件;
```

| 子句 | 作用 |
|------|------|
| `SET` | 指定**要修改的列**以及**修改后的值**（可以是具体值、表达式、函数等） |
| `WHERE` | 指定**要修改哪些行**（缺省则修改表中**所有行**） |

---

## 二、三种修改方式

### 1. 修改单个元组

```sql
-- 将学号为 20100201 的学生学院改为理学院
UPDATE student 
SET sdept = '理学院' 
WHERE sid = '20100201';
```

### 2. 修改多个元组（有条件）

```sql
-- 将所有软件学院学生的年龄增加1岁
UPDATE student 
SET sage = sage + 1 
WHERE sdept = '软件学院';
```

### 3. 修改所有元组（无 WHERE）

```sql
-- 将所有学生的年龄增加1岁
UPDATE student 
SET sage = sage + 1;
```
> ⚠️ 没有 `WHERE` 子句时，整张表都会被更新，**务必小心**。

## 三、带子查询的更新语句

### 业务场景
> 将所有**软件学院**学生的**选课成绩**设为 0

### 涉及的表
- `student(sno, sname, sdept, sex, sbirthday)` — 学生信息
- `sc(sno, cno, grade)` — 选课成绩

### SQL 语句

```sql
UPDATE sc
SET grade = 0
WHERE sno IN (
    SELECT sno
    FROM student
    WHERE student.sdept = '软件学院'
);
```

### 执行逻辑
1. 子查询找出所有 `sdept = '软件学院'` 的学生学号
2. 外层 UPDATE 将 `sc` 表中这些学生的 `grade` 设为 0

> 💡 **注意**：子查询中不需要写 `sc.sno = student.sno`（课件中写多了），因为子查询只涉及 `student` 表，和外层 `sc` 没有直接关联。

## 四、SET 子句中的表达式

| 表达式类型 | 示例 |
|-----------|------|
| 常量 | `SET sdept = '理学院'` |
| 原值加减 | `SET sage = sage + 1` |
| 函数 | `SET birthday = DATEADD(year, 1, birthday)` |
| 多列 | `SET balance = balance - 100, branch = 'Downtown'` |
| NULL | `SET grade = NULL` |

## 五、完整例子汇总

| 场景 | SQL |
|------|-----|
| 修改单行 | `UPDATE student SET sdept='理学院' WHERE sid='20100201';` |
| 修改多行（有条件） | `UPDATE student SET sage=sage+1 WHERE sdept='软件学院';` |
| 修改所有行 | `UPDATE student SET sage=sage+1;` |
| 带子查询 | `UPDATE sc SET grade=0 WHERE sno IN (SELECT sno FROM student WHERE sdept='软件学院');` |

## 六、与 DELETE 的对比

| 操作 | DELETE | UPDATE |
|------|--------|--------|
| 作用 | 删除**整行** | 修改**指定列的值** |
| 是否保留行 | 行被移除 | 行保留，列值改变 |
| WHERE 缺省 | 删除所有行 | 更新所有行 |
| 子查询 | 支持 | 支持 |

---

## 七、注意事项

| 注意点 | 说明 |
|--------|------|
| **忘了写 WHERE** | 会更新整张表，后果严重。执行前先 `SELECT` 确认条件 |
| **子查询引用被更新的表** | 某些数据库（如 MySQL）有限制，需要改写或使用 JOIN |
| **外键约束** | 更新主键或外键列可能违反约束 |
| **CHECK 约束** | 新值必须满足 CHECK 条件（如 `balance >= 0`） |
| **NULL 处理** | 表达式中的 NULL 会导致结果为 NULL |

## 八、速查表

```sql
-- 单行更新
UPDATE table SET col = value WHERE pk = id;

-- 条件批量更新
UPDATE table SET col = col + 1 WHERE condition;

-- 全表更新
UPDATE table SET col = default_value;

-- 子查询条件
UPDATE A SET A.col = new_value 
WHERE A.fk IN (SELECT B.pk FROM B WHERE B.condition);
```
