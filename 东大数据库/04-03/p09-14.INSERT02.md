这部分课件讲的是通过 **SELECT 查询结果** 来插入数据，这是非常实用的功能，尤其适合做**数据汇总、备份、或生成中间表**。课件介绍了两种方式，我帮你分别讲解。

---

## 一、INSERT ... SELECT —— 将查询结果插入**已存在**的表

### 语法格式

```sql
INSERT INTO 目标表 (列1, 列2, ...)
SELECT 列1, 列2, ...
FROM 数据源
[WHERE 条件];
```

- 目标表必须**已经存在**
- SELECT 返回的列数、数据类型必须与 INTO 指定的列匹配

---

### 课件完整例子：计算每个系的平均年龄并存入新表

#### 第一步：创建目标表 `Dept_age`

```sql
CREATE TABLE Dept_age (
    Dept   CHAR(15),   -- 系名
    Avg_age SMALLINT   -- 学生平均年龄
);
```

#### 第二步：用 INSERT ... SELECT 填入数据

```sql
INSERT INTO Dept_age (Sdept, Avg_age)
SELECT Sdept, AVG(Sage)
FROM Student
GROUP BY Sdept;
```

**执行过程：**
1. 内层 SELECT 按系分组，计算每个系的平均年龄
2. 外层 INSERT 将每组结果作为一行插入 `Dept_age` 表

> ✅ 优点：一条语句完成数据统计和存储，避免手动逐条插入

---

## 二、SELECT ... INTO —— 查询的同时**创建新表**并插入数据

### 语法格式

```sql
SELECT 列1, 列2, ...
INTO 新表名
FROM 数据源
[WHERE 条件];
```

- 目标表**不需要提前创建**，执行时会**自动创建**
- 新表的列名、数据类型由 SELECT 决定

---

### 课件例子

```sql
SELECT Sdept, AVG(Sage) AS AVG_age
INTO DEPT_AVG
FROM Student
GROUP BY Sdept;
```

**执行结果：**
- 自动创建一张新表 `DEPT_AVG`
- 表中有两列：`Sdept`（系名）和 `AVG_age`（平均年龄）
- 直接填入按系分组后的统计结果

---

## 三、两种方式的对比（重要！）

| 特性 | INSERT ... SELECT | SELECT ... INTO |
|------|------------------|-----------------|
| 目标表是否需提前存在 | ✅ 必须存在 | ❌ 自动创建 |
| 是否需要 CREATE TABLE | 需要 | 不需要 |
| 适用场景 | 向已有表追加数据 | 临时创建汇总表、备份表 |
| 数据库兼容性 | 所有 DBMS 通用 | SQL Server / Access 支持；<br>MySQL 不支持（需用 `CREATE TABLE ... AS SELECT`） |

> ⚠️ **注意**：在 MySQL 中，`SELECT ... INTO` 有不同含义（用于变量赋值）。如果想实现“创建表并插入查询结果”，应该使用：
> ```sql
> CREATE TABLE 新表名 AS
> SELECT ... FROM ...;
> ```

---

## 四、实际应用场景举例

| 场景 | 推荐方式 |
|------|----------|
| 定期将历史数据归档到汇总表 | `INSERT ... SELECT` |
| 临时生成一份报表数据 | `SELECT ... INTO` 或 `CREATE TABLE ... AS` |
| 将多个表的数据合并到一个表中 | `INSERT ... SELECT` + UNION |
| 数据备份（创建副本） | `SELECT * INTO 备份表 FROM 原表` |

---

## 五、总结

- **INSERT ... SELECT**：表先建好，再往里追加查询结果（多次执行会累积数据）
- **SELECT ... INTO**：边查边建表，结果直接变成一张新表（通常一次性使用）
