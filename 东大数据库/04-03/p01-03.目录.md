## 一、DML（Data Manipulation Language，数据操作语言）

DML 用于对数据库中的数据进行**增、删、改**操作，也就是课件中提到的 **Insertion（插入）、Deletion（删除）、Update（更新）**。

### 1. 插入数据（Insert）

向表中添加新记录。

**基本语法：**
```sql
INSERT INTO 表名 (列1, 列2, ...)
VALUES (值1, 值2, ...);
```

**示例：**
```sql
INSERT INTO Students (ID, Name, Age)
VALUES (101, '张三', 20);
```

### 2. 删除数据（Delete）

从表中删除符合条件的记录（⚠️ 如果不加条件，会删除**整张表**的数据）。

**基本语法：**
```sql
DELETE FROM 表名 WHERE 条件;
```

**示例：**
```sql
DELETE FROM Students WHERE ID = 101;
```

### 3. 更新数据（Update）

修改表中已有记录的某些列的值。

**基本语法：**
```sql
UPDATE 表名
SET 列1 = 新值1, 列2 = 新值2
WHERE 条件;
```

**示例：**
```sql
UPDATE Students
SET Age = 21
WHERE Name = '张三';
```

> 🧠 **记忆技巧**：  
> Insert → 增加数据  
> Delete → 删除数据  
> Update → 修改数据  
> 合称 **增删改**，加上查询（Select）就是完整的 **CRUD** 操作。

## 二、View（视图）

视图是一个**虚拟表**，它不存储数据本身，而是存储一条 **SQL 查询语句**。你可以像使用普通表一样对视图进行查询，甚至在某些条件下对视图进行增删改（有限制）。

### 主要用途：
- **简化复杂查询**：把多表关联查询存成视图，下次直接 `SELECT * FROM 视图`。
- **提高安全性**：只暴露用户需要的列，隐藏敏感数据。
- **逻辑数据独立性**：如果底层表结构变化，可以修改视图定义而不影响应用程序。

### 创建视图的语法：
```sql
CREATE VIEW 视图名 AS
SELECT 列1, 列2, ...
FROM 表名
WHERE 条件;
```

### 示例：
```sql
CREATE VIEW YoungStudents AS
SELECT Name, Age
FROM Students
WHERE Age < 22;
```

### 使用视图：
```sql
SELECT * FROM YoungStudents;
```

### 删除视图：
```sql
DROP VIEW YoungStudents;
```

> ⚠️ **注意**：  
> - 视图通常只能用于查询（只读视图）。  
> - 如果视图基于单个表且不包含分组、聚合等，有时可以进行更新操作（插入、更新、删除），但会直接作用于原表。

## 总结对比

| 操作/对象 | 类型 | 是否存储数据 | 主要作用 |
|----------|------|-------------|----------|
| INSERT | DML | 修改原有表中的数据 | 添加新行 |
| DELETE | DML | 修改原有表中的数据 | 删除行 |
| UPDATE | DML | 修改原有表中的数据 | 修改行内容 |
| View | 数据库对象 | ❌ 不存储 | 封装查询、简化访问、安全控制 |
