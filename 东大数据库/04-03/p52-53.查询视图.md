这节课讲的是**查询视图**以及RDBMS如何通过**视图消解法**将视图查询转化为对基表的查询。我来帮你系统梳理。

---

## 一、核心观点

> **从用户角度：查询视图与查询基本表完全相同**

- 用户不需要知道某个数据是来自基表还是视图
- 用 `SELECT` 查询视图的语法和查询普通表一模一样

---

## 二、视图消解法（View Resolution）

### 什么是视图消解法？

当用户查询一个视图时，RDBMS **不是**直接从视图（不存数据）中取数据，而是：

1. 从数据字典中取出视图的定义（保存的 SELECT 语句）
2. 将用户的查询语句与视图定义**合并**
3. 转化为对**基表**的查询
4. 执行转化后的查询，返回结果

### 示意图

```
用户查询视图                    RDBMS内部处理
      │                              │
      ▼                              ▼
SELECT * FROM sf_stu    →   视图消解法   →   SELECT ... FROM Student
WHERE Sage < 20                       WHERE Sdept='软件学院' AND Sage<20
      │                              │
      └──────────────────────────────┘
                    │
                    ▼
               返回查询结果
```

---

## 三、具体例子

### 视图定义（例1）

```sql
CREATE VIEW sf_stu AS
SELECT Sid, Sname, Sage
FROM Student
WHERE Sdept = '软件学院';
```

### 用户查询

```sql
SELECT Sno, Sage
FROM sf_stu
WHERE Sage < 20;
```

### 视图消解法转化后的查询

```sql
SELECT Sno, Sage
FROM Student
WHERE Sdept = '软件学院' 
  AND Sage < 20;
```

### 转化过程拆解

| 步骤 | 操作 |
|------|------|
| 1 | 取视图定义中的 SELECT：`SELECT Sid, Sname, Sage FROM Student WHERE Sdept='软件学院'` |
| 2 | 将用户查询中的 `Sage<20` 与视图定义的 `WHERE` 条件**合并**（用 AND） |
| 3 | 将用户查询的 SELECT 列（`Sno, Sage`）替换视图定义中的 SELECT 列 |
| 4 | 生成最终对基表 Student 的查询 |

---

## 四、视图消解法的优点

| 优点 | 说明 |
|------|------|
| **逻辑数据独立性** | 基表结构变化时，只需修改视图定义，用户查询不变 |
| **安全性** | 通过视图隐藏敏感数据，用户只能看到授权的内容 |
| **简化查询** | 用户不需要写复杂的多表连接或条件筛选 |
| **动态性** | 每次查询都访问最新基表数据 |

---

## 五、注意事项

| 注意点 | 说明 |
|--------|------|
| 视图不存数据 | 每次查询视图都会**实时执行**消解后的查询 |
| 性能考虑 | 复杂视图（多层嵌套、多表连接）可能影响查询性能 |
| 视图消解的局限性 | 某些情况下（如分组视图、包含 DISTINCT 等）视图消解可能失败 |

---

## 六、视图消解法 vs 物化视图

| 对比项 | 普通视图 | 物化视图（如 Oracle） |
|--------|---------|---------------------|
| 数据存储 | 不存储 | 实际存储查询结果 |
| 查询方式 | 每次动态执行 | 直接读取存储的数据 |
| 数据新鲜度 | 总是最新 | 需要刷新 |
| 性能 | 可能有开销 | 查询快 |
| 存储空间 | 几乎不占 | 占用空间 |

---

## 七、知识点速查

```sql
-- 创建视图
CREATE VIEW sf_stu AS
SELECT Sid, Sname, Sage
FROM Student
WHERE Sdept = '软件学院';

-- 查询视图（用户写法）
SELECT Sno, Sage
FROM sf_stu
WHERE Sage < 20;

-- 实际执行的查询（RDBMS转化后）
SELECT Sno, Sage
FROM Student
WHERE Sdept = '软件学院' AND Sage < 20;
```

## 八、思考题（检验理解）

假设有以下视图定义和查询：

**视图定义：**
```sql
CREATE VIEW high_balance AS
SELECT account_number, branch_name, balance
FROM account
WHERE balance > 10000;
```

**用户查询：**
```sql
SELECT branch_name, COUNT(*)
FROM high_balance
GROUP BY branch_name;
```

**问题：视图消解后的查询是什么？**

<details>
<summary>点击查看答案</summary>

```sql
SELECT branch_name, COUNT(*)
FROM account
WHERE balance > 10000
GROUP BY branch_name;
```
</details>
