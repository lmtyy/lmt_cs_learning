## 一、什么是“行列子集视图”？（首先明确这个概念）

课件先给出了**唯一允许更新**的视图类型定义：

> **行列子集视图**
> - 从**单个基本表**导出
> - 只是去掉了基本表的**某些行**（通过 WHERE）和**某些列**（通过 SELECT）
> - **保留了主码**

### 例子

```sql
CREATE VIEW sf_stu AS
SELECT Sno, Sname, Sage   -- 去掉了 Sdept 等列
FROM Student
WHERE Sdept = '软件学院';  -- 只保留软件学院的行
```

✅ 这种视图**可以更新**（UPDATE / INSERT / DELETE）

## 二、DB2 对视图更新的 7 条限制（超重要，经常考）

下面这 7 种情况的视图，**不允许更新**（或部分更新不允许）：

---

### 1️⃣ 由两个以上基本表导出

```sql
-- ❌ 多表连接视图，不可更新
CREATE VIEW stu_course AS
SELECT Student.Sno, Sname, Cno, Grade
FROM Student, SC
WHERE Student.Sno = SC.Sno;
```

**原因**：更新时不知道要改哪个基表

---

### 2️⃣ 字段来自表达式或常数

```sql
-- ❌ 包含表达式/常数，不允许 INSERT/UPDATE
CREATE VIEW stu_age AS
SELECT Sno, Sname, 2026-Sage AS birth_year
FROM Student;
```

**原因**：无法反向计算基表的值

✅ 但**允许 DELETE**

---

### 3️⃣ 字段来自集函数（聚合函数）

```sql
-- ❌ 包含 AVG、SUM 等，不可更新
CREATE VIEW avg_grade AS
SELECT Sno, AVG(Grade) AS avg_g
FROM SC
GROUP BY Sno;
```

**原因**：无法将 AVG 反向拆解成具体行的值

---

### 4️⃣ 含有 GROUP BY 子句

```sql
-- ❌ 分组视图，不可更新
CREATE VIEW dept_count AS
SELECT Sdept, COUNT(*) AS cnt
FROM Student
GROUP BY Sdept;
```

**原因**：一对多映射，无法唯一确定要改哪一行

---

### 5️⃣ 含有 DISTINCT 短语

```sql
-- ❌ 去重视图，不可更新
CREATE VIEW unique_dept AS
SELECT DISTINCT Sdept
FROM Student;
```

**原因**：DISTINCT 丢失了重复行的信息，无法反向

---

### 6️⃣ 嵌套查询中涉及导出该视图的基本表（自引用）

```sql
-- ❌ 子查询中引用了 SC，而 SC 正是导出该视图的表
CREATE VIEW GOOD_SC AS
SELECT Sno, Cno, Grade
FROM SC
WHERE Grade > (
    SELECT AVG(Grade)
    FROM SC   -- 内层也用了 SC
);
```

**原因**：更新时会造成逻辑循环，无法确定更新语义

---

### 7️⃣ 在一个不允许更新的视图上定义的视图

```sql
-- V1 不可更新（例如分组视图）
CREATE VIEW V1 AS
SELECT Sdept, COUNT(*) FROM Student GROUP BY Sdept;

-- V2 基于 V1，也不可更新
CREATE VIEW V2 AS
SELECT * FROM V1;
```

**原因**：依赖不可更新源 → 自身也不可更新

## 三、7 条限制速记表

| # | 限制条件 | 关键字 | 可否 DELETE | 可否 INSERT/UPDATE |
|---|----------|--------|-------------|-------------------|
| 1 | 多表连接 | 2+ 表 | ❌ | ❌ |
| 2 | 表达式/常数 | 计算列 | ✅ | ❌ |
| 3 | 集函数 | AVG/SUM | ❌ | ❌ |
| 4 | GROUP BY | 分组 | ❌ | ❌ |
| 5 | DISTINCT | 去重 | ❌ | ❌ |
| 6 | 嵌套自引用 | 子查询中引用自身基表 | ❌ | ❌ |
| 7 | 基于不可更新视图 | 依赖链 | ❌ | ❌ |

## 四、行列子集视图（唯一安全可更新的）

```sql
-- ✅ 行列子集视图：可更新
CREATE VIEW sf_stu AS
SELECT Sno, Sname, Sage   -- 列子集
FROM Student
WHERE Sdept = '软件学院';  -- 行子集
-- Student 的主码 Sno 被保留了
```

**更新时自动补全或检查条件：**

```sql
-- 插入时自动补 Sdept='软件学院'
INSERT INTO sf_stu VALUES('001', '张三', 20);

-- 实际执行：
INSERT INTO Student(Sno,Sname,Sage,Sdept) 
VALUES('001','张三',20,'软件学院');
```

---

## 五、理解这些限制的核心思想

> **一条核心原则：**  
> 对视图的更新，必须能**唯一地、有意义地**转换成对基表的更新。

违反这条原则的，系统就拒绝更新。

## 六、考试常见题型

### 判断题示例

**题目**：判断以下视图是否可以更新，为什么？

```sql
CREATE VIEW V AS
SELECT Sno, Grade*1.1 AS new_grade
FROM SC;
```

**答案**：❌ 不可更新。因为 `new_grade` 来自表达式 `Grade*1.1`，无法反向确定 `Grade` 的原值。

---

### 选择题示例

**题目**：以下哪个视图允许执行 `INSERT` 操作？

A. `CREATE VIEW V AS SELECT Sdept, COUNT(*) FROM Student GROUP BY Sdept;`  
B. `CREATE VIEW V AS SELECT DISTINCT Sdept FROM Student;`  
C. `CREATE VIEW V AS SELECT Sno, Sname FROM Student WHERE Sdept='CS';`  
D. `CREATE VIEW V AS SELECT Sno, AVG(Grade) FROM SC GROUP BY Sno;`

**答案**：**C**（行列子集视图，可更新）
