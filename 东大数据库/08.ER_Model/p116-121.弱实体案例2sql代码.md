根据最后一页 PPT（完美的二元联系设计图），我们可以清晰地看出有三个表需要创建。请特别注意代码中的**注释**，我会把图中的**线（联系）**和**符号（完全参与、分辨符）**是如何变成代码的一一对应起来。

### 1. 创建强实体表：雇员 (Employees)

这是整个关系网的核心，它不依赖任何人，所以最先创建。

SQL

```mysql
CREATE TABLE Employees (
    ssn VARCHAR(20) PRIMARY KEY,  -- ssn 下面是实下划线，代表强实体的主键
    name VARCHAR(50),
    lot VARCHAR(50)
);
```

### 2. 创建保单表：(Policies)

在图中，`Policies` 是个单边框矩形（强实体），有自己的主键 `policyid`。

但是，注意看它和 `Employees` 之间的菱形 `Purchaser`：

- **1:N**：1个雇员买多份保单。规则是**“把 1 的一端的主键，作为外键放到 N 的一端”**。所以我们要把 `ssn` 放到这里。
- **双线（完全参与）**：`Policies` 到 `Purchaser` 是双线，说明“每份保单必须由雇员购买”。在代码里，这就变成了 `NOT NULL` 约束。

SQL

```mysql
CREATE TABLE Policies (
    policyid VARCHAR(20) PRIMARY KEY, -- 实下划线，自身的主键
    cost DECIMAL(10, 2),
    
    -- 【联系落地】：实现 1:N 的 Purchaser 联系
    purchaser_ssn VARCHAR(20) NOT NULL, -- NOT NULL 体现了图中的“双线”完全参与约束
    
    -- 设置外键
    FOREIGN KEY (purchaser_ssn) REFERENCES Employees(ssn)
);
```

### 3. 创建弱实体表：家属 (Dependents)

这是最复杂、也是最能体现弱实体精髓的一张表。

在图中：

- `Dependents` 是双边框（弱实体），必须依附于 `Employees`，分辨符是 `pname`（虚下划线）。
- 并且，它还要和 `Policies` 通过 `Beneficiary` 建立 1:N 的联系（1份保单对应多个家属），且家属端是双线（完全参与，每个家属记录必须对应一份保单）。

SQL

```mysql
CREATE TABLE Dependents (
    -- 1. 引入标识实体 (Employees) 的主键，作为确认家属归属的外键
    employee_ssn VARCHAR(20),
    
    -- 2. 弱实体自身的分辨符 (图中的虚下划线 pname)
    pname VARCHAR(50),
    
    -- 3. 普通属性
    age INT,
    
    -- 4. 【联系落地】：实现 1:N 的 Beneficiary 联系。
    -- 因为是双线(完全参与)，所以必须加上 NOT NULL
    policy_id VARCHAR(20) NOT NULL,

    -- 【核心1：弱实体的主键生成】
    -- 复合主键 = 标识实体的主键 + 自己的分辨符
    PRIMARY KEY (employee_ssn, pname),

    -- 【核心2：弱实体的存在依赖】
    -- 加上 ON DELETE CASCADE (级联删除)。如果雇员离职(记录被删)，他的家属记录也会被数据库自动清理
    FOREIGN KEY (employee_ssn) REFERENCES Employees(ssn) ON DELETE CASCADE,
    
    -- Beneficiary 联系的外键约束
    FOREIGN KEY (policy_id) REFERENCES Policies(policyid)
);
```

### 💡 核心总结：为什么这个设计比“三元联系”好？

回想一下 PPT 前面那个糟糕的“大菱形 (Covers)”。如果按那个画法，你需要建一张极其臃肿的中间表，把 `ssn`、`policyid`、`pname` 全塞进去，还会产生大量的数据冗余（比如同一个雇员的信息被反复记录）和更新异常。

而我们现在的代码：

1. **没有任何多余的中间表**（联系都被优雅地转化为了外键）。
2. **约束极其严密**：通过 `NOT NULL` 和外键，数据库在底层就帮你挡住了“凭空出现的保单”或“没有主人的家属”这种脏数据。
