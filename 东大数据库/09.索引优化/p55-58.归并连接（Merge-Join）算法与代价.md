### 第1页：Merge-Join 的基本思想与重复值问题

**知识点1：归并连接的两阶段流程**
1. **排序阶段**：如果两张表还没有按连接属性排序，首先分别对它们在连接属性上排序。
2. **归并阶段**：同时扫描两张已排序的表，像合并有序列表一样，找出匹配的元组对。

- “Join step is similar to the merge stage of the sort-merge algorithm.”  
  这里的归并阶段的确与经典二路归并的外排序合并过程十分相似，都使用双指针同步遍历。

**知识点2：重复值的特殊处理**
- 归并连接与标准合并算法的最大区别在于**连接属性上的重复值处理**。
- 当连接属性存在重复值时，必须保证**所有值相同的元组对上都要产生连接结果**。例如，外表在连接键 `d` 上有两个元组 `(d,8)` 和 `(d,13)`，内表也有一个 `(d, N)`，则最终必须生成两对匹配。
- 如果只做简单的单指针顺序比较，很容易漏掉某些组合。通常实现中会引入**标记指针回溯**或者**临时缓冲**来处理重复键。

**知识点3：示例表 R 与 S**
- R（a1, a2）：
  ```
  pr  ps
  a   3
  b   1
  d   8
  d   13
  f   7
  m   5
  q   6
  ```
- S（a, A）：
  ```
  a   A
  b   G
  c   L
  d   N
  m   B
  s   (空?)
  ```
- 两表连接属性都是第一列（a1 和 a）。  
  观察到 R 中 `d` 出现了两次，S 中 `d` 出现一次，归并连接必须生成两行结果，正是上面强调的重复值问题。

> **易错点**：很多学生误以为归并连接就像合并两个有序数组那样简单，忽略连接键重复时需要用嵌套循环或回溯指针来输出所有匹配组合。这是重要考点。

### 第2页：Merge-Join 的伪代码

**知识点4：伪代码流程**
```
sort R, S on join keys
cursorR ← R_sorted, cursorS ← S_sorted
while cursorR and cursorS:
    if cursorR > cursorS:
        increment cursorS
    if cursorR < cursorS:
        increment cursorR
    elif cursorR and cursorS match:
        emit
    increment cursorS
```
- 核心逻辑是根据当前游标所指键值的大小关系移动游标：
  - 如果 `cursorR` 的键值 > `cursorS` 的键值，说明 `cursorS` 落后了，推进 `cursorS`。
  - 如果 `cursorR` 的键值 < `cursorS` 的键值，推进 `cursorR`。
  - 如果两者相等，则输出匹配，然后推进 `cursorS`。
- **注意**：这里给出的伪代码是简化版，不能正确处理重复值（如果 R 侧有重复，移动 S 游标后会错过与同一个 R 的其他匹配）。实际上，在匹配分支还需要检查 R 的下一个元组是否键值相同，如果是则需要将 S 游标回退或缓存处理。高层次的 PPT 可能忽略了这个细节，但考试常考“如何处理重复值”。

> **易错点/考点**：伪代码中的 `increment cursorS` 在匹配后立即发生，会导致重复匹配漏掉。正确的做法是使用类似“标记备份”的手段，在学生自己编写算法时容易犯错。

### 第3页：适用条件与代价公式

**知识点5：适用条件**
- “Can be used only for equi-joins and natural joins”
- 归并连接只能用于**等值连接**和**自然连接**，因为它的比较是基于等值条件的排序扫描。对于非等值连接（如 `<`、`>`）无法直接使用这种双指针同步方式。

**知识点6：归并阶段的 I/O 特性**
- “Each block needs to be read only once (assuming all tuples for any given value of the join attributes fit in memory)”
- 在归并阶段，如果内存足够容纳某个连接键的所有重复元组（两表中），那么两张表的每一页都只需读一次。
- 归并阶段的 I/O 代价就是 **b_r + b_s**（外表页数 + 内表页数）。

**知识点7：总代价公式**
\[
\text{Total Cost} = \text{Sorting Cost} + (b_r + b_s)
\]
- 如果表已经按连接属性排好序（例如聚簇索引），则排序成本为 0，总代价就是 `b_r + b_s`，非常低。
- 如果要排序，则加上排序成本。

**知识点8：外排序的代价公式**
PPT 中给出的形式：
- Sort Cost (R)：\( 2M \cdot (1 + \lceil \log_{B-1} (M/B) \rceil ) \)
- Sort Cost (S)：\( 2N \cdot (1 + \lceil \log_{B-1} (N/B) \rceil ) \)

解释：
- 这里 `M` 和 `N` 是表的页数，`B` 是可用的内存缓冲区页数。
- 外排序通常采用**多路归并**：首先生成初始归并段（Pass 0），然后执行多路归并，归并的趟数（passes）为 `⌈log_{B-1} (M/B)⌉`。
- 每一趟（包括初始趟）都会读和写所有数据，因此每趟代价为 `2M`（读 M 页，写 M 页）。  
- `(1 + 趟数)` 表示初始趟（Pass 0）加上后续归并趟数。
- 公式中 `M/B` 计算初始段的数目，因为每个段在 Pass 0 时受限于内存大小 B。

> **易错点**：
> - 混淆 B 和 B-1：多路归并时，B 个缓冲区中，一个用于输出，其余 B-1 个用于输入，因此归并路数为 **B-1**，对数底数应为 B-1，而不是 B。
> - 认为一张表只要 `2M * (趟数)`，漏掉初始趟的“1”。

**知识点9：总代价**
\[
\text{Total} = 2M(1+\lceil \log_{B-1}(M/B)\rceil) + 2N(1+\lceil \log_{B-1}(N/B)\rceil) + (M+N)
\]

### 第4页：具体代价计算示例

**知识点10：示例参数**
- 表 R：M = 1000 页，m = 100,000 元组
- 表 S：N = 500 页，n = 40,000 元组
- 缓冲区 B = 100 页
- 题目指出两张表都可以在**两次趟完成排序**（即 Pass 0 和 一次归并）。

**知识点11：排序代价计算**
- 计算 `log_{B-1}(M/B)`：
  - B-1 = 99
  - M/B = 1000/100 = 10
  - log_{99}(10)：10 < 99，所以值为 0（或 ceil 为 1？但公式 `log_{B-1}(M/B)` 实际是计算归并趟数，通常归并趟数为 0 如果只一趟。然而示例说“sorted in two passes”，意味着总共两次趟，即一趟初始 + 一趟归并，所以 `⌈log_{99}(1000/100)⌉ = ⌈log_{99}(10)⌉ = 1`（因为99^0=1<10, 99^1=99≥10，所以需要1次归并）。  
- 因此，R 的排序代价 = `2 * 1000 * (1 + 1) = 4000 IOs`。
- S：M? N=500, N/B=500/100=5，log_{99}(5) ceil=1，同样两趟，排序代价 = `2 * 500 * (1+1) = 2000 IOs`。

**知识点12：归并代价与总代价**
- 归并代价 = `M + N = 1000 + 500 = 1500 IOs`。
- 总代价 = `4000 + 2000 + 1500 = 7500 IOs`。
- 按每次 I/O 0.1 ms 计算，总时间 ≈ **0.75 秒**。

**知识点13：与之前算法的对比**
- 这个 0.75 秒远低于块嵌套循环的 50 秒（501,000 IOs）和索引嵌套循环的 25,100 IOs（2.51 秒，未过滤情况）。
- 说明对于两张都不小的表，归并连接在内存适当的情况下非常高效。但它要求排序，如果表已经有序，归并连接几乎是 I/O 最优的。

> **易考点**：
> - 给定缓冲区大小 B，计算排序趟数，以及总代价。
> - 为什么公式中是 `B-1`？因为有一页用于输出。
> - 哪种情况下 Merge-Join 格外有优势？——表已经按连接键排序，或两个表都很大且不能通过索引有效访问。
> - 与 Hash Join 的比较（通常 Hash Join 在等值连接中更优，但 Merge Join 可用于排序输出，且不依赖哈希函数均匀分布）。

### 总体总结
这四页 PPT 完整展示了 Merge-Join 的原理、伪代码、代价模型与实例：
1. **思想**：排序 + 归并，双指针同步扫描，特别注意重复键。
2. **适用**：仅等值连接/自然连接。
3. **代价**：`(M+N)` 归并加上外部排序开销，排序开销按多路归并公式计算。
4. **性能**：示例中总 I/O 仅 7500，时间 0.75 秒，比之前嵌套循环类算法快两个数量级。
