好的，这几页讲的是 **Set Comparison（集合比较）**，重点是 `SOME` 操作符（在某些数据库中也叫 `ANY`）。

---

## 第1页：问题引入

### 需求
找出所有**资产大于 Brooklyn 某个分行**的分行名称。

### 表结构
**Branch**：`(branch-name, branch-city, assets)`

### 理解需求
- "大于 Brooklyn 的**某个**分行"：只要比 Brooklyn 的**任意一个**分行资产多就行
- 不是要求比**所有** Brooklyn 分行都多，而是比**至少一个**多

### 举例说明
假设 Brooklyn 有分行：资产 100 万、200 万
- 资产 150 万的分行 ✅（大于 100 万，满足"大于某个"）
- 资产 250 万的分行 ✅（大于 100 万和 200 万，当然也满足）
- 资产 90 万的分行 ❌（不大于任何一个）

---

## 第2页：用 `> SOME` 子查询实现

### SQL
```sql
SELECT branch_name
FROM branch
WHERE assets > SOME
    (SELECT assets
     FROM branch
     WHERE branch_city = 'Brooklyn');
```

### 执行过程
1. **子查询**：找出所有 Brooklyn 分行的资产值，比如 (100, 200)
2. **外层查询**：找出资产 > 100 或 > 200 的分行（即至少大于其中一个）

---

## 第3页：`SOME` 的数学定义

### 语法
```
F <comp> SOME r
```

### 含义
> 存在 r 中的某个值 t，使得 F 和 t 满足比较关系 `<comp>`

换句话说：**只要集合中有一个值满足条件，就返回 True**

### 例子

| 表达式 | 集合 r | 结果 | 原因 |
|--------|-------|------|------|
| `5 < SOME (0, 5)` | (0, 5) | ❌ false | 5 < 0? 否；5 < 5? 否（5 不小于 5） |
| `5 ≠ SOME (0, 5)` | (0, 5) | ✅ true | 5 ≠ 0 成立 |
| `5 < SOME (0, 5)` | (0, 5) | ❌ false | 前面已解释 |
| `5 = SOME (0, 5)` | (0, 5) | ✅ true | 5 = 5 成立 |

### 总结规律
| 条件 | 等价于 |
|------|--------|
| `= SOME` | `IN` |
| `≠ SOME` | 不是简单的 NOT IN，要小心 |
| `> SOME` | 大于集合中的最小值 |
| `< SOME` | 小于集合中的最大值 |

## 第4页：不使用子查询的等价写法（自连接）

### SQL
```sql
SELECT DISTINCT T.branch_name
FROM branch AS T, branch AS S
WHERE T.assets > S.assets
  AND S.branch_city = 'Brooklyn';
```

### 为什么需要 `DISTINCT`？

假设 Brooklyn 有两个分行，资产分别是 100 和 200：

| T.branch_name | T.assets | S.assets | 是否匹配 |
|---------------|----------|----------|----------|
| 某分行 | 150 | 100 | ✅ 匹配一次 |
| 某分行 | 150 | 200 | ❌ 不匹配 |
| 某分行 | 250 | 100 | ✅ 匹配一次 |
| 某分行 | 250 | 200 | ✅ 匹配第二次 |

如果一个分行比 Brooklyn 的两个分行资产都大，它会在结果中出现**多次**，所以需要 `DISTINCT` 去重。

## `SOME` vs `ALL` 对比（扩展知识）

| 操作符 | 含义 | 等价于 |
|--------|------|--------|
| `> SOME` | 大于集合中**至少一个** | 大于最小值 |
| `> ALL` | 大于集合中**所有** | 大于最大值 |
| `< SOME` | 小于集合中**至少一个** | 小于最大值 |
| `< ALL` | 小于集合中**所有** | 小于最小值 |
| `= SOME` | 等于集合中**至少一个** | `IN` |
| `= ALL` | 等于集合中**所有** | 通常无意义（除非集合只有一个值）|

### 举例
假设子查询返回 (100, 200)：
- `150 > SOME` → True（150 > 100）
- `150 > ALL` → False（150 > 200? 否）
- `50 < SOME` → True（50 < 200）
- `50 < ALL` → True（50 < 100 且 50 < 200）

---

## 本节小结

| 知识点 | 要点 |
|--------|------|
| `SOME` / `ANY` | 与子查询结果中的**至少一个**值比较 |
| 等价关系 | `= SOME` = `IN` |
| 自连接写法 | 可以实现相同功能，但需要 `DISTINCT` |
| 典型场景 | "大于某个"、"小于某个"、"等于某个" |
