## 第1页：GROUP BY 的基本用法

### 核心概念
- `GROUP BY` 将表中的数据按照某一列或多列的值**分成多个组**
- 然后**每个组**分别应用聚集函数（如 COUNT、SUM、AVG 等）

### 例子：统计每个分行的账户数量

**Account 表**：

| branch_name | account_number | balance |
|-------------|----------------|---------|
| Perryridge | a-102 | 400 |
| Brighton | a-217 | 750 |
| Perryridge | a-201 | 900 |
| Brighton | a-215 | 750 |
| Redwood | a-222 | 700 |

### SQL
```sql
SELECT branch_name, COUNT(account_number)
FROM account
GROUP BY branch_name;
```

### 执行过程

**步骤1：按 branch_name 分组**

| 组 | branch_name | 包含的 account_number |
|---|-------------|----------------------|
| 第1组 | Perryridge | a-102, a-201 |
| 第2组 | Brighton | a-217, a-215 |
| 第3组 | Redwood | a-222 |

**步骤2：对每组分别执行 COUNT**

| branch_name | COUNT(account_number) |
|-------------|----------------------|
| Perryridge | 2 |
| Brighton | 2 |
| Redwood | 1 |

### 结果

| branch_name | COUNT(account_number) |
|-------------|----------------------|
| Perryridge | 2 |
| Brighton | 2 |
| Redwood | 1 |

## 第2页：GROUP BY 的作用说明

### 关键点

| 情况 | 聚集函数的作用范围 |
|------|-------------------|
| 没有 GROUP BY | 作用于**整个查询结果**（返回一行） |
| 有 GROUP BY | 作用于**每个组**（每组返回一行） |

### 对比示例

```sql
-- 没有 GROUP BY：整个表只有一个平均值
SELECT AVG(balance) FROM account;  -- 返回1行

-- 有 GROUP BY：每个分行有自己的平均值
SELECT branch_name, AVG(balance)
FROM account
GROUP BY branch_name;  -- 返回3行（每个分行一行）
```

## 第5-6页：GROUP BY 的重要规则

### 核心原则（非常重要！）

> **SELECT 子句中，没有包含在聚集函数中的列，必须出现在 GROUP BY 子句中。**

### 为什么？

因为当使用 GROUP BY 后，每个组会返回**一行**结果。如果某个列没有被分组，也没有用聚集函数，数据库不知道该取这个组中的哪一行的值。

### ✅ 正确写法

```sql
-- 情况1：被分组的列
SELECT branch_name, COUNT(account_number)
FROM account
GROUP BY branch_name;  -- branch_name 在 GROUP BY 中 ✅

-- 情况2：把所有非聚集列都放入 GROUP BY
SELECT branch_name, balance, COUNT(account_number)
FROM account
GROUP BY branch_name, balance;  -- 两个非聚集列都在 GROUP BY 中 ✅

-- 情况3：非聚集列用聚集函数包装
SELECT branch_name, SUM(balance), COUNT(account_number)
FROM account
GROUP BY branch_name;  -- balance 被 SUM() 包装，不是普通列 ✅
```

### ❌ 错误写法

```sql
-- 错误：balance 既不在 GROUP BY 中，也不是聚集函数
SELECT branch_name, balance, COUNT(account_number)
FROM account
GROUP BY branch_name;  -- ❌ 会报错！
```

### 课件中的正确示例

```sql
-- 正确：branch_name 和 balance 都在 GROUP BY 中
SELECT branch_name, balance
FROM account
GROUP BY branch_name, balance;

-- 或者：balance 用聚集函数包装
SELECT branch_name, SUM(balance), COUNT(account_number)
FROM account
GROUP BY branch_name;
```

## 执行顺序总结

```
FROM → WHERE → GROUP BY → 聚集函数 → SELECT → ORDER BY
```

1. **FROM**：确定数据来源
2. **WHERE**：筛选行（分组前筛选）
3. **GROUP BY**：分成多个组
4. **聚集函数**：对每组进行计算
5. **SELECT**：选择要显示的列
6. **ORDER BY**：排序

---

## 本节小结

| 知识点 | 要点 |
|--------|------|
| GROUP BY 作用 | 按列值分组，每组返回一行 |
| 聚集函数 + GROUP BY | 对每组分别计算 |
| 无 GROUP BY | 聚集函数作用于整个表 |
| 重要规则 | SELECT 中的非聚集列必须在 GROUP BY 中 |
