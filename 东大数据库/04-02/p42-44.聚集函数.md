好的，这几页讲的是 **Aggregate Functions（聚集函数）**，用于对一列数据进行统计计算，返回一个单一的值。

---

## 第1页：五种基本聚集函数

| 函数 | 作用 | 适用类型 |
|------|------|----------|
| `AVG` | 平均值 | 数值列 |
| `MIN` | 最小值 | 任意可比较类型 |
| `MAX` | 最大值 | 任意可比较类型 |
| `SUM` | 总和 | 数值列 |
| `COUNT` | 计数 | 任意类型 |

### 特点
- 对**一列**数据进行计算
- 返回**一个值**（单行单列的结果）
- 自动忽略 NULL 值（除了 `COUNT(*)`）

---

## 第2页：AVG 的例子

### 场景
**Account 表**：`(account-number, balance, branch-name)`

### 需求
找出 **Perryridge 分行** 账户的平均余额

### SQL
```sql
SELECT AVG(balance)
FROM account
WHERE branch_name = 'Perryridge';
```

### 执行过程图解

```
account 表：
┌─────────────────────────────────────┐
│ account-number | balance | branch_name │
├─────────────────────────────────────┤
│ A-101          │ 500     │ Perryridge  │
│ A-102          │ 1000    │ Perryridge  │
│ A-103          │ 1500    │ Perryridge  │
│ A-104          │ 2000    │ Downtown    │
└─────────────────────────────────────┘

步骤1：WHERE 筛选 → 只保留 Perryridge 的行
┌──────────────┬─────────┐
│ account-number │ balance │
├──────────────┼─────────┤
│ A-101          │ 500     │
│ A-102          │ 1000    │
│ A-103          │ 1500    │
└──────────────┴─────────┘

步骤2：AVG(balance) → (500 + 1000 + 1500) / 3 = 1000

结果：120,000（课件图中写的 120,000 可能是笔误，应该是 1000）
```

## 第3页：COUNT 的两种用法

### 例子1：COUNT(*) - 统计行数

```sql
SELECT COUNT(*)
FROM customer;
```

- `COUNT(*)` 统计表中的**总行数**（包括 NULL 值）
- 不关心具体哪一列，就是数有多少条记录

### 例子2：COUNT(列名) - 统计非空值个数

```sql
SELECT COUNT(DISTINCT customer_name)
FROM depositor;
```

- `COUNT(customer_name)` 只统计该列非 NULL 的行数
- 加上 `DISTINCT` 可以**去重**，统计有多少个不同的客户名

### 对比

| 写法 | 含义 |
|------|------|
| `COUNT(*)` | 所有行数（包括重复、NULL） |
| `COUNT(列名)` | 该列非 NULL 的行数 |
| `COUNT(DISTINCT 列名)` | 该列不同的非 NULL 值的个数 |

### 实际例子

假设 depositor 表：

| customer_name | account_number |
|---------------|----------------|
| 张三 | A-101 |
| 张三 | A-102 |
| 李四 | A-103 |
| NULL | A-104 |

结果：
- `COUNT(*)` = 4
- `COUNT(customer_name)` = 3（NULL 被忽略）
- `COUNT(DISTINCT customer_name)` = 2（张三、李四）

## 重要规则总结

### 1. 聚集函数自动忽略 NULL
```sql
-- 假设 balance 列有 NULL
SELECT AVG(balance) FROM account;  -- 只计算非 NULL 值的平均
SELECT SUM(balance) FROM account;  -- 只对非 NULL 值求和
SELECT COUNT(balance) FROM account; -- 只统计非 NULL 的行数
```

### 2. 除非用 DISTINCT，否则不自动去重
```sql
SELECT COUNT(customer_name) FROM depositor;     -- 包括重复
SELECT COUNT(DISTINCT customer_name) FROM depositor; -- 去重
```

### 3. 聚集函数不能直接与普通列混用（除非使用 GROUP BY，后面会讲）

```sql
-- ❌ 错误：customer_name 是普通列，AVG 是聚集函数
SELECT customer_name, AVG(balance) FROM account;

-- ✅ 正确：只返回聚集函数的值
SELECT AVG(balance) FROM account;
```

## 快速参考卡

| 需求 | SQL |
|------|-----|
| 平均值 | `SELECT AVG(列名) FROM 表名` |
| 最大值 | `SELECT MAX(列名) FROM 表名` |
| 最小值 | `SELECT MIN(列名) FROM 表名` |
| 总和 | `SELECT SUM(列名) FROM 表名` |
| 总行数 | `SELECT COUNT(*) FROM 表名` |
| 去重计数 | `SELECT COUNT(DISTINCT 列名) FROM 表名` |
