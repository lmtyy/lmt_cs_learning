## 第1页：FROM 子句的基本作用

### 核心概念
- `FROM` 子句列出查询涉及到的**表**
- 对应关系代数中的**笛卡尔积（Cartesian Product）** 运算

### 例子：计算 borrower 和 loan 的笛卡尔积

```sql
SELECT *
FROM borrower, loan;
```

### 这是什么意思？
- 把 borrower 表的每一行，和 loan 表的每一行**两两组合**
- 如果 borrower 有 10 行，loan 有 10 行，结果就是 100 行

### 重要提示
> 这种写法**很少单独使用**，因为没有筛选条件会产生大量无意义的组合。通常配合 `WHERE` 子句来指定连接条件。

## 第2页：一个完整的多表查询例子

### 表结构
- **Loan**：`(loan-number, amount, branch-name)`
- **Borrower**：`(customer-name, loan-number)`

### 需求
找出所有在 **Perryridge** 分行有贷款的客户的：姓名、贷款号、贷款金额

### SQL
```sql
SELECT customer_name, borrower.loan_number, amount
FROM borrower, loan
WHERE borrower.loan_number = loan.loan_number 
  AND branch_name = 'Perryridge';
```

### 关键点讲解

1. **为什么写 `borrower.loan_number` 而不是直接 `loan_number`？**
   - 因为两个表都有 `loan_number` 这个列名，需要加上**表名前缀**来区分
   - 如果列名唯一，可以省略前缀

2. **连接条件**：`borrower.loan_number = loan.loan_number`
   - 这个条件确保 borrower 和 loan 配对的是**同一个贷款**
   - 没有这个条件，会产生无意义的笛卡尔积

3. **筛选条件**：`branch_name = 'Perryridge'`
   - 只保留 Perryridge 分行的贷款

### 执行逻辑
1. 先做笛卡尔积（borrower × loan）
2. 用 WHERE 筛选：贷款号匹配 + 分行是 Perryridge
3. 输出指定的三列

## 第3页：重命名操作（Rename）- 列别名

### 作用
用 `AS` 给查询结果中的**列**起一个新名字

### 例子
```sql
SELECT customer_name,
       borrower.loan_number AS loan_id,
       amount
FROM borrower, loan
WHERE borrower.loan_number = loan.loan_number;
```

### 效果
结果中的第二列，列名显示为 `loan_id`，而不是 `loan_number`

### 注意
- `AS` 可以省略（但推荐写上，更清晰）
- 列别名只影响**输出结果**的列名，不影响原表

## 第4页：元组变量（Tuple Variables）- 表别名

### 作用
给表起一个**临时的短名字**，方便书写，尤其是在**自连接**或多表查询时

### 例子
```sql
SELECT customer_name, T.loan_number, S.amount
FROM borrower AS T, loan AS S
WHERE T.loan_number = S.loan_number;
```

### 好处
- 不用写完整的表名 `borrower.loan_number`，写 `T.loan_number` 更短
- 特别适合**一个表在查询中出现多次**的情况（比如下面第5页的自连接）

## 第5页：自连接（Self-Join）的经典例子

### 需求
找出所有**资产大于 Brooklyn 某分行**的分行名称

### 表结构
**Branch**：`(branch-name, branch-city, assets)`

### SQL
```sql
SELECT DISTINCT T.branch_name
FROM branch AS T, branch AS S
WHERE T.assets > S.assets 
  AND S.branch_city = 'Brooklyn';
```

### 详细解释

**为什么要用两个别名（T 和 S）？**
- 因为我们要比较**同一个表**中的不同行
- T 代表"目标分行"，S 代表"参照分行（Brooklyn 的分行）"
- 没有别名，数据库分不清哪个是哪个

**逻辑拆解**：
1. 把 branch 表当作两个不同的表：T 和 S
2. S 筛选出位于 Brooklyn 的分行（可能有多个）
3. T 找出那些资产 > 某个 Brooklyn 分行资产的分行
4. `DISTINCT` 去重（因为一个分行可能比多个 Brooklyn 分行资产都大）

### 例子帮助理解

假设数据：

| branch_name | branch_city | assets |
|-------------|-------------|--------|
| A | Brooklyn | 100 |
| B | Brooklyn | 200 |
| C | Manhattan | 300 |
| D | Manhattan | 50 |

查询结果：
- C 资产 300 > A(100) ✅，C > B(200) ✅ → 输出 C
- D 资产 50 > A(100)? ❌，50 > B(200)? ❌ → 不输出

结果：`C`

---

## 重要提示：AS 可以省略

```sql
-- 这两种写法等价
borrower AS T   →   borrower T
branch AS T     →   branch T
```

但省略 `AS` 有时可读性会下降，建议保留。

## 本节小结

| 知识点 | 作用 | 例子 |
|--------|------|------|
| 笛卡尔积 | 两两组合 | `FROM A, B` |
| 多表查询 | 配合 WHERE 连接 | `WHERE A.key = B.key` |
| 列别名 | 重命名输出列 | `loan_number AS loan_id` |
| 表别名 | 给表起临时名字 | `borrower AS T` |
| 自连接 | 同一张表出现两次 | `branch AS T, branch AS S` |
