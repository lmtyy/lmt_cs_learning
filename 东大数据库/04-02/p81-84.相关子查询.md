好的，这几页讲的是 **IN / SOME / ALL 的等价关系**，以及**非相关子查询**与**相关子查询**的核心区别。这是子查询中非常重要的概念。

## 第1页：IN、SOME、ALL 的等价关系

### 核心等价公式

| 表达式 | 等价于 |
|--------|--------|
| `= SOME` | `IN` |
| `≠ ALL` | `NOT IN` |

### 重要提醒

| 表达式 | 是否等价于 |
|--------|-----------|
| `= SOME` | ✅ `IN` |
| `≠ SOME` | ❌ `NOT IN`（不等价！）|
| `= ALL` | ❌ `IN`（不等价！）|
| `≠ ALL` | ✅ `NOT IN` |

### 例子说明

假设子查询返回 (1, 2, 3)：

```sql
-- 这两个等价
WHERE x = SOME (1,2,3)  -- x 等于其中至少一个
WHERE x IN (1,2,3)      -- 相同含义

-- 这两个等价
WHERE x ≠ ALL (1,2,3)   -- x 不等于所有，即不等于任何一个
WHERE x NOT IN (1,2,3)  -- 相同含义

-- 注意：下面两个不等价！
WHERE x ≠ SOME (1,2,3)  -- x 只要不等于至少一个就 true（实际上几乎所有值都满足）
WHERE x NOT IN (1,2,3)  -- x 不能等于任何一个
```

### 记忆技巧
- `= SOME` = `IN`（等于某一个 = 在集合中）
- `≠ ALL` = `NOT IN`（不等于所有 = 不在集合中）

## 第2页：非相关子查询

### 定义
> 子查询**完全独立**于外层查询，可以单独执行。

### 特点
- 由**里向外**逐层处理
- 子查询先执行一次，结果用于外层查询
- 子查询不依赖外层查询的值

### 例子
```sql
SELECT customer_name
FROM Borrower
WHERE customer_name IN (SELECT customer_name FROM Depositor);
```

**执行顺序**：
1. 先执行子查询 `SELECT customer_name FROM Depositor` → 得到结果集 R
2. 再执行外层查询，用 R 作为条件

## 第3页：相关子查询

### 定义
> 子查询**依赖于**外层查询的值，不能单独执行。

### 特点
- 由**外向里**处理
- 外层查询的每一行，都要执行一次子查询
- 子查询中引用了外层查询的列

### 例子
```sql
SELECT sname
FROM students
WHERE EXISTS (SELECT *
              FROM sc
              WHERE sc.sid = students.sid);
```

### 执行过程
1. 外层取第一行学生（比如 sid=08001）
2. 执行子查询：`SELECT * FROM sc WHERE sid=08001`（看这个学生是否有选课记录）
3. 如果有记录，EXISTS 返回 True，该学生被选中
4. 外层取下一行学生，重复步骤 2-3

### 图解数据

| 学号 | 姓名 | 性别 |
|------|------|------|
| 08001 | 张小红 | 女 |
| 08002 | 王伟 | 男 |
| 08003 | 小刚 | 男 |

**SC 表**：

| 学号 | 课程号 | 成绩 |
|------|--------|------|
| 08001 | 001 | 90 |
| 08001 | 002 | 80 |
| 08003 | 002 | 60 |

**结果**：张小红（有选课）、小刚（有选课）、王伟（无选课，不输出）

## 非相关 vs 相关子查询对比

| 对比项 | 非相关子查询 | 相关子查询 |
|--------|-------------|-----------|
| 独立性 | 完全独立 | 依赖外层查询 |
| 能否单独运行 | ✅ 能 | ❌ 不能 |
| 执行次数 | 子查询执行 **1 次** | 子查询执行 **外层行数** 次 |
| 执行顺序 | 由内向外 | 由外向內 |
| 典型操作符 | `IN`, `SOME`, `ALL` | `EXISTS`, `NOT EXISTS` |

## 相关子查询的例子（补充）

### 找出没有选课的学生
```sql
SELECT sname
FROM students
WHERE NOT EXISTS (SELECT *
                  FROM sc
                  WHERE sc.sid = students.sid);
```

### 找出比所在系平均年龄大的学生
```sql
SELECT sname, sage
FROM students S1
WHERE sage > (SELECT AVG(sage)
              FROM students S2
              WHERE S2.sdept = S1.sdept);
```
这个子查询依赖外层当前行的 `sdept`，每个系的学生执行时子查询的 WHERE 条件不同。
