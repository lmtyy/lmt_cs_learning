好的，我们继续讲 **select 子句的细节**，这三页是关于**重复处理**和**通配符**的内容。

## 第1页：重复元组的处理（DISTINCT 与 ALL）

### 核心概念
SQL **允许**查询结果中出现重复行（这和关系代数默认去除重复不同）。

### 两种情况：

**1. 想去掉重复 → 用 `DISTINCT`**

> 找出 loan 表中所有分行名（不要重复）

```sql
select distinct branch_name
from loan;
```

- 如果 "Downtown" 分行有 5 笔贷款，结果中只会出现 **1 次**

**2. 不想去重 → 用 `ALL`**（这是默认行为，通常可以省略）

```sql
select all branch_name
from loan;
```

- 每一笔贷款对应的分行名都会出现，有 5 笔贷款就显示 5 次

### 一个小记忆技巧
- **DISTINCT** → 不同的 → 去重
- **ALL** → 全部 → 保留重复

> 课件最后两行有点乱，意思其实是：
> - `select distinct` → 强制去重
> - `select all`（或直接 `select`）→ 不去重

---

## 第2页：DISTINCT 作用于所有目标列（重要！）

### 关键点
**DISTINCT 是作用于 `select` 后面列出的所有列的组合，而不是只作用于第一列。**

### 正确写法
```sql
select distinct branch_name, amount
from loan;
```

**效果**：只有当 `(branch_name, amount)` 这个组合完全相同时，才会被去重。

举例：
| branch_name | amount |
|-------------|--------|
| Downtown    | 1000   |
| Downtown    | 1000   | ← 重复，会去掉一个
| Downtown    | 2000   | ← 不同，保留
| Uptown      | 1000   | ← 不同，保留

### 错误写法
```sql
select distinct branch_name, distinct amount  -- 语法错误！
```

**原因**：DISTINCT 只能在整个 select 后面写一次，不能对每个列单独写。这是 SQL 语法规则。

---

## 第3页：星号 `*` 表示所有属性

### 用法
如果你想把表中的 **所有列** 都查出来，不用一个一个列名字写，直接用 `*`。

### 例子
```sql
select *
from loan;
```

完全等价于：
```sql
select loan_number, branch_name, amount
from loan;
```

### 实际意义
- **方便**：特别是在写临时查询、想快速看表里有什么数据时
- **注意**：在生产代码中，通常建议**明确写出列名**，这样更清晰，也更容易避免表结构变化带来的问题

---

## 三个小总结

| 写法 | 含义 | 去重？ |
|------|------|--------|
| `select branch_name` | 默认 | ❌ 不去重 |
| `select all branch_name` | 明确不去重 | ❌ 不去重 |
| `select distinct branch_name` | 去重 | ✅ 去重 |
| `select distinct a, b` | 组合去重 | ✅ 组合去重 |
| `select *` | 所有列 | ❌ 不去重 |
