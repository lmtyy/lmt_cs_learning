好的，这几页讲的是 **Set Operations（集合操作）**，包括 UNION、INTERSECT 和 EXCEPT，以及它们的去重和保留重复的版本。

---

## 第1页：集合操作的类型

### 三种基本集合操作

| 操作 | SQL 关键字 | 关系代数符号 | 含义 |
|------|-----------|-------------|------|
| 并集 | `UNION` | ∪ | 在 A 或 B 中 |
| 交集 | `INTERSECT` | ∩ | 同时在 A 和 B 中 |
| 差集 | `EXCEPT` | − | 在 A 但不在 B 中 |

### 重要前提
- 两个查询必须返回**相同数量的列**
- 对应列的数据类型必须**兼容**（可以比较）

---

## 第2页：三个实际例子

### 表结构
- **Depositor**（存款客户）：`(customer_name, account_number)`
- **Borrower**（贷款客户）：`(customer_name, loan_number)`

### 例子1：UNION（并集）- 有贷款或有存款的客户

```sql
(SELECT customer_name FROM depositor)
UNION
(SELECT customer_name FROM borrower);
```

**结果**：只要客户有账户 OR 有贷款，就会出现在结果中。同一个人出现多次也只显示一次。

**图示**：
```
存款客户：{张三, 李四, 王五}
贷款客户：{李四, 王五, 赵六}
UNION 结果：{张三, 李四, 王五, 赵六}
```

---

### 例子2：INTERSECT（交集）- 既有贷款又有存款的客户

```sql
(SELECT customer_name FROM depositor)
INTERSECT
(SELECT customer_name FROM borrower);
```

**结果**：客户必须**同时**有账户 AND 有贷款。

**图示**：
```
存款客户：{张三, 李四, 王五}
贷款客户：{李四, 王五, 赵六}
INTERSECT 结果：{李四, 王五}
```

---

### 例子3：EXCEPT（差集）- 有存款但没有贷款的客户

```sql
(SELECT customer_name FROM depositor)
EXCEPT
(SELECT customer_name FROM borrower);
```

**结果**：客户有账户，但**没有**贷款。

**图示**：
```
存款客户：{张三, 李四, 王五}
贷款客户：{李四, 王五, 赵六}
EXCEPT 结果：{张三}
```

## 第3页：去重 vs 保留重复

### 默认行为：自动去重

`UNION`、`INTERSECT`、`EXCEPT` 默认会**自动去除重复行**。

### 如果想保留重复：使用 ALL 版本

| 默认（去重） | 保留重复 |
|-------------|----------|
| `UNION` | `UNION ALL` |
| `INTERSECT` | `INTERSECT ALL` |
| `EXCEPT` | `EXCEPT ALL` |

---

### 重复次数的计算规则

假设某个元组（某条记录）在第一个查询结果中出现 **m** 次，在第二个查询结果中出现 **n** 次：

| 操作 | 结果中出现的次数 |
|------|-----------------|
| `UNION ALL` | m + n |
| `INTERSECT ALL` | min(m, n) |
| `EXCEPT ALL` | max(0, m - n) |

### 举例说明

假设客户"张三"：
- 在 depositor 中出现 **3** 次（有3个账户）
- 在 borrower 中出现 **2** 次（有2笔贷款）

| 操作 | 计算公式 | 结果中张三出现次数 |
|------|----------|-------------------|
| `UNION ALL` | 3 + 2 | 5 次 |
| `INTERSECT ALL` | min(3,2) | 2 次 |
| `EXCEPT ALL` | max(0, 3-2) | 1 次 |

---

### UNION vs UNION ALL 的区别

```sql
-- 去重版本：每个客户只出现一次
SELECT customer_name FROM depositor
UNION
SELECT customer_name FROM borrower;

-- 保留重复版本：有多少条记录就出现多少次
SELECT customer_name FROM depositor
UNION ALL
SELECT customer_name FROM borrower;
```

**性能提示**：`UNION ALL` 不需要去重，通常比 `UNION` 快。如果确定数据不会有重复（或不需要去重），优先使用 `UNION ALL`。

---

## 完整对比表

| 操作 | 去重版本 | 保留重复版本 | 用途 |
|------|---------|-------------|------|
| 并集 | `UNION` | `UNION ALL` | 合并结果 |
| 交集 | `INTERSECT` | `INTERSECT ALL` | 找共同部分 |
| 差集 | `EXCEPT` | `EXCEPT ALL` | 找独有部分 |
