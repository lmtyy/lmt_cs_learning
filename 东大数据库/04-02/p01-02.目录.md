## 1. Basic Query Structure（回顾与扩展）

你已经知道基础结构：

```sql
SELECT A1, A2, ...
FROM R1, R2, ...
WHERE condition;
```

这一章里，重点会变成：
- **多表连接**（JOIN）
- **去重** `DISTINCT`
- **排序** `ORDER BY`
- **重命名** `AS`
- **模式匹配** `LIKE`

## 2. Set Operations（集合操作）

把查询结果当作**集合**（或包，multiset）来处理。

| 操作 | 含义 | SQL 关键字 |
|------|------|------------|
| 并集 | 出现在任一查询中 | `UNION`（去重） / `UNION ALL` |
| 交集 | 同时出现在两个查询 | `INTERSECT` |
| 差集 | 在第一个但不在第二个 | `EXCEPT`（有的 DB 用 `MINUS`） |

**例子**：  
找出“选修了 CS101 或者 选修了 MATH202”的学生：

```sql
SELECT sid FROM takes WHERE cid = 'CS101'
UNION
SELECT sid FROM takes WHERE cid = 'MATH202';
```

> `UNION` 自动去重，`UNION ALL` 保留重复行（更快）。

---

## 3. Aggregate Functions（聚集函数）

对一组值进行计算，返回**一个值**。

常用聚集函数：
- `COUNT` – 计数
- `SUM` – 总和
- `AVG` – 平均值
- `MAX` / `MIN` – 最大/最小

**结合 `GROUP BY`**：按某列分组，再对每组做聚集。

**例子**：每个系有多少学生？

```sql
SELECT dept_name, COUNT(*)
FROM student
GROUP BY dept_name;
```

**`HAVING`**：对分组后的结果做筛选（类似于 WHERE，但用于组条件）。

```sql
SELECT dept_name, AVG(gpa)
FROM student
GROUP BY dept_name
HAVING AVG(gpa) > 3.5;
```

> 执行顺序：  
> `FROM` → `WHERE` → `GROUP BY` → `HAVING` → `SELECT` → `ORDER BY`

## 4. Null Values（空值）

`NULL` 表示**未知或不存在**，不是 0，也不是空字符串。

**重要特性**：
- 任何与 `NULL` 的算术/比较运算 → `NULL`
- `WHERE` 中条件结果为 `NULL` 视为 **false**，该行不选中

**判断 NULL**：
```sql
WHERE age IS NULL
WHERE age IS NOT NULL
```

**对 NULL 的聚集**：
- `COUNT(*)` 包含 NULL 行
- `COUNT(column)` **不包含** NULL 行
- `SUM/AVG` 忽略 NULL（不参与计算）

## 5. Nested Subqueries（嵌套子查询）

子查询是**写在另一个 SQL 内部的查询**，可以出现在：
- `WHERE` / `HAVING`
- `FROM`
- `SELECT`（作为标量子查询）

### 常见用法

#### (1) `IN` / `NOT IN`
找出选修了“数据库”课程的学生：

```sql
SELECT name
FROM student
WHERE sid IN (
    SELECT sid
    FROM takes
    WHERE cid = 'DB101'
);
```

#### (2) 比较运算符 + `ANY` / `ALL`
- `> ANY(...)` – 大于子查询中**任意一个**
- `> ALL(...)` – 大于子查询中**所有**

#### (3) `EXISTS` / `NOT EXISTS`
检查子查询是否有结果行（常用相关子查询）。

**例子**：没有选修任何课程的学生

```sql
SELECT *
FROM student s
WHERE NOT EXISTS (
    SELECT 1
    FROM takes t
    WHERE t.sid = s.sid
);
```

> 相关子查询：内层引用外层表，**逐行执行**。

## 6. Complex Queries（复杂查询）

通常指**上面几种技术的组合**：
- 子查询 + 聚集
- 多层嵌套
- 集合操作 + GROUP BY + HAVING

**典型复杂查询例子**：  
找出平均成绩高于“所有 CS 系学生平均成绩”的非 CS 系学生。

```sql
SELECT dept_name, AVG(score)
FROM student
WHERE dept_name != 'CS'
GROUP BY dept_name
HAVING AVG(score) > (
    SELECT AVG(score)
    FROM student
    WHERE dept_name = 'CS'
);
```

| 模块 | 一句话核心 |
|------|-------------|
