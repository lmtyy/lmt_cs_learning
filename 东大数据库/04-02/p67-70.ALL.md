## 第1页：用 `> ALL` 实现需求

### 需求
找出**资产大于 Brooklyn 所有分行的分行**名称

### SQL
```sql
SELECT branch_name
FROM branch
WHERE assets > ALL
    (SELECT assets
     FROM branch
     WHERE branch_city = 'Brooklyn');
```

### 与 `> SOME` 的区别

| 操作符 | 含义 | 条件强度 |
|--------|------|----------|
| `> SOME` | 大于**至少一个** Brooklyn 分行 | 较松 |
| `> ALL` | 大于**每一个** Brooklyn 分行 | 较严格 |

### 举例
假设 Brooklyn 分行资产：100万、200万

| 分行资产 | `> SOME` | `> ALL` |
|----------|----------|---------|
| 150万 | ✅（>100） | ❌（>200？否） |
| 250万 | ✅（>100） | ✅（>200） |
| 90万 | ❌ | ❌ |

## 第2页：`ALL` 的数学定义

### 语法
```
F <comp> ALL r
```

### 含义
> 对于 r 中的**每一个**值 t，F 和 t 都满足比较关系 `<comp>`

### 例子解析

| 表达式 | 集合 r | 结果 | 原因 |
|--------|-------|------|------|
| `5 < ALL (0, 5, 6)` | (0,5,6) | ❌ false | 5 < 0? 否 |
| `5 < ALL (6, 10)` | (6,10) | ✅ true | 5 < 6 ✅ 且 5 < 10 ✅ |
| `5 ≠ ALL (4, 6)` | (4,6) | ✅ true | 5 ≠ 4 ✅ 且 5 ≠ 6 ✅ |
| `5 = ALL (4, 5, 5)` | (4,5,5) | ❌ false | 5 = 4? 否，不满足所有 |

### 关键理解
- `ALL` 要求**全部**满足，有一个不满足就为 false
- 空集合的情况：`ALL` 返回 true（没有反例）

## 第3页：`> ALL` 等价于 `> MAX`

### 两种写法等价

**写法1：用 ALL**
```sql
SELECT branch_name
FROM branch
WHERE assets > ALL
    (SELECT assets FROM branch WHERE branch_city = 'Brooklyn');
```

**写法2：用 MAX**
```sql
SELECT branch_name
FROM branch
WHERE assets > (SELECT MAX(assets)
                FROM branch
                WHERE branch_city = 'Brooklyn');
```

### 为什么等价？
- 大于**所有**值 = 大于这些值中的**最大值**
- `> ALL` → `> MAX`
- `< ALL` → `< MIN`

### 通用等价关系

| 操作 | 等价写法 |
|------|----------|
| `> ALL` | `> MAX(子查询)` |
| `< ALL` | `< MIN(子查询)` |
| `= ALL` | 很少用，要求子查询只有一个值 |
| `≠ ALL` | 等价于 `NOT IN`（注意处理 NULL 的区别）|

## 第4页：语法注意事项（重要！）

### 错误写法 ❌
```sql
SELECT branch_name
FROM branch
WHERE (SELECT MAX(assets)
       FROM branch
       WHERE branch_city = 'Brooklyn') <= assets;
```

### 正确写法 ✅
```sql
SELECT branch_name
FROM branch
WHERE assets > (SELECT MAX(assets)
                FROM branch
                WHERE branch_city = 'Brooklyn');
```

### 规则
> **子查询必须放在比较操作符的右边**（后面）

也就是说：
- ✅ `WHERE assets > (子查询)`
- ✅ `WHERE assets > ALL (子查询)`
- ❌ `WHERE (子查询) < assets`

这是 SQL 语法规定，不是逻辑错误。

## `SOME` vs `ALL` 总结对比

| 操作符 | 含义 | 等价写法 | 条件强度 |
|--------|------|----------|----------|
| `= SOME` | 等于至少一个 | `IN` | 宽松 |
| `= ALL` | 等于所有 | 很少用 | 严格 |
| `> SOME` | 大于至少一个 | `> MIN` | 宽松 |
| `> ALL` | 大于所有 | `> MAX` | 严格 |
| `< SOME` | 小于至少一个 | `< MAX` | 宽松 |
| `< ALL` | 小于所有 | `< MIN` | 严格 |
| `≠ SOME` | 不等于至少一个 | 特殊 | - |
| `≠ ALL` | 不等于所有 | `NOT IN` | 严格 |
