## 第1页：多表连接 + 分组统计

### 需求
统计**每个分行**的存款客户数量（去重）

### 涉及的表
- **Depositor**（存款客户）：`(customer-name, account-number)`
- **Account**（账户）：`(branch-name, account-number, balance)`

### SQL
```sql
SELECT branch_name, COUNT(DISTINCT customer_name)
FROM depositor, account
WHERE depositor.account_number = account.account_number
GROUP BY branch_name;
```

### 逻辑拆解
1. **连接**：通过 `account_number` 将 depositor 和 account 表连接起来，知道每个客户属于哪个分行
2. **分组**：按 `branch_name` 分组
3. **统计**：对每组统计不同的客户数（用 DISTINCT 去重）

---

## 第2页：逐步演算过程

### 步骤1：先做连接（不带 GROUP BY）

```sql
SELECT branch_name, customer_name
FROM depositor, account
WHERE depositor.account_number = account.account_number;
```

**连接结果**（中间表）：

| branch_name | customer_name |
|-------------|---------------|
| Perryridge | John Wong |
| Perryridge | Jacky Chan |
| Uptown | John Wong |
| Uptown | Mary Kwan |
| Downtown | John Wong |
| Downtown | Pat Lee |
| Downtown | May Cheung |

> 注意：课件中第一张表显示的是这个结果，第二张表增加了重复数据（可能是演示用）

---

### 步骤2：按 branch_name 分组

**分组后的各组**：

| 组 | branch_name | 包含的 customer_name |
|---|-------------|---------------------|
| 第1组 | Perryridge | John Wong, Jacky Chan |
| 第2组 | Uptown | John Wong, Mary Kwan |
| 第3组 | Downtown | John Wong, Pat Lee, May Cheung |

---

### 步骤3：对每组应用 COUNT(DISTINCT customer_name)

| branch_name | COUNT(DISTINCT customer_name) |
|-------------|------------------------------|
| Perryridge | 2 |
| Uptown | 2 |
| Downtown | 3 |

**最终结果**：

| branch_name | count |
|-------------|-------|
| Perryridge | 2 |
| Uptown | 2 |
| Downtown | 3 |

## 为什么需要 DISTINCT？

### 场景说明

如果一个客户在同一个分行有**多个账户**，连接结果中该客户会出现多次。

例如：John Wong 在 Perryridge 有两个账户：

| branch_name | customer_name |
|-------------|---------------|
| Perryridge | John Wong |
| Perryridge | John Wong |
| Perryridge | Jacky Chan |

### 对比

```sql
-- 不加 DISTINCT：会重复计算
SELECT branch_name, COUNT(customer_name)
...
-- Perryridge 的结果是 3（John Wong 被算了两次）

-- 加 DISTINCT：每个客户只算一次
SELECT branch_name, COUNT(DISTINCT customer_name)
...
-- Perryridge 的结果是 2（John Wong 只算一次）
```

**本题想要的是"客户数量"（每个客户只算一次），所以必须用 DISTINCT。**

---

## 多表连接 + GROUP BY 的执行顺序

```
1. FROM (depositor, account)      -- 做笛卡尔积
2. WHERE (连接条件)               -- 筛选出匹配的行
3. 得到连接后的中间表
4. GROUP BY branch_name           -- 按分行分组
5. COUNT(DISTINCT customer_name)  -- 对每组统计不同客户数
6. SELECT 输出结果
```

---

## 常见写法对比

| 需求 | SQL |
|------|-----|
| 每个分行的账户总数 | `SELECT branch_name, COUNT(account_number) FROM account GROUP BY branch_name` |
| 每个分行的客户数（去重） | `SELECT branch_name, COUNT(DISTINCT customer_name) FROM depositor, account WHERE ... GROUP BY branch_name` |
| 每个分行的总余额 | `SELECT branch_name, SUM(balance) FROM account GROUP BY branch_name` |
