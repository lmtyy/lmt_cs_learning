好的，这两页讲的是 **WITH 子句**（公共表表达式，CTE）。这是一个非常实用的功能，可以让复杂查询变得更清晰。

## 第1页：WITH 的基本用法

### 需求
找出所有**余额等于最大余额**的账户。

### 不用 WITH 的写法
```sql
SELECT account_number
FROM account
WHERE balance = (SELECT MAX(balance) FROM account);
```

这个写法没问题，但如果是更复杂的情况，子查询会变得很乱。

### 用 WITH 的写法
```sql
WITH max_balance(value) AS
    (SELECT MAX(balance)
     FROM account)
SELECT account_number
FROM account, max_balance
WHERE account.balance = max_balance.value;
```

### 解释
- `WITH max_balance(value) AS (...)`：定义了一个**临时视图**，名叫 `max_balance`，它有一列叫 `value`，内容就是子查询的结果（最大余额）
- 然后下面的查询可以像使用普通表一样使用 `max_balance`

### 图示

```
WITH 定义：
┌─────────────────┐
│ max_balance     │
├─────────────────┤
│ value = 5000    │  ← 子查询结果
└─────────────────┘

主查询：
account 表           max_balance
┌────────┬───────┐   ┌───────┐
│ acc_no │ bal   │   │ value │
├────────┼───────┤   ├───────┤
│ A-101  │ 1000  │   │ 5000  │
│ A-102  │ 5000  │   └───────┘
│ A-103  │ 3000  │
└────────┴───────┘
         ↓ JOIN
    只保留 bal = 5000 的行
```

## 第2页：WITH 的复杂用法

### 需求
找出所有**总存款大于所有分行平均总存款**的分行。

### 逻辑分解
1. 先计算每个分行的**总存款**（branch_total）
2. 再计算所有分行总存款的**平均值**（branch_total_avg）
3. 最后筛选出总存款 ≥ 平均值的分行

### SQL
```sql
WITH 
branch_total(branch_name, value) AS
    (SELECT branch_name, SUM(balance)
     FROM account
     GROUP BY branch_name),
branch_total_avg(value) AS
    (SELECT AVG(value)
     FROM branch_total)
SELECT branch_name
FROM branch_total, branch_total_avg
WHERE branch_total.value >= branch_total_avg.value;
```

### 执行过程

**步骤1**：`branch_total` 临时视图

| branch_name | value (总存款) |
|-------------|---------------|
| Perryridge | 15000 |
| Brighton | 8000 |
| Redwood | 10000 |

**步骤2**：`branch_total_avg` 临时视图

| value (平均值) |
|---------------|
| (15000+8000+10000)/3 = 11000 |

**步骤3**：主查询筛选

| branch_name | value | >= 11000? |
|-------------|-------|-----------|
| Perryridge | 15000 | ✅ |
| Brighton | 8000 | ❌ |
| Redwood | 10000 | ❌ |

**结果**：Perryridge

## 使用 WITH 的好处

| 场景 | 好处 |
|------|------|
| 复杂查询 | 把大查询拆成多个小步骤，逻辑更清晰 |
| 重复使用同一子查询 | 只需定义一次，可以多次引用 |
| 递归查询 | WITH 支持递归（WITH RECURSIVE），用于树形结构（如组织架构、地铁线路）|

## 与子查询的对比

| 写法 | 优点 | 缺点 |
|------|------|------|
| 嵌套子查询 | 在一个地方写完 | 层数多了很难读懂 |
| WITH 子句 | 分步骤、可读性强 | 某些老数据库不支持 |

### 同一个问题的两种写法对比

**不用 WITH（嵌套子查询）**：
```sql
SELECT branch_name
FROM account
GROUP BY branch_name
HAVING SUM(balance) >= (SELECT AVG(total)
                        FROM (SELECT SUM(balance) AS total
                              FROM account
                              GROUP BY branch_name) AS t);
```
很乱，不容易看懂。

**用 WITH**：
```sql
WITH branch_total AS (SELECT branch_name, SUM(balance) AS total
                      FROM account
                      GROUP BY branch_name)
SELECT branch_name
FROM branch_total
WHERE total >= (SELECT AVG(total) FROM branch_total);
```
清晰多了！
