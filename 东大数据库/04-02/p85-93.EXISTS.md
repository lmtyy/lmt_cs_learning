## 第1页：EXISTS 的基本概念

### 作用
`EXISTS` 测试子查询的结果集是否**非空**。

### 返回值
| 子查询结果 | EXISTS 返回 |
|-----------|-------------|
| 非空（有数据）| `true` |
| 空（无数据）| `false` |

### 数学定义
- `EXISTS r` ⇔ `r ≠ ∅`（r 非空为真）
- `NOT EXISTS r` ⇔ `r = ∅`（r 为空为真）

---

## 第2页：EXISTS 和 NOT EXISTS 的详细说明

### 关键点
1. **子查询不返回数据**，只返回逻辑真值 `true` 或 `false`
2. 子查询的 `SELECT` 列表通常用 `*`，因为不关心具体数据
3. 带 `EXISTS` 的子查询是**相关子查询**，引用了外层查询的列

### NOT EXISTS
| 子查询结果 | NOT EXISTS 返回 |
|-----------|-----------------|
| 非空 | `false` |
| 空 | `true` |

## 第3页：例子 - 选修了001号课程的学生（用连接）

### 表结构
**Students**：

| 学号 | 姓名 | 性别 |
|------|------|------|
| 08001 | 张小红 | 女 |
| 08002 | 王伟 | 男 |
| 08003 | 小刚 | 男 |

**SC**：

| 学号 | 课程号 | 成绩 |
|------|--------|------|
| 08001 | 001 | 90 |
| 08001 | 002 | 80 |
| 08003 | 002 | 60 |

### 连接写法
```sql
SELECT sname
FROM students, sc
WHERE students.sid = sc.sid AND cid = '001';
```

**结果**：张小红

---

## 第4页：用 EXISTS 实现（相关子查询）

```sql
SELECT sname
FROM students
WHERE EXISTS (SELECT *
              FROM sc
              WHERE sid = students.sid AND cid = '001');
```

### 执行过程
1. 取 students 第一行（08001，张小红）
   - 子查询：`SELECT * FROM sc WHERE sid=08001 AND cid='001'` → 有记录 ✅ → EXISTS 为 true → 输出"张小红"
2. 取 students 第二行（08002，王伟）
   - 子查询：`SELECT * FROM sc WHERE sid=08002 AND cid='001'` → 无记录 ❌ → EXISTS 为 false → 不输出
3. 取 students 第三行（08003，小刚）
   - 子查询：`SELECT * FROM sc WHERE sid=08003 AND cid='001'` → 无记录 ❌ → 不输出

**结果**：张小红（与连接写法相同）

---

## 第5页：NOT EXISTS - 没选修001号课程的学生

```sql
SELECT sname
FROM students
WHERE NOT EXISTS (SELECT *
                  FROM sc
                  WHERE sid = students.sid AND cid = '001');
```

### 执行过程
- 张小红：子查询有记录 → EXISTS 为 true → NOT EXISTS 为 false → ❌ 不输出
- 王伟：子查询无记录 → EXISTS 为 false → NOT EXISTS 为 true → ✅ 输出"王伟"
- 小刚：子查询无记录 → NOT EXISTS 为 true → ✅ 输出"小刚"

**结果**：王伟、小刚

### 思考题
> 能否用简单的连接来实现"没有选修001号课程"？

可以用 `NOT IN`：
```sql
SELECT sname
FROM students
WHERE sid NOT IN (SELECT sid FROM sc WHERE cid = '001');
```

但需要注意 `NOT IN` 对 NULL 的处理问题。

## 第6页：只选修了1门课程的学生（用 EXISTS + 子查询分组）

```sql
SELECT sname
FROM students
WHERE EXISTS (SELECT sid, COUNT(cid)
              FROM sc
              WHERE sid = students.sid
              GROUP BY sid
              HAVING COUNT(cid) = 1);
```

### 执行过程
1. 张小红（08001）：在 SC 中有2条记录 → `COUNT(cid)=2` → `HAVING 2=1` 为 false → 子查询返回空 → EXISTS 为 false → ❌
2. 王伟（08002）：在 SC 中无记录 → 子查询返回空 → EXISTS 为 false → ❌
3. 小刚（08003）：在 SC 中有1条记录 → `HAVING 1=1` 为 true → 子查询返回非空 → EXISTS 为 true → ✅ 输出"小刚"

**结果**：小刚

---

## 第7页：用 IN 实现"只选修了1门课程"

```sql
SELECT sname
FROM students
WHERE sid IN (SELECT sid
              FROM sc
              GROUP BY sid
              HAVING COUNT(cid) = 1);
```

这个写法更简洁，因为子查询返回的是满足条件的 sid 列表，然后外层用 `IN` 判断。

## 第8页：相关子查询 - 比较年龄与同职称平均年龄

### 需求
查询教师姓名，要求其年龄**大于**同职称教师的平均年龄。

### 表结构
**Teacher**：`(pid, pname, age, title)`

### SQL
```sql
SELECT pname
FROM teacher t1
WHERE age > (SELECT AVG(age)
             FROM teacher t2
             WHERE t1.title = t2.title);
```

### 说明
- 子查询计算**同职称**的平均年龄
- `t1.title = t2.title` 实现"同职称"的关联
- 这是一个**相关子查询**，每个教师执行一次子查询

## 第9页：所有课程都不及格的学生（用 `> ALL`）

### 需求
查询**所有课程都不及格**（score < 60）的学生姓名。

### SQL
```sql
SELECT sname
FROM student
WHERE 60 > ALL (SELECT score
                FROM sc
                WHERE student.sid = sc.sid);
```

### 逻辑
- `60 > ALL (成绩列表)` 等价于：所有成绩都小于 60
- 即该学生的每一门课成绩都 < 60

### 等价写法（用 NOT EXISTS）
```sql
SELECT sname
FROM student
WHERE NOT EXISTS (SELECT *
                  FROM sc
                  WHERE student.sid = sc.sid AND score >= 60);
```
即：不存在任何一门成绩 ≥ 60 的课程。

## EXISTS 与 IN 的对比总结

| 场景 | 推荐写法 | 原因 |
|------|----------|------|
| 简单成员测试 | `IN` | 简洁、直观 |
| 需要关联外层条件 | `EXISTS` | IN 无法直接关联 |
| 处理 NULL | `EXISTS` 更安全 | IN 遇到 NULL 会有问题 |
| 性能 | 视具体情况 | 优化器通常能处理 |
