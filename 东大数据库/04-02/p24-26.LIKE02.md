好的，这三页继续讲 **LIKE 字符匹配**，包括更多例子和**转义字符**（如何处理真正的 `%` 或 `_`）。

## 第1页：更多 LIKE 例子

### 例子1：查询所有姓刘的学生

```sql
SELECT Sname, Sno, Ssex
FROM Student
WHERE Sname LIKE '刘%';
```

- `'刘%'`：刘开头，后面任意字符（可以是0个或多个）
- 能匹配：刘、刘备、刘邦、刘晓明...

### 例子2：姓"欧阳"且全名为四个汉字

```sql
SELECT Sname
FROM Student
WHERE Sname LIKE '欧阳__';
```

- `'欧阳__'`：欧阳 + 后面恰好两个字符
- 能匹配：欧阳静香、欧阳无敌
- 不能匹配：欧阳（只有两个字）、欧阳锋

---

## 第2页：更灵活的模式

### 例子3：名字中第2个字为"阳"

```sql
SELECT Sname, Sno
FROM Student
WHERE Sname LIKE '_阳%';
```

- `'_阳%'`：
  - `_`：第1个字符（任意）
  - `阳`：第2个字符必须是"阳"
  - `%`：后面任意字符（可以没有）
- 能匹配：刘阳、欧阳XX（欧是第1个，阳是第2个）、张阳光...

### 例子4：查询所有不姓刘的学生

```sql
SELECT Sname, Sno, Ssex
FROM Student
WHERE Sname NOT LIKE '刘%';
```

- `NOT LIKE` 表示**不匹配**这个模式
- 能匹配：姓张、李、王... 以及不姓刘的所有人

## 第3页：转义字符（重要！）

### 问题场景
如果你想查询的名字中**本身就包含 `%` 或 `_`**，怎么办？

比如课程名：`DB_Design`（这里的 `_` 是普通下划线，不是通配符）

如果直接写：
```sql
WHERE Cname LIKE 'DB_Design'
```
`_` 会被解释为"任意单个字符"，所以 `DB_Design` 能匹配 `DBXDesign`、`DB-Design` 等，这不一定是想要的结果。

### 解决方案：使用转义字符（ESCAPE）

```sql
SELECT Cno, Ccredit
FROM Course
WHERE Cname LIKE 'DB\_Design' ESCAPE '\';
```

- `ESCAPE '\'` 声明：`\` 是转义字符
- `\_` 表示：把 `_` 当作普通的下划线字符，而不是通配符
- 同样，`\%` 可以把 `%` 当作普通百分号

### 第二个例子（更复杂）

```sql
SELECT *
FROM Course
WHERE Cname LIKE 'DB\_%i\__' ESCAPE '\';
```

我们来拆解这个模式 `'DB\_%i\__'`：

| 模式中的字符 | 实际含义 |
|-------------|----------|
| `D` | 字符 D |
| `B` | 字符 B |
| `\_` | 普通下划线 `_`（不是通配符） |
| `%` | 通配符，匹配任意字符串 |
| `i` | 字符 i |
| `\__` | 转义后：普通下划线 `_` + 通配符 `_`（匹配任意一个字符） |

所以匹配的是：
- 以 `DB_` 开头
- 中间任意字符串
- 然后有一个 `i`
- 然后一个普通下划线 `_`
- 最后再跟任意一个字符

能匹配的例子：`DB_Design_iX`、`DB_Data_i_5` 等。

---

## 转义字符总结

| 你想匹配的字符 | 写法（设 ESCAPE '\'） |
|--------------|----------------------|
| 通配符 `%` | `\%` |
| 通配符 `_` | `\_` |
| 转义符本身 `\` | `\\` |

### 也可以选其他转义字符
```sql
-- 用 # 作为转义字符
WHERE Cname LIKE 'DB#_Design' ESCAPE '#';
```
不一定非要用 `\`，可以选任何不会出现在数据中的字符。

---

## 本节小结

| 模式 | 含义 |
|------|------|
| `'刘%'` | 刘开头 |
| `'%阳%'` | 包含阳 |
| `'_阳%'` | 第二个字是阳 |
| `NOT LIKE '刘%'` | 不姓刘 |
| `ESCAPE` | 把通配符变成普通字符 |
