## 第1页：SQL 查询的基本形式

SQL 的查询结构其实来自关系代数（relational algebra）和关系演算（relational calculus），你不需要太纠结这两个理论术语，只要知道它们是 SQL 的数学基础就行。

一个典型的 SQL 查询长这样：

```sql
select A1, A2, ..., An
from r1, r2, ..., rm
where P
```

- **select** 后面写的是你想要 **显示出来的属性（列）**，比如名字、金额等。
- **from** 后面写的是 **从哪里查**，也就是表名，可以是一个表，也可以是多个表。
- **where** 后面写的是 **筛选条件**（predicate），比如“金额大于1000”。

这个形式和你以后见到的大多数 SQL 查询是一致的。

---

## 第2页：SQL 与关系代数的等价关系

这一页给出了一个关系代数表达式：

\[
\Pi_{A_1, A_2, \dots, A_n} (\sigma_P(r_1 \times r_2 \times \dots \times r_m))
\]

简单解释一下：
- \(\Pi\)（投影）对应 SQL 中的 `select` 子句，用来选择列。
- \(\sigma\)（选择）对应 SQL 中的 `where` 子句，用来筛选行。
- \(\times\)（笛卡尔积）对应 SQL 中的 `from` 多个表（如果没有写连接条件的话）。

**重点记住**：SQL 查询的结果 **仍然是一个关系（表）**，也就是说它是有行有列的二维结构，可以继续作为查询的输入。

## 第3页：例子中使用的表结构

这一页给出了后续例子会用到的几个表，比较重要的是：

- **Branch**（分行）：分行名、城市、资产
- **Loan**（贷款）：贷款号、金额、分行名
- **Borrower**（借款者）：客户名、贷款号

后面所有例子都会围绕这些表展开，你可以先有个印象，不急着背。

## 第4页：select 子句（对应投影）

`select` 子句的作用是 **选择你想要的列**，这和关系代数中的“投影”是一样的。

例子：

> 找出 loan 表中的所有分行名

表结构：Loan（loan-number, amount, branch-name）

SQL 语句：

```sql
select branch_name
from loan
```

等价于关系代数：

\[
\Pi_{\text{branch\_name}} (\text{loan})
\]

意思就是：从 loan 表里，把 `branch_name` 这一列拿出来，其他列不要。

**注意**：如果同一个分行有多笔贷款，它的名字会出现多次（除非你用 `distinct`，后面会讲）。

---

## 第5页：SQL 大小写不敏感

SQL 对大小写不敏感，也就是说下面三种写法 **完全一样**：

- Branch_Name
- BRANCH_NAME
- branch_name

不过在实际写 SQL 时，常见习惯是把 **关键字大写**（SELECT, FROM, WHERE），**表名和列名小写**，这样可读性更好。但数据库执行时不会区分大小写。

---

## 小结一下这一节的核心

| 部分 | 作用 | 对应关系代数 |
|------|------|--------------|
| `select` | 选哪些列 | 投影（Π） |
| `from` | 从哪些表 | 笛卡尔积（×） |
| `where` | 筛选哪些行 | 选择（σ） |

查询结果仍然是一个表。
