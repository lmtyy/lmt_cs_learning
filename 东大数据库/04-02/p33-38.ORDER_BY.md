## 第1页：ORDER BY 的基本概念

### 作用
让查询结果按照指定的列**排序**显示

### 语法
```sql
ORDER BY 列名1 [ASC|DESC], 列名2 [ASC|DESC], ...
```

### 排序方向
| 关键字 | 含义 | 默认 |
|--------|------|------|
| `ASC` | 升序（从小到大） | ✅ 默认 |
| `DESC` | 降序（从大到小） | |

### 例子
```sql
-- 按成绩升序（默认，小到大）
ORDER BY Grade

-- 按成绩降序（大到小）
ORDER BY Grade DESC

-- 先按系升序，再按年龄降序
ORDER BY Sdept ASC, Sage DESC
```

## 第2页：单列排序的例子

### 需求
查询选修了 **3号课程** 的学生学号和成绩，按成绩**降序**排列（高分在前）

### SQL
```sql
SELECT Sno, Grade
FROM SC
WHERE Cno = '3'
ORDER BY Grade DESC;
```

### 数据演示

**SC 表原始数据**（部分）：

| Sno | Cno | Grade |
|-----|-----|-------|
| 200215121 | 1 | 92 |
| 200215121 | 2 | 85 |
| 200215121 | 3 | 88 |
| 200215122 | 2 | 90 |
| 200215122 | 3 | 80 |

**WHERE Cno='3' 筛选后**：

| Sno | Grade |
|-----|-------|
| 200215121 | 88 |
| 200215122 | 80 |

**ORDER BY Grade DESC 排序后**：

| Sno | Grade |
|-----|-------|
| 200215121 | 88 | ← 高的在前
| 200215122 | 80 | ← 低的在后

## 第3页：多列排序的例子

### 需求
查询全体学生，按：
1. **系号升序**（先按系排序）
2. **同一系内按年龄降序**（年龄大的在前）

### SQL
```sql
SELECT *
FROM Student
ORDER BY Sdept, Sage DESC;
```

### 数据演示

**原始数据**：

| Sno | Sname | Ssex | Sage | Sdept |
|-----|-------|------|------|-------|
| 200215121 | 李勇 | 男 | 20 | CS |
| 200215122 | 刘晨 | 女 | 19 | IS |
| 200215123 | 王敏 | 女 | 18 | MA |
| 200215124 | 张立 | 男 | 19 | IS |

**排序后**：

| Sno | Sname | Ssex | Sage | Sdept |
|-----|-------|------|------|-------|
| 200215122 | 刘晨 | 女 | 19 | IS |
| 200215121 | 李勇 | 男 | 20 | CS |
| 200215124 | 张立 | 男 | 19 | IS |
| 200215123 | 王敏 | 女 | 18 | MA |

> 注意：IS 系的两条记录中，Sage 大的（19 岁）排在前面，但课件输出有点乱，核心意思是**先按 Sdept 排，再在每个系内部按 Sage 降序**。

### 多列排序的执行顺序
1. 先按第一列排序
2. 第一列相同时，再按第二列排序
3. 以此类推

## 第4页：LIMIT 子句（限制行数）

### 作用
只返回查询结果中的**前 N 行**，常用于分页或取 top N

### 重要提示
- **不是所有数据库都支持**：MySQL、PostgreSQL、SQLite 支持；SQL Server 用 `TOP`，Oracle 用 `ROWNUM`
- 通常与 `ORDER BY` 配合使用，否则结果不稳定

### 语法
```sql
SELECT 列名
FROM 表名
ORDER BY 列名
LIMIT row_count OFFSET offset;
```

| 参数 | 含义 |
|------|------|
| `row_count` | 返回的行数 |
| `OFFSET offset` | 跳过前 offset 行，从第 offset+1 行开始取 |

---

## 第5页：LIMIT 例子（前5行）

### 需求
查询员工表，按 `first_name` 排序，只取**前5条**

### SQL
```sql
SELECT employee_id, first_name, last_name
FROM employees
ORDER BY first_name
LIMIT 5;
```

### 效果
按 first_name 排序后，取前5条：

| employee_id | first_name | last_name |
|-------------|------------|-----------|
| 103 | Alexander | Lee |
| 115 | Alexander | Su |
| 114 | Avg | Su |
| 193 | Britney | Zhao |
| 104 | Bruce | Wong |

---

## 第6页：LIMIT + OFFSET（分页）

### 需求
按 `first_name` 排序，**跳过前3条**，取接下来的5条

### SQL
```sql
SELECT employee_id, first_name, last_name
FROM employees
ORDER BY first_name
LIMIT 5 OFFSET 3;
```

### 执行逻辑
1. 按 first_name 排序
2. 跳过前3条（第1、2、3条不要）
3. 从第4条开始，取5条

### 结果

| employee_id | first_name | last_name |
|-------------|------------|-----------|
| 193 | Britney | Zhao |
| 104 | Bruce | Wong |
| 179 | Charles | Yang |
| 109 | Daniel | Chen |
| 105 | David | Liang |

### MySQL 简写形式
```sql
-- 跳过前3条，取5条
LIMIT 3, 5   -- 注意：第一个是偏移量，第二个是行数
```

---

## 分页公式（重要！）

假设每页显示 `pageSize` 条，要查第 `pageNum` 页：

```sql
LIMIT pageSize OFFSET (pageNum - 1) * pageSize
```

例如：每页10条，查第3页 → `LIMIT 10 OFFSET 20`

---

## 本节小结

| 子句 | 作用 | 例子 |
|------|------|------|
| `ORDER BY 列名` | 按列排序（默认升序） | `ORDER BY Grade` |
| `ORDER BY 列名 DESC` | 降序排列 | `ORDER BY Grade DESC` |
| `ORDER BY 列1, 列2 DESC` | 多列排序 | `ORDER BY Sdept, Sage DESC` |
| `LIMIT n` | 取前 n 行 | `LIMIT 5` |
| `LIMIT n OFFSET m` | 跳过 m 行，取 n 行 | `LIMIT 5 OFFSET 3` |
