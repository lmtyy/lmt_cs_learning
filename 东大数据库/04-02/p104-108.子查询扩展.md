## 第1页：子查询在 FROM 子句中

### 需求
找出**平均账户余额大于 $1200** 的分行名称和平均余额。

### 一种错误思路（不能直接写）
```sql
-- 错误：WHERE 中不能直接用聚集函数
SELECT branch_name, AVG(balance)
FROM account
WHERE AVG(balance) > 1200
GROUP BY branch_name;  -- ❌ 错误！
```

### 正确写法：用子查询在 FROM 中

```sql
SELECT branch_name, avg_balance
FROM (SELECT branch_name, AVG(balance) AS avg_balance
      FROM account
      GROUP BY branch_name) AS branch_avg
WHERE avg_balance > 1200;
```

### 执行过程
1. **内层子查询**：先计算每个分行的平均余额，生成一个**临时表** `branch_avg`
   
   | branch_name | avg_balance |
   |-------------|-------------|
   | Perryridge | 650 |
   | Brighton | 750 |
   | Redwood | 700 |

2. **外层查询**：从这个临时表中筛选 `avg_balance > 1200` 的行

### 关键点
- FROM 子句中的子查询必须有一个**别名**（`AS branch_avg`）
- 可以给子查询的结果列起别名（`avg_balance`）
- 这种方法称为**派生表**或**内联视图**

## 第2页：子查询在 SELECT 子句中

### 场景
查询每个国家的名称，以及该国**人员的最大年龄**。

### 表结构
**Person**：

| ID | Name | Age | countryID |
|----|------|-----|-----------|
| 1 | John | 20 | 1 |
| 2 | Gina | 25 | 3 |
| 3 | Ping | 22 | 2 |
| 4 | Wei | 21 | 2 |
| 5 | Ann | 19 | 1 |

**Country**：

| countryID | Name | Capital |
|-----------|------|---------|
| 1 | Ireland | Dublin |
| 2 | China | Beijing |
| 3 | Italy | Rome |
| 4 | France | Paris |

### 期望结果

| Name | MaxAge |
|------|--------|
| Ireland | 20 |
| China | 22 |
| Italy | 25 |
| France | null |

---

## 第3页：两种实现方法对比

### 方法1：JOIN + GROUP BY（传统写法）

```sql
SELECT c.Name, MAX(p.Age) AS MaxAge
FROM Country c
JOIN Person p ON p.countryID = c.countryID
GROUP BY c.Name;
```

**结果**：

| Name | MaxAge |
|------|--------|
| Ireland | 20 |
| China | 22 |
| Italy | 25 |

**问题**：France 没有人员，JOIN 会把它**丢掉**，所以结果中不显示 France。

---

### 方法2：子查询在 SELECT 中

```sql
SELECT Name,
    (SELECT MAX(Age)
     FROM Person
     WHERE Person.countryID = Country.countryID) AS MaxAge
FROM Country;
```

**结果**：

| Name | MaxAge |
|------|--------|
| Ireland | 20 |
| China | 22 |
| Italy | 25 |
| France | **null** |

### 优势
- **保留了所有国家**，包括没有人员的 France（子查询返回 NULL）
- 这是一个**相关子查询**：每处理一个国家，就用它的 `countryID` 去 Person 表中找最大年龄

## 第4页：子查询在不同位置的总结

这一页给出了**三种位置**的对比：

| 子查询位置 | 作用 | 要求 |
|-----------|------|------|
| **WHERE 子句** | 生成一个集合，用于测试条件 | 返回一列（可以是多行） |
| **FROM 子句** | 生成一个临时表，供外层查询使用 | 必须有别名，可以是多行多列 |
| **SELECT 子句** | 生成结果中的一列 | **必须返回单个值**（一行一列） |

### SELECT 子句中的子查询：重要规则

```sql
-- ✅ 正确：子查询返回一个值
SELECT Name,
    (SELECT MAX(Age) FROM Person WHERE countryID = Country.countryID) AS MaxAge
FROM Country;

-- ❌ 错误：子查询返回多行
SELECT Name,
    (SELECT Age FROM Person WHERE countryID = Country.countryID) AS Ages  -- 可能返回多行！
FROM Country;
```

---

## 三种位置的对比总结

| 位置 | 语法示例 | 返回值要求 | 是否可相关 |
|------|----------|-----------|-----------|
| WHERE | `WHERE age > (SELECT AVG(age) FROM ...)` | 单值（或单列多行配合 IN/SOME/ALL）| ✅ |
| FROM | `FROM (SELECT ...) AS t` | 表（多行多列）| ❌（子查询独立）|
| SELECT | `SELECT (SELECT MAX(...) FROM ...) AS alias` | **单值**（一行一列）| ✅ |
