好的，我们开始讲 **where 子句**，这是 SQL 查询中非常核心的部分，用来**筛选行**。

---

## 第1页：where 子句的作用

### 核心概念
- `where` 子句指定了查询结果必须满足的**条件**
- 对应关系代数中的**选择（selection）**运算，符号是 σ

### 通俗理解
`from` 决定从哪些表拿数据，`where` 决定**只拿哪些行**。

---

## 第2页：一个完整的例子

### 场景
Loan 表：`loan(loan_number, amount, branch_name)`

### 需求
找出 **Perryridge 分行** 发放的、**金额大于 1200** 的所有贷款，只要贷款号。

### SQL
```sql
select loan_number
from loan
where branch_name = 'Perryridge' and amount > 1200;
```

### 执行逻辑（想象一下）
1. 从 loan 表一行一行地看
2. 检查该行是否满足：`branch_name = 'Perryridge'` **并且** `amount > 1200`
3. 满足条件的行，取出 `loan_number` 放到结果中

### 注意
- **字符串用单引号**：`'Perryridge'`，不能用双引号（虽然有些数据库兼容，但标准是单引号）
- `and` 表示两个条件**同时满足**

---

## 第3页：where 子句支持的条件类型（重要！）

这一页是**条件类型的汇总表**，以后写查询时经常要回头查：

| 条件类型 | 操作符 | 例子 |
|----------|--------|------|
| 比较 | `=`, `>`, `<`, `>=`, `<=`, `!=`, `<>` | `age > 18` |
| 范围 | `BETWEEN AND`, `NOT BETWEEN AND` | `amount BETWEEN 1000 AND 5000` |
| 集合 | `IN`, `NOT IN` | `city IN ('北京', '上海')` |
| 字符匹配 | `LIKE`, `NOT LIKE` | `name LIKE '张%'` |
| 空值判断 | `IS NULL`, `IS NOT NULL` | `phone IS NULL` |
| 逻辑组合 | `AND`, `OR`, `NOT` | `age>18 AND city='北京'` |

### 几点说明
- `!=` 和 `<>` **都表示不等于**，两种写法都可以
- `BETWEEN 90000 AND 100000` 包含边界（≥ 且 ≤）
- `IN` 相当于多个 `OR` 的简写，后面会细讲
- `LIKE` 用于模糊匹配，`%` 代表任意多个字符，`_` 代表单个字符
- **判断空值必须用 `IS NULL`**，不能用 `= NULL`（这是新手常犯的错误）

---

## 第4页：BETWEEN 的例子

### 需求
找出贷款金额在 **$90,000 到 $100,000 之间**（包含两端）的贷款号。

### 写法一：使用 BETWEEN（推荐，简洁）
```sql
select loan_number
from loan
where amount between 90000 and 100000;
```

### 写法二：不用 BETWEEN（等价）
```sql
select loan_number
from loan
where amount >= 90000 and amount <= 100000;
```

两种写法**完全等价**，但 `BETWEEN` 更简洁易读。

### NOT BETWEEN 用法
```sql
-- 金额不在 90000 到 100000 之间的
where amount not between 90000 and 100000
```

等价于：
```sql
where amount < 90000 or amount > 100000
```

## 本节小结

| 知识点 | 要点 |
|--------|------|
| where 作用 | 筛选行（对应关系代数的 σ） |
| 字符串 | 用单引号括起来 |
| 条件组合 | 用 AND、OR、NOT |
| BETWEEN | 包含边界，是 >= 且 <= 的简写 |
| 常见错误 | 空值判断要用 IS NULL，不是 = NULL |
