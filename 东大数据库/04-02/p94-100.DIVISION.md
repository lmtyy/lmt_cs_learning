这几页是 **SQL 中的除法（Division）** 问题，也是整个课件中**最难**的部分。它解决的是 **“选了全部课程”** 这类带 **“所有”** 的问题。

你刚才理解了相关子查询的原理，现在正好可以用上。核心技巧就是：**用“双重否定”来表示“全部”**。

---

### 第1页：问题的本质

**需求**：查询选修了**全部**课程的学生姓名。

**表结构**：
- **students**：学生
- **course**：课程
- **sc**：选课成绩表

**核心难题**：SQL 里没有“全称量词”（∀，表示“所有”），只能用 **存在量词（∃，表示“存在”）** 来表达。

---

### 第2页：核心数学转换（关键！）

这是解决这类问题的**核心公式**：

> **“选了全部课程”** 
> 
> 等价于 **“不存在一门课程，该学生没选”**

用逻辑符号表示：

> ∀y P(y)  ≡  ¬ ∃y ( ¬ P(y) )
> 
> 所有都满足  ≡  不存在（一个不满足的）

**套用到本题**：
- **P(课程)**：学生选了这门课
- **¬P(课程)**：学生**没选**这门课

**转换后**：
> **选修了全部课程** 
> 
> 等价于 **不存在（一门课程，该学生没选）**

---

### 第3页：用 SQL 实现（双重 NOT EXISTS）

这就是经典的 **“除法” SQL 模板**：

```sql
SELECT sname
FROM students
WHERE NOT EXISTS          -- 不存在
    (SELECT *
     FROM course
     WHERE NOT EXISTS     -- 一门课程，该学生没选
         (SELECT *
          FROM sc
          WHERE sno = students.sno
            AND cno = course.cno));
```

#### 执行逻辑（用相关子查询的思路）

想象外层循环遍历每个学生，问一个问题：**“是否存在一门课程，这个学生没选？”**

以学生 **08001**（张小红）为例：

1.  外层判断 `NOT EXISTS`，所以先看子查询 `SELECT * FROM course WHERE NOT EXISTS (...)`
2.  这个子查询会遍历 **course** 表里的每一门课。
    -   对于 **001 号课程**：内层 `NOT EXISTS` 的子查询问：“张小红选过 001 吗？” → **选过** → 内层 `NOT EXISTS` 为 **假** → 这门课 **不满足** `NOT EXISTS`
    -   对于 **002 号课程**：问：“张小红选过 002 吗？” → **选过** → 内层 `NOT EXISTS` 为 **假** → 这门课 **不满足**
    -   对于 **003 号课程**：问：“张小红选过 003 吗？” → **没选过** → 内层 `NOT EXISTS` 为 **真** → 这门课 **满足** `NOT EXISTS`
3.  只要 **course** 表里有 **任何一门** 课满足 `NOT EXISTS`，说明“存在一门课没选”，那么中间的 `SELECT * FROM course WHERE ...` 结果就 **非空**。
4.  最外层的 `NOT EXISTS` 看到子查询结果**非空**，就返回 **假**。所以张小红 **不满足** 条件。

学生 **08003**（小刚）选过 002，没选 001 和 003，因此也会被筛掉。

**最终结果**：只有选了 **course** 表里**所有课程**的学生才会被留下。

### 第4页：另一种写法（用 EXCEPT）

同样的逻辑，另一种更直观的表达方式：

```sql
SELECT DISTINCT sname
FROM students
WHERE NOT EXISTS
    ((SELECT cid FROM course)          -- 所有课程
     EXCEPT                             -- 减去
     (SELECT cid FROM sc WHERE students.sid = sc.sid) -- 该学生选的课程
    );
```

-   `所有课程 EXCEPT 该生选的课`：得到该生**没选**的课程列表。
-   `NOT EXISTS`：如果这个列表是空的（即不存在没选的课），那么该生满足条件。

### 第5-7页：变体问题

**需求**：查询至少选修了 **学生95002** 选修的**全部课程**的学生学号。

**分析**：从“选修了所有课程”变成了“选修了特定学生（95002）所选修的所有课程”。

**逻辑转换**：对于学生 X，**不存在**这样一门课程：95002 选了它，但 X 没选它。

#### 写法1：用 NOT EXISTS

```sql
SELECT Sno
FROM SC X
WHERE NOT EXISTS
    (SELECT *
     FROM SC Y
     WHERE Sno = '95002'
       AND NOT EXISTS
       (SELECT *
        FROM SC Z
        WHERE Sno = X.Sno AND Cno = Y.Cno));
```

#### 写法2：用 EXCEPT（更直观）

```sql
SELECT Sno
FROM SC X
WHERE NOT EXISTS
    ((SELECT cid FROM SC Y WHERE Sno = '95002')  -- 95002 选的课程
     EXCEPT
     (SELECT cid FROM SC Z WHERE Sno = X.Sno));   -- 该学生选的课程
```

### 总结：除法的解题模板

以后遇到 **“选了所有/全部”** 这类问题，直接套用这个模板：

```sql
-- 核心公式：不存在（某个应该满足的条件，但它不满足）

SELECT 目标列
FROM 主表
WHERE NOT EXISTS
    (SELECT *
     FROM 全量集合表
     WHERE NOT EXISTS
         (SELECT *
          FROM 关系表
          WHERE 主表.关联键 = 关系表.关联键
            AND 全量集合表.关联键 = 关系表.关联键));
```

如果你觉得这个逻辑还是有些绕，我们可以用一个具体的数字例子，把这个“双重循环”的执行过程完整推演一遍。
