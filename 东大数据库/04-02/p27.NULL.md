好的，这一页讲的是 **NULL 值（空值）** 的处理，这是 SQL 中一个非常重要但又容易出错的概念。

---

## 核心概念：什么是 NULL？

- **NULL** 表示**值未知**或**不存在**，不是 0，也不是空字符串，而是"没有值"
- 比如：学生选修了课程，但**没有参加考试**，所以成绩是 NULL（不是 0 分，0 分是考了但得 0 分）

---

## 关键规则：IS NULL / IS NOT NULL

### 错误写法 ❌
```sql
WHERE Grade = NULL      -- 错误！永远查不到任何数据
WHERE Grade != NULL     -- 错误！也查不到任何数据
```

### 正确写法 ✅
```sql
WHERE Grade IS NULL      -- 查询成绩为空的学生
WHERE Grade IS NOT NULL  -- 查询成绩不为空的学生
```

---

## 为什么不能用 `=` 判断 NULL？

因为 NULL 代表"未知"，两个未知值不一定相等，数据库无法判断 `NULL = NULL` 是真是假，所以**任何 NULL 与 NULL 的比较都返回 "未知"（不是真，也不是假）**，不会出现在结果中。

> 一个简单记法：NULL 就像"不知道"，你不知道的东西，怎么能说它等于另一个不知道的东西呢？

---

## 例子1：查询缺考的学生（成绩为 NULL）

### 场景
SC 表（选课表）：`Sno（学号）, Cno（课程号）, Grade（成绩）`

有些学生选了课，但没有考试成绩，Grade 字段是 NULL。

### SQL
```sql
SELECT Sno, Cno
FROM SC
WHERE Grade IS NULL;
```

**结果**：返回所有缺考学生的学号和课程号。

---

## 例子2：查询有成绩的学生（成绩不为 NULL）

```sql
SELECT Sno, Cno
FROM SC
WHERE Grade IS NOT NULL;
```

**结果**：返回所有正常参加了考试、有成绩（包括 0 分）的学生。

---

## 重要补充：NULL 参与运算

NULL 和任何值做算术运算，结果还是 NULL：

```sql
-- 假设某行的 Grade 是 NULL
SELECT Grade + 10 FROM SC;   -- 结果是 NULL
```

NULL 和任何值做比较（`>`, `<`, `=` 等），结果都是"未知"，不会出现在 WHERE 的结果中。

---

## 常见错误总结

| 错误写法 | 问题 | 正确写法 |
|----------|------|----------|
| `Grade = NULL` | 永远为假 | `Grade IS NULL` |
| `Grade != NULL` | 永远为假 | `Grade IS NOT NULL` |
| `NULL = NULL` | 结果未知 | 直接用 `IS NULL` |
