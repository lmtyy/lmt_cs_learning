好的，这一节讲的是 **LIKE 字符串匹配操作**，用于模糊查询。

---

## 第1页：LIKE 的作用

- SQL 提供了 `LIKE` 操作符，用于**对字符串进行模式匹配**
- 当你**不知道完整的字符串**，只知道一部分特征时，就可以用 LIKE

---

## 第2页：固定字符串匹配（和 = 等价）

### 需求
查询学号为 `200215121` 的学生的所有信息。

### 写法一：用 LIKE
```sql
SELECT *
FROM Student
WHERE Sno LIKE '200215121';
```

### 写法二：用等号
```sql
SELECT *
FROM Student
WHERE Sno = '200215121';
```

### 结论
如果 LIKE 后面没有通配符（`%` 或 `_`），它的效果**完全等价于** `=`。

> 所以平时精确查询直接用 `=` 更简洁，LIKE 主要用于模糊匹配。

---

## 第3页：通配符 `%`（百分号）

### 含义
`%` 代表 **任意长度的字符串**（长度可以是 0）

### 例子
模式 `'a%b'` 的含义：
- 以 `a` 开头
- 以 `b` 结尾
- 中间可以是任意字符、任意长度

### 能匹配的字符串
| 字符串 | 是否匹配 |
|--------|----------|
| `ab` | ✅（中间长度为 0） |
| `acb` | ✅ |
| `addgb` | ✅ |
| `a12345b` | ✅ |
| `b` | ❌（不是 a 开头） |
| `abc` | ❌（不是 b 结尾） |

### 常见用法示例
```sql
-- 查询姓张的学生（张开头，后面任意）
WHERE Sname LIKE '张%'

-- 查询名字中包含“伟”的学生
WHERE Sname LIKE '%伟%'

-- 查询邮箱是 163.com 结尾的
WHERE email LIKE '%@163.com'
```

---

## 第4页：通配符 `_`（下划线）

### 含义
`_` 代表 **任意单个字符**（且必须恰好一个，不能多也不能少）

### 例子
模式 `'a_b'` 的含义：
- 以 `a` 开头
- 以 `b` 结尾
- 中间**恰好一个**任意字符
- 整个字符串长度 = 3

### 能匹配的字符串
| 字符串 | 是否匹配 | 原因 |
|--------|----------|------|
| `acb` | ✅ | a _ b，中间是 c |
| `afb` | ✅ | a _ b，中间是 f |
| `a_b` | ✅ | 中间是 _ 这个字符本身 |
| `ab` | ❌ | 长度只有 2，中间没有字符 |
| `acdb` | ❌ | 长度是 4，中间有两个字符 |

### 多个下划线
```sql
-- 查询名字是三个字、并且姓“张”的学生
WHERE Sname LIKE '张__'   -- 两个下划线，表示后面恰好两个字

-- 查询手机号第4位是5的（假设手机号11位，用11个下划线... 太麻烦，实际会结合其他函数）
```

---

## % 和 _ 的对比

| 通配符 | 含义 | 匹配长度 | 例子 |
|--------|------|----------|------|
| `%` | 任意字符串 | 0 个或多个字符 | `'张%'` 匹配张、张三、张三丰 |
| `_` | 任意单个字符 | 恰好 1 个字符 | `'张_'` 匹配张三，但不匹配张、张三丰 |
