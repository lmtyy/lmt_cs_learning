  这几页PPT系统地介绍了SQL中的**数据类型（Data Type）**，包括数值类型、字符类型、布尔类型和日期类型，并结合示例展示了如何在`CREATE TABLE`语句中使用这些类型。下面将详细讲解每一部分的知识点。

---

## 一、数据类型概述

在关系数据库中，每个列（字段）都必须指定数据类型，它决定了：
- 该列可以存储什么类型的值（整数、字符串、日期等）；
- 存储格式和长度；
- 允许的运算和函数；
- 比较规则（如大小写敏感）。

  PPT中列出了四大类：
- **Numeric**（数值）
- **Character String**（字符串）
- **Boolean**（布尔）
- **Date**（日期）

---

## 二、数值类型（Numeric Types）

数值类型分为**整数**和**浮点数（含定点数）**两类。

### 1. 整数类型（Integer）

标准SQL定义了三种整数类型：
- `INTEGER` 或 `INT`：普通整数（通常4字节）
- `SMALLINT`：小范围整数（通常2字节）
- `BIGINT`：大范围整数（通常8字节）

但不同的数据库系统会提供额外的整数类型，例如PPT中展示了**MySQL的整数类型**，包括：

| 类型      | 字节数 | 有符号最小值 | 无符号最小值 | 有符号最大值 | 无符号最大值 |
| --------- | ------ | ------------ | ------------ | ------------ | ------------ |
| TINYINT   | 1      | -128         | 0            | 127          | 255          |
| SMALLINT  | 2      | -32768       | 0            | 32767        | 65535        |
| MEDIUMINT | 3      | -8388608     | 0            | 8388607      | 16777215     |
| INT       | 4      | -2147483648  | 0            | 2147483647   | 4294967295   |
| BIGINT    | 8      | -2^63        | 0            | 2^63-1       | 2^64-1       |

- **有符号**：可存储负数，范围对称（除0外）。
- **无符号**：只存储非负数，范围上限更大。
- 选择整数类型时，应根据实际数值范围决定，以节省存储空间并提高性能。

### 2. 浮点数与定点数类型

PPT中介绍了以下几种：

#### ① `DECIMAL[(M[,D])]`
- **精确数值类型**，用于存储精确的小数，如货币金额。
- `M` 表示**总位数**（精度，precision），`D` 表示**小数位数**（标度，scale）。
- 例如 `DECIMAL(5,2)` 表示总共5位数字，其中小数部分占2位，整数部分占3位，取值范围 `-999.99` 到 `999.99`。
- 数据库会以字符串或压缩十进制形式精确存储，不会出现浮点舍入误差。

#### ② `FLOAT(p)`
- **近似数值类型**，用于存储浮点数。
- `p` 表示**尾数精度**（mantissa precision），即有效数字的位数。但不同DBMS对`p`的解释可能不同（例如MySQL中`FLOAT`使用4字节，`DOUBLE`使用8字节）。
- 由于是近似存储，进行相等比较时需谨慎。

#### ③ `REAL` 和 `DOUBLE PRECISION`
- `REAL`：通常对应单精度浮点数（4字节），精度由DBMS定义。
- `DOUBLE PRECISION`：通常对应双精度浮点数（8字节），精度高于`REAL`。
- 它们本质上也是浮点类型，但标准规定精度由具体系统决定。

**注意**：`DECIMAL`是精确类型，而`FLOAT`/`REAL`/`DOUBLE`是近似类型，在需要精确计算（如财务）时应使用`DECIMAL`。

### 3. 示例：`CREATE TABLE test`（数值部分）

```sql
CREATE TABLE test (
    name VARCHAR(100),          -- 字符串类型，稍后讲解
    col1 DECIMAL(5,2),           -- 精确小数，共5位，小数2位
    col2 SMALLINT,               -- 小范围整数
    col3 INTEGER,                 -- 普通整数
    col4 BIGINT,                   -- 大范围整数
    col5 FLOAT(2),                 -- 浮点数，精度参数p=2（意义依赖DBMS）
    col6 REAL,                       -- 单精度浮点
    col7 DOUBLE PRECISION   -- 双精度浮点
);
```

**说明**：
- `DECIMAL(5,2)` 可存储类似 `236.95` 的值（整数部分3位，小数2位）。
- `SMALLINT`、`INTEGER`、`BIGINT` 分别适用于不同范围的整数。
- `FLOAT(2)` 中的 `2` 表示要求至少保留2位有效数字？实际上在标准中`p`是指二进制精度，但很多DBMS将其解释为十进制显示宽度，具体需参考手册。
- `REAL` 和 `DOUBLE PRECISION` 的精度由系统决定。

## 三、字符字符串类型（Character String Types）

字符串值必须用**单引号**括起来（例如 `'Hello'`）。PPT将字符类型分为**ASCII字符类型**和**Unicode字符类型**。

### 1. ASCII字符类型（非Unicode）

- **`CHAR(n)`**：**定长字符串**。固定占用 `n` 个字符的存储空间，如果实际字符串长度小于 `n`，则右侧用空格填充。适用于长度固定的数据，如国家代码、性别等。
- **`VARCHAR(n)`**：**可变长字符串**。最多可存储 `n` 个字符，实际存储时只占用实际长度（加上长度前缀）。适用于长度变化较大的数据，如姓名、地址。
- **`CLOB`**（Character Large Object）：**字符大对象**，用于存储超长文本（如文章、日志）。具体最大长度由DBMS定义（例如MySQL中的`LONGTEXT`）。

### 2. Unicode字符类型

为了支持多语言（如中文、日文、阿拉伯文），数据库提供了Unicode版本的类型，名称前通常加 **`N`**（National Character Set）：
- **`NCHAR(n)`**：定长Unicode字符串。
- **`NVARCHAR(n)`**：可变长Unicode字符串。
- **`NCLOB`**：Unicode字符大对象。

这些类型在存储时使用Unicode编码（如UTF-16或UTF-8），避免字符集转换问题。使用时同样需要单引号。

### 3. 示例：`CREATE TABLE test`（字符部分）

```sql
CREATE TABLE test (
    col1 CHAR(8),        -- 固定8个字符，不足补空格
    col2 VARCHAR(100),   -- 最多100个字符，按实际长度存储
    col3 CLOB            -- 存储大量文本
);
```

**注意**：
- 在MySQL中，`CLOB`通常对应`TEXT`类型家族（`TINYTEXT`, `TEXT`, `MEDIUMTEXT`, `LONGTEXT`）。
- 对于Unicode版本，可以使用`NCHAR`、`NVARCHAR`，但许多现代数据库默认字符集已支持Unicode（如UTF-8），所以普通`CHAR`/`VARCHAR`也能存储中文，但排序规则可能不同。具体根据DBMS设置。

## 四、布尔类型（Boolean）

标准SQL定义了`BOOLEAN`类型，可存储`TRUE`、`FALSE`或`UNKNOWN`（即NULL）。但并非所有数据库都原生支持：
- PostgreSQL、MySQL（5.7+）支持`BOOLEAN`（MySQL中实际映射为`TINYINT(1)`）。
- SQL Server使用`BIT`类型（0/1）模拟布尔。
- Oracle中没有单独的布尔类型，通常用`NUMBER(1)`或`CHAR(1)`代替。

使用时可用`TRUE`、`FALSE`字面量，例如：
```sql
SELECT * FROM users WHERE is_active = TRUE;
```

---

## 五、日期类型（Date）

标准SQL定义了多种日期时间类型，常见的有：
- **`DATE`**：仅存储日期（年-月-日），例如 `'2026-03-12'`。
- **`TIME`**：仅存储时间（时:分:秒），例如 `'14:30:00'`。
- **`TIMESTAMP`** 或 **`DATETIME`**：同时存储日期和时间，有的还带时区。
- **`INTERVAL`**：表示时间间隔。

不同数据库对日期类型的支持和格式略有差异，但都提供相应的函数进行处理。

---

## 六、总结

- 选择合适的数据类型对数据库性能和存储效率至关重要。
- **整数**：根据数值范围选择`SMALLINT`、`INTEGER`、`BIGINT`或数据库扩展类型（如`TINYINT`）。
- **小数**：需要精确计算时用`DECIMAL`，否则可用`FLOAT`/`DOUBLE`。
- **字符串**：定长用`CHAR`，变长用`VARCHAR`，超长文本用`CLOB`；多语言环境考虑`NCHAR`/`NVARCHAR`。
- **布尔**和**日期**类型可根据实际需求选用，但需注意数据库间的差异。

理解这些数据类型，可以帮助你正确设计表结构，避免数据截断、精度丢失或存储浪费。在实际创建表时，还需查阅所用数据库的官方文档，了解具体类型名称、范围及特性。
