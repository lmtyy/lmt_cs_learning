这两页PPT介绍了SQL中**数据定义语言（DDL）**的两个核心操作：创建数据库和创建表。DDL用于定义、修改和删除数据库中的结构（模式），它描述的语句一旦执行，就会在数据库系统中持久化存储这些定义，并成为后续数据操作（DML）的依据。下面分别深入讲解这两部分内容。

---

### 一、创建数据库：`CREATE DATABASE`

**基本语法**  
```sql
CREATE DATABASE database_name;
```
例如：`CREATE DATABASE soft2018;`

#### 1. 功能与作用
- 在数据库管理系统（DBMS）中创建一个新的**逻辑容器**，用于存放后续创建的表、视图、索引、存储过程等对象。
- 在物理层面，DBMS会为该数据库分配独立的存储空间（如文件系统中的目录或文件），并记录数据库的元数据（如名称、字符集、所有者等）到系统目录中。

#### 2. 命名规则与注意事项
- 数据库名称必须唯一，不能与已有数据库重名。
- 名称通常区分大小写取决于操作系统（如Windows下不区分，Linux下区分），建议使用小写字母、数字和下划线，避免使用SQL保留字（如`CREATE`、`TABLE`等）。
- 不同DBMS对名称长度的限制不同（例如MySQL最多64字符，Oracle最多30字符）。

#### 3. 可选的扩展子句
许多DBMS允许在创建数据库时指定更详细的属性，常见的有：
- **字符集和排序规则**  
  ```sql
  CREATE DATABASE soft2018
    CHARACTER SET utf8mb4
    COLLATE utf8mb4_general_ci;
  ```
  这决定了该数据库中字符串的默认存储编码和比较方式，对国际化应用至关重要。
- **存储设置**（以MySQL为例）  
  ```sql
  CREATE DATABASE soft2018
    DEFAULT ENCRYPTION='Y';   -- 启用透明数据加密
  ```
  也可以指定数据文件存放路径（某些数据库支持），但通常由DBMS自动管理。

#### 4. 相关管理命令
- 查看现有数据库：`SHOW DATABASES;`（MySQL）或 `SELECT name FROM sys.databases;`（SQL Server）。
- 切换到指定数据库：`USE soft2018;`（MySQL、PostgreSQL等）。
- 删除数据库：`DROP DATABASE soft2018;`（谨慎操作，会永久删除所有对象）。

#### 5. 为什么要显式创建数据库？
- 实现**逻辑隔离**：不同的应用或项目可以放在不同的数据库中，避免对象名冲突。
- 便于**权限管理**：可以针对整个数据库授予访问权限。
- 便于**备份和恢复**：大多数DBMS支持按数据库进行备份。

### 二、创建表：`CREATE TABLE`

**基本语法**  
```sql
CREATE TABLE Students (
    sid  CHAR(8),
    name CHAR(20),
    age  INT
);
```

#### 1. 定义关系模式
- 表名：`Students`，遵循与数据库类似的命名规则。
- 列定义：每个列包括名称和数据类型，这是关系模型中的**属性**及其**域**的具体体现。
- **域（Domain）** 在这里由数据类型实现，它限定了列可以存储的值的集合。例如 `INT` 只能存储整数，`CHAR(8)` 只能存储长度不超过8个字符的字符串。

#### 2. 列的数据类型详解
- **`sid CHAR(8)`**：固定长度字符串，占用8个字符的空间，无论实际存储内容长短（不足8字符时会在尾部用空格填充）。适合存储学号、身份证号等长度固定的编码。
- **`name CHAR(20)`**：同理，但若姓名长度变化较大，使用 `VARCHAR(20)` 更节省存储空间（只占用实际长度+1或2字节）。PPT中使用 `CHAR` 可能是为了简化教学。
- **`age INT`**：整数类型，通常占用4字节，取值范围约 -21亿~21亿。对于年龄，也可以使用 `SMALLINT` 或 `TINYINT` 以节省空间（取决于DBMS）。

#### 3. 其他常见数据类型
- **数值类型**：`DECIMAL(10,2)`（定点数，适合金额）、`FLOAT`、`DOUBLE`。
- **字符串类型**：`VARCHAR(n)`（可变长度，n为最大字符数）、`TEXT`（长文本）、`BLOB`（二进制大对象）。
- **日期时间类型**：`DATE`（仅日期）、`TIME`（仅时间）、`DATETIME`（日期和时间）、`TIMESTAMP`（带时区的时间戳）。
- **布尔类型**：`BOOLEAN`（某些DBMS用`TINYINT(1)`代替）。
- **特殊类型**：`JSON`（MySQL 5.7+）、`XML`（SQL Server）等。

#### 4. 完整性约束的强制执行
PPT中提到“**The type (domain) of each field is specified, and enforced by the DBMS whenever tuples are added or modified.**” 这是DDL的核心作用之一：DBMS会强制保证数据的域完整性。
- **类型检查**：如果尝试将字符串 `'abc'` 插入 `age` 列，DBMS会拒绝操作并返回类型转换错误。
- **长度检查**：如果向 `sid CHAR(8)` 插入超过8个字符的字符串，可能会被截断（或报错，取决于SQL模式）。
- **空值约束**：默认情况下，所有列都允许 `NULL`，除非显式加上 `NOT NULL`。
- **其他约束**：可以在列定义后附加，例如：
  ```sql
  sid  CHAR(8) PRIMARY KEY,                -- 主键（隐含非空且唯一）
  name CHAR(20) NOT NULL,                   -- 非空约束
  age  INT CHECK (age >= 0 AND age <= 150), -- 检查约束
  ```
  这些约束由DBMS在插入、更新时自动检查，确保数据满足业务规则。

#### 5. 数据字典（元数据存储）
执行 `CREATE TABLE` 后，DBMS会将表的结构信息存入**数据字典**（也称系统目录）。数据字典是一组特殊的表，记录了所有数据库对象的定义。例如：
- 表名、列名、数据类型、是否可为空、默认值等。
- 索引、约束的定义。
- 当用户查询时，DBMS会从数据字典中获取模式信息，解析并优化SQL语句。

#### 6. 物理存储的隐含意义
尽管PPT未展开，但创建表也隐含着为表分配物理存储结构（如数据页、索引页）。不同的存储引擎（如MySQL的InnoDB、MyISAM）在磁盘上组织数据的方式不同，例如：
- InnoDB按主键聚簇存储数据。
- 可以指定存储参数，如 `ENGINE=InnoDB`、`ROW_FORMAT=DYNAMIC` 等。

#### 7. 扩展：修改表结构（ALTER）
DDL还包括 `ALTER TABLE` 语句，用于添加、修改或删除列，以及添加/删除约束。例如：
```sql
ALTER TABLE Students ADD COLUMN email VARCHAR(50);
ALTER TABLE Students MODIFY age SMALLINT;
```
这些操作也会立即更新数据字典，并可能触发数据重组。

---

### 总结
这两页PPT简明扼要地展示了SQL DDL的基本功能：
1. **`CREATE DATABASE`** 建立数据库环境，为后续对象提供容器。
2. **`CREATE TABLE`** 定义表的结构，通过数据类型和约束实现数据完整性，并由DBMS在运行时强制执行。

理解DDL是学习数据库设计和管理的第一步，它不仅是语法问题，更涉及到数据模型、完整性策略、存储优化和安全性等深层次概念。
