作为一名计算机专业的学霸，你可能已经接触过一些 SQL 语句，但课件从学术和系统的角度为你梳理了它的前世今生和基本构成。我为你总结了其中的核心要点：

---

## 1. SQL 的历史演进
SQL 的诞生极大地改变了数据管理的方式：
* **起源：** 它最初由 IBM 在 **System R** 项目中开发，原名叫 **SEQUEL**。
* **标准化：** 为了防止各大数据库厂商（如 Oracle, Microsoft, IBM）各自为政，ANSI 和 ISO 制定了标准（如 SQL-92, SQL:1999 等）。
* **现状：** 虽然有标准，但不同系统（MySQL, PostgreSQL, SQL Server）仍有各自的**方言**（Proprietary features）。这意味着你在课件里学的标准语法，在实际使用的特定数据库中可能会有细微差别。

---

## 2. SQL 的主要特点
课件强调了 SQL 的两个重要属性：
* **非过程化 (Non-procedure)：** 这是 SQL 与 C++ 或 Python 最本质的区别。你只需要告诉数据库“我要什么”（What to do），而不需要告诉它“怎么去拿”（How to do）。底层的搜索路径由数据库优化器决定。
* **简单易学：** 核心功能仅由 9 个关键词驱动，极其精炼。

---

## 3. SQL 的三大组成部分（重点）
这是你考试和日常开发中最常接触的分类方法：

| 类别 | 全称 | 核心功能 | 常用关键词 |
| :--- | :--- | :--- | :--- |
| **DDL** | Data Definition Language (数据定义语言) | 定义数据库的**结构**（表长什么样） | `CREATE`, `DROP`, `ALTER` |
| **DML** | Data Manipulation Language (数据操纵语言) | 对表里的**数据**进行增删改查 | `SELECT`, `INSERT`, `DELETE`, `UPDATE` |
| **DCL** | Data Control Language (数据控制语言) | 管理**权限**和安全性 | `GRANT`, `REVOKE` |


---

## 4. 深入理解 DDL (数据定义语言)
DDL 不仅仅是简单的建表，它负责规定数据的“游戏规则”，包括：
* **Schema (模式)：** 规定每个关系（表）的逻辑结构。
* **Domain (域)：** 规定属性的取值范围（比如年龄必须是整数）。
* **Integrity Constraints (完整性约束)：** 比如主键唯一、外键关联等，保证数据不乱套。
* **Security (安全性)：** 谁有权看这张表。
* **Physical Storage (物理存储)：** 决定数据在磁盘上如何存放（虽然作为程序员我们通常只关注逻辑层）。
