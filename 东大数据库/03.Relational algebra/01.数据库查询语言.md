## 数据库查询语言 (Query Languages) 核心知识点

### 1. 定义

**查询语言**是用户用来从数据库中请求、获取或操作信息的专用语言。

------

### 2. 核心分类

根据操作逻辑的不同，查询语言主要分为两大类：

#### **A. 过程化语言 (Procedural Languages)**

- **逻辑：** 用户必须指定**“做什么 (What)”**以及**“怎么做 (How)”**。
- **特点：** 需要详细指明获取数据的具体步骤、算法和路径。
- **代表：** **关系代数 (Relational Algebra)**。它通过一系列运算（如选择、投影、连接）逐步构建查询结果。

#### **B. 非过程化语言 / 声明式语言 (Non-procedural / Declarative Languages)**

- **逻辑：** 用户只需指定**“做什么 (What)”**，无需关心**“怎么做 (How)”**。
- **特点：** 用户只需描述所需数据的特征，由数据库管理系统 (DBMS) 自动寻找最优的执行路径。
- **代表：** **SQL (Structured Query Language)** 和 **关系演算 (Relational Calculus)**。

------

### 3. 对比总结

| **维度**     | **过程化 (Procedural)** | **非过程化 (Non-procedural)** |
| ------------ | ----------------------- | ----------------------------- |
| **关注点**   | 步骤 + 结果             | 仅结果                        |
| **用户要求** | 需了解底层操作逻辑      | 只需描述需求                  |
| **执行效率** | 取决于用户编写的步骤    | 由 DBMS 优化器决定            |
| **典型实例** | 关系代数                | SQL                           |
