这两张内容都在讲**关系代数的差运算（Set Difference）**，下面完整展开。

---

## 1. 符号与读法

- **记号**：$r - s$
- **读法**：r 减 s，或"r 差 s"
- **直觉**：从 r 里"挖掉"那些同时也出现在 s 里的元组，剩下的就是结果。

---

## 2. 形式化定义

$$r - s = \{\, t \mid t \in r \ \text{and} \ t \notin s \,\}$$

- $t$ 是一个元组（一行）
- 含义：结果中只保留"在 r 里有，但在 s 里没有"的元组
- **不是对称的**：$r - s \neq s - r$（这一点极其重要，后面细讲）

---

## 3. 例子解读（第一张图）

给定两个关系 r 和 s，属性均为 $(A, B)$：

| r 的元组      | s 的元组      |
| ------------- | ------------- |
| $(\alpha, 1)$ | $(\alpha, 2)$ |
| $(\alpha, 2)$ | $(\beta, 3)$  |
| $(\beta, 1)$  |               |

做 $r - s$：
- $(\alpha, 1)$：在 r 里 ✅，不在 s 里 ✅ → **保留**
- $(\alpha, 2)$：在 r 里 ✅，也在 s 里 ❌ → **去掉**
- $(\beta, 1)$：在 r 里 ✅，不在 s 里 ✅ → **保留**

所以 $r - s$ 的结果为：$(\alpha, 1),\ (\beta, 1)$

> **注意**：s 里的 $(\beta, 3)$ 对结果没有任何贡献，因为差运算只从 r 里挑元组，s 里有什么不影响结果"增加"任何行。

---

## 4. 差运算成立的前提：并相容（Union-compatibility）

差运算和并运算一样，要求 r 和 s **并相容**：

1. **属性个数相同**：r 和 s 的列数必须一致
2. **对应属性域兼容**：每一列的数据类型/取值域要匹配
3. **属性名一致（或先重命名对齐）**：若列名不同，需先用 $\rho$ 重命名

> 直觉：你不能把"(学号, 姓名)"的表减去"(课程号, 学分)"的表，这没有意义。

---

## 5. 差运算的非对称性（最重要的特性）

$$r - s \neq s - r \quad \text{（一般情况下）}$$

以上面的例子验证：

$s - r$：
- $(\alpha, 2)$：在 s 里 ✅，也在 r 里 ❌ → **去掉**
- $(\beta, 3)$：在 s 里 ✅，不在 r 里 ✅ → **保留**

所以 $s - r = \{(\beta, 3)\}$，与 $r - s = \{(\alpha,1),(\beta,1)\}$ 完全不同。

> 这与加法、并运算的交换律形成鲜明对比，做题时要特别注意操作数的顺序。

---

## 6. 差运算的代数性质

| 性质       | 说明                                             |
| ---------- | ------------------------------------------------ |
| **非交换** | $r - s \neq s - r$                               |
| **非结合** | $(r-s)-t \neq r-(s-t)$（一般情况下）             |
| **与空集** | $r - \emptyset = r$；$\emptyset - r = \emptyset$ |
| **自差**   | $r - r = \emptyset$                              |
| **单调性** | $r - s \subseteq r$，结果一定是 r 的子集         |

---

## 7. 差运算可以表达"交集"（图中未提，但极重要）

关系代数的 6 个基本运算里**没有交运算**，但交集可以用差来推导：

$$r \cap s = r - (r - s)$$

**验证**（用上面的例子）：
- $r - s = \{(\alpha,1),(\beta,1)\}$
- $r - (r-s) = r - \{(\alpha,1),(\beta,1)\} = \{(\alpha,2)\}$
- 而 r 和 s 共同都有的元组恰好是 $(\alpha,2)$ ✅

> 这说明**差是比交更基础的运算**，交可以由差派生，所以教材只把差列为基本运算。

---

## 8. 差运算的典型应用场景

差运算天然适合表达**"不存在"、"没有"、"排除"**类的查询：

**场景 1：找出"选了课程A但没选课程B"的学生**
$$\Pi_{学号}(\sigma_{课程='A'}(选课)) - \Pi_{学号}(\sigma_{课程='B'}(选课))$$

**场景 2：找出"从未借过书"的读者**
$$\Pi_{读者ID}(读者) - \Pi_{读者ID}(借阅记录)$$

**场景 3：找出"所有部门都有员工，但某部门没有经理"的部门**
$$\Pi_{部门ID}(部门) - \Pi_{部门ID}(\sigma_{职位='经理'}(员工))$$

---

## 9. 差运算 vs SQL（对应关系）

| 关系代数 | SQL                             |
| -------- | ------------------------------- |
| $r - s$  | `(query_r) EXCEPT (query_s)`    |
| 同上     | `WHERE NOT EXISTS (SELECT ...)` |
| 同上     | `WHERE id NOT IN (SELECT ...)`  |

> **注意**：`EXCEPT` 在 PostgreSQL、SQL Server 中支持；MySQL 不直接支持 `EXCEPT`（需用 `NOT IN` 或 `NOT EXISTS` 替代）。

---

## 10. 易错点总结

| 易错点                       | 说明                                                         |
| ---------------------------- | ------------------------------------------------------------ |
| 顺序搞反                     | $r-s$ 和 $s-r$ 结果不同，不能随意调换                        |
| 忘记并相容前提               | 两表结构不同时差运算无意义                                   |
| 误以为结果会包含 s 的元素    | 结果永远是 r 的子集，s 只起"过滤"作用                        |
| 与 SQL 的 `NOT IN` 混用 NULL | SQL 中若子查询结果含 NULL，`NOT IN` 会返回空集，需改用 `NOT EXISTS` |
