这两张都在讲**选择运算（Select Operation）**，第一张给出例子，第二张给出完整的形式化定义和谓词的构成规则。下面完整展开。

---

# 一、符号与核心含义

- **记号**：$\sigma_p(r)$
- **下标 $p$**：选择谓词（selection predicate），即筛选条件
- **直觉**：按条件筛选**行**，列结构完全不变，相当于 SQL 的 `WHERE`

---

# 二、形式化定义

$$\sigma_p(r) = \{\, t \mid t \in r \ \text{and} \ p(t) \,\}$$

- $t$ 是关系 r 中的一个元组（一行）
- $p(t)$ 是谓词作用于 $t$ 的结果，为 **true** 时该行保留，为 **false** 时丢弃
- 结果仍然是一个关系（闭包性），可以继续参与其他运算

---

# 三、谓词 $p$ 的完整构成（第二张图的核心）

图中明确指出：$p$ 是一个**命题演算（propositional calculus）**中的公式，由若干**项（term）**通过逻辑连接词组合而成。

## 3.1 逻辑连接词

| 符号    | 含义          | 说明             |
| ------- | ------------- | ---------------- |
| $\land$ | **and（与）** | 两个条件同时满足 |
| $\lor$  | **or（或）**  | 至少一个条件满足 |
| $\lnot$ | **not（非）** | 条件取反         |

## 3.2 每个"项（term）"的形式

$$\langle \text{attribute} \rangle \ op \ \langle \text{attribute} \rangle \quad \text{或} \quad \langle \text{attribute} \rangle \ op \ \langle \text{constant} \rangle$$

其中 $op$ 是以下比较运算符之一：

$$=,\quad \neq,\quad >,\quad \geq,\quad <,\quad \leq$$

**两种项的区别**：

| 类型         | 示例    | 含义                           |
| ------------ | ------- | ------------------------------ |
| 属性 op 常量 | $D > 5$ | 列 D 的值大于常量 5            |
| 属性 op 属性 | $A = B$ | 列 A 的值等于同一行的列 B 的值 |

> **注意**：$A = B$ 是**同一行内两列之间的比较**，不是跨行比较，这与 SQL 的 JOIN 条件含义不同。

---

# 四、例子解读（第一张图）

关系 r 有属性 $(A, B, C, D)$，4 行：

| A        | B        | C    | D    |
| -------- | -------- | ---- | ---- |
| $\alpha$ | $\alpha$ | 1    | 7    |
| $\alpha$ | $\beta$  | 5    | 7    |
| $\beta$  | $\beta$  | 12   | 3    |
| $\beta$  | $\beta$  | 23   | 10   |

执行 $\sigma_{A=B \land D>5}(r)$，谓词是：**A=B 且 D>5**

逐行检查：

| 行                    | A=B？               | D>5？       | 两者都满足？ |
| --------------------- | ------------------- | ----------- | ------------ |
| $(\alpha,\alpha,1,7)$ | ✅ $\alpha=\alpha$   | ✅ $7>5$     | ✅ **保留**   |
| $(\alpha,\beta,5,7)$  | ❌ $\alpha\neq\beta$ | ✅           | ❌ 去掉       |
| $(\beta,\beta,12,3)$  | ✅                   | ❌ $3\not>5$ | ❌ 去掉       |
| $(\beta,\beta,23,10)$ | ✅                   | ✅ $10>5$    | ✅ **保留**   |

结果：

| A        | B        | C    | D    |
| -------- | -------- | ---- | ---- |
| $\alpha$ | $\alpha$ | 1    | 7    |
| $\beta$  | $\beta$  | 23   | 10   |

# 五、选择运算的基本特性

| 特性              | 说明                                       |
| ----------------- | ------------------------------------------ |
| **只过滤行**      | 列数与原关系完全相同，不增不减             |
| **行数 ≤ 原关系** | 满足条件的行才保留，行数只会减少或不变     |
| **闭包性**        | 结果仍是关系，可继续参与其他运算           |
| **不去重**        | 选择不会消除重复行（原来有几份就保留几份） |

---

# 六、选择运算的重要代数性质（图中未提，但极重要）

## 6.1 幂等性
$$\sigma_p(\sigma_p(r)) = \sigma_p(r)$$
对同一个条件重复选择，结果不变。

## 6.2 谓词合并（最常用的优化规则）
$$\sigma_{p_1}(\sigma_{p_2}(r)) = \sigma_{p_1 \land p_2}(r)$$
两层嵌套的选择可以合并成一个，减少扫描次数。

**例**：
$$\sigma_{D>5}(\sigma_{A=B}(r)) = \sigma_{A=B \land D>5}(r)$$

## 6.3 选择的交换律
$$\sigma_{p_1}(\sigma_{p_2}(r)) = \sigma_{p_2}(\sigma_{p_1}(r))$$
两个选择的顺序可以互换（结果相同）。

## 6.4 选择与笛卡尔积的分配（谓词下推）
当谓词 $p$ 只涉及 r 的属性时：
$$\sigma_p(r \times s) = \sigma_p(r) \times s$$

**意义**：先过滤 r，再做笛卡尔积，中间结果更小，这是**查询优化中"谓词下推（Predicate Pushdown）"**的理论基础。

## 6.5 选择与并的分配
$$\sigma_p(r \cup s) = \sigma_p(r) \cup \sigma_p(s)$$

## 6.6 选择与差的分配
$$\sigma_p(r - s) = \sigma_p(r) - \sigma_p(s)$$

# 七、谓词的更多写法示例

| 谓词                                    | 含义                           |
| --------------------------------------- | ------------------------------ |
| $\sigma_{A = 'alpha'}(r)$               | A 列等于常量 'alpha'           |
| $\sigma_{D \geq 10}(r)$                 | D 列大于等于 10                |
| $\sigma_{A = B \land C > 5}(r)$         | A=B 且 C>5（图中的例子）       |
| $\sigma_{A = B \lor D > 5}(r)$          | A=B 或 D>5（满足任一即可）     |
| $\sigma_{\lnot(A = B)}(r)$              | A 不等于 B                     |
| $\sigma_{C \geq 10 \land C \leq 20}(r)$ | C 在 10 到 20 之间（范围查询） |

---

# 八、SQL 对应

| 关系代数                                | SQL                                         |
| --------------------------------------- | ------------------------------------------- |
| $\sigma_{A=B \land D>5}(r)$             | `SELECT * FROM r WHERE A=B AND D>5`         |
| $\sigma_{A='alpha' \lor D<3}(r)$        | `SELECT * FROM r WHERE A='alpha' OR D<3`    |
| $\sigma_{\lnot(A=B)}(r)$                | `SELECT * FROM r WHERE A <> B`              |
| $\sigma_{C \geq 10 \land C \leq 20}(r)$ | `SELECT * FROM r WHERE C BETWEEN 10 AND 20` |

---

# 九、选择 vs 投影（两者经常混淆）

| 对比项   | 选择 $\sigma$                  | 投影 $\Pi$                   |
| -------- | ------------------------------ | ---------------------------- |
| 操作对象 | **行**（元组）                 | **列**（属性）               |
| 条件     | 谓词（布尔条件）               | 属性列表                     |
| 结果列数 | 与原关系**相同**               | **≤ 原关系**（只保留指定列） |
| 结果行数 | **≤ 原关系**（满足条件才保留） | 可能因去重而减少             |
| SQL 对应 | `WHERE`                        | `SELECT 列名`                |

---

# 十、易错点总结

| 易错点                    | 说明                                                         |
| ------------------------- | ------------------------------------------------------------ |
| 谓词中 $A=B$ 是列间比较   | 不是"A等于字符串B"，而是"A列的值等于B列的值"                 |
| 选择不改变列结构          | 结果的列与原关系完全一致，不会减少列                         |
| $\land$ 与 $\lor$ 混淆    | $\land$ 是"且"（更严格，结果更少）；$\lor$ 是"或"（更宽松，结果更多） |
| 忘记谓词下推              | 应尽早做选择，减少后续运算（尤其是笛卡尔积）的数据量         |
| 与 SQL WHERE 的 NULL 差异 | 经典关系代数不讨论 NULL；SQL 中 NULL 参与比较结果为 UNKNOWN，既不是 TRUE 也不是 FALSE，需特别处理 |
