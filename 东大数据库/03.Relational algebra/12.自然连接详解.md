这四张都在讲**自然连接（Natural Join）**，从定义、规则、抽象例子到实际银行场景完整呈现。下面全面展开。

---

# 一、自然连接的符号与定义（第一张图）

- **记号**：$r \bowtie s$（无下标）
- **结果的模式**：$R \cup S$（两张表属性的并集，去掉重复列）

## 形式化执行逻辑（图中逐步给出）

设 r 基于模式 R，s 基于模式 S，则 $r \bowtie s$ 的执行步骤：

1. **考察每一对元组** $t_r \in r$ 和 $t_s \in s$
2. **检查公共属性**（$R \cap S$，即同名属性）上的值是否完全相同
3. 若相同，则把 $t_r$ 和 $t_s$ **拼接成一个元组 $t$** 加入结果（公共属性只保留一份）
4. 若不同，则**丢弃**这对配对

---

# 二、自然连接的三个核心规则（第二张图）

图中用中文明确给出三条规则：

### 规则 1：自然连接是特殊的等值连接
> 自然连接是一种特殊的等值连接

- 等值连接需要**手动指定**连接条件
- 自然连接**自动**在所有同名属性上做等值连接
- 自然连接 ⊂ 等值连接 ⊂ θ-连接（包含关系）

### 规则 2：比较的分量必须是相同的属性组
> 要求两个关系中进行比较的分量必须是**相同的属性组**

- 连接键 = $R \cap S$（两个模式的交集，即所有同名属性）
- 若有多个同名属性，**全部**参与等值比较，缺一不可

### 规则 3：结果中去掉重复列（图中最重要的补充）
> 在结果中将相同的属性组去掉
> **一般连接是从行的角度进行计算，但自然连接还要从列的角度进行计算，消除重复的列。**

这句话是自然连接与等值连接最本质的区别：
- 等值连接：只从**行**的角度过滤（连接列出现两次）
- 自然连接：既从**行**过滤，又从**列**消除重复

# 三、抽象例子解读（第三张图）

图中给出：

**关系 r**（属性：A, B, C, D）：

| A        | B    | C        | D    |
| -------- | ---- | -------- | ---- |
| $\alpha$ | 1    | $\alpha$ | a    |
| $\beta$  | 2    | $\gamma$ | a    |
| $\gamma$ | 4    | $\beta$  | b    |
| $\alpha$ | 1    | $\gamma$ | a    |
| $\delta$ | 2    | $\beta$  | b    |

**关系 s**（属性：B, D, E）：

| B    | D    | E          |
| ---- | ---- | ---------- |
| 1    | a    | $\alpha$   |
| 3    | a    | $\beta$    |
| 1    | a    | $\gamma$   |
| 2    | b    | $\delta$   |
| 3    | b    | $\epsilon$ |

## 确定公共属性（连接键）

$R \cap S = \{B, D\}$（B 和 D 同时出现在两张表中）

→ 自然连接条件：**r.B = s.B 且 r.D = s.D**（两个条件同时满足）

## 逐行匹配

| r 的行                | r.B  | r.D  | 在 s 中找 B=? D=? | 匹配的 s 行                       |
| --------------------- | ---- | ---- | ----------------- | --------------------------------- |
| $(\alpha,1,\alpha,a)$ | 1    | a    | B=1,D=a           | s行1($\alpha$) ✅ s行3($\gamma$) ✅ |
| $(\beta,2,\gamma,a)$  | 2    | a    | B=2,D=a           | 无匹配 ❌                          |
| $(\gamma,4,\beta,b)$  | 4    | b    | B=4,D=b           | 无匹配 ❌                          |
| $(\alpha,1,\gamma,a)$ | 1    | a    | B=1,D=a           | s行1($\alpha$) ✅ s行3($\gamma$) ✅ |
| $(\delta,2,\beta,b)$  | 2    | b    | B=2,D=b           | s行4($\delta$) ✅                  |

## 结果（属性为 A, B, C, D, E，B 和 D 各只出现一次）

| A        | B    | C        | D    | E        |
| -------- | ---- | -------- | ---- | -------- |
| $\alpha$ | 1    | $\alpha$ | a    | $\alpha$ |
| $\alpha$ | 1    | $\alpha$ | a    | $\gamma$ |
| $\alpha$ | 1    | $\gamma$ | a    | $\alpha$ |
| $\alpha$ | 1    | $\gamma$ | a    | $\gamma$ |
| $\delta$ | 2    | $\beta$  | b    | $\delta$ |

共 **5 行**，结果模式为 $R \cup S = \{A, B, C, D, E\}$，B 和 D 各只出现一次。

> **注意**：r 中 $(\beta,2,\gamma,a)$ 和 $(\gamma,4,\beta,b)$ 因为在 s 中找不到 B 和 D 同时匹配的行，所以被**丢弃**。这是自然连接（内连接）的特性——不匹配的行不出现在结果中。

# 四、银行实际例子（第四张图）

**需求**：找出在银行有账户的所有客户姓名、账号和余额

**两张表**：

**customer 表**（客户-账号关系）：

| customer_name | account_number |
| ------------- | -------------- |
| Hayes         | A-102          |
| Johnson       | A-101          |
| Johnson       | A-201          |
| Jones         | A-217          |
| Lindsay       | A-222          |
| Smith         | A-215          |
| Turner        | A-305          |

**account 表**（账号详情）：

| account_number | branch_name | balance |
| -------------- | ----------- | ------- |
| A-101          | Downtown    | 500     |
| A-102          | Perryridge  | 400     |
| A-201          | Brighton    | 900     |
| A-215          | Mianus      | 700     |
| A-217          | Brighton    | 750     |
| A-222          | Redwood     | 700     |
| A-305          | Round Hill  | 350     |

## 图中给出的等值连接写法

$$\Pi_{\text{customer\_name, account.account\_number, balance}} \Big( \sigma_{\text{account.account\_number = customer.account\_number}} (\text{account} \times \text{customer}) \Big)$$

这是用**笛卡尔积 + 选择 + 投影**来实现等值连接的完整写法。

## 等价的自然连接写法（更简洁）

两张表的公共属性是 **account_number**，因此：

$$\Pi_{\text{customer\_name, account\_number, balance}} (\text{customer} \bowtie \text{account})$$

执行步骤：
1. 自动在 account_number 上做等值连接
2. 去掉重复的 account_number 列
3. 再投影取出需要的三列

## 结果

| customer_name | account_number | balance |
| ------------- | -------------- | ------- |
| Hayes         | A-102          | 400     |
| Johnson       | A-101          | 500     |
| Johnson       | A-201          | 900     |
| Jones         | A-217          | 750     |
| Lindsay       | A-222          | 700     |
| Smith         | A-215          | 700     |
| Turner        | A-305          | 350     |

Johnson 有两个账户（A-101 和 A-201），所以出现两行。

# 五、自然连接的重要补充知识

## 5.1 无公共属性时退化为笛卡尔积
若 $R \cap S = \emptyset$（没有同名属性），自然连接退化为笛卡尔积：
$$r \bowtie s = r \times s$$

## 5.2 所有属性都相同时退化为交集
若 $R = S$（两张表结构完全相同），自然连接退化为交运算：
$$r \bowtie s = r \cap s$$

## 5.3 自然连接的代数性质

| 性质           | 公式                                                         |
| -------------- | ------------------------------------------------------------ |
| **交换律**     | $r \bowtie s = s \bowtie r$（列顺序可能不同）                |
| **结合律**     | $(r \bowtie s) \bowtie t = r \bowtie (s \bowtie t)$          |
| **与选择分配** | $\sigma_\theta(r \bowtie s) = \sigma_\theta(r) \bowtie s$（θ 只涉及 r 的属性时） |

结合律非常重要：多表连接时可以**任意调整连接顺序**，查询优化器会选择代价最小的顺序。

## 5.4 自然连接会丢失不匹配的行（内连接语义）

自然连接是**内连接（Inner Join）**，不匹配的行直接丢弃。若需保留不匹配的行，需要**外连接**：

| 类型     | 说明                              | 符号    |
| -------- | --------------------------------- | ------- |
| 左外连接 | 保留左表所有行，右表无匹配填 NULL | $r ⟕ s$ |
| 右外连接 | 保留右表所有行，左表无匹配填 NULL | $r ⟖ s$ |
| 全外连接 | 两边都保留，无匹配填 NULL         | $r ⟗ s$ |

# 六、SQL 对应

| 关系代数      | SQL                                        |
| ------------- | ------------------------------------------ |
| $r \bowtie s$ | `FROM r NATURAL JOIN s`                    |
| 等价写法      | `FROM r JOIN s ON r.B = s.B AND r.D = s.D` |
| 左外连接      | `FROM r LEFT OUTER JOIN s ON ...`          |

> **实际开发中不推荐 `NATURAL JOIN`**：因为它依赖同名列自动匹配，若表结构变化（新增同名列），查询结果会悄悄改变，难以排查。生产环境应明确写出连接条件。

# 七、易错点总结

| 易错点                 | 说明                                             |
| ---------------------- | ------------------------------------------------ |
| 多个同名属性都参与连接 | 不能只选其中一个，全部同名属性都必须满足等值条件 |
| 不匹配的行被丢弃       | 自然连接是内连接，无法匹配的行不出现在结果中     |
| 结果列数 = \|R∪S\|     | 不是 \|R\|+\|S\|，重复列已去掉                   |
| 与等值连接混淆         | 等值连接保留重复列；自然连接去掉重复列           |
| 无公共属性时变笛卡尔积 | 使用前要确认两表确实有同名属性，否则结果行数爆炸 |
