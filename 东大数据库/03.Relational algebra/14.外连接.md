这五张都在讲**外连接（Outer Join）**，从定义、三种类型到完整的银行例子逐一展开。

---

# 一、为什么需要外连接（第二张图）

图中明确指出：

> **An extension of the join operation that avoids loss of information**
> （外连接是连接运算的扩展，用于**避免信息丢失**）

> **Computes the join and then adds tuples from one relation that does not match tuples in the other relation**
> （先计算连接，再把**不匹配的行**也加入结果，用 NULL 填充缺失值）

**内连接（自然连接）的问题**：不匹配的行直接丢弃，导致信息丢失。

**外连接的解决方案**：不匹配的行也保留，缺失的列用 **NULL** 填充。

---

# 二、数据准备（第一、三张图）

**loan 表**（贷款信息）：

| loan_number | branch_name    | amount   |
| ----------- | -------------- | -------- |
| L-170       | Downtown       | 3000     |
| L-230       | Redwood        | 4000     |
| **L-260**   | **Perryridge** | **1700** |

**borrower 表**（借款人信息）：

| customer_name | loan_number |
| ------------- | ----------- |
| Jones         | L-170       |
| Smith         | L-230       |
| **Hayes**     | **L-155**   |

**关键观察**：
- loan 中的 **L-260** 在 borrower 中**没有对应借款人**（无人借这笔贷款）
- borrower 中的 **Hayes（L-155）** 在 loan 中**没有对应贷款记录**（贷款号不存在）
- 这两条"孤立"的行在内连接中会被丢弃，外连接则会保留它们

---

# 三、内连接（自然连接）作为对比基准（第四张图上半部分）

$$\text{loan} \bowtie \text{borrower}$$

在 loan_number 上自然连接，**丢弃**不匹配的行：

| loan_number | branch_name | amount | customer_name |
| ----------- | ----------- | ------ | ------------- |
| L-170       | Downtown    | 3000   | Jones         |
| L-230       | Redwood     | 4000   | Smith         |

- L-260（无借款人）→ **丢弃**
- L-155/Hayes（无贷款记录）→ **丢弃**

结果只有 **2 行**，信息有损失。

---

# 四、三种外连接详解

## 4.1 左外连接（Left Outer Join）

**符号**：$\text{loan} ⟕ \text{borrower}$（符号左边有横线）

**规则**：
- 保留**左表（loan）的所有行**
- 右表（borrower）中能匹配的正常连接
- 右表中**无法匹配**的左表行，右边列填 **NULL**

**结果**（第四张图下半部分）：

| loan_number | branch_name    | amount   | customer_name |
| ----------- | -------------- | -------- | ------------- |
| L-170       | Downtown       | 3000     | Jones         |
| L-230       | Redwood        | 4000     | Smith         |
| **L-260**   | **Perryridge** | **1700** | ***null***    |

- L-260 在 borrower 中没有匹配 → 保留，customer_name 填 **null**
- Hayes/L-155 在 loan 中没有匹配 → **不保留**（右表不匹配的行不保留）

结果 **3 行**。

---

## 4.2 右外连接（Right Outer Join）

**符号**：$\text{loan} ⟖ \text{borrower}$（符号右边有横线）

**规则**：
- 保留**右表（borrower）的所有行**
- 左表（loan）中能匹配的正常连接
- 左表中**无法匹配**的右表行，左边列填 **NULL**

**结果**（第五张图上半部分）：

| loan_number | branch_name | amount     | customer_name |
| ----------- | ----------- | ---------- | ------------- |
| L-170       | Downtown    | 3000       | Jones         |
| L-230       | Redwood     | 4000       | Smith         |
| **L-155**   | ***null***  | ***null*** | **Hayes**     |

- Hayes/L-155 在 loan 中没有匹配 → 保留，branch_name 和 amount 填 **null**
- L-260 在 borrower 中没有匹配 → **不保留**（左表不匹配的行不保留）

结果 **3 行**。

---

## 4.3 全外连接（Full Outer Join）

**符号**：$\text{loan} ⟗ \text{borrower}$（两边都有横线）

**规则**：
- **两张表的所有行都保留**
- 能匹配的正常连接
- 左表不匹配的行：右边列填 NULL
- 右表不匹配的行：左边列填 NULL

**结果**（第五张图下半部分）：

| loan_number | branch_name    | amount     | customer_name |
| ----------- | -------------- | ---------- | ------------- |
| L-170       | Downtown       | 3000       | Jones         |
| L-230       | Redwood        | 4000       | Smith         |
| **L-260**   | **Perryridge** | **1700**   | ***null***    |
| **L-155**   | ***null***     | ***null*** | **Hayes**     |

- L-260 无匹配 → 保留，customer_name = null
- Hayes/L-155 无匹配 → 保留，branch_name 和 amount = null

结果 **4 行**，信息**完全无损失**。

# 五、四种连接结果对比总结

| 连接类型           | 符号      | 保留左表不匹配行 | 保留右表不匹配行 | 结果行数 |
| ------------------ | --------- | ---------------- | ---------------- | -------- |
| 内连接（自然连接） | $\bowtie$ | ❌                | ❌                | 2行      |
| 左外连接           | $⟕$       | ✅                | ❌                | 3行      |
| 右外连接           | $⟖$       | ❌                | ✅                | 3行      |
| 全外连接           | $⟗$       | ✅                | ✅                | 4行      |

# 六、NULL 的本质与影响（图中未展开，极重要）

外连接引入了 **NULL**，NULL 在关系数据库中有特殊语义：

## 6.1 NULL 的含义
- **未知值**（Unknown）：该值存在但不知道是什么
- **不适用**（Not Applicable）：该属性对此行不适用

## 6.2 NULL 的三值逻辑
SQL 中涉及 NULL 的比较结果不是 TRUE/FALSE，而是 **UNKNOWN**：

| 表达式         | 结果                             |
| -------------- | -------------------------------- |
| `NULL = NULL`  | UNKNOWN（不是 TRUE！）           |
| `NULL = 5`     | UNKNOWN                          |
| `NULL > 5`     | UNKNOWN                          |
| `NULL IS NULL` | TRUE（唯一正确判断 NULL 的方式） |

> **重要**：WHERE 条件中，只有结果为 TRUE 的行才被保留；UNKNOWN 和 FALSE 都会导致行被过滤掉。

## 6.3 NULL 对聚合函数的影响
- `COUNT(*)` 计算所有行（包含 NULL）
- `COUNT(列名)` 忽略 NULL
- `SUM/AVG/MAX/MIN` 都忽略 NULL

# 七、外连接的代数表达（用基本运算推导）

外连接可以由内连接 + 差 + NULL 填充来表达：

**左外连接**：
$$r ⟕ s = (r \bowtie s) \cup \big(\Pi_{R}(r - \Pi_R(r \bowtie s)) \times \{(null, \ldots, null)\}\big)$$

直觉：内连接结果 + "r 中没有匹配的行，s 的列填 null"

> 这说明外连接是**派生运算**，可由基本运算表达，但引入 NULL 后经典关系代数的集合语义需要扩展。

---

# 八、SQL 对应

| 关系代数      | SQL                                                          |
| ------------- | ------------------------------------------------------------ |
| $r \bowtie s$ | `FROM loan NATURAL JOIN borrower`                            |
| $r ⟕ s$       | `FROM loan LEFT OUTER JOIN borrower ON loan.loan_number = borrower.loan_number` |
| $r ⟖ s$       | `FROM loan RIGHT OUTER JOIN borrower ON loan.loan_number = borrower.loan_number` |
| $r ⟗ s$       | `FROM loan FULL OUTER JOIN borrower ON loan.loan_number = borrower.loan_number` |

> **注意**：MySQL 不支持 `FULL OUTER JOIN`，需用 `LEFT JOIN UNION RIGHT JOIN` 模拟。

---

# 九、左外连接 vs 右外连接的互换

左外连接和右外连接可以互相转换，只需交换操作数顺序：

$$r ⟕ s = s ⟖ r$$

因此实际中只需要左外连接就够了，右外连接只是书写习惯不同。

---

# 十、易错点总结

| 易错点                       | 说明                                                         |
| ---------------------------- | ------------------------------------------------------------ |
| NULL ≠ NULL                  | NULL 与任何值比较结果都是 UNKNOWN，判断 NULL 要用 IS NULL    |
| 左/右方向搞混                | 左外连接保留**左表**所有行；右外连接保留**右表**所有行       |
| 全外连接两边都保留           | 两边不匹配的行都出现，各自用 NULL 填充对方的列               |
| 外连接后 WHERE 过滤 NULL     | 若 WHERE 条件涉及外连接产生的 NULL 列，该行会因 UNKNOWN 被过滤，等于变回内连接 |
| MySQL 不支持 FULL OUTER JOIN | 需用 UNION 模拟                                              |
