这两张内容都在讲**关系代数的交运算（Set-Intersection Operation）**，下面完整展开。

---

## 1. 符号与读法

- **记号**：$r \cap s$
- **读法**：r 交 s
- **直觉**：只保留**同时出现在 r 和 s 里**的元组，两边都有才留，缺一不可。

---

## 2. 形式化定义

$$r \cap s = \{\, t \mid t \in r \ \text{and} \ t \in s \,\}$$

- $t$ 是一个元组（一行）
- "and" 是集合意义上的"既属于 r，又属于 s"
- 结果是 r 和 s 的**公共部分**

---

## 3. 关键结论：交可以由差推导（图中明确给出）

$$\boxed{r \cap s = r - (r - s)}$$

**为什么这个等式成立？**

逐步拆解：
1. $r - s$：从 r 里去掉"也在 s 里的元组"，剩下的是"只在 r 里、不在 s 里"的元组
2. $r - (r-s)$：再从 r 里去掉"只在 r 里、不在 s 里"的元组，剩下的恰好是"在 r 里、也在 s 里"的元组 = $r \cap s$ ✅

> **这就是为什么交运算不是关系代数的基本运算**：它可以完全由差运算（基本运算之一）来表达，因此是**派生运算（derived operation）**。

---

## 4. 例子解读（第二张图）

给定两个关系 r 和 s，属性均为 $(A, B)$：

| r 的元组      | s 的元组      |
| ------------- | ------------- |
| $(\alpha, 1)$ | $(\alpha, 2)$ |
| $(\alpha, 2)$ | $(\beta, 3)$  |
| $(\beta, 1)$  |               |

逐行检查：
- $(\alpha, 1)$：在 r ✅，在 s ❌ → **不保留**
- $(\alpha, 2)$：在 r ✅，在 s ✅ → **保留**
- $(\beta, 1)$：在 r ✅，在 s ❌ → **不保留**

所以 $r \cap s = \{(\alpha, 2)\}$

**用公式验证**：
- $r - s = \{(\alpha,1),\ (\beta,1)\}$（上一节已算过）
- $r - (r-s) = \{(\alpha,1),(\alpha,2),(\beta,1)\} - \{(\alpha,1),(\beta,1)\} = \{(\alpha,2)\}$ ✅

---

## 5. 交运算成立的前提：并相容

与并、差运算一样，交运算要求 r 和 s **并相容**：

1. **属性个数相同**
2. **对应属性域兼容**（类型一致）
3. **属性名一致**（或先用 $\rho$ 重命名对齐）

> 原因：交运算本质上是差运算的组合，差运算要求并相容，因此交运算也必须满足。

---

## 6. 交运算的代数性质

| 性质       | 公式                                             | 说明                         |
| ---------- | ------------------------------------------------ | ---------------------------- |
| **交换律** | $r \cap s = s \cap r$                            | 顺序不影响结果（与差不同！） |
| **结合律** | $(r \cap s) \cap t = r \cap (s \cap t)$          | 多个交可任意结合             |
| **幂等律** | $r \cap r = r$                                   | 自己交自己还是自己           |
| **与空集** | $r \cap \emptyset = \emptyset$                   | 任何关系与空集的交是空集     |
| **子集性** | $r \cap s \subseteq r$ 且 $r \cap s \subseteq s$ | 结果同时是 r 和 s 的子集     |

> 注意：**交运算满足交换律**，而差运算不满足，这是两者的重要区别。

---

## 7. 并、差、交三者的关系（图中未提，但非常重要）

三者都要求并相容，且相互之间有推导关系：

$$r \cap s = r - (r - s) \quad \text{（交由差推导）}$$

$$r \cup s = (r - s) \cup s \quad \text{（并可拆分）}$$

还有一个重要的**De Morgan 对偶律**（类比集合论）：

$$\overline{r \cup s} = \overline{r} \cap \overline{s}$$
$$\overline{r \cap s} = \overline{r} \cup \overline{s}$$

（在关系代数语境下，"补"通常通过差运算相对于全集来表达。）

---

## 8. 基本运算 vs 派生运算（图中隐含的核心概念）

| 类型         | 运算                                      | 说明                               |
| ------------ | ----------------------------------------- | ---------------------------------- |
| **基本运算** | $\sigma,\ \Pi,\ \cup,\ -,\ \times,\ \rho$ | 不可由其他运算推导，构成最小完备集 |
| **派生运算** | $\cap,\ \bowtie,\ \div$                   | 可由基本运算组合表达               |

**交是派生运算**的意义：
- 理论上只需要 6 个基本运算就能表达所有关系查询
- 交运算是"语法糖"，方便书写，但不增加表达能力
- 查询优化器可以把交替换成差的组合，反之亦然

---

## 9. 典型应用场景

**场景 1：找出"既选了课程A，又选了课程B"的学生**
$$\Pi_{学号}(\sigma_{课程='A'}(选课)) \cap \Pi_{学号}(\sigma_{课程='B'}(选课))$$

**场景 2：找出"在两个分支机构都有账户"的客户**
$$\Pi_{客户ID}(\sigma_{分支='北京'}(账户)) \cap \Pi_{客户ID}(\sigma_{分支='上海'}(账户))$$

**场景 3：找出"同时是教师又是学生"的人员**
$$\Pi_{人员ID}(教师表) \cap \Pi_{人员ID}(学生表)$$

---

## 10. 交运算 vs SQL

| 关系代数   | SQL                                        |
| ---------- | ------------------------------------------ |
| $r \cap s$ | `(query_r) INTERSECT (query_s)`            |
| 同上       | `WHERE id IN (SELECT id FROM s)`           |
| 同上       | `WHERE EXISTS (SELECT 1 FROM s WHERE ...)` |

> **注意**：`INTERSECT` 在 PostgreSQL、SQL Server、Oracle 中支持；MySQL 8.0+ 才支持 `INTERSECT`，旧版本需用 `IN` 或 `EXISTS` 替代。

---

## 11. 并、差、交三者对比速记

| 运算 | 符号       | 保留的元组             | 是否基本运算 | 是否满足交换律 |
| ---- | ---------- | ---------------------- | ------------ | -------------- |
| 并   | $r \cup s$ | 在 r **或** s 中       | ✅ 是         | ✅ 是           |
| 差   | $r - s$    | 在 r 中**且不在** s 中 | ✅ 是         | ❌ 否           |
| 交   | $r \cap s$ | 在 r **且**在 s 中     | ❌ 派生       | ✅ 是           |

---

## 12. 易错点总结

| 易错点             | 说明                                                         |
| ------------------ | ------------------------------------------------------------ |
| 误以为交是基本运算 | 交是派生运算，可由差推导                                     |
| 忘记并相容前提     | 两表结构不同时交运算无意义                                   |
| 交与连接混淆       | 交要求并相容、结构完全一致；连接是不同结构的表组合           |
| SQL NULL 问题      | `INTERSECT` 会把 NULL 视为相等（两个 NULL 算匹配），而 `IN` 子句中 NULL 会导致不匹配，需注意 |
