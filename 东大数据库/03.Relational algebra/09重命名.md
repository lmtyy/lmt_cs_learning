这两张都在讲**重命名运算（Rename Operation）**，第一张给出定义和两种形式，第二张给出一个完整的综合例子。下面全面展开。

---

# 一、重命名运算的作用（为什么需要它）

图中给出两个核心用途：

1. **给关系代数表达式的结果命名**，使其可以被后续运算引用
   - 关系代数表达式的中间结果是"匿名的"，无法直接被再次引用，重命名给它一个名字
2. **允许一个关系用多个名字引用**
   - 最典型的场景：**自连接**（一张表与自身做笛卡尔积/连接），必须用重命名区分两个"副本"

---

# 二、两种形式（第一张图）

## 形式 1：只重命名关系名

$$\rho_X(E)$$

- 把表达式 $E$ 的结果命名为 $X$
- **列名不变**，只是给整个关系起了一个新名字
- 读法："把 E 重命名为 X"

## 形式 2：同时重命名关系名和属性名

$$\rho_{X(A_1, A_2, \ldots, A_n)}(E)$$

- 把表达式 $E$ 的结果命名为 $X$
- 同时把各列依次重命名为 $A_1, A_2, \ldots, A_n$
- **$n$ 必须等于 $E$ 的属性个数**，一一对应
- 读法："把 E 重命名为 X，属性依次为 $A_1, A_2, \ldots, A_n$"

---

# 三、重命名的三大典型使用场景

## 场景 1：解决自连接的同名列冲突（最重要）

一张表与自身做笛卡尔积时，两边属性名完全相同，必须用重命名区分：

**例**：找出工资比 Jones 高的所有员工姓名

```
Employee(name, salary)
```

直接写 $\text{Employee} \times \text{Employee}$ 会有两个 name 列、两个 salary 列，无法区分。

正确写法：
$$\Pi_{\text{e1.name}}(\sigma_{\text{e1.salary} > \text{e2.salary}}(\rho_{\text{e1}}(\text{Employee}) \times \rho_{\text{e2}}(\text{Employee})))$$

先把两份 Employee 分别命名为 e1 和 e2，再做笛卡尔积，列就变成 e1.name、e1.salary、e2.name、e2.salary，不再冲突。

---

## 场景 2：给中间结果命名，方便复用

**例**：
$$\text{temp} \leftarrow \sigma_{\text{Sdept}='CS'}(\text{Student})$$
$$\Pi_{\text{Sname}}(\text{temp})$$

用重命名把中间结果存起来，避免重复写长表达式（在关系代数的"赋值"写法中常用）。

---

## 场景 3：并相容对齐（属性名不一致时）

两个关系做并/差/交时，若属性名不同但语义相同，需先重命名对齐：

**例**：
$$\Pi_{\text{Sname}}(\text{Student}) \cup \Pi_{\text{Tname}}(\text{Teacher})$$

这两个投影结果列名不同（Sname vs Tname），不能直接并。需先：
$$\Pi_{\text{Sname}}(\text{Student}) \cup \Pi_{\text{Sname}}(\rho_{\text{T(Sname)}}(\Pi_{\text{Tname}}(\text{Teacher})))$$

---

# 四、综合例子解读（第二张图）

**需求**：找出所有与 Smith 住在同一城市的客户姓名

**Customer 表**（部分数据）：

| customer_name | customer_street | customer_city |
| ------------- | --------------- | ------------- |
| Adams         | Spring          | Pittsfield    |
| Brooks        | Senator         | Brooklyn      |
| Curry         | North           | Rye           |
| Smith         | North           | **Rye**       |
| ...           | ...             | ...           |

**完整表达式**：

$$\Pi_{\text{customer.customer\_name}} \Big( \sigma_{\text{customer.customer\_city = smith\_infor.customer\_city}} \big( \text{customer} \times \rho_{\text{smith\_infor}}(\sigma_{\text{customer\_name="smith"}}(\text{customer})) \big) \Big)$$

---

## 逐步拆解执行过程

### 第 1 步：找出 Smith 的信息
$$\sigma_{\text{customer\_name="smith"}}(\text{customer})$$

从 customer 表中选出 Smith 那一行：

| customer_name | customer_street | customer_city |
| ------------- | --------------- | ------------- |
| Smith         | North           | Rye           |

---

### 第 2 步：重命名，避免与原表冲突
$$\rho_{\text{smith\_infor}}(\sigma_{\text{customer\_name="smith"}}(\text{customer}))$$

把上一步的结果命名为 **smith\_infor**，列名变为：
- smith\_infor.customer\_name
- smith\_infor.customer\_street
- smith\_infor.customer\_city

> **为什么必须重命名？** 因为接下来要把 customer 表与这个结果做笛卡尔积，两边都有 customer\_name、customer\_city 等列，不重命名就无法区分"哪边的 customer\_city"。

---

### 第 3 步：笛卡尔积
$$\text{customer} \times \rho_{\text{smith\_infor}}(\ldots)$$

customer 有 12 行，smith\_infor 有 1 行，结果有 $12 \times 1 = 12$ 行，每行有 6 列：

| customer.customer\_name | customer.customer\_city | smith\_infor.customer\_name | smith\_infor.customer\_city |
| ----------------------- | ----------------------- | --------------------------- | --------------------------- |
| Adams                   | Pittsfield              | Smith                       | Rye                         |
| Brooks                  | Brooklyn                | Smith                       | Rye                         |
| Curry                   | **Rye**                 | Smith                       | **Rye**                     |
| Smith                   | **Rye**                 | Smith                       | **Rye**                     |
| ...                     | ...                     | Smith                       | Rye                         |

---

### 第 4 步：选择——城市相同的行
$$\sigma_{\text{customer.customer\_city = smith\_infor.customer\_city}}(\ldots)$$

只保留 customer\_city 相等的行（即与 Smith 同城 Rye 的）：

| customer.customer\_name | customer.customer\_city | ...  |
| ----------------------- | ----------------------- | ---- |
| Curry                   | Rye                     | ...  |
| Smith                   | Rye                     | ...  |

---

### 第 5 步：投影——只取客户姓名
$$\Pi_{\text{customer.customer\_name}}(\ldots)$$

最终结果：

| customer\_name |
| -------------- |
| Curry          |
| Smith          |

> **注意**：Smith 本人也在结果中（他/她确实与自己住在同一城市）。若需要排除 Smith 本人，可以在第 4 步的选择条件中加上 $\land\ \text{customer.customer\_name} \neq \text{"smith"}$。

---

# 五、重命名的代数性质

| 性质               | 说明                                                |
| ------------------ | --------------------------------------------------- |
| **不改变数据内容** | 重命名只改变名字，元组的值完全不变                  |
| **不去重、不过滤** | 行数、列数、数据值均不变                            |
| **可嵌套**         | $\rho_Y(\rho_X(E)) = \rho_Y(E)$，外层重命名覆盖内层 |
| **闭包性**         | 结果仍是关系，可继续参与其他运算                    |

---

# 六、重命名 vs SQL 的别名

| 关系代数                      | SQL                                                         |
| ----------------------------- | ----------------------------------------------------------- |
| $\rho_X(E)$                   | `(query) AS X` 或 `FROM table AS X`                         |
| $\rho_{X(A_1,\ldots,A_n)}(E)$ | `SELECT col1 AS A1, col2 AS A2 ... FROM ...`                |
| 自连接中的重命名              | `FROM Employee e1, Employee e2 WHERE e1.salary > e2.salary` |

---

# 七、易错点总结

| 易错点                 | 说明                                                         |
| ---------------------- | ------------------------------------------------------------ |
| 忘记自连接需要重命名   | 同一张表做笛卡尔积/连接时，必须用 $\rho$ 区分两个副本        |
| 属性个数不匹配         | $\rho_{X(A_1,\ldots,A_n)}(E)$ 中 $n$ 必须等于 $E$ 的属性个数 |
| 重命名后仍用旧名引用   | 重命名后必须用新名字，旧名字在该表达式中不再有效             |
| 误以为重命名会改变数据 | 重命名只改名字，数据内容、行数、列数完全不变                 |
| 并相容时忘记对齐列名   | 做并/差/交前若列名不同，需先用重命名统一列名                 |
