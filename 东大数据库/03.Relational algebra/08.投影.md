这三张都在讲**投影运算（Project Operation）**，从定义、去重规则到实际例子完整呈现。下面全面展开。

---

# 一、符号与核心含义

- **记号**：$\Pi_{A_1, A_2, \ldots, A_k}(r)$
- **下标**：要保留的**属性名列表**（列名）
- **直觉**：从关系 r 中只保留指定的列，丢掉其余列，相当于 SQL 的 `SELECT 列名`
- **一元运算（Unary operation）**：只作用于**一个**关系（不像笛卡尔积需要两个）

---

# 二、形式化定义（第二张图）

$$\Pi_{A_1,A_2,\ldots,A_k}(r) = \text{保留 r 中第 } A_1, A_2, \ldots, A_k \text{ 列，删去其余列，并去除重复行}$$

图中给出两个关键规则：

## 规则 1：删去未列出的列
> "The result is defined as the relation of **k columns** obtained by **erasing the columns that are not listed**"

结果只有 k 列，其余列全部丢弃。

## 规则 2：自动去除重复行
> "**Duplicate rows removed** from result, since relations are sets"

因为关系是**集合**，不允许重复元组，投影后若出现重复行，只保留一份。

> 这是投影与 SQL `SELECT`（不加 DISTINCT）的核心区别：关系代数投影**强制去重**，SQL 默认**不去重**。

---

# 三、属性的两种引用方式（第三张图揭示）

图中给出了两种等价写法：

$$\Pi_{\text{Sname, Sdept}}(\text{Student}) \quad \text{或} \quad \Pi_{2,5}(\text{Student})$$

| 方式           | 写法                        | 说明                                               |
| -------------- | --------------------------- | -------------------------------------------------- |
| **按属性名**   | $\Pi_{\text{Sname, Sdept}}$ | 直接写列名，清晰易读，推荐                         |
| **按位置编号** | $\Pi_{2,5}$                 | 用列的序号（第2列=Sname，第5列=Sdept），不依赖列名 |

> 位置编号方式在某些教材中使用，但可读性差，实际写查询时应优先使用属性名方式。

---

# 四、例子解读

## 4.1 抽象例子（第一张图）

关系 r 有属性 $(A, B, C)$，4 行：

| A        | B    | C    |
| -------- | ---- | ---- |
| $\alpha$ | 10   | 1    |
| $\alpha$ | 20   | 1    |
| $\beta$  | 30   | 1    |
| $\beta$  | 40   | 2    |

执行 $\Pi_{A,C}(r)$：只保留 A 和 C 列，丢掉 B 列：

**投影后（未去重）**：

| A        | C    |
| -------- | ---- |
| $\alpha$ | 1    |
| $\alpha$ | 1    |
| $\beta$  | 1    |
| $\beta$  | 2    |

**去重后（最终结果）**：

| A        | C    |
| -------- | ---- |
| $\alpha$ | 1    |
| $\beta$  | 1    |
| $\beta$  | 2    |

原来 4 行，去重后变 3 行。图中用红圈标出了被去掉的重复行 $(\alpha, 1)$。

---

## 4.2 实际例子（第三张图）

**需求**：查询所有学生的姓名和所在系

**Student 表**：

| Sno   | Sname | Ssex | Sage | Sdept |
| ----- | ----- | ---- | ---- | ----- |
| 95001 | 李勇  | 男   | 20   | CS    |
| 95002 | 刘晨  | 女   | 19   | IS    |
| 95003 | 王敏  | 女   | 18   | MA    |
| 95004 | 张立  | 男   | 19   | IS    |

**关系代数表达式**：
$$\Pi_{\text{Sname, Sdept}}(\text{Student})$$

**结果**（只保留第2列 Sname 和第5列 Sdept，丢掉 Sno、Ssex、Sage）：

| Sname | Sdept |
| ----- | ----- |
| 李勇  | CS    |
| 刘晨  | IS    |
| 王敏  | MA    |
| 张立  | IS    |

本例中每行的 (Sname, Sdept) 组合都不同，所以没有发生去重。若有两个同名同系的学生，结果里只会出现一行。

# 五、投影运算的基本特性

| 特性              | 说明                             |
| ----------------- | -------------------------------- |
| **只操作列**      | 行不增加，只可能因去重而减少     |
| **列数 ≤ 原关系** | 结果只有 k 列（k ≤ 原列数）      |
| **强制去重**      | 关系代数语义下，投影结果自动去重 |
| **闭包性**        | 结果仍是关系，可继续参与其他运算 |
| **一元运算**      | 只需要一个输入关系               |

# 六、投影的重要代数性质（图中未提）

## 6.1 投影的幂等性（列集合相同时）
$$\Pi_L(\Pi_L(r)) = \Pi_L(r)$$
对同一列集合重复投影，结果不变。

## 6.2 投影的级联（列集合有包含关系时）
若 $L_1 \subseteq L_2$（$L_1$ 是 $L_2$ 的子集）：
$$\Pi_{L_1}(\Pi_{L_2}(r)) = \Pi_{L_1}(r)$$
先投影到更多列，再投影到更少列，等价于直接投影到更少列。

**例**：
$$\Pi_{A}(\Pi_{A,C}(r)) = \Pi_{A}(r)$$

> **注意**：若 $L_1 \not\subseteq L_2$（即 $L_1$ 中有 $L_2$ 没有的列），则 $\Pi_{L_1}(\Pi_{L_2}(r))$ 是**非法的**，因为第一次投影后那些列已经不存在了。

## 6.3 投影与选择的组合（最常用）
当谓词 $p$ 只涉及属性列表 $L$ 中的属性时：
$$\Pi_L(\sigma_p(r)) = \sigma_p(\Pi_L(r))$$
先筛行再取列 = 先取列再筛行（条件允许时可互换，用于优化）。

## 6.4 投影与笛卡尔积的分配
$$\Pi_{L_1 \cup L_2}(r \times s) = \Pi_{L_1}(r) \times \Pi_{L_2}(s)$$
（当 $L_1$ 只含 r 的属性，$L_2$ 只含 s 的属性时）

# 七、投影 vs 选择（对比总结）

| 对比项   | 投影 $\Pi$                   | 选择 $\sigma$                  |
| -------- | ---------------------------- | ------------------------------ |
| 操作对象 | **列**（属性）               | **行**（元组）                 |
| 指定内容 | 属性名列表                   | 布尔谓词                       |
| 结果列数 | **≤ 原关系**（只保留指定列） | 与原关系**相同**               |
| 结果行数 | 可能因去重而减少             | **≤ 原关系**（满足条件才保留） |
| 去重     | ✅ 自动去重                   | ❌ 不去重                       |
| SQL 对应 | `SELECT DISTINCT 列名`       | `WHERE 条件`                   |

---

# 八、SQL 对应

| 关系代数                                                     | SQL                                                   |
| ------------------------------------------------------------ | ----------------------------------------------------- |
| $\Pi_{\text{Sname, Sdept}}(\text{Student})$                  | `SELECT DISTINCT Sname, Sdept FROM Student`           |
| $\Pi_{A,C}(r)$                                               | `SELECT DISTINCT A, C FROM r`                         |
| $\Pi_{\text{Sname}}(\sigma_{\text{Sdept}='CS'}(\text{Student}))$ | `SELECT DISTINCT Sname FROM Student WHERE Sdept='CS'` |

> **关键**：关系代数投影对应 SQL 的 `SELECT DISTINCT`，而不是 `SELECT`（因为关系代数强制去重，SQL 默认不去重）。

---

# 九、投影与选择的组合（最典型的查询模式）

实际查询中，投影和选择几乎总是一起使用：**先用选择筛行，再用投影取列**。

**例**：查询 CS 系所有学生的姓名：
$$\Pi_{\text{Sname}}(\sigma_{\text{Sdept}='CS'}(\text{Student}))$$

执行步骤：
1. $\sigma_{\text{Sdept}='CS'}(\text{Student})$：先筛出 CS 系的行 → 得到李勇那行
2. $\Pi_{\text{Sname}}(...)$：再只取 Sname 列 → 得到"李勇"

> **顺序很重要**：应先选择（减少行数）再投影（减少列数），这样中间结果最小，效率最高。

# 十、易错点总结

| 易错点                 | 说明                                                         |
| ---------------------- | ------------------------------------------------------------ |
| 忘记去重               | 投影后必须去重，这是关系代数的集合语义要求                   |
| 投影列不在关系中       | 指定的属性名必须存在于原关系，否则非法                       |
| 级联投影顺序错误       | $\Pi_{L_1}(\Pi_{L_2}(r))$ 要求 $L_1 \subseteq L_2$，否则非法 |
| 与 SQL SELECT 混淆     | SQL `SELECT` 默认不去重；关系代数投影强制去重                |
| 误以为投影会改变行顺序 | 关系是集合，本身无顺序，投影不保证任何行的顺序               |
