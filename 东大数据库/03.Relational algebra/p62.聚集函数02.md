这一页课件讲解了聚集运算中一个非常实用的小细节：**聚集结果的命名问题**。

当你执行类似 `sum`、`avg` 这样的操作时，产生的结果列在原始表中是没有名字的。为了方便后续引用或让结果表更易读，我们需要给它起个名字。

---

### 1. 核心问题：无名列
当你写 $_{branch\_name}\mathcal{G}_{sum(balance)}(account)$ 时，生成的那一列结果默认就是 `sum(balance)`。在严格的关系代数定义中，这个结果列是没有正式属性名的。

### 2. 解决方案：更名 (Rename)
课件提到了两种处理方式：
* **使用标准的更名运算符 ($\rho$)**：你可以把整个聚集运算的结果嵌套在一个 $\rho$ 运算符里。但这会让表达式变得非常长且难以阅读。
* **聚集运算内的简写 (as)**：为了方便，关系代数允许我们在聚集函数后面直接加上 `as` 关键字来指定新名字。

### 3. 示例拆解
课件展示了这种语法的进化：

* **进化前**：
    $$branch\_name\ \mathcal{G}\ sum(balance)\ (account)$$
    结果列的名字可能就叫 `sum(balance)`，不够美观。

* **进化后**：
    $$branch\_name\ \mathcal{G}\ sum(balance)\ \mathbf{as}\ sum\text{-}balance\ (account)$$
    这里明确指定了结果列的名字为 **`sum-balance`**。

---

### 💡 与 SQL 的关联
这完全对应了 SQL 中的 `AS` 别名语法。作为一个 CS 专业学生，你可以直接在大脑中进行转换：

```sql
/* 课件中表达式对应的 SQL */
SELECT branch_name, SUM(balance) AS sum_balance
FROM account
GROUP BY branch_name;
```

### 总结
这一页告诉我们，关系代数不仅关注数学逻辑，也考虑到了**工程实践中的可读性**。通过 `as` 命名，我们可以把复杂的计算结果变成一个清晰的属性名，方便在更外层的嵌套查询中继续使用。
