### 第1页：基本查询结构概述

**内容**：
- SQL 基于关系代数和关系演算。
- 一个典型的SQL查询形式是：
  ```sql
  SELECT A1, A2, ..., An
  FROM r1, r2, ..., rm
  WHERE P;
  ```
  其中：
  - `A1, A2, ..., An` 是目标属性（Attributes）。
  - `r1, r2, ..., rm` 是参与查询的关系（表）。
  - `P` 是谓词（条件），用于筛选行。

**详细讲解**：
- **SQL的理论基础**：SQL（结构化查询语言）是建立在**关系代数**和**关系演算**之上的。关系代数提供了一系列操作（如选择、投影、连接等）来操作关系，而关系演算则用逻辑公式表达查询。SQL结合了这两种思想，使得用户可以直观地描述想要的数据。
- **查询结构**：这是最基础的SQL查询模板。`SELECT`子句指定要输出的列，`FROM`子句指定数据来源的表，`WHERE`子句指定过滤条件。三者共同定义了查询的语义。
- **类比自然语言**：这个结构类似于英语中的“从...中选择...当...时”，例如：“从员工表中选择姓名和工资，当部门为'销售部'时”。

### 第2页：SQL查询与关系代数的等价性

**内容**：
- SQL查询等价于关系代数表达式：
  \[
  \prod_{A_1, A_2, \dots, A_n} \left( \sigma_P \left( r_1 \times r_2 \times \dots \times r_m \right) \right)
  \]
- SQL查询的结果是一个关系（即一张表）。

**详细讲解**：
- **关系代数解释**：上面的表达式清晰地展示了SQL查询的底层操作步骤：
  1. 首先，计算所有表的**笛卡尔积**（\(r_1 \times r_2 \times \dots \times r_m\)）。这会产生一个巨大的中间表，包含所有可能的行组合。
  2. 然后，应用**选择操作**（\(\sigma_P\)），只保留满足谓词 \(P\) 的行。
  3. 最后，应用**投影操作**（\(\prod_{A_1,\dots,A_n}\)），从结果中提取所需的属性列。
- **为什么这样等价？**：虽然实际数据库执行时会进行优化（如使用索引、连接算法代替笛卡尔积），但从语义上讲，任何SQL查询都可以用这个关系代数表达式表示。
- **查询结果**：SQL查询总是返回一个**关系**（表），这意味着结果集具有行和列的二维结构，并且可以再次作为查询对象（即子查询）。这一特性称为**关系闭包**，是SQL强大的基础。

### 第4页：`SELECT`子句详解

**内容**：
- `SELECT`子句列出结果中需要的属性，对应于关系代数中的**投影**操作。
- 示例：从 `Loan` 表中找出所有分行的名称。
  ```sql
  SELECT branch_name
  FROM loan;
  ```
  这等价于关系代数：\(\pi_{\text{branch\_name}}(\text{loan})\)。

**详细讲解**：
- **投影操作**：在关系代数中，投影（\(\pi\)）用于从关系中选取指定的列，并去除重复行。但在SQL中，默认情况下`SELECT`不会去除重复行（除非使用`DISTINCT`）。这一点与关系代数略有不同：SQL的`SELECT`保留了重复行（因为数据库通常不自动去重），而关系代数的投影会自动去重。这体现了SQL更贴近实际使用场景（用户有时需要所有行，有时需要去重）。
- **示例分析**：假设 `Loan` 表中有多笔贷款，每笔贷款都有对应的分行名。执行 `SELECT branch_name FROM loan;` 会返回所有贷款的所属分行名，包括重复的分行名。如果你想获得唯一的分行列表，可以使用 `SELECT DISTINCT branch_name FROM loan;`，这更接近关系代数的投影语义。
- **属性列表**：可以列出多个属性，用逗号分隔。例如 `SELECT loan_number, amount FROM loan;`。

### 第5页：SQL名称的大小写敏感性

**内容**：
- SQL名称（如表名、列名）通常不区分大小写。例如 `Branch_Name`、`BRANCH_NAME` 和 `branch_name` 被视为相同标识符。

**详细讲解**：
- **SQL标准**：根据SQL标准，标识符（如表名、列名）在默认情况下是不区分大小写的。但实际数据库系统（如Oracle、MySQL、PostgreSQL等）可能有自己的实现差异：
  - 在MySQL中，表名在Windows上不区分大小写，但在Linux上可能区分（取决于文件系统）。列名始终不区分大小写。
  - 在Oracle中，默认情况下所有标识符以大写形式存储，但在SQL语句中可以任意大小写。
  - 在PostgreSQL中，未加引号的标识符会被自动转换为小写，因此大小写不敏感；但若使用双引号括起来的标识符，则区分大小写。
- **建议**：为了可移植性和可读性，通常采用小写加下划线的命名风格（如 `branch_name`），并在SQL语句中保持一致。
- **示例**：在大多数数据库中，以下三个查询是等价的：
  ```sql
  SELECT branch_name FROM loan;
  SELECT BRANCH_NAME FROM LOAN;
  SELECT Branch_Name FROM Loan;
  ```
  但需要注意，某些数据库可能要求表名大小写与存储时一致（尤其在区分大小写的文件系统中），所以最好统一风格。
