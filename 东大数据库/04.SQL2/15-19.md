这几页PPT详细介绍了SQL查询中 **`WHERE` 子句**的核心功能和常用谓词。下面我将逐一讲解每页的内容，并结合示例说明其用法和注意事项。

---

### 第1页：WHERE子句的作用

**内容回顾**  
- `WHERE` 子句用于指定结果必须满足的条件。
- 它对应于关系代数中的**选择（selection）操作**。

**详细讲解**  
- **关系代数中的选择**（\(\sigma\)）是从关系中选取满足给定谓词的元组（行）。例如，\(\sigma_{\text{amount}>1200}(\text{loan})\) 选出所有贷款金额大于1200的贷款记录。
- SQL中的 `WHERE` 子句正是实现这一功能的语法。它出现在 `FROM` 子句之后，对 `FROM` 子句产生的中间表（可能是多表的笛卡尔积或连接结果）进行行级过滤，只有满足条件的行才会进入下一步处理（如投影、排序等）。
- **重要**：`WHERE` 子句中的条件可以使用列名、常量、表达式、函数以及子查询。

---

### 第2页：WHERE子句的简单示例

**内容回顾**  
给出了一个具体的例子：从 `loan` 表中查询在 Perryridge 分行且贷款金额大于1200的贷款编号。

```sql
SELECT loan_number
FROM loan
WHERE branch_name = 'Perryridge' AND amount > 1200;
```

**详细讲解**  
- **条件组合**：使用了 `AND` 逻辑运算符连接两个条件，要求同时满足。
- **字符串比较**：`branch_name = 'Perryridge'` 中，`'Perryridge'` 是字符串常量，必须用**单引号**括起来。注意字符串的比较通常区分大小写取决于数据库的排序规则（collation）。
- **数值比较**：`amount > 1200` 直接比较数值。
- **结果**：返回满足所有条件的行中的 `loan_number` 列。

### 第3页：WHERE子句支持的条件谓词汇总

**内容回顾**  
以表格形式列出了常见的谓词类型：

| 查询条件             | 谓词                            |
| -------------------- | ------------------------------- |
| 比较                 | =, >, <, >=, <=, !=, <>, !>, !< |
| 确定范围             | BETWEEN AND, NOT BETWEEN AND    |
| 确定集合             | IN, NOT IN                      |
| 字符匹配             | LIKE, NOT LIKE                  |
| 空值                 | IS NULL, IS NOT NULL            |
| 多重条件（逻辑运算） | AND, OR, NOT                    |

**详细讲解**  
- **比较运算符**：
  - 标准SQL支持 `=`、`>`、`<`、`>=`、`<=`、`<>`（不等于）。部分数据库还支持 `!=`（同样表示不等于）、`!>`（不大于）等非标准扩展，但为了可移植性，推荐使用标准符号。
  - 注意：比较运算符只能用于可比较的数据类型（数值、字符串、日期等），且涉及 `NULL` 时结果总是 `UNKNOWN`（见第6页）。
- **范围谓词**：`BETWEEN ... AND ...` 包含边界值（即闭区间），`NOT BETWEEN` 则排除区间内的值。
- **集合谓词**：`IN` 用于判断值是否在给定的列表或子查询结果中，`NOT IN` 相反。
- **字符匹配**：`LIKE` 用于模式匹配，支持通配符 `%`（匹配任意字符串）和 `_`（匹配单个字符）。`NOT LIKE` 否定匹配。
- **空值判断**：由于 `NULL` 不能使用 `=` 或 `<>` 直接比较，必须使用 `IS NULL` 或 `IS NOT NULL`。
- **逻辑运算**：`AND`、`OR`、`NOT` 用于组合多个条件，优先级为 `NOT` > `AND` > `OR`，建议使用括号明确优先级。

### 第4页：BETWEEN 的用法

**内容回顾**  
介绍了 `BETWEEN` 操作符，用于判断值是否在某个区间内（包含边界）。示例：查询贷款金额在 90,000 到 100,000 之间的贷款编号。

```sql
SELECT loan_number
FROM loan
WHERE amount BETWEEN 90000 AND 100000;
```

**详细讲解**  
- **语法**：`value BETWEEN lower AND upper` 等价于 `value >= lower AND value <= upper`。
- **包含性**：BETWEEN 包含上下界，因此 `amount BETWEEN 90000 AND 100000` 会包括金额等于 90000 和 100000 的行。
- **NOT BETWEEN**：`value NOT BETWEEN lower AND upper` 等价于 `value < lower OR value > upper`。
- **数据类型**：`lower` 和 `upper` 必须与 `value` 的数据类型兼容，且 `lower` 应小于等于 `upper`，否则结果为空集。
- **性能**：BETWEEN 通常会被优化为范围扫描，使用索引时效率较高。

### 第5页：IN 的用法（确定集合）

**内容回顾**  
展示了 `IN` 和 `NOT IN` 的用法，用于判断某个列的值是否在指定的值列表中。示例：

- 查询信息系（IS）、数学系（MA）和计算机科学系（CS）学生的姓名和性别：
  ```sql
  SELECT Sname, Ssex
  FROM Student
  WHERE Sdept IN ('IS', 'MA', 'CS');
  ```

- 查询不在这三个系的学生：
  ```sql
  SELECT Sname, Ssex
  FROM Student
  WHERE Sdept NOT IN ('IS', 'MA', 'CS');
  ```

**详细讲解**  
- **IN 操作符**：`column IN (value1, value2, ...)` 等价于 `column = value1 OR column = value2 OR ...`。当列表较长时，IN 更简洁。
- **子查询**：IN 后面也可以跟一个子查询（返回单列多行），例如 `WHERE Sdept IN (SELECT dept_name FROM department WHERE ...)`。这将在后续嵌套子查询章节详细讲解。
- **NOT IN 的陷阱**：如果子查询或列表中包含 `NULL`，`NOT IN` 的行为可能不符合预期。例如，`WHERE Sdept NOT IN ('IS', 'MA', NULL)` 的结果永远为空，因为 `Sdept NOT IN (..., NULL)` 等价于 `Sdept != 'IS' AND Sdept != 'MA' AND Sdept != NULL`，而 `Sdept != NULL` 的结果是 `UNKNOWN`，导致整个条件为 `UNKNOWN`，所以不会返回任何行。因此使用 `NOT IN` 时要确保列表或子查询结果不包含 `NULL`。
- **字符串比较**：列表中的字符串常量需加单引号，数值常量不需加引号。

### 扩展知识点与注意事项

1. **NULL 的三值逻辑**  
   SQL 使用三值逻辑：`TRUE`、`FALSE` 和 `UNKNOWN`。任何与 `NULL` 的比较（如 `amount = NULL`）结果都是 `UNKNOWN`。只有 `IS NULL` 和 `IS NOT NULL` 能正确处理 `NULL`。在 `WHERE` 子句中，只有条件为 `TRUE` 的行才会被选中，`FALSE` 和 `UNKNOWN` 的行会被过滤掉。

2. **字符匹配（LIKE）**  
   虽然本组PPT未详细展开，但表格中已列出。LIKE 常用于模糊查询，例如查找所有以“王”开头的学生：`WHERE Sname LIKE '王%'`。需要转义通配符时，可使用 `ESCAPE` 子句定义转义字符。

3. **逻辑运算符的优先级**  
   - `NOT` 优先级最高。
   - 其次为 `AND`。
   - 最低为 `OR`。
   - 可以使用括号改变运算顺序，建议在复杂条件中显式加括号以提高可读性。

4. **性能考虑**  
   - 在 `WHERE` 子句中对列应用函数（如 `LOWER(Sdept) = 'cs'`）可能会导致索引失效，应尽量保持列原始形式。
   - 使用 `IN` 比多个 `OR` 更清晰，但性能上通常相同，数据库优化器会做等价转换。

5. **WHERE 与 HAVING 的区别**  
   - `WHERE` 用于在分组前对行进行过滤。
   - `HAVING` 用于在分组后对组进行过滤，通常与 `GROUP BY` 一起使用。
