这几页PPT主要围绕SQL查询中 `SELECT` 子句的进阶用法展开，包括选择所有列、在查询结果中进行算术运算、使用字符串常量、调用函数以及为结果列指定别名。下面我将逐一详细讲解这些知识点，并结合示例进行说明。

---

### 1. 使用星号（*）选择所有列

**内容回顾**  
第一页PPT展示了如何使用 `*` 来表示“所有属性”，并指出以下两个查询是等价的：

```sql
SELECT * FROM loan;
```

```sql
SELECT loan_number, branch_name, amount FROM loan;
```

**详细讲解**  
- **`*` 的作用**：在 `SELECT` 子句中，`*` 是一个通配符，表示从 `FROM` 子句指定的表中选择**所有列**。这在快速查看表内容或编写临时查询时非常方便。
- **等价性**：正如PPT所示，`SELECT *` 等价于显式列出所有列名。但需要注意：
  - 如果表结构将来发生变化（例如增加新列），使用 `*` 的查询结果会自动包含新列，而显式列出列名的查询则不会。这可能是期望的行为，也可能导致意想不到的结果，因此在实际生产环境中，**推荐显式列出所需列**，以提高查询的可读性和稳定性。
  - 在多表连接（JOIN）的查询中，`*` 会返回所有表的全部列，这可能导致列名重复（需要通过别名区分），且结果集可能过于庞大。
- **示例表**：PPT中给出的 `Loan` 表包含三列：`loan_number`、`amount`、`branch_name`。因此 `SELECT *` 的结果就是这三列的全部数据。

### 2. 在检索结果中进行算术运算

**内容回顾**  
第二页PPT介绍了 `SELECT` 子句中可以使用算术表达式，包括加（+）、减（-）、乘（*）、除（/），操作数可以是常量或列的值。示例：

```sql
SELECT loan_number, branch_name, amount * 100 FROM loan;
```

**详细讲解**  
- **算术表达式的位置**：算术表达式可以出现在 `SELECT` 子句中，作为结果集的一列。该列的值是根据每一行的列值计算得出的。
- **表达式示例**：`amount * 100` 表示将 `amount` 列的值乘以100。如果 `amount` 是数值类型，这个操作是合法的；如果 `amount` 是字符串或其他非数值类型，数据库会报错（或尝试隐式转换，取决于具体DBMS）。
- **运算符优先级**：SQL中的算术运算符遵循标准的数学优先级（乘除高于加减），可以使用括号改变运算顺序。例如：`(amount + 100) * 1.1`。
- **空值处理**：如果参与运算的列值为 `NULL`，那么整个算术表达式的结果也为 `NULL`。这是因为 `NULL` 表示未知值，任何与 `NULL` 的运算结果仍然是未知的。
- **实际应用**：算术运算常用于财务报表、价格计算、百分比增长等场景。例如，计算税后工资、折扣价格等。

### 3. 更复杂的算术运算示例：出生年份计算

**内容回顾**  
第三页PPT使用一个学生表（`Student`）展示了如何计算出生年份。表结构：`Student(sno, sname, sage, ssex, sdept)`。查询全体学生的姓名及其出生年份：

```sql
SELECT Sname, 2019 - Sage FROM Student;
```

输出结果示例（假设2019为当前年份）：
| Name |      |
| ---- | ---- |
| 李勇 | 1984 |
| 刘晨 | 1985 |
| 王敏 | 1986 |
| 张立 | 1985 |

**详细讲解**  
- **表达式 `2019 - Sage`**：这里 `Sage` 是学生的年龄，用当前年份减去年龄得到出生年份。这演示了如何在查询中利用算术运算生成新的信息。
- **结果列的默认名称**：在SQL中，如果没有为表达式指定别名，结果集中的列名通常是表达式本身（如 `2019-Sage`），这在某些数据库（如MySQL）中可能显示为 `2019-Sage`，而在其他数据库（如PostgreSQL）中可能是一个默认的列名如 `?column?`。为了可读性，通常建议为这类列指定别名（见下文）。
- **与关系代数的关系**：这种操作类似于在关系代数中先进行投影，但投影只能选择已有列，不能生成新列。SQL的 `SELECT` 子句允许包含任意表达式，因此比纯关系代数更灵活。

### 4. 结合字符串常量和函数

**内容回顾**  
第四页PPT进一步扩展了查询，添加了字符串常量 `'Year of Birth: '` 和函数 `LOWER(Sdept)`，要求用小写字母表示系名：

```sql
SELECT Sname, 'Year of Birth: ', 2000 - Sage, LOWER(Sdept)
FROM Student;
```

输出结果（输出表格标题有错位，但内容是清晰的）：
| 姓名 |                |      |      |
| ---- | -------------- | ---- | ---- |
| 李勇 | Year of Birth: | 1984 | cs   |
| ...  | ...            | ...  | ...  |

**详细讲解**  
- **字符串常量**：`'Year of Birth: '` 是一个字符串字面量，在结果集中作为一列出现，每一行的该列值都是相同的字符串。这可以用来添加静态描述文本。
- **函数 `LOWER(Sdept)`**：`LOWER` 是一个字符串函数，将传入的字符串转换为小写字母。这里将 `Sdept`（系名）转换为小写输出。类似地，还有 `UPPER` 函数用于转换为大写。不同的DBMS可能有不同的函数名（例如 `LCASE` 在某些数据库中可用），但 `LOWER` 是SQL标准的一部分。
- **结果列的顺序**：`SELECT` 子句中列出的项的顺序决定了结果集中列的顺序。此处依次是：学生姓名、字符串常量、计算出的出生年份、小写的系名。
- **输出表格的标题**：由于没有指定别名，结果集的列标题可能是表达式或函数名，可读性较差。下一张PPT将解决这个问题。

### 5. 使用列别名（Alias）改善结果标题

**内容回顾**  
第五页PPT演示了如何使用**列别名**为表达式或列指定更有意义的名称。示例：

```sql
SELECT Sname NAME, 'Year of Birth: ' BIRTH,
       2000 - Sage BIRTHDAY, LOWER(Sdept) DEPARTMENT
FROM Student;
```

输出结果（表格标题已被替换）：
| NAME | BIRTH          | BIRTHDAY | DEPARTMENT |
| ---- | -------------- | -------- | ---------- |
| 李勇 | Year of Birth: | 1984     | cs         |
| 刘晨 | Year of Birth: | 1985     | is         |
| 王敏 | Year of Birth: | 1986     | ma         |
| 张立 | Year of Birth: | 1985     | is         |

**详细讲解**  
- **列别名的语法**：可以在列名或表达式后紧跟一个别名，用空格分隔。也可以使用可选的 `AS` 关键字，例如 `Sname AS NAME`，这样更清晰，建议使用 `AS`。
- **别名的用途**：
  - 为表达式或函数的结果提供一个易于理解的列名。
  - 在结果集中重命名列，使其更符合输出要求。
  - 在多表连接中，如果不同表有相同列名，使用别名可以避免混淆。
  - 在子查询或视图定义中，别名可以为列指定名称。
- **别名的作用域**：列别名只在结果集中生效，不能在 `WHERE`、`GROUP BY` 或 `HAVING` 子句中直接引用（除非在支持这种引用的数据库如MySQL或PostgreSQL中，但这是非标准行为，不推荐依赖）。如果需要基于别名进行过滤，应该使用子查询或重复表达式。
- **示例分析**：PPT中的别名将 `Sname` 显示为 `NAME`，将字符串常量显示为 `BIRTH`，将计算表达式显示为 `BIRTHDAY`，将 `LOWER(Sdept)` 显示为 `DEPARTMENT`。这使得输出结果清晰易读。

### 补充知识点

1. **SQL中的表达式类型**  
   - 算术表达式：`+`, `-`, `*`, `/`
   - 字符串表达式：字符串连接（在SQL标准中使用 `||`，但某些数据库如MySQL使用 `CONCAT` 函数或 `+`）
   - 函数调用：`LOWER`, `UPPER`, `LENGTH`, `SUBSTR`, `ROUND` 等
   - 逻辑表达式：虽然不能在 `SELECT` 子句中直接返回布尔值（某些数据库支持），但可以用 `CASE` 表达式实现条件逻辑。

2. **数据类型兼容性**  
   当在表达式中混合不同数据类型时，数据库通常会进行隐式类型转换。但为了可移植性和避免意外，最好显式转换（如使用 `CAST` 或 `CONVERT`）。

3. **使用别名的最佳实践**  
   - 别名应简洁明了，反映列的含义。
   - 如果别名包含空格或特殊字符，需要用双引号括起来（例如 `"Year of Birth"`），但一般推荐使用下划线连接单词（如 `Year_of_Birth`）以避免引号。
   - 虽然 `AS` 关键字是可选的，但使用它可以使意图更清晰，尤其是在复杂查询中。

### 总结

这几页PPT涵盖了SQL查询中 `SELECT` 子句的核心扩展功能：
- **选择所有列**（`*`）方便快捷，但显式列出列名更规范。
- **算术运算**允许在查询时动态计算新值，增强了查询的表达力。
- **字符串常量**和**函数**可以丰富输出内容，实现数据格式化。
- **列别名**是提升查询结果可读性的重要工具。

掌握这些基础后，你就能编写更灵活、更实用的SQL查询，为后续学习更复杂的查询（如连接、分组、子查询等）打下坚实基础。
