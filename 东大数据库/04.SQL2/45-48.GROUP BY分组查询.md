## 第1页：GROUP BY 基本用法示例

**内容回顾**  
- 表结构：`Account(account-number, balance, branch-name)`
- 需求：查询每个分支的账户数量。
- SQL语句：
  ```sql
  SELECT branch_name, COUNT(account_number)
  FROM account
  GROUP BY branch_name;
  ```
- 图示：给出了一个 `account` 表的示例数据，并说明对每个 `branch_name` 分组，然后对每个组应用 `COUNT` 函数计算账户数。

**详细讲解**  
- **`GROUP BY` 的作用**：将表中的行按照指定列（或列组合）的值进行分组，值相同的行被分到同一组。然后，可以对每个组应用聚合函数（如 `COUNT`、`SUM`、`AVG` 等），得到每个组的汇总结果。
- **执行逻辑**：
  1. 先根据 `GROUP BY` 子句指定的列（这里是 `branch_name`）将表分成若干组。
  2. 对每个组独立地执行聚合函数（这里是 `COUNT(account_number)`），计算该组内 `account_number` 的非空值个数。
  3. 最后返回每个组的 `branch_name` 和对应的计数。
- **示例数据**：假设表中数据如下：

| branch_name | account_number | balance |
| ----------- | -------------- | ------- |
| Perryridge  | a-102          | 400     |
| Brighton    | a-217          | 750     |
| Perryridge  | a-201          | 900     |
| Brighton    | a-215          | 750     |
| Redwood     | a-222          | 700     |

分组后：
- Perryridge 组：包含 a-102 和 a-201，计数为 2。
- Brighton 组：包含 a-217 和 a-215，计数为 2。
- Redwood 组：包含 a-222，计数为 1。

结果：
| branch_name | COUNT(account_number) |
| ----------- | --------------------- |
| Perryridge  | 2                     |
| Brighton    | 2                     |
| Redwood     | 1                     |

- **注意**：`SELECT` 子句中只能出现**分组列**（即 `GROUP BY` 后的列）和**聚合函数**。如果尝试选择其他非聚合列且未出现在 `GROUP BY` 中，会导致错误（除非数据库支持非标准的功能依赖，但标准SQL不允许）。

## 第2页：GROUP BY 的细化说明

**内容回顾**  
- 分组：按指定的一列或多列值分组，值相等的为一组。
- 未分组时，聚合函数作用于整个查询结果。
- 分组后，聚合函数分别作用于每个组。

**详细讲解**  
- **未分组的情况**：如果查询中没有 `GROUP BY` 子句，那么整个查询结果被视为一个整体，聚合函数会对所有满足 `WHERE` 条件的行进行计算，返回一个单行结果。例如：
  ```sql
  SELECT COUNT(*) FROM account;
  ```
  返回整个表的行数。

- **分组的情况**：当使用 `GROUP BY` 后，查询结果被划分为多个组，聚合函数在每个组内独立计算，返回的行数等于组数。

- **多列分组**：可以按多列分组，例如按 `branch_name` 和 `account_type` 分组。语法：
  ```sql
  SELECT branch_name, account_type, COUNT(*)
  FROM account
  GROUP BY branch_name, account_type;
  ```
  此时，只有这些列的组合值完全相同的行才被分到同一组。

- **分组与 `WHERE` 的关系**：`WHERE` 子句在分组前执行，用于筛选参与分组的行。分组后如果需要过滤组，应使用 `HAVING` 子句（后续会讲到）。

## 第4页：分组查询示例

**内容回顾**  
- 展示了 `SC` 表的示例数据：

| Sno       | Cno  | Grade |
| --------- | ---- | ----- |
| 200215121 | 1    | 92    |
| 200215121 | 2    | 85    |
| 200215121 | 3    | 88    |
| 200215122 | 2    | 90    |
| 200215122 | 3    | 80    |

- 需求：求各个课程号及相应的选课人数。
- SQL语句：
  ```sql
  SELECT Cno, COUNT(Sno)
  FROM SC
  GROUP BY Cno;
  ```

**详细讲解**  
- **查询目标**：每个课程（Cno）有多少学生选修。这里统计的是选课记录数，即每个课程在 `SC` 表中出现的次数。
- **分组依据**：按 `Cno` 分组，相同课程号的记录归为一组。
- **聚合函数**：`COUNT(Sno)` 统计每组中 `Sno` 的非空值个数。由于 `Sno` 是主键的一部分，不会为 NULL，所以等价于 `COUNT(*)`。但通常统计选课人数时，如果一门课程有学生选课但成绩为 NULL，记录仍然存在，所以 `COUNT(Sno)` 和 `COUNT(*)` 结果相同。
- **结果分析**：
  - Cno=1 组：只有一条记录（200215121），计数为 1。
  - Cno=2 组：有两条记录（200215121 和 200215122），计数为 2。
  - Cno=3 组：有两条记录（200215121 和 200215122），计数为 2。
- **输出**：
  | Cno  | COUNT(Sno) |
  | ---- | ---------- |
  | 1    | 1          |
  | 2    | 2          |
  | 3    | 2          |

- **扩展思考**：如果要统计每个课程的平均成绩，可以使用 `AVG(Grade)`。但注意 `Grade` 可能为 NULL，`AVG` 会忽略 NULL，只计算有成绩的记录。

## 补充知识点与注意事项

### 1. `GROUP BY` 与 `SELECT` 列的规则
- 在标准 SQL 中，`SELECT` 子句中只能包含：
  - 分组列（即 `GROUP BY` 中出现的列）。
  - 聚合函数。
- 如果选择其他非分组列，数据库无法确定该列在组内取哪个值（除非该列在组内所有行中值相同，或数据库支持功能依赖，但为可移植性，应避免）。

### 2. `COUNT(*)` 与 `COUNT(列)` 的区别
- `COUNT(*)` 统计组内的总行数，包括所有列都为 NULL 的行。
- `COUNT(列)` 统计组内该列非 NULL 值的个数。
- 在分组查询中，通常用 `COUNT(*)` 统计每组记录数，用 `COUNT(列)` 统计有值的个数。

### 3. 多列分组
- 可以按多个列分组，例如按系和性别分组统计学生人数：
  ```sql
  SELECT Sdept, Ssex, COUNT(*)
  FROM Student
  GROUP BY Sdept, Ssex;
  ```

### 4. `HAVING` 子句
- 分组后如需对组进行筛选（例如只显示选课人数大于 1 的课程），应使用 `HAVING` 而不是 `WHERE`。
- 例如：
  ```sql
  SELECT Cno, COUNT(*)
  FROM SC
  GROUP BY Cno
  HAVING COUNT(*) > 1;
  ```
  `HAVING` 在分组后执行，可以使用聚合函数作为条件。

### 5. 执行顺序
- SQL 查询的逻辑执行顺序：
  1. `FROM` + `JOIN`：确定数据源。
  2. `WHERE`：过滤行。
  3. `GROUP BY`：分组。
  4. `HAVING`：过滤组。
  5. `SELECT`：选择列和计算表达式。
  6. `ORDER BY`：排序。
  7. `LIMIT`：限制行数。

理解这个顺序有助于正确编写查询。
