## 第一页：SQL 的历史与标准

### 1. SQL 的起源
- **IBM Sequel 语言**：20世纪70年代，IBM公司位于圣何塞的研究实验室开展了**System R项目**，该项目旨在验证关系数据库模型的可行性。在项目过程中，开发了一种名为**Sequel**（Structured English Query Language，结构化英语查询语言）的语言，用于对关系数据库进行数据定义和操纵。System R被认为是第一个实现关系数据库管理系统的原型，为后来的商业关系数据库（如Oracle、DB2等）奠定了基础。
- **更名为 SQL**：后来，由于法律商标方面的原因，Sequel 更名为 **Structured Query Language（结构化查询语言）**，缩写为 **SQL**。尽管名称改变，但许多人仍习惯将其读作“sequel”。

### 2. SQL 的标准化历程
随着SQL被越来越多的数据库厂商采用，不同产品之间的语法和功能差异逐渐显现，为了解决兼容性问题，ANSI（美国国家标准学会）和ISO（国际标准化组织）开始制定SQL标准。主要版本包括：

| 标准版本     | 发布时间 | 主要新增特性                                                 |
| ------------ | -------- | ------------------------------------------------------------ |
| **SQL-86**   | 1986     | 第一个SQL标准，定义了基本的SQL语法，包括数据定义、数据操纵和数据控制的基础功能。 |
| **SQL-89**   | 1989     | 小幅修订，增加了完整性约束（如PRIMARY KEY、FOREIGN KEY）等特性。 |
| **SQL-92**   | 1992     | 重大更新，极大地扩展了SQL的功能：支持更丰富的数据类型（如DATE、TIME）、多表连接（JOIN）、子查询、集合操作（UNION、INTERSECT、EXCEPT）、事务控制（COMMIT、ROLLBACK）等。SQL-92成为关系数据库的事实基础标准，几乎所有现代数据库都支持其核心特性。 |
| **SQL:1999** | 1999     | 引入了面向对象特性（如行类型、引用类型）、递归查询（WITH RECURSIVE）、触发器、正则表达式、新的数据类型（BOOLEAN、LOB）等。 |
| **SQL:2003** | 2003     | 增加了XML相关功能（SQL/XML）、窗口函数（OLAP函数）、序列生成器、IDENTITY列、MERGE语句等。 |
| **SQL:2008** | 2008     | 完善了TRUNCATE语句、FETCH子句（用于分页）、复合触发器、INSTEAD OF触发器等功能。 |

此后还有SQL:2011、2016、2019等版本，增加了JSON支持、多态表函数等新特性，但该页PPT仅列举到2008年。

## 第二页：商业系统对 SQL 标准的实现

### 1. SQL-92 的广泛支持
- **核心标准已普及**：目前所有主流的商业关系数据库管理系统（如 Oracle、Microsoft SQL Server、IBM Db2、MySQL、PostgreSQL 等）都实现了 **SQL-92** 的绝大部分甚至全部核心功能。这意味着如果你编写的 SQL 语句严格遵循 SQL-92 标准，那么它通常可以在不同的数据库系统之间顺利移植。
- **后续标准的差异化**：对于 SQL:1999、SQL:2003 及以后的新特性，不同厂商的支持程度各不相同。例如，窗口函数（SQL:2003）现在被大多数主流数据库支持，但某些较新的特性（如 JSON 功能）可能只有部分数据库支持，且实现方式可能存在差异。

### 2. 专有扩展与兼容性问题
- **专有特性**：为了提升性能或提供独特功能，每个数据库厂商都会开发自己的扩展语法和功能。例如：
  - Oracle 的 PL/SQL、层次查询（CONNECT BY）
  - SQL Server 的 T-SQL、TOP 子句
  - MySQL 的 LIMIT 子句、存储过程语法
  - PostgreSQL 的丰富数据类型和索引方法
- **示例可能不通用**：教材或课程中给出的示例代码，如果使用了某个特定数据库的专有语法或较新的标准特性，那么在你的数据库系统上可能无法直接运行。因此，在学习和实践时，需要根据你所使用的数据库查阅相应的文档，必要时对示例进行调整。

## 第三页：SQL 的核心特性

### 1. 非过程化（Non‑procedural）
- **定义**：SQL 是一种**非过程化语言**，也称为**描述性语言**。这意味着在使用 SQL 进行数据操作时，用户只需**声明“做什么”**（what to do），而不需要指定“如何做”（how to do it）。
- **对比过程化语言**：传统的过程化语言（如 C、Java、PL/SQL）需要编写详细的步骤（循环、条件分支等）来逐步实现任务。而 SQL 则由数据库系统的**查询优化器**自动决定最优的执行路径（如使用哪个索引、以什么顺序连接表等）。
- **示例**：要查询所有年龄大于 30 的员工，只需写 `SELECT * FROM employees WHERE age > 30;`，无需关心数据库如何扫描数据。

### 2. 简单易用
- **核心词汇仅九个**：SQL 的核心功能可以仅用**九个关键字**（或称为动词）来概括，这体现了其简洁性。这九个词分别是：
  - **DDL（数据定义语言）**：`CREATE`、`DROP`、`ALTER`（用于定义、删除、修改数据库对象）
  - **DML（数据操纵语言）**：`SELECT`、`INSERT`、`UPDATE`、`DELETE`（用于查询和修改数据）
  - **DCL（数据控制语言）**：`GRANT`、`REVOKE`（用于权限管理）
- 当然，SQL 的实际词汇远不止这些（还包括 `TRUNCATE`、`COMMIT`、`ROLLBACK` 等），但这九个动词构成了 SQL 最基本、最常用的功能骨架。

### 3. SQL 的主要内容分类
SQL 按照功能通常分为三大类：

| 分类    | 全称                                       | 作用                                                     | 主要命令                               |
| ------- | ------------------------------------------ | -------------------------------------------------------- | -------------------------------------- |
| **DDL** | Data Definition Language（数据定义语言）   | 定义数据库的结构（创建、修改、删除表、视图、索引等对象） | `CREATE`、`DROP`、`ALTER`              |
| **DML** | Data Manipulation Language（数据操纵语言） | 对数据库中的数据进行查询和更新                           | `SELECT`、`INSERT`、`UPDATE`、`DELETE` |
| **DCL** | Data Control Language（数据控制语言）      | 管理用户权限和访问控制                                   | `GRANT`、`REVOKE`                      |

- **DDL 示例**：
  ```sql
  CREATE TABLE students (id INT, name VARCHAR(50));  -- 创建表
  ALTER TABLE students ADD COLUMN age INT;            -- 添加列
  DROP TABLE students;                                 -- 删除表
  ```
- **DML 示例**：
  ```sql
  SELECT * FROM students WHERE age > 18;               -- 查询
  INSERT INTO students (id, name) VALUES (1, '张三');  -- 插入
  UPDATE students SET age = 20 WHERE id = 1;           -- 更新
  DELETE FROM students WHERE id = 1;                    -- 删除
  ```
- **DCL 示例**：
  ```sql
  GRANT SELECT ON students TO user1;                    -- 授予查询权限
  REVOKE INSERT ON students FROM user1;                  -- 撤销插入权限
  ```

## 总结
这三页 PPT 从历史发展、标准实现到语言特性，全面介绍了 SQL：
- **历史与标准**：SQL 起源于 IBM 的 System R 项目，经过多年标准化，功能不断丰富。
- **商业实现**：大多数数据库都支持 SQL-92 核心，但对后续标准和专有扩展的支持存在差异，使用时需注意兼容性。
- **核心特性**：SQL 是一种非过程化、简洁的语言，其核心功能由 DDL、DML、DCL 三部分构成，仅用九个基本动词即可完成主要任务。

理解这些内容，有助于你从宏观上把握 SQL 的本质，并在实际开发中更灵活地运用它。
