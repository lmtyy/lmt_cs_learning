### 1. Some Concepts（基本概念）
这部分主要定义“什么是数据库”以及相关的几个易混淆术语：

- **数据**：描述事物的符号记录。
- **数据库**：长期存储在计算机内、有组织、可共享的大量数据的集合。
- **数据库管理系统**：位于用户与操作系统之间的一层数据管理软件（如 MySQL）。
- **数据库系统**：由数据库、DBMS、应用程序和数据库管理员组成的整体。

### 2. Purpose of Database Systems（数据库系统的目的/优点）
主要解释**为什么不用文件系统存数据，而要用数据库**：

- **数据冗余度低、共享性高**：避免同一个数据在多个文件里重复存（比如学生信息在教务处和财务处各存一份导致不一致）。
- **数据独立性高**：物理存储变了，应用程序不用改（物理独立性）；逻辑结构变了，应用程序也不用大改（逻辑独立性）。
- **统一控制**：提供**安全性**（权限控制）、**完整性**（数据正确性约束）、**并发控制**（多人在线订票不出错）和**恢复**（断电数据不丢）。

### 3. Data Model（数据模型）
这是数据库的**核心与灵魂**，PPT 会重点讲以下分层抽象：

- **两类模型**：
    - **概念模型**：按用户观点建模，典型代表是 **E-R 模型**（实体-联系图）。
    - **逻辑/物理模型**：
        - **层次模型**（树状，如 IBM IMS）
        - **网状模型**（图状）
        - **关系模型**（二维表，**当前主流**）
        - **面向对象模型**、**半结构化模型**（XML/JSON）

### 4. Database Architecture（数据库体系结构）
这是必考点，即 **三级模式两级映像**：

- **三级模式**：
    - **外模式**：用户看到的局部数据视图（视图）。
    - **模式**：数据库全体数据的逻辑结构和特征（表结构定义）。
    - **内模式**：数据物理存储方式（索引、文件存储路径）。
- **两级映像**：
    - 外模式/模式映像：保证**逻辑独立性**。
    - 模式/内模式映像：保证**物理独立性**。

### 5. Database Users and Administrators（用户与管理员）
- **数据库管理员**：决定数据库结构和策略，负责监控和优化。
- **专业用户/数据库设计者**：写复杂查询的工程师、架构师。
- **应用程序员**：写 Java/Python 代码调用数据库的人。
- **最终用户**：通过界面点鼠标查询报表的人。

### 6. Overall Structure（总体结构）
这里通常指 DBMS 的内部**功能组件结构**，一张典型的流程图包括：
查询处理器（解析、优化） -> 存储管理器（文件、缓冲区） -> 磁盘存储。
