### 第一页：数据库语言总览

> **PPT 内容：**
> - Database system provides
>   - Data Definition Language (DDL) to specify database schema
>   - Data Manipulation Language (DML) to express database queries and updates

**知识点详解：**

#### 1. 为什么需要“数据库语言”？

- 数据库是一个**软件系统**，用户和应用程序需要一种**标准化的方式**与它沟通。
- 就像你需要用 **SQL** 告诉数据库“建个表、存点数据、查个报表”，而不能直接用鼠标在硬盘上画表格。
- 数据库语言就是**人与数据库之间的通信协议**。

#### 2. 两大类语言的分工

| 语言类型 | 全称                       | 作用对象 | 核心任务                     | 类比                                         |
| :------- | :------------------------- | :------- | :--------------------------- | :------------------------------------------- |
| **DDL**  | Data Definition Language   | **模式** | 定义和修改数据库的**结构**   | 盖楼时画**设计图纸**（几室几厅、承重墙位置） |
| **DML**  | Data Manipulation Language | **实例** | 查询和更新数据库中的**数据** | 入住后**搬家具、换装饰、查人数**             |

**核心认知**：
- DDL 操作的是**空架子**（比如创建一个空表，定义它有几列）。
- DML 操作的是**架子里的东西**（往表里插入一行行数据，修改某行的值）。

### 第二页：数据定义语言 语法示例

> **PPT 内容：**
> - Specification notation for defining the database schema
> - Example:
>   ```sql
>   create table account (
>       account-number char(10),
>       balance integer
>   )
>   ```

**知识点详解：**

#### 1. DDL 是“规范性符号体系”

- “Specification notation” 意味着 DDL 有一套严格的**语法规则**。
- 你写 `CREATE TABLE` 而不是 `MAKE TABLE`，你写 `CHAR(10)` 而不是 `STRING(10)`。
- 这套规则由 **SQL 标准** 定义，各大数据库厂商（Oracle, MySQL）基本遵循。

#### 2. 示例语句的逐成分解析

| 成分          | 示例内容                  | 含义                                                         | 涉及的数据模型三要素              |
| :------------ | :------------------------ | :----------------------------------------------------------- | :-------------------------------- |
| **命令动词**  | `create table`            | 告诉 DBMS 要**创建一个关系**                                 | 数据结构                          |
| **关系名**    | `account`                 | 关系的**名称**，未来在查询中使用                             | 数据结构                          |
| **属性定义1** | `account-number char(10)` | 定义一个名为 `account-number` 的列，数据类型为**定长字符串**（最多10个字符） | 数据结构 + 完整性约束（类型约束） |
| **属性定义2** | `balance integer`         | 定义一个名为 `balance` 的列，数据类型为**整数**              | 数据结构 + 完整性约束             |

#### 3. DDL 执行后的立即效果

当你在数据库客户端执行这行语句后：
- 数据库的**数据字典**里会多一条记录，描述 `account` 表的结构。
- 磁盘上可能会**预分配**一个初始大小的文件空间给这个表（虽然表还是空的）。
- 此后，你可以对这个表执行 DML 操作（插入数据）。

### 第三页：DDL 编译器与数据字典

> **PPT 内容：**
> - DDL compiler generates tables according to DDL statements and store the schema definitions of these tables in **data dictionary**.
> - Data dictionary contains metadata (i.e., data about data)
>   - Database schema
>   - Integrity constraints
>   - Authorization

**知识点详解：**

#### 1. DDL 编译器的工作流程

```
用户输入 DDL 语句
        ↓
   [DDL 编译器]
        ↓
  ┌─────┴─────┐
  ↓           ↓
生成物理表      更新数据字典
（分配空间）    （记录元数据）
```

- **DDL 编译器**是 DBMS 内部的一个模块，负责解析和执行 DDL 语句。
- 它做的事情比普通编译器简单：主要是**翻译成内部格式**并**更新系统表**。

#### 2. 数据字典 —— 数据库的“元数据中心”

| 对比项   | 普通数据                         | 元数据                                                    |
| :------- | :------------------------------- | :-------------------------------------------------------- |
| 内容     | 客户的姓名、账户余额             | `account` 表有几列、每列什么类型、谁有权访问              |
| 存储位置 | 用户表空间（如 `user_data.dbf`） | 系统表空间（如 `system.dbf` 或 `information_schema`）     |
| 谁写入   | 用户通过 DML 写入                | **DBMS 自动维护**（DDL 触发）                             |
| 谁读取   | 用户通过 SELECT 读取             | 用户**可以**查询（如 `SHOW TABLES`），DBMS 内部也频繁读取 |

#### 3. 数据字典中的三类元数据（对应 PPT 三点）

| 类别           | 具体存储内容举例                                             | 作用                                                         |
| :------------- | :----------------------------------------------------------- | :----------------------------------------------------------- |
| **数据库模式** | 表名 = `account`，列名 = `account-number`，类型 = `char(10)`，列名 = `balance`，类型 = `integer` | 当用户执行 `SELECT * FROM account` 时，DBMS 先去数据字典查出 `*` 代表哪两列，再去取数据。 |
| **完整性约束** | 主键约束（`account-number` 唯一非空）、检查约束（`balance >= 0`） | 当用户插入 `balance = -100` 时，DBMS 检查数据字典中的约束，**拒绝执行**。 |
| **授权信息**   | 用户 `Alice` 对 `account` 表有 `SELECT` 权限；用户 `Bob` 有 `SELECT, INSERT` 权限。 | 当用户执行任何操作时，DBMS 先查数据字典，**无权限则拒绝**。  |

#### 4. 为什么数据字典如此重要？

- **实现数据独立性**：应用程序可以动态查询数据字典获取表结构，而不必将列信息写死在代码里。
- **支持系统管理**：DBA 通过查询数据字典了解数据库整体状况（有多少表、多少索引、多少用户）。
- **保证安全与完整**：所有约束和权限检查都依赖数据字典。

### 第四页：数据操纵语言 的定义与四大功能

> **PPT 内容：**
> - Language for accessing and manipulating the data organized by the appropriate data model
> - Retrieval of information
> - Insertion of new information
> - Deletion of information
> - Modification of information
> - DML also known as query language

**知识点详解：**

#### 1. DML 的操作对象是“由数据模型组织的数据”

- 这句话强调：DML 不是直接操作磁盘上的字节，而是操作**已经按某种数据模型（如关系模型）组织好的逻辑数据**。
- 在关系数据库中，DML 操作的是**表**（关系）和**行**（元组）。

#### 2. 四大核心功能详解

| 功能     | SQL 关键字 | 典型语法示例                                                 | 对数据库状态的影响                       |
| :------- | :--------- | :----------------------------------------------------------- | :--------------------------------------- |
| **检索** | `SELECT`   | `SELECT balance FROM account WHERE account-number = 'A-101';` | **只读**，不改变任何数据                 |
| **插入** | `INSERT`   | `INSERT INTO account VALUES ('A-106', 1000);`                | 增加新元组，**数据库实例变大**           |
| **删除** | `DELETE`   | `DELETE FROM account WHERE balance < 100;`                   | 移除元组，**数据库实例变小**             |
| **修改** | `UPDATE`   | `UPDATE account SET balance = balance * 1.05;`               | 改变已有元组的属性值，**数据库状态改变** |

#### 3. 为什么 DML 也叫“查询语言”？

- **历史原因**：早期数据库的主要用途是查询（Query），所以人们习惯用“查询语言”泛指所有数据操作语言。
- **术语辨析**：
    - **Query** 严格来说仅指 `SELECT` 检索操作。
    - **Query Language** 在日常语境中常等同于 DML。
    - **SQL** 的名字就叫 **Structured Query Language**（结构化查询语言），虽然它包含了 DDL、DML、DCL。
- 所以，PPT 说 *"DML also known as query language"* 是一种**宽泛的习惯说法**。

### 第五页：DML 的两大类别 —— 过程式 vs 声明式

> **PPT 内容：**
> - Two classes of languages
>   - **Procedural** – user specifies what data is required and **how** to get those data
>   - **Declarative (nonprocedural)** – user specifies **what** data is required **without specifying how** to get those data
> - **SQL** is the most widely used nonprocedural query language

**知识点详解：**

这一页是理解数据库语言**进化史**和**易用性**的关键。

#### 1. 过程式 DML —— “自己动手，丰衣足食”

| 维度         | 详细说明                                                     |
| :----------- | :----------------------------------------------------------- |
| **核心特征** | 用户必须编写**详细的执行步骤**，告诉数据库**如何**一步步找到所需数据。 |
| **操作粒度** | **记录级**。一次操作只处理一条记录，需要用循环遍历。         |
| **典型代表** | 层次数据库的 `GET NEXT`，网状数据库的 `FIND` + `NEXT`。      |
| **类比**     | 你去图书馆查书，管理员告诉你：“书架在 3 楼，编号 A 区，你自己去一本本翻吧。” |
| **优缺点**   | 优点：程序员可精细控制访问路径，理论上可以榨干性能。<br>缺点：代码量巨大，维护困难，数据独立性差。 |

**过程式代码示例（伪代码）：**
```pseudo
// 查余额大于 500 的账户对应的客户姓名
FIND FIRST account WHERE balance > 500   // 找到第一笔符合条件的账户
DO WHILE (found) {
    FIND OWNER OF depositor              // 沿着指针链找到客户
    PRINT customer.name
    FIND NEXT account WHERE balance > 500 // 找下一笔
}
```

#### 2. 声明式 DML —— “动口不动手”

| 维度         | 详细说明                                                     |
| :----------- | :----------------------------------------------------------- |
| **核心特征** | 用户**只描述想要什么结果**，**不指定**执行路径。DBMS 自动决定**如何**执行。 |
| **操作粒度** | **集合级**。一条语句处理一批数据。                           |
| **典型代表** | **SQL**。                                                    |
| **类比**     | 你去图书馆查书，告诉管理员：“我要《数据库系统概念》。” 管理员自己去系统里查索书号、去书架上拿、递给你。 |
| **优缺点**   | 优点：代码简洁，易学易用，数据独立性极高。<br>缺点：极端情况下，DBMS 自动生成的执行计划可能不如专家手工优化（但现在优化器已经非常强大）。 |

**声明式代码示例（SQL）：**
```sql
SELECT customer.customer_name
FROM customer, depositor, account
WHERE customer.customer_id = depositor.customer_id
  AND depositor.account_number = account.account_number
  AND account.balance > 500;
```
你只说了**三表怎么关联、筛选条件是什么**。至于先查哪个表、用不用索引、怎么连接，由**查询优化器**自动完成。

#### 3. SQL 为何成为“最广泛使用的非过程化查询语言”？

- **数据独立性**：物理结构变了（比如加了索引），SQL 语句不用改，程序不用重编译。
- **标准化**：ANSI/ISO SQL 标准使得跨数据库移植相对容易。
- **表达能力强**：几行 SQL 可以完成非常复杂的集合运算和统计。
- **易学性**：非程序员（如数据分析师）也能快速上手。

### 核心术语总结

| 术语           | 一句话定义                                                   |
| :------------- | :----------------------------------------------------------- |
| **DDL**        | 定义数据库**结构**的语言，如 `CREATE`、`ALTER`、`DROP`。     |
| **DML**        | 操作数据库**数据**的语言，如 `SELECT`、`INSERT`、`UPDATE`、`DELETE`。 |
| **数据字典**   | 存储**元数据**（表结构、约束、权限）的系统表集合，由 DBMS 自动维护。 |
| **过程式 DML** | 需指定 **“做什么”** 和 **“怎么做”**，如层次数据库的导航语句。 |
| **声明式 DML** | 只需指定 **“做什么”**，由 DBMS 决定 **“怎么做”**，如 SQL。   |
| **SQL**        | 结构化查询语言，是**声明式 DML** 的事实标准，同时包含 DDL 和 DCL 功能。 |
