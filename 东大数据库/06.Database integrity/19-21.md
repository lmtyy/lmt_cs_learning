## 第一页PPT：存储过程和函数概述

### 1. 共同目标：提高数据库性能
- **减少频繁访问数据库**：将一组SQL语句封装在数据库端，应用程序只需一次调用即可完成多个操作，避免了多次网络往返。
- **减少网络延迟**：由于代码在数据库服务器上执行，只有调用命令和结果在网络中传输，大量中间数据无需来回传递，显著降低延迟。
- **加速执行效率**：存储过程和函数在首次执行时被编译并优化，后续调用直接使用编译后的版本，节省了重复解析和编译的时间。

### 2. 适用场景分工
- **函数（Function）**：更适合**简单的计算型任务**，例如：
  - 字符串拼接、格式化
  - 日期计算（如计算年龄、日期差）
  - 数学运算
  - 返回单个值（标量值）或表对象
- **存储过程（Stored Procedure）**：适合**复杂的业务逻辑**，例如：
  - 涉及多张表的更新、删除操作
  - 需要事务控制的流程
  - 包含条件判断、循环等复杂控制结构
  - 需要返回多个结果集或多个输出参数

这种分工体现了“各司其职”的设计思想：函数聚焦于计算和返回值，存储过程聚焦于动作和流程。

## 第二页PPT：创建和调用函数

### 1. 函数的基本语法格式
```sql
CREATE FUNCTION 函数名 ([参数, 参数...])
RETURNS 数据类型
[特性 ...]
BEGIN
    -- 函数体
    RETURN 值;
END
```
- **参数列表**：函数的参数只能是**IN类型**（默认，无需显式指定），即输入参数，不能有OUT或INOUT。
- **RETURNS子句**：必须指定函数返回值的**数据类型**（如INT、VARCHAR、DATE等），这是函数的声明，表示“这个函数会返回什么类型”。
- **RETURN语句**：在函数体内，必须至少有一个`RETURN 值;`语句，用于实际返回计算结果。注意`RETURNS`（带S）是声明部分，`RETURN`（不带S）是执行语句。

### 2. 示例：创建函数返回学生某门课程的成绩
```sql
DELIMITER //
CREATE FUNCTION FUNC_STU_COU_GRADE(student_id CHAR(10), course_id CHAR(4))
RETURNS INT
BEGIN
    DECLARE grade_val INT;
    SELECT Grade INTO grade_val FROM SC WHERE Sno = student_id AND Cno = course_id;
    RETURN grade_val;
END //
DELIMITER ;
```
**说明**：
- 函数名：`FUNC_STU_COU_GRADE`
- 参数：两个IN参数，学号和课程号
- 返回类型：`INT`（假设成绩为整数）
- 函数体：先声明一个局部变量`grade_val`，然后用`SELECT ... INTO`将查询到的成绩存入该变量，最后用`RETURN`返回。如果查询无结果，`grade_val`将为NULL，函数返回NULL。
- **注意**：函数体内不允许执行导致数据变化的操作（如INSERT、UPDATE、DELETE），否则会报错。

### 3. 调用函数
函数不能像存储过程那样用`CALL`调用，而是作为表达式的一部分，通常用在SELECT语句中：
```sql
SELECT FUNC_STU_COU_GRADE('2021310722', '1') AS 成绩;
```
也可以用在WHERE子句、SET赋值等任何允许表达式的地方：
```sql
SET @score = FUNC_STU_COU_GRADE('2021310722', '1');
```

### 4. 关于RETURNS和RETURN的强调
- `RETURNS`：定义函数返回类型，位于函数头。
- `RETURN`：在函数体中实际返回值，可以出现在多个分支中（如IF-ELSE），但最终只有一个值被返回。

## 第三页PPT：存储过程和函数的区别

这一页列出了五点核心区别，下面逐一详细解释：

### 1. **返回值的方式不同**
- **存储过程**：通过**OUT或INOUT参数**返回多个值，也可以返回多个结果集（通过SELECT直接输出）。它没有“返回值”的概念，调用后可以通过输出参数获取数据。
- **函数**：只能通过**RETURNS子句**指定返回一个**单值**（标量值）或**表对象**（在某些数据库中，如SQL Server的表值函数）。函数体内必须有`RETURN`语句返回这个值。

**示例对比**：
- 存储过程返回平均分：`CALL proc_get_avg('1', @avg);`
- 函数返回平均分：`SELECT func_get_avg('1');`

### 2. **参数类型限制**
- **存储过程**：参数可以是 **IN、OUT、INOUT**，允许数据传入、传出或双向传递。
- **函数**：参数只能是 **IN**（输入）类型，不能用作输出。函数的所有“输出”都通过返回值实现，参数仅用于传入数据。

### 3. **调用方式不同**
- **存储过程**：使用 `CALL` 语句独立执行，不能直接嵌套在SQL表达式中（除非通过某些特殊方式，如MySQL的`SELECT proc()`，但不推荐且有限制）。
- **函数**：可以像内置函数一样**嵌入在SQL语句**中，例如在SELECT列表、WHERE条件、SET赋值等处调用。此外，如果函数返回表对象（如MySQL的`RETURNS TABLE`，或SQL Server的内联表值函数），还可以在FROM子句中作为虚拟表使用。

**示例**：在FROM子句中使用表值函数（假设数据库支持）
```sql
SELECT * FROM TABLE(func_get_students_by_course('1'));
```

### 4. **创建时的强制要求**
- **函数**：创建时必须使用`RETURNS`指定返回值数据类型，且函数体内必须至少包含一个`RETURN`语句，否则会报错。
- **存储过程**：没有返回值类型的声明，也不需要`RETURN`语句（虽然过程体内可以使用`RETURN`提前退出，但并非必须）。

### 5. **对数据库的修改权限**
- **存储过程**：可以执行**数据更新操作**（INSERT、UPDATE、DELETE）以及DDL操作（如CREATE TABLE），适合实现业务逻辑。
- **函数**：通常**不允许执行修改数据的操作**（如INSERT、UPDATE、DELETE）。函数的设计初衷是计算和查询，保持“无副作用”，以确保在SQL语句中调用时不会意外改变数据库状态。大多数数据库系统（如MySQL、Oracle）严格禁止函数内执行写操作，否则会报错。

## 补充说明：存储过程与函数的相似之处
尽管有上述区别，它们也有共同点：
- 都是存储在数据库中的SQL代码块，可以重复使用。
- 都能接收参数，并包含流程控制语句（IF、LOOP等）。
- 都能提高性能（减少网络交互、编译一次多次运行）。
- 都能增强安全性（通过权限控制访问底层表）。

---

## 总结
- **存储过程**适合复杂的业务操作，可以返回多个值或结果集，能修改数据。
- **函数**适合计算和查询任务，只能返回单个值（或表），不能修改数据，调用灵活。
- 在实际开发中，应根据需求选择合适的技术：需要修改数据、多步骤业务逻辑时用存储过程；需要封装计算、简化查询时用函数。
