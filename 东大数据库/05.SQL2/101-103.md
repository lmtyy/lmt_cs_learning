## 第1题：检索学过001号老师主讲所有课程的学生的姓名

**表结构**  
- Student(S#, sname, ...)  
- Course(C#, cname, T#, ...)  
- SC(S#, C#, score)  
- Teacher(T#, tname, ...)  

**需求**  
找出选修了001号教师所讲授的**所有**课程的学生姓名。

### SQL语句
```sql
SELECT Sname
FROM Student
WHERE NOT EXISTS (
    SELECT *
    FROM Course
    WHERE Course.T# = '001'
      AND NOT EXISTS (
          SELECT *
          FROM SC
          WHERE S# = Student.S# AND C# = Course.C#
      )
);
```

### 逻辑解析（双重否定）

这个查询采用了标准的**双重NOT EXISTS**模式，用于表达“对所有……都满足”。

1. **最内层子查询**：
   ```sql
   SELECT * FROM SC
   WHERE S# = Student.S# AND C# = Course.C#
   ```
   检查当前学生（Student.S#）是否选修了当前课程（Course.C#）。如果有记录，说明学生选修了这门课。

2. **中间层NOT EXISTS**：
   ```sql
   NOT EXISTS (最内层子查询)
   ```
   它的含义是：**对于当前课程，不存在该学生的选课记录**，即该学生**没有**选修这门课。这个子查询是相关的，依赖于外层的学生（来自最外层Student）和内层循环的课程（来自Course表）。

3. **最外层NOT EXISTS**：
   ```sql
   NOT EXISTS (
       SELECT * FROM Course
       WHERE Course.T# = '001'
         AND (中间层NOT EXISTS为真)
   )
   ```
   它的含义是：**不存在这样一门001号老师教的课程，使得该学生没有选修它**。  
   换句话说，该学生选修了001号老师教的**所有**课程。

### 执行过程（示例）
假设：
- 001号老师教了C1和C2两门课。
- 学生A选修了C1和C2 → 对于C1和C2，中间层NOT EXISTS都为假，最外层子查询无返回 → 最外层NOT EXISTS为真 → 学生A被选中。
- 学生B只选修了C1 → 对于C2，中间层NOT EXISTS为真（因为没选C2），最外层子查询返回C2 → 最外层NOT EXISTS为假 → 学生B被排除。
- 学生C一门都没选 → 对于C1和C2，中间层都为真，最外层子查询返回两门课 → 最外层NOT EXISTS为假 → 学生C被排除。

### 关键点
- 这里用到了两次NOT EXISTS，内层否定表示“没选某课”，外层否定表示“不存在没选的课”。
- 注意表之间的关联：最内层引用了Student和Course，它们分别来自最外层和中间层。

## 第2题：检索没学过李明老师讲授的任意一门课程的学生姓名

**需求**  
找出那些**没有**选修过李明老师所讲授的**任何一门**课程的学生。  
注意：“任意一门”指的是“至少一门”，所以“没学过任意一门”等于“一门都没学过”。

### SQL语句
```sql
SELECT Sname
FROM Student
WHERE NOT EXISTS (
    SELECT *
    FROM Course, SC, Teacher
    WHERE Tname = '李明'
      AND Course.T# = Teacher.T#
      AND Course.C# = SC.C#
      AND S# = Student.S#
);
```

### 逻辑解析
- 子查询要找出**该学生选修过的、且是李明老师教的课程**。它通过连接Course、SC、Teacher表，并限定Tname='李明'和S#=Student.S#。
- 如果子查询返回至少一行，说明该学生至少选修了一门李明的课。
- 外层NOT EXISTS则取反：如果子查询为空，说明该学生没有选修任何李明的课，满足条件。

### 与第1题的区别
- 第1题是“学过所有”，用双重否定。
- 第2题是“没学过任何”，直接用NOT EXISTS + 存在性子查询即可。
- 注意第2题的子查询是一个**相关子查询**，因为它引用了外层的Student.S#。

### 可能的问题
- 如果李明老师没有教任何课程，那么子查询对所有学生都返回空，所有学生都会被选中。这符合逻辑吗？如果李明没教课，那么“没学过李明老师的课”自然对所有人都成立，结果正确。
- 如果学生没有选任何课，那么子查询也不会有结果，该学生被选中，也正确。

## 第3题：有两门以上不及格课程的同学学号和平均成绩

**需求**  
找出那些**不及格课程（score<60）数量超过2门**的学生，并输出他们的学号和**平均成绩**（注意：平均成绩是**所有课程**的平均，还是仅不及格课程的平均？从题意看，通常是指该学生的所有课程的平均成绩）。

### SQL语句
```sql
SELECT S#, AVG(score)
FROM sc
WHERE S# IN (
    SELECT S#
    FROM sc
    WHERE score < 60
    GROUP BY S#
    HAVING COUNT(*) > 2
)
GROUP BY S#;
```

### 逻辑解析
1. **子查询**（找出符合条件的学号）：
   - 从sc表中筛选出成绩不及格的记录（score<60）。
   - 按学号分组。
   - 用HAVING COUNT(*) > 2过滤出那些不及格课程数大于2的学生学号。
   - 注意：这里COUNT(*)统计的是每个学生不及格课程的条数。

2. **外层查询**：
   - 从sc表中选取所有记录，但只限于学号出现在子查询结果中的学生。
   - 按学号分组，计算每个学生的平均成绩（AVG(score)）。这里的平均成绩基于该学生的**所有**选课记录，而不仅仅是及格的课程。
   - 输出学号和平均成绩。

### 示例
假设学生A有选课记录：
- 课程1：55（不及格）
- 课程2：45（不及格）
- 课程3：30（不及格）
- 课程4：80（及格）
子查询会选中学生A（因为3门不及格>2），外层计算平均成绩：(55+45+30+80)/4 = 52.5。

### 注意点
- **平均成绩的范围**：外层GROUP BY是对所有选课记录求平均，包括及格的课程。如果只想统计不及格课程的平均成绩，则需要在子查询中直接计算并输出，但题目明确写了“平均成绩”，通常指所有课程的平均。
- **去重**：如果学生有多条选课记录，学号自然会在分组后唯一。
- **HAVING的位置**：HAVING是在分组后过滤组，这里正确。
- **空值处理**：如果某学生有成绩为NULL，AVG会忽略NULL，但COUNT(*)会统计所有行，包括NULL。如果NULL代表缺考，是否算不及格？题目未明确，通常score<60只比较非空值，NULL不会计入不及格，也不会被AVG计算。

### 改进建议
如果要求平均成绩只针对不及格课程，可以这样写：
```sql
SELECT S#, AVG(score)
FROM sc
WHERE score < 60
GROUP BY S#
HAVING COUNT(*) > 2;
```
这样就直接在子查询中完成分组和平均，但外层就不需要了。不过原题可能期望的是所有课程的平均。

## 总结

| 题目  | 考察重点         | 关键技巧                                   |
| ----- | ---------------- | ------------------------------------------ |
| 第1题 | 全称量词（所有） | 双重NOT EXISTS，将“所有”转化为“不存在反例” |
| 第2题 | 存在性否定       | 单层NOT EXISTS + 相关子查询                |
| 第3题 | 分组后筛选       | GROUP BY + HAVING，子查询与主查询结合      |

这三道题覆盖了SQL中比较复杂的查询模式，掌握它们对处理实际业务中的“包含”、“排除”、“分组统计”等需求非常有帮助。
