这几页PPT主要介绍了SQL中的**聚合函数**，它们用于对一组值进行计算并返回一个单一的结果。下面我将逐一详细讲解每一页的知识点，并补充一些重要的注意事项和扩展内容。

---

## 第1页：聚合函数概述

**内容回顾**  
- 聚合函数作用于**关系（表）中某一列的值**，并返回一个**单一的值**。
- 列举了五种基本聚合函数：
  - **avg**：计算某列的平均值（仅适用于数值列）。
  - **min**：找出某列的最小值（适用于数值、字符串、日期等可比较类型）。
  - **max**：找出某列的最大值。
  - **sum**：计算某列的总和（仅适用于数值列）。
  - **count**：统计某列中值的个数（或行数）。

**详细讲解**  
- **作用**：聚合函数将多行数据压缩为一行汇总信息，常用于报表生成、统计分析等场景。
- **语法**：通常出现在 `SELECT` 子句中，例如：
  ```sql
  SELECT AVG(balance) FROM account;
  ```
- **NULL值的处理**：除了 `COUNT(*)` 外，其他聚合函数都会**忽略** `NULL` 值。例如，`AVG` 只计算非空值的平均值，`SUM` 忽略空值，`COUNT(column)` 只统计该列非空值的个数。而 `COUNT(*)` 统计的是表中的所有行数，包括某些列全为 NULL 的行。
- **DISTINCT 与聚合函数**：可以在聚合函数中使用 `DISTINCT` 关键字来排除重复值，例如 `COUNT(DISTINCT customer_name)` 统计不同客户的数量。对于 `AVG`、`SUM` 等，使用 `DISTINCT` 也会先去除重复值再计算。

## 第2页：聚合函数计算示例

**内容回顾**  
- 表结构：`Account(account-number, balance, branch-name)`
- 需求：查询 Perryridge 分行的平均账户余额。
- SQL语句：
  ```sql
  SELECT AVG(balance)
  FROM account
  WHERE branch_name = 'Perryridge';
  ```
- 图示说明：先通过 `WHERE` 筛选出 Perryridge 分行的所有账户，再对这些账户的 `balance` 列应用 `AVG` 函数，最后返回平均值（例如 120,000）。

**详细讲解**  
- **执行逻辑**：
  1. `FROM` + `WHERE`：从 `account` 表中筛选出 `branch_name = 'Perryridge'` 的行。
  2. `SELECT AVG(balance)`：对筛选结果中的 `balance` 列计算平均值。
  3. 返回一个单行单列的结果。
- **注意事项**：
  - 如果没有匹配的行（即 Perryridge 分行不存在账户），则 `AVG` 会返回 `NULL`。
  - 如果筛选后的 `balance` 列全部为 `NULL`，则 `AVG` 也返回 `NULL`。
- **类似用法**：可以将 `AVG` 替换为 `SUM`、`MIN`、`MAX` 等，完成其他统计。

## 第3页：COUNT 函数的两种用法

**内容回顾**  
- 第一个例子：统计 `customer` 表中的元组（行）数。
  ```sql
  SELECT COUNT(*) FROM customer;
  ```
  这返回表中的总行数，无论某些列是否为空。

- 第二个例子：统计银行中存款客户的数量（即 `depositor` 表中不同的客户名个数）。
  ```sql
  SELECT COUNT(DISTINCT customer_name) FROM depositor;
  ```
  这里使用了 `DISTINCT` 来去除重复的客户名，确保每个客户只计数一次。

- 涉及的表：
  - `Customer(customer-name, customer-street, customer-city)`
  - `Depositor(customer-name, account-number)`

**详细讲解**  
- **COUNT(*) 与 COUNT(column) 的区别**：
  - `COUNT(*)` 统计表中的行数，包括所有行，不管是否有 NULL 值。
  - `COUNT(column)` 统计该列中**非 NULL** 值的个数。
  - 例如，如果 `customer` 表有10行，但 `customer_name` 列中有1个 NULL，则 `COUNT(customer_name)` 返回9，而 `COUNT(*)` 返回10。

- **DISTINCT 的作用**：
  - 在 `COUNT(DISTINCT column)` 中，先去除该列中的重复值（并忽略 NULL），然后统计剩余值的个数。
  - 这对于统计唯一值数量非常有用，例如统计不同客户数、不同产品种类数等。

- **示例场景**：
  - `COUNT(*)` 常用于快速获取表的行数。
  - `COUNT(DISTINCT customer_name)` 用于获取银行中实际有存款的客户人数（一个客户可能有多个账户，但只计一次）。

- **其他聚合函数与 DISTINCT**：
  - 除了 `COUNT`，`SUM(DISTINCT column)`、`AVG(DISTINCT column)` 等也可以使用，但实际场景较少，因为对数值去重后再求和或平均通常意义不大。
  - `MIN` 和 `MAX` 使用 `DISTINCT` 无意义，因为最值不受重复影响。

## 补充知识：聚合函数与 GROUP BY 的关联

虽然这几页PPT没有直接涉及 `GROUP BY`，但聚合函数通常与 `GROUP BY` 子句配合使用，以实现对数据的分组统计。例如，要计算每个分行的平均余额，可以写：

```sql
SELECT branch_name, AVG(balance)
FROM account
GROUP BY branch_name;
```

这里 `GROUP BY` 将数据按 `branch_name` 分组，然后对每个组分别应用 `AVG` 函数。`WHERE` 子句在分组前过滤行，而 `HAVING` 子句在分组后过滤组。

## 总结

| 函数                   | 作用                       | 注意事项                        |
| ---------------------- | -------------------------- | ------------------------------- |
| **AVG**                | 求平均值                   | 忽略 NULL，仅用于数值列         |
| **MIN**                | 求最小值                   | 忽略 NULL，可用于多种类型       |
| **MAX**                | 求最大值                   | 忽略 NULL，可用于多种类型       |
| **SUM**                | 求和                       | 忽略 NULL，仅用于数值列         |
| **COUNT(*)**           | 统计行数                   | 不忽略 NULL（包括全 NULL 的行） |
| **COUNT(列)**          | 统计某列非空值的个数       | 忽略 NULL                       |
| **COUNT(DISTINCT 列)** | 统计某列不重复的非空值个数 | 忽略 NULL，先去重               |

掌握这些聚合函数是编写 SQL 统计查询的基础，后续结合 `GROUP BY` 可以实现更复杂的数据分析。
