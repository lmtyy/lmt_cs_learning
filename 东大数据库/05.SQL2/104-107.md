这几页PPT主要介绍了SQL中**子查询在FROM子句和SELECT子句中的用法**，并通过具体示例展示了它们的应用场景和语法要点。下面我将逐一详细讲解每一页的内容，并补充相关知识点。

---

## 第1页：子查询在FROM子句中

**核心概念**  
- SQL允许在`FROM`子句中使用子查询，此时子查询的结果充当一个临时表（称为**派生表**）。
- 派生表必须有一个**别名**，以便外层查询引用它。
- 别名可以同时指定表别名和列别名，如 `AS 表别名(列别名列表)`。

**示例场景**  
表：`Account(account-number, balance, branch-name)`  
需求：找出那些平均账户余额大于1200的分支，并显示分支名和平均余额。

---

## 第2页：具体实现

```sql
SELECT branch_name, avg_balance
FROM (
    SELECT branch_name, AVG(balance) 
    FROM account
    GROUP BY branch_name
) AS branch_avg (branch_name, avg_balance)
WHERE avg_balance > 1200;
```

**详细解析**  
1. **内层子查询**：先按`branch_name`分组，计算每个分支的平均余额，结果包含两列：`branch_name`和`AVG(balance)`。注意这个结果没有列名，因为`AVG(balance)`是一个表达式，所以我们需要在派生表中为它指定别名。
2. **派生表**：内层查询的结果作为一个临时表，被命名为`branch_avg`，并显式指定了列名`(branch_name, avg_balance)`。这样外层就可以直接使用这些列名。
3. **外层查询**：从派生表中选择满足`avg_balance > 1200`的行。

**优点**  
- 这种写法将分组聚合与过滤分离，逻辑清晰。
- 实际上，这个查询也可以用`HAVING`直接实现：  
  ```sql
  SELECT branch_name, AVG(balance)
  FROM account
  GROUP BY branch_name
  HAVING AVG(balance) > 1200;
  ```
  但使用派生表更灵活，例如当我们需要对聚合结果进行多次计算或与其他表连接时。

**注意**  
- 派生表必须有一个别名，即使有时可以省略`AS`关键字，但推荐显式使用。
- 列别名可以在派生表定义时指定，也可以在子查询内部用`AS`定义，两者等价。

## 第3页：子查询在SELECT子句中

**核心概念**  
- 子查询也可以出现在`SELECT`子句中，通常作为**标量子查询**（返回单个值），用于为每一行计算一个额外列。
- 标量子查询可以是相关的，即引用外层查询的列，此时对外层每一行都会执行一次。

**示例数据**  
- `Person`表：`(ID, Name, Age, countryID)`
- `Country`表：`(countryID, Name, Capital)`

需求：查询每个国家的名称及其对应的人员最大年龄（如果该国没有人员，则显示`NULL`）。

结果表格：
| Name    | MaxAge |
| ------- | ------ |
| Ireland | 20     |
| China   | 22     |
| Italy   | 25     |
| France  | null   |

---

## 第4页：两种实现方法对比

### 方法1：使用JOIN和GROUP BY
```sql
SELECT c.Name, MAX(p.Age) AS MaxAge
FROM Country c
LEFT JOIN Person p ON c.countryID = p.countryID
GROUP BY c.Name;
```
- 这里使用了`LEFT JOIN`，确保所有国家都保留（即使没有匹配的人员）。
- 然后按国家分组，用`MAX(p.Age)`计算最大年龄，没有人员的国家`MAX`结果为`NULL`。
- 注意：原PPT中写的是`JOIN`，但如果是内连接，法国将不会出现，所以正确应为`LEFT JOIN`。这可能是PPT的疏漏。

### 方法2：使用标量子查询
```sql
SELECT Name,
    (SELECT MAX(Age)
     FROM Person
     WHERE Person.countryID = Country.countryID
    ) AS MaxAge
FROM Country;
```
- 对于`Country`表中的每一行，子查询找出该国人员的最大年龄。
- 子查询是相关的，因为引用了外层的`Country.countryID`。
- 如果子查询没有找到任何行（如法国），则返回`NULL`，这正是我们想要的。

**对比**  
- **可读性**：子查询方法更直观，直接对应“每个国家，计算其人员最大年龄”。
- **性能**：相关子查询可能较慢（外层有多少国家就执行多少次），但若表不大，差异可忽略。`LEFT JOIN`+分组通常一次扫描即可，可能更高效。
- **适用性**：当需要从同一张表获取聚合值时，标量子查询很简洁；若需多列聚合，则`JOIN`+分组更合适。

**注意事项**  
- 标量子查询必须保证**只返回一行一列**，否则会引发运行时错误。如果可能返回多行，应使用聚合函数（如`MAX`）或限制条件（如`TOP 1`）确保单值。
- 如果子查询没有匹配行，结果就是`NULL`，这符合SQL的`NULL`语义。

---

## 总结

- **子查询在FROM子句**：创建派生表，必须起别名，常用于对中间结果进行再查询。
- **子查询在SELECT子句**：作为标量子查询，用于计算每行的额外列，常与相关条件结合。
- 两种用法都极大增强了SQL的表达能力，但需注意性能影响和语法要求（如别名、单值保证）。
- 实际开发中，应根据需求选择最清晰、高效的方式，有时`JOIN`和子查询可以互换，但要注意空值处理和连接类型。

理解这些高级用法，能帮助你编写更灵活、更强大的SQL查询。
