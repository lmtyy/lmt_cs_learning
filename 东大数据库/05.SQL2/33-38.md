## 1. ORDER BY 子句基础

**概念**  
- `ORDER BY` 子句用于对查询结果集进行排序，使其按照指定的列的顺序显示。
- 可以指定排序方向：
  - `ASC`：升序（从小到大，默认，可不写）
  - `DESC`：降序（从大到小）
- 可以对**多个列**进行排序，先按第一个列排序，如果值相同，再按第二个列排序，以此类推。

**语法**  
```sql
SELECT 列1, 列2, ...
FROM 表名
[WHERE 条件]
ORDER BY 列1 [ASC|DESC], 列2 [ASC|DESC], ...;
```

---

## 2. 单列排序示例

**示例需求**：查询选修了课程号为“3”的学生的学号和成绩，并按成绩降序排列。

```sql
SELECT Sno, Grade
FROM SC
WHERE Cno = '3'
ORDER BY Grade DESC;
```

**解释**：
- `WHERE Cno='3'` 先筛选出课程3的选课记录。
- `ORDER BY Grade DESC` 将结果按成绩从高到低排序。
- 结果集只包含两列：学号（Sno）和成绩（Grade）。

**注意**：PPT中给出的表格可能有些混乱，但核心是展示了排序后的结果（例如成绩88排在80前面）。实际使用中，排序方向清晰可见。

---

## 3. 多列排序示例

**示例需求**：查询全体学生信息，先按所在系（Sdept）升序排列，同一系内的学生按年龄（Sage）降序排列。

```sql
SELECT *
FROM Student
ORDER BY Sdept ASC, Sage DESC;
```

**解释**：
- 首先按照 `Sdept` 升序排序，即系名字母顺序（或字符集顺序）。
- 当 `Sdept` 相同时，再按 `Sage` 降序，即年龄大的在前。
- 这样可以得到例如“CS系中年龄最大的学生排在该系最前面”的效果。

**排序规则**：
- 对于数值，按数字大小。
- 对于字符串，按字符集定义的顺序（通常与字典序一致）。
- 对于日期，按时间先后。
- `NULL` 值的处理：不同数据库有差异，通常 `ASC` 时 `NULL` 排在最前或最后，但标准未规定。建议测试确认。

## 4. LIMIT 子句介绍

**作用**  
- `LIMIT` 子句用于限制查询返回的行数，常用于分页或只获取前N条记录。
- 它通常与 `ORDER BY` 结合使用，先排序再取前几条。

**语法（标准形式）**  
```sql
SELECT 列
FROM 表
[WHERE ...]
[ORDER BY ...]
LIMIT row_count [OFFSET offset];
```
- `row_count`：要返回的最大行数。
- `OFFSET offset`：可选，表示在返回结果前跳过前 `offset` 行。`OFFSET 0` 表示不跳过。
- 如果只写 `LIMIT 5`，相当于 `LIMIT 5 OFFSET 0`，返回前5行。

**兼容性**  
- `LIMIT` 子句在 **MySQL、PostgreSQL、SQLite** 等数据库中广泛支持。
- 其他数据库有不同实现：
  - SQL Server 使用 `SELECT TOP 5 ...` 或 `OFFSET-FETCH`。
  - Oracle 使用 `ROWNUM` 或 `OFFSET-FETCH`（12c+）。
  - DB2 也支持 `FETCH FIRST 5 ROWS ONLY`。
- 因此，当编写跨数据库应用时需注意兼容性。

---

## 5. LIMIT 5 示例

**示例需求**：从 employees 表中按 first_name 排序，返回前5行。

```sql
SELECT employee_id, first_name, last_name
FROM employees
ORDER BY first_name
LIMIT 5;
```

**执行逻辑**：
1. 先按 `first_name` 对整个表排序。
2. 然后从排序后的结果中取出前5行。

**结果说明**：PPT中左侧显示全表（40行），右侧显示取出的前5行，但截图中的具体数据可能有错位，重点在于理解 `LIMIT 5` 只返回5行。

---

## 6. LIMIT 5 OFFSET 3 示例（分页）

**示例需求**：按 first_name 排序，跳过前3行，然后返回接下来的5行。

```sql
SELECT employee_id, first_name, last_name
FROM employees
ORDER BY first_name
LIMIT 5 OFFSET 3;
```

**执行逻辑**：
- 排序后，跳过第1、2、3行（即OFFSET 3，从第4行开始取）。
- 取5行，即第4~8行。

**MySQL 简写形式**：  
在 MySQL 中，`LIMIT 5 OFFSET 3` 可以简写为 `LIMIT 3, 5`，注意顺序：**先偏移量，后行数**。但标准形式更清晰，推荐使用标准写法。

**分页常用公式**：  
如果每页显示 `page_size` 行，第 `page_num` 页（从1开始）的数据为：
```sql
LIMIT page_size OFFSET (page_num - 1) * page_size;
```

## 7. 关键注意事项

1. **ORDER BY 与 NULL**  
   排序时 NULL 的位置因数据库而异：
   - MySQL、PostgreSQL：`ASC` 时 NULL 排在最前，`DESC` 时 NULL 排在最后。
   - SQL Server：`ASC` 时 NULL 排在最前（可配置）。
   - Oracle：`ASC` 时 NULL 排在最后，`DESC` 时 NULL 排在最前。
   若需统一行为，可使用 `COALESCE` 或 `CASE` 将 NULL 替换为特定值。

2. **LIMIT 与性能**  
   - 通常先执行 `ORDER BY` 再 `LIMIT`，如果排序的列有索引，可大幅提升性能。
   - 大偏移量的分页（如 `LIMIT 10 OFFSET 10000`）可能效率低下，因为数据库需要扫描并丢弃前10000行。优化方法包括使用键集分页（seek method）或基于索引的子查询。

3. **列别名与 ORDER BY**  
   在 `ORDER BY` 中可以使用列别名（因为 `ORDER BY` 在 `SELECT` 之后执行），例如：
   ```sql
   SELECT salary * 12 AS annual FROM emp ORDER BY annual;
   ```
   这是标准允许的，但某些数据库可能要求严格。

4. **多个排序列的方向**  
   每个列可以独立指定 `ASC` 或 `DESC`，不要求统一。

---

## 总结

- **ORDER BY** 使查询结果有序，支持单列和多列排序，方向可指定。
- **LIMIT** 限制返回行数，常与 `ORDER BY` 联用实现分页。
- 不同数据库对 `LIMIT` 的支持形式不同，注意兼容性。
- 理解排序时 `NULL` 的处理方式，避免逻辑错误。

掌握这些内容后，你就能灵活地对查询结果进行排序和分页，满足常见的报表和前端展示需求。
