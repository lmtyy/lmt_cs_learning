## 第1页：GROUP BY 的核心原则

**内容回顾**  
- 提出了一个问题：为什么在 `SELECT` 子句中，**没有使用聚合函数的列必须出现在 `GROUP BY` 列表中**？
- 给出了原则：`SELECT` 后面的所有列中，没有使用聚合函数的列，必须出现在 `GROUP BY` 后面。

**详细讲解**  
- **原因**：当使用 `GROUP BY` 对数据进行分组后，每个分组会包含多行数据。对于出现在 `SELECT` 中的列：
  - 如果是**分组列**（即 `GROUP BY` 中指定的列），则该列在组内所有行的值都是相同的，因此可以确定性地输出这个值。
  - 如果是**聚合函数**（如 `COUNT`、`SUM`、`AVG` 等），它对组内多行进行计算，返回一个单一值，也是确定的。
  - 但如果一个列**既不是分组列，也没有被聚合**，那么它在组内可能有多个不同的值（例如账户余额在同一个支行组内可能不同），数据库无法确定应该输出哪一个值，这会导致结果的不确定性。因此，标准SQL禁止这种查询。

- **举例**：假设我们想按支行分组，同时输出每个支行的名称和某个账户的余额。但一个支行有多个账户，余额各不相同，无法确定输出哪一个。正确的做法要么将余额也加入分组（此时分组粒度变细，每个不同的余额值单独成组），要么对余额进行聚合（如求平均值、总和等）。

## 第2页：正确与错误示例

**内容回顾**  
- 给出了一个错误示例：
  ```sql
  SELECT branch_name, balance, COUNT(DISTINCT account_number)
  FROM account
  GROUP BY branch_name;
  ```
  这个查询试图按 `branch_name` 分组，但 `balance` 既不在 `GROUP BY` 中，也没有被聚合，因此是错误的。

- 给出了两种正确的写法：
  1. 将 `balance` 也加入 `GROUP BY`：
     ```sql
     SELECT branch_name, balance
     FROM account
     GROUP BY branch_name, balance;
     ```
     这种写法会按照 `(branch_name, balance)` 的组合进行分组，输出每个组合及其出现次数（虽然这里没写聚合函数，但 `GROUP BY` 本身就会去重，相当于输出所有唯一的组合）。
  2. 使用聚合函数处理 `balance`：
     ```sql
     SELECT branch_name, SUM(balance), COUNT(...)
     FROM account
     GROUP BY branch_name;
     ```
     这里对 `balance` 求和，使其成为聚合结果。

- 同时还展示了一个示例表格：

| branch_name | account_number | balance |
| ----------- | -------------- | ------- |
| Perryridge  | a-102          | 400     |
| Perryridge  | a-201          | 900     |
| Brighton    | a-217          | 750     |
| Brighton    | a-215          | 750     |
| Redwood     | a-222          | 700     |

**详细讲解**  
- **错误示例分析**：假设执行错误查询，数据库会报错，因为对于 Perryridge 组，有 400 和 900 两个不同的 balance 值，无法确定输出哪一个。
- **正确写法1（多列分组）**：`GROUP BY branch_name, balance` 会将数据分成更细的组：
  - (Perryridge, 400) 一组，包含 a-102 一行。
  - (Perryridge, 900) 一组，包含 a-201 一行。
  - (Brighton, 750) 一组，包含 a-217 和 a-215 两行（因为 balance 相同）。
  - (Redwood, 700) 一组，包含 a-222 一行。
  此时 `SELECT branch_name, balance` 会输出所有唯一的组合，相当于对原始表做了去重（但注意，如果某个组合有多行，输出中只出现一次，因为 `GROUP BY` 默认去重）。如果想输出每个组合的计数，可以加上 `COUNT(*)`。
- **正确写法2（聚合 balance）**：使用 `SUM(balance)` 可以计算每个支行的总余额，这是常见的分组汇总需求。

- **注意**：在错误示例中，`COUNT(DISTINCT account_number)` 是聚合函数，它本身没问题，问题在于 `balance` 未处理。因此修正时必须处理 `balance`。

## 第3页：GROUP BY 与 JOIN 结合

**内容回顾**  
- 表结构：
  - `depositor` (customer-name, account-number)
  - `account` (branch-name, account-number, balance)
- 需求：查询每个支行的存款人数（即每个支行有多少不同的客户）。
- SQL语句：
  ```sql
  SELECT branch_name, COUNT(DISTINCT customer_name)
  FROM depositor, account
  WHERE depositor.account_number = account.account_number
  GROUP BY branch_name;
  ```

**详细讲解**  
- **连接**：首先通过 `WHERE` 条件将 `depositor` 和 `account` 按 `account_number` 进行等值连接，得到每个账户对应的客户和所属支行。
- **分组**：按 `branch_name` 分组，同一个支行的所有账户记录被分到一组。
- **聚合**：对每个组，使用 `COUNT(DISTINCT customer_name)` 统计该支行下不同客户的数量，因为一个客户可能在同一个支行有多个账户，需要去重。
- **为什么需要 DISTINCT**：如果某个客户在同一个支行有多个账户，连接后该客户会出现多次，如果不加 `DISTINCT`，就会重复计数。因此必须使用 `COUNT(DISTINCT customer_name)` 来得到正确的存款人数。
- **注意**：这里使用了隐式的逗号连接语法，等价于 `INNER JOIN`。在实际开发中，更推荐使用显式的 `JOIN` 语法，以提高可读性。

## 第4页：分组评估过程

**内容回顾**  
- 展示了连接后的中间结果表（可能有多行，包含 `branch_name` 和 `customer_name`）。
- 然后通过 `GROUP BY` 分组，并应用 `COUNT(DISTINCT customer_name)` 得到每个支行的计数。
- PPT中给出了两个表格，但内容有些混乱，需要理清。

**详细讲解**  
- **连接结果**：假设执行连接查询：
  ```sql
  SELECT branch_name, customer_name
  FROM depositor, account
  WHERE depositor.account_number = account.account_number;
  ```
  可能得到如下结果（示例数据）：

| branch_name | customer_name |
| ----------- | ------------- |
| Perryridge  | John Wong     |
| Perryridge  | Pat Lee       |
| Uptown      | John Wong     |
| Uptown      | Jacky Chan    |
| Downtown    | John Wong     |
| Downtown    | Mary Kwan     |
| Downtown    | Pat Lee       |
| Downtown    | May Cheung    |

  注意：这里假设每个客户在每个支行可能只有一个账户（否则会有重复），但PPT中似乎有重复行（如 Downtown 出现两次 John Wong？），这可能是为了说明 DISTINCT 的必要性。

- **分组聚合**：按 `branch_name` 分组后，对每个组应用 `COUNT(DISTINCT customer_name)`：
  - Perryridge 组：有 John Wong 和 Pat Lee → 计数 2。
  - Uptown 组：有 John Wong 和 Jacky Chan → 计数 2。
  - Downtown 组：有 John Wong、Mary Kwan、Pat Lee、May Cheung（假设没有重复）→ 计数 4？但PPT中显示 Downtown 计数 3，可能数据不同。关键是理解过程。

- **DISTINCT 的作用**：如果某个客户在同一个支行有多个账户，连接后会出现多行相同 `(branch_name, customer_name)` 组合。使用 `COUNT(DISTINCT customer_name)` 会去除同一组内重复的客户名，从而得到正确的客户数。

- **分组评估步骤**：
  1. 先执行 `FROM` 和 `WHERE`，得到中间结果（包含所有符合条件的行）。
  2. 然后根据 `GROUP BY` 列将中间结果分成若干组。
  3. 对每组分别计算聚合函数。
  4. 最后输出每组的分组列和聚合结果。

## 总结与扩展

- **核心规则**：`SELECT` 中出现的非聚合列必须出现在 `GROUP BY` 中，以保证结果的确定性。
- **多列分组**：可以按多个列分组，此时分组粒度更细。
- **与 JOIN 结合**：`GROUP BY` 常与多表连接一起使用，先连接后分组。
- **DISTINCT 在聚合中的使用**：如 `COUNT(DISTINCT col)` 用于统计唯一值，避免重复计数。
- **执行顺序**：理解 `WHERE` → `GROUP BY` → `SELECT` → `ORDER BY` 的逻辑顺序有助于正确编写查询。

掌握这些知识点后，你就能灵活运用 `GROUP BY` 进行各种分组统计，同时避免常见的语法错误。如果有任何疑问，欢迎继续探讨！
