### 1. Basic Query Structure（基本查询结构）

**概念**  
这是SQL查询的起点，描述了如何从一个或多个表中检索数据。最核心的结构是 `SELECT-FROM-WHERE` 块，有时也包含 `DISTINCT`、`ORDER BY` 等子句。

**详细解析**  
- **SELECT 子句**：指定要查询的列（属性），可以是具体的列名、表达式、常量或函数（如 `UPPER(name)`）。  
- **FROM 子句**：指定数据来源的表，可以是单个表或多个表的连接（JOIN）。  
- **WHERE 子句**（可选）：对行进行过滤，只有满足条件的行才会出现在结果中。  
- **其他常用子句**：  
  - `DISTINCT`：去除结果中重复的行。  
  - `ORDER BY`：对结果进行排序（ASC升序/DESC降序）。  
  - `LIMIT` / `FETCH`：限制返回的行数（不同数据库语法略有差异）。

**示例**  
```sql
SELECT DISTINCT dept_name, salary * 12 AS annual_salary
FROM employee
WHERE hire_date >= '2020-01-01'
ORDER BY annual_salary DESC
LIMIT 10;
```
该查询从 `employee` 表中选取2020年后入职的员工，计算其年薪，按年薪降序返回前10条记录，并去除完全重复的行。

**意义**  
这是所有复杂查询的基础，后续的集合操作、聚合、子查询等都是在基本查询结构之上进行的扩展。

### 2. Set Operations（集合操作）

**概念**  
集合操作用于组合两个或多个查询的结果集，类似于数学中的并、交、差运算。SQL提供了三个主要的集合操作符：`UNION`、`INTERSECT` 和 `EXCEPT`（某些数据库用 `MINUS`）。

**详细解析**  
- **UNION**：合并两个查询的结果，并自动去除重复行。若需保留所有重复，使用 `UNION ALL`（性能更好）。  
- **INTERSECT**：返回同时出现在两个查询结果中的行。  
- **EXCEPT**：返回出现在第一个查询结果但不在第二个查询结果中的行。  
- **使用要求**：参与运算的各个查询必须具有相同数量的列，且对应列的数据类型兼容。

**示例**  
```sql
-- 查询既是教师又是学生的所有人的ID（交集）
SELECT id FROM teacher
INTERSECT
SELECT id FROM student;

-- 查询所有在A部门或B部门工作的员工（并集）
SELECT name FROM employee WHERE dept = 'A'
UNION
SELECT name FROM employee WHERE dept = 'B';

-- 查询选修了课程1但没有选修课程2的学生（差集）
SELECT student_id FROM enroll WHERE course_id = 1
EXCEPT
SELECT student_id FROM enroll WHERE course_id = 2;
```

**与基本查询的关系**  
集合操作通常作用于两个独立的 `SELECT` 语句之上，因此它建立在基本查询结构的基础上，用于解决需要跨多个查询进行逻辑组合的问题。

### 3. Aggregate Functions（聚合函数）

**概念**  
聚合函数对一组值进行计算并返回一个单一的值。常与 `GROUP BY` 子句配合使用，实现数据的分组统计。常见的聚合函数有：`COUNT`、`SUM`、`AVG`、`MAX`、`MIN`。

**详细解析**  
- **COUNT**：统计行数，`COUNT(*)` 统计所有行，`COUNT(column)` 统计该列非空值的个数。  
- **SUM** / **AVG**：对数值列求和或求平均值，会自动忽略 `NULL` 值。  
- **MAX** / **MIN**：求最大值或最小值，可用于任何可排序的数据类型。  
- **GROUP BY**：将数据按指定列分组，然后对每个分组应用聚合函数。  
- **HAVING**：对分组后的结果进行过滤，与 `WHERE` 类似，但 `WHERE` 是在分组前过滤行，而 `HAVING` 是在分组后过滤分组。

**示例**  
```sql
-- 统计每个部门的员工人数和平均工资
SELECT dept_id, COUNT(*) AS emp_count, AVG(salary) AS avg_salary
FROM employee
WHERE status = 'active'
GROUP BY dept_id
HAVING AVG(salary) > 50000;
```

**与之前概念的结合**  
- 可以在基本查询中使用聚合函数，但注意：如果 `SELECT` 中同时包含普通列和聚合函数，通常需要将普通列放入 `GROUP BY` 中。  
- 集合操作也可以对聚合后的结果进行操作，例如合并两个部门统计结果。

### 4. Null Values（空值处理）

**概念**  
`NULL` 在数据库中代表“未知”或“不存在”的值。它的存在给查询带来了特殊的逻辑，必须谨慎处理。

**详细解析**  
- **三值逻辑**：与 `NULL` 的比较结果不是 `TRUE` 或 `FALSE`，而是 `UNKNOWN`。只有 `WHERE` 条件为 `TRUE` 的行才会被选中。  
- **IS NULL / IS NOT NULL**：判断值是否为 `NULL` 必须使用这两个操作符，而不能用 `= NULL`。  
- **聚合函数对 NULL 的处理**：除了 `COUNT(*)`，其他聚合函数都会忽略 `NULL` 值（例如 `SUM` 忽略 `NULL` 行，`AVG` 只计算非空值）。  
- **算术运算与 NULL**：任何包含 `NULL` 的算术表达式结果都是 `NULL`。  
- **在集合操作中**：`NULL` 被视为相互相等（例如两个 `NULL` 在 `UNION` 去重时被视为相同），但在比较时仍遵循三值逻辑。

**示例**  
```sql
-- 查找没有填写邮箱的员工
SELECT name FROM employee WHERE email IS NULL;

-- 计算员工的平均奖金（忽略NULL值）
SELECT AVG(bonus) FROM employee;  -- 只对有奖金的员工求平均

-- 注意：COUNT(*) 与 COUNT(column) 的区别
SELECT COUNT(*) AS total, COUNT(bonus) AS has_bonus FROM employee;
```

**与聚合函数的交互**  
理解 `NULL` 对于正确使用聚合函数至关重要，因为 `NULL` 的忽略可能导致统计结果与预期不符。

### 5. Nested Subqueries（嵌套子查询）

**概念**  
子查询是指嵌套在另一个查询（外层查询）中的 `SELECT` 语句。子查询可以出现在 `SELECT`、`FROM`、`WHERE`、`HAVING` 等子句中，用于提供动态的中间结果。

**详细解析**  
- **标量子查询**：返回单个值（一行一列），常用于 `SELECT` 列表或 `WHERE` 条件中。  
- **行子查询**：返回一行多列，可用于比较。  
- **表子查询**：返回多行多列，常用于 `FROM` 子句（作为派生表）或 `IN`/`EXISTS` 条件中。  
- **相关子查询**：子查询中引用了外层查询的列，每处理一行外层数据，子查询就执行一次。  
- **存在性测试**：`EXISTS` 检查子查询是否返回至少一行，通常用于相关子查询。  
- **集合成员测试**：`IN` / `NOT IN` 检查某值是否在子查询的结果集中（注意 `NULL` 的陷阱）。

**示例**  
```sql
-- 标量子查询：查询每个员工的姓名及其所在部门的平均工资
SELECT name, 
       (SELECT AVG(salary) FROM employee e2 WHERE e2.dept_id = e1.dept_id) AS dept_avg
FROM employee e1;

-- 相关子查询：查询工资高于其所在部门平均工资的员工
SELECT name, salary
FROM employee e1
WHERE salary > (SELECT AVG(salary) FROM employee e2 WHERE e2.dept_id = e1.dept_id);

-- 使用 EXISTS：查询至少有一名员工的部门
SELECT dept_name
FROM department d
WHERE EXISTS (SELECT 1 FROM employee e WHERE e.dept_id = d.dept_id);
```

**与之前概念的结合**  
子查询可以出现在 `WHERE` 条件中与集合操作结合，也可以在 `FROM` 中作为派生表，然后在派生表上再进行聚合或连接，极大地提升了查询的灵活性。

### 6. Complex Queries（复杂查询）

**概念**  
这一部分是对前面所有知识点的综合运用，解决现实业务中更为复杂的查询需求。通常涉及多表连接、子查询嵌套、集合运算、聚合函数以及条件逻辑的混合使用。

**常见复杂查询场景**  
- **多表连接 + 聚合**：例如连接订单表和客户表，统计每个客户的订单总额。  
- **相关子查询 + 集合操作**：例如找出既选修了所有必修课的学生（使用 `NOT EXISTS` 实现除法）。  
- **自连接**：将一张表与自己连接，用于比较同一表中不同行的数据（如查询比同事工资高的员工）。  
- **窗口函数**（虽未在目录中明确列出，但常作为复杂查询的一部分）：如 `ROW_NUMBER()`、`RANK()` 等，用于排名、移动平均等高级分析。  
- **条件逻辑（CASE WHEN）**：在查询中实现 if-then-else 逻辑，可用于创建新列或复杂的聚合条件。

**示例**  
```sql
-- 复杂综合查询：找出每个部门中工资最高的员工，同时显示该员工的姓名、工资以及部门名称
SELECT d.dept_name, e.name, e.salary
FROM employee e
JOIN department d ON e.dept_id = d.dept_id
WHERE (e.dept_id, e.salary) IN (
    SELECT dept_id, MAX(salary)
    FROM employee
    GROUP BY dept_id
);
```
该查询使用了子查询找出每个部门的最高工资，然后用多列 `IN` 匹配出对应的员工。

**与前面各节的关系**  
复杂查询是前五个主题的融会贯通，要求能够根据需求选择合适的语法组合，优化查询性能，并正确处理边界情况（如 `NULL`、重复值等）。
