这两页PPT主要讲解了SQL查询中如何处理重复行的问题，重点介绍了 `DISTINCT` 和 `ALL` 关键字的使用。下面我将详细解释这些知识点，并补充一些相关的概念和注意事项。

---

### 1. SQL 中的重复行

在关系数据库中，**关系（表）** 在理论上是一个**集合**，集合中的元素（行）是唯一的，不允许重复。然而，在实际的SQL实现中，**表可能包含重复行**（如果没有定义主键或唯一约束），而且**查询结果默认也会保留重复行**。这是因为SQL更贴近实际数据处理需求，有时我们需要保留所有记录（包括重复），有时则需要去重。

例如，在 `Loan` 表中，同一个分行可能有多笔贷款，因此查询所有分行名称时，结果中会出现重复的分行名。

---

### 2. 使用 `DISTINCT` 去除重复

**`SELECT DISTINCT`** 的作用是强制去除查询结果中的重复行，只返回唯一的组合。

**语法示例**：
```sql
SELECT DISTINCT branch_name
FROM loan;
```
这条查询会返回 `loan` 表中所有**不重复**的分行名称。如果同一个分行有多笔贷款，结果中该分行只出现一次。

**等价于关系代数**：在关系代数中，投影操作（\(\pi\)）默认会去除重复行，因此 `SELECT DISTINCT` 更符合关系代数的语义。

---

### 3. 使用 `ALL` 保留重复

**`SELECT ALL`** 的作用是明确表示保留重复行。实际上，**`ALL` 是默认行为**，即如果你不写任何关键字，SQL默认就是 `SELECT ALL`。因此，以下两个查询是等价的：

```sql
SELECT branch_name FROM loan;                 -- 默认保留重复
SELECT ALL branch_name FROM loan;              -- 显式保留重复
```

`ALL` 关键字通常可以省略，但在某些上下文中（如与 `UNION` 等集合操作配合时）它也有用。

---

### 4. 注意事项：`DISTINCT` 作用于所有目标列

PPT第二页特别强调了两点：

#### a. `DISTINCT` 作用于**所有**选择的列

`DISTINCT` 是基于 `SELECT` 子句中列出的**所有列的组合**来判断重复的。也就是说，只有当这些列的值组合完全相同时，才被视为重复行并去除。

**示例**：
```sql
SELECT DISTINCT branch_name, amount
FROM loan;
```
这条查询返回的是 `(branch_name, amount)` 的所有唯一组合。假设 `loan` 表中有以下数据：

| branch_name | amount |
| ----------- | ------ |
| Downtown    | 1000   |
| Downtown    | 2000   |
| Downtown    | 1000   |

执行上述查询后，结果会保留 `(Downtown, 1000)` 一次和 `(Downtown, 2000)` 一次，即去除了重复的 `(Downtown, 1000)` 行。

#### b. 不能对单个列单独使用 `DISTINCT`

以下写法是错误的：
```sql
SELECT DISTINCT branch_name, DISTINCT amount FROM loan;   -- 错误
```
因为 `DISTINCT` 是作用于整个 `SELECT` 子句的修饰符，不能为每个列单独指定。如果你需要对不同列做不同的去重处理，通常需要使用其他方法（如子查询或分组）。

---

### 5. 与关系代数的对比

- **关系代数中的投影**：\(\pi_{A_1,\dots,A_n}(r)\) 的结果是一个集合，自动去除重复行。
- **SQL中的投影**：默认的 `SELECT` 不带 `DISTINCT` 相当于**多重集**（multiset），保留重复行。`SELECT DISTINCT` 才将结果转为集合。

这种设计使得SQL既能处理集合，也能处理多重集，更加灵活。

---

### 6. 性能考虑

- 使用 `DISTINCT` 通常需要对结果进行排序或哈希操作以识别重复行，这可能会增加查询的开销。因此，**只在必要时才使用 `DISTINCT`**。
- 如果你确定查询结果本身不会包含重复（例如查询主键列），就不需要加 `DISTINCT`，以避免不必要的性能损耗。

---

### 7. 扩展：`DISTINCT` 与 `GROUP BY`

`DISTINCT` 和 `GROUP BY` 都可以用于去重，但用途不同：
- `DISTINCT` 纯粹是去除重复行。
- `GROUP BY` 通常与聚合函数配合使用，用于分组统计；但如果只 `GROUP BY` 而不加聚合函数，效果类似于 `DISTINCT`（但可能涉及排序，且语义上是为了分组）。

例如：
```sql
SELECT branch_name, amount FROM loan GROUP BY branch_name, amount;
```
等价于：
```sql
SELECT DISTINCT branch_name, amount FROM loan;
```
但 `GROUP BY` 的意图是分组，而 `DISTINCT` 意图是去重，建议按语义选择合适的关键字。

---

### 总结

- **默认情况下**，SQL查询保留重复行（多重集语义）。
- 使用 **`SELECT DISTINCT`** 去除结果中的重复行，它作用于所有选定列的组合。
- 使用 **`SELECT ALL`** 显式保留重复行（可省略）。
- 注意 `DISTINCT` 不能分别指定于单个列，只能修饰整个 `SELECT` 列表。

掌握这些基础概念后，你就能在处理重复数据时做出正确的选择，写出高效且语义清晰的SQL查询。
