## 第1页：获取没有成绩的学生姓名（连接写法）

**原始SQL**：
```sql
SELECT sname
FROM students, sc
WHERE students.sid = sc.sid AND grade IS NULL;
```

### 解析
- **意图**：找出那些有选课记录但成绩为空的学生。
- **逻辑**：通过内连接将`students`和`sc`关联，然后筛选出`grade IS NULL`的行。
- **问题**：这种写法**只能找出有选课记录且成绩为空的学生**。如果某个学生根本没有选课记录（即不在`sc`表中），那么他不会出现在连接结果中，因此不会被选出。这取决于对“没有成绩”的定义：
  - 如果“没有成绩”仅指已选课但未考试（成绩为空），则此写法正确。
  - 如果“没有成绩”也包括从未选课的学生，则需要使用**左外连接**或**子查询**来包含所有学生。

### 正确写法（包含未选课学生）
```sql
-- 使用左连接
SELECT sname
FROM students LEFT JOIN sc ON students.sid = sc.sid
WHERE sc.grade IS NULL;  -- 包括未选课（sc.sid为NULL）和选课但成绩为空的
```

## 第2页：获取没有成绩的学生姓名（子查询写法）

**原始SQL**：
```sql
SELECT sname
FROM students
WHERE sid IN (SELECT sid FROM sc WHERE grade IS NULL);
```

### 解析
- **逻辑**：子查询找出所有有选课记录且成绩为空的学生ID，外层查询从中选出学生姓名。
- **同样的问题**：与第一页一样，只考虑了有选课记录但成绩为空的学生，忽略了未选课的学生。
- **结论**：这两种写法在语义上等价，都只覆盖了部分情况。

### 扩展思考
如果需求是“找出所有没有成绩的学生”，需要明确是否包括未选课的学生。通常“没有成绩”可能指没有考试成绩，那么未选课的学生自然也没有成绩，应该包括在内。因此，更严谨的写法是：

```sql
SELECT sname
FROM students
WHERE NOT EXISTS (SELECT 1 FROM sc WHERE sc.sid = students.sid AND sc.grade IS NOT NULL);
-- 或者使用左连接判断
```

## 第3页：列出没学过李明老师讲授所有课程的学生姓名

**表结构**：
- `student(s#, sname, sage, ssex, scclass)`
- `course(c#, cname, chours, credit, T#)`
- `teacher(T#, Tname, D#)`
- `sc(s#, c#, score)`

**需求**：找出那些**没有学过李明老师所教的所有课程**的学生。即存在至少一门李明教的课，该学生没学过。

### 给出的两种错误写法

#### 写法一
```sql
SELECT sname
FROM student
WHERE s# NOT IN (
    SELECT s#
    FROM sc, course, teacher
    WHERE Tname='李明' AND course.T# = teacher.T# AND sc.c# = course.c#
);
```
- **子查询含义**：找出所有**学过李明老师任意一门课**的学生ID（注意：是任意一门，不是所有）。
- **外层NOT IN**：选出那些**从未学过李明老师任何一门课**的学生。
- **错误**：需求是“没学过所有课程”，即只要有一门没学就符合，而不是一门都没学。所以这个查询选出的范围太窄了。例如，如果李明教了3门课，一个学生学了其中2门，那么他应该被选出（因为他没学第3门），但此查询会把他排除，因为他出现在子查询中。

#### 写法二
```sql
SELECT sname
FROM student, sc, course, teacher
WHERE Tname <> '李明' 
  AND course.T# = teacher.T# 
  AND sc.c# = course.c# 
  AND sc.s# = student.s#;
```
- **逻辑**：找出所有学过非李明老师课程的学生。这与需求完全无关，因为学生可能同时学李明和非李明的课。
- **错误**：完全不相关。

### 正确写法分析
要表达“没学过所有李明老师的课”，等价于“存在一门李明老师的课，该学生没学过”。这可以用`NOT EXISTS`或分组计数来实现。

#### 方法一：使用 NOT EXISTS
```sql
SELECT sname
FROM student s
WHERE NOT EXISTS (
    SELECT 1
    FROM course c
    JOIN teacher t ON c.T# = t.T#
    WHERE t.Tname = '李明'
      AND NOT EXISTS (
          SELECT 1 FROM sc
          WHERE sc.s# = s.s# AND sc.c# = c.c#
      )
);
```
- 内层`NOT EXISTS`：对于李明老师的每一门课，检查该学生是否没有选课记录。如果存在至少一门课没有选，则条件成立。

#### 方法二：使用分组计数
先找出李明老师所教课程的总数，再对每个学生统计他学过李明老师课程的数量，如果这个数量小于总数，则符合条件。
```sql
SELECT s.sname
FROM student s
LEFT JOIN sc ON s.s# = sc.s#
LEFT JOIN course c ON sc.c# = c.c#
LEFT JOIN teacher t ON c.T# = t.T# AND t.Tname = '李明'
GROUP BY s.s#, s.sname
HAVING COUNT(DISTINCT c.c#) < (SELECT COUNT(*) FROM course c2 JOIN teacher t2 ON c2.T# = t2.T# WHERE t2.Tname = '李明');
```
注意：需要考虑学生可能重复选同一门课，所以要用`DISTINCT`。

#### 方法三：使用 NOT IN 但需要处理多门课
可以先用子查询找出所有李明老师的课程ID集合，然后对于每个学生，检查他是否遗漏了其中任何一门。这通常需要借助`NOT EXISTS`。

## 第4页：IN、SOME、ALL 的等价关系

**内容回顾**：
- `(= some) ≡ in`  
- `(≠ some) ≠ not in`  
- `(≠ all) ≡ not in`  
- `(= all) ≠ in` 

### 详细解释

#### 1. `= SOME` 等价于 `IN`
- `x = SOME (子查询)` 表示 x 等于子查询结果中的**至少一个**值，这正是`IN`的定义。
- 例如：`x IN (1,2,3)` 等价于 `x = SOME (1,2,3)`。

#### 2. `≠ SOME` 与 `NOT IN` 不等价
- `x ≠ SOME (集合)` 表示存在集合中的一个值使得 x 不等于它。由于集合中通常有多个不同值，只要集合不是所有值都等于 x，这个条件就为真。实际上，除了 x 等于集合中每一个值这种极端情况外，`≠ SOME` 总是真。所以它几乎没有实用意义。
- `NOT IN` 表示 x 不等于集合中的**每一个**值，即 `x ≠ ALL (集合)`。两者完全不同。
- 例如：集合 {1,2,3}，x=2，则 `2 ≠ SOME` 为真（因为存在1≠2），但 `2 NOT IN` 为假。

#### 3. `≠ ALL` 等价于 `NOT IN`
- `x ≠ ALL (集合)` 要求 x 不等于集合中的每一个值，这正是`NOT IN`的含义。
- 注意：两者在遇到`NULL`时行为相同，都会导致结果为`UNKNOWN`（空结果），除非集合不包含`NULL`。

#### 4. `= ALL` 与 `IN` 的关系
- `x = ALL (集合)` 要求 x 等于集合中的**每一个**值，这意味着集合中所有值必须相等且等于 x。这通常只在集合只有一个元素时成立。
- 因此，`= ALL` 绝不等价于 `IN`。

### 补充：关于 NULL 的影响
- 如果子查询结果中包含`NULL`，`IN`和`NOT IN`的行为需要特别注意：
  - `IN`：如果匹配到非NULL值，则返回真；如果所有值都是NULL且没有匹配，则结果为`UNKNOWN`（通常被视为假）。
  - `NOT IN`：如果集合中包含NULL，则整个`NOT IN`的结果为`UNKNOWN`（因为`x ≠ NULL`是未知的），导致不返回任何行。这是一个经典陷阱。
