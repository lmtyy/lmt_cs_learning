在SQL中，别名使用**双引号**（`"`）而不是单引号（`'`），是因为**单引号和双引号在SQL标准中有完全不同的用途**。下面详细解释两者的区别，以及为什么在别名中必须使用双引号（或特定数据库的方括号`[]`）。

---

### 1. 单引号（`'`）的用途：**字符串字面量**
- 在SQL中，**单引号专门用于表示字符串常量**（文本值）。
- 例如：
  ```sql
  SELECT 'Hello World' AS greeting;
  ```
  这里的 `'Hello World'` 是一个字符串，直接作为结果输出。
- 如果你在别名上使用单引号，比如：
  ```sql
  SELECT Sname 'Student Name'   -- 错误用法
  FROM Student;
  ```
  数据库会尝试将 `'Student Name'` 解析为**字符串常量**，而不是别名，从而导致语法错误（或意外的行为，比如在MySQL中这竟然可以工作，但它将别名视为字符串，违反了标准）。

---

### 2. 双引号（`"`）的用途：**标识符（如列名、表名、别名）**
- 双引号用于**引用数据库对象名称**（标识符），例如列名、表名、别名，特别是当这些名称：
  - **包含空格**（例如 `"Year of Birth"`）。
  - **是SQL保留字**（例如 `"select"`、`"table"`）。
  - **区分大小写**（某些数据库如PostgreSQL、Oracle对未加引号的标识符自动转为大写，加双引号可保留大小写）。
- 标准SQL中，双引号是引用标识符的唯一方式。例如：
  ```sql
  SELECT Sname AS "Student Name"
  FROM Student;
  ```
  这里的 `"Student Name"` 是一个别名，它包含空格，所以必须用双引号括起来。

---

### 3. 为什么不能混用？
- **单引号用于数据，双引号用于结构**。这是SQL语法的基础规则。
- 如果混用，数据库会混淆意图。例如：
  ```sql
  SELECT Sname AS 'Student Name'   -- 错误
  ```
  大多数数据库（如PostgreSQL、Oracle、SQLite）会直接报错，因为它期望一个标识符，却得到了一个字符串。
- **MySQL的例外**：MySQL在默认配置下允许使用单引号作为别名，但这**不符合SQL标准**，且可能导致可移植性问题。例如：
  ```sql
  SELECT Sname 'Student Name'   -- 在MySQL中可能工作，但不应依赖
  ```
  但在严格模式下或未来版本中可能被禁止。

---

### 4. 不同数据库的别名引用方式
| 数据库         | 引用标识符的符号                                | 示例                          |
| -------------- | ----------------------------------------------- | ----------------------------- |
| **SQL标准**    | 双引号 (`"`)                                    | `SELECT col AS "My Alias"`    |
| **MySQL**      | 反引号（``` ` ```）或双引号（取决于`sql_mode`） | ``SELECT col AS `My Alias` `` |
| **SQL Server** | 方括号（`[]`）或双引号                          | `SELECT col AS [My Alias]`    |
| **PostgreSQL** | 双引号 (`"`)                                    | `SELECT col AS "My Alias"`    |
| **Oracle**     | 双引号 (`"`)                                    | `SELECT col AS "My Alias"`    |

- 为了**可移植性**，建议始终使用**双引号**作为标识符引用符，这是SQL标准的要求。

---

### 5. 什么时候必须加引号？
- **如果别名符合以下所有条件，可以不加引号**：
  - 只包含字母、数字、下划线（`_`）。
  - 以字母开头。
  - 不是SQL保留字（如 `SELECT`、`WHERE`）。
  - 不区分大小写（大多数数据库自动转为大写或小写）。
- **如果别名违反以上任意一条，必须加双引号**（或对应数据库的引用符）：
  ```sql
  SELECT Sname AS "Student Name"   -- 包含空格，必须加
  SELECT Sname AS "2025_Year"      -- 以数字开头，必须加
  SELECT Sname AS "select"         -- 保留字，必须加
  ```

---

### 6. 别名的最佳实践
- **尽量使用简单、无空格的别名**（如 `student_name`），这样通常可以省略引号，避免麻烦。
- 如果必须使用特殊字符或保留字，**始终使用双引号**（或根据数据库调整）。
- **使用 `AS` 关键字**使意图更清晰，例如 `SELECT col AS "My Alias"`，而不是直接 `SELECT col "My Alias"`（后者在MySQL中有效，但标准SQL要求 `AS` 或空格都可，但加 `AS` 更清晰）。

---

### 总结
| 符号 | 用途                       | 示例                       |
| ---- | -------------------------- | -------------------------- |
| `'`  | 字符串常量                 | `SELECT 'Hello'`           |
| `"`  | 标识符（别名、列名、表名） | `SELECT col AS "My Alias"` |

因此，当别名需要特殊处理（空格、保留字、大小写敏感）时，必须使用**双引号**（`"`），而不能用单引号（`'`）。
