## 第1页：查询只选修了1门课程的学生姓名（错误示例）

**原始SQL**：
```sql
SELECT sname
FROM students
WHERE EXISTS (
    SELECT sid, COUNT(cid)
    FROM sc
    WHERE sid = students.sid
    GROUP BY sid
    HAVING COUNT(cid) = 1
);
```

### 错误分析
- **`EXISTS` 的语义**：`EXISTS` 只关心子查询是否返回任何行，而不关心子查询返回的具体数据。它通常用于存在性测试，而不是用于聚合后的条件判断。
- **子查询中的 `GROUP BY` 和 `HAVING`**：虽然子查询可以包含 `GROUP BY`，但这里 `EXISTS` 只需要知道是否有分组满足 `HAVING` 条件。如果存在这样的分组，子查询就会返回行，`EXISTS` 为真。因此，从逻辑上讲，这个写法**可能正确**，但存在几个问题：
  - 子查询返回了 `sid` 和 `COUNT(cid)` 两列，而 `EXISTS` 并不需要这些列，但语法上允许。不过，通常我们会写 `SELECT 1` 或 `SELECT *`，避免无意义的列。
  - 更重要的是，这个子查询是**相关子查询**，对于每个学生，它都会执行一次分组统计。如果学生选课很多，分组后 `HAVING COUNT(cid)=1` 可能不成立，子查询返回空，`EXISTS` 为假。如果学生只选了一门课，则分组后计数为1，子查询返回一行，`EXISTS` 为真，学生被选中。
- **潜在问题**：如果学生没有选任何课程，那么子查询中 `WHERE sid = students.sid` 会筛选出0行，`GROUP BY` 后没有分组，子查询返回空，`EXISTS` 为假，该学生不会被选中。这符合需求吗？题目要求“只选修了1门课程”，未选课的学生不应该被包含，所以没问题。

### 改进建议
虽然这个写法能工作，但不够简洁。更常见的是使用 `IN` 结合子查询（如第2页），或者使用连接后分组。但注意，这个 `EXISTS` 写法实际上等同于：
```sql
SELECT sname
FROM students
WHERE (SELECT COUNT(*) FROM sc WHERE sc.sid = students.sid) = 1;
```
这样更清晰。

## 第2页：查询只选修了1门课程的学生姓名（改进示例）

**原始SQL**：
```sql
SELECT sname
FROM students
WHERE sid IN (
    SELECT sid
    FROM sc
    WHERE sid = students.sid   -- 这里有问题！
    GROUP BY sid
    HAVING COUNT(cid) = 1
);
```

### 错误分析
- **问题在于子查询中的 `WHERE sid = students.sid`**：这里子查询引用了外层表的 `sid`，使得子查询变成了**相关子查询**。但 `IN` 通常用于非相关子查询，即子查询独立执行一次，返回一个值列表。如果子查询是相关的，那么对于外层每一行，子查询都会执行一次，但 `IN` 的语义是检查当前值是否在子查询的结果集中，而子查询的结果集依赖于当前行，这会导致逻辑混乱，且效率极低。
- **实际上，这个写法在语法上可能被某些数据库接受，但结果往往不正确**。因为子查询中的 `WHERE sid = students.sid` 使得子查询只返回当前学生的 `sid`（如果满足 `HAVING` 条件），那么 `sid IN (这个学生的sid)` 对于该学生来说，如果子查询返回了他的sid，则条件为真，否则为假。这等价于检查该学生是否满足 `HAVING` 条件，但写法极其绕，且因为相关，性能差。
- **正确做法**：应该去掉子查询中的 `WHERE sid = students.sid`，使其成为非相关子查询：
  ```sql
  SELECT sname
  FROM students
  WHERE sid IN (
      SELECT sid
      FROM sc
      GROUP BY sid
      HAVING COUNT(cid) = 1
  );
  ```
  这样，子查询先找出所有只选了一门课的学生ID，然后外层从中选出姓名。

### 为什么原写法可能“碰巧”正确？
如果数据库执行时，对于外层每个学生，子查询只返回该学生的sid（如果满足条件），那么 `IN` 实际上变成了一个复杂的 `EXISTS` 等价形式。但这是不可靠的，且违背了 `IN` 的通常用法。因此，一定要避免这种混用。

## 第3页：查询教师的姓名，要求其年龄大于其所任职称的教师的平均年龄

**原始SQL**：
```sql
SELECT pname
FROM teacher t1
WHERE age > (SELECT AVG(age)
             FROM teacher t2
             WHERE t1.title = t2.title);
```

### 正确分析
- 这是一个典型的**相关子查询**，用于计算每个教师的职称平均年龄，并与其自身年龄比较。
- **执行逻辑**：
  1. 外层从 `teacher` 取一行（例如某职称的教师）。
  2. 子查询计算同一职称的所有教师的平均年龄（注意不包括当前行？这里子查询中 `t2` 是全部教师，包括当前行，所以平均年龄包括了自己。但通常我们想要的是“同职称其他教师的平均年龄”还是“所有同职称教师的平均年龄”？题目说“大于其所任职称的教师的平均年龄”，这个平均年龄通常包括他自己。如果包括自己，那么条件就是年龄大于平均年龄，这相当于年龄大于组内平均，即高于平均水平。这没问题。
  3. 如果当前教师的年龄大于这个平均年龄，则被选中。
- **注意事项**：
  - 如果某个职称的教师人数为0，子查询返回 `NULL`，则 `age > NULL` 结果为 `UNKNOWN`，该教师不会被选中。但这种情况不会发生，因为外层教师存在，该职称至少有一人。
  - 如果需要排除自己，可以在子查询中加 `AND t2.pid != t1.pid`，但题目未明确。
- **性能**：相关子查询会为每个教师执行一次，如果教师表很大，可能较慢。可以改写为连接+分组，但子查询更直观。

### 等价写法（使用连接）
```sql
SELECT t1.pname
FROM teacher t1
JOIN (SELECT title, AVG(age) AS avg_age FROM teacher GROUP BY title) t2
ON t1.title = t2.title
WHERE t1.age > t2.avg_age;
```
这种写法先计算每个职称的平均年龄，然后连接，通常效率更高。

## 第4页：查询所有课程都不及格的学生姓名

**原始SQL**：
```sql
SELECT sname
FROM student
WHERE 60 > ALL (SELECT score
                FROM sc
                WHERE student.sid = sc.sid);
```

### 正确分析
- **`> ALL` 的含义**：`60 > ALL (子查询)` 要求60大于子查询返回的**每一个**值，即所有成绩都小于60（因为成绩小于60才是不及格）。所以这个查询选出所有成绩都小于60的学生。
- **执行逻辑**：
  1. 对于每个学生，子查询返回该学生的所有成绩（可能有多行，也可能为空）。
  2. 如果子查询为空（学生没有成绩），那么 `60 > ALL (空集)` 的结果是**真**（因为空集中所有值都满足条件，空真）。这会导致从未选课的学生也被认为“所有课程都不及格”，这可能不符合题意。通常我们可能希望只考虑有成绩的学生，或者明确排除未选课的情况。
  3. 如果子查询中有成绩，则检查是否每个成绩都小于60。如果有一个成绩大于等于60，则条件为假。
- **陷阱**：子查询中如果存在 `NULL` 成绩，`NULL` 与60比较的结果是 `UNKNOWN`，那么 `ALL` 条件会怎样？根据SQL标准，如果子查询中有 `NULL`，`x > ALL` 的结果通常是 `UNKNOWN`（因为存在未知值），导致该行不被选中。但实际中，成绩字段通常不会有 `NULL`，或者我们应排除 `NULL`（例如用 `WHERE score IS NOT NULL`）。如果成绩可能为 `NULL`，且我们想忽略 `NULL`，可以在子查询中加 `WHERE score IS NOT NULL`。
- **改进**：为了确保只考虑有成绩的学生，可以添加条件确保学生有成绩，或者用 `NOT EXISTS` 结合其他逻辑。例如，找出所有成绩都小于60且至少有一门成绩的学生：
  ```sql
  SELECT sname
  FROM student s
  WHERE EXISTS (SELECT 1 FROM sc WHERE sid = s.sid)  -- 有成绩
    AND 60 > ALL (SELECT score FROM sc WHERE sid = s.sid AND score IS NOT NULL);
  ```
  或者用 `NOT EXISTS` 找出没有一门成绩大于等于60的学生：
  ```sql
  SELECT sname
  FROM student s
  WHERE NOT EXISTS (SELECT 1 FROM sc WHERE sid = s.sid AND score >= 60);
  ```
  这个写法更清晰，且自动处理空成绩（如果没有成绩，`NOT EXISTS` 为真，会包括未选课学生，需结合需求）。

### 注意事项
- `ALL` 操作符需要子查询返回的集合中每个值都满足条件。如果子查询为空，结果恒为真，这可能导致逻辑错误，务必注意。
- 对于成绩为 `NULL` 的情况，最好在子查询中显式排除，或者使用 `IS NULL` 判断。

## 总结

这些示例展示了SQL中几种高级查询技巧，同时也揭示了常见的错误和陷阱。通过分析，我们应记住：
- **`EXISTS`** 只用于存在性测试，子查询中通常写 `SELECT 1`，不要滥用聚合。
- **`IN` 子查询**应为非相关，避免将相关条件混入。
- **相关子查询**常用于逐行比较，但可能影响性能，可考虑改写为连接。
- **`ALL` 和 `ANY`** 用于集合比较，注意空集和 `NULL` 的影响。
- 对于“所有……都……”的逻辑，优先考虑用 `NOT EXISTS` 转换，更安全。

掌握这些细节，能帮助你写出正确且高效的SQL查询。
