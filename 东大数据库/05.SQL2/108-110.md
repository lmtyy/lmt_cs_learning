这三页PPT主要介绍了SQL中的**子查询在不同子句中的角色**，以及**`WITH`子句（公用表表达式，CTE）**的用法。下面我将逐一详细讲解。

---

## 第1页：子查询在不同子句中的角色总结

### 1. 子查询在 `WHERE` 子句中
- **作用**：生成一个数据集（通常是一个值列表或一组值），外层查询用这个数据集进行条件过滤。
- **常见用法**：与 `IN`、`EXISTS`、`ANY`、`ALL` 等运算符结合。
- **示例**：
  ```sql
  SELECT sname FROM student
  WHERE sid IN (SELECT sid FROM sc WHERE grade > 90);
  ```
  子查询生成所有成绩>90的学生ID，外层查询据此筛选学生。

### 2. 子查询在 `FROM` 子句中
- **作用**：生成一个临时表（派生表），外层查询从这个临时表中查询数据。
- **必须为派生表指定别名**，因为临时表没有名字，外层需要引用它。
- **示例**：
  ```sql
  SELECT dept_avg.dept_name, dept_avg.avg_salary
  FROM (SELECT dept_name, AVG(salary) AS avg_salary FROM employee GROUP BY dept_name) AS dept_avg
  WHERE dept_avg.avg_salary > 5000;
  ```
  子查询生成每个部门的平均工资，外层再过滤。

### 3. 子查询在 `SELECT` 子句中
- **作用**：为外层查询的每一行计算一个额外的列值。这种子查询必须是**标量子查询**，即**只返回一行一列**。
- **如果子查询返回多行或空值，处理规则**：
  - 返回多行会导致运行时错误。
  - 返回空行，则结果为 `NULL`。
- **示例**：
  ```sql
  SELECT name,
         (SELECT MAX(age) FROM person WHERE person.country_id = country.id) AS max_age
  FROM country;
  ```
  对于每个国家，子查询返回该国人员的最大年龄，没有人员则返回 `NULL`。

### 小结
这一页清晰地区分了子查询的三种位置及其产出物的类型：
- `WHERE` → 值集（用于条件）
- `FROM` → 表（用于查询）
- `SELECT` → 标量值（用于输出）

## 第2页：`WITH` 子句（公用表表达式，CTE）介绍

**`WITH` 子句**允许你在一个查询中定义**临时视图**，这个视图仅在当前查询中有效。它可以把复杂的查询分解成多个可读的步骤，类似于把子查询提取出来命名。

### 基本语法
```sql
WITH cte_name (column_list) AS (
    subquery
)
SELECT ...
FROM cte_name ...;
```

### 示例：找出所有余额最大的账户
```sql
WITH max_balance (value) AS (
    SELECT MAX(balance) FROM account
)
SELECT account_number
FROM account, max_balance
WHERE account.balance = max_balance.value;
```

**解析**：
1. **`WITH max_balance (value) AS (...)`**：定义了一个名为 `max_balance` 的临时视图，它有一个列 `value`，内容为账户表的最大余额。
2. 然后外层查询将 `account` 表与这个临时视图进行笛卡尔积（实际是交叉连接），筛选出余额等于最大值的账户。
3. 等价于：
   ```sql
   SELECT account_number
   FROM account
   WHERE balance = (SELECT MAX(balance) FROM account);
   ```

**优点**：
- 将最大余额的计算独立出来，提高可读性。
- 如果后续需要多次使用这个中间结果，CTE 可以避免重复写子查询（但这里只用了一次，主要是清晰）。

### 注意
- `WITH` 子句定义的临时视图只在紧随其后的查询中有效。
- 可以定义多个 CTE，用逗号分隔。

## 第3页：使用 `WITH` 子句的复杂查询

**需求**：找出所有分支的存款总额大于所有分支平均存款总额的分支。

### SQL语句（原PPT内容整理）
```sql
WITH branch_total(branch_name, value) AS (
    SELECT branch_name, SUM(balance)
    FROM account
    GROUP BY branch_name
),
branch_total_avg(value) AS (
    SELECT AVG(value) FROM branch_total
)
SELECT branch_name
FROM branch_total, branch_total_avg
WHERE branch_total.value >= branch_total_avg.value;
```

### 逐步解析
1. **第一个CTE：`branch_total`**
   - 计算每个分支的存款总额（`SUM(balance)`），结果是一个表，包含两列：`branch_name` 和 `value`（每个分支的总存款）。
2. **第二个CTE：`branch_total_avg`**
   - 基于第一个CTE的结果，计算所有分支总存款的平均值。结果是一个单行单列的表，列名为 `value`。
3. **最终查询**
   - 将 `branch_total` 与 `branch_total_avg` 进行笛卡尔积（因为没有连接条件，所以每一行都会与平均值组合）。
   - 筛选出 `branch_total.value >= branch_total_avg.value` 的分支。

### 这个查询的优点
- **分步清晰**：先算每个分支的总和，再算这些总和的平均，最后比较。每一步都用一个CTE表达，非常直观。
- **避免多层嵌套**：如果不使用CTE，可能需要多层子查询嵌套，可读性差。例如等价写法：
  ```sql
  SELECT branch_name
  FROM account
  GROUP BY branch_name
  HAVING SUM(balance) >= (SELECT AVG(total) 
                          FROM (SELECT SUM(balance) AS total 
                                FROM account 
                                GROUP BY branch_name) AS totals);
  ```
  这个嵌套写法虽然正确，但逻辑不如CTE清晰。

### 注意点
- CTE 是按顺序定义的，后面的CTE可以引用前面定义的CTE。
- 最终查询可以引用所有CTE，就像引用普通表一样。
- 这里使用了笛卡尔积，但因为是常数平均值，实际上没有性能问题。如果平均值也是按组变化的，可能需要连接条件。

## 总结

| 知识点         | 作用                       | 关键点                                                       |
| -------------- | -------------------------- | ------------------------------------------------------------ |
| **子查询位置** | 决定子查询的输出用途       | WHERE：值集；FROM：派生表；SELECT：标量值                    |
| **标量子查询** | 在SELECT中返回单值         | 必须保证返回一行一列，否则报错或返回NULL                     |
| **WITH子句**   | 定义临时视图，简化复杂查询 | 可定义多个，用逗号分隔；后定义的可以引用前面的；仅在当前查询有效 |

CTE 是编写复杂SQL的利器，它让查询逻辑更接近人类思维，特别适合需要多步计算或多次引用同一中间结果的场景。
