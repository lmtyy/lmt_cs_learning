这三页PPT讲解的是数据库规范化理论中非常核心、但也最容易让人头晕的操作：**如何测试并剔除函数依赖集（规则库）中的“多余属性（Extraneous Attribute）”**。

这其实就是求“最小函数依赖集（Minimal Cover）”的核心武功秘籍。

为了让你彻底搞懂，我把这个过程比作**“抓出规则库里的混子（吃白食的属性）”**。这三页PPT分为两部分：第一、二页教你怎么抓“左边的混子”，第三页教你怎么抓“右边的混子”。

下面我为你逐页详细拆解：

------

### 第一页：测试左侧属性是否多余 (理论篇)

这页PPT定义了：在规则 $\alpha \rightarrow \beta$ 中，如何判断左边（前提条件 $\alpha$）里是不是混进了没用的属性。

- **前提设定**：
  - $F$ 是我们手里所有的规则集合。
  - $\alpha \rightarrow \beta$ 是 $F$ 里面的某一条具体规则（比如 $ABC \rightarrow D$，这里 $\alpha$ 就是 $ABC$，$\beta$ 就是 $D$）。
- **测试目标**：我们要测试左边集合 $\alpha$ 里面的某个特定属性 $A$（比如 $ABC$ 里的 $A$），是不是个没用的“混子”。

**PPT给出的两步算法：**

1. **第一步（把嫌疑人踢出去）**：`compute ({a} - A)+ using the dependencies in F`
   - **大白话解释**：把嫌疑人 $A$ 从左边的小团体 $\alpha$ 里踢出去，看看剩下的人（即 $\alpha - A$）拿着整个规则库 $F$ 去求闭包（滚雪球）。
   - *举个例子*：在 $ABC \rightarrow D$ 中测试 $A$，我们就去算 $(BC)^+$。
2. **第二步（看剩下的人行不行）**：`check that ({a} - A)+ contains all attributes of \beta; if it does, A is extraneous`
   - **大白话解释**：看看踢掉 $A$ 之后，剩下的人求出的闭包里，**还包不包含原来的目标 $\beta$**。
   - 如果**包含**：说明没有 $A$，剩下的人照样能推导出 $\beta$。那 $A$ 就是个纯纯的“混子”（extraneous），可以直接从这条规则里删掉！
   - 如果**不包含**：说明缺了 $A$ 就推导不出 $\beta$ 了，$A$ 是不可或缺的核心骨干，不能删。

------

### 第二页：测试左侧属性是否多余 (实战演练)

这页PPT直接用一个具体的例子，带你跑一遍上一页的理论。

- **已知规则库 $F$**：$\{abd \rightarrow e, ab \rightarrow g, b \rightarrow f, c \rightarrow j, cj \rightarrow i, g \rightarrow h\}$

**任务 1：测试 $abd \rightarrow e$ 里面有没有混子**

- *PPT只写了开头，我帮你补全整个思维过程：*
  - 要测试 $a$，就求踢掉 $a$ 之后的闭包 $(bd)^+$。查规则库，$(bd)^+ = bdf$。里面没有 $e$，说明 $a$不能删。
  - 要测试 $b$，求 $(ad)^+$。查规则库，$(ad)^+ = ad$。没有 $e$，说明 $b$ 不能删。
  - 要测试 $d$，求 $(ab)^+$。查规则库，$(ab)^+ = abgf h$。没有 $e$，说明 $d$ 也不能删。
  - **结论**：$abd \rightarrow e$ 里面全是骨干，没有混子。

**任务 2：测试 $cj \rightarrow i$（PPT的核心演示）**

- **目标**：测试 $cj$ 这个左边小团体里，有没有混子。我们先**怀疑 $j$ 是混子**。
- **套用算法第一步**：把 $j$ 踢出去，剩下 $c$。我们在整个规则库 $F$ 里去求 $(c)^+$。
- **计算 $(c)^+$**：
  - 一开始有 $c$。
  - 根据 $c \rightarrow j$，我们得到了 $j$。现在集合里有 $cj$。
  - 根据 $cj \rightarrow i$，我们得到了 $i$。现在集合里有 $cji$。
  - 所以，$(c)^+ = cji$。
- **套用算法第二步**：看看 $(c)^+$ 里有没有我们原来的目标 $i$？
  - 有！
- **得出结论**：原来只要有 $c$ 自己，绕了一圈也能推导出 $i$。那个 $j$ 待在 $cj \rightarrow i$ 的左边完全是多余的。$j$ 是 extraneous（多余的）。
- **最终净化**：规则库里的 $cj \rightarrow i$ 被精简成了 **$c \rightarrow i$**。最下面那行给出了精简后的全新规则库。

------

### 第三页：测试右侧属性是否多余 (最容易踩坑的地方)

第一页测试的是左边的条件，这页PPT测试的是**右边的结果**里有没有废话。

- *比如规则 $A \rightarrow BC$ 中，如果别的规则已经能证明 $A \rightarrow B$ 了，那这条规则右边的 $B$ 就是一句废话，应该精简成 $A \rightarrow C$。*
- **前提设定**：测试 $\alpha \rightarrow \beta$ 中，右侧结果 $\beta$ 里面的属性 $A$ 是不是多余的。

**PPT给出的两步算法（注意这里的致命细节）：**

1. **第一步（伪造一个没有A的新规则库）**：`compute a+ using only the dependencies in F' = (F - {a -> \beta}) \cup {a -> (\beta - A)}`
   - **大白话解释**：这步非常非常关键！你要测试 $\alpha$ 到底能不能通过*其他路径*推导出 $A$，你**绝对不能用原来的规则库 $F$**。
   - 为什么？因为原库 $F$ 里明明白白写着 $\alpha \rightarrow \beta$（里面包含 $A$）。你如果用原库算 $\alpha^+$，那百分之百会包含 $A$。这就成了“用你要证明的东西去证明它自己”，毫无意义。
   - **正确做法**：你要“蒙住眼睛”。把原规则库 $F$ 里的 $\alpha \rightarrow \beta$ 这条规则抠出来，把右边的 $A$ 强行抹掉，变成 $\alpha \rightarrow (\beta - A)$，然后再塞回规则库。这个被动过手脚的新规则库叫 **$F'$**。
   - 然后，用这个**新规则库 $F'$** 去算左边 $\alpha$ 的闭包 $\alpha^+$。
2. **第二步（验证结论）**：`check that a+ contains A; if it does, A is extraneous`
   - **大白话解释**：我们已经把“$\alpha$ 能直接决定 $A$”这条显式规则给抹掉了。现在看看在 $F'$ 下求出的 $\alpha^+$，**是不是依然奇迹般地包含了 $A$**？
   - 如果**包含**：这说明什么？说明即使我不直接写出来，靠着别的规则绕来绕去，$\alpha$ 依然能推导出 $A$！那么原规则右边写上 $A$ 就是一句废话，$A$ 是多余的。
   - 如果**不包含**：说明抹掉之后，就真的推导不出 $A$ 了。那这个 $A$ 就不是废话，必须老老实实写在规则右边。

**总结一下：**

- **查左边的混子**：把嫌疑人从左边踢掉，用**原规则库 $F$** 求剩下人的闭包，看还能不能推导出结果。
- **查右边的废话**：把嫌疑人从右边抹掉，构造**残缺的新规则库 $F'$**，用左边的原班人马去求闭包，看还能不能找回嫌疑人。
