## 第一页：原因与解决方法

**原文内容**  
> 原因  
> Sdept、Sloc部分函数依赖于码。  
>
> 解决方法  
> 采用投影分解法，把SLC分解为两个关系模式，以消除这些部分函数依赖。  
> SC (Sno, Cno, Grade)  
> SL (Sno, Sdept, Sloc)

### 知识点讲解

1. **问题根源回顾**  
   - 在SLC(Sno, Sdept, Sloc, Cno, Grade)中，码是(Sno, Cno)。  
   - 非主属性Sdept和Sloc部分依赖于码（因为Sno→Sdept，Sno→Sloc，而Sno是码的真子集）。  
   - 正是这种部分依赖导致了之前讲过的插入异常、删除异常、数据冗余和修改复杂。

2. **解决思路**  
   - 将**部分依赖的属性**与**它所依赖的码的一部分**分离出去，形成独立的关系。  
   - 这样，每个新关系中的非主属性都完全依赖于该关系的码。

3. **投影分解法**  
   - 将原关系R分解为两个（或多个）关系R1和R2，使得：  
     - 属性集是R的子集，且并集等于R的所有属性。  
     - 分解后的关系能够通过自然连接恢复原信息（无损连接分解）。  
   - 这里的具体分解：  
     - **SC(Sno, Cno, Grade)**：存放选课与成绩，码为(Sno, Cno)。  
       - Grade完全依赖于(Sno, Cno)。  
       - Sdept和Sloc被移走，不再部分依赖。  
     - **SL(Sno, Sdept, Sloc)**：存放学生基本信息，码为Sno。  
       - 由于Sno是单属性，Sdept和Sloc自动完全依赖于Sno。  
       - 注意：Sdept→Sloc仍然存在（传递依赖），这留待3NF解决。

4. **分解后的效果**  
   - 插入异常消除：未选课的学生可以单独插入SL表。  
   - 删除异常消除：学生退选所有课程，只删除SC中的记录，SL中学生信息保留。  
   - 冗余消除：Sdept和Sloc只在SL中存储一次。  
   - 修改简化：转系只需修改SL中的一条记录。

5. **关键术语**  
   - **投影分解**：关系代数中的投影操作，选择部分属性构成新关系。  
   - **无损连接分解**：分解后的关系自然连接能还原原关系（此处因Sno是公共属性，满足无损）。  
   - **保持函数依赖**：原函数依赖是否被分解后的模式所保持（这里Sdept→Sloc在SL中保持，但Sno→Sdept、Sno→Sloc也都保持）。

## 第三页：分解后的函数依赖图

**原文内容**（有乱码，但整理后应为）  
> 函数依赖图：  
> SC  
> Grade  
> Sno  Cno  
>
> SL  
> Sno → Sdept, Sno → Sloc, Sdept → Sloc

### 知识点讲解

1. **SC 关系的依赖图**  
   - 关系：SC(Sno, Cno, Grade)  
   - 码：(Sno, Cno)  
   - 依赖：  
     - (Sno, Cno) → Grade（完全依赖）。  
     - 不存在部分依赖，因为Grade是唯一的非主属性，且码是复合属性，但Sno单独不决定Grade，Cno单独也不决定Grade。  
   - 图中可能显示：Grade 同时依赖于 Sno 和 Cno（两者共同决定）。

2. **SL 关系的依赖图**  
   - 关系：SL(Sno, Sdept, Sloc)  
   - 码：Sno  
   - 依赖：  
     - Sno → Sdept  
     - Sno → Sloc  
     - Sdept → Sloc（传递依赖，在2NF中允许，但属于3NF要消除的）  
   - 图中箭头：从Sno指向Sdept和Sloc，再从Sdept指向Sloc。

3. **函数依赖图的作用**  
   - 直观展示依赖关系，便于分析范式等级。  
   - 对于SC：完全依赖，满足2NF。  
   - 对于SL：Sno是单属性，自动满足2NF，但存在传递依赖，因此SL ∈ 2NF，但 ∉ 3NF。

4. **分解后的范式等级总结**  
   - SC ∈ 2NF（且实际上也满足3NF和BCNF，因为没有传递依赖）。  
   - SL ∈ 2NF（但不满足3NF，因为存在传递依赖Sno→Sdept→Sloc）。  
   - 原SLC ∉ 2NF，分解后两个关系均达到2NF，消除了部分依赖引起的异常。

## 整体知识点总结

| 项目          | 内容                                            |
| ------------- | ----------------------------------------------- |
| **问题原因**  | 非主属性Sdept、Sloc部分依赖于码(Sno,Cno)        |
| **解决方法**  | 投影分解法，将SLC拆成SC和SL                     |
| **分解结果**  | SC(Sno,Cno,Grade)，SL(Sno,Sdept,Sloc)           |
| **SC依赖**    | (Sno,Cno) → Grade（完全依赖）                   |
| **SL依赖**    | Sno→Sdept, Sno→Sloc, Sdept→Sloc（存在传递依赖） |
| **分解后2NF** | 满足（消除了部分依赖）                          |
| **仍需改进**  | SL需要进一步分解到3NF（消除传递依赖）           |

这三页PPT的核心是：**通过投影分解消除部分函数依赖，将关系提升到第二范式**，并展示了分解前后的函数依赖图，为后续学习第三范式做铺垫。
