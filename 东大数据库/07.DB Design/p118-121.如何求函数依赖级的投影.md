这四页 PPT 详细讲解了数据库规范化过程中的一个核心操作：**如何求函数依赖集的投影（Projection of Functional Dependencies）**。

在上一节课中，我们知道了“拆表（模式分解）”不仅要把列分家，还要把规则（依赖）也分家。这四页 PPT 就是在教你，当你把大表 $R$ 拆出一个小表 $R_1$ 后，**究竟该怎么算出 $R_1$ 能够带走哪些规则**。

我将按照“理论定义 $\rightarrow$ 算法步骤 $\rightarrow$ 实例演练”的逻辑，为你详细拆解这四页的内容：

------

### 第一页：明确任务 —— 函数依赖集投影的输入与输出

这页 PPT 相当于我们写代码时的函数签名声明，定义了我们要解决的问题的边界。

**1. 输入 (Input):**

- **关系 $R$**：拆分前的原始大表（包含所有的属性）。
- **模式分解计算得到的关系 $R_1$**：我们刚刚从 $R$ 中切分出来的一张小表（它只包含 $R$ 的一部分属性）。
- **在 $R$ 成立的 FD 集合 $S$**：大表原本遵循的所有函数依赖（规则库）。*(注：为了不跟前面的 $F$ 混淆，这里 PPT 把大表的规则库叫成了 $S$)*。

**2. 输出 (Output):**

- **在 $R_1$ 中成立的 FD 集合**：我们要求出的目标。即 $R_1$ 这张小表，根据大表的家规 $S$，能够“合法继承”并且在自己内部“自圆其说”的那些规则。

------

### 第二页：核心算法 —— 怎么算投影？

这页 PPT 给出了求投影的标准算法（伪代码）。这个算法的精髓在于**“穷举子集求闭包”**。

**1. 初始化：**

- `令 T 为最终输出的 FD 集合，初始化 T 为空。`
- **解释**：准备一个空篮子 $T$，用来装我们给 $R_1$ 找到的规则。

**2. 核心大循环（穷举与测试）：**

- `对于 R1 的属性集合的每一个子集 X，计算 X+。该计算依据 FD 集合 S，可能会涉及一些在 R 中但不在 R1 中的属性。`
- **解释**：这是最烧脑的一步。假设 $R_1$ 包含属性 $\{A, C\}$。我们需要找出 $\{A, C\}$ 的**所有非空子集**（即 $\{A\}, \{C\}, \{A, C\}$），把它们分别当做“前提条件 $X$”。
- 然后，拿着这个前提条件 $X$，去**大表的规则库 $S$** 里去滚雪球算闭包 $X^+$。注意，虽然我们在为小表找规则，但算闭包必须借用大表的全部力量。
- `对于所有在 X+ 且属于 R1 的属性 A，将所有非平凡的 FD X -> A 添加到 T 中。`
- **解释**：算出了闭包 $X^+$ 后，我们在这个大闭包里**“淘宝”**。我们要找的是满足以下条件的属性 $A$：
  1. $A$ 是在这个闭包里被推导出来的（意味着 $X \rightarrow A$ 成立）。
  2. $A$ **必须属于小表 $R_1$**（不能把不属于自己表的属性强行拉进来定规矩）。
  3. $X \rightarrow A$ 必须是**非平凡的**（即 $A$ 不能本来就是 $X$ 里面的一部分，别弄一堆废话加进去）。
- 找到符合条件的 $A$ 后，把 $X \rightarrow A$ 这条规则放进篮子 $T$ 里。

**3. 后处理（精简）：**

- `T 是在 R1 中成立的 FD 基本集。对 T 进行改造成最小化基本集。`
- **解释**：循环结束后，篮子 $T$ 里可能装了一大堆繁杂的、有冗余的规则。这就需要用到你前几节课学到的功夫——把 $T$ 化简成**Canonical Cover（最小函数依赖集）**。化简后的结果，才是小表最终带走的干净家规。

------

### 第三页：实战演练（题目设定）

这页 PPT 给出了一个经典的例题，用来演示上一页的算法。

**1. 题目已知条件：**

- **大表 $R$** = $(A, B, C, D)$
- **大表的规则库 $S$ (这里写成了 FD)** = $\{A \rightarrow B, B \rightarrow C, C \rightarrow D\}$ （一条长长的推导链）
- **拆出的小表 $R_1$** = $(A, C, D)$。 *注意：小表里没有 $B$！*

**2. 任务：**

- 求 $R_1$ 的 FD（即求 $S$ 在 $R_1$ 上的投影）。

------

### 第四页：实战演练（步骤拆解）

这页 PPT 一步步展示了如何用“穷举子集求闭包”的方法解开上一页的题。

**前情提要**：小表 $R_1$ 包含 $\{A, C, D\}$。它的子集有 $A, C, D, AC, AD, CD$（通常从单个属性开始测）。

**1. 测试子集 $A$：**

- `A+ = {A, B, C, D}`：在大表规则下算闭包（因为 $A \rightarrow B \rightarrow C \rightarrow D$）。
- **去闭包里“淘宝”**：闭包里有 $B, C, D$。但是 $B$ 不在小表 $R_1$ 里，必须剔除。剩下的 $C$ 和 $D$ 都在小表里，且 $A \rightarrow C$ 和 $A \rightarrow D$ 都是非平凡的。
- `T: A -> C, A -> D`：把这两条规则放进篮子。

**2. 测试子集 $C$：**

- `C+ = {C, D}`：在大表规则下算闭包。
- **淘宝**：闭包里多出了 $D$，$D$ 在小表里，且非平凡。
- `T: {C -> D}`：把规则放进篮子。

**3. 测试子集 $D$：**

- `D+ = {D}`：推不出任何新东西。不产生新规则。

**4. 测试子集 $CD$：**

- `CD+ = {C, D}`：也推不出新东西。（实际上，由于 $A, C, D$ 的单属性闭包已经求完，往往不需要穷尽所有的多属性组合，除非有类似 $AC \rightarrow D$ 这样只有组合才能触发的隐藏规则。这题比较简单，不需要考虑）。

**5. 汇总初步结果（未化简）：**

- `5 T: A -> C, A -> D, C -> D`：这就是把所有合格的规则都装进篮子后的状态。这叫**投影的基本集**。

**6. 最终精简（求最小依赖集）：**

- 观察篮子里的集合 $\{A \rightarrow C, A \rightarrow D, C \rightarrow D\}$。
- 你能一眼看出里面的**冗余**吗？
- 因为有 $A \rightarrow C$，且有 $C \rightarrow D$。根据传递律，系统自己就能推导出 $A \rightarrow D$！
- 所以，$A \rightarrow D$ 是一条彻头彻尾的废话规则，必须从篮子里剔除。
- `6 T: A -> C, C -> D`：**这就是小表 $R_1$ 最终合法继承、并且精简到了极致的函数依赖集（投影的最终结果）**。

**总结这四页：**

这四页告诉你，分家（拆表）时分家规，不是简单地看一眼原来的规则就把包含这几个字母的规则抄过来。你必须利用大表的隐含逻辑（比如那个被丢掉的 $B$ 充当了传递的桥梁），重新推导出一套适用于新环境的全新规则。
