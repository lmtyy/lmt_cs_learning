### 第一页：Relation Definition（关系的定义 —— 行与列的术语）

这一页给出了关系模型中两个最基础的称谓：**元组** 和 **属性**。

**原文内容：**
> - Each row of a relation can be referred to as **tuple**
> - Each column of a relation has a name → named columns of a relation are referred to as **attributes**

**详细知识点讲解：**

**1. Tuple（元组）**
- **定义**：关系（表）中的**一行**。
- **术语来源**：来自数学中的“n 元组”（n-tuple），表示 n 个值的有序序列。
- **核心特性**：
    - 元组代表一个**具体的实体**或**联系实例**。
    - 在一个规范的关系中，**元组是无序的**——从数学集合的角度看，行的先后没有意义（尽管物理存储有先后）。
- **例子**：在 `FAMILY(MAN, WOMAN, CHILD)` 中，`('王兵', '丁梅', '王一')` 就是一个 **3 元组**。

**2. Attribute（属性）**
- **定义**：关系（表）中的**一列**，每一列都有一个**名称**。
- **术语来源**：实体 - 联系模型中，实体具有“属性”。
- **核心特性**：
    - 属性名在同一个关系模式中必须**唯一**。
    - 每个属性都绑定一个**域**，限制该列可取的值的范围。
    - 属性的顺序在关系模型中理论上也是**无序的**（实践中通过列名访问，而非位置）。
- **例子**：`MAN`、`WOMAN`、`CHILD` 都是 `FAMILY` 关系的属性。

> **小结**：这一页是术语统一页。
> - 行 = **元组**
> - 列 = **属性**
> - 后续学习中，这两个词会频繁出现。

### 第二页：Basic Structure（基本结构度量）

这一页引入了衡量关系大小的两个关键量化指标：**基数** 和 **度**。

**原文内容：**
> - The **cardinality** of a table T is the number of tuples in T, denoted by |T|.
> - The **arity**, or **degree**, of a table is the number of columns or attributes.

**详细知识点讲解：**

**1. Cardinality（基数）**
- **符号**：|T| 或 |r|
- **定义**：关系（表）中**元组的个数**。
- **通俗理解**：表有多少**行**。
- **重要性**：
    - 是查询优化器估计查询代价的核心参数。
    - 是数据库统计信息中最基础的一项。
- **例子**：
    - 前面讲过的 `FAMILY` 表有 3 行，因此它的基数 |FAMILY| = 3。
    - 一个空表（没有任何元组）的基数为 0。

**2. Arity / Degree（目 / 度）**
- **术语**：Arity（来自英文后缀 -ary，如 binary 二元的）或 Degree。
- **定义**：关系中**属性的个数**。
- **通俗理解**：表有多少**列**。
- **分类**：
    - **一元关系**：度 = 1（只有一列）。例如：`ACTIVE_ACCOUNT_NUMBERS(A-101, A-102, ...)`。
    - **二元关系**：度 = 2。最常见于表示多对多联系的关系表，例如 `DEPOSITOR(customer_name, account_number)`。
    - **n 元关系**：度 = n。
- **例子**：
    - `FAMILY(MAN, WOMAN, CHILD)` 的度为 3。

**3. 关系的大小描述**
- 一个关系可以描述为：**基数为 3、度为 3 的关系**。
- 这两个指标分别从**纵向规模**（数据量）和**横向结构**（属性数）定义了表。

### 第三页：Relation Schema（关系模式的语法形式）

这一页给出了关系模式的形式化**记法**，并引入了**域与属性的绑定**。

**原文内容：**
> - \( A_1, A_2, \dots, A_n \) are attribute names
> - \( R = (A_1 : D_1, A_2 : D_2, \dots, A_n : D_n) \) is a relation schema
>
> `customer_schema = (customer_name char(20), customer_street char(50), customer_city char(50))`

**详细知识点讲解：**

**1. 属性名序列**
- 一个关系模式首先需要确定属性的**名称**：\( A_1, A_2, ..., A_n \)。
- 属性名在同一模式中不能重复。

**2. 完整的模式定义语法**
- **形式化记法**：\( R = (A_1 : D_1, A_2 : D_2, \dots, A_n : D_n) \)
- **含义**：
    - \( R \) 是关系名。
    - \( A_i \) 是第 i 个属性的名称。
    - \( D_i \) 是该属性对应的**域**（数据类型 + 约束）。
    - 冒号 `:` 读作“类型为”或“取值于”。

**3. 实际 SQL 风格的例子**
PPT 给出了 `customer_schema` 的例子：

```
customer_schema = (customer_name char(20), customer_street char(50), customer_city char(50))
```

- 关系名：`customer`
- 三个属性：
    - `customer_name`：定长 20 的字符串。
    - `customer_street`：定长 50 的字符串。
    - `customer_city`：定长 50 的字符串。

**4. 关系模式 vs. SQL 建表语句**
关系模式的这种记法，与 SQL 的 **DDL（数据定义语言）** 几乎一一对应：

```sql
CREATE TABLE customer (
    customer_name CHAR(20),
    customer_street CHAR(50),
    customer_city CHAR(50)
);
```

- 关系模式是逻辑设计文档中的**抽象描述**。
- CREATE TABLE 是在具体数据库管理系统中的**物理实现命令**。

### 第四页：Relation（关系实例的表示）

这一页将**模式**与**实例**联系起来，给出了用符号表示“某个模式下的具体关系”的方法。

**原文内容：**
> - \( r(R) \) denotes a relation \( r \) on the relation schema \( R \)
>
>   customer (customer_schema)
>
> - The current values (relation instance) a relation are specified by a table

**详细知识点讲解：**

**1. 符号约定：\( r(R) \)**
- **含义**：表示**名为 \( r \) 的关系**，该关系遵循**关系模式 \( R \)** 的定义。
- **解读**：小写 \( r \) 代表**实例**，大写 \( R \) 代表**模式**。
- 这一符号在教材和论文中广泛使用，用来区分“表的定义”和“表里的数据”。

**2. 举例：customer**
- `customer(customer_schema)` 表示：存在一个具体的关系（表），它的名字叫 `customer`，它的结构定义由 `customer_schema` 给出。
- 这里的 `customer_schema` 即上一页定义的：
    - `(customer_name char(20), customer_street char(50), customer_city char(50))`

**3. 关系实例的具体表示**
- **原文**：*The current values (relation instance) ... are specified by a table.*
- **解释**：模式是**骨架**，实例是**血肉**。在任意一个时间点，`customer` 表中的具体数据行构成的集合，就是 `customer` 关系的**当前实例**。

**4. 概念对照总结**

| 概念         | 符号 / 记法                    | 物理对应     | 变化频率             |
| :----------- | :----------------------------- | :----------- | :------------------- |
| **关系模式** | \( R(A_1:D_1, ..., A_n:D_n) \) | 表头定义     | 极少变化（结构稳定） |
| **关系实例** | \( r(R) \)                     | 表中的数据行 | 频繁变化（增删改）   |

**5. 一个完整的例子串联四页内容**

假设我们有如下定义：

- **关系模式**：`Student_schema = (sid: integer, name: char(20), age: integer)`
- **关系实例** \( student(Student\_schema) \) 在某一时刻的内容如下：

| sid  | name | age  |
| :--- | :--- | :--- |
| 101  | 张三 | 20   |
| 102  | 李四 | 21   |
| 103  | 王五 | 19   |

那么我们可以说：
- 这个关系有 **3 个元组**（第一页：tuple），因此**基数**为 3（第二页：cardinality）。
- 这个关系有 **3 个属性**（第一页：attribute），因此**度**为 3（第二页：degree）。
- 它的结构由**关系模式** `Student_schema` 定义（第三页）。
- 上面的表格就是关系 `student` 在某一时刻的**关系实例**（第四页）。
