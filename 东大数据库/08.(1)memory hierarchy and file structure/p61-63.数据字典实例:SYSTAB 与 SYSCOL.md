好的，这三页PPT是对上一讲**数据字典存储**的具象化展开。如果说上一讲是讲“数据字典存了什么信息”，这三页就是直接展示“这些信息在系统内部具体长什么样”。

我们按PPT顺序逐页拆解，重点分析第一页的**模式定义**和后两页的**实例数据**。

---

### 第一页：数据字典的关系模式定义

这一页给出了数据字典中五张核心系统表的**逻辑结构定义**。用数据库术语说，就是给出了每个表的**关系模式**——即表名和属性列。

#### 1. 关系元数据表 (Relation-metadata)
记录数据库中**每个表（关系）的概要信息**。

| 属性名                 | 含义         | 说明                                       |
| :--------------------- | :----------- | :----------------------------------------- |
| `relation-name`        | 表名         | 表的唯一标识。                             |
| `number-of-attributes` | 列的数量     | 该表有多少个字段。                         |
| `storage-organization` | 存储组织方式 | 例如 `Heap`、`Sequential`、`Hash`。        |
| `location`             | 物理存储位置 | 可能是一个操作系统文件名，或者起始页地址。 |

#### 2. 属性元数据表 (Attribute-metadata)
记录数据库中**每个表的每一列（属性）的详细信息**。

| 属性名           | 含义               | 说明                         |
| :--------------- | :----------------- | :--------------------------- |
| `attribute-name` | 列名               | 例如 `student_id`。          |
| `relation-name`  | 所属表名           | 表明该列属于哪张表。         |
| `domain-type`    | 数据类型           | 例如 `INTEGER`、`CHAR(10)`。 |
| `position`       | 列在表中的顺序位置 | 第1列、第2列等。             |
| `length`         | 数据长度（字节数） | 用于定长记录的计算。         |

#### 3. 用户元数据表 (User-metadata)
记录**数据库用户账户和权限分组**信息。

| 属性名               | 含义                           |
| :------------------- | :----------------------------- |
| `user-name`          | 登录用户名。                   |
| `encrypted-password` | 加密后的密码哈希值。           |
| `group`              | 所属用户组，用于权限批量管理。 |

#### 4. 索引元数据表 (Index-metadata)
记录**每个索引的定义信息**。

| 属性名             | 含义                            |
| :----------------- | :------------------------------ |
| `index-name`       | 索引的唯一名称。                |
| `relation-name`    | 该索引建立在哪个表上。          |
| `index-type`       | 索引类型，如 `B+Tree`、`Hash`。 |
| `index-attributes` | 索引包含的列（搜索键）。        |

#### 5. 视图元数据表 (View-metadata)
记录**视图的定义信息**。视图是虚拟表，本身不存数据，只存查询逻辑。

| 属性名       | 含义                                                         |
| :----------- | :----------------------------------------------------------- |
| `view-name`  | 视图的名称。                                                 |
| `definition` | 视图的创建语句，即 `CREATE VIEW ... AS ...` 中的 `SELECT` 查询文本。 |

> 📌 **小结**：第一页相当于定义了五张系统表的 **CREATE TABLE** 语句。

### 第二页：系统表实例 —— SYSTAB（表元数据表）

这一页展示了第一页中 `Relation-metadata` 表的具体**实例数据**，PPT 中将其命名为 `SYSTAB`。

#### 表结构说明（根据表头翻译）

| 列名         | 全称/含义            | 说明                     |
| :----------- | :------------------- | :----------------------- |
| `RNAME`      | Relation Name        | 表名。                   |
| `NO_OF_ATTR` | Number of Attributes | 列数。                   |
| `ROW_SIZE`   | Row Size             | 每行记录的字节数。       |
| `NO_OF_ROW`  | Number of Rows       | 表中的行数（统计信息）。 |
| `CREATOR`    | Creator              | 创建者。                 |
| `OWNER`      | Owner                | 当前所有者。             |

#### 实例数据逐行解读

| RNAME       | NO_OF_ATTR | ROW_SIZE | NO_OF_ROW | CREATOR | OWNER |
| :---------- | :--------- | :------- | :-------- | :------ | :---- |
| **SYSTAB**  | 6          | 82       | （空）    | SYS     | SYS   |
| **STUDENT** | 4          | 18       | 10        | USER1   | USER1 |

**第 1 行数据解读：SYSTAB 表自身的元数据**
- **RNAME = `SYSTAB`**：说明这张元数据表的名字就叫 `SYSTAB`。
- **NO_OF_ATTR = `6`**：`SYSTAB` 这张表自身有 6 个列（对应表头的 6 列）。
- **ROW_SIZE = `82`**：`SYSTAB` 表的一行记录占用 82 字节。
- **CREATOR/OWNER = `SYS`**：这张系统表由超级用户 `SYS` 创建和维护。

> 💡 这是一个非常经典的**自举**示例：`SYSTAB` 这张表里存了它**自己**的元数据。

**第 2 行数据解读：用户表 STUDENT 的元数据**
- **RNAME = `STUDENT`**：用户创建了一张名为 `STUDENT` 的表。
- **NO_OF_ATTR = `4`**：该表有 4 个字段。
- **ROW_SIZE = `18`**：每行记录定长 18 字节（说明列都是定长类型）。
- **NO_OF_ROW = `10`**：当前表中有 10 行数据（优化器会用到这个值）。
- **CREATOR/OWNER = `USER1`**：由普通用户 `USER1` 创建和拥有。

### 第三页：系统表实例 —— SYSCOL（属性元数据表）

这一页展示了第一页中 `Attribute-metadata` 表的具体**实例数据**，PPT 中将其命名为 `SYSCOL`。它详细描述了每张表的每个字段。

#### 表结构说明（根据表头翻译）

| 列名           | 全称/含义             | 说明                           |
| :------------- | :-------------------- | :----------------------------- |
| `RNAME`        | Relation Name         | 该列属于哪个表。               |
| `ATTR_NAME`    | Attribute Name        | 列名。                         |
| `ATTR_TYPE`    | Attribute Type        | 数据类型（带长度）。           |
| `MEMBER_OF_PK` | Member of Primary Key | 是否为**主键**成员（YES/NO）。 |
| `MEMBER_OF_FK` | Member of Foreign Key | 是否为**外键**成员（YES/NO）。 |
| `FK_RELATION`  | Foreign Key Relation  | 如果是外键，它**引用**哪张表。 |
| `NULL`         | Nullable              | 是否允许为空（YES/NO）。       |

#### 实例数据逐行解读

**第 1 行：描述 `SYSTAB` 表的 `RNAME` 列**
| 字段           | 值       | 解读                                               |
| :------------- | :------- | :------------------------------------------------- |
| `RNAME`        | SYSTAB   | 这一行记录描述的是 `SYSTAB` 表。                   |
| `ATTR_NAME`    | RNAME    | 描述的是 `SYSTAB` 表中的 `RNAME` 这一列。          |
| `ATTR_TYPE`    | CHAR(10) | 定长字符串，最大 10 字符。                         |
| `MEMBER_OF_PK` | YES      | `RNAME` 是 `SYSTAB` 表的**主键**（表名必须唯一）。 |
| `MEMBER_OF_FK` | NO       | 不是外键。                                         |
| `FK_RELATION`  | （空）   | 无引用关系。                                       |
| `NULL`         | NO       | **不允许为空**（主键必然非空）。                   |

**第 2 行：描述 `SC` 表（选课表）的 `SNO` 列（学号）**
| 字段           | 值      | 解读                                                         |
| :------------- | :------ | :----------------------------------------------------------- |
| `RNAME`        | SC      | 描述的是选课表 `SC`。                                        |
| `ATTR_NAME`    | SNO     | 描述的是学号列 `SNO`。                                       |
| `ATTR_TYPE`    | CHAR(6) | 定长 6 字符。                                                |
| `MEMBER_OF_PK` | YES     | `SNO` 是 `SC` 表**主键的一部分**（选课表主键通常是 `SNO + CNO`）。 |
| `MEMBER_OF_FK` | YES     | **同时也是外键**。                                           |
| `FK_RELATION`  | STUDENT | 引用了 `STUDENT` 表。                                        |
| `NULL`         | NO      | 不允许为空。                                                 |

> 💡 这一行完美展示了**外键约束**在数据字典中的存储方式：明确记录了引用目标表 `STUDENT`。

### 三页PPT的整体逻辑总结

| PPT 页码   | 内容层次              | 作用                                                         |
| :--------- | :-------------------- | :----------------------------------------------------------- |
| **第一页** | **逻辑模式设计**      | 定义了数据字典需要哪些表、每个表有哪些字段。相当于画了五张表的 **E-R 图/表结构图**。 |
| **第二页** | **SYSTAB 表实例数据** | 展示了表元数据表的具体填充内容，揭示了**自举**现象。         |
