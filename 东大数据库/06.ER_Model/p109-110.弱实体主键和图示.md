### 第一页 PPT：弱实体集主键的构成公式

虽然这张图片的文字有些截断，但结合上下文，它的核心知识点非常明确，即阐述**弱实体的主键（Primary Key）到底是什么**。

**PPT上的核心逻辑：**

- 弱实体自身没有足够区分度的属性，所以它不能只靠自己生成主键。
- **核心公式：弱实体集的主键 = 强实体集的主键 + 弱实体集的分辨符（Discriminator）。**

**详细讲解：**

- 既然弱实体是依附于强实体（标识实体）存在的，我们要想在数据库海量的数据中精准找到某一个具体的弱实体实例，就必须**先找到它的“靠山”（强实体），再根据“特征”（分辨符）在靠山名下找人**。
- 这个组合起来的键，在数据库理论中被称为**复合主键（Composite Key）**。

### 第二页 PPT：ER图表示法与经典案例解析

这张PPT极其重要，它给出了绘制ER图的标准规范，并用一个经典的银行业务场景做了解释。

**PPT上的知识点（ER图规范）：**

1. **双边框矩形（Double rectangles）**：用来表示弱实体集（图中的 `payment`）。
2. **虚下划线（Dashed line）**：用来标注弱实体集的分辨符（图中的 `payment_number` 下方有一条虚线）。

**案例详细解析：贷款（loan）与 还款记录（payment）**

- **强实体 `loan`（贷款）**：它有自己的主键 `loan_number`（贷款编号，实下划线）。只要知道贷款编号，就能在银行系统里查到这笔贷款的金额（amount）。
- **弱实体 `payment`（还款记录）**：它包含属性：还款日期（payment_date）、还款期数/编号（payment_number，如第1期、第2期）、还款金额（payment_amount）。
- **为什么 `payment` 是弱实体？** 假设你只拿到一张单子，上面写着“第3期还款（payment_number = 3）”，你根本不知道这是张三的房贷第3期，还是李四的车贷第3期。“第3期”这个信息**只有依附于某一笔具体的贷款时才有意义**。
- **标识联系 `loan_payment`**：图中的**双边框菱形**代表标识联系，说明 `payment` 是通过这个联系依附于 `loan` 的。注意 `payment` 到联系之间的**双实线**，这代表前面提到的“完全参与”——任何一笔还款记录，必定属于某一笔贷款。
- **最终结论**：`payment` 实体集的主键是 `(loan_number, payment_number)`。

### 💡 补充相关知识点（实战与进阶）

为了让你更全面地掌握这个概念，补充以下几个在考试和实际开发中经常遇到的知识点：

#### 1. 如何将这个ER图转化为 SQL 建表语句？

这是数据库考试中最爱考的题型。在实际建表时，弱实体的表必须把强实体的主键作为外键拉进来，并和自己的分辨符一起组成主键。

SQL

```mysql
-- 1. 先创建强实体表 (Loan)
CREATE TABLE Loan (
    loan_number VARCHAR(20) PRIMARY KEY,
    amount DECIMAL(10,2)
);

-- 2. 再创建弱实体表 (Payment)
CREATE TABLE Payment (
    loan_number VARCHAR(20),       -- 引入强实体主键作为外键
    payment_number INT,            -- 弱实体自己的分辨符
    payment_date DATE,
    payment_amount DECIMAL(10,2),
    
    -- 核心：复合主键
    PRIMARY KEY (loan_number, payment_number),
    
    -- 核心：外键约束（通常带有级联删除）
    FOREIGN KEY (loan_number) REFERENCES Loan(loan_number) ON DELETE CASCADE
);
```

*注意代码中的 `ON DELETE CASCADE`（级联删除）：因为弱实体有“存在依赖”，如果银行把某一笔贷款（loan）的记录删除了，那么属于这笔贷款的所有还款记录（payment）也应该被数据库自动清理掉。*

#### 2. 弱实体可以有多层嵌套吗？

**可以的。** 一个弱实体集，也可以成为另一个更底层弱实体集的“标识实体集”（强实体）。

- **举个例子**：大学里的实体关系。
  - **大学 (University)**：强实体（主键：大学名称）。
  - **学院 (College)**：相对大学是弱实体。比如“计算机学院”，分辨符是“学院名称”。主键是 `(大学名称, 学院名称)`。
  - **系 (Department)**：相对学院也是弱实体。比如“软件工程系”，分辨符是“系名称”。它的最终主键要带上上面所有老大的信息，变成复合主键：`(大学名称, 学院名称, 系名称)`。
