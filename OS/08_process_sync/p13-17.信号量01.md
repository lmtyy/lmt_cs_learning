### 图1：SEMAPHORES (信号量) —— 定义与基本操作

这张 PPT 定义了信号量是什么。

1.  **本质**：信号量是一个**整数变量**，可以初始化为非负值。
2.  **三种原子操作**：
    *   **初始化**：设定一个初始值。
    *   **`wait(S)` (P 操作)**：减 1。如果减完之后小于 0，进程就**阻塞 (Block)**。
    *   **`signal(S)` (V 操作)**：加 1。如果加完之后小于等于 0，就**唤醒 (Unblock)** 一个被阻塞的进程。

**通俗理解**：如果把信号量看作停车场的“空车位计数器”：
*   `S` = 5 (有5个空位)。
*   `wait(S)` = 进来一辆车，空位减1。如果变成 -1，说明车位满了，新来的车得排队等待。
*   `signal(S)` = 离开一辆车，空位加1。如果有人排队，就让排在第一辆的车进来。

### 图2：Semaphores: Critical Section (用信号量解决临界区问题)

这张 PPT 展示了最基础的应用——**互斥锁 (Mutex)**。

```c
semaphore mutex; // 定义一个信号量
// mutex = 1;    // 初始化为 1 (表示锁是开的)

do {
    wait(mutex);    // 进门前：申请锁 (P操作)
        critical section // 临界区
    signal(mutex);  // 出门后：释放锁 (V操作)
    remainder section
} while (1);
```

**运作原理**：
*   **初值 1**：表示一个人都没有在临界区。
*   **`wait(mutex)`**：进来一个人，变成 0。再有人进来，变成 -1、-2…… 这些人全被**阻塞**在门外排队。
*   **`signal(mutex)`**：里面的人出来，变成 0 或正数，唤醒下一个排队的人。
*   **结果**：每次最多只有 1 个进程在临界区里执行。**完美实现了“互斥 (Mutual Exclusion)”**。

### 图3 & 图4：Implementation (信号量的内部实现)

这两张 PPT 告诉你，信号量在操作系统内核里是怎么写出来的。

**图3：定义了数据结构**
```c
typedef struct {
    int value;          // 计数器
    struct process *L;  // 等待队列 (存放阻塞的进程)
} semaphore;
```

**图4：定义了 `wait` 和 `signal` 的具体操作（伪代码）**

```c
// P操作 (申请)
wait(S) {
    S.value--;
    if (S.value < 0) {
        add this process to S.L; // 加到等待队列
        block();                 // 阻塞进程，让出CPU
    }
}

// V操作 (释放)
signal(S) {
    S.value++;
    if (S.value <= 0) {
        remove a process P from S.L; // 从队列里拿一个
        wakeup(P);                   // 唤醒它
    }
}
```

**关键点**：注意 `wait` 和 `signal` 这两个操作本身必须是**原子的 (Atomic)**。操作系统内核确保执行这两段代码时**不会被中断**。如果它们不是原子的，信号量本身也会出现竞态条件。

### 图5：As a General Synchronization Tool (作为通用的同步工具)

这张 PPT 展示了信号量的**第二种用法**（图2是第一种：互斥）。

**场景**：控制**执行顺序**。比如，必须等进程 A 干完某件事，进程 B 才能开始。

**代码结构**：
```c
semaphore flag = 0; // 初始化为 0

// 进程 Pi (先做)
A;
signal(flag);   // V操作：通知 B "我做完了"

// 进程 Pj (后做)
wait(flag);     // P操作：等 A 的信号
B;
```

**运作原理**：
*   `flag` 初始为 0。
*   `Pj` 执行到 `wait(flag)` 时，`0-1 = -1`，小于 0，所以**阻塞**。它被挂起，等待唤醒。
*   等 `Pi` 执行完 `A`，执行 `signal(flag)`，`-1+1 = 0`，小于等于 0，于是唤醒 `Pj`。
*   `Pj` 恢复运行，执行 `B`。

**结果**：保证了 **A 永远在 B 之前执行**。这叫**前趋关系 (Precedence constraints)**。

---

### 💡 总结：分清两种用法 (考试必辩)

很多同学考不好，是因为分不清这两个信号量的初值和含义：

1.  **互斥 (Mutex)**：
    *   **初值** = 1
    *   用于：保护临界区，防止同时进入。 (比如 `counter++`)
    *   口诀：**锁**

2.  **同步 (Synchronization)**：
    *   **初值** = 0
    *   用于：控制执行顺序。A 做完，B 才能做。 (比如先存款，后取款)
    *   口诀：**信号**
