这几张 PPT 是操作系统课程中 **“经典同步问题”** 的入门篇，正式开始讲解 **有界缓冲区问题 (Bounded-Buffer Problem)**，即 **生产者-消费者问题** 的正式解决方案。

这是本章**考试的核心**，也是面试/考研常考的高频题。无论你是为了应付考试还是理解并发编程，这部分都必须掌握。

### 图 1：CLASSICAL PROBLEMS (经典问题总览)

这张图列出了本章要解决的三个“经典案例”。后续的课程会逐一展开：
1.  **Bounded-buffer problem (有限缓存问题)**：也就是现在的 PPT 重点。
2.  **Readers-writers problem (读者-写者问题)**：控制读写权限。
3.  **Dining-philosophers problem (哲学家就餐问题)**：控制资源分配，防止死锁。

**其中第 1 和 第 3 最常考。**

### 图 2：Shared Data (共享数据定义)

首先要定义使用的信号量。这是解决这个问题的**最关键的初始化配置**。

```c
semaphore mutex = 1;  // 互斥锁：保护缓冲区，防止同时操作
semaphore full = 0;   // 同步信号量：缓冲区里“已满”的数据个数 (初始为0)
semaphore empty = n;  // 同步信号量：缓冲区里“空闲”的槽位数 (n为大小)
```

**为什么这么初始化？**
*   **`mutex = 1`**：保证在任何时候，**最多只有一个进程**可以操作缓冲区的数组（执行 `in` 或 `out` 指针修改）。
*   **`full = 0`**：消费者必须先看有没有数据。0 表示没有数据，消费者会**阻塞等待**。
*   **`empty = n`**：生产者先看有没有空位。n 表示有 n 个空位可以放东西，生产者**不会阻塞**。

### 图 3：Producer Process (生产者进程代码)

这是生产者的标准写法。请**务必注意顺序**：

```c
while (1) {
    // 1. 生产数据
    produce an item in nextp;

    // 2. 申请一个“空位” (P操作) —— 如果满了，这里就会阻塞
    wait(empty);

    // 3. 申请互斥锁 (P操作) —— 确保这一小段操作是原子的
    wait(mutex);
        // 临界区：把数据加入缓冲区
        ...; add nextp to buffer; ...
    // 4. 释放互斥锁 (V操作)
    signal(mutex);

    // 5. 通知消费者“多了一个数据” (V操作)
    signal(full);
}
```

**注意 `empty` 和 `mutex` 的顺序**：
*   一定要 **`wait(empty)` 在前，`wait(mutex)` 在后**！
*   如果把 `wait(mutex)` 放在前面，缓冲区满了，生产者会拿着 `mutex` 锁死等 `empty`。此时消费者想拿数据释放空间，却因拿不到 `mutex` 锁进不去。**导致死锁！**

---

### 图 4：Consumer Process (消费者进程代码)

消费者的代码与生产者对称：

```c
while (1) {
    // 1. 申请一个“数据” (P操作) —— 如果空了，这里就会阻塞
    wait(full);

    // 2. 申请互斥锁 (P操作) —— 防止同时读写指针
    wait(mutex);
        // 临界区：从缓冲区取出数据
        ...; remove one buffer to nextc; ...
    // 3. 释放互斥锁 (V操作)
    signal(mutex);

    // 4. 通知生产者“多了一个空位” (V操作)
    signal(empty);

    // 5. 消费数据
    ...; consume the item in nextc; ...
}
```

### 💡 核心总结与考试答题模板

这一题在考试中，通常要求学生**写伪代码**。请按照以下**模板**来写，确保不扣分：

#### 1. 定义信号量 (初始化)
```c
semaphore mutex = 1;
semaphore full = 0;
semaphore empty = BUFFER_SIZE; // 用 N 或具体的数字
```

#### 2. 生产者过程
```c
while (1) {
    produce an item;
    wait(empty);       // 检查空位
    wait(mutex);       // 上锁
        // 把 item 放入 buffer[in];
        in = (in + 1) % BUFFER_SIZE;
    signal(mutex);     // 解锁
    signal(full);      // 通知有数据
}
```

#### 3. 消费者过程
```c
while (1) {
    wait(full);        // 检查数据
    wait(mutex);       // 上锁
        // 从 buffer[out] 取 item;
        out = (out + 1) % BUFFER_SIZE;
    signal(mutex);     // 解锁
    signal(empty);     // 通知有空位
    consume the item;
}
```

---

### ⚠️ 易错点 (考试常坑)

1.  **`wait(empty)` 和 `wait(full)` 必须放在 `wait(mutex)` 的外面**。
    *   如果放在里面（锁内部），会导致**死锁**。
2.  **生产者是 `signal(full)`，消费者是 `signal(empty)`**。不要搞混了。
3.  **`signal(mutex)` 必须释放**。如果在临界区里写了 `return` 或者抛异常导致无法释放，整个系统会崩。
4.  **不要用 `while(counter == n)` 这种忙等待**。要用信号量的 `wait` 阻塞。这是信号量存在的意义。
