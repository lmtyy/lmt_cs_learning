这组幻灯片是上一组的延续，重点讲解了**内存数据结构（In-memory structures）**以及**文件操作的实现流程**。

这部分内容非常关键，因为它解释了当你编写代码调用 `open()`, `create()`, `close()` 等函数时，操作系统内部到底发生了什么。**尤其是关于“进程打开文件表”和“系统级打开文件表”的区分，是操作系统课程中的高频考点。**

下面我为你详细梳理这5张PPT的核心逻辑：

---

### 1. 内存数据结构 (In-memory structures) —— 图1
这部分回答了内存里到底存了什么来管理文件系统。既然磁盘上的数据（FCB/Inode）读起来很慢，操作系统就在内存里维护了一份用于**加速**和**管理**的副本。

PPT中列出的四个结构，**重点在于后两个（关于打开文件的表格）**：

1.  **In-memory partition table（内存分区表）**：存有哪些分区（比如 C盘、D盘）被挂载了。
2.  **In-memory directory structures（内存目录结构）**：缓存最近访问过的目录信息，加速查找。
3.  **System-wide open-file table（系统级打开文件表）**：
    *   **层级**：全局唯一。
    *   **内容**：包含每个**正在被使用**的文件的 FCB（文件控制块）副本。
    *   **意义**：保证同一个文件即使被多个进程打开，物理 FCB 也只读入内存一次。
4.  **Per-process open-file table（进程级打开文件表）**：
    *   **层级**：每个进程一个（进程私有的）。
    *   **内容**：包含一个指向“系统级打开文件表”的**指针**，以及一些进程私有的信息（比如文件读写的当前偏移量）。
    *   **意义**：这就是为什么两个进程同时打开同一个文件，读取互不影响的原因。

---

### 2. 创建文件的过程 (To create a new file) —— 图2
当你执行 `touch newfile.txt` 或调用 `creat()` 时：

1.  **请求**：应用程序调用逻辑文件系统（Logical File System）。
2.  **分配 FCB**：在磁盘上分配一个新的 FCB 空间。
3.  **读取目录**：找到该文件所在的目录（PPT特别提到：UNIX把目录当作文件处理；Windows NT 把目录当作记录处理）。
4.  **添加条目**：在目录中添加一条新记录，将“文件名”和“新的FCB”关联起来。
5.  **写回磁盘**：把更新后的目录和新的FCB写回磁盘（此时文件大小为0）。

---

### 3. FCB (文件控制块) 的内容详情 —— 图3
这张图直观地展示了 FCB（在 Linux 中称为 `inode`）里到底存了什么属性。你可以把它看作文件的“身份证信息”：

*   **File permissions**：权限位（如 rwx）。
*   **File dates**：创建时间、最后访问时间、最后修改时间。
*   **File owner, group, ACL**：所有者ID、所属组ID、访问控制列表。
*   **File size**：文件大小（字节数）。
*   **File data blocks**：指向文件实际数据块的指针（这是地址映射的关键）。

---

### 4. 打开文件的过程 (To open a file) —— 图4 **(重点！)**
当你调用 `fopen()` 或系统调用 `open()` 时，操作系统经历以下步骤：

1.  **传递文件名**：逻辑文件系统接收文件名。
2.  **搜索目录**：在目录结构中寻找该文件名。
3.  **读取 FCB**：找到后，把磁盘上的 FCB 读入内存。
4.  **存入系统表**：把该 FCB 的副本放入 **System-wide open-file table**。
5.  **存入进程表**：在 **Per-process open-file table** 中添加一个新条目，该条目包含一个指向系统级表的指针。
6.  **返回句柄**：返回 **文件描述符 (file descriptor)** 或 **文件句柄 (file handle)** 给应用程序。从此，应用程序只需使用这个数字（如 fd=3）来操作文件，无需再重复查找目录。

**核心机制**：两个进程打开同一个文件，它们各自的“进程表”指向同一个“系统表”条目。但如果它们读写，通过私有指针互不影响。

---

### 5. 关闭文件的过程 (To close a file) —— 图5
当你调用 `fclose()` 或 `close()` 时：

1.  **移除条目**：从 **Per-process open-file table** 中移除该文件的条目。
2.  **减少计数**：减少 **System-wide open-file table** 中该条目的 **open count**（打开计数）。
3.  **写回磁盘（重要！）**：
    *   如果 `open count` 变为 **0**（意味着没有任何进程在使用这个文件了），操作系统会：
        *   将内存中更新过的文件信息（如文件大小、修改时间、最后访问时间）**写回磁盘的目录结构**。
        *   删除内存中的系统级条目。
    *   **这解释了为什么“关闭文件”很重要**：如果不关闭，最新的修改可能没写回磁盘，导致数据丢失。

---

### 💡 考试/面试重点提示
1.  **区分两种表**：一定要能说清楚 **Per-process**（进程私有，存指针/偏移量）和 **System-wide**（系统全局，存FCB数据）的区别。
2.  **理解文件描述符**：PPT 最后提到的 "file descriptor" 其实就是一个整数索引（如 0, 1, 2 代表 stdin, stdout, stderr），它指向 **进程表** 中的条目。
3.  **open() vs create()**：`create()` 涉及分配新的 FCB 并写回磁盘；`open()` 涉及将磁盘 FCB 读入内存缓存。
