### 1. 基础概念：为什么需要文件系统？（图1）
首先，课件定义了为什么要用文件系统。
*   **物理特性**：磁盘是“二级存储”的主力。它有一个特点：**数据是按“块（Block）”读写，而不是按“字节”读写的**。并且磁盘支持**顺序存取**和**直接存取**。
*   **文件系统的作用**：因为磁盘是物理设备，管理起来很复杂。文件系统就像一个**翻译官兼管家**：
    *   **对用户**：提供一个友好的逻辑视图（看起来像文件夹和文件）。
    *   **对磁盘**：把逻辑文件映射到物理磁盘块上。

### 2. 核心架构：分层文件系统（图2、图3）
操作系统设计的一个核心思想是**“分层”**。每一层只负责一个小任务，**下层为上层服务，上层调用下层的功能**。

图3展示了经典的结构图：
`Application Programs` (应用程序) -> `Logical File System` -> `File-Organization Module` -> `Basic File System` -> `I/O Control` -> `Devices` (硬件设备)

### 3. 每一层的具体职责（图4、图5）
这是考试可能考查的重点，我们从上往下拆解：

#### 1) I/O Control (最底层 - 图4)
*   **包含**：设备驱动（Device Drivers）和中断处理程序。
*   **任务**：直接和硬件对话。它负责把内存里的数据搬到磁盘上，或者把磁盘上的数据读到内存里。它是**最靠近物理层**的。

#### 2) Basic File System (基本文件系统 - 图4)
*   **任务**：向对应的设备驱动发送**通用命令**。
*   **关键点**：它只关注“读写物理磁盘块”，把物理块地址发下去。它不知道什么是“文件名”或“文件夹”。

#### 3) File-Organization Module (文件组织模块 - 图4)
*   **任务**：做**转换**和**映射**。
*   **具体工作**：
    *   它知道文件有哪些**逻辑块**，以及这些逻辑块对应磁盘上的哪些**物理块**。
    *   它包含**空闲空间管理器**，知道磁盘上哪些块是空的，需要存新文件时就把空块分配出去。

#### 4) Logical File System (逻辑文件系统 - 图5)
*   **任务**：管理**元数据（Metadata）**。**元数据**是指“关于数据的数据”，即除了文件内容之外的所有信息。
*   **具体工作**：
    *   **管理目录结构**：把文件名（如 `A.txt`）转换为文件组织模块需要的内部标识。
    *   **管理FCB（文件控制块）**：FCB里存的是**文件属性**，比如：所有者是谁？权限是什么？文件内容存放在哪几个物理块上？
    *   **负责安全与保护**。

### 4. 现实中的文件系统例子（图6）
课件最后列举了不同操作系统的具体实现案例：

*   **Windows阵营**：
    *   **FAT** (12/16/32)：早期文件系统，简单但功能弱（如没有权限管理）。
    *   **NTFS**：现代Windows主力文件系统，功能强大，支持大文件、权限、压缩等。
*   **UNIX/Linux阵营**：
    *   **UFS** -> **ext2** -> **ext3**：展示了发展路径（ext3增加了日志功能，防断电损坏数据）。
    *   课件也给出了在Linux下查看文件系统的命令 `more /proc/filesystems`，让你知道系统支持哪些文件系统。

### 💡 学习小窍门
如果你在备考，建议把**图3的箭头层级**和**图4、图5中每一层的职责**对应着背下来。

考试中常出现的一种题型是：**“如果用户想读取 /home/user/a.txt 这个文件，请描述文件系统各层级是如何协作的？”**
答案按以下顺序：
1.  **Logical FS**：查找目录，找到 `a.txt` 的FCB（获取物理块地址）。
2.  **File-Organization Module**：将逻辑块地址转为物理块地址。
3.  **Basic FS**：基于物理地址，构造读写命令发给驱动。
4.  **I/O Control**：驱动控制磁盘硬件，把数据搬到内存。
