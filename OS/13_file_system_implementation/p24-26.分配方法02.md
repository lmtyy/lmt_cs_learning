这组幻灯片讲解了分配方法中的第二种：**链接分配 (Linked Allocation)**。

这部分内容与上一讲的“连续分配”形成了鲜明对比。连续分配讲究“连续存放”，而链接分配讲究“分散存放，通过指针串联”。理解这一章，就掌握了文件系统如何在不连续的空间里灵活管理文件。

下面我为你详细拆解这3张PPT的核心逻辑：

---

### 1. 链接分配的基本原理 (图1)

#### **数据结构**
*   **文件在磁盘上**：一个文件被拆分成若干个磁盘块，这些块可以**分散在磁盘的任何地方**。每个磁盘块中不仅包含数据，还包含一个**指向下一个块的指针**。
*   **目录项**：目录中只需要记录**起始块号 (Start)** 和 **结束块号 (End)**。
*   **磁盘块结构**（图1下方）：
    *   `Block = Pointer + Data`。即：一部分空间存指针，另一部分存实际数据。

---

### 2. 实例演示 (图2) —— **重点理解**

这张图直观演示了文件 `jeep` 是如何存储在磁盘上的。

*   **目录 (右图)**：
    *   文件 `jeep` 的目录项记录：**start = 9**, **end = 25**。
*   **磁盘块 (左图)**：
    *   块 **9** 是起始块，它里面有指针指向下一个块 **16**。
    *   块 **16** 里面有指针指向下一个块 **1**。
    *   块 **1** 里面有指针指向下一个块 **10**。
    *   块 **10** 里面有指针指向最后一个块 **25**。
    *   块 **25** 里存的是结束标记（通常记为 `-1` 或 `NULL`）。

> **关键**：文件在磁盘上是**不连续**的。要读取第100个块的数据，必须从块9开始，顺着指针一个个往下数，直到找到第100块。这就是为什么它**不能直接访问（随机访问）**。

---

### 3. 链接分配的优缺点 (图3) —— **考试重点**

这张PPT总结了链接分配的核心利弊，请务必记下：

#### **✅ 优点**
1.  **实现简单**：目录只需存起始地址。
2.  **无空间浪费 (No waste of space)**：
    *   **没有外部碎片**：因为文件可以存在任何空闲块里。
    *   **文件易于扩展**：无论文件多大，只需找到一个空闲块，把它链接到文件末尾即可，无需移动已有数据。

#### **❌ 缺点**
1.  **不能直接访问 (No random access)**：
    *   如果你想读文件结尾的内容，必须从头开始顺着指针一个个读（即**顺序访问**）。这在处理大文件的随机读写时效率极低。
2.  **指针浪费空间 (Pointers waste space)**：
    *   每个磁盘块都要拿出一部分空间来存指针。如果块很小（如512字节），指针占用的比例会很大。
    *   *改进方案*：使用 **簇 (Clusters)**。把多个物理块合并成一个逻辑簇，减少指针开销。
3.  **可靠性差 (Poor reliability)**：
    *   如果某个中间的指针损坏了（比如块16的指针坏了），从块16之后的所有数据都会**丢失**。因为系统不知道去哪里找下一块。
    *   *改进方案*：使用**双向链表 (Doubly linked list)**（存前后两个指针），虽然浪费更多空间，但增加了容错性。

---

### 💡 核心总结与对比 (对比连续分配)

| 特性 | 连续分配 (Contiguous) | 链接分配 (Linked) |
| :--- | :--- | :--- |
| **空间占用** | 需连续空间，有外部碎片 | 可分散存储，无外部碎片 |
| **访问方式** | 支持随机访问 (Direct/Random) | 仅支持顺序访问 (Sequential) |
| **文件扩展** | 难，可能需移动整个文件 | 易，只需加一个新的块 |
| **可靠性** | 较高（单点损坏不影响整体） | 较低（指针损坏导致断链） |
| **目录结构** | 存 (Start, Length) | 存 (Start, End) |

### 🚩 备考小贴士
在考研或期末考试中，**链接分配**往往与 **FAT (File Allocation Table)** 文件系统联系在一起。

*   **FAT 的改进**：为了克服“指针占用磁盘块空间”和“可靠性差”的问题，FAT 把指针**单独提取出来存到一个表中**（即 FAT 表），而不是混在数据块里。这保留了链接分配灵活性，同时提升了可靠性和读取效率。
