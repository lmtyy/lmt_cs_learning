课件这里非常清晰地划分了文件系统在存储上的两大“阵地”：**磁盘（On-disk）** 和 **内存（In-memory）**。并且详细拆解了磁盘上存放的各种关键数据结构。这对理解操作系统如何在磁盘上“布局”至关重要。

下面我为你详细梳理这几张PPT的核心内容：

---

### 1. 核心分类：On-disk Structures vs. In-memory Structures
文件系统实现的核心在于它维护两组不同的数据结构：
*   **On-disk structures（磁盘数据结构）**：
    *   **永久存储**：数据保存在硬盘上，电脑关机后还在。
    *   主要用于**信息存放**，比如文件内容、目录结构、空闲块信息。
*   **In-memory structures（内存数据结构）**：
    *   **临时存储**：在内存里，电脑关机会消失。
    *   主要用于**性能优化**和**管理**，例如缓存经常访问的数据、追踪当前打开的文件信息。（这部分在后面PPT会详述）

---

### 2. 磁盘数据结构 (On-disk Structures) 详解（图2、图3）
磁盘上存放着文件系统的元数据（Metadata），这部分数据决定了文件系统能否正常运作。按照课件，主要包含这几部分：

#### A. 引导控制块 (Boot Control Block) —— 图3
*   **用途**：用于从该分区启动操作系统。
*   **位置**：通常是分区的**第一个块**（即“引导块”或“分区引导扇区”）。
*   **特性**：如果这个分区没有存操作系统（只是存数据），这个块可以为空。

#### B. 分区控制块 (Partition Control Block) —— 图3
*   **别名**：**超级块 (Super Block)** 或 **主文件表 (Master File Table, MFT)**。
*   **用途**：包含该**分区的所有全局细节**。就像这个分区的“身份证”。
*   **关键信息**：
    *   块总数、块大小。
    *   空闲块的数量和位置指针。
    *   空闲 FCB（文件控制块）的数量和位置。

---

### 3. 目录结构与 FCB (File Control Block) —— 图4
这部分结合了之前的内容，说明了具体单个文件的信息怎么存。

#### A. 目录结构 (Directory Structure)
目录本质上是一个“特殊文件”，它记录着该文件夹下有哪些文件。
*   **实现方法**：
    *   **线性列表**：简单的链表，查找慢。
    *   **哈希表**：查找快，但可能有冲突。

#### B. 文件控制块 (FCB)
这是**单个文件**的身份信息集合。
*   **包含内容**：文件权限、所有者、文件大小、数据块的**位置指针**等。
*   **不同系统的称呼**：
    *   **UNIX/Linux (UFS)**：叫 **Inode**（索引节点）。
    *   **Windows (NTFS)**：这些信息存在 **Master File Table (MFT)** 中，MFT采用**关系数据库**的结构，每个文件占一行记录。

---

### 💡 图片中的重点对比（考试经常考）
这张PPT里最值得关注的一个对比点是：
*   **UFS (Linux)** 使用 **inode** 来存放文件属性。
*   **NTFS (Windows)** 使用 **Master File Table (MFT)**（实际上就是把所有文件的FCB集中存放到一个大型关系表里）。

**如何区分“目录项”和“FCB”？**
*   **目录项**：通常只包含“文件名” + “指向FCB的指针”。
*   **FCB/Inode**：包含真正的文件属性（权限、大小、物理地址）。
*   这样设计的好处是：**查找目录时，不需要把每个文件的详细属性都读进内存，只需要读文件名列表就行，速度更快。**
