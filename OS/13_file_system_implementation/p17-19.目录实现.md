这组幻灯片详细讲解了**目录实现（Directory Implementation）**的两种核心方法：**线性列表（Linear List）**和**哈希表（Hash Table）**。

这部分内容对应目录章节的具体实现细节。在上一组 PPT 中我们知道了目录里存的是“文件名+指针”，现在我们要学习操作系统是**如何组织这些目录项以实现高效的查找、创建和删除**的。

下面我为你详细拆解这两种方法的原理、优缺点和具体操作流程。

---

### 1. 线性列表 (Linear List) —— 图1 & 图2

这是最简单、最直观的实现方式。

#### **数据结构**
目录本质上就是一个**数组或链表**，其中每一项包含：
*   **文件名**
*   **指向数据块的指针**（实际上是指向 FCB 或 inode 的指针）

#### **具体操作逻辑**

| 操作 | 具体步骤 | 复杂度 |
| :--- | :--- | :--- |
| **查找 (Find a file)** | 从头到尾遍历整个列表，逐个比较文件名。 | **O(n)** (线性搜索) |
| **创建 (Create a file)** | 1. 遍历列表，确保不存在同名文件。<br>2. 在列表末尾追加一个新条目。 | 需要搜索一次 (O(n)) |
| **删除 (Delete a file)** | 1. 搜索列表找到该文件。<br>2. 移除该条目。<br>3. 释放该文件占用的磁盘空间。 | 需要搜索一次 (O(n)) |
| **复用条目 (Reuse an entry)** (图2) | 将删除条目的位置标记为 **“未使用”**，并将该位置挂载到一个“空闲目录项链表”中。下次创建文件时，优先使用这些已标记的空位，以节省空间。 | - |

#### **讨论与优缺点 (图2)**
*   **优点**：编程实现非常简单。
*   **缺点**：**执行耗时 (time-consuming)**。查找一个文件需要遍历整个列表（例如在一个包含数千个文件的文件夹中查找 `readme.txt` 效率极低）。

---

### 2. 哈希表 (Hash Table) —— 图3

为了解决线性列表查找慢的问题，引入了**哈希表**。它本质上是在线性列表之上增加了一个 **“索引层”**。

#### **基本原理**
1.  **存储**：实际目录项依然存储在**线性列表**中（或类似结构）。
2.  **索引**：使用一个**哈希表**（数组），通过哈希函数将**文件名映射为一个数字（索引值）**。
3.  **查找**：通过哈希值直接定位到目录项的位置，无需遍历整个列表。

#### **处理哈希冲突**
PPT 中提到了两种处理冲突的方式：

**方法一：不允许冲突 (No collisions allowed)**
*   **做法**：每个哈希槽只能存一个值。如果发生冲突（两个不同文件名哈希到同一个位置），则改变哈希函数。
*   **特点**：速度**最快**，但哈希函数需要动态调整，工程实现较麻烦。

**方法二：允许冲突 (Collisions allowed) —— 常用方法（拉链法）**
*   **做法**：每个哈希槽指向一个**链表**。
    *   先通过哈希函数找到对应的槽位。
    *   然后在该槽位的**链表**中线性搜索目标文件名。
*   **特点**：速度**较快**（比纯线性列表快得多，特别是冲突很少时）。

#### **优缺点**
*   **优点**：
    *   **查找速度极快**：平均情况下接近 O(1)。
    *   适合需要频繁查找和操作的大目录。
*   **缺点**：
    *   需要处理哈希冲突（增加了复杂度）。
    *   如果目录是固定的或文件很少，线性列表可能反而更简单。

---

### 💡 学习总结与对比表格

| 特性 | 线性列表 (Linear List) | 哈希表 (Hash Table) |
| :--- | :--- | :--- |
| **实现难度** | 简单 | 较复杂 |
| **查找速度** | 慢 (O(n)) | 快 (平均 O(1)) |
| **删除操作** | 需要遍历标记/释放 | 需要处理哈希冲突 |
| **内存开销** | 小 | 较大（需要额外的哈希表数组） |
| **适用场景** | 小型目录、简单文件系统 (FAT) | 大型目录、现代文件系统 (NTFS, ext4) |

### 🚩 考试/面试点
考试中常会问：**“在什么情况下，线性列表可能比哈希表更有优势？”**
*   **答案**：当目录中的文件**数量很少**（比如只有几个文件）时，线性列表的O(n)实际上很快，而且它不需要维护哈希表和解决冲突的代码，实现简单，内存占用更小。对于极小的目录，线性列表的效率可能高过哈希表。
