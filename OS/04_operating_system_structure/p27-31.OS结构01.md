这一组PPT开始讲解 **“OS Structures”**（操作系统结构）——这是第一章大纲里最后的重点。现在，我们将从**功能分类**（组件、服务、API）转向**代码如何组织**。

这组PPT重点介绍了结构中的第一种：**Monolithic Structure（单内核/宏内核结构）**。

---

### 第一张图：OS Structures 总览

这张图列出了课程将要讲解的**四种经典的OS结构模型**（括号里是PPT中的中文翻译）：

1.  **Monolithic (单层)**：大家熟悉的 **Linux, UNIX, MS-DOS**。
2.  **Layered (分层)**：将OS分成一圈一圈的同心圆或一层一层的方块。
3.  **Microkernel (微内核)**：内核很小，只负责最基础的通信和调度（如 **Mach, QNX**）。
4.  **Virtual machine (虚拟机)**：在一个物理机器上模拟出多个虚拟硬件环境。

目前这几张PPT都在讲解第一类：**Monolithic Structure**。

---

### 第二、三张图：Monolithic Structure - MS-DOS

**MS-DOS 是非常典型的“简单单内核”代表。**

*   **设计目标**：在**最小空间**（当时内存很小）内提供**最多功能**。
*   **结构特点**：
    *   **不分模块**：所有功能都堆在一起，没有清晰的模块划分。
    *   **接口不清**：不同层次的接口没有很好地分离。
*   **图解（图3）**：
    *   可以看到 **Application Program** 可以跳过中间层**直接**调用 **ROM BIOS device drivers**（底层硬件驱动）。
    *   这种“直通”方式效率很高，但**极度不安全**。一个错误的应用程序如果直接操纵BIOS，可能会让整个系统崩溃。

---

### 第四、五张图：Monolithic Structure - 原始 UNIX

**原始 UNIX 是比 MS-DOS 更“结构化”的单内核。**

*   **结构划分**：它将整个系统分为**两部分**：
    1.  **System Programs**（用户空间）：Shell、编译器、各种命令。
    2.  **Kernel**（内核空间）：所有底层功能。
*   **内核的职责**：
    *   文件系统、CPU调度、内存管理、I/O系统……所有这些都放在**同一个大内核**里。
    *   PPT总结：**“A large number of functions for one level”**（在一个层级里集成了大量的功能）。

**图5（核心图解 - 需要仔细看！）**：

这张图是**经典单内核（Monolithic Kernel）的解剖图**。从上到下分为三个大层次：

1.  **用户空间（Top）**：
    *   **Users**：用户。
    *   **Shells and Commands / Compilers / System Libraries**：这是系统程序和应用层。
2.  **中间界限**：**System-call interface to the kernel**（系统调用接口）。
    *   这是用户程序进入内核的**唯一大门**。
3.  **内核空间（Middle - 核心！）**：
    *   这是**大内核**，所有服务都在这里：
    *   **左侧**：信号处理、终端处理、字符I/O、终端驱动。
    *   **中间**：文件系统、块I/O系统、磁盘和磁带驱动。
    *   **右侧**：CPU调度、页面置换、虚拟内存。
    *   **特点**：所有上述组件都运行在**同一个地址空间**、同一个特权级（内核态）里。
4.  **底层界限**：**Kernel interface to the hardware**。
5.  **硬件层（Bottom）**：终端控制器、设备控制器、内存控制器等物理硬件。

---

### 💡 单内核（Monolithic Kernel）的核心特点总结

| 特点 | 说明 |
| :--- | :--- |
| **定义** | 整个操作系统（除了一些特殊驱动）都运行在**同一个大内核空间**里。 |
| **代表系统** | **Linux, UNIX, MS-DOS**。 |
| **优点** | **性能高**。因为所有的模块都在一起，系统调用通过简单的函数调用即可完成，没有跨进程通信的额外开销。 |
| **缺点** | **稳定性差**。如果文件系统模块崩溃，整个内核就崩溃，系统会死机。 |
| **开发难度** | 庞大、耦合度高，所有开发者必须在内核空间工作，调试困难。 |

*   **Layered Structure**（分层结构）：如何解决单内核的耦合问题？
*   **Microkernel**（微内核）：如何解决单内核的稳定性问题？
