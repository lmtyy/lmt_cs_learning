这一组PPT深入讲解了 **“系统调用的分类”** ，并重点对比了 **“进程控制”** 这一类别在 **MS-DOS（单进程）** 和 **UNIX（多进程）** 下的不同内存布局和执行模型。

### 第一张图：Types of system calls（系统调用的5大类型）

这张图是对OS提供的所有API功能的分类。你可以把系统调用看作“服务窗口”，根据功能不同分为5类：

1.  **Process control（进程控制）**：管理运行中的程序（创建、结束、等待）。
2.  **File management（文件管理）**：对文件的操作（打开、读取、写入、关闭）。
3.  **Device management（设备管理）**：对硬件的操作（请求设备、释放设备、读/写设备）。
4.  **Information maintenance（信息维护）**：获取或设置系统信息（如获取当前时间、设置进程优先级、获取系统配置）。
5.  **Communications（通信）**：进程之间或网络上的数据交换。

这一章接下来的PPT重点讲解 **第一类：Process control**。

### 第二张图：OS API: Process control（进程控制系统调用）

这里列出了在编程时，用于控制进程生命周期的具体API指令：

*   **Create & terminate**：`fork`（创建进程），`exit`（终止进程）。
*   **Execute**：`exec`（加载并执行一个新程序，替换当前进程）。
*   **Get/Set attributes**：获取或设置进程的属性（如优先级）。
*   **Abort, end**：强制中止或正常结束进程。
*   **Wait for time/event**：进程可以**挂起**自己，等待某个时间点或某个特定事件发生。
*   **Allocate/free memory**：申请或释放内存资源。

这些是编写高级程序（如Shell、浏览器、服务器）的基础。

### 第三张图：Process control (MS-DOS) —— 早期的单进程系统

这张图展示了 **MS-DOS** 操作系统处理进程的方式。它代表了 **“单线程、单进程”** 的古老模型。

*   **图 (a)**：Shell（命令解释器）在运行，位于内存中间位置，上面是空闲内存，下面是Kernel。
*   **图 (b)**：当用户在Shell中输入命令（如 `edit`）时：
    *   **发生了什么？** DOS把命令解释器加载进内存，启动了新进程（Process）。
    *   **特点**：新进程被加载到**内存空闲区的顶部**。此时命令解释器依然存在于内存中。
    *   **后果**：这是一种**非常简陋且不安全**的方式。如果新进程程序本身写得很大，它可能会**覆盖**掉Shell所在的内存区域。
    *   **总结**：一旦新进程运行，用户就无法终止它或切换回Shell，必须等新进程结束，才能重新启用Shell。

### 第四张图：Process control (UNIX) —— 现代多进程系统

这张图展示了 **UNIX**（以及Linux、macOS等现代OS）如何处理进程。

*   **内存布局**：你可以看到Kernel始终在底部。上方可以同时存在多个进程（Process B, Process C, Process D）和自由内存。
*   **核心机制（fork + exec）**：
    *   **Fork**：创建一个**完全相同**的进程副本（子进程）。
    *   **Exec**：在子进程中，用新程序（如 `ls`, `vi`）**替换**掉子进程的代码段。
    *   **结果**：父进程（Shell）和子进程可以**同时存在**。父进程可以继续运行（甚至等待子进程结束），而子进程在后台或前台运行。
*   **优势**：
    *   **并行性**：Shell（解释器）在输入命令后，不会被新程序覆盖或阻塞（除非用户主动要求）。
    *   **保护**：内存分区明确，不同的进程不会互相覆盖。
    *   **灵活性**：你可以在运行一个程序的同时，切换到Shell执行其他命令。

### 💡 核心对比：DOS vs UNIX

| 特征 | MS-DOS（图3） | UNIX/Linux（图4） |
| :--- | :--- | :--- |
| **并行能力** | 一次只允许一个进程运行（单进程） | 允许多个进程同时执行（多进程） |
| **加载模型** | 新进程覆盖Shell（覆盖式加载） | 新进程通过 `fork()` 作为子进程创建 |
| **Shell可用性** | 运行新程序时，Shell消失 | Shell 依然存活，可以等待子进程或继续工作 |
| **内存布局** | 容易发生内存覆盖 | 严格的内存隔离和保护 |

**考试/面试重点**：
*   必须牢记 **`fork()`** 和 **`exec()`** 的区别。
*   理解DOS的缺陷（程序崩溃可能拖垮整个系统）与UNIX的优势（一个进程崩溃不影响其他进程或Shell）。
