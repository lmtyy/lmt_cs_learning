### 第一张图：OS API / System Calls 总览

**核心概念**：
*   **System Call（系统调用）**：这是**运行中的程序**和**操作系统内核**之间的接口。简单说，当你想让操作系统为你做事（比如读写文件、创建进程、打印）时，你就是在调用系统调用。
*   **语言层面**：
    *   **底层**：系统调用本质上是一条**汇编指令**（比如早期的 `int 0x80` 或现代的 `syscall`）。
    *   **高层**：为了不让程序员写汇编，高级语言（如 **C, C++, Perl**）允许**直接**调用系统调用。你平时写C语言时用的 `printf()` 或 `fopen()`，底层其实就是在调用系统调用。

**例子说明：`copy file1 file2`**
这是一个经典的Unix复制文件命令的底层逻辑。程序员在写这个程序时，需要做以下几件事（每一件事都需要调用一个或多个系统调用）：
1.  **如何提供文件名**：找到 `file1` 和 `file2`。
2.  **如何准备输入/输出文件**：`open()` 打开源文件，`creat()` 或 `open()` 创建目标文件。
3.  **如何进入循环**：`read()` 从源文件读数据块，`write()` 把数据块写入目标文件，然后循环。
4.  **如何释放资源**：`close()` 关闭两个文件。

### 第二张图：系统调用的参数传递（三种方法）

**核心问题**：当用户程序（运行在**用户态**）要调用操作系统（运行在**内核态**）时，它们之间**如何传递数据（参数）**？

操作系统规定在用户态和内核态之间，数据不能随意传递（为了安全）。PPT列出了三种经典方法：

1.  **通过寄存器传递参数（Pass parameters in registers）**
    *   **最直接**：把参数直接塞进CPU的寄存器（如 `eax`, `ebx`）。
    *   **缺点**：CPU寄存器数量有限，如果参数太多（比如要传10个参数），寄存器就不够用了。

2.  **通过内存中的表传递参数（Store parameters in a table）** —— **这是本节课件的重点**
    *   **流程**：先在内存里建一个**参数表**，把参数放在表里。然后只把**这个表的地址**放在寄存器里传给OS。
    *   **好处**：可以传任意数量的参数。
    *   **实例**：**Linux** 系统调用就广泛使用这种方法。（请参考第三张图的详细解释）。

3.  **通过栈（Stack）传递参数**
    *   **流程**：程序把参数**压入（Push）** 栈中，然后OS从栈里**弹出（Pop）** 取出参数。
    *   **好处**：不需要指定寄存器，代码更通用。但堆栈操作在切换特权级时会有一些额外开销。

### 第三张图：参数传递的详细图解（Linux 方法）

这张图完美地演示了**“方法2：通过内存中的表传递参数”**的实际工作流。

**图解步骤（从左到右）：**

1.  **用户程序（左侧 User Program）的准备阶段**：
    *   程序员在代码里把需要的参数（如文件名、读写标志等）放入一个叫 **X** 的内存块（参数表）里。
    *   **关键动作1**：用户程序执行 **`load address X`**（把参数表 X 的内存地址装入 CPU 寄存器，比如 `eax`）。
    *   **关键动作2**：用户程序执行 **`system call 13`**（把系统调用号 13 装入一个特定的寄存器，比如 `ebx`）。这里的 **13** 代表“我想做第13号系统调用”（比如 `read` 是某个编号，`write` 是另一个编号）。

2.  **硬件/内核介入**：
    *   CPU遇到系统调用指令，立刻暂停用户程序，从**用户态**切换到**内核态**。硬件会根据寄存器的值（地址和调用号）告诉操作系统：“去这个地址 X 找我的参数，帮我做第13号系统调用”。

3.  **操作系统（右侧 Operating System）处理阶段**：
    *   操作系统内核根据调用号（13）找到了对应的处理代码（**code for system call 13**）。
    *   内核去读取寄存器里的地址 X，找到内存里的参数表 X。
    *   内核从参数表中取出参数（**use parameters from table X**），执行相应的工作（比如读写磁盘）。

4.  **返回**：
    *   执行完成后，内核把结果放回寄存器，切换回用户态，用户程序继续往下运行。

### 💡 重点总结与学习建议

1.  **分清角色**：
    *   **OS Services** 是你（作为用户）希望操作系统**做什么**。
    *   **System Calls / OS API** 是你（作为程序员）编写的代码**如何告诉操作系统**你要做什么。

2.  **图3必须看懂**：这是考试中非常容易考到的**计算/原理题**。老师可能会问：“请画出用户程序与操作系统之间通过参数表传递参数的过程。” 答案就是这张图。

3.  **三个方法需要记忆**：寄存器传递、表传递（Linux）、栈传递。了解各自的优缺点。
