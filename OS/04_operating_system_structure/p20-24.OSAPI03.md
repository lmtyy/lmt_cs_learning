这几张PPT详细介绍了 **OS API** 中剩下的四种系统调用分类：**文件管理、设备管理、通信** 以及 **信息维护**。

结合之前讲过的**进程控制**，至此，操作系统提供的所有API功能分类已经完整呈现。

下面我为你逐一解读这些功能以及其中最重要的**通信机制对比图**。

---

### 第一张图：OS API: File management（文件管理系统调用）

这是程序员日常打交道最多的系统调用类别。它对应的是之前讲过的 **OS Components: File management**。

*   **Create/Delete**：创建和删除文件。
*   **Open/Close**：打开和关闭文件（打开后才能读写）。这是为了操作系统可以管理文件表。
*   **Read/Write/Reposition**：
    *   **Read/Write**：从文件中读取数据或向文件写入数据。
    *   **Reposition**：移动文件指针（比如在文件里跳到第1000个字节）。
*   **Get/Set attributes**：获取或设置文件的属性，比如文件大小、修改时间、权限（读/写/执行）。
*   **Directory operations**：对**文件夹（目录）** 的操作，比如创建/删除目录、列出目录内容。

---

### 第二张图：OS API: Device management（设备管理系统调用）

这部分与文件管理非常相似，因为**在UNIX/Linux系统中，设备也被当作文件来处理**。

*   **Request/Release**：请求和释放设备。例如，你需要使用打印机，就要先请求它，用完后再释放。
*   **Read/Write/Reposition**：读写设备（如键盘、显示器、磁带机）以及移动设备指针。
*   **Get/Set attributes**：获取或设置设备属性（如打印机的分辨率、终端的波特率）。
*   **Attach/Detach**：逻辑上挂载或卸载设备。比如插入U盘，系统通过挂载（Attach）让你看到它的内容。

---

### 第三张图：OS API: Communication（通信管理系统调用）

这部分负责进程之间的数据交换（IPC：Inter-Process Communication）。

*   **Create/Delete connection**：建立或断开通信连接。
*   **Send/Receive messages**：发送和接收消息（基于消息传递的模型）。
*   **Transfer status information**：传递状态信息（例如确认发送是否成功）。
*   **Attach/Detach remote devices**：挂载或卸载远程设备（如通过网络连接远程打印机）。

---

### 💡 第四张图：OS API: Communication（通信模型对比图）—— **最重要的知识点**

这张图展示了进程间通信的**两种核心模型**。请仔细看这张图，它往往包含在期末考试或考研试题中。

#### **左图：Message Passing（消息传递）**
*   **机制**：进程之间**不共享任何内存**。进程A把要发送的数据打包成“消息”，通过**操作系统内核**（中间的Kernel区）作为中介进行传递。
*   **流程**：
    1.  Process A 把消息发送给 Kernel（交给系统调用）。
    2.  Kernel 接收到消息，把它复制到内核缓冲区。
    3.  Kernel 再把消息发送给 Process B。
*   **特点**：
    *   **安全**：进程间互不干扰。
    *   **开销大**：需要在内核态和用户态之间**两次拷贝数据**（慢）。
    *   **例子**：网络通信（Socket）、管道（Pipe）。

#### **右图：Shared Memory（共享内存）**
*   **机制**：操作系统在内存中划出一块**共享区域**，让进程A和进程B都映射到这块区域。
*   **流程**：
    1.  Process A 直接写入共享内存（不需要内核介入）。
    2.  Process B 直接从共享内存读取（同样不需要内核介入）。
*   **特点**：
    *   **最快**：不需要内核拷贝，是**最快的IPC方式**。
    *   **需要同步**：因为A和B可以同时读写，所以需要程序员自己加锁或信号量来防止冲突。
    *   **例子**：大型数据库、图像处理软件。

**💡 考试对比黄金口诀**：
*   **Message Passing** = 安全（有内核把关） + 慢（双拷贝）。
*   **Shared Memory** = 超快（无拷贝） + 危险（需程序员自己同步）。

---

### 第五张图：OS API: Information maintenance（信息维护系统调用）

这部分主要是获取或设置系统运行过程中产生的各种信息和数据。

*   **Get/Set time or date**：获取或设置系统时间、日期。
*   **Get/Set system data**：获取或设置系统数据，例如主机名、系统版本号。
*   **Get/Set process/file/device attributes**：获取或设置进程/文件/设备的属性信息（例如获取当前进程的ID、设置文件权限等）。

---

### 💡 总结：OS API 功能一览表

至此，五大类系统调用已经全部讲完，帮你用一句话总结：

| 分类 | 核心任务 | 典型例子 |
| :--- | :--- | :--- |
| **Process control** | 管理程序的生老病死 | `fork()`, `exec()`, `exit()` |
| **File management** | 管理文件和目录 | `open()`, `read()`, `write()` |
| **Device management** | 管理硬件设备 | `ioctl()`, `read()` |
| **Communication** | 让进程之间交换数据 | `pipe()`, `shmget()`, `socket()` |
| **Information maintenance** | 获取和设置系统信息 | `gettimeofday()`, `getpid()` |
