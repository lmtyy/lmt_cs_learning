这三张幻灯片专门讲解了**FIFO（First-In, First-Out，先进先出）页面置换算法**。这是最基础的算法，也是后续学习其他复杂算法（如LRU、Optimal）的“对照组”。

---

### 1. 算法核心思想（第1张PPT）

*   **规则**：操作系统会跟踪每个页进入内存的时间。当需要置换时，**替换掉内存中存在时间最长（最老）的那一页**。
*   **类比**：就像排队结账——最早来排队的人，最先被服务（或被踢走）。
*   **实现方式**：使用一个 **FIFO队列**。
    *   新页入队时：加到队尾（Tail）。
    *   发生缺页时：从队头（Head）踢出最老的页。
*   **优缺点**：
    *   ✅ **优点**：简单，容易理解和实现，开销小。
    *   ❌ **缺点**：性能不一定好。因为一个经常被访问的“热门页”，如果它是早期加载的，它可能被无辜地踢走，导致很快又缺页。

### 2. 模拟示例（第2张PPT）

这张图展示了一个具体的运行轨迹：
*   **Reference String**（页面引用串）：`7 0 1 2 0 3 0 4 2 3 0 3 2 1 2 0 1 7 0 1`
*   **Frame**（物理帧数）：这里假设分配了 **3个帧**。

**如何看懂这排小方块：**
1.  每一个垂直的列代表一个时刻的内存状态。
2.  数字串代表访问的页号。
3.  框内的数字是当前内存里的页。
4.  当出现新数字且内存已满时，最上面的（最老的）被踢出，新的数字被加到最下面。
    *   *例如：当访问 `2`，内存已有 `7,0,1`。`7`被踢出，`0`变成最老，`1`次老，`2`进来。*

**结果**：对于这个特定的序列和3个帧，FIFO发生了多次缺页。

### 3. 史上著名坑点：Belady异常（第3张PPT）

这是FIFO算法最著名的“坏脾气”表现，也是**考试高频考点**。

*   **常识推理**：按常理，如果把分配给进程的内存**增加**（帧数变多），缺页次数应该**减少**才对（如之前那个下降的曲线图）。
*   **Belady异常**：但FIFO算法下，**增加帧数，反而可能导致缺页次数增加！**
    *   **例子**：参考串 `1, 2, 3, 4, 1, 2, 5, 1, 2, 3, 4, 5`
    *   **3个帧时**：发生了9次缺页。
    *   **4个帧时**：发生了10次缺页。
    *   **图**：你看折线图，在 `3` 帧和 `4` 帧之间，曲线竟然是**向上**走的（缺页数从9上升到了10）。

**为什么会发生这种现象？**
因为FIFO完全不考虑页的“使用频率”，它只按“进入时间”踢人。当帧数增加时，队尾更长，会导致某些关键页在错误的时间点被保留或踢出。

---

### 📝 考试速记与考点

1.  **原理**：踢最老的。
2.  **数据结构**：队列 (Queue)。
3.  **最大缺点**：**Belady异常**（即 `增加物理帧数 → 反而缺页增多`）。
4.  **后续对比**：接下来的课程中，你会学到 **LRU (Least Recently Used)**，这个算法可以完全避免 Belady异常，因为它不仅看“进入时间”，更看“使用时间/频率”。
