这两张幻灯片介绍了**LRU（Least Recently Used，最近最少使用）页面置换算法**。

这是实际操作系统中最常使用的算法之一（或它的变种），也是考试中非常喜欢考的题目。

### 1. LRU 的核心思想（第1张PPT）

这张PPT做了一组非常棒的对比，帮你理清三个算法的区别：

*   **FIFO（先进先出）**：只看**进入的时间**，不看是否被用过。谁最老，踢谁。
*   **OPT（最佳置换）**：**看未来**。谁在未来最晚才用到，踢谁。这是理想情况，但现实中不可能实现。
*   **LRU（最近最少使用）**：**看过去，预测未来**。它依据“局部性原理”推断：如果一页在过去很久都没被用过，那么它未来大概率也不会被立刻用到。
    *   **规则**：替换掉**最长时间没有被使用**的那一页。

**通俗理解**：就像清理家里的冰箱。有一包菜放在冰箱里，你每天都会拿它做菜（最近使用了），你会把它放回去。但有一包冷冻薯条，你放进去半年了，从来没碰过（最近最少使用），你会优先把它丢掉，因为它大概率以后也不会吃。

### 2. 如何手动模拟 LRU（第2张PPT）

这是考试中需要掌握的核心技能。结合这张图，我来教你如何手动计算：

**给定引用串：** `7 0 1 2 0 3 0 4 2 3 0 3 2 1 2 0 1 7 0 1`
**假设有3个帧。**

**关键步骤拆解（用手工模拟方法）：**

1.  **头部空位**：`7, 0, 1` 依次进入。内存：[7, 0, 1]。（缺页 x3）
2.  **访问 `2`**：
    *   内存满了：[7, 0, 1]。
    *   **找受害者**：比较这三个页的“最近使用时间”。
        *   `1` 刚被用过（最近）。
        *   `0` 在 `1` 之前被用过。
        *   `7` 是最久远的（最久没被用过）。
    *   **操作**：踢掉 `7`。内存变为 [2, 0, 1]。（页面顺序反映最近使用情况）
3.  **访问 `0`**：
    *   `0` 已经在内存中。**这是命中（Hit）**。
    *   **重要操作**：你需要**更新 `0` 的“最近使用时间”**。把它标记为“最新鲜”的。
    *   此时内存的“新旧程度”顺序变为：[2, 1, 0]（`0` 最新，`2` 最久远）。
4.  **访问 `3`**：
    *   内存中有 [2, 1, 0]。需要置换。
    *   **找受害者**：`0` 和 `1` 最近刚被用过。`2` 在“访问 `0`”之后就没被用过。
    *   **操作**：踢掉 `2`。内存变为 [3, 1, 0]。
5.  **访问 `0`**：
    *   命中 `0`。更新它为最新。内存顺序变为 [3, 1, 0] → [3, 1, 0]（调整0为最新）。

### 💡 考试做题技巧：使用“时间戳”或“栈”

为了避免手动模拟时搞混顺序，考试时推荐这两种记录方法：

**方法A：时间戳**
给每一行写一个时间数字。某页被访问时，就给它打上最新的时间戳。置换时，选择时间戳**最小（最旧）**的那个。

**方法B：栈（Stack）**
用“堆栈”来模拟。
*   每次访问一个页：如果它在栈里，就把它**拿出来，放到栈顶**。
*   如果需要置换：直接踢掉**栈底（最底部）**的那个元素（因为它最久没被用到）。
*   这个方法在纸上模拟最快！

**以此题为例（栈法）：**
*   访问 `7`：栈 [7]（7最旧）
*   访问 `0`：栈 [0, 7]
*   访问 `1`：栈 [1, 0, 7]
*   访问 `2`：栈满。栈底是 `7`，踢掉。栈变 [2, 1, 0]
*   访问 `0`：命中。把 `0` 移到栈顶。栈变 [0, 2, 1]
*   访问 `3`：栈满。栈底是 `1`，踢掉。栈变 [3, 0, 2]

这样是不是清晰多了？

### 📝 考点总结

1.  **看过去**：根据最近使用的时间。
2.  **与FIFO对比**：FIFO不管“访问”，LRU照顾“频率”。
3.  **性能**：比FIFO好（通常更接近OPT），且**不会发生Belady异常**（这点要记住，常考选择题）。
4.  **硬件开销**：比FIFO大，因为每次访问都要更新“时间戳”或维护栈结构。
