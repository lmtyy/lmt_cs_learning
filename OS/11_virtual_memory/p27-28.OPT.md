这两张幻灯片介绍的是**OPT（Optimal，最佳置换算法）**。

在操作系统的教学和考试中，OPT通常被称为“**理论上的天花板**”。它无法在现实中实现，但却是衡量其他算法好坏的标尺。

---

### 1. OPT 的核心定义（第1张PPT）

*   **规则**：当需要置换时，替换掉**在未来最长时间内不会被访问**的那一页。如果在未来会用到，那就选那个**用得最晚**的。如果某些页永远不再被访问，那就优先替换它们。
*   **特点**：
    *   拥有所有算法中**最低的缺页率**。
    *   绝对不会发生 **Belady异常**。
*   **最大的软肋**：
    *   **难以实现**：你需要知道程序**未来**的访问顺序。这对计算机来说是“算命”，操作系统无法预知未来。
*   **实际用途**：
    *   它主要被用作**衡量标准**。
    *   考试时，你会对比：**OPT的缺页数** vs. **FIFO/LRU等算法的缺页数**，来判断谁更优秀。

### 2. 如何手动计算 OPT（第2张PPT）

这张图展示了 OPT 是如何运作的。由于PPT只截取了部分步骤，我为你补充一下**手动模拟的思路**（**这在考试中很常见**）：

**给定引用串：** `7 0 1 2 0 3 0 4 2 3 0 3 2 1 2 0 1 7 0 1`
**假设有3个帧。**

1.  **第一步**：`7, 0, 1` 依次进入内存。由于内存未满，全部放入。（缺页 x3）
2.  **关键步骤：访问 `2`**：
    *   内存里有 `7, 0, 1`，满了。
    *   现在你需要**看未来**：在 `2` 之后，引用串是 `0 3 0 4 2...`
    *   `0` 将在下一个时刻被用到（很近）。
    *   `1` 将在很久以后（在第13个位置）才被再次用到。
    *   `7` 将在第17个位置才被再次用到。
    *   **决定**：`7` 和 `1` 都是很久以后才用到。但 `1` 是未来出现得最晚的（或者说，比 `7` 稍近一点）。通常如果有一个页在未来永远不会再被访问，它会优先被替换。在这里，为了逻辑清晰，标准模拟中通常比较它们的“下一次出现距离”。
    *   按照最优原则，`7` 是最晚被重新使用的。所以替换掉 `7`。内存变为 `2, 0, 1`。
3.  **关键步骤：访问 `3`**：
    *   查看未来：`0` 即将出现（位置6），`2` 在位置9左右出现。
    *   `1` 在很远的地方才出现（位置15左右）。
    *   所以，踢掉 `1`。内存变为 `2, 0, 3`。

**💡 选择题/计算题技巧**：考试中如果考 OPT，唯一的难点是**“挑选受害者”**。你只需要拿着题目给出的完整引用串，**向下看（看未来）**，谁在未来最长的时间里不会出现，就踢谁。这是与其他算法（看过去、看时间）的根本区别。
