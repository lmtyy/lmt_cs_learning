好的，这四张PPT讲解了操作系统保护模型中最经典的理论工具：**访问矩阵（Access Matrix）**。

你在之前看到的“保护域”概念，在这里被具象化为一张二维表格。这是操作系统教材中必考的核心模型。我们一步一步拆解：

---

### 第一张图：访问矩阵的定义

这张图给出了理论基础。

*   **行**：代表 **域（Domain）**（即发起访问的实体，比如进程或用户）。
*   **列**：代表 **对象（Object）**（即被访问的资源，比如文件、打印机、内存段）。
*   **单元格 \(Access(i,j)\)**：代表域 \(i\) 可以对对象 \(j\) 执行的操作集合（如 `读`、`写`、`执行` 等）。

**一句话总结**：这张矩阵集中了整个系统的所有访问权限。

---

### 第二张图：示例图（Figure A）

这是一个具体的例子。注意看这个表格：

| 域 | F1 | F2 | F3 | Printer |
| :--- | :--- | :--- | :--- | :--- |
| \(D_1\) | read | | read | |
| \(D_2\) | | | | print |
| \(D_3\) | | read | execute | |
| \(D_4\) | read/write | | read/write | |

**解读：**
*   **行是域**：这里有 \(D_1, D_2, D_3, D_4\) 四个域。
*   **列是对象**：有文件 \(F_1, F_2, F_3\) 和一台打印机。
*   **空白单元格**：意味着该域对此对象没有任何权限。例如 \(D_2\) 对 \(F_1\) 的权限是空白，说明 \(D_2\) 里的进程**完全不能碰**文件 \(F_1\)。
*   **内容**：\(D_1\) 能读 \(F_1\) 和 \(F_3\)；\(D_2\) 只能用打印机；\(D_3\) 能读 \(F_2\) 并执行 \(F_3\)；\(D_4\) 拥有 \(F_1\) 和 \(F_3\) 的读写权限。

---

### 第三张图：如何控制域切换（Switch）

这张PPT提出了一个非常巧妙的思考：
我们在之前讲到，进程是可以“切换保护域”的。但在操作系统里，**切换域本身也是一种操作，必须受到控制**。

*   **机制**：
    1.  把 **“域本身”也当作访问矩阵里的“对象”**（即加到列里去）。
    2.  引入一个新的特殊操作：**`switch`**。
*   **规则**：
    *   如果域 \(D_i\) 想要切换到域 \(D_j\)，那么在访问矩阵的单元格 \(Access(i, j)\) 中，必须包含 **`switch`** 这个权限。
    *   换句话说：**你是由于在 \(D_i\) 里有切换到 \(D_j\) 的权限，才能完成切换**。

**意义**：这防止了进程随意跳来跳去。只有被明确授予了“切换权”的域，才能切换到目标域。

### 第四张图：扩充后的访问矩阵（Figure B）

这张图是 **Figure A** 的**升级版**。它把第三张PPT的理论应用到了实际例子中。

现在的列不仅有文件 (\(F_1, F_2, F_3\)) 和打印机，还增加了域 (\(D_1, D_2, D_3, D_4\)) 作为对象。

**让我们来找找“切换权”在哪里：**

*   **看 \(D_1\) 行**：在 \(D_2\) 这一列下，写着 `switch`。这意味着：**在 \(D_1\) 域里运行的进程，可以切换去 \(D_2\) 域。**
*   **看 \(D_2\) 行**：在 \(D_3\) 和 \(D_4\) 列下，写着 `switch`。意味着：**在 \(D_2\) 域里运行的进程，可以切换到 \(D_3\) 或 \(D_4\) 域。**
*   **看 \(D_4\) 行**：在 \(D_1\) 列下，写着 `switch`。意味着：**在 \(D_4\) 域里，可以切换回 \(D_1\) 域。**

**这就形成了一个完整的动态切换图：**
\(D_1 \rightarrow D_2 \rightarrow (D_3 \text{ 或 } D_4) \rightarrow D_1\) (循环)

### 💡 核心考点总结

1.  **什么是访问矩阵？**：一个二维表格，**行=域，列=对象**，它是系统保护策略的完整数学描述。
2.  **如何表示无权限？**：直接留空/空白。这也是保护的核心所在。
3.  **如何实现域切换？**：**把域设为对象**，并在相应的单元格里填入 `switch` 操作。
4.  **动态控制的妙处**：这个模型非常统一。
    *   要控制对**文件**的访问？看矩阵里对应**文件列**。
    *   要控制对**域**的切换？看矩阵里对应**域列**。

**一句话记忆**：
访问矩阵就是把“谁能在哪个对象上做什么”写成一张表。为了控制域切换，我们只需要把**域自己也当成对象**，并在表格里写明谁能“switch”到谁。
