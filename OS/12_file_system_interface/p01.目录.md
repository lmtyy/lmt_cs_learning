这是**第 11 章**的内容。它列出了这一章要讲的三个核心主题。我为你逐一拆解这些概念，帮你把这个章节的骨架搭建起来：

### 1. File Concept（文件概念）
这是整个章节的基础。操作系统眼中的“文件”是什么？
*   **定义**：文件是被命名的、存储在磁盘等外存上的相关信息的集合。
*   **文件属性**：一个文件不仅有内容（数据），还有“身份证”（元数据）。包括：文件名、标识符、类型（如 .txt, .exe）、位置（在磁盘上的地址）、大小、保护机制（谁可以读/写）、时间戳（创建/修改/访问时间）。
*   **文件结构**：有的系统视文件为**字节流**（Linux/UNIX，按字节存取），有的为**记录流**（如早期的数据库视文件为定长记录）。

### 2. File Access（文件访问）
这一节讲的是用户/程序如何“读写”文件。主要有两种模式：
*   **顺序访问（Sequential Access）**：
    *   像读磁带一样。必须从头开始读，读完第一个字节才能读第二个。
    *   原理简单，是早期系统最常见的方式。
*   **直接访问（Direct Access / Random Access）**：
    *   像读DVD或U盘。可以直接跳到任意位置（例如第 1000 个字节）开始读，不需要读出前面的部分。
    *   现代数据库和大多数通用操作系统都支持这种模式（通过 `lseek` 等操作）。

### 3. Directory Structure（目录结构）
这一节讲的是如何**组织**成千上万个文件。
*   **为什么需要目录？** 为了用户方便查找，并防止文件重名。
*   **常见的目录结构**：
    *   **单级目录**：所有文件都在同一个大目录下（类似把所有文件堆在桌面上，不同用户文件全混在一起，难找且易重名）。
    *   **两级目录**：不同用户有各自的目录，不同用户的文件可以同名。
    *   **树形目录（现代最常用）**：类似现在电脑的文件夹树，根目录下可以有子目录，子目录还可以有子目录。非常灵活，路径名（如 `/home/user/project/main.c`）清晰。
    *   **无环图目录**：在树的基础上，允许文件/目录之间有“快捷方式”（链接）。同一个文件可以属于两个不同的目录。

### 💡 学习建议：
这个章节主要讲的是**用户视角（接口）**，而不是底层实现。
*   **文件接口**：对应你的 C 语言 `fopen`, `fread` 等函数。
*   **目录接口**：对应 `mkdir`, `ls`, `cd` 等命令。

学习这一章时，重点理解 **“路径名是如何映射到文件实体”** 以及 **“用户是如何通过系统调用（System Call）来操作文件的”**。
