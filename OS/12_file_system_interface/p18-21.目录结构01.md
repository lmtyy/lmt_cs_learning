这组幻灯片进入了本章的最后一部分：**目录结构（Directory Structure）**。

前面我们讲了单个文件长什么样、怎么读写。现在我们要讲的是：**几十万个文件放在一起，系统该怎么组织它们？** 这部分内容逻辑非常清晰，涉及从物理划分到逻辑组织。

### 第一张图：物理划分（Partition / Volume）

在讨论“目录树”之前，必须知道文件系统在磁盘上是如何物理划分的。
1.  **问题**：现代计算机存储着百万级文件，必须对它们进行组织。
2.  **两步走**：
    *   **第一步（物理划区）**：磁盘被划分为一个或多个**分区（Partitions）**。在 IBM 系统中称为 **Minidisk**。每个分区是一个低级的结构，文件和目录就住在里面。
    *   **第二步（建立索引）**：每个分区包含一个 **设备目录（Device Directory）** 或 **卷目录表（Volume Table of Contents, VTOC）**。这个表记录了该分区内所有文件的信息（Name, Location, Size, Type）。
*   **直观理解**：
    *   **分区** 就像你的 C盘、D盘。每个盘是一个独立的空间。
    *   **设备目录** 就像本书的“目录页”，告诉你在哪里能找到哪一页。

### 第二张图：目录的逻辑定义与操作

这里定义了什么是“目录”以及我们能对目录做什么。
*   **定义**：目录是用来组织文件（以及其他目录）的工具。它是一个**符号表（Symbol Table）**，负责将**文件名**（人读）映射到**目录项**（系统读）。
*   **操作**：
    *   Search a file（搜索文件）
    *   Create a file（创建文件）
    *   Delete a file（删除文件）
    *   List a directory（列出目录内容 - 也就是 `ls` 命令的原理）
    *   Rename a file（重命名文件）
    *   Traverse the file system（遍历文件系统）

### 第三张图：目录设计的标准（Criteria）

一个好的目录系统应该满足什么要求？这不仅是考点，也是设计目录的理由：
1.  **Efficiency（效率）**：能快速定位一个文件（不要搜索几百万个文件还找不到）。
2.  **Naming（命名）**：方便用户。
    *   **异名同实**：允许两个用户有相同的文件名（比如 UserA 有 `proj.txt`，UserB 也有 `proj.txt`，它们是不冲突的）。
    *   **同名异实**：同一个物理文件可以有多个名字（即硬链接/软链接的基础）。
3.  **Grouping（分组）**：能够根据文件的属性进行逻辑分组（比如所有 Java 程序放在一个文件夹里，所有图片放在另一个）。

### 第四张图：五种目录结构（核心架构图）

这是本章**最常考的知识点**。列举了目录结构进化的 5 个阶段：

1.  **Single-level directory（单级目录）**
    *   **形式**：所有文件都在一个大根目录下。
    *   **缺点**：难以支持多个用户（重名问题严重）；查找效率低；无法分组。

2.  **Two-level directory（两级目录）**
    *   **形式**：根目录下是每个用户的 UFD（User File Directory），用户目录下才是文件。
    *   **优点**：不同用户文件可以同名（解决了命名冲突）；不同用户彼此隔离。
    *   **缺点**：用户自己内部依然混乱；没有子目录概念。

3.  **Tree-structured directory（树形目录）**
    *   **形式**：现代操作系统标准。根目录 `/` 或 `C:\`，可以有任意深度的子目录。
    *   **优点**：完美解决了查找效率和分组问题。路径名（Path Name）唯一标识一个文件。
    *   **缺点**：不能共享文件（一个文件必须只属于一个父目录）。

4.  **Acyclic graph directory（无环图目录）**
    *   **形式**：在树形目录的基础上，允许**共享（共享文件/共享目录）**。
    *   **特点**：可以让同一个文件出现在两个不同的目录中（通过链接 Link 实现）。结构中没有环（即 A 指向 B，B 不会指向 A）。
    *   **缺点**：必须解决“垃圾回收”（删除文件时：如果两个目录都指向它，删一个目录不能真的删除文件，必须等到所有引用都删除才能删）。

5.  **General graph directory（通用图目录）**
    *   **形式**：允许结构中出现**环（Cycle）**。
    *   **问题**：如果出现环（比如 A 包含 B，B 又包含 A），当你遍历文件系统时就会陷入无限循环。
    *   **解决方案**：必须引入垃圾回收机制（Garbage Collection）或引用计数来解决。

### 💡 考试攻略

这部分内容在考试中非常清晰：

1.  **简答题/选择题**：
    *   **哪一种是现代操作系统最常用的？** → 树形目录（Tree-structured）。
    *   **当需要文件共享时怎么办？** → 无环图目录（Acyclic graph）。
    *   **单级目录最大的问题是什么？** → 命名冲突（Name collision）。

2.  **理解路径**：
    *   **绝对路径**：从根目录开始（如 `/home/user/doc.txt`）。
    *   **相对路径**：从当前目录开始（如 `./doc.txt` 或 `../other/file.txt`）。这部分是目录结构的具体使用，课件后续可能会讲到。

3.  **无环图的实现**：通常通过 **Link（链接）** 实现，也就是你在 Windows 的“快捷方式”或 Linux 的 `ln` 命令。重点在于链接删除时的引用计数处理（当计数归零时才释放数据块）。
