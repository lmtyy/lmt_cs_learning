这几张幻灯片讲的是目录结构进化的**最后两个阶段**，也是最复杂的部分。

前面的**树形目录**解决了组织和唯一命名的问题，但它有一个无法避免的痛点：**无法共享**。如果你想在多个不同文件夹里引用同一个文件，树形结构里你只能复制一份（浪费空间且难以同步），或者必须把所有相关东西放进同一个文件夹（不方便）。

为了解决这个问题，我们引入了**无环图（Acyclic Graph）** 和 **通用图（General Graph）** 结构。

---

### 第一张图：无环图目录（Acyclic-graph directories）

*   **示意图解读**：
    *   你可以看到，左下角名为 `list` 的**目录**，同时被左上角的 `count` 目录和右侧的 `list` 目录指向。
    *   同样，左侧 `all` 目录下的那个**文件**，也被下方的 `list` 目录指向。
    *   这就是**共享**。同一个文件或子目录可以出现在“树”的多个不同分支上。
    *   **“无环”的意思是**：虽然你可以从 A 走到 B，从 B 走到 C，但你**不能绕一圈走回来**（比如 A 包含 B，B 包含 C，C 又包含 A）。结构里没有闭合回路。

### 第二张图：无环图的特征与问题

*   **公式**：`Tree-structured directory + Sharing → Acyclic-Graph Directories`
    *   意思是：在树形结构的基础上，允许建立“链接”（Links），就变成了无环图结构。这正是 Linux 中的 **Hard Link（硬链接）** 和 Windows 里的 **Shortcut / Symlink（软链接）** 的物理基础。
*   **潜在问题**：
    1.  **Traverse the FS（遍历文件系统）**：如果你要全盘搜索某个文件，你可能会在共享节点上重复进出，需要设计复杂的算法避免重复遍历。
    2.  **To delete a file（删除文件）**：这是**核心考点**。
        *   **情景**：假设文件 F 有 2 个链接（分别来自目录 A 和 目录 B）。如果你删除目录 A 中的 F，你能直接把文件 F 的磁盘数据删掉吗？
        *   **错误做法**：不行！如果你删了，目录 B 里的那个链接就成了空指针（悬空指针）。
        *   **正确做法**：操作系统必须维护一个**引用计数（Reference Count / Link Count）**。
            *   当你在目录 A 删除 F 时：系统把引用计数从 2 减到 1。
            *   当你在目录 B 删除 F 时：系统把引用计数从 1 减到 0。
            *   **当引用计数为 0 时，才真正释放磁盘空间。**

### 第三、四张图：通用图目录（General graph directory）

*   **无环图的“严重问题”**：确保图里没有环（No cycles）。
*   **什么是环？**
    *   举个例子：你在 `A` 文件夹里创建了一个链接指向 `B` 文件夹，又在 `B` 文件夹里创建了一个链接指向 `A` 文件夹。这就形成了一个环：`A → B → A → B → ...`。
    *   如果不做特殊处理，如果你让程序“遍历并删除 `A` 文件夹下的所有内容”，程序就会陷入死循环（因为它会顺着链接跳回 `A` 再跳回 `B` 再跳回 `A`...）。
*   **解决办法**：
    1.  **引用计数（Reference Count）**：需要额外的垃圾回收机制（Garbage Collection）来处理环。
        *   在普通的引用计数中，环里每个节点的计数都至少为 1，永远无法被回收（导致内存泄漏）。
    2.  **标记-清除（Mark-Sweep）** 或其他图论算法：周期性遍历文件系统，标记所有“从根出发能到达的节点”，把无法到达的节点（悬空节点）回收。

### 💡 核心考点对比表

| 结构 | 共享能力 | 是否有环 | 删除的难点 | 典型例子 |
| :--- | :--- | :--- | :--- | :--- |
| **树形目录** | ❌ 无 | ❌ 无 | 简单删除 | 单用户系统 |
| **无环图** | ✅ 有 (硬链接) | ❌ 无 | **引用计数** (必须维护 Link Count) | Linux 的 `ln` (硬链接) |
| **通用图** | ✅ 有 | ✅ 有 (可以有环) | **垃圾回收** (Garbage Collection) | 允许创建软链接指向父目录的情况 |

### 🎯 必背知识点（来自幻灯片）：

1.  **Hard Link vs. Symbolic Link**：虽然在课件里没有细讲，但要注意无环图结构中，删除一个文件不仅仅是删目录项，而是**减少引用计数**。这是硬链接（Hard Link）的核心机制。
2.  **垃圾回收**：通用图结构必须用垃圾回收算法来处理环，否则会出现“删不掉的垃圾”或者“遍历死循环”。
3.  **无环图的定义**：无环图 = 树 + 共享链接。它是大多数现代操作系统支持硬链接的基础结构。
