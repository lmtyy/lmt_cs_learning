这组 PPT 讲解了**文件访问模式（File Access Methods）**。这是文件系统接口中非常基础但又很重要的概念，决定了用户和程序如何读取文件中的信息。

我们分两张图的核心内容来拆解：

### 第一张图：顺序访问（Sequential Access）

*   **定义**：文件中的信息必须**按照顺序**处理，一个记录接着一个记录。
*   **视觉示意图（方框）**：
    *   有一个“当前位置（current position）”指针。
    *   默认操作是“read or write”，指针会随着读写**自动向后移动**。
    *   如果你想到前面去，只能使用 **rewind**（倒带）操作，将指针重置回开头。
*   **形象比喻**：这就像**老式磁带机**或**读卡片机**。你必须听完第1首歌，才能听第2首；没法直接跳到第10首。你唯一能做的控制操作就是“播放下一个”和“倒回开头”。

### 第二、三张图：直接访问（Direct Access）

*   **定义**：文件由**固定长度的逻辑记录**组成。程序可以**不按特定顺序**快速读/写任意记录。这里的“记录”通常对应磁盘上的物理块（block）。
*   **核心操作**：
    *   关键是 **`read n`**、**`write n`**、**`position to n`**。这里的 `n` 是相对块号（relative block number）。
    *   有了这个能力，你就能瞬间跳到文件的第100号位置，而不需要读完前99个。
*   **典型应用**：数据库（Database）通常使用这种访问模式，因为数据库需要快速查询第 X 行，而不管前面的行是什么。

### 第四张图：直接访问伪装成顺序访问（考点）

这张图展示了**直接访问为什么比顺序访问更强大**。

*   **表格内容**：
    *   左边是顺序访问命令（reset, read next, write next）。
    *   右边是**如何在直接访问环境中实现这些命令**。
    *   比如，要实现 **read next**：
        1.  先读当前指针 `cp` 位置的块。
        2.  然后把指针 `cp` 加 1。

*   **PPT 底部的两个结论句子（划重点！）**：
    *   **✅ `To simulate sequential access by direct access is easy`**：
        *   因为直接访问系统允许你随机跳转，你只需要维护一个**当前指针变量**，每次读完后将指针+1，就能模拟出顺序访问的效果。
    *   **❌ `(To simulate direct access by sequential access is NOT easy)`**：
        *   如果你用的是老式磁带机（纯顺序访问），你想读第100块，必须得手动倒带并从头开始读100次。不仅慢，而且机械操作复杂。想模拟直接访问是极其低效甚至不可能实现的。

---

### 💡 考点总结与对比

| 特性 | 顺序访问 (Sequential) | 直接访问 (Direct) |
| :--- | :--- | :--- |
| **物理模型** | 磁带（Tape） | 磁盘（Disk / SSD） |
| **读取第N块** | 必须读完前 N-1 块 | 直接跳到第N块，瞬间读出 |
| **核心操作** | `read next`, `reset` | `read n`, `write n`, `position to n` |
| **应用场景** | 日志文件处理、流媒体（按顺序播放） | 数据库、随机查询、内容寻址 |
| **灵活性** | 低 | 高（能完美模拟顺序访问） |

**考试常考**：
1.  **选择题**：数据库系统通常使用哪种文件访问模式？（答案：Direct Access）
2.  **辨析题**：为什么说“直接访问”可以完全替代“顺序访问”，但反之不行？（答案：因为直接访问通过维护一个变量 `cp` 就能模拟顺序访问，而顺序访问没法高效地随机跳转）。
3.  **实际应用**：如果想在 Linux 的 C 语言里实现直接访问，你需要使用 `lseek()` 系统调用来改变文件指针位置，然后配合 `read()` 使用。
