## 一、进程 vs 程序（第一张图）

**核心定义：**
> **进程是执行中的程序**

- 同一个程序（比如 Chrome、微信）可以有多个进程实例
- 进程可以产生新进程（父子关系）
- 操作系统早期称为 job（批处理）或 task（分时），现在统一叫 **process**

**进程包含四个部分：**
- 程序代码（代码段）
- 程序计数器 + 寄存器（体现“执行位置”）
- 栈（函数调用、局部变量）
- 数据段（全局变量）

✅ 考试常见问法：  
> 程序和进程的区别是什么？  
> 答：程序是静态文件，进程是动态执行实体；一个程序可对应多个进程。

## 二、进程状态（第二、三张图）

这是**操作系统中最重要的基础图之一**。

### 五大状态（第二张图）

| 状态 | 含义 |
|------|------|
| new | 进程正在被创建 |
| ready | 可以运行，等待 CPU |
| running | 正在 CPU 上执行 |
| waiting | 等待某事件（如 I/O） |
| terminated | 已结束，等待回收 |

### 状态转换图（第三张图）

```
   new
    │ admitted
    ▼
  ready ── scheduler dispatch ──▶ running
    ▲                              │
    │                              │ interrupt / timeout
    │ I/O or event completion      ▼
    │                            waiting
    │                              │
    └──── I/O or event wait ◀──────┘
                                   │ exit
                                   ▼
                               terminated
```

✅ 你务必记住：
- `ready → running`：调度
- `running → ready`：时间片用完或被抢占
- `running → waiting`：等待 I/O
- `waiting → ready`：I/O 完成

## 三、PCB（进程控制块）（第四、五张图）

> **PCB 是操作系统中描述一个进程的唯一数据结构**

### PCB 包含的信息（第四、五张图综合）

| 类别 | 具体内容 |
|------|----------|
| 进程标识 | 进程号（PID） |
| 状态 | new / ready / running … |
| 程序计数器 | 下一条指令地址 |
| CPU 寄存器 | 恢复进程时必须 |
| 调度信息 | 优先级、时间片 |
| 内存信息 | 基址、界限、页表 |
| I/O 状态 | 打开的文件列表 |
| 记账信息 | CPU 使用时间等 |

✅ 重点理解：
- **保存/恢复寄存器**是实现“进程切换”的关键
- Linux 中的 PCB 对应 `task_struct`（在 `sched.h` 中）
