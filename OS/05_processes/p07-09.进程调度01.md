## 一、操作系统中的三种队列（第 1 张图核心内容）

| 队列 | 包含哪些进程 | 对应状态 |
|------|--------------|----------|
| **Job queue** | 系统中所有进程 | new + ready + running + waiting + terminated |
| **Ready queue** | 准备好了，只等 CPU | **ready** |
| **Device queue** | 正在等某个 I/O 设备 | **waiting** |

> 每个 I/O 设备（磁盘、磁带、终端）都有自己的 device queue

✅ 关键点：**一个进程的一生会在这些队列之间迁移**

---

## 二、队列与状态转换的关系（非常重要）

你之前学的状态转换：
- `ready → running`
- `running → waiting`
- `waiting → ready`

现在用**队列**来理解：

```
                     ┌─────────────────┐
                     │                 │
    new ──▶ ready queue ──▶ CPU ──┬──▶ terminated
                     │            │
                     │            └──▶ device queue（等I/O）
                     │                      │
                     └──────────────────────┘
                              I/O完成
```

- **CPU**：不是队列，是一个资源
- **Ready queue**：存所有 ready 状态的进程
- **Device queue**：存所有 waiting 状态的进程（每个设备一个）

---

## 三、第 2 张图（队列 + PCB 示意图）

这张图画的是操作系统内部的数据结构：

- **Ready queue**：通过指针把多个 PCB 串起来
- **每个 device queue**：也是 PCB 链表（比如磁带 0、磁带 1、磁盘 0）

> 实际实现中，队列就是 **PCB 的链表**  
> 队头指针 + 队尾指针 + 每个 PCB 里有“下一个 PCB”的指针

✅ 理解这个对后续学**调度算法**很有帮助

---

## 四、第 3 张图（调度流程图）

这张图展示了**进程在不同队列/资源之间的流动**：

```
         ready queue
              │
              ▼ (调度)
             CPU
              │
    ┌─────────┼─────────┐
    │         │         │
    ▼         ▼         ▼
 I/O请求   时间片用完   fork子进程
    │         │         │
    ▼         ▼         ▼
 I/O队列  回到ready   子进程执行
    │
    └──(I/O完成)──▶ ready
```

**图中关键动作：**

| 事件 | 结果 |
|------|------|
| I/O request | CPU → device queue（`running → waiting`） |
| I/O 完成 | device queue → ready queue（`waiting → ready`） |
| time slice expired | CPU → ready queue（`running → ready`） |
| fork a child | 父进程可能等待或继续 |
| interrupt occurs | 当前进程被放回 ready |

## 五、帮你把“状态”和“队列”对应死记

| 状态 | 在哪个队列/位置 |
|------|----------------|
| new | 不在队列（正在被创建） |
| ready | **ready queue** |
| running | **CPU**（不在任何队列） |
| waiting | **某个 device queue** |
| terminated | 不在队列（已销毁） |

✅ 考试常考：  
> 一个 waiting 状态的进程在哪个队列？  
> 答：在对应的 device queue 里。

---

## 六、一句话总结本章节

> **调度 = 从 ready queue 中选一个进程去 CPU 运行**  
> **I/O 管理 = 把进程从 CPU 移到 device queue，完成后再移回 ready queue**
