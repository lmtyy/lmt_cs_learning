## 1. Process concept（进程概念）

**进程是正在运行的程序实例**，它包含：
- **程序代码**（文本段）
- **当前活动**（程序计数器、寄存器）
- **数据**（全局变量、堆、栈）

> 区别：**程序是被动存储在磁盘上的代码**，**进程是主动执行中的实体**。

进程有状态：新建 → 就绪 → 运行 → 等待 → 终止。

---

## 2. Process scheduling（进程调度）

操作系统的调度器决定**哪个进程何时使用 CPU**。

- **调度队列**：就绪队列、等待队列（I/O 等）
- **调度器类型**：
  - 长期调度器（从磁盘选进程加入内存）
  - 短期调度器（选下一个运行进程）
  - 中期调度器（交换，用于内存管理）

常见调度算法（后面章节会细讲）：FCFS、SJF、优先级调度、时间片轮转等。

---

## 3. Process operations（进程操作）

基本操作包括：
- **创建进程**（`fork()` 或 `CreateProcess`）  
  父进程创建子进程，形成进程树。
- **终止进程**（`exit()`）  
  可自行结束或被其他进程强制结束（`kill`）。

例子（Linux）：
```c
pid_t pid = fork();
if (pid == 0) {
    // 子进程
} else {
    // 父进程
}
```

---

## 4. Process cooperating（进程协作）

**独立进程**：不影响其他进程，也不被影响。

**协作进程**：
- 共享数据、任务拆分（如管道、多线程下载）
- **好处**：信息共享、计算加速、模块化
- **问题**：可能发生**竞争条件**（race condition）

例如：两个进程同时修改一个文件 → 结果取决于执行顺序。

---

## 5. Inter-process communication（进程间通信，IPC）

协作进程需要交换数据。两种常见模型：

### 共享内存
- 两个进程直接读写同一块内存
- **快**，但要自己加锁（如信号量、互斥量）

### 消息传递
- 操作系统提供 `send()` / `receive()`
- **慢一些**，但自动同步、更安全
- 分两种：阻塞 / 非阻塞发送与接收

---

## 6. Communication in client-server systems（客户-服务器系统通信）

常见三种方式：

| 方式 | 特点 |
|------|------|
| **Socket** | 通过网络或本地（如 TCP、UDP），适合不同机器 |
| **RPC（远程过程调用）** | 像调用本地函数一样调用远程函数（参数需序列化） |
| **Pipe** | 单向字节流，常用于父子进程或 shell 命令（`grep a | wc -l`） |

> 补充：管道有匿名管道（只限有亲缘关系的进程）和命名管道（任意进程可用）。

---

## 总结（帮你考试 / 复习）

- **进程 = 运行中的程序**
- **调度**决定谁用 CPU
- **创建 / 终止**是基本操作
- **协作进程**需要同步，避免竞争条件
- **IPC**：共享内存（快但复杂） vs 消息传递（慢但简单）
- **客户-服务器**：Socket、RPC、管道
