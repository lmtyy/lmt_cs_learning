## 一、进程的四个基本操作（第 1 张图总览）

| 操作 | 含义 |
|------|------|
| **进程创建** | 产生一个新进程 |
| **进程执行** | 加载程序、设置属性、等待事件 |
| **进程终止** | 进程结束，释放资源 |
| **进程通信** | 共享内存 / IPC / 分布式通信 |

这些就是操作系统对进程的“生、跑、死、聊”。

## 二、进程创建的核心（第 3、4 张图）

### 2.1 创建时的三个决策

| 决策点 | 选项 |
|--------|------|
| **资源如何共享** | 1. 父子共享所有<br>2. 共享子集<br>3. 完全不共享 |
| **执行方式** | 1. 父子并发执行<br>2. 父进程等子进程结束 |
| **地址空间** | 1. 子进程是父进程的副本（clone）<br>2. 子进程加载新程序（exec） |

### 2.2 UNIX 的经典做法：fork + exec

这是**最重要**的部分，你给的代码完美展示了它。

| 系统调用 | 作用 |
|----------|------|
| **fork()** | 创建一个**几乎完全一样**的子进程（复制父进程的内存、寄存器、PCB） |
| **exec()** | 把当前进程的地址空间**替换成**一个新程序 |

> 💡 为什么分两步？  
> 因为这样可以在 exec 之前做各种准备工作（比如重定向输入输出、设置环境变量）。

## 三、逐行解读你的代码（第 5 张图）

```c
#include <stdio.h>
void main(int argc, char *argv[])
{
    int pid;
    
    /* fork another process */
    pid = fork();        // ① 创建子进程
    
    if (pid < 0) {       // ② fork 失败
        fprintf(stderr, "Fork Failed");
        exit(-1);
    }
    else if (pid == 0) { // ③ 子进程执行的代码
        execlp("/bin/ls", "ls", NULL);  // 子进程变成 ls 命令
    }
    else {               // ④ 父进程执行的代码
        wait(NULL);      // 父进程等子进程结束
        printf("Child Complete");
        exit(0);
    }
}
```

### 3.1 fork() 的返回值是理解的关键

| 返回值 | 含义 |
|--------|------|
| `-1` | 创建失败 |
| `0` | 我是子进程 |
| `>0` | 我是父进程，返回值是子进程的 PID |

✅ **同一个 fork() 调用，在两个进程中各返回一次**

### 3.2 执行流程

```
        父进程
          │
     fork() 调用
       │    │
       │    └──────► 子进程（pid == 0）
       │                 │
       │            execlp("/bin/ls")
       │                 │
       │            子进程变成 ls
       │                 │
       │            执行 ls，然后结束
       │                 │
   wait() ◄──────────────┘
       │
   printf("Child Complete")
       │
     exit(0)
```

## 四、进程终止

| 方式 | 说明 |
|------|------|
| **正常退出** | 调用 `exit()` 或 main 返回 |
| **异常退出** | 段错误、除以零等 |
| **被其他进程终止** | 调用 `kill()` |

父进程通常用 `wait()` 回收子进程的退出状态，避免子进程变成**僵尸进程**。

## 五、帮你总结“进程创建”考点

| 问题 | 答案 |
|------|------|
| UNIX 创建进程的两个核心调用 | `fork()` + `exec()` |
| fork() 返回 0 表示什么 | 当前是子进程 |
| fork() 返回正数表示什么 | 当前是父进程，返回值是子进程 PID |
| 父进程如何等子进程结束 | `wait()` |
| exec() 做了什么 | 替换当前进程的代码段、数据段 |

## 七、进阶问题（考题常见）

**问：** 下面这段代码会打印几个 "Hello"？
```c
fork();
printf("Hello\n");
```

**答：** 2 个。因为 fork 后有两个进程，每个都执行 printf。

**追问：** 如果是 `printf("Hello ");`（没有 `\n`）呢？  
**答：** 可能 2 个或更多，因为缓冲区被复制了。这是个经典陷阱题。
