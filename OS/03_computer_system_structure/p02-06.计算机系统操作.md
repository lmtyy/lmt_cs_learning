### 幻灯片1：物理架构图（硬件的“骨架”）
这张图展示了现代计算机最基础的**硬件拓扑结构**。
*   **System bus (系统总线)**：这是所有组件共享的“高速公路”。
*   **CPU**：大脑。
*   **Device controllers (设备控制器)**：每个外设（磁盘、打印机、磁带机）都由一个专门的控制器来管理。控制器里有一个**本地缓存 (Local buffer)**。
*   **Memory controller & memory**：内存控制器负责管理主存。

**重点理解**：CPU 和 I/O 设备（通过它们的控制器）是**分离**但**相连**的。它们竞争总线，但可以并行工作。

---

### 幻灯片2：基本原理与并发（如何工作？）
这是对上一张图的逻辑解释。
*   **CPU 和 I/O 设备可以并发执行 (Concurrently)**：比如你在打字（CPU执行指令）的同时，硬盘在后台默默写入数据。
*   **CPU的职责**：取指令、执行指令，以及在**主存**和**控制器本地缓存**之间搬运数据。
*   **设备控制器的职责**：负责处理具体的物理设备，处理完毕后通过发送 **Interrupt (中断)** 来通知 CPU。

---

### 幻灯片3：中断驱动的本质（核心机制）
这张幻灯片解释了操作系统为什么被称为**“中断驱动 (Interrupt driven)”**。
*   **中断 (Interrupt)**：硬件设备通知 CPU “我活干完了”或“我有事儿找你”的方式。
*   **中断向量 (Interrupt Vector)**：CPU 接收中断后，需要一个“电话号码本”来查找去执行哪段代码。这个“电话号码本”就是中断向量，里面存着所有**中断服务程序 (Interrupt Service Routine, ISR)** 的入口地址。
*   **中断禁止 (Incoming interrupts are disabled)**：CPU 在服务一个中断的过程中，通常会暂时**关闭**对其他中断的响应，防止新来的打断旧的处理，导致数据丢失。
*   **陷阱 (Trap)**：这是一种由**软件**生成的中断，通常由错误（如除零、缺页）或系统调用（用户程序请求操作系统服务）引起。

---

### 幻灯片4：中断处理（操作系统的动作）
当 CPU 收到中断信号后，操作系统在底层具体执行了哪些动作？这里给出了标准流程：
1.  **保存现场 (Preserves the state)**：必须立即保存当前的 CPU 寄存器状态和**程序计数器 (PC)**。这样中断结束后，被中断的程序还能从断点接续执行。
2.  **确定中断类型**：查表判断是哪个设备发来的中断。
3.  **执行代码**：执行对应的中断服务代码。
4.  **恢复现场**：返回之前保存的状态，恢复被中断的程序。

---

### 幻灯片5：中断时间线（动态演示）
这是最直观的一张图，用于演示**单进程执行 I/O 时的“时间”流转**。
*   **X轴是时间，Y轴是状态**。
*   **CPU的动作**：用户程序执行 -> 发出 I/O 请求 -> **CPU 暂停此进程（或切换到其他进程）** -> 收到中断 -> 执行中断处理程序 -> 恢复用户程序。
*   **I/O 设备的动作**：空闲 -> 接收到请求开始传输 (transferring) -> 传输完成 (transfer done) -> 发送中断 -> 恢复空闲。
*   **关键结论**：CPU 不是傻傻地等待 I/O 设备完成工作，而是在发出请求后，利用设备工作的间隙去执行其他指令。只有当设备 **“done”** 了，CPU 才被迫打断当前工作去处理数据。

### 💡 考试/学习锦囊
这几张幻灯片构成了考试中常见的高频考点，请务必掌握以下问题：

1.  **硬盘读一个数据块，这过程中谁向谁发送中断？**
    *   **答案**：磁盘控制器 -> 发中断 -> CPU。
2.  **中断和陷阱的区别？**
    *   **答案**：中断是硬件发起的（外部事件），陷阱是软件发起的（内部异常/系统调用）。
3.  **中断处理流程的第一步是什么？**
    *   **答案**：保存 CPU 状态（寄存器、PC）。
4.  **图5的启示是什么？**
    *   **答案**：解释了为什么操作系统要设计**多道程序设计 (Multiprogramming)**。如果CPU在I/O期间硬等着，就是巨大的资源浪费。

**一句话串联：**
计算机的硬件架构（图1）支持CPU和设备并行（图2），通过中断机制（图3）实现及时响应。操作系统负责在中断发生时保存现场、处理事务、恢复现场（图4），最终形成了我们看到的错落有致的执行时间线（图5）。
