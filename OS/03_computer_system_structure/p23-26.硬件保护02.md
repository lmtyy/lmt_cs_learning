这四张幻灯片深入讲解了“硬件保护”中最重要的两个领域：**内存保护 (Memory Protection)** 和 **CPU保护 (CPU Protection)**。

它们是理解操作系统如何同时运行多个程序、防止恶意或错误程序破坏系统的关键。

---

### 幻灯片1：内存保护的概念与寄存器
这张幻灯片定义了内存保护的核心机制。

*   **保护对象**：操作系统必须保护 **中断向量 (Interrupt vector)** 和 **中断服务程序 (Interrupt service routines)**，因为它们是系统控制的核心。如果用户程序能修改它们，就可以劫持系统。
*   **实现方法**：硬件增加两个特殊的寄存器：
    1.  **基址寄存器 (Base register)**：存放该进程在物理内存中合法的**最小物理地址**。
    2.  **界限寄存器 (Limit register)**：存放该进程合法地址范围的**大小（长度）**。
*   **结论**：任何试图访问基址+界限范围之外的内存地址都是非法的。

---

### 幻灯片2：基址与界限寄存器的使用（图示）
这张图展示了实际分配情况。
*   **物理内存布局**：图中是一块从 `0` 到 `1024000` 的连续物理内存。
    *   操作系统（Monitor）占用低地址区域。
    *   多个用户作业（Job 1, Job 2, Job 3, Job 4）被依次加载到内存中。
*   **示例（Job 2）**：
    *   **基址寄存器 (Base)** = `300040`。这意味着 Job 2 被分配在了物理地址 `300040`。
    *   **界限寄存器 (Limit)** = `120900`。这意味着 Job 2 的长度是 `120900` 个字节。
    *   **合法范围**：从 `300040` 到 `300040 + 120900 = 420940`。
*   **为何这样设计？** 当一个程序（比如 Job 2）运行时，操作系统会把它的基址和界限值加载到 CPU 的这两个寄存器中。硬件会自动检查每一个访问地址是否落在 `[Base, Base+Limit)` 之间。

### 幻灯片3：硬件地址保护（流程图）
这张图揭示了硬件层面的**执行逻辑**。
*   **特权指令**：**加载基址和界限寄存器的指令是特权指令**。这意味着只有操作系统（在Monitor mode）才能设置这些值。用户程序无法修改它们来欺骗硬件。
*   **检查流程**：
    1.  CPU 生成一个虚拟/逻辑地址 `address`（由用户程序发出）。
    2.  **检查一**：`address >= base` 吗？
        *   如果不满足（`No`）：说明试图访问比基址还低的内存地址（比如试图访问操作系统区域）。触发陷阱（Trap）。
    3.  **检查二**：`address < base + limit` 吗？
        *   如果不满足（`No`）：说明试图访问超出自己范围的内存地址。触发陷阱。
    4.  如果两个检查都通过（`Yes`）：硬件允许访问该内存。
*   **结果**：一旦越界，硬件会自动触发一个**陷阱 (Trap)**，操作系统会接管并杀死这个非法进程，保护其他进程和操作系统本身。

---

### 幻灯片4：CPU保护（计时器）
这张幻灯片介绍了防止单一程序霸占 CPU 的机制。

*   **问题**：如果没有 CPU 保护，一个用户程序可能进入无限循环（死循环），导致操作系统和其他程序永远无法运行。
*   **解决方案**：**计时器 (Timer)**。
    *   **工作原理**：
        1.  操作系统在将 CPU 交给用户程序之前，**设置一个计时器**（比如 100 毫秒）。
        2.  计时器每个时钟滴答递减。
        3.  **当计时器减到 0 时，硬件会自动触发一个中断 (Timer Interrupt)**。
    *   **效果**：操作系统收到计时器中断，重新获得控制权，就可以暂停当前运行超时的进程，切换到另一个进程。这就实现了 **分时系统 (Time Sharing)**。
*   **关键点**：**加载计时器 (Load-timer) 是一个特权指令**。用户程序无法修改或禁用计时器，也无法关闭中断。这确保了操作系统永远不会失去对 CPU 的控制。

---

### 💡 考试与核心总结

这部分内容构建了现代操作系统的 **“监狱系统”**：

1.  **内存保护**（基址+界限寄存器）：
    *   确保每个程序在物理内存中的**领土边界**。
    *   越界访问 = **致命错误**。
    *   **特权**：操作系统负责设置边界。

2.  **CPU 保护**（计时器）：
    *   确保每个程序的 **“刑期”** 是有限的。
    *   超时 = **定时炸弹**（中断）。
    *   **特权**：操作系统负责设置倒计时。

**考点速记：**

| 保护类型 | 核心硬件 | 越界后果 | 设置权限 |
| :--- | :--- | :--- | :--- |
| **内存保护** | Base & Limit Registers | **Trap** (地址错误) | 特权指令 (OS设置) |
| **CPU保护** | Timer | **Interrupt** (时间片耗尽) | 特权指令 (OS设置) |

**一句话串联**：
双重模式（Kernel/User）构建了权力架构；**内存保护**锁住了程序的活动范围；**CPU保护**限制了程序的活动时间。这三者共同协作，确保了即使有一个恶意或包含错误的用户程序在运行，操作系统也能掌控全局、保护其他进程并维持系统的稳定运行。
