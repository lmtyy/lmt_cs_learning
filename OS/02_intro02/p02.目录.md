## 1. Operating Systems Structure（操作系统结构）

操作系统结构指的是 OS 内部各模块的组织方式。常见结构有：

- **简单结构**：早期 OS（如 MS-DOS）没有清晰模块划分，内核与应用混在一起，易崩溃。
- **分层结构**（layered approach）：每层只调用更低层提供的功能，最底层是硬件，最上层是用户接口。如 THE 操作系统。
- **微内核**（microkernel）：把文件系统、设备驱动等移到用户态，内核只保留 IPC、内存管理等基本功能，如 QNX、L4。
- **宏内核**（monolithic kernel）：所有系统服务都在内核里，效率高但模块耦合紧，如 Linux（虽是宏内核但可加载模块）。
- **混合内核**：结合宏内核的性能与微内核的部分模块化，如 Windows NT。

## 2. History of Operating Systems（操作系统历史）

你课件里指出 **“really a history of resource-driven choices”**（本质是资源驱动选择的历史）——这一点很精辟。早期计算机昂贵、资源有限，OS 设计围绕如何高效利用 CPU、内存、I/O 等资源。

关键发展阶段：

1. **1940s-1950s：无 OS**，程序员直接操作硬件，人工插拔线路。
2. **1950s-60s：单道批处理系统**，一个作业接着一个，CPU 空闲浪费。
3. **1960s：多道批处理**、**分时系统**（如 CTSS）、**实时系统**。资源复用提升利用率，同时开始关注交互性。
4. **1970s：Unix** 出现（C 语言编写，可移植、简洁）。由于 PDP-11 等小型机资源比大型机少，Unix 采用了简洁的多用户分时模型。
5. **1980s：个人电脑时代**，资源（CPU、内存）依然有限，早期 DOS 是单用户单任务，后出现 Windows 及 Mac OS（图形界面，更关注用户体验而非硬件效率）。
6. **1990s 以后：网络资源驱动**（Web、分布式 OS）、移动设备资源（低功耗、有限算力）驱动的嵌入式 OS（如 Android、iOS）。

每次资源瓶颈的变化（CPU 慢→引入多道；内存小→引入虚拟内存；网络带宽/延迟→引入异步 I/O 和网络栈优化）都推动 OS 设计变革。

## 3. Operating Systems Organizations（操作系统组织方式）

组织方式可以理解为 OS 如何构造自身并向上提供接口：

- **单体内核**：所有服务在内核模式运行，系统调用进入内核执行。
- **微内核**：最小化内核，服务用进程间通信。
- **外核**（exokernel）：只负责资源复用与保护，让应用程序直接管理物理资源。
- **虚拟机监视器**（Hypervisor）：硬件之上的薄层，多 OS 共享物理资源。

另外也可以从**进程/内存/文件/设备管理**等子系统如何组织来理解。  

常见分类对应不同资源需求：
- 服务器 OS（Linux/Windows Server）→ 宏内核或混合，侧重吞吐与并发。
- 移动 OS（iOS/Android）→ 宏内核（修改版）+ 电源管理，侧重资源受限。
- 实时 OS（FreeRTOS）→ 微内核或精简宏内核，侧重可预测延迟。

## 4. Abstractions and Layering（抽象与分层）

这是 OS 设计的核心原则。

### 分层（Layering）
- OS 从上到下分为多个逻辑层，每层只依赖下层提供的抽象。
- 典型层次（由高到低）：
  - 用户程序 / 命令行 / GUI  
  - 系统调用接口  
  - 文件系统 / 进程管理 / 内存管理 / 设备驱动  
  - 中断处理 / CPU 调度 / 硬件接口  
  - 硬件

优点：模块化，隔离变化；  
缺点：层间调用可能有性能开销（但现代硬件下可接受）。

### 抽象（Abstraction）
OS 为复杂的硬件提供简洁、统一的接口，隐藏底层资源细节。  
例子：

| 物理资源 | OS 提供的抽象 |
|---------|--------------|
| CPU      | 进程（或线程） |
| 内存     | 地址空间（虚拟内存） |
| 磁盘     | 文件（通过文件系统） |
| 网卡     | 网络 socket / 流 |
| 终端/显示器 | 标准输入输出 / 图形窗口 |

为什么需要抽象？  
- 简化用户编程（不需要直接操作磁盘扇区）。  
- 资源复用与保护（多个进程共享 CPU 且互不影响）。  
- 硬件多样性（不同硬盘都可以用“读写文件”访问）。

---

结合你的课件，可以理解为：

> OS 随着资源变化（硬件更便宜、更快或更受限）演进其结构。现代 OS 利用分层组织，并在每一层提供抽象，以隐藏硬件复杂性、管理共享资源。
