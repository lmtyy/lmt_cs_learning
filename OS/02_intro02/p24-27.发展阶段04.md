这几张图讲的是 **Phase 4（1988 年起）：分布式系统和互联网时代**。  
这是操作系统范围从“单机”扩展到“网络”的阶段，OS 设计必须考虑通信、资源共享、安全等新维度。

---

## 1. Phase 4: 分布式系统（1988–）

### 驱动力：局域网（LAN）
- 不同机器可以共享资源：**打印机、文件服务器、Web 服务器**
- 模型：**客户端 – 服务器**（Client–Server Model）

### 共享的服务：
- **计算服务**（如远程执行任务）
- **文件存储服务**（如网络文件系统）

> 这时 OS 不再只管自己这台机器，还要参与网络协调。  
> 典型技术：NFS（Network File System）、AFS（Andrew File System）

---

## 2. Phase 4: 互联网（1988–）

### 特点：
- 由**研究社区**开发（而非单一公司）
- 基于**开放标准**：IP 协议（Internet Protocol）
- 由 **IETF（互联网工程任务组）** 负责标准化

### 对 OS 的意义：
- 互联网协议栈（TCP/IP）成为 OS 网络子系统的核心
- 几乎所有现代 OS 都内置 TCP/IP 栈

### 互联网提供的服务类型（按时间发展）：

| 年代 | 服务类型 | 例子 |
|------|----------|------|
| 1970s | 远程计算访问 | telnet |
| 1980s | 数据/文件共享 | FTP, NFS, AFS, USENET |
| 1980s | 人际通信 | email |
| 1990s | 即时通信、万维网 | 聊天室、WWW |
| 1990s–00s | 多媒体、文件共享 | 音频、视频、P2P 文件共享 |

> 注意：很多服务最初是“在 OS 之上的应用”，但最终影响 OS 设计（如网络协议栈优化、安全模型）。

---

## 3. 网络“云”（Network “Cloud”）

这张图是一个抽象概念图，大概画了：
- 云的外部设备
- 处理器
- 本地存储器

### 实际含义：
- “云” 代表**由网络连接的多个计算机**，对外表现为一个整体
- 用户不在意数据存在哪台机器、计算在哪台机器执行
- 这是现代**云计算**的前身（尽管当时还不这么叫）

> 对 OS 的挑战：**分布式透明性**（用户感觉像在用一台机器，实际背后是多台）

---

## 4. Regional Nets + Backbone（区域网络 + 骨干网）

这张图说明了互联网的层次结构：

```
骨干网（Backbone）
   ↑
区域网（Regional Net）←→ 区域网
   ↑            ↑
  LAN          LAN
   ↑
 终端/PC
```

### 概念：
- **LAN（局域网）**：一个房间、一栋楼内的网络（如以太网）
- **区域网**：多个 LAN 连接成更大的区域网络
- **骨干网**：高速连接各区域网络，构成互联网的主干

> OS 需要支持从 LAN 到 Internet 的不同层级网络配置。

---

## Phase 4 对 OS 的核心影响

| 维度 | 变化 |
|------|------|
| 资源范围 | 从“单机资源”扩展到“网络资源” |
| 新抽象 | socket、文件句柄（可指远程文件）、RPC |
| 新挑战 | 网络延迟、分区、安全、认证、并发访问 |
| OS 模块 | TCP/IP 栈成为核心组件 |
| 分布式服务 | NFS、AFS、DNS 等成为 OS 或基础服务 |

---

## 与前面阶段的对比

| 阶段 | 核心资源 | OS 主要任务 |
|------|----------|-------------|
| Phase 1 | 昂贵 CPU | 批处理，最大化吞吐 |
| Phase 2 | 终端 + CPU | 分时，响应时间 |
| Phase 3 | 个人机器 | GUI，人的效率 |
| Phase 4 | 网络连接 | 资源共享，通信，分布式透明 |
