这张课件讲的是**操作系统历史的 Phase 2（1970–1985）**，核心转变是：**硬件变便宜，人力变贵**，因此设计目标从“最大化硬件利用率”转向“提高用户体验和响应速度”。

---

## 时代背景

| 对比项 | Phase 1 (1948–1970) | Phase 2 (1970–1985) |
|--------|---------------------|----------------------|
| 硬件成本 | 数百万美元 | 数万美元 |
| 人力成本 | 便宜 | 昂贵 |
| 优化目标 | 最大化硬件利用率 | 最小化用户等待时间 |
| 用户交互 | 无（批处理） | 有（交互式分时） |
| 失败后果 | 作业丢失 | 响应慢、系统颠簸 |

> 关键转折：计算机从“稀有资源”变成“可负担的工具”，人的时间变成更贵的资源。

---

## 核心技术：交互式分时（Interactive Timesharing）

### 怎么做：
- 使用廉价终端（约 $1000/台）
- 多个用户同时登录同一台计算机
- CPU 在用户之间快速切换（时间片轮转）
- 每个用户感觉自己是唯一使用者

### 代价：
- **牺牲 CPU 吞吐量**换取更好的响应时间
- 单个任务完成得更慢（因为 CPU 被切分），但用户等待响应的时间更短

### 用户能做什么（第一次在线）：
- 调试程序（而不是提交批处理等几小时）
- 编辑文档
- 发邮件

> 这是现代“在线”工作方式的起源。

---

## 新问题：颠簸（Thrashing）

### 表现：
- 系统负载轻微增加 → 响应时间急剧恶化
- 性能不是线性下降，而是**悬崖式崩溃**

### 为什么会颠簸（此处提及）：
- **Swapping**：内存不足时，系统把进程换出到磁盘。如果内存太小，系统会花大量时间换入换出，而不是做实际工作。
- **Queueing**：请求排队太长（CPU 队列、磁盘队列），导致响应时间爆炸（排队论中的非线性增长）。

### 简单理解：
```
轻负载：每个请求立即处理
中等负载：需要排队，但可忍受
重负载（颠簸）：大部分时间花在换页/排队/调度上，几乎不做有用功
```

> 这是分时系统第一次遇到“资源过度竞争”问题，后来用**虚拟内存 + 工作集模型 + 负载控制**来缓解。

---

## OS 技术的成熟与稳定

Phase 2 期间（1970–1985）：
- **UNIX** 诞生并传播（1970s 初）
- **虚拟内存** 普遍化
- **文件系统** 成熟（层次结构、权限）
- **进程间通信（IPC）** 成为标准组件
- 操作系统不再是实验品，而成为产品（商业 UNIX、BSD）

---

## 与 Phase 1 的直接对比

| 目标 | Phase 1 (批处理) | Phase 2 (分时) |
|------|------------------|----------------|
| 吞吐量 | 高（最大化作业数/小时） | 可接受（不是首要） |
| 响应时间 | 几小时/天 | 秒级或更短 |
| 用户交互 | 无 | 有（终端） |
| 失败模式 | 系统崩溃（无保护） | 颠簸（性能崩溃） |
| 硬件使用 | 接近 100% CPU 利用率 | CPU 有时空闲（等待用户输入） |

> 注意：Phase 2 并没有完全放弃批处理——大型机仍运行批处理作业，但**分时成为主导模式**。

---

## 总结：Phase 2 的核心贡献

1. **交互式计算成为主流**：人不再排队等机器，而是机器为人服务。
2. **响应时间成为核心指标**：从“每秒钟执行多少指令”转向“用户按下回车后多久看到结果”。
3. **发现新问题（颠簸）**：资源的非线性失效模式，推动内存管理和调度理论的发展。
4. **OS 技术成熟**：分时系统需要的所有组件（调度、虚拟内存、文件系统、终端 I/O）基本定型。

> 一句话总结 Phase 2：**“硬件足够便宜，可以让机器等人，而不是人等机器。”**  
> 代价是：当太多人同时等机器时，系统会颠簸崩溃——这也启发了后续的负载控制和服务质量（QoS）研究。
