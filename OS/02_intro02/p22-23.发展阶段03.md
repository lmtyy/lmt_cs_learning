这两张课件图讲的是 **Phase 3（1981 年起）—— 硬件极便宜、人力极贵**，以及 **GUI 的发展**。  
这是个人电脑（PC）兴起的时代，OS 经历了“简化 → 再次复杂化”的轮回。

---

## 1. Phase 3 核心逻辑：人力比硬件贵得多

### 数据对比：
- 计算机成本：约 $1,000  
- 程序员年薪：约 $100,000

### 推论：
> 如果一台计算机能让程序员效率提高 1%，一年就省下 $1,000 —— 正好够买这台电脑。  
> 所以**用计算机帮人提高效率是划算的**，哪怕硬件利用率不高。

这与 Phase 1（最大化硬件利用）完全相反。

---

## 2. 个人计算（Personal Computing）

- 计算机便宜 → 给每个人一台 PC
- 不再共享机器，一个人独占所有资源

### 初期：硬件资源仍然有限
- PC 的 CPU 慢、内存小（早期 640KB 限制）
- OS 退化成**子程序库**（subroutine library）
- 一次只运行一个应用程序（MS-DOS, CP/M）

> 这正是你前面看到的 MS-DOS 结构：单任务，简单分层，无内存保护。

---

## 3. 后来 PC 变得强大

- CPU、内存、磁盘容量快速增长
- **OS 重新获得大型机的复杂性**：
  - 多道程序（multiprogramming）
  - 内存保护
  - 虚拟内存
  - 抢占式多任务

### 代表系统：
- **Windows NT**（1993）：微软的全新内核，支持保护、多用户（设计上）、多任务
- **OS/2**（IBM 与微软合作，后分裂）

> 有趣的是：PC OS 先**简化**（从大型机到单任务 DOS），然后随着硬件变强**再次复杂化**（回到类似 UNIX 的能力）。

---

## 4. 一个关键问题（留作思考）

> **As hardware gets cheaper does need for OS go away?**  
> 硬件越来越便宜，还需要 OS 吗？

课件没有直接回答，但潜台词是：
- 如果硬件无限便宜，是否可以不用 OS，每个程序直接占满机器？
- 但现实中：OS 不止是“资源复用器”，还提供**抽象、隔离、安全、便捷**。即使硬件便宜，这些需求仍在。

（这个问题后面可能会讨论，比如 exokernel、unikernel 的思路。）

## 5. Phase 3 续：图形用户界面（GUI）

### GUI 发展时间线（课件列举）：

| 年份 | 系统 | 关键点 |
|------|------|--------|
| 1981 | Xerox Star | 原型 Alto 研究项目，第一个“鼠标+窗口” |
| 1984 | Apple Lisa/Macintosh | 商业化成功，“Look and Feel”诉讼（1988） |
| 1985 | Windows 1.0 | 微软进入 GUI 市场 |
| 1990 | Windows 3.1 | 真正流行 |
| 1993 | Windows NT | 全新内核，保护模式，HAL |
| 1995 | Windows 95 | 集成 GUI + DOS 基础 |
| 2000 | Windows 2000 | NT 路线消费化 |
| 2001 | Windows XP | 经典 |
| 2007 | Windows Vista | … |

### 底部注释：
- **Single Level** / **HAL/Protection** / **No HAL/Full Prot**  
  暗示 Windows 版本在底层架构上的差异：  
  - Win 9x（95/98/Me）：无硬件抽象层（HAL），保护弱  
  - Win NT 系列（NT/2000/XP/...）：有 HAL，完整保护

## Phase 3 总结

| 特征 | 内容 |
|------|------|
| 时间 | 1981 年起 |
| 经济逻辑 | 硬件极便宜，人力极贵 → 用计算机提升人的效率 |
| 硬件模式 | 个人电脑（每人一台） |
| OS 初期 | 子程序库，单任务（MS-DOS） |
| OS 后期 | 重新获得大型机的复杂功能（NT, OS/2） |
| UI 革命 | GUI 成为标准（Xerox → Apple → Microsoft） |
| 悬置问题 | 硬件便宜后是否还需要 OS？ |
