## 1. History Phase 5 (1995—): Mobile Systems（移动系统）

### 驱动力：无处不在的移动设备
- 笔记本电脑、PDA、手机
- **小、便携、便宜**
- 趋势：**每人拥有多台计算机**（与早期“多用户共享一台”完全相反）

### 资源限制：
- 内存小、CPU 弱（早期）、**功耗受限**（电池）
- 这导致 OS 需要特别关注：省电调度、后台限制、快速休眠/唤醒

### 无线/广域组网：
- 利用现有基础设施（蜂窝网络、WiFi）
- 巨大的分布式资源池（云）扩展了设备能力
- 传统计算机被“拆分”：
  - 无线键盘/鼠标
  - CPU 可能在云端
  - 存储在远程

> 对 OS 的影响：**分布式不再可选，而是默认**。一个“应用”可能涉及本地 UI + 远程计算 + 云存储。

### 对等系统（Peer-to-Peer）
- 许多设备平等协作，没有中心服务器
- OS 的“组件”可能分布在全球
- 例子：BitTorrent、区块链、某些协作编辑系统

## 2. Datacenter is the Computer（数据中心就是计算机）

这张图引用 Google 的 Luiz Barroso：

> Google 程序 = Web 搜索、Gmail ……  
> Google 计算机 = **成千上万台计算机 + 网络 + 存储**（仓库级设施）

### 核心观点：
- 对于云服务来说，**单一 PC 或服务器不再是“计算机”**
- 真正的计算机是**整个数据中心**
- OS 概念需要向上抽象：从单机 OS → 数据中心 OS（如 Kubernetes、Borg、MapReduce 等）

> 现在虽然不常见，但未来会更普遍。

## 3. Migration of Operating-System Concepts and Features（OS 概念与特征的迁移）

这是教材/课件里的经典总结图。我帮你按**行（平台）**和**列（年代）**解读：

### 行表示不同平台（按出现时间）：
1. **mainframes**（大型机）—— 最早
2. **minicomputers**（小型机）
3. **desktop computers**（桌面计算机）
4. **handheld computers**（手持计算机）—— 最晚

### 列表示年代（1950–2000）
每个单元格里是该年代该平台的主要 OS 特征。

### 关键模式（从上到下看）：
- 新技术先出现在**大型机**，然后逐步向下迁移到小型机、桌面、手持设备
- 例如：
  - **time shared**（分时）：1960 年代大型机 → 1970 年代小型机 → 1980 年代桌面
  - **multiuser**（多用户）：1970 大型机 → 1980 小型机 → 1990 桌面
  - **networked**（网络化）：1980 大型机 → 1990 小型机 → 2000 桌面/手持

### 底部画的关系图（网状）：
- 展示了各个里程碑之间的关系
- MULTICS → UNIX → 各种系统
- 分时、多用户、网络、分布式、容错、多处理器 等概念相互关联

### 结论：
> OS 的概念和技术**从上（大型机）向下（手持设备）迁移**，只是**时间滞后**。  
> 今天手机上用的虚拟内存、多任务、保护模式，都是 1960-70 年代大型机上发明的。

## Phase 5 总结

| 特征 | 内容 |
|------|------|
| 时间 | 1995 年起 |
| 设备 | 移动设备（每人多台） |
| 新约束 | 功耗、无线、有限本地资源 |
| OS 新任务 | 省电、网络透明、分布协作 |
| 终极形态 | 数据中心作为计算机 |
| 技术迁移 | 大型机概念 → 手持设备 |

## 全历史阶段回顾（Phase 1–5）

| 阶段 | 时间 | 稀缺资源 | 核心模式 | 典型系统 |
|------|------|----------|----------|----------|
| Phase 1 | 1948–1970 | CPU/内存 | 批处理 | 早期大型机 |
| Phase 1½ | late 60s–70s | I/O 延迟 | 多道程序 | OS/360 |
| Phase 2 | 1970–1985 | 人的时间 | 分时系统 | UNIX, Multics |
| Phase 3 | 1981– | 人的效率 | 个人计算 + GUI | MS-DOS, Mac, Windows |
| Phase 4 | 1988– | 网络资源 | 分布式/互联网 | NFS, WWW, TCP/IP |
| Phase 5 | 1995– | 功耗/移动性 | 移动 + 云 | iOS, Android, 数据中心 OS |

---

## 最终核心思想（贯穿所有 Phase）

> **操作系统的演进，始终是被“当前最稀缺的资源”驱动的。**  
> 硬件越来越强、越来越便宜，但人的时间、网络带宽、功耗、安全性相继成为新的稀缺资源，OS 因此不断重塑。
