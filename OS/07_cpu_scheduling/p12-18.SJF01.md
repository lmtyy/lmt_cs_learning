## 模块一：核心思想 & 两种模式（图1、图2）

### 图1：SJF的核心定义
*   **规则**：当CPU空闲时，选择**下一个CPU爆发时间（Next CPU Burst）最短**的进程。
*   **打破平局**：如果两个进程的爆发时间相同，用FCFS处理。
*   **关键点**：SJF是基于“**下一个**爆发”，而不是“总运行时间”。（这影响了预测机制，见模块三）

### 图2：非抢占 vs 抢占（SRTF）
SJF分为两种截然不同的模式，这也是考试计算题的核心区别：

| 模式 | 行为 | 英文简称 |
| :--- | :--- | :--- |
| **非抢占式 (Non-preemptive)** | 一旦开始运行，必须跑完当前CPU爆发。即使新来了一个更短的任务，也必须等当前任务结束。 | SJF |
| **抢占式 (Preemptive)** | 当新进程到达时，如果它的爆发时间 < **当前进程的剩余爆发时间**，立即抢占。 | **SRTF** (Shortest-Remaining-Time-First) |

> **⚠️ 重要结论（本页底部）**：SJF（无论是否抢占）在理论上能**给出最小的平均等待时间**。它是**最优的**（在最小化平均等待时间的目标下）。

## 模块二：具体计算演示（图3、图4）

这是考试中最可能考到的部分。我们来看同一个进程集合在不同模式下的表现。

**进程数据：**
*   \(P_1\): 到达0, 爆发7
*   \(P_2\): 到达2, 爆发4
*   \(P_3\): 到达4, 爆发1
*   \(P_4\): 到达5, 爆发4

### 图3：非抢占式SJF 计算
*   **过程**：
    1.  **\(0 \sim 7\)**：\(P_1\) 先到达，先运行7ms（非抢占，不会中断）。
    2.  **\(7\)时刻**：\(P_1\)结束。此时 \(P_2, P_3, P_4\) 都已到达。
    3.  **选择下一个**：爆发时间分别是 4, 1, 4。选最短的 \(P_3\)。
    4.  **\(7 \sim 8\)**：\(P_3\) 运行1ms，结束。
    5.  **\(8\)时刻**：剩下 \(P_2(4), P_4(4)\)，平局，按FCFS（\(P_2\)先到）处理。
    6.  **\(8 \sim 12\)**：\(P_2\) 运行。
    7.  **\(12 \sim 16\)**：\(P_4\) 运行。
*   **计算等待时间**：
    *   \(P_1\): 0 (不用等)
    *   \(P_2\): 到达2，运行8，等 \(8-2=6\)
    *   \(P_3\): 到达4，运行7，等 \(7-4=3\)
    *   \(P_4\): 到达5，运行12，等 \(12-5=7\)
*   **平均等待时间**：\((0+6+3+7)/4 = 4\) ms

### 图4：抢占式SJF (SRTF) 计算
*   **过程**（重点关注每次新进程到达时的检查）：
    1.  **\(0 \sim 2\)**：\(P_1\) 运行2ms（剩余5）。
    2.  **\(2\)时刻**：\(P_2\) 到达（爆发4）。当前 \(P_1\) 剩余5。\(4 < 5\)，**抢占**！\(P_1\)被挂起，\(P_2\)上。
    3.  **\(2 \sim 4\)**：\(P_2\) 运行2ms（剩余2）。
    4.  **\(4\)时刻**：\(P_3\) 到达（爆发1）。当前 \(P_2\) 剩余2。\(1 < 2\)，**抢占**！\(P_2\)被挂起，\(P_3\)上。
    5.  **\(4 \sim 5\)**：\(P_3\) 运行1ms，结束。
    6.  **\(5\)时刻**：\(P_4\) 到达（爆发4）。当前就绪队列有：\(P_1\)(剩5), \(P_2\)(剩2), \(P_4\)(4)。最短的是 \(P_2\)(剩2)。
    7.  **\(5 \sim 7\)**：\(P_2\) 运行完最后2ms，结束。
    8.  **\(7\)时刻**：就绪队列：\(P_1\)(剩5), \(P_4\)(4)。最短是 \(P_4\)。
    9.  **\(7 \sim 11\)**：\(P_4\) 运行4ms，结束。
    10. **\(11 \sim 16\)**：\(P_1\) 运行完最后5ms，结束。
*   **计算等待时间**：
    *   \(P_1\): 运行了(2+5=7ms)，完成时间16，等待 \(16-7=9\) ms
    *   \(P_2\): 到达2，结束7，等 \(7-2-4=1\) ms (实际等待了1ms)
    *   \(P_3\): 到达4，结束5，等 \(5-4=0\) ms
    *   \(P_4\): 到达5，结束11，等 \(11-5-4=2\) ms
*   **平均等待时间**：\((9+1+0+2)/4 = 3\) ms

### 💡 核心对比（图3 vs 图4）
| 指标 | 非抢占式 | 抢占式 (SRTF) |
| :--- | :--- | :--- |
| 平均等待时间 | 4ms | **3ms** (更优) |
| 对短进程响应 | 较慢 | **极快** (\(P_3\)立即响应) |
| 开销 | 低 (切换少) | 高 (频繁抢占切换) |

## 模块三：如何预测爆发时间？（图5、图6）

SJF有一个致命问题：**操作系统怎么知道下一个爆发是多长？** 通常采用**指数平均法（Exponential Averaging）**来预测。

### 图5：预测公式
*   **公式**：\(\tau_{n+1} = \alpha t_n + (1-\alpha)\tau_n\)
    *   \(\tau_{n+1}\)：下一次的预测值
    *   \(t_n\)：上一次的实际爆发值
    *   \(\tau_n\)：上一次的预测值
    *   \(\alpha\)：权重系数 (\(0 \le \alpha \le 1\))

### 图6：系数 \(\alpha\) 的影响
*   **\(\alpha = 0\)**：不看历史。\(\tau_{n+1} = \tau_n\)（预测值永远不变，忽略最近的实际行为）。
*   **\(\alpha = 1\)**：只看最近。\(\tau_{n+1} = t_n\)（完全相信上一次的实际爆发值，忽略所有历史）。
*   **公式展开**：\(\tau_{n+1} = \alpha t_n + (1-\alpha)\alpha t_{n-1} + (1-\alpha)^2\alpha t_{n-2} + ...\)
    *   **结论**：越早的历史数据，权重 \((1-\alpha)^k \alpha\) 越小。这符合直觉：**近期的爆发值比远期的更能反映当前行为。**

### 图7：实例图表
*   **实线 (\(t_n\))**：实际的CPU爆发时间（阶梯状波动）。
*   **虚线 (\(\tau_n\))**：预测的爆发时间（平滑曲线）。
*   **现象**：当实际值稳定时，预测值逼近；当实际值跳变（如从4跳到13），预测值会逐渐跟进，但不会瞬间完全跟上（因为 \(\alpha < 1\)）。这体现了“平滑”效果。

## 📝 总结：SJF 在全章中的位置

| 对比项 | FCFS | SJF (非抢占) | SRTF (抢占式) |
| :--- | :--- | :--- | :--- |
| **核心逻辑** | 先到先得 | 挑最短的跑完 | 谁剩得最少谁上 |
| **平均等待时间** | 可能很差 | 最优（非抢占） | **最优（理论最低）** |
| **饥饿问题** | 无 | **有** (长进程可能永远得不到CPU) | **更严重** |
| **实现难度** | 极简 | 难 (需要预测) | 更难 (需要预测+抢占) |

**学习建议**：
1.  务必掌握 **非抢占 vs 抢占 的甘特图绘制**。
2.  理解 **指数平均** 背后的逻辑：用“近大远小”的权重来预测，这是现代系统预估行为的经典方法。
3.  **SJF（特别是SRTF）是理论上最优算法**，但实际操作系统很少直接用纯SJF，因为“饥饿”和“预测不准”是硬伤。后续的**优先级调度**和**多级反馈队列**会解决这些问题。
