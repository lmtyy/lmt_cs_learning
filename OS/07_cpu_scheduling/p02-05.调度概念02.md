您观察得非常仔细！这是一个非常经典且容易混淆的知识点。

简单直接的答案是：**第2种情况（进程从运行→等待）虽然是调度触发点，但它不是“被强行赶走的”，而是进程自己主动放弃了CPU。**

让我们把这个概念掰开揉碎来看：

### 1. 核心定义：什么是“抢占”？

*   **抢占（Preemptive）** = **强行剥夺**。即：进程正在运行，还没用完时间片，也没有主动说“我让位”，但是操作系统**强行把CPU从它手里抢走**，交给别的进程。
*   **非抢占（Non-preemptive）** = **自愿让渡**。即：进程一旦获得CPU，除非它自己说“我不行了/我需要等个东西/我搞完了”，否则操作系统**绝对不会**动它。

### 2. 为什么“运行→等待”属于非抢占？

请看这两个场景的区别：

#### 场景A：你在写代码，突然想去上厕所
*   **动作**：你主动对经理（操作系统）说：“我不写了，我去上个厕所（I/O/等待）。”
*   **结果**：经理把工作排给别人。
*   **关键**：你是**主动**放弃CPU的，不是被经理强行拉走的。

#### 场景B：你在写代码，被经理强行拉走
*   **动作**：经理看了一眼表说：“你的工作时间到了，换人！”你还没写完，但被强行拉去休息室。
*   **结果**：经理把工作排给别人。
*   **关键**：你是**被动**被赶走的。

**结论：**

| 调度触发条件 | 性质 | 是否为抢占？ |
| :--- | :--- | :--- |
| **1. 进程终止** | 进程主动说“我搞完了” | **非抢占** (自愿) |
| **2. 运行→等待** (比如I/O) | 进程主动说“我去等会儿，你换人吧” | **非抢占** (自愿) |
| **3. 运行→就绪** (时间片用完) | 操作系统说“时间到，换人” | **抢占** (强制) |
| **4. 等待→就绪** (I/O完成) | 优先级变化，操作系统决定是否打断 | **抢占** (强制) |

---

### 3. 为什么您会误认为2是抢占？

因为教材上会把**条件1和条件2**列为“非抢占调度发生的**时机**”。

翻译一下就是：
*   **在非抢占调度模式下**：只有当发生**条件1**（进程终止）或**条件2**（进程主动请求I/O）时，调度器才会**有机会**介入工作。
*   其他任何时候（即使时间片用完了），调度器都不能插手。

**用大白话总结：**
*   **非抢占** = **只有这一种情况，调度器才能动**。
*   **抢占** = **除了这一种情况，调度器随时都能动**。

---

### 4. 一个小记忆技巧

请记住这一句话：
> **“非抢占”不是指“不能发生调度”，而是指“不能发生强制中断”。**

只要进程是**主动**离开CPU的（1.结束，2.等I/O），它就是非抢占的。
只要进程是**被动**离开CPU的（3.时间片用尽，4.高优先级来了），它就是抢占的。

**如果不理解这一点，后续看SJF（短作业优先）和RR（时间片轮转）算法时会非常痛苦。** 您能提出这个问题，说明已经抓住了调度的核心矛盾！
