这三张PPT完整地展示了 **FCFS（先来先服务）算法** 的核心知识点，包括**计算方法、致命缺陷和适用场景**。

既然你已经理解了“非抢占”的概念（上一轮讨论的），那么现在这张PPT就是该概念最生动的体现。

## 第一张图：最糟糕的情况（大车堵路）

这张图展示了FCFS最典型的糟糕场景：**护航效应（Convoy Effect）**。

*   **数据**：
    *   \(P_1\)：运行 24ms（大任务）
    *   \(P_2\)：运行 3ms（小任务）
    *   \(P_3\)：运行 3ms（小任务）
*   **到达顺序**：\(P_1\) 先来，然后 \(P_2\)，然后 \(P_3\)。
*   **Gantt Chart（甘特图）分析**：
    *   \(0 \sim 24\)ms：\(P_1\) 独占CPU。
    *   \(24 \sim 27\)ms：\(P_2\) 终于开始。
    *   \(27 \sim 30\)ms：\(P_3\) 终于开始。
*   **等待时间**：
    *   \(P_1\)：0（没等，直接上）
    *   \(P_2\)：24（等24ms）
    *   \(P_3\)：27（等27ms）
*   **平均等待时间**：**17ms**

### 💡 核心结论
**虽然 \(P_2, P_3\) 只需要3ms就能跑完，但它们被迫在队列里傻等了24ms。** 就像你在高速公路上被一辆占道的大卡车堵在后面一样。

## 第二张图：最好的情况（小任务先上）

为了对比，这张图展示了**如果大任务最后来**的情况。

*   **到达顺序**：\(P_2\) 先来，\(P_3\) 第二，\(P_1\) 最后。
*   **甘特图分析**：
    *   \(0 \sim 3\)ms：\(P_2\) 跑完。
    *   \(3 \sim 6\)ms：\(P_3\) 跑完。
    *   \(6 \sim 30\)ms：\(P_1\) 跑完（虽然后面还有很长的车，但不会堵住别人）。
*   **等待时间**：
    *   \(P_1\)：6（从第6ms才开始，等了前两个小任务）
    *   \(P_2\)：0
    *   \(P_3\)：3
*   **平均等待时间**：**3ms**

> PPT中写道：**"Much better than previous case."**（比前一种情况好得多）

### 💡 核心结论
**同样的进程集合，仅仅是到达顺序不同，平均等待时间就从 17ms 降到了 3ms。** 这说明FCFS算法严重依赖于进程到达的顺序。

## 第三张图：核心概念总结

这张PPT总结了FCFS的核心特性：

### 1. 护航效应（Convoy Effect）
*   **定义**：**短进程排在长进程后面**，导致短进程等待时间极长。
*   **比喻**：一艘小快艇（\(P_2\)）被一艘大航母（\(P_1\)）堵在港口，动弹不得。小快艇明明能很快完成装卸，却要等航母完成全部操作。

### 2. 非抢占性（Non-preemptive）
*   **结合你之前的问题**：这就是非抢占的典型案例。
*   一旦 \(P_1\) 获得了CPU，即便 \(P_2\) 只需要1ms，它也**绝对不能打断** \(P_1\)。必须等到 \(P_1\) 自己结束，或者主动让出CPU。

### 3. 对分时系统简直是灾难
*   **分时系统**：多用户共享一台计算机（比如：几十个学生连到一台大学服务器）。
*   **灾难原因**：如果一个人运行一个大的计算任务（比如疯狂编译代码，持续占CPU几十秒），其他所有用户都会感到**死机**。
*   **结论**：FCFS完全不适合交互式系统。

---

## 📝 考试重点总结

| 特性 | FCFS |
| :--- | :--- |
| **模式** | 非抢占式 |
| **优点** | 简单，实现成本低 |
| **缺点** | 平均等待时间波动极大；存在严重的护航效应 |
| **适用场景** | 批处理系统（非交互式，比如打印队列、早期大型机） |
| **不适用场景** | 分时系统、桌面交互系统、实时系统 |

## 💡 思考题（巩固理解）

**问题**：如果有三个进程到达时间分别为 0、0、0，但运行时间分别是 5ms、10ms、100ms。如果采用FCFS，你能算出平均等待时间是多少吗？

*   **提示**：它们同时到达，顺序通常按进程ID或队列中的顺序。
*   算出来的结果会让你更直观地感受“护航效应”有多可怕。

答案是：\((0 + 5 + 15)/3 = 6.67\)ms。\(100\)ms的任务拖累了所有后续短任务。
