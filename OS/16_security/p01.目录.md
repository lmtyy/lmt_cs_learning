这张幻灯片是操作系统课程中关于**“安全（Security）”**章节的目录（通常是大学教材《操作系统概念》即“恐龙书”的第15章内容）。

根据这五个标题，我为你梳理一下这个章节的核心逻辑和重点内容。这一章主要讨论的是：**操作系统如何应对恶意攻击，以及如何保护自身和用户数据的安全。**

---

### 1. The Security Problem (安全问题)
这是全章的引言。
*   **核心问题**：计算机系统非常复杂，存在各种漏洞。操作系统需要平衡“安全性”与“便利性”。
*   **基本概念**：你会接触到**机密性（保密）**、**完整性（不被篡改）**、**可用性（能随时访问）**这三大安全目标。
*   **保护机制**：比如用户空间和内核空间的隔离，以及文件系统的权限管理。

### 2. Authentication (身份认证)
这是安全的第一道大门。系统如何确认使用者的身份？
*   **方式**：
    *   **你知道的**：密码（最常见的，但容易泄露或破解）。
    *   **你拥有的**：物理钥匙、智能卡。
    *   **你是什么**：生物特征（指纹、虹膜、人脸识别）。
*   **课堂重点**：操作系统会讨论如何安全存储密码（如使用哈希和加盐技术，防止管理员直接看到明文密码）。

### 3. Program Threats (程序级威胁)
这一节讲的是恶意软件（Malware），这些程序会利用系统漏洞来破坏安全。
*   **常见类型**：
    *   **木马**：伪装成合法软件，进去后窃取数据或开后门。
    *   **蠕虫**：利用网络漏洞自我复制，疯狂传播（如感染整栋楼甚至全球）。
    *   **病毒**：依附在合法程序（如Word文档、邮件附件）上，用户一启动它就作恶。
*   **技术手段**：还会讲到**缓冲区溢出（Buffer Overflow）**攻击——这是历史最悠久的漏洞之一。

### 4. System Threats (系统级威胁)
这部分比程序威胁更底层，直接攻击操作系统内核或网络环境。
*   **拒绝服务 (DoS)**：攻击者发送海量请求，让服务器忙不过来，导致合法用户无法访问。
*   **端口扫描**：探测系统开启了哪些端口，寻找攻击入口。
*   **Rootkit**：这是一种极度危险的威胁。它不仅能获得管理员权限，还能**隐藏自己**（即使你用杀毒软件查杀，可能也查不到它篡改了系统调用表）。

### 5. Securing Systems (系统加固/保护)
前面讲了问题，这里讲解决方案。操作系统如何“免疫”或“治疗”？
*   **打补丁**：及时更新操作系统修复漏洞。
*   **日志与审计**：记录谁在什么时候干了什么，以便事后追溯。
*   **防火墙与入侵检测**：在网络层面过滤恶意流量。
*   **沙箱**：让程序在受限环境中运行，防止它破坏核心系统。
*   **加密**：对存储的硬盘数据或传输的网络数据加密。
