这道题是标准的**分段系统地址翻译题**。解答它的核心逻辑在之前的PPT中已经详细讲过：**先检查越界，再计算物理地址**。

我带你一步步推导出这5个小题的答案。

---

### 核心公式

在分段系统中：
1.  **合法性检查**：`偏移量 (d) < 段长度 (Length)`，如果 `d >= Length`，则是非法访问（越界错误）。
2.  **物理地址计算**：`物理地址 = 基址 (Base) + 偏移量 (d)`（仅当第一步检查通过时）。

---

### 逐题解析

#### **a. (0, 430)**
*   **段号**：0
*   **查表**：段0的 Base = `219`，Length = `600`
*   **检查**：偏移量 `430` 是否 < 600？
    *   430 < 600 ✅ （合法）
*   **计算**：物理地址 = 219 + 430 = **649**

#### **b. (1, 10)**
*   **段号**：1
*   **查表**：段1的 Base = `2300`，Length = `14`
*   **检查**：偏移量 `10` 是否 < 14？
    *   10 < 14 ✅ （合法）
*   **计算**：物理地址 = 2300 + 10 = **2310**

#### **c. (2, 500)**
*   **段号**：2
*   **查表**：段2的 Base = `90`，Length = `100`
*   **检查**：偏移量 `500` 是否 < 100？
    *   500 < 100 ❌ （非法！越界了）
*   **结果**：**非法内存访问 (Illegal / Trap Error)**。

#### **d. (3, 400)**
*   **段号**：3
*   **查表**：段3的 Base = `1327`，Length = `580`
*   **检查**：偏移量 `400` 是否 < 580？
    *   400 < 580 ✅ （合法）
*   **计算**：物理地址 = 1327 + 400 = **1727**

#### **e. (4, 112)**
*   **段号**：4
*   **查表**：段4的 Base = `1952`，Length = `96`
*   **检查**：偏移量 `112` 是否 < 96？
    *   112 < 96 ❌ （非法！越界了）
*   **结果**：**非法内存访问 (Illegal / Trap Error)**。

### 💡 考试/作业提醒

这类题目的**陷阱**就是**忘记做越界检查**。

很多同学看到 `Base` 和 `Offset` 就直接相加，结果算出了物理地址。但这在分段系统中是**错误的**，因为分段系统中的每个段长度不同。如果偏移量超过了段的长度，意味着程序试图访问不属于它的内存区域，操作系统必须抛出异常（Trap）来保护系统。
