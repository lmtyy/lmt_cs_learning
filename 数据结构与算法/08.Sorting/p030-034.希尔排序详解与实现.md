# 希尔排序详解与实现（p030-034）

这五页介绍希尔排序（Shell Sort），一种改进的插入排序，通过"逐次缩小间隔"的策略突破相邻交换的 Ω(n²) 下界，实现 O(n^1.25) 的时间复杂度。

---

## 第030页：Shell Sort Introduction（"希尔排序简介"）

### 历史与命名

**发明者**：D.L. Shell（1959 年）

> Shell sort 既以发明人命名，又因其"逐次缩小的壳层"的直观比喻而得名——每一层都是一个"壳"，逐次收缩。

### 核心思想

希尔排序是**插入排序的改进版本**，它通过一个巧妙的策略解决了插入排序的主要问题：

**问题**：小元素在数组右侧时，需要一步步向左移动。

**解决方案**：先让相距较远的元素进行比较和交换，从而在较少的步数内把小元素移到更接近最终位置。

### 性能突破

**时间复杂度**：

$$
O(n^{1.25})
$$

> 这是一个介于 O(n log n) 和 O(n²) 之间的时间复杂度，实现了对相邻交换下界的突破。

### 动机：减少插入排序的工作量

希尔排序的目标就是：

$$
\text{减少插入排序所做的工作}
$$

具体来说，通过前期的"粗排序"减少最终插入排序的逆序对数量 I，从而减小 O(I + n) 的 I 部分。

## 第031页：Shell Sort - Gap Sequence（"希尔排序的间隔序列"）

### 间隔序列的概念

希尔排序使用一个**间隔序列**（gap sequence）：

$$
h_1, h_2, \ldots, h_t
$$

其中 $h_1 > h_2 > \cdots > h_t = 1$。

### h_k-Sorted 的定义

在使用间隔 $h_k$ 进行一轮处理后，我们说数组是**h_k-sorted**，意思是：

$$
\text{对所有满足} \quad i + h_k \leq n \quad \text{的} \quad i, \quad \text{有} \quad a[i] \leq a[i + h_k]
$$

**直观理解**：所有相距 $h_k$ 的元素都满足大小关系。

```
例：h_k = 3，n = 9
要求：a[1] ≤ a[4], a[2] ≤ a[5], a[3] ≤ a[6], ...

数组可以看作 3 个独立的子序列（间隔为 3）：
  子序列1：位置 1, 4, 7, ...
  子序列2：位置 2, 5, 8, ...
  子序列3：位置 3, 6, 9, ...

h-sorted 意味着每个子序列都已排序
```

### 处理方式

**每个阶段**，对使用间隔 $h_k$ 的所有子序列进行**插入排序**。

```
伪代码思路：

for each gap h_k:
    for i = h_k+1 to n:           // 遍历从 h_k+1 开始的每个位置
        把 a[i] 插入到间隔为 h_k 的子序列的正确位置
        ├─ 与 a[i-h_k] 比较
        ├─ 与 a[i-2*h_k] 比较
        └─ 直到找到正确位置
```

## 第032页：Shell Sort - Key Property（"希尔排序的关键性质"）

### 重要定理

**定理**：如果一个数组已经是 h_k-sorted，那么在对其进行 h_{k-1}-sorted 处理后，它**仍然保持 h_k-sorted**。

**符号表示**：

$$
\text{如果} \quad \text{h}_k\text{-sorted，然后做} \quad \text{h}_{k-1}\text{-sorted} \quad \Rightarrow \quad \text{仍然} \quad \text{h}_k\text{-sorted}
$$

### 为什么这很重要？

这个性质确保了**前期的工作不会被后期的工作推翻**。

#### 反例说明

假如这个性质不成立：

```
阶段1：用 h_1 = 4 做排序 → 数组变成 4-sorted
阶段2：用 h_2 = 2 做排序 → 破坏了阶段1的工作！

结果：
  ├─ 阶段1 的努力被浪费
  ├─ 每个阶段都独立工作，缺乏积累
  └─ 算法效率很低

➜ 这种算法"几乎没有价值"（little value）
```

### 为什么性质成立？

**关键观察**：h_{k-1}-sorted 的处理**不会改变相距 h_k 的元素对之间的大小关系**。

```
例：已有 a[i] ≤ a[i+h_k]（h_k-sorted）

做 h_{k-1}-sorted 处理时：
  ├─ 移动的元素距离是 h_{k-1}
  ├─ 由于 h_{k-1} < h_k
  ├─ 相距 h_k 的两个元素不会互相交换位置
  └─ 它们的大小关系保持不变：a[i] ≤ a[i+h_k]

因此：h_k-sorted 性质得到保持
```

### 算法的优点

这个性质使得希尔排序能够**递进地改进数组**：

```
初始：无序数组（大间隔）
  ↓ gap = h_1（很大的间隔）
  ├─ 快速消除大量逆序对
  ├─ 数组变得"更有序"但不完全有序
  ↓ gap = h_2（较小的间隔）
  ├─ 继续消除逆序对
  ├─ 数组变得"更有序"
  ↓ gap = 1（插入排序）
  ├─ 在已经"几乎有序"的基础上进行
  ├─ 只需很少的移动
  ↓
最终：完全有序
```

## 第033页：Shell Sort - Gap Sequence Selection（"间隔序列的选择"）

### Shell 原始建议（但性能不佳）

**Shell 的方案**：

$$
h_t = \left\lfloor \frac{n}{2} \right\rfloor, \quad h_k = \left\lfloor \frac{h_{k+1}}{2} \right\rfloor
$$

**序列示例**（n=100）：

$$
50, 25, 12, 6, 3, 1
$$

### 标注："Popular choice (but poor)"

这个序列虽然被广泛使用（historically），但**性能并不理想**：

| 方面 | 评价 |
|-----|------|
| 实现简单 | ✅ 非常简单 |
| 性能 | ❌ 不够优 |
| 理论下界 | ❌ 无法达到 O(n^1.25) |

### 为什么不理想？

```
问题 1：间隔变化不合理
  ├─ 相邻间隔的比例 hk / h(k+1) = 1/2
  ├─ 这个比例不适合所有数据规模
  └─ 无法始终高效地消除逆序对

问题 2：1/2 递推不是最优的
  ├─ 其他比例（如 1/3）有更好的理论性质
  ├─ Shell 序列无法证明达到 O(n^1.25)
  └─ 实际性能通常在 O(n^1.5) 到 O(n²) 之间
```

### 更好的替代方案

| 提议者 | 序列 | 时间复杂度 |
|------|------|----------|
| Shell | n/2, n/4, ... | O(n²) 最坏 |
| Hibbard | 1, 3, 7, 15, ... (2^k-1) | O(n^1.5) |
| Sedgewick | 特殊构造 | O(n^1.35) |
| Knuth | (3^k - 1)/2 | O(n^1.5) |

> 💡 **Knuth 序列**：$h_k = (3^k - 1)/2$，例如 1, 4, 13, 40, 121, ...

## 第034页：Shell Sort - Code Implementation（"希尔排序的代码实现"）

### 完整代码

```c
void shellsort(input_type a[], unsigned int n)
{
    unsigned int i, j, increment;
    input_type tmp;

    /*1*/ for(increment = n/2; increment > 0; increment /= 2)
    /*2*/   for(i = increment+1; i <= n; i++)
            {
    /*3*/       tmp = a[i];
    /*4*/       for(j = i; j > increment; j -= increment)
    /*5*/           if(tmp < a[j-increment])
    /*6*/               a[j] = a[j-increment];
    /*7*/           else
    /*8*/               break;
    /*8*/       a[j] = tmp;
            }
}
```

### 代码逐行精讲

#### 外层循环（第 1 行）：间隔序列

```c
/*1*/ for(increment = n/2; increment > 0; increment /= 2)
```

- 初值：`increment = n/2`（第一个间隔）
- 条件：`increment > 0`（直到间隔为 0）
- 更新：`increment /= 2`（每次减半）

**序列**（n=16）：8 → 4 → 2 → 1

#### 中层循环（第 2 行）：遍历每个起始位置

```c
/*2*/ for(i = increment+1; i <= n; i++)
```

- 起始：`i = increment+1`（跳过前 increment 个元素）
- 条件：`i <= n`（遍历到末尾）
- 更新：`i++`（逐个处理）

**作用**：对每个位置 i，进行间隔为 increment 的子序列的插入排序。

#### 保存待插入元素（第 3 行）

```c
/*3*/ tmp = a[i];
```

这是待插入元素，需要在间隔为 increment 的子序列中找到正确位置。

#### 内层循环（第 4 行）：向左查找位置

```c
/*4*/ for(j = i; j > increment; j -= increment)
```

- 从位置 i 开始
- 向左以 increment 的步长移动
- 直到 `j <= increment`（找到子序列的开头或合适位置）

#### 比较与移动（第 5-7 行）

```c
/*5*/     if(tmp < a[j-increment])
/*6*/         a[j] = a[j-increment];   // 向右移动
/*7*/     else
/*8*/         break;                    // 找到位置，退出
```

**逻辑**：
- 如果待插入的元素小于当前位置左边 increment 距离的元素
- 就把那个元素向右移动
- 否则说明找到了正确位置，退出循环

#### 插入到正确位置（第 8 行）

```c
/*8*/ a[j] = tmp;
```

找到了位置 j，把待插入元素放进去。

### 算法演示

**初始数组**：[49, 38, 65, 97, 76, 13, 27, 49]，n=8

**gap = 4**：

```
处理 i=5 (a[5]=13)：
  比较链：13 < 97 → 移动
  结果：[49, 38, 13, 97, 76, 65, 27, 49]

处理 i=6 (a[6]=27)：
  比较链：27 < 65 → 移动
  结果：[49, 27, 13, 97, 38, 65, 76, 49]

处理 i=7 (a[7]=49)：
  比较链：49 < 49 → 否
  结果：保持不变
```

**gap = 2**：进行微调排序

**gap = 1**：最终的标准插入排序

### 时间复杂度分析

| 阶段 | 间隔 | 工作量 | 累计 |
|-----|-----|-------|------|
| 1 | n/2 | O(n² / h₁²) | O(n²/4) |
| 2 | n/4 | O(n² / h₂²) | O(n²/16) |
| ... | ... | ... | ... |
| log n | 1 | O(n²) | O(n²) |

通过适当的间隔选择（如 Knuth 序列），总工作量可以降到：

$$
T(n) = O(n^{1.5}) \quad \text{或更优}
$$

### 稳定性与特性

| 特性 | 值 |
|-----|-----|
| **稳定性** | ❌ 不稳定（远距离交换会打乱相同键值的相对位置） |
| **空间复杂度** | O(1)（原地排序） |
| **实际表现** | ✅ 很好（特别是对中等规模数据） |
| **最坏情况** | 取决于间隔序列 |
| **最好情况** | O(n)（已有序，gap=1 时直接通过） |

## 核心要点总结

```
希尔排序的演进过程：

问题：
  └─ 插入排序在逆序对多时速度慢 (Ω(n²))

解决思路：
  ├─ 不用相邻交换 ✓
  ├─ 用"远距离交换"替代
  └─ 逐步缩小交换距离，最后用普通插入排序

关键性质：
  ├─ h_k-sorted 后仍是 h_k-sorted（性质保持）
  ├─ 前期工作不被后期推翻
  └─ 算法可以递进改进

性能改进：
  ├─ 相邻交换：Ω(n²)
  ├─ 希尔排序（Shell 序列）：O(n^2) 最坏
  ├─ 希尔排序（Knuth 序列）：O(n^1.5)
  └─ 理想情况：O(n^1.25)

代码特点：
  ├─ 三层嵌套循环
  ├─ 外层控制间隔
  ├─ 中层遍历位置
  └─ 内层进行移位
```
