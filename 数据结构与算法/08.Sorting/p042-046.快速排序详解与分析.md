## 第042页：QuickSort Introduction（"快速排序简介"）

### 实践中最快的排序算法

**名言**：

> QuickSort is the **fastest known sorting algorithm in practice** （快速排序是实践中已知的最快排序算法）

### 历史背景

**发明人**：C.A.R. Hoare（霍尔）

**年代**：1962 年

> 这个算法已经沿用超过 60 年，在工程界仍然是首选排序算法。

### 性能特征

| 指标 | 复杂度 | 说明 |
|-----|-------|------|
| **平均情况** | **O(n log n)** | 快速且稳定的性能 |
| **最坏情况** | O(n²) | 理论上存在，但非常罕见 |
| **实际表现** | 接近 O(n log n) | 工程中的首选 |

### 为什么这么快？

1. **分治思想**：每次 partition 将问题规模减半
2. **缓存友好**：数据访问局部性好
3. **常数系数小**：相比 Merge Sort 不需要额外空间
4. **递归树高度低**：平均情况下只需 O(log n) 层

### 最坏情况的避免

虽然理论最坏情况是 O(n²)，但通过**简单的措施可以使其非常罕见**：

- 随机选择 pivot（而不是第一个元素）
- 使用中位数-中位数选择 pivot
- 三路分割（处理重复元素）

> 💡 **实践指导**：在大多数现实场景中，即使是"坏"的 pivot 选择也很难触发 O(n²)。

## 第043页：QuickSort Algorithm（"快速排序的算法思想"）

### 核心思想：分治法

快速排序基于一个简单而强大的想法：**选择一个枢轴（pivot），然后分割**。

### 完整算法描述

#### 步骤 1：基础情况

```
if (元素个数 ≤ 1)
    return;  // 0 个或 1 个元素，天然有序
```

#### 步骤 2：选择枢轴

```
Pick any element v (this is called the pivot)

// v 可以是：
// - 第一个元素（简单但可能最坏）
// - 最后一个元素（同样简单）
// - 随机元素（避免最坏情况）
// - 中位数（最优但计算成本高）
```

#### 步骤 3：分割

**将其他元素分成两个不相交的集合**：

$$
S_1 = \{x : x \leq v\}
$$

$$
S_2 = \{x : x > v\}
$$

> 🔑 **关键**：S₁ 中的所有元素都小于等于 pivot，S₂ 中的所有元素都大于 pivot。

#### 步骤 4：递归处理

```
return QuickSort(S₁) + [v] + QuickSort(S₂)
```

**逻辑**：
1. 递归排序 S₁（得到有序的较小部分）
2. 放入 pivot v
3. 递归排序 S₂（得到有序的较大部分）
4. 三部分连接 = 完全有序

### 伪代码

```
QuickSort(array):
    if length(array) ≤ 1:
        return array
    
    pivot = SelectPivot(array)
    S₁ = {x in array : x ≤ pivot}
    S₂ = {x in array : x > pivot}
    
    return QuickSort(S₁) + [pivot] + QuickSort(S₂)
```

### 关键特性

| 特性 | 描述 |
|-----|------|
| **分治** | 每次分割问题规模 |
| **就地排序** | 标准实现不需要额外空间 |
| **不稳定** | 分割过程可能改变相同元素的相对顺序 |
| **缓存友好** | 顺序访问数组，缓存利用率高 |

## 第044页：QuickSort - Partition Example（"快速排序的分割演示"）

### 示例数据

**原始数组**：

$$
[46, 55, 13, 42, 94, 05, 17, 70]
$$

**选择的 pivot**：46（第一个元素）

### 分割过程（初始状态到第一趟结果）

#### 初始状态

```
数组：[46, 55, 13, 42, 94, 05, 17, 70]
pivot = 46
i = 0（从左开始，找 ≥ pivot 的元素）
j = 7（从右开始，找 < pivot 的元素）
```

#### 第一次扫描

```
步骤 1：
  i 扫描：46 = 46（停），i = 0
  j 扫描：70 > 46（继续），55 > 46（继续），...，17 < 46（停），j = 6
  交换 a[0] 和 a[6]：[17, 55, 13, 42, 94, 05, 46, 70]
  
步骤 2：
  i 扫描：17 < 46（继续），55 > 46（停），i = 1
  j 扫描：46 = 46（停），j = 6
  交换 a[1] 和 a[6]：[17, 46, 13, 42, 94, 05, 55, 70]
  
步骤 3：
  i 扫描：46 = 46（停），i = 1
  j 扫描：55 > 46（继续），05 < 46（停），j = 5
  交换 a[1] 和 a[5]：[17, 05, 13, 42, 94, 46, 55, 70]
  
步骤 4：
  i 扫描：05 < 46（继续），13 < 46（继续），42 < 46（继续），94 > 46（停），i = 4
  j 扫描：46 = 46（停），j = 5
  i > j，分割完成
```

#### 第一趟结果

```
数组分为三部分：
  S₁ (< 46)：[17, 05, 13, 42]
  pivot：46
  S₂ (> 46)：[94, 55, 70]

最终数组：[17, 05, 13, 42, 46, 94, 55, 70]
         └─ S₁ ──┘     └─ S₂ ──┘
```

> 注意：46 现在在正确的最终位置！

### 递归继续

```
QuickSort([17, 05, 13, 42])  // S₁ 的排序
QuickSort([94, 55, 70])       // S₂ 的排序
```

#### 动态过程描述

```
时刻 t0 - 初始化：
  数组：[46, 55, 13, 42, 94, 05, 17, 70]
  pivot = 46，设置 i=左，j=右

时刻 t1 - 第一次交换：
  i 和 j 同时扫描，交换违规元素
  数组变化：→ [17, 55, 13, 42, 94, 05, 46, 70]

时刻 t2 - 继续交换：
  i 继续右移，j 继续左移
  数组变化：→ [17, 05, 13, 42, 46, 94, 55, 70]

时刻 t3 - 分割完成：
  i 和 j 交错，pivot 到达最终位置 46
  结果：[17, 05, 13, 42] [46] [94, 55, 70]
        └─ 需要递归 ──┘     └─ 需要递归 ──┘

时刻 t4-∞ - 递归处理左右部分，最终得到：
  [05, 13, 17, 42, 46, 55, 70, 94]
```

### 关键观察

1. **Pivot 快速到位**：每一趟都有一个元素达到最终位置
2. **问题规模递减**：每一趟后，问题分成两个更小的子问题
3. **指针对向移动**：i 和 j 从两端向中间移动，高效定位违规元素
4. **交换最少化**：只有违规元素才被交换

## 第046页：QuickSort - Not Stable（"快速排序的不稳定性"）

### 定义回顾

排序是**稳定的**，当且仅当**相同键值的记录在排序前后的相对位置不变**。

### 快速排序不稳定的原因

**分割过程中的远距离交换打乱相对顺序**。

### 反例

**原始数组**（用下标区分相同值）：

$$
[\underbrace{98}_a, \underbrace{2}_1, \underbrace{2}_2, 100]
$$

### 快速排序的执行

#### 选择 pivot = 98

```
pivot = 98（第一个元素）
```

#### 分割过程

```
i 从左扫描，找 > 98 的元素：100（位置 3）
j 从右扫描，找 ≤ 98 的元素：2₂（位置 2）

交换 a[3] 和 a[2]：
  原：[98, 2₁, 2₂, 100]
  后：[98, 2₁, 100, 2₂]
  
继续扫描...
最终结果（一可能的排列）：
  [2, 2, 98, 100]
```

#### 稳定性检验

```
原始中两个 2 的相对顺序：2₁ 在 2₂ 前面
排序后的两个 2：可能顺序反转！

在交换 100 和 2₂ 的过程中，2₁ 和 2₂ 的相对位置被改变。
```

### 对比插入排序

| 算法 | 交换方式 | 稳定性 |
|-----|--------|-------|
| **Insertion Sort** | 相邻元素 | ✅ 稳定 |
| **Bubble Sort** | 相邻元素 | ✅ 稳定 |
| **Quick Sort** | 远距离交换 | ❌ 不稳定 |
| **Merge Sort** | 不交换（比较+合并） | ✅ 稳定 |
| **Heap Sort** | 堆调整 | ❌ 不稳定 |

### 为什么不稳定仍被接受？

```
原因 1：性能优势
  ├─ 快速排序比稳定的 Merge Sort 更快
  ├─ 大多数应用只关心最终排序结果
  └─ 单键值排序时稳定性无关紧要

原因 2：实际应用
  ├─ 多关键字排序时使用稳定排序作为基础
  ├─ 结合其他方法保证整体稳定性
  └─ 例：先用稳定排序排次要键，再用任意排序排主键

原因 3：工程实践
  ├─ 对于绝大多数数据集，快速排序性能远超稳定排序
  ├─ 稳定性需求时明确使用 Merge Sort
  └─ 混合策略（Timsort）在稳定性和性能间平衡
```

## 快速排序的核心特性总结

```
算法框架：
  ├─ 选择 pivot
  ├─ 分割成 S₁ (≤ pivot) 和 S₂ (> pivot)
  ├─ 递归排序 S₁ 和 S₂
  └─ 结果：S₁ + pivot + S₂

性能特征：
  ├─ 平均：O(n log n)（分治树高 O(log n)）
  ├─ 最坏：O(n²)（pivot 总是最小或最大值）
  ├─ 但最坏情况很罕见
  └─ 实际表现接近 O(n log n)

实现特点：
  ├─ 原地排序（不需额外空间）
  ├─ 不稳定（远距离交换）
  ├─ 缓存友好（顺序访问）
  └─ 常数系数小（操作简单）

选择 pivot 的影响：
  ├─ 第一个元素：简单但可能触发最坏情况
  ├─ 随机元素：避免最坏情况概率
  ├─ 中位数：最优但计算成本高
  └─ 三数中位数：实践中的好选择
```

- "最坏情况什么时候发生？" → pivot 选择不当，总是最小/最大值
