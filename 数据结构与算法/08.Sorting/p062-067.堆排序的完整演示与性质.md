这六页通过完整的实例逐步演示堆排序的执行过程，最后分析其不稳定性。

## 第062页：Heapsort - Build Max Heap（"建立最大堆"）

### 核心原则

**升序排列 → 使用最大堆（Max Heap）**

### 实例：从无序数组到最大堆

**原始数组**：

$$
[49, 38, 65, 97, 76, 13, 27, 70]
$$

**建堆后的最大堆**：

$$
[97, 76, 65, 70, 49, 13, 27, 38]
$$

### 建堆过程（树形表示）

**建堆前**（按原始数组直接画成完全二叉树）：

```
           49
         /    \
       38      65
      / \     / \
    97   76  13   27
   /
  70
```

**建堆后**（经过下滤调整后的最大堆）：

```
           97
         /    \
       76      65
      / \     / \
    70   49  13   27
   /
  38
```

### 建堆操作说明

使用 Floyd 线性建堆法（从最后一个非叶节点开始向上调整）：

```
步骤 1：从 n/2 = 4 号节点开始
  节点 97 > 70 ✓（无需调整）

步骤 2：节点 3（值 65）
  65 > 13, 65 > 27 ✓（无需调整）

步骤 3：节点 2（值 38）
  38 < 97 → 交换 38 和 97
  继续：38 < 70 → 交换 38 和 70

步骤 4：节点 1（值 49）
  49 < 97 → 交换 49 和 97
  继续：49 < 76 → 交换 49 和 76
  继续：49 在叶节点附近 → 停止
```

**数组对应**：

| 下标 | 1 | 2 | 3 | 4 | 5 | 6 | 7 | 8 |
|-----|---|---|---|---|---|---|---|---|
| 建堆前 | 49 | 38 | 65 | 97 | 76 | 13 | 27 | 70 |
| 建堆后 | **97** | **76** | **65** | **70** | **49** | **13** | **27** | **38** |

## 第063页：Heapsort - First DeleteMax（"第一次删除堆顶"）

### 第 1 次 DeleteMax

**当前最大堆**：

```
           97
         /    \
       76      65
      / \     / \
    70   49  13   27
   /
  38
```

**操作**：
1. 取出堆顶 97（最大值）
2. 把最后一个元素 38 放到堆顶
3. 对堆顶进行下滤（percolate down）

**下滤过程**：

```
38 放到堆顶：
           38
         /    \
       76      65
      / \     / \
    70   49  13   27

38 < 76 → 与 76 交换：
           76
         /    \
       38      65
      / \     / \
    70   49  13   27

38 < 70 → 与 70 交换：
           76
         /    \
       70      65
      / \     / \
    38   49  13   27
```

**结果**：堆大小从 8 缩小到 7，97 被放到数组末尾（位置 8）

**数组状态**：`[76, 70, 65, 38, 49, 13, 27 | 97]`

## 第065页：Heapsort - Final Steps（"堆排序的最终步骤"）

### 继续 DeleteMax

#### 第 5 次：取出 49

```
结果堆：[38, 27, 13]
数组：[38, 27, 13 | 49, 65, 70, 76, 97]
```

#### 第 6 次：取出 38

```
结果堆：[27, 13]
数组：[27, 13 | 38, 49, 65, 70, 76, 97]
```

#### 第 7 次：取出 27

```
结果堆：[13]
数组：[13 | 27, 38, 49, 65, 70, 76, 97]
```

### 最终结果

$$
[13, 27, 38, 49, 65, 70, 76, 97]
$$

✅ **升序排列完成！**

### 排序过程汇总

| 步骤 | DeleteMax | 堆大小 | 已排序部分 |
|-----|----------|-------|---------|
| 初始 | - | 8 | [] |
| 1 | 97 | 7 | [97] |
| 2 | 76 | 6 | [76, 97] |
| 3 | 70 | 5 | [70, 76, 97] |
| 4 | 65 | 4 | [65, 70, 76, 97] |
| 5 | 49 | 3 | [49, 65, 70, 76, 97] |
| 6 | 38 | 2 | [38, 49, 65, 70, 76, 97] |
| 7 | 27 | 1 | [27, 38, 49, 65, 70, 76, 97] |
| 8 | 13 | 0 | [13, 27, 38, 49, 65, 70, 76, 97] |

> 💡 **关键**：每次 DeleteMax 后，取出的最大值放到堆末尾释放的位置。最终数组从左到右就是升序。

## 第067页：Heapsort - Not Stable（"堆排序的不稳定性"）

### 定义回顾

排序是**稳定的**，当且仅当相同键值的记录排序后相对位置不变。

### 堆排序不稳定的反例

**原始数组**（用下标区分相同值）：

$$
[2_a, 1, 2_b]
$$

（其中 $2_a$ 在位置 1，$2_b$ 在位置 3）

### 排序过程

#### 建最大堆

```
原数组：[2ₐ, 1, 2ᵦ]

作为完全二叉树：
       2ₐ
      / \
     1   2ᵦ

已满足最大堆性质（2ₐ ≥ 1, 2ₐ ≥ 2ᵦ）
```

#### 第 1 次 DeleteMax

```
取出堆顶 2ₐ
将最后一个元素 2ᵦ 放到堆顶
堆：[2ᵦ, 1]
已排序：[2ₐ]

调整：2ᵦ > 1，不需要调整
堆：[2ᵦ, 1]
```

#### 第 2 次 DeleteMax

```
取出堆顶 2ᵦ
堆：[1]
已排序：[2ᵦ, 2ₐ]
```

#### 最终结果

$$
[1, 2_b, 2_a]
$$

### 稳定性检验

```
原始顺序：2ₐ 在 2ᵦ 前面（位置 1 < 位置 3）
排序结果：2ᵦ 在 2ₐ 前面（位置 2 < 位置 3）

相对位置反转！ → 不稳定 ❌
```

### 为什么不稳定？

**根本原因**：堆的 DeleteMax 操作把**最后一个元素**放到堆顶，这种操作无法保持相同键值元素的原始相对位置。

```
具体机制：
  1. DeleteMax 时，堆顶被取出
  2. 最后一个叶节点（可能是相同键值的元素）被放到堆顶
  3. 下滤过程中，相同键值的元素可能被安排在"错误"的相对位置
  4. 已排序部分的累积顺序 = 出堆的逆序 ← 这里出问题
```

### 所有排序算法的稳定性总结

| 算法 | 稳定性 | 原因 |
|-----|-------|------|
| Insertion Sort | ✅ | 相邻移位，相同键值不交换 |
| Bubble Sort | ✅ | 相邻交换，相同键值不交换 |
| Merge Sort | ✅ | 合并时保持相等元素原始顺序 |
| Shell Sort | ❌ | 远距离交换，不同子序列破坏位置 |
| Quick Sort | ❌ | Partition 中远距离交换 |
| Selection Sort | ❌ | 选最小值交换到前端 |
| **Heap Sort** | ❌ | DeleteMax 破坏相同键值的相对位置 |

## 核心总结

```
堆排序的完整执行流程：

Step 1：建最大堆
  ├─ 使用 Floyd 线性建堆法 O(N)
  └─ 堆顶 = 数组最大值

Step 2：重复 N-1 次 DeleteMax
  ├─ 交换堆顶和堆的最后一个元素
  ├─ 堆大小 -= 1
  ├─ 对新的堆顶进行下滤
  └─ 被交换出去的元素放在已排序位置

结果：数组升序排列

性能：
  ├─ 建堆：O(N)
  ├─ N 次 DeleteMax：O(N log N)
  ├─ 总计：O(N log N)
  ├─ 空间：O(1)（原地）
  └─ 稳定性：不稳定
```
