## 第008页：The Family of Sorting Methods（"排序算法大家族"）

### 排序算法的分类体系

整个排序算法世界可以按以下方式分类：

```
                    排序算法
                      │
          ┌───────────┼───────────┐
          │           │           │
     比较排序      址址排序    其他方法
  (Comparison)  (Address-based)
        │           │
    ┌──┴──┬──────┬─────┬─────────┐
    │     │      │     │         │
  换位  插入  选择  分治  其他
  (Trans) (Insert) (Select) (Divide)

╔═══════════════════════════════════════════════════╗
║        主要排序算法分支 (Main Sorting Themes)       ║
╠═══════════════════════════════════════════════════╣
║                                                   ║
║  [比较排序]                [址址排序]              ║
║    ├─ 换位排序              ├─ ProxmapSort        ║
║    │   ├─ BubbleSort        └─ RadixSort         ║
║    │   └─ QuickSort                              ║
║    ├─ 插入排序                                    ║
║    │   ├─ InsertionSort                          ║
║    │   ├─ TreeSort                               ║
║    │   └─ ShellSort                              ║
║    └─ 选择排序                                    ║
║        ├─ SelectionSort                          ║
║        └─ HeapSort                               ║
║                                                   ║
║  [分治排序]                                       ║
║    ├─ QuickSort                                  ║
║    └─ MergeSort                                  ║
║                                                   ║
╚═══════════════════════════════════════════════════╝
```

> 💡 **分类思想**：
> - **比较排序**：通过比较键值大小来决定相对位置（如 BubbleSort）
> - **址址排序**：根据键值直接计算存储位置（如 RadixSort）
> - **分治排序**：分解问题，递归求解（如 MergeSort、QuickSort）

### 关键观察

- **BubbleSort** 和 **QuickSort** 都属于**换位排序**，但复杂度差异巨大（O(n²) vs O(n log n)）
- **HeapSort** 属于**选择排序**，通过优先队列高效地选出最小/最大元素
- **ShellSort** 是**插入排序**的改进版，通过**逐次缩小间隔**提升效率
- **MergeSort** 和 **QuickSort** 都是**分治**思想，但稳定性不同

## 第009页：Sorting Algorithms - Comparison-based Methods（"比较排序算法总览"）

### 比较排序的主要类型

所有比较排序都通过 `if (key1 > key2)` 这样的比较来决定元素位置。根据**比较和移动的操作方式**，分为以下几类：

#### 1. 插入排序（Insert and keep sorted）

```
操作逻辑：逐个取出未排序的元素，插入到已排序部分的正确位置
代表算法：
  ├─ Insertion Sort（基础插入排序，O(n²)）
  └─ Shell Sort（希尔排序，O(n^1.5) ~ O(n²)）

动画演示思路：
  初始：[5, 3, 7, 1, 2]
  第1步：[3, 5, 7, 1, 2]  ← 3 插入到 5 前面
  第2步：[3, 5, 7, 1, 2]  ← 7 已在正确位置
  第3步：[1, 3, 5, 7, 2]  ← 1 插入到最前面
  第4步：[1, 2, 3, 5, 7]  ← 2 插入到正确位置
```

✅ **优点**：
- 稳定
- 对几乎有序的数据效率高
- 在线排序（可边读入边排序）

❌ **缺点**：
- 最坏情况 O(n²)
- 大数据集效率低

#### 2. 换位排序（Transposition Sorting）

```
操作逻辑：不断交换相邻或远距离的元素，让大元素逐步移到后面
代表算法：
  ├─ Bubble Sort（冒泡排序，O(n²)）
  └─ Quick Sort（快速排序，O(n log n) 平均）

特点：通过"交换"而非"插入"来重排元素
```

#### 3. 选择排序（Selection Sort）

```
操作逻辑：每次都找出未排序部分的最小/最大元素，放到已排序部分
代表算法：
  ├─ Selection Sort（基础选择排序，O(n²)）
  └─ Heap Sort（堆排序，O(n log n)）使用优先队列加速查找

与插入排序的对比：
  Insertion Sort：每次选一个未排序元素，插入到已排序部分
  Selection Sort：每次从未排序部分选最小的，放到已排序部分末尾
```

#### 4. 分治排序（Divide and Conquer）

```
操作逻辑：分解问题 → 递归排序 → 合并结果
代表算法：
  ├─ Quick Sort（快速排序）：partition + 递归
  └─ Merge Sort（归并排序）：分组 + 合并
```

递推式：

$$
T(n) = 2 \cdot T(n/2) + O(n) \Rightarrow O(n \log n)
$$

## 第010页：Class Definition of Sorting List（"排序列表的C语言数据结构定义"）

### 核心数据结构

```c
#define MAXSIZE 20          // 最大元素个数

typedef int KeyType;        // 键的类型（可改为其他类型）

typedef struct {
    KeyType key;            // 排序关键字
    InfoType Otherinfo;     // 其他信息
} RedType;                  // "红" = Record Element Data Type

typedef struct {
    Redtype r[MAXSIZE+1];   // 数组存储，下标从1开始
    int length;             // 当前元素个数
} SqList;                   // Seq List = 顺序表
```

### 结构解析

#### 为什么数组下标从 1 开始？

```c
r[MAXSIZE+1]    // 数组大小是 MAXSIZE+1（0到MAXSIZE）
                // 但实际只用 r[1] 到 r[MAXSIZE]
                // 为什么要空出 r[0]？
```

**原因**（典型的教科书做法）：

| 下标位置 | 用途 | 好处 |
|---------|------|------|
| r[0] | **哨兵位置**或**临时存储** | 简化边界判断，避免数组越界 |
| r[1] ~ r[MAXSIZE] | 实际存储 n 个元素 | 下标 = 逻辑位置（易于理解） |

> 💡 **哨兵的作用**：
> 某些排序算法（如冒泡排序）会在 r[0] 暂时放置一个极值元素，用来简化循环条件。

**现代 C 的做法**：从 0 开始，下标空出一个不用，或直接用动态数组。但教科书通常保持从 1 开始的传统。

#### 记录结构 (RedType)

```c
typedef struct {
    KeyType key;           // 必须有：排序依据
    InfoType Otherinfo;    // 可选：其他信息（姓名、地址等）
} RedType;
```

- **key**：决定元素的排序位置
- **Otherinfo**：与 key 关联的其他数据（在排序时一起移动）

> ⚠️ **重要**：整个 RedType 记录会被移动，不仅是 key！这对"数据移动次数"指标有影响。

#### 顺序表结构 (SqList)

```c
typedef struct {
    Redtype r[MAXSIZE+1];   // 记录数组
    int length;             // 关键：记录当前有多少个元素
} SqList;
```

- **length**：当前实际存储的元素个数（≤ MAXSIZE）
- 排序时只需要处理前 `length` 个元素

### 简单初始化例子

```c
SqList myList;
myList.length = 0;          // 初始为空表

// 插入第一个元素
myList.r[1].key = 90;
myList.r[1].Otherinfo = "宋青";
myList.length = 1;

// 再插入一个
myList.r[2].key = 87;
myList.r[2].Otherinfo = "张红";
myList.length = 2;
```

### 与之前学过的数据结构对比

| 数据结构 | 优点 | 缺点 | 排序适用 |
|---------|------|------|---------|
| 数组（SqList） | 随机访问快，缓存友好 | 插删困难 | ✅ 最常用 |
| 链表 | 插删快 | 随机访问慢，缓存不友好 | ⚠️ 有限制 |
| 树结构 | 搜索快 | 构造复杂 | ✅ 特定场景 |

> 💡 **为什么排序优先选数组**？
> - 排序不需要频繁插删，只需要比较和交换
> - 数组的随机访问对排序算法友好
> - 数组的缓存局部性对大规模排序性能关键
