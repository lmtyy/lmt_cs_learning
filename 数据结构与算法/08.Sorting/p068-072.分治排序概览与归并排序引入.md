# 分治排序概览与归并排序引入（p068-072）

这五页从宏观视角对比分治排序的两种策略（QuickSort vs MergeSort），并通过完整实例展示分治排序的分割与合并过程。

---

## 第068页：Overview - Divide and Conquer Sorting（"分治排序概述"）

### 分治排序的核心思想

**策略**：不断将 n 个元素的集合划分成**两个部分**。

$$
\text{效果：显著减少排序所需的比较次数}
$$

### 为什么分治能减少比较？

```
朴素方法（O(n²)）：
  每个元素都和其他所有元素比较
  总比较次数 ≈ n × n = n²

分治方法（O(n log n)）：
  先分成两部分，各自排序
  排完后合并
  总比较次数 ≈ n × log n

关键：分治把"全局比较"变成了"局部比较 + 合并"
```

> 💡 **本质**：分治通过缩小问题规模来减少冗余比较。

## 第069页：Overview - Two Strategies（"分治排序的两种策略"）

### 策略 1：分割时排序（QuickSort）

```
我们在分割的过程中做一点排序工作
→ 每次 partition 后，pivot 到位
→ 当所有 partition 完成时，数组已完全有序
→ 不需要合并步骤
```

**特点**：
- 排序工作在**分割阶段**完成
- 分割结束 = 排序完成
- **不需要合并**

### 策略 2：合并时排序（MergeSort）

```
我们在合并的过程中做一点排序工作
→ 先把数组拆分到每个元素都独立
→ 逐步合并相邻部分，合并时排好序
→ 当所有分区都合并完成时，数组完全有序
```

**特点**：
- 排序工作在**合并阶段**完成
- 分割是简单的"对半切"
- **合并时比较和排序**

### 复杂度

| 算法 | 平均时间 | 最坏时间 | 稳定性 |
|-----|--------|--------|-------|
| QuickSort | O(n log n) | O(n²) | ❌ |
| MergeSort | O(n log n) | O(n log n) | ✅ |

## 第070页：Divide and Conquer Sorting - Direction（"分治排序的方向"）

### 两种方向的可视化

```
                [原始数组]
               /          \
         [左半部分]    [右半部分]
          /    \        /    \
        [.]   [.]    [.]   [.]
       / \   / \   / \   / \
      .  .  .  .  .  .  .  .

↑ MergeSort                QuickSort ↓
（从底向上合并）         （从顶向下分割）
```

### QuickSort 的方向：**自顶向下**

```
QuickSort（↓ 向下分割）：
  原数组 → Partition → 左部分 + pivot + 右部分
                          ↓           ↓
                     Partition     Partition
                      ↓   ↓         ↓   ↓
                    ...  ...      ...  ...
                    
  排序发生在"向下"的过程中（Partition 步骤）
  到达叶子时，排序已经完成
```

### MergeSort 的方向：**自底向上**

```
MergeSort（↑ 向上合并）：
  单个元素  单个元素  单个元素  单个元素
      \      /          \      /
    [合并排序]        [合并排序]
         \                /
          [合并排序]
               ↓
         最终有序数组
         
  排序发生在"向上"的过程中（Merge 步骤）
  到达根节点时，排序完成
```

### 核心对比

| 维度 | QuickSort | MergeSort |
|-----|-----------|-----------|
| **排序方向** | 自顶向下（分割时排序） | 自底向上（合并时排序） |
| **分割方式** | 按 pivot 的值分（不等分） | 按位置对半分（等分） |
| **合并方式** | 无需合并 | 有序合并（双指针） |
| **递归树高度** | 平均 O(log n)，最坏 O(n) | 总是 O(log n) |

## 第071页：Divide - QuickSort Direction（"分割阶段：QuickSort 的分治树"）

### 完整实例

**原始数组**：

$$
[5, 1, 4, 2, 10, 3, 9, 15]
$$

### QuickSort 的分割过程（自顶向下）

```
第 1 层（pivot = 5）：
  [5, 1, 4, 2, 10, 3, 9, 15]
        ↓ Partition
  [1, 4, 2, 3] | 5 | [10, 9, 15]

第 2 层：
  左部分（pivot = 1）：      右部分（pivot = 10）：
  [1, 4, 2, 3]              [10, 9, 15]
      ↓                         ↓
  [] | 1 | [4, 2, 3]        [9] | 10 | [15]

第 3 层：
  [4, 2, 3]（pivot = 4）：
      ↓
  [2, 3] | 4 | []

第 4 层：
  [2, 3]（pivot = 2）：
      ↓
  [] | 2 | [3]
```

### 关键特点

1. **每一步都让 pivot 到达最终位置**
2. **子问题的大小不一定平衡**（取决于 pivot 选择）
3. **分割完成 = 排序完成**（不需要额外的合并）

### 最终结果

$$
[1, 2, 3, 4, 5, 9, 10, 15]
$$

## 第072页：And Conquer - MergeSort Direction（"合并阶段：MergeSort 的分治树"）

### 完整实例

**原始数组**：

$$
[5, 1, 4, 2, 10, 3, 9, 15]
$$

### MergeSort 的合并过程（自底向上）

#### 第 1 层（拆分到单个元素）：

```
[5] [1] [4] [2] [10] [3] [9] [15]
```

#### 第 2 层（两两合并）：

```
[5] + [1] → [1, 5]       ← 比较 5 和 1
[4] + [2] → [2, 4]       ← 比较 4 和 2
[10] + [3] → [3, 10]     ← 比较 10 和 3
[9] + [15] → [9, 15]     ← 比较 9 和 15
```

#### 第 3 层（四四合并）：

```
[1, 5] + [2, 4] → [1, 2, 4, 5]
  比较过程：1<2 取1, 2<5 取2, 4<5 取4, 取5

[3, 10] + [9, 15] → [3, 9, 10, 15]
  比较过程：3<9 取3, 9<10 取9, 10<15 取10, 取15
```

#### 第 4 层（最终合并）：

```
[1, 2, 4, 5] + [3, 9, 10, 15] → [1, 2, 3, 4, 5, 9, 10, 15]
  比较过程：1<3 取1, 2<3 取2, 3<4 取3, 4<9 取4,
           5<9 取5, 9<10 取9, 10<15 取10, 取15
```

### 最终结果

$$
[1, 2, 3, 4, 5, 9, 10, 15]
$$

### 关键特点

1. **分割是"无脑对半切"**（不做任何比较）
2. **合并时进行排序**（双指针合并两个有序子序列）
3. **子问题大小总是平衡的**（每次对半）
4. **保证 O(n log n)**（不依赖输入数据）

### 合并操作的细节

```
合并两个有序数组 A 和 B：
  i = 0, j = 0, k = 0
  while i < len(A) and j < len(B):
      if A[i] <= B[j]:       ← 注意 <=，这保证了稳定性
          result[k] = A[i]
          i++
      else:
          result[k] = B[j]
          j++
      k++
  // 把剩余元素复制过来
```

> 💡 **稳定性的来源**：合并时，当 A[i] == B[j]，我们总是**先取 A[i]**（左边的），这保持了相同键值元素的原始相对位置。

## 核心总结

```
分治排序的两种策略对比：

╔═══════════════╦════════════════════╦════════════════════╗
║               ║    QuickSort       ║    MergeSort       ║
╠═══════════════╬════════════════════╬════════════════════╣
║ 分割方式      ║ 按值分（pivot）    ║ 按位置分（对半）   ║
║ 排序时机      ║ 分割时（Partition）║ 合并时（Merge）    ║
║ 方向          ║ 自顶向下 ↓         ║ 自底向上 ↑         ║
║ 平衡性        ║ 不保证             ║ 保证等分           ║
║ 平均时间      ║ O(n log n)         ║ O(n log n)         ║
║ 最坏时间      ║ O(n²)              ║ O(n log n)         ║
║ 稳定性        ║ ❌ 不稳定          ║ ✅ 稳定            ║
║ 额外空间      ║ O(log n)（栈）     ║ O(n)（临时数组）   ║
║ 实际速度      ║ 更快（常数系数小） ║ 稍慢（常数系数大） ║
╚═══════════════╩════════════════════╩════════════════════╝
```
