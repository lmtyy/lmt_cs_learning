## 逐算法解读

### 直接插入排序

| 指标 | 最好 | 最差 | 说明 |
|-----|------|------|------|
| 比较 | **n** | n² | 已有序时只需 n-1 次比较 |
| 移动 | **0** | n² | 已有序时不需要移动 |
| 稳定 | ✅ | | 相邻移位不破坏相对位置 |
| 空间 | 1 | | 只需一个哨兵位置 |

### 折半插入排序

| 指标 | 最好 | 最差 | 说明 |
|-----|------|------|------|
| 比较 | **n log₂n** | | 用二分查找减少比较次数 |
| 移动 | **0** | n² | 移动次数不变（仍需逐个移位） |
| 稳定 | ✅ | | 与直接插入相同 |
| 空间 | 1 | | 只需一个哨兵位置 |

> 💡 折半插入排序减少了**比较次数**，但**移动次数不变**。

### 起泡排序（冒泡排序）

| 指标 | 最好 | 最差 | 说明 |
|-----|------|------|------|
| 比较 | **n** | n² | 已有序时用 flag 优化，一趟即可 |
| 移动 | **0** | n² | 已有序时不需要交换 |
| 稳定 | ✅ | | 相邻交换不破坏相对位置 |
| 空间 | 1 | | 原地排序 |

### 快速排序

| 指标 | 最好 | 最差 | 说明 |
|-----|------|------|------|
| 比较 | **n log₂n** | n² | 平衡分割 vs 极端分割 |
| 移动 | **n log₂n** | n² | 与比较次数同阶 |
| 稳定 | ❌ | | Partition 远距离交换 |
| 空间 | **log₂n** | **n²** | 递归栈深度 |

> ⚠️ **快速排序的附加存储**：最好 O(log₂n)（平衡分割时递归栈深度），最差 O(n²)（极端退化时递归栈深度接近 n）。

### 简单选择排序

| 指标 | 最好 | 最差 | 说明 |
|-----|------|------|------|
| 比较 | **n²** | n² | 无论输入如何，总是 n(n-1)/2 次 |
| 移动 | **0** | n | 已有序不需交换；最多 n-1 次交换 |
| 稳定 | ❌ | | 选最小值交换到前端 |
| 空间 | 1 | | 原地排序 |

> 💡 选择排序的**比较次数固定**，但**移动次数最少**（最多 O(n)）。

### 锦标赛排序（Tournament Sort）

| 指标 | 最好 | 最差 | 说明 |
|-----|------|------|------|
| 比较 | **n log₂n** | | 通过比赛树高效选最小值 |
| 移动 | **n log₂n** | | 每次选出一个需要 O(log n) 调整 |
| 稳定 | ✅ | | 可以保持相对位置 |
| 空间 | **n** | | 需要额外的比赛树空间 |

### 堆排序

| 指标 | 最好 | 最差 | 说明 |
|-----|------|------|------|
| 比较 | **n log₂n** | | 所有情况下都保证 O(n log n) |
| 移动 | **n log₂n** | | 每次 DeleteMax 需要 O(log n) 调整 |
| 稳定 | ❌ | | 堆调整破坏相对位置 |
| 空间 | **1** | | 原地排序 |

> 💡 堆排序是**唯一同时满足 O(n log n) 和 O(1) 空间**的排序算法。

### 归并排序

| 指标 | 最好 | 最差 | 说明 |
|-----|------|------|------|
| 比较 | **n log₂n** | | 所有情况下都保证 O(n log n) |
| 移动 | **n log₂n** | | 每趟合并需要 O(n) |
| 稳定 | ✅ | | 合并时 ≤ 优先取左边 |
| 空间 | **n** | | 需要临时数组 |

> 💡 归并排序是**唯一同时满足 O(n log n) 和稳定性**的比较排序算法。

## 快速记忆口诀

```
稳定排序（"稳坐插并"）：
  ├─ 直接插入 ✅
  ├─ 折半插入 ✅
  ├─ 起泡排序 ✅
  ├─ 锦标赛排序 ✅
  └─ 归并排序 ✅

不稳定排序（"快选堆希"）：
  ├─ 快速排序 ❌
  ├─ 选择排序 ❌
  ├─ 堆排序 ❌
  └─ 希尔排序 ❌
```

### 选择建议

| 需求 | 推荐算法 | 原因 |
|-----|--------|------|
| 实际最快 | **快速排序** | 平均 O(n log n)，常数系数小 |
| 保证 O(n log n) | **堆排序** | 最坏情况也是 O(n log n) |
| 需要稳定性 | **归并排序** | O(n log n) 且稳定 |
| 数据几乎有序 | **插入排序** | 最好情况 O(n) |
| 内存受限 | **堆排序** | O(1) 额外空间 |
| 小数据集 | **插入排序** | 简单高效，常数系数小 |
| 外排序 | **归并排序** | 顺序访问友好 |
