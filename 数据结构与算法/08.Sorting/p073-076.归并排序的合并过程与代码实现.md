## 第073页：MergeSort - Algorithm Description（"归并排序的算法描述"）

### 归并排序的三个关键步骤

1. **反复对半分割**：将初始数组不断切成两半，直到每个子数组只剩 1 个元素
2. **逐层合并**：在每一层回溯中，将两个已排序的子数组**合并**成一个有序数组
3. **合并操作**：通过比较两个数组的当前元素，选较小的放入新数组

### 合并操作的数据结构

**三个指针**：

```
A 数组（已排序）：[1, 13, 24, 26]
                   ↑
                  Aptr

B 数组（已排序）：[2, 15, 27, 38]
                   ↑
                  Bptr

C 数组（结果）：  [_, _, _, _, _, _, _, _]
                   ↑
                  Cptr
```

| 指针 | 作用 | 初始位置 |
|-----|------|---------|
| **Aptr** | 指向 A 数组当前待比较的元素 | A[0] |
| **Bptr** | 指向 B 数组当前待比较的元素 | B[0] |
| **Cptr** | 指向 C 数组下一个待填入的位置 | C[0] |

### 合并规则

```
while (Aptr 未到末尾 AND Bptr 未到末尾):
    if A[Aptr] <= B[Bptr]:
        C[Cptr] = A[Aptr]
        Aptr++
    else:
        C[Cptr] = B[Bptr]
        Bptr++
    Cptr++

// 把剩余元素复制到 C
```

> 💡 **关键**：使用 `<=` 而非 `<`，当两个元素相等时优先取左边的（A 数组的），这保证了**稳定性**。

## 第074页：Merging - Step by Step（"合并过程的逐步演示"）

### 实例

**A 数组**：[1, 13, 24, 26]
**B 数组**：[2, 15, 27, 38]

### 逐步合并

#### 初始状态

```
A: [1, 13, 24, 26]    B: [2, 15, 27, 38]    C: [_, _, _, _, _, _, _, _]
    ↑ Aptr                 ↑ Bptr                 ↑ Cptr
```

#### 第 1 步：比较 A[0]=1 vs B[0]=2

```
1 < 2 → 取 A[0]=1 放入 C

A: [1, 13, 24, 26]    B: [2, 15, 27, 38]    C: [1, _, _, _, _, _, _, _]
       ↑ Aptr             ↑ Bptr                    ↑ Cptr
```

#### 第 2 步：比较 A[1]=13 vs B[0]=2

```
13 > 2 → 取 B[0]=2 放入 C

A: [1, 13, 24, 26]    B: [2, 15, 27, 38]    C: [1, 2, _, _, _, _, _, _]
       ↑ Aptr                ↑ Bptr                      ↑ Cptr
```

#### 第 3 步：比较 A[1]=13 vs B[1]=15

```
13 < 15 → 取 A[1]=13 放入 C

A: [1, 13, 24, 26]    B: [2, 15, 27, 38]    C: [1, 2, 13, _, _, _, _, _]
           ↑ Aptr            ↑ Bptr                         ↑ Cptr
```

#### 继续...

| 步骤 | 比较 | 选取 | C 数组状态 |
|-----|------|------|----------|
| 4 | 24 vs 15 | B: 15 | [1, 2, 13, 15, _, _, _, _] |
| 5 | 24 vs 27 | A: 24 | [1, 2, 13, 15, 24, _, _, _] |
| 6 | 26 vs 27 | A: 26 | [1, 2, 13, 15, 24, 26, _, _] |
| 7 | A用完 | B: 27 | [1, 2, 13, 15, 24, 26, 27, _] |
| 8 | A用完 | B: 38 | [1, 2, 13, 15, 24, 26, 27, 38] |

### 最终结果

$$
C = [1, 2, 13, 15, 24, 26, 27, 38]
$$

### 合并操作的复杂度

| 指标 | 值 | 说明 |
|-----|-----|------|
| **比较次数** | 最多 n-1 次 | 每次比较确定一个元素的位置 |
| **移动次数** | 恰好 n 次 | 每个元素都被复制到 C 中 |
| **时间复杂度** | O(n) | 线性时间 |
| **空间复杂度** | O(n) | 需要额外的 C 数组 |

> 💡 **核心特性**：合并两个长度分别为 m 和 n 的有序数组，总时间为 O(m+n)。

## 第076页：The MergeSort Algorithm（"归并排序的代码实现"）

### 完整代码

```java
static void mergeSort(Comparable[] a, Comparable[] temp,
                      int left, int right)
{
    if (left < right)    // terminating condition
    {
        int centre = (left + right) / 2;
        
        mergeSort(a, temp, left, centre);       // 递归排序左半部分
        mergeSort(a, temp, center + 1, right);  // 递归排序右半部分
        merge(a, temp, left, center + 1, right); // 合并两个有序部分
    }
}
```

### 代码逐行精讲

#### 函数签名

```java
static void mergeSort(Comparable[] a, Comparable[] temp,
                      int left, int right)
```

**参数**：
- `a[]`：待排序数组
- `temp[]`：临时数组（用于合并操作）
- `left`：当前子数组的左边界
- `right`：当前子数组的右边界

#### 终止条件

```java
if (left < right)    // terminating condition
```

- `left >= right`：子数组长度 ≤ 1，天然有序，返回
- `left < right`：子数组长度 ≥ 2，需要继续分割

#### 计算中点

```java
int centre = (left + right) / 2;
```

**作用**：找到数组的中间位置，准备对半分割

**注意**：整数除法自动向下取整

#### 递归排序左半部分

```java
mergeSort(a, temp, left, centre);
```

**范围**：`[left, centre]`（包含中点）

#### 递归排序右半部分

```java
mergeSort(a, temp, center + 1, right);
```

**范围**：`[centre+1, right]`

#### 合并两个有序部分

```java
merge(a, temp, left, center + 1, right);
```

**作用**：将 `[left, centre]` 和 `[centre+1, right]` 这两个已排序的子数组合并成一个有序数组

### 递归调用的可视化

```
mergeSort(a, temp, 0, 7)
    ├── mergeSort(a, temp, 0, 3)      [左半]
    │       ├── mergeSort(a, temp, 0, 1)
    │       │       ├── mergeSort(a, temp, 0, 0)  [终止]
    │       │       └── mergeSort(a, temp, 1, 1)  [终止]
    │       │       └── merge(a, temp, 0, 1, 1)
    │       └── mergeSort(a, temp, 2, 3)
    │               ├── mergeSort(a, temp, 2, 2)  [终止]
    │               └── mergeSort(a, temp, 3, 3)  [终止]
    │               └── merge(a, temp, 2, 3, 3)
    │       └── merge(a, temp, 0, 2, 3)
    └── mergeSort(a, temp, 4, 7)      [右半]
            ├── ...（同上结构）
            └── merge(a, temp, 4, 6, 7)
    └── merge(a, temp, 0, 4, 7)       [最终合并]
```

### 时间复杂度分析

**递推关系**：

$$
T(n) = 2T(n/2) + O(n)
$$

其中：
- $2T(n/2)$：两次递归调用（左半和右半）
- $O(n)$：merge 操作（线性时间）

**Master Theorem 求解**：

$$
a = 2, \quad b = 2, \quad f(n) = O(n)
$$

$$
\log_b a = \log_2 2 = 1
$$

$$
f(n) = O(n^1) \Rightarrow \text{Case 2: } T(n) = O(n \log n)
$$

### 空间复杂度

| 项目 | 值 | 说明 |
|-----|-----|------|
| **temp 数组** | O(n) | 合并时的临时存储 |
| **递归栈** | O(log n) | 递归深度 = log₂n |
| **总空间** | **O(n)** | temp 数组是主导项 |

> ⚠️ **MergeSort 的主要缺点**：需要 O(n) 的额外空间。这是它比 QuickSort 和 HeapSort 差的唯一地方。

## 核心总结

```
归并排序的完整框架：

算法结构（三步）：
  1. 分割：对半切（不做任何比较）
  2. 递归：分别排序左半和右半
  3. 合并：双指针合并两个有序子数组

合并操作（核心）：
  ├─ 三个指针：Aptr、Bptr、Cptr
  ├─ 每次比较取较小者放入 C
  ├─ 相等时取左边的（保证稳定性）
  └─ 时间 O(n)，空间 O(n)

性能：
  ├─ 时间：O(n log n)（所有情况）
  ├─ 空间：O(n)（临时数组）
  ├─ 稳定性：✅ 稳定
  └─ 递推：T(n) = 2T(n/2) + O(n)

适用场景：
  ├─ 需要稳定排序
  ├─ 需要保证 O(n log n) 最坏情况
  ├─ 外排序（磁盘上的大文件排序）
  └─ 链表排序（不需要额外空间）
```
