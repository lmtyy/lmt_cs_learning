## 第002页：Motivation（"为什么要学排序"）

### 核心论点：排序是为搜索服务的

| 搜索方式 | 时间复杂度 | 前提条件 |
|---------|-----------|---------|
| 顺序搜索 Sequential Search | **O(n)** | 无需排序 |
| 二分搜索 Binary Search | **O(log n)** | 必须有序 |

> 💡 类比：在图书馆找一本书——如果书架上的书是乱放的，你只能一本一本翻（O(n)）；如果按编号排好了，你可以直接折半定位（O(log n)）。排序就是"整理书架"的过程。

✅ **核心结论**：高效排序方法之所以重要，是因为它让后续的搜索操作从线性降到对数级别。

### 术语表

| 术语 | 英文 | 含义 | 类比 |
|-----|------|-----|------|
| 列表 | list | 能放进内存的数据集合 | 你手边的一叠卡片 |
| 文件 | file | 存在外部设备上的数据集合 | 图书馆里的一柜子档案 |
| 记录 | record | 描述一个对象的全部信息 | 一张学生信息卡 |
| 字段 | field | 记录中的一个属性 | 卡片上的"姓名"栏 |
| 键 | key | 用来比较/排序的那个字段 | 你决定按"学号"来排卡片 |

> 🔑 关系链：file/list → 包含多个 record → 每个 record 有多个 field → 其中一个 field 被选为 key 来排序。

## 第003页：Introduction（"排序问题的正式定义"）

### 问题三要素

$$
\text{Given: } \text{一个包含 } n \text{ 个元素的集合（数组、单词集等）}
$$

$$
\text{Suppose: } \text{元素之间存在一种序关系（如 } \leq \text{）}
$$

$$
\text{Goal: } \text{把元素按升序（或降序）排列}
$$

### 示例

```
排序前: 1  23  2  56  9  8  10  100
排序后: 1   2  8   9  10 23  56  100
```

> ⚠️ 注意：排序前后元素不变（不会凭空多出/少掉），只是位置重新安排。这叫做**就地重排**的基本约束。

### 深入思考：什么是"序关系"？

PPT说 "there is an order relation"，这其实有严格数学定义：

- **自反性**：a ≤ a
- **反对称性**：若 a ≤ b 且 b ≤ a，则 a = b
- **传递性**：若 a ≤ b 且 b ≤ c，则 a ≤ c
- **完全性**：任意两个元素都可比较

满足这四条的叫**全序关系（Total Order）**，这是排序能成立的前提。

> 💡 反例：如果元素之间不是全序（比如"石头剪刀布"的关系），就没法排序。

## 总结

```
排序的价值链：
│
├─ 动机：提升搜索效率 O(n) → O(log n)
│
├─ 数据组织：file/list ⊃ record ⊃ field ⊃ key
│          （存储地点）（信息单位）（属性）（排序依据）
│
└─ 定义：给定 n 个可比较元素，重新排列使得 key 升序/降序
```

> 💡 **核心考点**：排序的本质是利用元素间的全序关系，将搜索时间从 O(n) 优化到 O(log n)。术语区分（list vs file, record vs field vs key）是选择题高频考点。
