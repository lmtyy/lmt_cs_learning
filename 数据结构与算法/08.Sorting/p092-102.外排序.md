# 外排序（p092-102）

这十一页介绍外排序（External Sorting），当数据量太大无法放入内存时，如何利用磁带/磁盘进行排序。核心思想是多路归并。

---

## 第092页：External Sorting - Motivation（"外排序的动机"）

### 内排 vs 外排

到目前为止学过的所有排序算法都有一个前提：

$$
\text{输入数据能够完全放入主存（RAM）}
$$

### 现实问题

在很多应用中，输入数据**太大**，无法全部放入内存：
- 数据库排序（TB 级）
- 日志文件排序（GB 级）
- 大规模科学计算数据

### 磁带/磁盘的访问特点

**磁带（Tape）上的数据只能顺序访问**：

```
内存（RAM）：
  ├─ 随机访问 O(1)
  ├─ 大小有限（GB 级）
  └─ 所有内排算法都适用

磁带/磁盘：
  ├─ 只能顺序访问（Sequential Access）
  ├─ 大小几乎无限（TB 级）
  ├─ 随机访问极慢（seek time）
  └─ 需要专门的外排序算法
```

> 💡 **核心挑战**：磁带上的元素只能顺序读取，不能随机跳转。因此 QuickSort、HeapSort 等依赖随机访问的算法**完全不适用**。

## 第093页：External Sorting - Basic Algorithm（"外排序的基本算法"）

### 基本外排序算法 = 归并排序的外部版本

**核心方法**：使用 MergeSort 的合并操作

### 设备配置

假设我们有 **4 条磁带**：

| 磁带 | 角色 | 说明 |
|-----|------|------|
| **Ta1** | 输入磁带 1 | 初始数据在这里 |
| **Ta2** | 输入磁带 2 | 辅助输入 |
| **Tb1** | 输出磁带 1 | 交替写入 |
| **Tb2** | 输出磁带 2 | 交替写入 |

### 前提条件

- 数据初始存放在 **Ta1** 上
- 内存一次能容纳（并排序）**m 条记录**

> 💡 **关键参数**：m = 内存能容纳的记录数。这决定了初始 run 的大小。

---

## 第094页：External Sorting - Creating Runs（"创建初始有序段"）

### 第一步：创建初始 runs

**操作**：

1. 从输入磁带 Ta1 中每次读入 **m 条记录**
2. 在内存中用任意内排算法排序
3. 将排好的 m 条记录**交替写入** Tb1 和 Tb2

**术语**：每组排好的 m 条记录称为一个 **run**（有序段）

### 写入策略

```
第 1 组（m 条记录） → 排序 → 写入 Tb1
第 2 组（m 条记录） → 排序 → 写入 Tb2
第 3 组（m 条记录） → 排序 → 写入 Tb1
第 4 组（m 条记录） → 排序 → 写入 Tb2
...交替进行...
```

### 完成后

- Ta1 已清空
- Tb1 和 Tb2 各有约 n/(2m) 个 run
- 每个 run 内部是有序的
- **所有磁带回卷到开头（rewind）**

### 示例数据

**输入磁带 Ta1 上的数据**（共 13 条记录）：

[81, 94, 11, 96, 12, 35, 17, 99, 28, 58, 41, 75, 15]

| Ta1 | 81, 94, 11, 96, 12, 35, 17, 99, 28, 58, 41, 75, 15 |
| Ta2 | （空） |
| Tb1 | （空） |
| Tb2 | （空） |

## 第096-097页：External Sorting - After Run Creation（"创建 runs 后的状态"）

### 参数设置

$$
m = 3 \quad \text{（内存一次能排序 3 条记录）}
$$

### 创建 runs 的过程

```
读入第 1 组：[81, 94, 11] → 排序 → [11, 81, 94] → 写入 Tb1
读入第 2 组：[96, 12, 35] → 排序 → [12, 35, 96] → 写入 Tb2
读入第 3 组：[17, 99, 28] → 排序 → [17, 28, 99] → 写入 Tb1
读入第 4 组：[58, 41, 75] → 排序 → [41, 58, 75] → 写入 Tb2
读入第 5 组：[15]（不足 m）→ 排序 → [15] → 写入 Tb1
```

### 创建 runs 后的磁带状态

| 磁带 | 内容 | 说明 |
|-----|------|------|
| Ta1 | （空） | 已全部读完 |
| Ta2 | （空） | 未使用 |
| **Tb1** | **[11, 81, 94]** \| **[17, 28, 99]** \| **[15]** | 3 个 runs |
| **Tb2** | **[12, 35, 96]** \| **[41, 58, 75]** | 2 个 runs |

> 竖线 `|` 分隔不同的 run。

### 下一步：多路归并

接下来需要将 Tb1 和 Tb2 中的 runs **两两合并**，合并后写入 Ta1 和 Ta2：

```
第 1 趟合并（2-路归并）：
  Tb1 的 run1 [11,81,94] + Tb2 的 run1 [12,35,96]
  → 合并 → [11,12,35,81,94,96] → 写入 Ta1

  Tb1 的 run2 [17,28,99] + Tb2 的 run2 [41,58,75]
  → 合并 → [17,28,41,58,75,99] → 写入 Ta2

  Tb1 的 run3 [15]（没有对应的 Tb2 run）
  → 直接复制 → [15] → 写入 Ta1

第 2 趟合并：
  Ta1 的 run1 [11,12,35,81,94,96] + Ta2 的 run1 [17,28,41,58,75,99]
  → 合并 → [11,12,17,28,35,41,58,75,81,94,96,99] → 写入 Tb1

  Ta1 的 run2 [15]（没有对应的 Ta2 run）
  → 直接复制 → [15] → 写入 Tb2

第 3 趟合并：
  Tb1 [11,12,17,28,35,41,58,75,81,94,96,99] + Tb2 [15]
  → 合并 → [11,12,15,17,28,35,41,58,75,81,94,96,99]
  → 最终有序！
```

### 外排序的趟数分析

**初始 runs 数量**：

$$
\lceil n / m \rceil = \lceil 13 / 3 \rceil = 5 \text{ 个 runs}
$$

**每趟合并后 runs 数量减半**：

$$
5 \to 3 \to 2 \to 1
$$

**需要的合并趟数**：

$$
\lceil \log_2(\lceil n/m \rceil) \rceil = \lceil \log_2 5 \rceil = 3 \text{ 趟}
$$

### 时间复杂度

每趟合并需要读写全部 n 条记录：

$$
T = O\left(\frac{n}{m} \cdot \log_2 \frac{n}{m}\right) \text{ 次磁盘 I/O}
$$

> 💡 **外排序的瓶颈是 I/O 次数，而不是比较次数。**

## 第098-099页：External Sorting - Merge Process（"归并过程的详细描述"）

### 归并操作规则

现在 Tb1 和 Tb2 各包含一组 runs。归并的具体步骤：

#### 步骤 1：合并第一对 runs

```
取 Tb1 的第 1 个 run + Tb2 的第 1 个 run
→ 合并（merge）→ 产生一个两倍长的 run
→ 写入 Ta1
```

#### 步骤 2：合并第二对 runs

```
取 Tb1 的第 2 个 run + Tb2 的第 2 个 run
→ 合并 → 产生一个两倍长的 run
→ 写入 Ta2
```

#### 步骤 3：继续交替

```
继续交替写入 Ta1 和 Ta2，直到 Tb1 或 Tb2 为空
```

### 结果

- 每次归并后，run 的长度**翻倍**（从 m 变成 2m，再变成 4m...）
- 继续这个过程，直到只剩下 **1 个长度为 n 的 run**

### 趟数分析

$$
\text{总趟数} = \lceil \log_2(n/m) \rceil + 1 \text{（加上初始建 run 趟）}
$$

### 阶段 1：创建 runs（m=3）

| 磁带 | 内容 |
|-----|------|

| Ta2 | （空） |
| Tb1 | [11, 81, 94] \| [17, 28, 99] \| [15] |
| Tb2 | [12, 35, 96] \| [41, 58, 75] |

合并 Tb1 和 Tb2 的 runs，交替写入 Ta1 和 Ta2：

```
Tb1 run1 [11,81,94] + Tb2 run1 [12,35,96] → merge → [11,12,35,81,94,96] → Ta1
Tb1 run2 [17,28,99] + Tb2 run2 [41,58,75] → merge → [17,28,41,58,75,99] → Ta2
Tb1 run3 [15] (无对应) → 直接复制 → [15] → Ta1
```

| 磁带 | 内容 |

| Tb2 | （空） |

### 阶段 3：第 2 趟归并（Ta→Tb）

合并 Ta1 和 Ta2 的 runs，交替写入 Tb1 和 Tb2：

```
Ta1 run1 [11,12,35,81,94,96] + Ta2 run1 [17,28,41,58,75,99]
→ merge → [11,12,17,28,35,41,58,75,81,94,96,99] → Tb1

Ta1 run2 [15] (无对应) → 直接复制 → [15] → Tb2
```

| 磁带 | 内容 |
|-----|------|
| Ta1 | （空） |
| Ta2 | （空） |
| Tb1 | [11, 12, 17, 28, 35, 51, 58, 75, 81, 94, 96, 99] |

### 阶段 4：第 3 趟归并（Tb→Ta）

```
Tb1 [11,12,17,28,35,41,58,75,81,94,96,99] + Tb2 [15]
→ merge → [11, 12, 15, 17, 28, 35, 41, 58, 75, 81, 94, 96, 99] → Ta1
```

| 磁带 | 内容 |

## 第102页：External Sorting - Pass Count（"趟数总结"）

### 趟数公式

$$
\text{总趟数} = \lceil \log_2(n/m) \rceil + 1
$$

其中：
- $\lceil \log_2(n/m) \rceil$：归并趟数
- $+1$：初始建 run 趟

### 本例验证

$$
n = 13, \quad m = 3
$$

$$
\text{初始 runs 数} = \lceil 13/3 \rceil = 5
$$

$$
\text{归并趟数} = \lceil \log_2 5 \rceil = 3
$$

$$
\text{总趟数} = 3 + 1 = 4
$$

### 每趟的 I/O 量

| 趟 | 操作 | 读取量 | 写入量 |
|---|------|-------|-------|
| 0 | 建 runs | 读 n | 写 n |
| 1 | 第 1 趟归并 | 读 n | 写 n |
| 2 | 第 2 趟归并 | 读 n | 写 n |
| 3 | 第 3 趟归并 | 读 n | 写 n |

**总 I/O 量**：

$$
2n \times (\lceil \log_2(n/m) \rceil + 1)
$$

> 💡 **外排序的优化方向**：减少趟数 → 增大 m 或使用 k-路归并（每趟合并 k 个 runs）

### 外排序完整流程图

```
外排序的完整执行流程：

输入：Ta1 上的 n 条记录，内存大小 m

阶段 0：建 runs
  ├─ 每次读 m 条记录到内存
  ├─ 内排序
  ├─ 交替写入 Tb1 和 Tb2
  └─ 产生 ⌈n/m⌉ 个 runs

阶段 1-k：归并
  ├─ 从一对磁带读取 runs
  ├─ 两两合并
  ├─ 交替写入另一对磁带
  ├─ runs 长度翻倍
  └─ 重复直到只剩 1 个 run

最终：一条磁带上有完整的有序序列
```
