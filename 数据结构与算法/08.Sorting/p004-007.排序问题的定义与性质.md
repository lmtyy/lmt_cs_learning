## 第004页：Sorting Problem - Definition（"排序问题的数学定义"）

### 核心定义

给定记录序列 $(R_0, R_1, \ldots, R_{n-1})$，其中每个 $R_i = \text{key} + \text{data}$。

**目标**：找到一个排列 $f$（permutation），使得：

$$
K_{f(i-1)} \leq K_{f(i)}, \quad 0 < i < n-1
$$

> 🔑 **术语解析**：
> - $R_i$：第 i 个记录
> - $K_i$：第 i 个记录的键值
> - $f$：一个映射，$f(i)$ 告诉你原始的第 i 个记录现在排在哪个位置
> - 不等式读法："排列后，前一个键值总是 ≤ 后一个键值"

✅ **Sorted** 就是满足上面的不等式条件。

## 第006页：Sorting Problem - Stability（"排序的稳定性"）

### 稳定性的定义

**定义**：如果在原始序列中 $i < j$ 且 $K_i = K_j$，那么排序后 $R_i$ 仍然排在 $R_j$ 前面。

> 💡 **通俗说法**：键值相同的两条记录，排序后它们的相对位置不变。

### 实例：看第005页成绩排序

两个 90 分的学生（020140 吴紫，020101 宋青）。

- **稳定排序**的结果（按成绩排序）：
  ```
  020140 吴紫 90    ← 原始第5个
  020101 宋青 90    ← 原始第4个
  ```
  ✅ 两个 90 分的相对位置保持：020140 在前（因为原本靠后）。

- **不稳定排序**可能产生：
  ```
  020101 宋青 90    ← 原始第4个，被提到前面
  020140 吴紫 90    ← 原始第5个，被往后推
  ```
  ❌ 两个 90 分的相对位置反转了。

> 🎯 **为什么稳定性重要**？
> - **多关键字排序的前提**：先按次要键排序，再按主要键排序，稳定性保证了前一步的排序结果不被破坏。
> - 如果排序不稳定，多关键字排序无法保证最终结果的正确性。

### 常见排序算法的稳定性

| 排序算法 | 稳定性 | 说明 |
|---------|-------|------|
| Bubble Sort | ✅ 稳定 | 相邻交换才能维持相对位置 |
| Insertion Sort | ✅ 稳定 | 从前往后插入不破坏前面的相对位置 |
| Selection Sort | ❌ 不稳定 | 选最小值交换会打乱相同元素的相对位置 |
| Quick Sort | ❌ 不稳定 | partition 过程会打乱相同元素的位置 |
| Merge Sort | ✅ 稳定 | 合并时保持相等元素的原始顺序 |
| Heap Sort | ❌ 不稳定 | 堆调整会改变相同元素的相对位置 |

### 1️⃣ 内排 vs 外排（Internal vs External Sort）

| 维度 | 内排 | 外排 |
|------|------|------|
| 数据存储位置 | 全部在内存（RAM）| 大部分在外部设备（硬盘） |
| 应用场景 | 数据量小（MB 级） | 数据量大（GB 级以上） |
| 时间复杂度瓶颈 | 比较和交换次数 | **I/O 次数**（远比计算慢） |
| 优化目标 | 减少比较/交换 | 减少磁盘读写 |
| 例子 | 排序数组 | 排序数据库表 |

> 💡 **直觉**：外排的核心任务是**降低 I/O 次数**，而非减少比较次数。这导致外排有独特的算法（K-路归并）。

### 2️⃣ 排序性能的两个评估指标（Criteria）

| 指标 | 英文 | 含义 | 为什么重要 |
|-----|------|-----|----------|
| **关键字比较次数** | # of key comparisons | 排序过程中执行了多少次 `if (a > b)` | 直接反映时间复杂度 |
| **数据移动次数** | # of data movements | 排序过程中执行了多少次交换/赋值 | 影响内存总线压力和缓存效率 |

> ⚠️ **重要**：这两个指标有时候是冲突的！
> - Bubble Sort：比较 O(n²)，移动最多 O(n²)
> - Quick Sort：比较 O(n log n)，但移动也是 O(n log n)
> - Insertion Sort：最坏情况下比较 O(n²)，但如果数据"几乎有序"，移动次数会很少

**实战例子**：
```
数据：[1, 2, 3, 4, 5, 6, 7, 100]（几乎有序）

Insertion Sort：
- 比较次数：O(n)（少）
- 移动次数：O(n)（少）
✅ 非常高效

Quick Sort：
- 比较次数：O(n log n)（正常）
- 移动次数：O(n log n)（正常）
❌ 浪费了"几乎有序"的优势
```

## 总结

```
排序问题的完整框架：

定义层
  └─ 给定 n 个记录，找排列 f 使得：

$$
K_{f(i-1)} \leq K_{f(i)}
$$

性质层
  ├─ 稳定性：相同 key 的相对位置保持
  │  └─ 应用：多关键字排序必须用稳定排序
  ├─ 内外排：看数据存储位置和 I/O 瓶颈
  └─ 评估指标：比较次数 + 移动次数

选择层
  └─ 根据数据特点选择合适的算法
```

> 1. 稳定性的定义与多关键字排序的关系
> 2. 各种排序算法的稳定性判断
> 3. 内排 vs 外排的本质区别
> 4. 评估指标的独立性（比较次数少 ≠ 移动次数少）
