这六页介绍选择排序（Selection Sort），从基本思想、详细演示到复杂度分析再到代码实现，建立对选择排序的完整理解。

## 第051页：Selection Sorting Overview（"选择排序概览"）

### 选择排序的两种实现

**1. 简单选择排序（Simple Selection Sort）**
- 直接扫描找最小值
- 时间复杂度 O(n²)
- 不稳定

**2. 优先队列实现（Priority Queue Implementation）**
- 使用堆结构加速
- 每次从堆中提取最小元素
- 时间复杂度 O(n log n)
- 稳定性取决于堆的实现

### 核心思想

**选择排序的本质**：每次都从未排序部分**选出最小元素**，放到已排序部分的末尾。

> 与插入排序对比：
> - 插入排序：取出一个元素，找合适位置插入
> - 选择排序：找最小元素，放到当前位置

## 第052页：Simple Selection Sort（"简单选择排序的定义"）

### 标记：Not Stable（不稳定）

这是选择排序的关键特性，后面会详细说明。

### 算法描述

给定一个长度为 **n** 的数组，重复以下步骤直到数组有序：

#### 第 1 轮

```
搜索元素 0 到 n-1，找出最小元素
交换最小元素与位置 0 的元素
```

#### 第 2 轮

```
搜索元素 1 到 n-1，找出最小元素
交换最小元素与位置 1 的元素
```

#### 第 3 轮

```
搜索元素 2 到 n-1，找出最小元素
交换最小元素与位置 2 的元素
```

#### 继续...

```
按上述方式继续进行，直到：
搜索元素 n-2 到 n-1，交换后就完成了
```

> 注意：最后一轮只需要检查最后两个元素。

### 伪代码

```
SelectionSort(array):
    for i = 0 to n-2:
        // 找从 i 到 n-1 中的最小值
        minIndex = i
        for j = i+1 to n-1:
            if array[j] < array[minIndex]:
                minIndex = j
        
        // 交换
        swap(array[i], array[minIndex])
```

### 关键观察

**"Already in place" 的元素**：
- 如果最小值本身就在位置 i，那么交换是自身与自身的交换
- 这是无害的，可以不检查就执行
- 不需要特殊处理

```
选择排序的特点：
  ├─ 每次确定一个元素的最终位置
  ├─ 不需要递归
  ├─ 总是 O(n²)（无论输入如何）
  └─ 相对"均匀"的工作分布

快速排序的特点：
  ├─ 每次确定多个元素的相对位置
  ├─ 利用分治和递归
  ├─ 平均 O(n log n)，最坏 O(n²)
  └─ 工作分布不均（大多数工作在上层）
```

## 第055页：Analysis of Selection Sort（"选择排序的复杂度分析"）

### 关键性质

**选择排序可能会和自身交换**：
- 如果最小值已在目标位置，就会出现自身交换
- 这是无害的（值不变），可以不特殊处理
- 算法不需要检查以避免这种情况

### 复杂度分析

#### 外层循环

```
for i = 0 to n-2:  // 执行 n-1 次
    ...
```

**次数**：**n-1 次**（最后一个元素不需要处理）

#### 内层循环

```
for j = i+1 to n-1:  // 从 i+1 到 n-1
    if array[j] < array[minIndex]:
        minIndex = j
```

**每轮的比较次数**：

| 轮次 i | 内层循环范围 | 比较次数 |
|-------|-----------|--------|
| 0 | 1 到 n-1 | n-1 |
| 1 | 2 到 n-1 | n-2 |
| 2 | 3 到 n-1 | n-3 |
| ... | ... | ... |
| n-2 | n-1 到 n-1 | 1 |

**平均情况**：每轮平均比较 **n/2** 次

### 工作量计算

**比较总数**：

$$
(n-1) + (n-2) + \cdots + 1 = \frac{(n-1)n}{2} = O(n^2)
$$

**交换次数**：

$$
\text{最多 } n-1 \text{ 次交换（每轮至多 1 次）}
$$

**总时间复杂度**：

$$
T(n) = (n-1) \times \frac{n}{2} = O(n^2)
$$

### 关键结论

| 指标 | 值 | 说明 |
|-----|-----|------|
| **比较次数** | Θ(n²) | 总是 n(n-1)/2 次，无论输入 |
| **交换次数** | O(n) | 最多 n-1 次 |
| **总时间复杂度** | **Θ(n²)** | 固定，不依赖输入 |
| **空间复杂度** | O(1) | 原地排序 |
| **稳定性** | ❌ 不稳定 | 交换可能改变相同元素的相对位置 |

### 与其他 O(n²) 算法的对比

| 算法 | 最好情况 | 平均情况 | 最坏情况 | 稳定性 | 移动次数 |
|-----|--------|--------|--------|-------|--------|
| **Insertion Sort** | O(n) | O(n²) | O(n²) | ✅ 稳定 | O(n²) |
| **Bubble Sort** | O(n) | O(n²) | O(n²) | ✅ 稳定 | O(n²) |
| **Selection Sort** | O(n²) | O(n²) | O(n²) | ❌ 不稳定 | **O(n)** |

> 💡 **选择排序的优势**：虽然比较次数多，但**交换次数最少**（只有 O(n)）。在交换操作昂贵的场景下，选择排序可能更快。

### 为什么不稳定？

```
例：[3ₐ, 2, 3ᵦ]

第0轮：找最小值 2，与位置 0 的 3ₐ 交换
       [2, 3ₐ, 3ᵦ]

第1轮：在 3ₐ 和 3ᵦ 中找最小值
       两者相等，可能交换 3ₐ 和 3ᵦ
       [2, 3ᵦ, 3ₐ]  ← 相对位置改变！

结果：原来 3ₐ 在 3ᵦ 前，现在反了。
      这说明选择排序不稳定。
```

## 第056页：Algorithm for Simple Selection Sorting（"选择排序的代码实现"）

### 完整代码

```c
void SelectSort(SqList &L)
{
    int i, j;
    
    for(i=1; i<L.length; ++i)
    {
        j = SelectMinKey(L, i);
        
        if(i != j)
            swap(L.r[i], L.r[j]);
    }
}
```

### 代码逐行精讲

#### 函数签名

```c
void SelectSort(SqList &L)
```

**参数**：顺序表的引用，原地排序

> 初始调用：`SelectSort(L);`

#### 变量声明

```c
int i, j;
```

- `i`：当前位置（已排序部分的末尾）
- `j`：最小值所在的位置

#### 外层循环

```c
for(i=1; i<L.length; ++i)
```

**循环范围**：从 `i=1` 到 `i=L.length-1`

**原因**：
- i=1 表示已有 1 个元素在位置 0（天然有序）
- 从位置 1 开始逐步扩展已排序部分
- 最后一次 i=L.length-1 时，只剩最后一个元素

#### 找最小值

```c
j = SelectMinKey(L, i);
```

**作用**：调用辅助函数找从位置 `i` 到末尾中的最小值位置

**返回值**：最小值所在的位置（下标 j）

#### 优化判断

```c
if(i != j)
    swap(L.r[i], L.r[j]);
```

**逻辑**：
- 如果最小值已在位置 i，就不需要交换
- 这是一个小优化，避免无意义的自身交换
- 虽然自身交换无害，但这个检查可以省略（对正确性无影响）

> 注意：即使没有这个 if 判断，算法仍然正确，只是多了一个无用的交换。

### 辅助函数：SelectMinKey

这个函数在代码中被调用但未实现，它的作用是找最小值位置：

```c
int SelectMinKey(SqList &L, int startPos)
{
    int minPos = startPos;
    
    for(int k = startPos + 1; k < L.length; ++k)
    {
        if(L.r[k].key < L.r[minPos].key)
            minPos = k;
    }
    
    return minPos;
}
```

**逻辑**：
- 初始化最小值位置为 startPos
- 从 startPos+1 开始扫描到末尾
- 每次找到更小的值就更新最小值位置
- 返回最终的最小值位置

### 完整流程示例

**数组**：`[8, 3, 2, 1, 7, 4, 6, 5]`

```
第1次循环（i=1）：
  SelectMinKey(L, 1) 返回 3（元素 1 的位置）
  交换 L.r[1] 和 L.r[3]
  结果：[8, 1, 2, 3, 7, 4, 6, 5]

第2次循环（i=2）：
  SelectMinKey(L, 2) 返回 2（元素 2 的位置）
  i==j，不交换
  结果：[8, 1, 2, 3, 7, 4, 6, 5]

第3次循环（i=3）：
  SelectMinKey(L, 3) 返回 5（元素 3 的位置）
  交换 L.r[3] 和 L.r[5]
  结果：[8, 1, 2, 3, 7, 4, 6, 5]

...继续...

最后：[1, 2, 3, 4, 5, 6, 7, 8]
```

### 时间复杂度

| 操作 | 次数 | 说明 |
|-----|-----|------|
| 外层循环 | n-1 次 | 从 i=1 到 L.length-1 |
| 内层循环（SelectMinKey） | 平均 n/2 次 | 第1次 n-1，第2次 n-2...最后 1 次 |
| 交换 | O(n) | 最多 n-1 次 |
| **总比较** | **(n-1)·n/2** | **O(n²)** |

### 代码特点

| 特性 | 值 | 说明 |
|-----|-----|------|
| **时间复杂度** | O(n²) | 稳定，不依赖输入 |
| **空间复杂度** | O(1) | 原地排序，只用了 i、j 两个变量 |
| **稳定性** | ❌ 否 | 交换操作破坏相同元素相对位置 |
| **最少代码行数** | 仅 6 行 | 非常简洁 |

### 与插入排序代码的对比

```c
// 插入排序
for(i=2; i<=n; i++) {
    if(L.r[i].key < L.r[i-1].key) {
        ...移位...
    }
}

// 选择排序
for(i=1; i<n; i++) {
    j = SelectMinKey(L, i);  // 找最小值
    if(i != j) swap(...);     // 交换到位
}
```

**对比**：
- 插入排序：找合适位置，移位插入
- 选择排序：找最小值，交换到位

## 核心总结

```
选择排序的完整框架：

基本思想：
  ├─ 每次从未排序部分找最小元素
  ├─ 放到已排序部分的末尾
  └─ 重复直到整个数组有序

算法特点：
  ├─ 时间复杂度：总是 O(n²)，独立于输入
  ├─ 空间复杂度：O(1)（原地排序）
  ├─ 稳定性：不稳定
  └─ 交换次数：O(n)（比其他 O(n²) 少）

性能对比：
  ├─ vs 插入排序：最坏情况相同，但插入排序最好情况更好
  ├─ vs 快速排序：平均情况快排更快（O(n log n) vs O(n²)）
  └─ vs 堆排序：使用优先队列实现可达 O(n log n)

应用场景：
  ├─ 小数据集
  ├─ 交换操作代价高，比较代价低的场景
  └─ 不需要稳定性的排序
```
