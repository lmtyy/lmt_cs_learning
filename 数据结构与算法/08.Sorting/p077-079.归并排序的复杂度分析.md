# 归并排序的复杂度分析（p077-079）

这三页通过严格的数学推导证明归并排序的时间复杂度为 O(n log n)，并讨论其在实际应用中的限制。

---

## 第077页：Analysis of MergeSort - Setup（"归并排序分析：建立递推关系"）

### 问题设定

设 $T(n)$ 为对 $n$ 个元素执行 MergeSort 的时间。

### 假设

- 假设 $n$ 是 **2 的幂**（$n = 2^k$），这样每次都能等分成两半
- 一般情况可以类似分析，结论相同

### 基础情况

$$
T(1) = 1
$$

> 对 1 个元素的数组排序，时间为常数。

### 递推关系的建立

对于 $n > 1$，MergeSort 的时间包含三部分：
1. 对左半部分排序：$T(n/2)$
2. 对右半部分排序：$T(n/2)$
3. 合并两个有序部分：$O(n)$（线性时间）

因此：

$$
T(n) = 2T(n/2) + n
$$

> 💡 **这就是归并排序的核心递推式**。接下来只需要求解它。

---

## 第078页：Analysis of MergeSort - Solving（"归并排序分析：求解递推"）

### 递推式

$$
T(n) = 2T(n/2) + n
$$

### 求解方法：除以 n 消去系数

#### 第 1 步：两边除以 n

$$
\frac{T(n)}{n} = \frac{T(n/2)}{n/2} + 1
$$

#### 第 2 步：将 n 替换为 n/2（递推展开）

$$
\frac{T(n/2)}{n/2} = \frac{T(n/4)}{n/4} + 1
$$

#### 第 3 步：继续替换

$$
\frac{T(n/4)}{n/4} = \frac{T(n/8)}{n/8} + 1
$$

### 展开模式

每次展开都多出一个 "+1"：

$$
\frac{T(n)}{n} = \frac{T(n/2)}{n/2} + 1
$$

$$
= \frac{T(n/4)}{n/4} + 1 + 1
$$

$$
= \frac{T(n/8)}{n/8} + 1 + 1 + 1
$$

$$
= \cdots
$$

> 每一步 n 除以 2，直到 n 变成 1。

---

## 第079页：Analysis of MergeSort - Conclusion（"归并排序分析：最终结论"）

### 递推的终止

一直展开到 $n = 2$ 时：

$$
\frac{T(2)}{2} = \frac{T(1)}{1} + 1
$$

由于 $T(1) = 1$，所以：

$$
\frac{T(2)}{2} = 1 + 1 = 2
$$

### 共展开了多少步？

$$
n \to n/2 \to n/4 \to \cdots \to 2 \to 1
$$

共除了 $\log_2 n$ 次。因此：

$$
\frac{T(n)}{n} = \frac{T(1)}{1} + \log_2 n = 1 + \log_2 n
$$

### 最终结论

$$
\frac{T(n)}{n} = \log_2 n + 1
$$

两边乘以 $n$：

$$
T(n) = n \log_2 n + n = O(n \log n)
$$

### 核心公式

$$
\boxed{T(n) = n \log_2 n + n = O(n \log n)}
$$

### 归并排序的实际应用限制

虽然 MergeSort 是 O(n log n) 的算法，但**在主存排序中很少使用**，原因是：

$$
\text{需要线性额外内存（O(n)）}
$$

### 具体分析

```
为什么额外内存是致命缺点？

内存排序场景：
  ├─ 数据量大时（如 100 万个元素）
  ├─ 需要额外 100 万个元素的空间
  ├─ 内存消耗翻倍
  └─ 对于内存敏感的系统不可接受

对比：
  ├─ QuickSort：O(log n) 额外空间（递归栈）
  ├─ HeapSort：O(1) 额外空间（原地）
  └─ MergeSort：O(n) 额外空间（临时数组）← 最大
```

### 适合 MergeSort 的场景

尽管内存开销大，MergeSort 在以下场景中仍是首选：

| 场景 | 原因 |
|-----|------|
| **外排序** | 磁盘 I/O 是瓶颈，MergeSort 的顺序访问模式最友好 |
| **链表排序** | 链表不需要额外数组（合并操作只需改指针） |
| **需要稳定性** | MergeSort 是唯一 O(n log n) 的稳定比较排序 |
| **并行计算** | MergeSort 易于并行化（左右子问题独立） |

## 完整推导总结

```
归并排序复杂度推导的完整链条：

递推式：T(n) = 2T(n/2) + n

除以 n：T(n)/n = T(n/2)/(n/2) + 1

递推展开：
  T(n)/n     = T(n/2)/(n/2) + 1
  T(n/2)/(n/2) = T(n/4)/(n/4) + 1
  T(n/4)/(n/4) = T(n/8)/(n/8) + 1
  ...
  T(2)/2     = T(1)/1 + 1

累加 log₂n 个 "+1"：
  T(n)/n = T(1)/1 + log₂n = 1 + log₂n

结论：T(n) = n(1 + log₂n) = n log₂n + n = O(n log n)
```

### 各排序算法的最终对比

| 算法 | 最好 | 平均 | 最坏 | 空间 | 稳定 |
|-----|------|------|------|------|------|
| Insertion Sort | O(n) | O(n²) | O(n²) | O(1) | ✅ |
| Bubble Sort | O(n) | O(n²) | O(n²) | O(1) | ✅ |
| Shell Sort | O(n) | O(n^1.5) | O(n²) | O(1) | ❌ |
| Selection Sort | O(n²) | O(n²) | O(n²) | O(1) | ❌ |
| Quick Sort | O(n log n) | O(n log n) | O(n²) | O(log n) | ❌ |
| Heap Sort | O(n log n) | O(n log n) | O(n log n) | O(1) | ❌ |
| **Merge Sort** | **O(n log n)** | **O(n log n)** | **O(n log n)** | **O(n)** | **✅** |
