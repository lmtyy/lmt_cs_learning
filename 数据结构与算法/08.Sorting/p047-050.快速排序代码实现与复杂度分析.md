# 快速排序的代码实现与复杂度分析（p047-050）

这四页从代码实现层面讲解快速排序的核心函数，分析其最坏情况和平均情况的复杂度，建立对性能的深层理解。

---

## 第047页：QuickSort Recursive Function（"快速排序的递归实现"）

### 主函数：QSort

```c
void QSort(SqList &L, int low, int high)
{
    if(low < high)
    {
        pivotloc = Partition(L, low, high);
        QSort(L, low, pivotloc-1);
        QSort(L, pivotloc+1, high);
    }
}  // QSort
```

### 代码逐行精讲

#### 函数签名

```c
void QSort(SqList &L, int low, int high)
```

**参数说明**：
- `SqList &L`：顺序表的引用（修改原数组）
- `int low`：当前子数组的左边界（包含）
- `int high`：当前子数组的右边界（包含）

> 初始调用：`QSort(L, 1, L.length)`

#### 基础情况（递归终止）

```c
if(low < high)
```

**逻辑**：
- 如果 `low >= high`，说明子数组只有 0 或 1 个元素，天然有序
- 无需进一步处理，函数返回

```
low == high：1 个元素 → 有序 ✓
low > high：0 个元素（不会出现）
```

#### 分割操作

```c
pivotloc = Partition(L, low, high);
```

**作用**：
- 调用 Partition 函数分割数组
- 返回值 `pivotloc` 是 pivot 的最终位置
- **重要**：pivot 现在在正确的位置，不需要再比较它

#### 左递归

```c
QSort(L, low, pivotloc-1);
```

**处理**：递归排序 pivot 左边的所有元素

**范围**：`[low, pivotloc-1]`（不包括 pivot 本身）

> 这些元素都 ≤ pivot

#### 右递归

```c
QSort(L, pivotloc+1, high);
```

**处理**：递归排序 pivot 右边的所有元素

**范围**：`[pivotloc+1, high]`（不包括 pivot 本身）

> 这些元素都 > pivot

### 递归树

```
QuickSort(L, 1, 8)
    │
    ├─ Partition(...) → pivotloc = 5
    ├─ QuickSort(L, 1, 4)        [左子问题]
    └─ QuickSort(L, 6, 8)        [右子问题]
         │
         ├─ Partition(...) → pivotloc = 2
         ├─ QuickSort(L, 1, 1)    [递归终止]
         └─ QuickSort(L, 3, 4)
              │
              ├─ Partition(...)
              ├─ ...
```

### 时间复杂度直观

```
阶段        工作量
─────────────────────
第1层      O(n)        ← 第一次 Partition 处理所有 n 个元素
第2层      O(n)        ← 两个 Partition，共处理 n 个元素
第3层      O(n)        ← 四个 Partition，共处理 n 个元素
...
第 h 层    O(n)        ← 树高为 h 层

总工作量 = O(n × h)
  ├─ 最好情况（平衡分割）：h = O(log n) → O(n log n)
  ├─ 最坏情况（极端分割）：h = O(n) → O(n²)
  └─ 平均情况（随机分割）：h = O(log n) → O(n log n)
```

## 第048页：Partition Function（"分割函数的实现"）

### 完整代码

```c
int Partition(SqList &L, int low, int high)
{
    // 交换顺序表L中子表L.r[low...high]的记录
    // 使枢轴记录到位
    
    L.r[0] = L.r[low];
    pivotkey = L.r[low];
    
    while(low < high)
    {
        // 从右向左找第一个 < pivotkey 的元素
        while(low < high && L.r[high].key >= pivotkey)
            --high;
        L.r[low] = L.r[high];  // 将 high 处小的元素移至 low 处
        
        // 从左向右找第一个 > pivotkey 的元素
        while(low < high && L.r[low].key <= pivotkey)
            ++low;
        L.r[high] = L.r[low];  // 将 low 处大的元素移至 high 处
    }
    
    L.r[low] = L.r[0];  // 将 pivot 放入最终位置
    return low;
}
```

### 代码精讲

#### 哨兵保存

```c
L.r[0] = L.r[low];
pivotkey = L.r[low];
```

**作用**：
- `L.r[0]`：保存 pivot 元素（成为哨兵）
- `pivotkey`：保存 pivot 的键值，用于比较

> 这样做避免了 pivot 在移位过程中丢失，类似于插入排序中的哨兵技巧。

#### 外层循环（双指针扫描）

```c
while(low < high)
{
    ...
}
```

**条件**：`low < high` 保证两个指针没有交错

**流程**：
1. 每次循环处理一对违规元素
2. 进行一次或两次移位
3. 指针向中间靠拢

#### 从右向左的扫描

```c
while(low < high && L.r[high].key >= pivotkey)
    --high;
L.r[low] = L.r[high];  // 将 high 处小的元素移至 low 处
```

**逻辑**：
- 从右开始，找第一个 **小于** pivotkey 的元素
- 停止条件 1：找到了这样的元素（`L.r[high].key < pivotkey`）
- 停止条件 2：指针相遇（`low >= high`）
- 找到后，把它移到 `L.r[low]`

**目的**：把"不该在右边"的小元素移向左边

#### 从左向右的扫描

```c
while(low < high && L.r[low].key <= pivotkey)
    ++low;
L.r[high] = L.r[low];  // 将 low 处大的元素移至 high 处
```

**逻辑**：
- 从左开始，找第一个 **大于** pivotkey 的元素
- 停止条件 1：找到了这样的元素（`L.r[low].key > pivotkey`）
- 停止条件 2：指针相遇（`low >= high`）
- 找到后，把它移到 `L.r[high]`

**目的**：把"不该在左边"的大元素移向右边

#### Pivot 落位

```c
L.r[low] = L.r[0];
return low;
```

**过程**：
- 当 `low == high` 时，外层循环结束
- `low`（或 `high`，此时相等）就是 pivot 的最终位置
- 把哨兵中的 pivot 放回

**结果**：
- 位置 `low` 左边的所有元素都 ≤ pivot
- 位置 `low` 右边的所有元素都 > pivot
- pivot 在正确的最终位置

### 分割示例（复习）

```
原数组：[46, 55, 13, 42, 94, 05, 17, 70]
pivot = 46, pivotkey = 46

步骤 1：从右找 < 46 的：找到 17（位置 7）
       L.r[1] = 17 → [46, 17, 13, 42, 94, 05, 55, 70]

步骤 2：从左找 > 46 的：找到 55（位置 2）
       L.r[7] = 55 → [46, 17, 13, 42, 94, 05, 55, 55]

...（继续扫描）...

最终：[17, 05, 13, 42, 46, 94, 55, 70]
              ↑
         pivot 到位
```

## 第049页：Worst-Case Analysis（"最坏情况分析"）

### 最坏情况的发生条件

**条件 1**：Pivot 选择不当 ❌
- 总是选择第一个元素作为 pivot
- 这在预先排序的数据中特别糟糕

**条件 2**：输入数据已排序 📊
- 如果输入数组已经完全升序排列
- 每次分割都把 pivot 放在最后位置

### 具体例子

**输入**：`[1, 2, 3, 4, 5, 6, 7, 8]`（已升序）

**Pivot 选择**：总是第一个元素

```
第1次：pivot = 1
  分割结果：[] 1 [2,3,4,5,6,7,8]
  左子问题：空
  右子问题：7 个元素
  
第2次：pivot = 2
  分割结果：[] 2 [3,4,5,6,7,8]
  左子问题：空
  右子问题：6 个元素
  
...依此类推...

第7次：pivot = 7
  分割结果：[] 7 [8]
  左子问题：空
  右子问题：1 个元素
```

### 递归树退化成链

```
QuickSort(L, 1, 8)
    │
    └─ Partition(...) → 1 (最后位置)
       │
       └─ QuickSort(L, 2, 8)
          │
          └─ Partition(...) → 2 (最后位置)
             │
             └─ QuickSort(L, 3, 8)
                │
                └─ ... (继续单边递归)
```

> 树高度变成 O(n) 而不是 O(log n)！

### 复杂度推导

设 $T(n)$ 为排序 $n$ 个元素的时间，则：

$$
T(n) = T(n-1) + O(n)
$$

**递推关系**：
- 每次 Partition 都处理 $n$ 个元素
- 左子问题：0 个元素
- 右子问题：$n-1$ 个元素

**展开递推**：

$$
T(n) = T(n-1) + cn
$$

$$
= T(n-2) + c(n-1) + cn
$$

$$
= T(n-3) + c(n-2) + c(n-1) + cn
$$

$$
= \cdots
$$

$$
= T(1) + c(2 + 3 + \cdots + n)
$$

$$
= c \sum_{i=2}^{n} i = c \cdot \frac{n(n+1)}{2} - c
$$

$$
= O(n^2)
$$

### 关键结论

**最坏情况的讽刺**：

> QuickSort takes $O(n^2)$ time to do absolutely nothing!

**解释**：
- 即使每次分割都把 pivot 放在最终位置
- 但由于选择的总是最小元素
- 数组实际上已经排好序了
- 算法却花了 O(n²) 的时间

这说明 **pivot 的选择至关重要**。

## 第050页：Average-Case Analysis（"平均情况分析"）

### 假设：随机 Pivot 选择

我们假设每次都**随机选择** pivot（而不是总是第一个元素）。

### 递推关系

设 $T(n)$ 为平均排序时间，则：

$$
T(n) = T(k) + T(n-1-k) + O(n)
$$

其中 $k$ 是 pivot 最终位置的期望值。

**理解**：
- Partition 耗时 O(n)
- 左子问题：$k$ 个元素
- 右子问题：$n-1-k$ 个元素

### 关键洞察

对于**随机 pivot**，任何位置都等概率出现：

$$
\mathbb{E}[k] = \frac{n}{2}
$$

这意味着，平均而言，pivot 会把数组分成两个大致相等的部分。

### 树高度分析

若每次都把问题分成两个大小为 $n/2$ 的子问题，则：

**递推树的高度**：

$$
h \text{ 满足 } \log_2 n < h < \log_2 n + 1
$$

或简记为：

$$
h = O(\log_2 n)
$$

### 时间复杂度

**平均情况下，QuickSort 的时间复杂度**：

$$
T(n) = O(n \log_2 n)
$$

**推导**：
- 树高度：O(log n) 层
- 每层处理的总工作量：O(n)
- 总时间：O(n × log n)

### 复杂度对比

| 情形 | Pivot 选择 | 复杂度 | 特点 |
|-----|----------|-------|------|
| **最坏情况** | 总是第一个 | O(n²) | 有序数组触发 |
| **最好情况** | 总是中位数 | O(n log n) | 完美平衡分割 |
| **平均情况** | 随机选择 | **O(n log n)** | 实践中最常见 |

> 💡 **实际应用**：随机选择 pivot 使得 O(n²) 最坏情况几乎不可能发生（概率接近 0）

## 核心总结

```
QuickSort 的完整框架：

递归结构：
  QSort(L, low, high)
    ├─ 检查基础情况（low < high）
    ├─ 调用 Partition 分割
    ├─ 递归处理左部分
    └─ 递归处理右部分

Partition 的核心逻辑：
  ├─ 保存 pivot（哨兵）
  ├─ 双指针从两端向中间扫描
  ├─ 交换违规元素
  └─ 放置 pivot 到最终位置

复杂度分析：
  ├─ 最坏：O(n²)（不推荐的 pivot 选择）
  ├─ 最好：O(n log n)（完美分割）
  └─ 平均：O(n log n)（随机选择）

关键洞察：
  ├─ Pivot 选择决定分割平衡性
  ├─ 随机选择使最坏情况极其罕见
  ├─ 实践中是最快的排序算法
  └─ 缓存友好且空间高效
```
