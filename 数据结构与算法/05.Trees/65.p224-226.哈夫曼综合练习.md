### 第一页（Slide 224）：数学理论与证明（必考定理）
**核心知识点：二叉树的数学性质与哈夫曼编码的数学证明**

*   **题目 4：证明一棵有 \(n\) 个节点的二叉树有 \(n+1\) 个空指针。**
    *   **解析**：这是数据结构中最经典的“铁律”之一。
    *   **证明过程**：
        1.  每个二叉树节点有 2 个指针域（`left` 和 `right`），所以 \(n\) 个节点的**总指针域**为 **\(2n\)**。
        2.  为了连接这 \(n\) 个节点成一棵树，必然使用了 **\(n-1\)** 个指针（即树的边数）。
        3.  因此，**空指针的数量 = 总指针域 - 被使用的指针数 = \(2n - (n-1) = n + 1\)**。
    *   *延伸思考*：正是因为有这 \(n+1\) 个空指针，我们才引出了**“线索二叉树”**，去利用这些空指针提升遍历效率。

*   **题目 5：克拉夫特不等式（Kraft Inequality）证明。**
    *   **公式**：\(\sum_{i=1}^{M} l_i \cdot 2^{-d_i} \le 1\)。
    *   **解析**：这是**哈夫曼树（最优前缀编码）**存在的**数学理论基石**。其中 \(l_i\) 代表某个叶子节点，\(d_i\) 代表该叶子节点的深度。
    *   **取等号的条件（当等式成立时）**：**当这棵树是一棵“满二叉树”（Full Binary Tree）时，等式正好等于 1。** 也就是说，如果树在每一层都紧密排列没有任何空洞，这个求和结果就是 1。在哈夫曼树的构建中，最终生成的树满足 \(=1\) 的情况。

### 第二页（Slide 225）：经典面试题——重构二叉树（🟡 必考大题）
**核心知识点：已知前序+中序，唯一确定二叉树**

*   **题目 6**：已知前序序列 `ABCDFGHIJKL`，中序序列 `BCDAGHFJLKI`。画出该二叉树。
*   **解题秘籍（“三步走”）**：
    1.  **前序定根**：前序第一个字符 `A` 必是根节点。
    2.  **中序定左右**：在中序串 `BCD A GHFJLKI` 中，找到 `A`。`A` 左边的 `BCD` 是左子树，右边的 `GHFJLKI` 是右子树。
    3.  **递归拆分**：
        *   **左子树 `BCD`（前序也是 BCD）**：根是 `B`。中序中 `B` 无左孩子，右孩子是 `CD`。继续拆分 `CD`，根是 `C`，右孩子是 `D`。
        *   **右子树 `GHFJLKI`（对应前序的 `FGHIJKL`）**：根是 `F`。中序中 `GH` 是 F 的左子树，`JLKI` 是 F 的右子树。
        *   **F的左子树 `GH`（前序 GH）**：根 `G`，右孩子 `H`。
        *   **F的右子树 `JLKI`（前序 I J K L，中序 J L K I）**：根 `I`。中序中 `JLK` 是 I 的左子树。继续拆分 `J L K`（前序 J K L，中序 J L K），根 `J`，右子树 `LK`。`L K` 的根是 `K`，左孩子是 `L`。
*   **最终得到的树结构**：
    *   根 `A`，左子树是一条向右的单链：`A左->B，B右->C，C右->D`。
    *   右子树比较复杂，基干是：`A右->F`。
    *   `F` 的左孩子是 `G`（G的右孩子是 `H`）。
    *   `F` 的右孩子是 `I`。`I` 的左孩子是一条单链 `J->K->L`（K是J的右孩子，L是K的左孩子）。

### 第三页（Slide 226）：哈夫曼树的实战与编程设想
**核心知识点：哈夫曼编码构建、理论证明与工程实现**

*   **题目 7：根据字符频率构建哈夫曼编码。**
    *   **考点**：这是典型的**手工建树**题。
    *   **解题思路**：把所有频率列出来（`colon(100)`, `space(605)`, `newline(100)`, `comma(705)`, `0(431)`...）。
    *   **算法执行**：反复执行**“找到当前频率最小的两个节点，合并成一个新节点（权重相加），并把新节点放回候选中”**。
    *   *考试防坑提示*：当权重相同时（比如 colon 和 newline 都是 100），选取顺序不同可能导致树的形状不同，但最终的**带权路径长度 WPL 一定是一样的**。

*   **题目 8：证明哈夫曼算法生成最优前缀编码。**
    *   **考点**：证明题。
    *   **核心证明思路**：证明需要用到 **“贪心选择性质”** 和 **“最优子结构性质”**。
        1.  **贪心性**：首先证明权重最小的两个叶子节点，必定在最深的底层，且互为兄弟。
        2.  **交换论证**：如果能证明每次合并两个最小节点，最后得到的 WPL 一定小于等于其他任意组合，就能证明它是全局最优的。

*   **题目 9：编写哈夫曼压缩/解压程序。**
    *   **考点**：工程代码设计。
    *   **核心实现逻辑**：
        *   **压缩过程**：
            1.  `统计频率`：遍历文件，统计每个字符的出现次数。
            2.  `构建哈夫曼树`：使用**最小堆（Min-Heap / 优先队列）**实现贪心合并，生成哈夫曼树。
            3.  `生成编码表`：通过**树的深度优先遍历（前序）**，生成每个字符对应的二进制编码（如 01, 110 等），存入哈希表（Map）。
            4.  `写入压缩文件`：按位（Bit）将字符替换为对应编码，并拼凑成字节（Byte）写入文件。必须把**频率表/编码表**写在压缩包头部，以便解压。
        *   **解压过程**：
            1.  读取压缩包头部的频率表/编码表，**重构哈夫曼树**。
            2.  按位读取压缩数据，从哈夫曼树的根节点开始走（0左、1右）。
            3.  一旦走到**叶子节点**，就输出该字符，并**立刻把指针重新指向根节点**，继续读取下一位。
