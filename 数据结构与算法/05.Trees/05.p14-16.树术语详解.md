### 第一页 (Slide 14)：基本关系与节点分类
这一页主要定义了 **“谁是谁的谁”** 以及 **“节点长什么样”**。

**1. 关系术语**
*   **Root（根节点）**：没有任何父节点的节点（图中为 **A**）。树只有一个根。
*   **Siblings（兄弟节点）**：拥有同一个父节点的节点互为兄弟。
    *   例：图中的 **B** 和 **C** 都是 **A** 的孩子，所以 B 和 C 是兄弟节点。

**2. 节点类型术语（非常重要！）**
*   **Internal node（内部节点 / 分支节点）**：**至少拥有 1 个孩子** 的节点（图中标为橙色的 **A, B, C, F**）。
    *   理解：它是树内部的分支点，向下有延伸。
*   **External node (leaf)（外部节点 / 叶子节点）**：**没有孩子** 的节点（图中标为绿色的 **E, I, J, K, G, H, D**）。
    *   理解：它位于树的末端，数据流的终点。

**3. 血缘关系术语**
*   **Ancestors（祖先）**：从一个节点向上追溯到根节点路径上的所有节点。包括父节点、祖父节点、曾祖父节点等。
    *   例：图中 **K** 的祖先是 **F、B、A**。（通常不包含自己）。
*   **Descendant（后代）**：从一个节点向下延伸的所有子节点。包括孩子、孙子、曾孙等。
    *   例：图中 **B** 的后代是 **E、F、I、J、K**。

### 第二页 (Slide 15)：树的核心维度指标
这一页定义了衡量树“大小”、“高度”和“延伸度”的量化指标。**注意：不同的教材对 Depth/Height 的定义（是否从0开始计数）可能有细微差别，请以你当前这门课老师的定义为准（此PPT给出了明确公式）。**

**1. 距离指标**
*   **Depth（深度）**：**节点祖先的数量**（或者说**边数**，即从根走到该节点需要走几步）。
    *   例：在图里，A 没有祖先 -> 深度为 0。B 的祖先是 A（1个） -> 深度为 1。K 的祖先是 F, B, A（共3个） -> 深度为 3。（注：有些教材深度从1算起，但这里明确是“数量”，所以根是0）。
*   **Height（树的高度）**：**树中所有节点深度的最大值**。
    *   例：这棵树最深的是 **I, J, K**，深度是 3。所以整个树的高度 = 3。

**2. 扩展度指标**
*   **Degree of a node（节点的度）**：该节点**拥有的孩子数量**（不是有多少个分支，是有几个孩子）。
    *   例：A 有 B,C,D 三个孩子 -> 度为 3。F 有 I,J,K 三个孩子 -> 度为 3。E 没有孩子 -> 度为 0。
*   **Degree of a tree（树的度）**：**整棵树中，所有节点的度数的最大值**。
    *   例：A 度为3，F 度为3，C 度为2，其他为0或1。最大值为 3，所以这是一棵 **3 叉树**（三元树）。

**3. 结构术语**
*   **Subtree（子树）**：树中任意一个节点及其所有的后代所构成的结构，本身也是一棵树。
    *   图中画了 **C 节点及其后代（G, H）** 作为一个整体。它可以被视作以 C 为根的一棵独立的小树。这是**递归定义**的物理表现形式。

### 第三页 (Slide 16)：路径 (Path) 的数学定义
这一页给出了“路径”的严谨数学描述，这在计算树的性能和时间复杂度时非常重要。

**1. 路径的定义**
*   **Path（路径）**：一个节点序列 \(n_1, n_2, ..., n_k\)。其中，每一个节点 \(n_i\) 都是下一个节点 \(n_{i+1}\) 的**父节点**。
*   **Path Length（路径长度）**：这条路径上**边的数量**，等于 **\(k - 1\)**（也就是经过的步数）。
    *   例：从 A 走到 K：路径是 A -> B -> F -> K。k = 4，路径长度 = 4 - 1 = **3**。

**2. 特殊情况：零长度路径**
*   *“There is a path of length **zero** from every node to itself.”*
*   解释：每个节点到它自身也视为一条路径，长度为 0。

**3. 树的核心特性（这个非常重要！）**
*   *“Notice that in a tree there is **exactly one** path from the root to each node.”*
*   **解释**：**在树中，从根节点到任何一个其他节点，有且仅有一条路径。**
*   **深层含义**：这是树和“图（Graph）”最大的区别。如果在树里某两个节点之间有两条不同的路可以走，那它就不是树（说明存在环路）。树是一种**无环的连通图**。

### 💡 助记与复习建议

这 14-16 页的内容非常密集，但逻辑极其清晰。你可以这样记忆：

*   **身份**：看自己有没有爹（**根**），有没有娃（**内部**还是**叶子**）。
*   **亲戚**：同一个爹的是**兄弟**，往上数是**祖先**，往下数是**后代**。
*   **尺寸**：往上走几步是**深度**，最深的是**高度**；有几个娃叫**度**，最大的度叫**树的度**。
*   **路径**：走几步就是**长度**，而且从根往下走，**只有一条唯一的路**。

**考试题预测**：老师大概率会给你一棵类似第14页的图，让你填空：“节点X的祖先是谁？”、“节点X的度是多少？”、“树的高度是多少？”、“树的度是多少？”、“叶子节点有哪些？”、“根节点是哪个？”等等。吃透这三张图，这种送分题就能轻松拿下。
