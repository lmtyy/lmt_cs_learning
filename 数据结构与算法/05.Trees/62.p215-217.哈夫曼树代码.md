### 第一页（Slide 215）：【合并动作】`CreateBT()` 函数
**核心知识点：如何将两个子节点合并为一个父节点**

这个函数的作用是**创建一个新的内部节点**，挂载两个给定的子树。

*   **参数说明**：传入哈夫曼树数组 `HT`，新节点的下标 `NR`，以及两个要合并的子节点下标 `l`（左）和 `r`（右）。
*   **代码逐行拆解**：
    1.  `HT[NR].weight = HT[l].weight + HT[r].weight;`
        *   *(📝 纠错：PPT 此处有一个笔误，写成了 `HR[r].weight`，正确的应该是 `HT[r].weight`)*。新节点的权重等于左右两个孩子权重相加。
    2.  `HT[NR].parent = 0;`
        *   新节点刚产生，目前是游离的“根节点”，所以它的父节点暂定为 0。
    3.  `HT[NR].lchild = l;` `HT[NR].rchild = r;`
        *   将传入的 `l` 和 `r` 作为新节点的左孩子和右孩子。
    4.  `HT[l].parent = NR; HT[r].parent = NR;`
        *   **最核心的一步**：把两个旧孩子的父节点指针，都指向刚刚创建的新节点 `NR`。这就意味着这两棵树**“从森林里被移除，并加入了新树中”**。

### 第二页（Slide 216）：【初始化环境】`Huffman()` 函数第一部分
**核心知识点：动态分配数组，并初始化所有叶子节点**

*   **代码解析**：
    ```c
    HT = (HTNode *)malloc(2 * n * sizeof(HTNode)); 
    ```
    *   **为什么分配 `2*n` 大小的数组？** 数学上，哈夫曼树如果有 \(n\) 个叶子节点，则总节点数是 \(2n - 1\)。分配 `2*n` 是为了保证数组下标从 1 开始也能存得下所有的节点（下标 1 到 \(2n-1\)）。
*   **初始化叶子节点（绿色高亮部分）**：
    ```c
    for (i=1; i<=n; i++) {
        HT[i].weight = WA[i];            // 把用户传入的权重（频率）存入前 n 个位置
        HT[i].parent = 0;                // 初始状态下，所有叶子都是独立的根
        HT[i].lchild = HT[i].rchild = 0; // 初始状态下，叶子没有孩子
    }
    ```
*   **💡 工程细节**：PPT 特意强调注释 `//weights to n leaf nodes n save to array WA`。意思就是：**数组的前 \(n\) 个位置只放原始字符（叶子），代码里下标 1 到 \(n\) 都是初始节点。** 数组后面的位置（\(n+1\) 到 \(2n-1\)）将用来存放合并出的**内部节点**。

### 第三页（Slide 217）：【循环合并】`Huffman()` 函数第二部分
**核心知识点：\(n-1\) 次循环，构建内部节点**

*   **循环条件**：`for (i=1; i<n; i++)`
    *   如果有 \(n\) 个叶子节点，就要循环 **\(n-1\) 次**，构造出 \(n-1\) 个内部节点。
*   **第一步：挑选两个最小的（`Select`）**
    ```c
    Select(HT, n+i-1, l, r);
    ```
    *   这行代码调用了上一页 PPT（Slide 214）写的 `Select` 函数。
    *   `n+i-1` 代表当前“活着的”节点总数。
    *   函数执行后，会把当前森林中权重最小且没有父节点（`parent==0`）的两个节点的下标，赋给变量 `l` 和 `r`。
*   **第二步：合并（`CreateBT`）**
    ```c
    CreateBT(HT, n+i, l, r);
    ```
    *   调用 Slide 215 的函数进行合并。
    *   仔细看新节点的下标是 `n+i`。
        *   第一轮合并（`i=1`），新节点存到 `n+1` 的位置。
        *   第二轮合并（`i=2`），新节点存到 `n+2` 的位置。
        *   ……以此类推。
*   **循环结束**：经过 \(n-1\) 次循环后，所有的叶子都合并成了一棵完整的树，最后新生成的那个节点（下标为 `2n-1`）也就是整棵哈夫曼树的**根节点**。

### 💡 总结：学完这三页，你必须明白的“哈夫曼数组构建”精髓

1.  **下标分配规则**：哈夫曼树在数组里是 **“前 \(n\) 个存叶子，后面 \(n-1\) 个存内部节点”**。这是一个非常约定俗成的实现标准。
2.  **三个核心辅助函数协作**：
    *   `Select()`：负责**找对象**（挑出两个最小的游离树）。
    *   `CreateBT()`：负责**办婚礼**（把两个树合并，生成新节点）。
    *   `Huffman()`：负责**策划整场婚礼**（分配房间、跑 \(n-1\) 次循环）。
3.  **考试怎么考**：如果让你补全哈夫曼树构建的代码，一般会挖空 **`Select` 和 `CreateBT` 的调用部分**，或者考你**新生成的父节点下标是多少**（记住 `n+i` 这个公式即可满分）。
