### 第一页（Slide 104）：严格定义——什么是“完全二叉树”？
**核心知识点：它是满二叉树的“左侧连续截断版”**

*   **定义解读**：
    *   PPT明确说明，一棵拥有 \(n\) 个节点的二叉树，如果它的节点**严格按照满二叉树中“从1到 \(n\)”的编号顺序填充**，那么它就是**完全二叉树（Complete Binary Tree）**。
    *   **通俗理解**：想象把一棵“满二叉树”的最下面一层，**从最右边开始，依次删除**0个、1个、2个……节点。剩下的就是完全二叉树。
*   **🖼️ 看图对比（必须分清**）：
    *   **左图（Full binary tree）**：所有内部节点都有2个孩子，所有叶子都在同一层。**绝对的完美。**
    *   **右图（Complete binary tree）**：它有5个节点（编号1到5）。请仔细看它的“占位”方式——**它在节点5的下方空出了位置6和7，但绝对没有跳过节点5去放节点6。**
    *   **关键规矩（必考判断）**：如果一棵树有编号 1, 2, 3, 4, **6**（跳过了5），那它**绝对不是**完全二叉树。完全二叉树的填充必须是**“从左到右，紧密相连，中间无空档”**。

### 第二页（Slide 105）：两种存储结构——链表 vs 数组
**核心知识点：针对不同需求，有两种放树到内存的方法。**

*   **1. 链式结构（Linked structure）—— 通用型**
    *   这是我们前面 PPT (Slide 62) 讲过的经典结构体：`数据域 + left指针 + right指针`。
    *   **优点**：极其灵活，任意形状的树都能存。
    *   **缺点**：需要额外分配左右指针的内存空间，而且因为指针是随机分配的，在内存里是零散的，**容易出现 CPU 缓存未命中（Cache Miss），遍历效率相对较低**。

*   **2. 连续数组结构（Contiguous array structure）—— 专为完全二叉树打造**
    *   **存储方式**：`A[1] = 根节点`，`A[2], A[3] = A[1] 的孩子`，`A[4], A[5] = A[2] 的孩子`，`A[6], A[7] = A[3] 的孩子`。
    *   **优点**：
        1.  **极致省空间**：不需要存指针，只存数据，空间利用率 100%。
        2.  **速度极快**：符合计算机的**“局部性原理”**，放在连续的内存块里，CPU 缓存加载速度远快于指针跳跃。
        3.  **找父母/孩子不用指针**：直接通过我们前一页背的公式算：**`父 = i/2`，`左 = 2i`，`右 = 2i+1`**。

---

### 💡 重要考点提醒：为什么数组结构不用于普通树？
这是很多初学者容易迷糊的地方。**数组顺序存储的前提，必须是“完全二叉树”**。
*   **错误示范**：如果是一个单链形状的树（A -> B -> C，全部是右孩子），你用数组存：
    `A[1]=A`, `A[2]=B`, `A[3]=C`, `A[4],A[5]`, `A[6],A[7]` 就得全部留空，以此类推。
    *   **结果**：为了存 \(n\) 个节点，你可能会需要大小为 \(2^n\) 的巨型数组，**99.99% 的空间全是空的，纯属浪费内存**！
