这三页 PPT（221、222、223）是数据结构（二叉树）章节的**综合练习题**。它们覆盖面非常广，从**基础概念填空**、**递归算法设计**，一直到**表达式树的实战实现**。这通常是期末考试的经典题型组合。

下面我为您逐页详细拆解这三页 PPT 的题目与解题思路：

---

### 第一页（Slide 221）：二叉树基础与遍历综合题
**（这道题主要考察你对着具体的一棵树，能否快速准确写出各项属性）**

我们已知右侧的二叉树：根 A，左子树 B（带孩子 D、E，E 带左孩子 G），右子树 C（只带右孩子 F）。

*   **(1) The binary tree height（树的高度）**
    *   **答案：3**（基于前几页 PPT 中“边数”的定义）。最长路径是 `A -> B -> E -> G`，路径上有 3 条边。
*   **(2) List leaves（列出叶子节点）**
    *   **答案：D, G, F**。（叶子节点是没有孩子的节点。注意 E 有孩子 G，所以 E 不是叶子）。
*   **(3) List the full nodes（列出满节点，即有两个孩子的节点）**
    *   **答案：A, B**。（A 有 B 和 C；B 有 D 和 E。注意 C 只有 F 一个孩子；E 只有 G 一个孩子，所以不算满节点）。
*   **(4) List the children of node C（列出 C 的孩子）**
    *   **答案：F**。
*   **(5) The sequence of preorder, postorder, inorder and level-order traversals（四种遍历的序列）**
    *   **前序（根-左-右）**：`A, B, D, E, G, C, F`
    *   **中序（左-根-右）**：`D, B, G, E, A, C, F`
    *   **后序（左-右-根）**：`D, G, E, B, F, C, A`
    *   **层序（从上到下，从左到右）**：`A, B, C, D, E, F, G`
*   **(6) Add the threads of postorder（画出后序线索）**
    *   **解答思路**：找出“后序遍历序列”（D, G, E, B, F, C, A），把每个没有左孩子的节点的左边 **连向后继**，把每个没有右孩子的节点的右边 **连向前驱**。
    *   *考试关键点*：序列的**第一个节点 D**（没有前驱）和**最后一个节点 A**（没有后继），它们的线索通常需要指向**头结点**。

### 第二页（Slide 222）：树统计算法设计与时间复杂度分析
**（这道题考察你写递归算法的能力，以及分析算法效率的能力）**

要求：编写高效的函数，接收根节点 `T`，并且**给出它们的大 O 运行时间**。
*   **核心提示**：求解这种问题，**统一使用“递归后序遍历（左-右-根）”**，时间复杂度均为 **\(O(n)\)**（因为每个节点正好被访问一次）。

*   **(a) 计算叶子节点数（没有孩子的节点）**
    *   **解题思路**：`if (T == NULL) return 0;` `if (T->lchild==NULL && T->rchild==NULL) return 1;` `return (CountLeaves(T->lchild) + CountLeaves(T->rchild));`
*   **(b) 计算只有一个非空孩子的节点数**
    *   **解题思路**：判断左右两个孩子是否**“一空一非空”**。如果是，则返回 `1 + 递归左 + 递归右`；如果不是，则返回 `0 + 递归左 + 递归右`。
*   **(c) 计算有两个非空孩子的节点数**
    *   **解题思路**：判断 `T->lchild != NULL && T->rchild != NULL`。如果是，则返回 `1 + 递归左 + 递归右`。

### 第三页（Slide 223）：表达式树的完整功能设计（综合大项目）
**（这道题考察你能不能结合之前学过的表达式树、栈、哈希表，写一个简易的电脑计算器程序）**

*   **(a) 接受表达式，并生成二叉树**
    *   **做法**：使用我们之前 PPT 讲过的**后缀表达式（逆波兰式）加栈**的方法。先用栈把中缀转后缀，再通过另一个栈把后缀构建成表达式树。
*   **(b) 表达式可以包含变量**
    *   **做法**：建立一棵表达式树后，叶子节点除了存放常数（如 `2`, `3`），还需要存放**变量名**（如 `x`, `y`）。
*   **(c) 用户可以设置变量的值**
    *   **做法**：编写一个解析器，读取用户的输入如 `x = 5; y = 10;`，把这些变量名和对应的数字存到一张**哈希表（映射表，Map）**或者结构体数组中。
*   **(d) 对一组值进行表达式求值**
    *   **做法**：求值的核心就是调用我们之前写过的**后序遍历求值**函数。
    *   **流程**：当程序遇到叶子节点时，如果是常数直接返回；如果是变量，就去刚才的“哈希表”里查出它当前的值。如果是内部节点的运算符，就递归算出左值和右值，然后进行加减乘除运算。

### 💡 学习建议（如果这是你的期末复习）：
*   第 221 页的题目非常基础，**必须要手算无误**。特别是中序和后序，很容易搞错 E 和 G 的顺序，建议在草稿纸上老老实实画一遍递归路径。
*   第 222 页的代码是 **“递归万能公式”** 的典型应用。这类题目不要用全局变量计数，**一定要用 `return 左子树结果 + 右子树结果 + 自身判断`** 的写法，这是教科书的标准答案。
