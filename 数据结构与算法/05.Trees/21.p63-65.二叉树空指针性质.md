### 第一页（Slide 63）：与链表一致的动态内存管理规则
**核心知识点：连接“线性表”与“树”的底层代码逻辑**

*   **内容解析**：PPT 明确指出，链表中使用的许多内存管理规则，完全适用于二叉树。
*   **为什么一样？** 这是因为在 C 语言中，二叉树的节点结构体（Slide 62）本质上和双向链表非常相似，都是**“数据域 + 两个指针域”**。只不过链表的两个指针是“前驱”和“后继”，而二叉树是“左孩子”和“右孩子”。
*   **实际编程应用**：
    *   **插入节点时（Insertion）**：必须使用 `malloc` 函数在堆（Heap）中动态开辟内存来存放新节点。
    *   **删除节点后（Deletion）**：必须使用 `free` 函数手动释放被删除节点的内存，防止内存泄漏（Memory Leak）。
    *   *💡 提示*：如果你是在 C++ 或 Java 中写代码，`malloc`/`free` 对应的是 `new`/`delete`，或者依赖垃圾回收机制（GC），但底层的逻辑（动态分配与释放）是一样的。

### 第二页（Slide 64）：画图约定与 🌟 终极数学冷知识（n+1个空指针）
**核心知识点：视觉抽象与经典数学定理**

*   **画图方式的区别**：PPT 指出，链表通常用矩形框表示，而树通常用**圆圈**和线连接。这是因为树本质上是一种“图（Graph）”，用圆圈更能体现节点之间的拓扑关系。
*   **🌟 为什么不用画 NULL 指针？（红色重点）**
    *   PPT 提到了一个非常经典的数据结构定理：**拥有 \(n\) 个节点的二叉树，必然存在 \(n+1\) 个 NULL 指针。**
    *   **严谨的数学推导**：
        1.  二叉树的每个节点有 2 个指针域（`left` 和 `right`），所以 \(n\) 个节点总共有 **\(2n\)** 个指针槽位。
        2.  要连接 \(n\) 个节点形成一棵树，只需要 **\(n-1\)** 条边（也就是 \(n-1\) 个非空指针）。
        3.  那么，**空指针（NULL）的数量 = 总槽位 - 有意义的连线 = \(2n - (n-1) = n + 1\)**。
    *   **工程意义**：如果我们非要把这 \(n+1\) 个空指针都画在图中，图面会彻底变成密密麻麻的虚线或者 `null` 标记，变得完全不可读。所以，在计算机科学的所有教材和笔记中，默认**省略一切 NULL 指针，只画实际存在的连线**。

### 第三页（Slide 65）：新应用预告——编译器设计（表达式树）
**核心知识点：二元运算的天然数据结构**

*   **转折点**：PPT 提醒大家，二叉树**不只有“搜索（BST）”这一个用途**，它还有一个极为重要的核心应用场景。
*   **预告**：接下来，课程将进入**编译器设计（Compiler Design）**领域，讲解二叉树在其中的应用。
*   **猜想与铺垫**：在编译器设计中，二叉树用来表示**算术表达式**。例如，算式 `a + b * c` 在编译器的内部就会被表示为一棵**表达式树（Expression Tree）**。根节点是运算符（`+`），左孩子是 `a`，右孩子是包含 `b * c` 的子树。
*   *💡 学习期待*：接下来的 PPT 大概率会通过这棵表达式树，来讲解二叉树的**中序遍历**（得到人类习惯的 `a+b*c` 数学式）和**后序遍历**（得到计算机直接执行的 `abc*+` 后缀表达式/逆波兰式）。

### 💡 总结与复习重点（考试要点）
1.  **代码实操**：如果手写二叉树，要有意识地写出 `malloc` 创造新节点和 `free` 释放节点的逻辑。
