### 第一页（Slide 106）：二叉树的“地基”——C 语言结构体
**核心知识点：链式存储结构的标准定义**

这是任何一本数据结构教材里都会出现的经典二叉链表结构体代码：
```c
typedef struct BiTNode
{
    TElemType data;            // 数据域：存放节点实际的值
    struct BiTNode *lchild, *rchild; // 指针域：分别指向左孩子和右孩子
} BiTNode, *BiTree;
```
*   **`lchild` 和 `rchild`**：左右孩子的指针。这是二叉树“有方向、分左右”特征在代码上的完全体现。
*   **`BiTree`**：注意这里的 `typedef` 定义。它实际上是一个指向 `BiTNode` 的指针类型。**在外部代码中，我们只需要维护一个 `BiTree T;`，这个 `T` 就代表了整棵树的“根节点入口”，通过 `T->lchild` 就能顺藤摸瓜找到整棵树。**

### 第二页（Slide 107 上半部分）：物理内存布局对比
**核心知识点：把“画在纸上的树”变成“内存里的盒子”**
*   这张图将左边的**逻辑树（黄圆圈）**，映射成了右边内存中实际的**物理块（蓝方块）**。
*   每个蓝方块里包含一个数据（比如 A）和两个指向其他方块的内存指针。
*   **特别注意下方的斜树（链表退化情况）**：如果插入顺序不对（比如按顺序插入 1,2,3,4...），二叉树就会退化成一条长长的右斜链。此时在内存里，每个节点的右孩子指针都是非空的，而左孩子指针全是空的。这就是为什么我们需要后续引入**“平衡二叉树（AVL树/红黑树）”**来避免这种情况。

### 第三页（Slide 107 下半部分/黄条）：终极定理——为什么有 `n+1` 个空指针？
**核心知识点：计算机内存被“浪费”的证据，以及“线索二叉树”的引入**

PPT 黄条标出了极其重要的一句话：**“In one linked Binary tree there are n+1 NULL pointers.”**（一棵含有 n 个节点的链式二叉树，必然存在 n+1 个空指针）。

*   **数学证明（必考推算题）**：
    1. 有 `n` 个节点，每个节点有 2 个指针域，总共有 **`2n`** 个指针空间。
    2. 为了把这 `n` 个节点连成一棵树，必须使用 `n-1` 条边（也就是 `n-1` 个非空指针）。
    3. 所以，**空指针的数量 = 总指针空间 - 有效连接的指针 = 2n - (n-1) = n + 1**。
*   **这个定理有什么用（高能预警）？**
    因为这些 `n+1` 个空指针如果白白浪费太可惜了。于是聪明的计算机科学家发明了 **“线索二叉树（Threaded Binary Tree）”**。它把这些空指针利用起来，让它们指向前驱或后继节点，这样以后遍历二叉树时**连递归和栈都不需要用了，极大地提高了遍历速度**。

### 第五页（Slide 109）：二叉树 ADT 的核心操作接口
**核心知识点：对外暴露的操作函数（四个遍历是重头戏）**

这一页列出了二叉树外部调用的标准接口：
1.  **`Status CreateBiTree(BiTree &T);`**：构造一棵二叉树。注意 `&T`，这在 C++ 中代表“引用传递”，意味着这个函数会直接修改传入的根节点指针，完成树的初始化。
2.  **四种标准遍历方法（重点）**：
    *   `PreOrderTraverse`（前序）：根 -> 左 -> 右
    *   `InOrderTraverse`（中序）：左 -> 根 -> 右
    *   `PostOrderTraverse`（后序）：左 -> 右 -> 根
    *   `LevelOrderTraverse`（层次/层序）：从上到下、从左到右（借助队列实现）

**✨ 这段代码里最精彩的设计：函数指针 `(*visit)(TElemType e)`**
*   你可以看到这些遍历函数都有一个参数 `status (*visit)(TElemType e)`。这是一个**回调函数（函数指针）**。
*   为什么要这么设计？**实现了“遍历逻辑”和“数据处理”的解耦！**
*   比如，如果你想**打印**树里的值，你定义一个 `visit_print`；如果你想把树的值**存入数据库**，你定义一个 `visit_save`。树的遍历算法本身不需要动，只要把不同的 `visit` 函数传进去，它就能自动完成不同的业务任务。这是非常优秀的系统接口设计思想。
