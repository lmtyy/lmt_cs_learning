### 第一页（Slide 211）：哈夫曼树的“贪心”构建算法
**核心知识点：三步法构建最优树**

PPT 列出了构建哈夫曼树的最基础三步：
1.  **初始化**：把每个字符当作一棵单节点树（叶子），按频率（权重）排成一列。
2.  **贪心合并**：**找到权重最小的两棵树，把它们合并成一棵树**。合并后新节点的权重 = 这两个节点权重之和。并且规定，**选中的这两个节点的权重和，必须小于或等于当前任意其他组合的权重和**（这是保证最终结果WPL最小的贪心策略）。
3.  **重复迭代**：把合并后的新树放回集合中，继续重复第2步，直到所有节点合并成一棵完整的树。

### 第二页（Slide 212）：动态图解推演（必会的手算题）
**核心知识点：跟着箭头走一遍建树过程**

这页 PPT 用 5 个初始权重 `5, 15, 40, 30, 10` 画出了完整的建树推演（橙色箭头表示合并方向）：
1.  **第一步**：从 `5, 15, 40, 30, 10` 中挑出最小的两个：**5 和 10**。合并成新的父节点 **15**。此时集合剩下：`[15(新), 15(旧), 40, 30]`。
2.  **第二步**：从集合里挑出最小的两个：**15(新) 和 15(旧)**。合并成新的父节点 **30**。此时集合剩下：`[30(新), 40, 30(旧)]`。
3.  **第三步**：挑出最小的两个：**30(新) 和 30(旧)**。合并成新的父节点 **60**。此时集合剩下：`[40, 60]`。
4.  **第四步**：挑出最后的 **40 和 60**。合并成最终根节点 **100**。
*   🌟 **最终树结构（看右上角）**：根为 100，左子树40，右子树60。`60` 下面分出来了 `30` 和 `30`，左下角的 `30` 下面挂着 `15`（含5和10）和 `15`（原始）。

### 第三页（Slide 213）：物理存储结构（数组实现）
**核心知识点：哈夫曼树的 C 语言结构体**

在实际工程和考试手写代码中，哈夫曼树通常不画图，而是**用【数组】来实现**。PPT 给出了标准的节点定义：

```c
typedef struct {
    unsigned int weight;   // 权重（出现频率）
    Index parent, lchild, rchild; // 父节点、左孩子、右孩子的数组下标
} HTNode, *HuffmanTree;
```
**💡 关键设计：为什么需要一个 `parent` 指针？**
在普通的二叉树里，我们通常只需要 `lchild` 和 `rchild`。但在构建哈夫曼树时，**我们每次挑选的都是“当前没有父节点”的树根**。一旦两棵树被合并，它们的新父节点就被确定。因此，代码中必须通过标记 `parent` 来判断一个节点是否已经被合并过。
*   `HuffmanCode`：这是一个字符指针的指针（二维数组/字符串数组），用来存放最终生成的**哈夫曼编码**（如 `"01"`, `"110"` 等）。

### 第四页（Slide 214）：核心挑选函数 `Select` 的代码实现
**核心知识点：如何在数组中找出“两个最小且未被合并的节点”**

PPT 给出了在构建哈夫曼树时，负责“挑出最小的两个节点”的 `Select` 函数代码。这段代码极有技巧性：

```c
void select(HuffmanTree &HT, Index m, Index &S1, &Index &S2)
{
    S1=S2=0; m1=m2=infinity; // 初始化最小值和次小值为无穷大
    for (j=1; j<=m; j++)
    {
        if (HT[j].parent == 0) // 🌟 灵魂条件：只有还没被合并的树根才能被挑选！
        {
            if (HT[j].weight < m1) 
            {
                m2=m1; S2=S1;   // 原来的最小值降级为次小值
                m1=HT[j].weight; S1=j; // 更新最小值为当前节点
            }
            else if (HT[j].weight < m2) 
            { 
                m2=HT[j].weight; S2=j; // 更新次小值为当前节点
            }
        }
    }
}
```
*   **双重条件**：你可能会疑惑，如果两个节点权重一样怎么办？注意 `if (HT[j].weight < m1)`，如果 `HT[j].weight == m1`，它不会执行前一个分支，而是会走 `else if (HT[j].weight < m2)`，从而将相同的权重完美分配给 `S2`（次小值）。
