### 第一页（Slide 115）：深度优先遍历的三种方式图解
**核心知识点：三种遍历方式的“访问时机”**

这张图用一个简单的“根节点 + 左右子树”三角模型，清晰地展示了三种深度优先遍历的访问顺序（中间的黑色小圆点代表“访问/处理当前节点”的时刻）：
*   **(a) 前序遍历 (Preorder)**：**根 -> 左 -> 右**。（访问完根，再一头扎进左子树）。
*   **(b) 中序遍历 (Inorder)**：**左 -> 根 -> 右**。（先跑完左子树，再回来访问根，最后去右子树）。
*   **(c) 后序遍历 (Postorder)**：**左 -> 右 -> 根**。（把左右子树都跑完，最后才轮到访问根）。
*   *注：这张图是后面前、中、后序算法的直观几何基础。*

### 第二页（Slide 116）：前序遍历的定义与经典应用
**核心知识点：先父后子，用于“逐层传递状态”**

*   **定义**：在前序遍历中，节点是在它的左右子树**之前**被访问和处理的。
*   **💡 经典应用场景：给节点标记深度 (Label depth)**
    *   为什么计算**“每个节点的深度（距离根节点有几条边）”**必须用前序遍历？
    *   **逻辑推导**：如果你想知道孩子节点的深度，必须先知道它父节点的深度。因为前序遍历的顺序是 `根 -> 左 -> 右`。当你走到根节点时，假设它的深度是 `0`；那么左子树的根节点的深度就是 `0+1=1`；再往下就是 `1+1=2`。这种**“上层算好传给下层”**的需求，完美契合前序遍历的访问顺序。

### 第三页（Slide 117）：前序遍历的算法实现与图例验证
**核心知识点：递归代码与实际树形图的映射**

*   **右侧伪代码**：`Algorithm PreOrder(v)`
    *   步骤1：`visit(v)`（先处理当前节点）。
    *   步骤2：`if isInternal(v) PreOrder(leftChild(v))`（如果当前节点是内部节点，递归处理左孩子）。
    *   步骤3：`if isInternal(v) PreOrder(rightChild(v))`（递归处理右孩子）。
*   **左侧树图验证**：
    *   最外层的根节点是 `1`。根据前序遍历，访问顺序严格为：
        **1** -> 进入左子树 **2** -> 进入左子树 **3** (叶子) -> 返回2 -> 进入右子树 **4** -> 进入左子树 **5** (叶子) -> 返回4 -> 进入右子树 **6** (叶子) -> 返回4 -> 返回2 -> 返回1 -> 进入右子树 **7** -> 进入左子树 **8** -> 进入右子树 **9**。
    *   最终前序遍历序列就是图上节点的编号：**1, 2, 3, 4, 5, 6, 7, 8, 9**。这完美印证了算法。

### 第四页（Slide 118）：⚠️ 核心难点对比——处理顺序 vs 行走顺序
**核心知识点：计算机在内存里究竟是怎么“走”的？**

这张图是教学精华，它区分了两个极其容易混淆的概念：
*   **(a) 处理顺序 (Processing order)**：
    *   这就是我们平时做题、写代码时说的遍历序列。以这棵树为例，前序遍历“处理”到的有效节点序列是：**`A -> B -> C -> D -> E -> F`**。
*   **(b) 行走顺序 ("Walking" order)**：
    *   **这是计算机递归函数在内存中真实的物理执行路径！** 图中的**黑色方块**代表指向 `NULL` 的空指针。
    *   如果你在脑海中模拟计算机走一遍递归代码，它的路径实际上是这样的：
        `A` -> 找A的左孩子(`B`) -> 找B的左孩子(`C`) -> 找C的左孩子(`NULL方块`) -> **回溯**到C -> 找C的右孩子(`NULL方块`) -> **回溯**到C -> **回溯**到B -> 找B的右孩子(`D`) -> 找D的孩子(`NULL方块`) -> **回溯**到D -> **回溯**到B -> **回溯**到A -> 找A的右孩子(`E`) ...
*   **💡 核心结论（必背）**：
    *   拥有 `n` 个节点的二叉树，它**真实的物理行走轨迹**不仅会经过 \(n\) 个节点，还会**额外经过 \(n+1\) 个 NULL 空指针**（对应图中的黑色方块）。
    *   **这个原理有什么用？** 当你在学习“线索二叉树（Threaded Binary Tree）”时，这个空指针的理论就是它的核心基石。它告诉我们树里有很多空指针空间是可以被“废物利用”的。

### 学习总结：
*   **前序遍历口诀**：**“根左右，先父后子”**。
*   **适用场景**：适用于**“依赖上层信息”**的操作，比如求节点的深度、复制一棵二叉树、打印带缩进的目录树。
*   **底层思维**：学完 Slide 118，你可以明白了，**其实树的递归遍历本质上就是“左、右、上（回溯）”的物理走迷宫过程**，处理节点只是这个过程中的“暂停拍照”动作。
