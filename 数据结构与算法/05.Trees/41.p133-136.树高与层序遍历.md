### 第一页（Slide 133）：后序遍历的终极实战——求树的高度
**核心知识点：自底向上的递归计算（注意这行 -1 的坑！）**

*   **代码逻辑**：
    ```c
    if (T == NULL) return -1; // 特殊处理空节点
    else return 1 + max( Height(T->Left), Height(T->Right) );
    ```
*   **算法思想（后序）**：要算出当前节点的高度，必须**先**递归算出左子树的高度，**再**算出右子树的高度，取最大值，**最后**加 1 返回。这就完全契合了“先处理孩子，再处理父亲”的**后序遍历**思想。
*   **⚠️ 重点坑点：为什么空节点返回 -1 而不是 0？**
    *   这正是很多新手容易混淆的地方。
    *   我们来算一笔账：假设 `T` 指向一个**叶子节点**。它的左孩子是 `NULL`。
    *   当代码走到 `Height(T->Left)` 时，触发基准情况，返回 **`-1`**。
    *   右孩子同理，返回 **`-1`**。
    *   最后执行 `1 + max(-1, -1) = 1 - 1 = 0`。
    *   **结论**：**正因为 `NULL` 返回了 `-1`，才使得叶子节点的高度被正确计算为 `0`**。（这与很多教材中“叶子节点高度为0，空树高度为-1”的定义是完美统一的）。

### 第二页（Slide 134）：递归边界条件检查与时间复杂度
**核心知识点：递归核心逻辑与效率分析**

*   **验证逻辑**：PPT 明确指出，返回 -1 的处理使得叶子节点的高度被声明为 0，这完全符合树高度的标准定义。
*   **复杂度分析**：再次强调，整个遍历过程每个节点只访问一次，在每个节点上执行的操作是常数（`1+max` 计算），因此总时间复杂度是 **\(O(n)\)**。

### 第三页（Slide 135）：递归算法代码的极简哲学
**核心知识点：优雅的工程规范**

这页 PPT 总结了所有树递归算法的**通用核心理念**：
1.  **防微杜渐**：**“先处理 NULL 情况，再处理其余情况”**。这是最稳妥的做法。如果把 `if (T == NULL)` 去掉，直接写 `return 1 + max(...)`，必然会因为空指针解引用导致程序直接崩溃。
2.  **代码极简**：PPT 强调“代码中不声明或传递任何额外的变量，只传入树节点 `T` 即可”。在递归中，**“越紧凑的代码，越不容易产生愚蠢的 bug”**。不要试图在递归函数里用全局变量或者多参数去维护状态，尽量纯粹地使用函数的返回值（如计算高度，直接返回 `int`）。

### 第四页（Slide 136）：全新的遍历视角——层序遍历（BFS）
**核心知识点：深度优先（DFS）vs 广度优先（BFS）**

*   **什么是层序遍历（Level-order traversal）？**
    *   **规则**：它严格按层级进行。**深度为 `d` 的所有节点，必须在深度为 `d+1` 的任何节点处理之前，全部处理完。**
    *   **简单理解**：从根节点开始，一层一层、从上往下、从左到右像扫描仪一样扫过去。
*   **⚡️ 重大区别（数据结构完全不同）**：
    *   **前序、中序、后序（DFS）**：它们的底层靠的是**系统递归栈（Stack）**。一条路走到黑，撞墙了再回来。
    *   **层序遍历（BFS）**：PPT 明确指出，它**绝对不使用递归**，而是**必须使用一个【队列（Queue）】**来实现。
    *   *原理*：先把根节点放进队列，然后从队列中取出一个节点，处理它，并把它的左右孩子放进队列的末尾。一直循环直到队列为空，这样就能保证上一层节点永远先被取出和处理。

### 💡 总结：给你一张对比表，秒懂 PPT 最后一页的深意

| 特征             | 前、中、后序遍历 (DFS)               | 层序遍历 (Level-Order / BFS)                   |
| :--------------- | :----------------------------------- | :--------------------------------------------- |
| **遍历方向**     | **垂直方向**（先尽可能深入最下面）   | **水平方向**（一层一层向下扫）                 |
| **底层数据结构** | **栈（Stack）**（通过递归隐式调用）  | **队列（Queue）**（显式声明并操作）            |
| **编程模式**     | 递归调用                             | 迭代循环（`while` 循环搭配队列）               |
| **经典应用**     | 求树的高度、表达式求值、打印目录结构 | 求树的最小深度、图的连通性判断、按层打印二叉树 |
