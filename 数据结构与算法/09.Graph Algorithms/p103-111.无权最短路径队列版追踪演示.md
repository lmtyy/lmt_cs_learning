### 路径重构示例

**v3 → v5 的最短路径**（dist=3）：
```
p_v[v5] = v2  →  p_v[v2] = v1  →  p_v[v1] = v3（起点）
反转：v3 → v1 → v2 → v5
```

**v3 → v7 的最短路径**（dist=3）：
```
p_v[v7] = v4  →  p_v[v4] = v1  →  p_v[v1] = v3
反转：v3 → v1 → v4 → v7
```

## 第111页：复杂度总结

> **"Using the same analysis as was performed for topological sort, we see that the running time is O(|E| + |V|), as long as adjacency lists are used."**

- 与拓扑排序 Kahn 算法**同构**的分析方法：
  - 每个顶点入队/出队恰好 1 次 → O(|V|)
  - 每条边被检查恰好 1 次（出发点被处理时检查）→ O(|E|)
  - 总计 **O(|V| + |E|)**

---

## 总结

```
BFS 队列版无权最短路的追踪模板：

第 k 步：
  1. 出队队头 v → known[v] = 1
  2. 遍历 v 的邻居 w：
     如果 dist[w] == ∞：
       dist[w] = dist[v] + 1
       path[w] = v
       enqueue(w)
  3. 记录队列当前状态

验证结果：
  - 所有顶点 known = 1 → 完成
  - 分层验证：距离0→1→2→...逐层递增
  - 路径重构：沿 path 指针回溯

复杂度：O(|V| + |E|)（邻接表）
```
