## 第73页：队列版拓扑排序代码（初始化部分）

### 完整代码（上半部分）

```c
void topsort(graph G)
{
    QUEUE Q;
    unsigned int counter = 0;
    vertex v, w;
    /*1*/  Q = create_queue(NUM_VERTEX); make_null(Q);
    /*2*/  for each vertex v
    /*3*/      if(indegree[v] == 0)
    /*4*/          enqueue(v, Q);
    // ... 主循环部分见下页
}
```

### 逐段精讲

**行 /*1*/：初始化队列**

```c
Q = create_queue(NUM_VERTEX); make_null(Q);
```

- **动作**：创建一个容量为 `NUM_VERTEX`（顶点数）的队列，并清空。
- **为什么容量是 `NUM_VERTEX`？** 因为每个顶点最多入队 1 次（入度变为 0 时入队，之后不会再变），队列最大长度不会超过 N。

> 💡 **对比 BFS**：BFS 队列也是同样思路——每个顶点最多入队 1 次，容量 N 足够。

**行 /*2*/-/*4*/：把所有初始入度为 0 的顶点入队**

```c
for each vertex v
    if(indegree[v] == 0)
        enqueue(v, Q);
```

- **动作**：扫描一遍所有顶点，把入度为 0 的直接放入队列。
- **目的**：这些是**"没有先修课"的顶点**，可以立刻开始处理。它们是 BFS 的"起点集合"。
- **前提**：`indegree[]` 数组已经算好——通常在算法开始前遍历一遍所有边完成计算。

> ⚠️ **注意**：这里假设 `indegree[]` 已经预先计算完毕。实际工程中一般会先跑一遍：
> ```c
> for each edge (u, w) in G:
>     indegree[w]++;
> ```
> 时间 O(|E|)。

> 💡 **多个"零度种子"共存**：可能有多个顶点初始入度都是 0，它们的相对顺序在队列中决定了拓扑序列的开头。这就是拓扑排序不唯一的直接体现。

## 第74页：主循环 + 判环 + 清理

### 完整代码（下半部分）

```c
/*5*/  while(!is_empty(Q))
       {
/*6*/      v = dequeue(Q);
/*7*/      top_num[v] = ++counter;    /* assign next number */
/*8*/      for each w adjacent to v
/*9*/          if(--indegree[w] == 0)
/*10*/             enqueue(w, Q);
       }
/*11*/ if(counter != NUM_VERTEX)
/*12*/     error("Graph has a cycle");
/*13*/ dispose_queue(Q);   /* free the memory */
```

### 逐行精讲

**行 /*5*/：主循环——队列非空则继续**

```c
while(!is_empty(Q))
```

- **循环终止条件**：队列空了。
- **两种可能**：
  - 输出了 N 个顶点 → 正常结束，拓扑排序成功
  - 输出的顶点数 < N，但队列空了 → 剩余顶点有环，无法输出

**行 /*6*/：出队队头**

```c
v = dequeue(Q);
```

- **动作**：取出队头顶点 v。
- **语义**：v 是"当前所有入度为 0 的候选者中，最先被发现的一个"。

**行 /*7*/：分配拓扑序号**

```c
top_num[v] = ++counter;
```

- **动作**：先自增 counter，再赋值给 `top_num[v]`。
- **语义**：`top_num[v] = k` 表示 v 是拓扑序中的第 k 个（从 1 开始）。
- **`++counter` vs `counter++`**：这里用**前置自增**，因为要用增加后的值。第一次进入时 counter=0，`++counter` 变成 1，所以 top_num 从 1 开始编号。

> 💡 **考试小坑**：题目让你写出拓扑序列时，就是按 `top_num` 从小到大的顺序输出顶点名。

**行 /*8*/：遍历 v 的所有邻居**

```c
for each w adjacent to v
```

- **动作**：v 是 w 的前驱之一。现在 v 被"处理完"了，w 少了一个"前置依赖"。
- **实现**：邻接表版就是走 v 的邻接链表；邻接矩阵版就是扫矩阵第 v 行。

**行 /*9*/：入度减 1 并检查**

```c
if(--indegree[w] == 0)
```

- **动作**：`--indegree[w]` **前置自减**——先减 1，再和 0 比较。
- **等价写法**：
  ```c
  indegree[w]--;
  if(indegree[w] == 0) ...
  ```
- **含义**：如果 w 的所有前驱都处理完了（入度归零），w 现在具备输出资格了。

> 💥 **常见错误**：写成 `if(indegree[w]-- == 0)`（后置自减）。这会先比较再减，逻辑完全错误！**要用 `--indegree[w]`（前置）**。

**行 /*10*/：新入度为 0 的顶点入队**

```c
enqueue(w, Q);
```

- **动作**：把刚变为入度 0 的 w 加入队列。
- **保证**：每个顶点只会被入队一次——因为只有 `indegree[w]` 从 1 变到 0 的那一刻才入队，之后 `indegree[w]` 就不会再变了。

**行 /*11*/-/*12*/：判环**

```c
if(counter != NUM_VERTEX)
    error("Graph has a cycle");
```

- **动作**：循环结束后（队列空），检查输出的顶点数是否等于总顶点数。
- **判定原理**：
  - 若图是 DAG → 每次都能找到入度 0 的顶点 → 最终输出 N 个 → counter == N
  - 若图有环 → 环中所有顶点入度永远 > 0（互相制约）→ 无法进入队列 → counter < N

> ✅ **副产品**：**这个算法能同时完成"拓扑排序"和"判环"两件事**。

**行 /*13*/：释放队列内存**

```c
dispose_queue(Q);
```

- **动作**：手动释放队列占用的内存。
- **为什么**：C 语言没有垃圾回收。前面 `create_queue` 分配的内存必须手动释放，否则**内存泄漏**。

> 💡 **RAII 类比**：C++ 和现代语言中通常用析构函数自动处理。C 语言需要程序员自己保证"每个 create 都有对应的 dispose"。

## 第75~81页：完整执行实例（v1~v7 图）

### 图结构

```
v1 → v2
v1 → v3, v1 → v4
v2 → v4, v2 → v5
v3 → v6
v4 → v6, v4 → v7
v5 → v4, v5 → v7
v7 → v6
```

### 初始入度

| 顶点 | v1 | v2 | v3 | v4 | v5 | v6 | v7 |
|------|----|----|----|----|----|----|----|
| 入度 | 0  | 1  | 2  | 3  | 1  | 3  | 2  |

### 追踪表（PPT 中的核心）

| 时刻 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|------|---|---|---|---|---|---|---|
| v1   | **0** | 0 | 0 | 0 | 0 | 0 | 0 |
| v2   | 1 | **0** | 0 | 0 | 0 | 0 | 0 |
| v3   | 2 | 1 | 1 | 1 | **0** | 0 | 0 |
| v4   | 3 | 2 | 1 | **0** | 0 | 0 | 0 |
| v5   | 1 | 1 | **0** | 0 | 0 | 0 | 0 |
| v6   | 3 | 3 | 3 | 3 | 2 | 1 | **0** |
| v7   | 2 | 2 | 2 | 1 | **0** | 0 | 0 |
| **enqueue** | v1 | v2 | v5 | v4 | v3,v7 |   | v6 |
| **dequeue** | v1 | v2 | v5 | v4 | v3 | v7 | v6 |

### 逐步追踪解读

**时刻 1**：只有 v1 入度 0 → 入队 v1 → 出队 v1（top_num[v1]=1）
- 处理 v1 的邻居：v2(3-1=没到0), v3(2-1=1), v4(3-1=2)... 

**时刻 2**：v1 出队后，v2 入度归 0（v2 只被 v1 指） → 入队 v2 → 出队 v2（top_num[v2]=2）
- 处理 v2 的邻居：v4(2-1=1), v5(1-1=0)

**时刻 3**：v5 入度归 0 → 入队 v5 → 出队 v5（top_num[v5]=3）
- 处理 v5 的邻居：v4(1-1=0), v7(2-1=1)

**时刻 4**：v4 入度归 0 → 入队 v4 → 出队 v4（top_num[v4]=4）
- 处理 v4 的邻居：v6(3-1=2), v7(1-1=0)

**时刻 5**：v3 和 v7 同时入度 0！按邻接表顺序，v3 先入队，然后 v7 入队 → 出队 v3（top_num[v3]=5）
- 处理 v3 的邻居：v6(2-1=1)

**时刻 6**：队列中还剩 v7 → 出队 v7（top_num[v7]=6）
- 处理 v7 的邻居：v6(1-1=0) → 入队 v6

**时刻 7**：出队 v6（top_num[v6]=7）
- v6 无出边，处理完毕

### 最终拓扑序列

**dequeue 顺序 = 拓扑序列**：**v1, v2, v5, v4, v3, v7, v6**

> ✅ **PPT 关键点**：**"The topological ordering is the order in which the vertices dequeue."**

> 💡 **观察时刻5**：v3 和 v7 同时入度为 0 时，谁先入队取决于**邻接表中邻居的顺序**。如果换一种邻接表顺序，可能得到 v1, v2, v5, v4, **v7, v3**, v6 —— 也是合法拓扑序。这就是"拓扑排序不唯一"的实操表现。

### 手动模拟技巧

> 🎯 **考试解题四步法**：
> 1. **写下所有顶点的初始入度**
> 2. **列一个表**：每列记录当前入度快照 + 谁入队 + 谁出队
> 3. **每次出队后**：更新它所有邻居的入度，检查有没有新的 0，有就入队
> 4. **出队顺序 = 拓扑序**

## 第82页：时间复杂度总结

### 核心结论

> **邻接表实现时，Kahn 算法的时间复杂度是 O(|E| + |V|)。**

### 复杂度推导

| 步骤 | 操作 | 次数 | 单次时间 | 小计 |
|------|------|------|---------|------|
| 计算初始入度 | 扫所有边 | \|E\| | O(1) | O(\|E\|) |
| 初始入队扫描 | 扫所有顶点 | \|V\| | O(1) | O(\|V\|) |
| 主循环出队 | 每个顶点1次 | \|V\| | O(1) | O(\|V\|) |
| 主循环内更新邻居 | 每条边处理1次 | \|E\| | O(1) | O(\|E\|) |
| **总计** | | | | **O(\|V\| + \|E\|)** |

### 为什么"每条边处理 1 次"？

- 边 (u, w) 只在"u 出队时"被用来减少 `indegree[w]`
- u 出队恰好 1 次（DAG 保证每个顶点被处理 1 次）
- ⇒ 每条边只被扫描 1 次

### 邻接矩阵版

- 每个顶点出队后要扫矩阵第 v 行 → O(|V|)
- |V| 个顶点 → 总时间 **O(|V|²)**
- 和 DFS/BFS 一样，矩阵版永远是 O(|V|²)

### 完整复杂度对照

| 版本 | 邻接表 | 邻接矩阵 |
|------|--------|---------|
| 朴素版（扫全表找入度0） | O(\|V\|²) | O(\|V\|²) |
| **Kahn 队列版** | **O(\|V\| + \|E\|)** | O(\|V\|²) |

> 💡 **优化的两个前提**：Kahn 算法要达到 O(|V|+|E|)，必须满足：
> 1. 用**邻接表**存图（否则扫邻居就是 O(|V|)）
> 2. 用**队列**维护零度顶点（否则找入度0还是 O(|V|)）

## 总结

```
Kahn 队列版拓扑排序 —— 代码骨架：

初始化阶段：
  ├── 建立队列 Q
  ├── 计算所有顶点入度（预处理，O(|E|)）
  └── 初始入度为0的顶点全部入队

主循环阶段（while Q 非空）：
  ├── 出队 v，分配拓扑序号 top_num[v] = ++counter
  ├── 遍历 v 的所有邻居 w：
  │    ├── --indegree[w]
  │    └── if indegree[w] == 0 → enqueue(w)
  └── 循环

结束阶段：
  ├── counter != N → 有环，报错
  └── 释放队列内存

复杂度：邻接表 O(|V|+|E|) | 邻接矩阵 O(|V|²)

拓扑序列 = 顶点出队的顺序
```

> 🎯 **一句话记忆**：**"零入度先出，出一个减一遍，减到零再入队；出不完就有环。"**
