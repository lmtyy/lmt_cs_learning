# 拓扑排序 Topological Sort（p062-066）

进入 9.3 节。图算法从"如何遍历"转向"如何利用图结构解决实际问题"。拓扑排序是有向图独有的算法——把一堆有依赖关系的任务排成一个不违反依赖的线性顺序。

---

## 第62页：拓扑排序的形式化定义（"依赖顺序"）

### 核心定义

**拓扑排序（Topological Sort）**：对**有向无环图（DAG）**的顶点进行**线性排序**，满足：

> 如果图中存在从 vᵢ 到 vⱼ 的路径，那么在排序中 **vⱼ 必须出现在 vᵢ 之后**。

### 三个关键前提

| 条件 | 含义 | 为什么必需 |
|------|------|-----------|
| **有向图** | 边有方向 | 依赖关系天然有方向：A 依赖 B ≠ B 依赖 A |
| **无环** | 没有循环依赖 | 有环就没法排（下页讲） |
| **线性排序** | 所有顶点排成一列 | 目的是产生一个可执行的"时间表" |

> 🎯 **生活类比**：
> - **穿衣服顺序**：袜子 → 鞋子（不能反）、内裤 → 裤子 → 皮带（不能反）。这些"必须先..."的关系构成有向图，找一个不违反这些关系的顺序就是拓扑排序。
> - **早晨流程**：起床 → 刷牙 → 洗脸 → 吃饭 → 出门。可以先刷牙后洗脸，也可以反过来（没依赖关系），但吃饭必须在出门之前。
> - **软件构建**：编译 A.c 前必须先编译它依赖的 B.h。`make` 工具做的核心工作就是拓扑排序。

### 术语澄清：DAG

**DAG = Directed Acyclic Graph（有向无环图）**

- 拓扑排序**只能对 DAG** 进行
- DAG 是数据结构和算法中一类极其重要的图：任务调度、编译依赖、Git 版本历史、数据流图、区块链... 都是 DAG

## 第64页：两个关键性质（"能不能排"和"唯不唯一"）

### 性质1：有环 ⇒ 无拓扑排序

> **"如果图有环，拓扑排序不可能存在。"**

**为什么？** 假设图中存在环 A → B → C → A。

- 按定义，A 到 B 有路径 → B 必须排在 A 后
- B 到 C 有路径 → C 必须排在 B 后  
- C 到 A 有路径 → A 必须排在 C 后

综合起来：**A < B < C < A** —— 逻辑矛盾！

> 💡 **哲学理解**：环意味着"循环依赖"——A 等 B 完成，B 等 C 完成，C 又等 A 完成。谁也无法先开始。就像"先有鸡还是先有蛋"。

> ✅ **判环 = 判定 DAG 的一种方法**：拓扑排序算法执行时，如果**排不出来**（所有顶点入度都>0 但还有顶点未输出），就说明图**有环**。这是一个副产品——拓扑排序算法本身就能检测环。

### 性质2：拓扑排序不唯一

> **"排序不必唯一；任何合法排序都可以。"**

**为什么？** 因为图定义的是**偏序**（Partial Order），不是**全序**（Total Order）。

| 概念 | 含义 | 例子 |
|------|------|------|
| **偏序** | 只对部分元素有先后关系 | 早上刷牙和洗脸，两者之间没规定顺序 |
| **全序** | 所有元素之间都有唯一顺序 | 一个数组的从小到大排列 |

拓扑排序做的事就是——**把一个偏序扩展成一个合法的全序**。这种扩展**通常有多种方式**。

> 🎯 **类比**：假设你要早上完成三件事——**穿衣、吃饭、看手机**。只有一条规定："穿衣必须在吃饭之前"。合法的顺序有：
> 1. 穿衣 → 吃饭 → 看手机
> 2. 穿衣 → 看手机 → 吃饭
> 3. 看手机 → 穿衣 → 吃饭
>
> 三种都合法！**看手机**这个动作和其他两个都没依赖关系，可以放在任何位置。

## 第65页：非唯一性的具体示例（v1~v7 有向图）

### 图结构

```
v1 ──→ v2
 ↓ ↘   ↓ ↘
 v3  v4 ← v5
  ↘  ↓    ↓
    v6 ← v7
```

依赖关系（部分）：
- v1 → v2, v1 → v3, v1 → v4
- v2 → v4, v2 → v5
- v3 → v6
- v4 → v6, v4 → v7
- v5 → v4, v5 → v7
- v7 → v6

### 两种合法拓扑排序

| 序列 | 顺序 |
|------|------|
| 序列1 | v1, v2, v5, v4, **v3**, **v7**, v6 |
| 序列2 | v1, v2, v5, v4, **v7**, **v3**, v6 |

差异：v3 和 v7 谁先谁后？——它们之间**没有依赖关系**，所以任意顺序都合法。

### 验证方法

检查每条有向边 (u, v)：在序列中 u 必须在 v 之前。

以序列1为例：
- v1 → v2 ✅ v1 在 v2 前
- v1 → v3 ✅ v1 在 v3 前
- v4 → v6 ✅ v4 在 v6 前
- v5 → v7 ✅ v5 在 v7 前
- ... 所有边都满足 ✅

> ✅ **考试技巧**：验证一个序列是否为合法拓扑排序，就检查**每条边的两个端点在序列中的先后关系**。所有边都满足"起点在前，终点在后"即为合法。

> 💡 **产生不同顺序的原因**：算法执行时，如果多个顶点同时**入度为0**（都可以立刻输出），选谁都可以。不同的选择产生不同的拓扑序列。

## 第66页：拓扑排序的朴素算法（"剥洋葱"策略）

### 算法三步走

> 1. **找一个没有入边的顶点**（入度=0）
> 2. **输出它，并从图中删除**（连同它出发的所有边）
> 3. **对剩下的图，重复以上策略**

### 为什么"入度=0"的点先输出？

- **入度=0** 意味着"没有任何东西依赖它先完成"
- 换句话说，**它不依赖任何其他顶点** → 可以立即执行
- 输出它之后，它对下游顶点的"限制"就解除了 → 下游顶点的入度减1
- 就像剥洋葱：先剥掉最外层（没人依赖它），再剥次外层...

### 图解演示（v1~v7 图）

**初始状态**（各顶点入度）：
```
v1: 0  ← 入度0！可以先输出
v2: 1
v3: 1
v4: 3
v5: 1
v6: 3
v7: 2
```

**第1步**：输出 v1，删除 v1 出发的所有边。

新入度：
```
v2: 0  ← 变为0
v3: 0  ← 变为0
v4: 2
v5: 1
v6: 3
v7: 2
```

现在 v2 和 v3 都可选！

**如果选 v2**：输出 v2，v4 入度变1，v5 入度变0...
**如果选 v3**：输出 v3，v6 入度变2...

这就是拓扑排序**不唯一**的源头。

### 算法伪代码

```
while (还有顶点未输出):
    找一个入度为0的顶点 v
    if (找不到): 
        图有环，无拓扑排序，退出
    输出 v
    从图中删除 v 及其所有出边
    (即：v 的所有邻居 w 的入度 -1)
```

### 复杂度分析

**朴素实现**：每轮扫描找入度为0的顶点 → O(N)，共 N 轮 → **O(N²)**

**优化实现（下节讲）**：用**队列**维护所有入度为0的顶点，避免重复扫描 → **O(N + E)**

> 💡 **算法本质**：这个"找入度0 → 输出 → 删除 → 重复"的过程，本质是一种**BFS 变种**——从"入度0的顶点集"出发，逐波扩散。这个优化版本称为 **Kahn 算法**（后续讲）。

> 🎯 **另一种做法**：**DFS + 后序遍历的逆序**。对 DAG 做 DFS，每个顶点在"递归返回时"记录下来，最后反转顺序，就是一个合法拓扑序。这种方法叫做 **DFS-based Topological Sort**（也会讲到）。

## 总结

```
拓扑排序核心：

定义：DAG 中顶点的线性排序 → 保证所有边都是"前指向后"

前提：
├── 有向：无向图没有依赖概念
├── 无环：环导致循环依赖，无解
└── 输出：所有顶点，恰好一次

性质：
├── 有环 ⇒ 无拓扑排序（可用于判环）
├── 无环 ⇒ 至少存在一个拓扑排序
└── 排序通常不唯一（偏序→全序有多种扩展方式）

朴素算法（"剥洋葱"）：
  循环 N 次：{找入度0的顶点 → 输出 → 删除}
  复杂度：O(N²)

优化算法：
├── Kahn 算法（BFS 版）：O(N + E)
└── DFS 后序逆序：O(N + E)

应用：
  课程规划 | 编译依赖 | 任务调度 | Git 提交历史 | Excel 公式计算顺序
```

> 💡 **前后呼应**：
> - **前面 DFS/BFS 是通用遍历**——不管有向无向，只关心"访问所有"
> - **拓扑排序是首个"利用有向图结构"的算法**——它开启了后续"最短路径、网络流、最小生成树"等应用型算法的大门
> - **判环技巧**：用拓扑排序副产品能判断有向图是否有环。而无向图判环用 DFS 找回边即可，两者不同。
