# 逆邻接表与邻接表 vs 邻接矩阵对比（p035-039）

解决有向图邻接表"查入度慢"的痛点——逆邻接表，然后用一张完整对比表终结"到底该用哪种存储"的决策问题。

---

## 第35页：逆邻接表 Inverse Adjacency List（"谁指向我"）

### 问题回顾

上一节讲过，有向图的（正）邻接表存的是**出边**——第 i 个链表记录"从 i 出发能到哪"。

但如果你想知道"**谁能到 i**"（即求入度），就得扫全部链表找 dest==i 的节点，复杂度 O(N+E)。

### 解决方案：逆邻接表

**逆邻接表（Inverse Adjacency List）**：把存储方向反过来——第 i 个链表记录的是所有**指向 i** 的顶点。

### 示例（5顶点有向图 G7）

正邻接表（出边表，上一节讲过）：
```
0  A → [3] → [1] → ∧     A→D, A→E... (PPT中: A出发到的点)
1  B → [0] → ∧           
2  C → [1] → ∧           
3  D → [0] → [2] → ∧    
4  E → [3] → ∧           
```

**逆邻接表（入边表）**：
```
0  A → [4] → ∧            谁指向A？E(4)指向A
1  B → [0] → ∧            谁指向B？A(0)指向B
2  C → [1] → ∧            谁指向C？B(1)指向C
3  D → [0] → [2] → ∧     谁指向D？A(0)和C(2)指向D
4  E → [3] → ∧            谁指向E？D(3)指向E
```

### 术语对照

| 术语 | 含义 | 链表存的是 |
|------|------|-----------|
| **出边表**（正邻接表） | 从 i 出发的所有边 | "i 能到谁" |
| **入边表**（逆邻接表） | 指向 i 的所有边 | "谁能到 i" |

> ✅ **记忆口诀**：正邻接表 = "我认识谁"；逆邻接表 = "谁认识我"。

> 💡 **核心价值**：有了逆邻接表，求入度就变成了数链表长度——O(ID(i))，和求出度一样快。

## 第36页：有向图正邻接表示例（4顶点）

### 图：V0→V1, V0→V2, V2→V3, V3→V0

```
下标 | data | 出边链表
  0  |  V₀  | → [1] → [2] → ∧     V0出发到V1和V2
  1  |  V₁  | → (空)               V1没有出边，出度=0
  2  |  V₂  | → [3] → ∧           V2出发到V3
  3  |  V₃  | → [0] → ∧           V3出发到V0
```

> **关键观察**：V1 的链表为空 → V1 是一个"死胡同"（出度=0），能进去但出不来。对应第9页讲的"dead end"概念。

## 第37页：逆邻接表示例（同一个4顶点图）

### 同一个图：V0→V1, V0→V2, V2→V3, V3→V0

**逆邻接表**（存"谁指向我"）：

```
下标 | data | 入边链表
  0  |  V₀  | → [3] → ∧           谁指向V0？V3
  1  |  V₁  | → [0] → ∧           谁指向V1？V0
  2  |  V₂  | → [0] → ∧           谁指向V2？V0
  3  |  V₃  | → [2] → ∧           谁指向V3？V2
```

### 正/逆对比表

| 顶点 | 出度（正邻接表链表长） | 入度（逆邻接表链表长） |
|------|---------------------|---------------------|
| V0 | 2 | 1 |
| V1 | 0 | 1 |
| V2 | 1 | 1 |
| V3 | 1 | 1 |

> 💡 **实际工程做法**：如果算法同时需要出度和入度信息（如拓扑排序），通常不会真开两份邻接表，而是用**十字链表（Orthogonal List）**——每个边节点同时挂在"出边链"和"入边链"上，空间只需 O(N+E)。

## 第38页：邻接表 vs 邻接矩阵——空间对比

### 核心结论

| 存储方式 | 空间复杂度 | 适用场景 |
|---------|-----------|---------|
| 邻接矩阵 | **O(N²)** | 稠密图（E 接近 N²） |
| 邻接表 | **O(E + N)** | 稀疏图（E << N²） |

### 分界点在哪？

设每个邻接矩阵格占 1 单位空间，每个边节点占 c 单位（通常 c≈2~3，因为要存 dest+link+可能的cost）。

```
邻接矩阵空间 = N²
邻接表空间   = N + c·E（无向图是 N + 2c·E）

当 N² < N + c·E  即 E > (N²-N)/c ≈ N²/c 时，矩阵反而更省
```

> **实际判断**：如果 E < N²/4（边数不到最大可能的四分之一），邻接表通常更省空间。

## 第39页：邻接表 vs 邻接矩阵——时间对比（终极决策表）

### 完整操作复杂度对比

| 操作 | 邻接矩阵 | 邻接表 |
|------|---------|--------|
| **加边 (add edge)** | O(1) | O(1)（头插法） |
| **删边 (remove edge)** | O(1) | O(出度)，出度最大时为O(N) |
| **查边 (find edge)** | **O(1)** ✅ | O(出度)，出度最大时为O(N) |
| **列出所有邻居 (list all successors)** | O(N)（扫一行） | **O(出度)** ✅ |
| **初始化 (initialize)** | O(N²) | **O(N)** ✅ |

### 深度解析每个操作

**加边 O(1)**：

- 矩阵：直接 `A[i][j]=1`。无向图还有`A[j][i]=1`
- 邻接表：在 HEAD[i] 链表头部插入新节点（头插法），不需要遍历。

**删边**：
- 矩阵：`A[i][j]=0`，一步到位。无向图还有`A[j][i]=1`
- 邻接表：要先在 HEAD[i] 的链表里**找到** dest==j 的节点再删，最坏遍历整个链表。

**查边（find edge）**：

- 矩阵：`return A[i][j]`，直接下标访问 → O(1)。这是邻接矩阵**最大优势**。
- 邻接表：遍历 HEAD[i] 链表找 dest==j，最坏 O(degree(i))。

**列出所有邻居**：
- 矩阵：扫描第 i 行所有 N 个格子，跳过0的 → O(N)，无论实际邻居有几个。
- 邻接表：遍历 HEAD[i] 链表，只访问实际存在的邻居 → O(degree(i))。这是邻接表**最大优势**。

**初始化**：
- 矩阵：双重循环填 N×N 个格子 → O(N²)。
- 邻接表：开 N 个头节点置空 → O(N)。

> 💡 **核心考点**：DFS/BFS 遍历的复杂度直接取决于"列出所有邻居"的效率：
> - 用邻接矩阵：遍历总时间 = O(N²)（每个顶点扫一行）
> - 用邻接表：遍历总时间 = **O(N+E)**（每个顶点只看实际邻居）

> ✅ **考试决策公式**：
> - 题目问"判断 u 和 v 之间是否有边" → 邻接矩阵 O(1) 胜
> - 题目问"遍历图 / 找所有邻居 / BFS / DFS" → 邻接表 O(N+E) 胜
> - 题目是稀疏图 → 邻接表（空间和遍历都赢）
> - 题目是稠密图 + 频繁查边 → 邻接矩阵

---

## 总结

```
有向图的完整存储工具箱：

正邻接表（出边表）→ 快速查出度、列出后继
逆邻接表（入边表）→ 快速查入度、列出前驱
十字链表           → 同时支持出入度，空间 O(N+E)

决策总结：
┌─────────────────┬───────────────┬───────────────┐
│     考量维度     │   邻接矩阵     │    邻接表      │
├─────────────────┼───────────────┼───────────────┤
│ 空间             │ O(N²) 固定    │ O(N+E) 按需   │
│ 查边             │ O(1) ★        │ O(degree)     │
│ 遍历（BFS/DFS） │ O(N²)         │ O(N+E) ★      │
│ 初始化           │ O(N²)         │ O(N) ★        │
│ 适用             │ 稠密图         │ 稀疏图 ★      │
└─────────────────┴───────────────┴───────────────┘
```
