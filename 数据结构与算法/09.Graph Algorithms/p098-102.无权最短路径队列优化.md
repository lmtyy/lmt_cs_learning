# 无权最短路径的队列优化（p098-102）

从 O(|V|²) 到 O(|V|+|E|) 的优化之路。这段展示了一个精彩的思维演进：**观察 → 两个盒子 → 一个队列**——每一步都在解决前一步的浪费。

---

## 第98页：关键观察——距离状态只有两种

### 核心观察

> **"At any point in time, there are only two types of unknown vertices that have d_v ≠ ∞."**
>
> **"Some have d_v = curr_dist, and the rest have d_v = curr_dist + 1."**

**翻译**：在算法执行的任何时刻，那些**距离已知但未处理**（`!known` 且 `dist ≠ ∞`）的顶点，**只有两种可能**：

| 类别 | 距离值 | 语义 |
|------|-------|------|
| **当前层** | d_v = curr_dist | 本轮要处理的 |
| **下一层** | d_v = curr_dist + 1 | 下一轮要处理的 |

### 为什么只有两种？

**归纳证明**：

- 初始：只有起点，distance = 0
- 处理 curr_dist=0 时：邻居们的 distance 被设为 1（curr_dist+1）
- 处理 curr_dist=1 时：邻居们（还没被发现的）distance 被设为 2

**关键**：**BFS 的分层扩展性质**保证——从距离为 k 的顶点出发，只能发现距离为 k+1 的新顶点（因为距离更小的顶点已经被更早发现）。

> 💡 **推论**：如果有个顶点 d_v = 5，而当前 curr_dist = 3，怎么可能？——**不可能！**  
> 因为 d_v = 5 意味着它在第 4 轮扩展的邻居里，那第 4 轮必须先发生，但 curr_dist 还在 3，矛盾。所以 d_v 要么等于 curr_dist，要么等于 curr_dist+1，中间不会有断层。

### 这个观察的价值

- **朴素版**：需要遍历全表找出"当前层"的顶点 → O(V) 每轮
- **利用观察**：如果我们**分别记住这两类顶点**，就能 O(1) 找到当前层
- **优化方向**：**用容器保存这两类顶点，避免全表扫描**

## 第99页：优化1——"两个盒子"（简单但抽象）

### 优化方案

维护 **两个盒子（box）**：

| | 盒子 #1 | 盒子 #2 |
|---|---|---|
| 存什么 | 当前层顶点（d_v = curr_dist） | 下一层顶点（d_v = curr_dist + 1） |
| 何时用 | 本轮从这里取顶点处理 | 本轮扩展的新邻居放这里 |

### 修改后的算法

**替换朴素版的两个操作**：

| 朴素版行号 | 朴素做法 | 优化后做法 |
|-----------|---------|-----------|
| /*2*/-/*3*/ | 扫全表找符合条件的 v | **直接从盒子#1取出一个 v** → O(1) |
| /*8*/ 之后 | 更新完 w 的 dist、path | **额外把 w 加入盒子#2** |

### 第100页：单轮结束后的转换

> **"After the outside for loop terminates, box #1 is empty, and box #2 can be transferred to box #1 for the next pass of the for loop."**

**翻译**：一轮外层循环结束时（盒子#1处理光了），把盒子#2里的顶点**转移到盒子#1**，然后 curr_dist++ 开始下一轮。

### 图示

```
初始：
盒子#1 = { s }（起点，距离0）
盒子#2 = {}

轮次1（curr_dist=0）：
  取出 s，扩展邻居 → 邻居距离=1，放入盒子#2
  盒子#1 变空，盒子#2 = { 距离1的顶点 }
  转移：盒子#1 ← 盒子#2，盒子#2 = {}

轮次2（curr_dist=1）：
  从盒子#1 取出顶点，扩展邻居 → 邻居距离=2，放入盒子#2
  ...

直到两个盒子都空 → 算法结束
```

### 复杂度分析

- 每个顶点最多被放入盒子1次、取出1次 → 顶点操作总计 O(V)
- 每条边最多被扫描1次（其起点被处理时）→ 边操作总计 O(E)
- **总计 O(V + E)** ✅

### 缺点

- 需要两个数据结构，编码复杂
- "转移盒子#2到盒子#1"这个操作虽然是 O(box_size)，但有额外开销
- **可以进一步简化——只用一个队列！**（下页）

## 第101页：优化2——"一个队列"（最终版）

### 关键洞察

> **"We can refine this idea even further by using just one queue."**

**核心思想**：**两个盒子其实可以合并成一个队列！**

**为什么可以？** 队列的性质：
- **FIFO（先进先出）**
- 早入队的先出队 → 距离小的先处理

### 队列的分层性质

> **"At the start of the pass, the queue contains only vertices of distance curr_dist."**
>
> **"When we add adjacent vertices of distance curr_dist + 1, since they enqueue at the rear, we are guaranteed that they will not be processed until after all the vertices of distance curr_dist have been processed."**

**翻译**：
- 每一"轮"开始时，队列里都是**当前距离**的顶点
- 新加入的下一层顶点会加到**队尾**
- 由于 FIFO，**下一层顶点不会在当前层处理完之前被处理**

### 队列的"隐式分层"

```
初始状态：
Queue = [ s ]                              (距离0)

处理 s 后：
Queue = [ 邻居1, 邻居2, 邻居3 ]            (全是距离1)

处理邻居1 后（假设邻居1的新邻居是 A、B）：
Queue = [ 邻居2, 邻居3, A, B ]             (距离1, 距离1, 距离2, 距离2)
                       ↑ 这里是隐式分层线

处理邻居2、邻居3 后（各自加入距离2的新邻居 C、D）：
Queue = [ A, B, C, D ]                     (全是距离2)

...
```

> ✅ **神奇的事情**：队列的**头部永远是当前层**，**尾部永远是下一层**。分层是"自动"的——不需要显式的"两个盒子"！

### 与"两个盒子"的对应

| 两个盒子 | 一个队列 |
|---------|---------|
| 盒子#1 | 队列前半部分（距离 curr_dist） |
| 盒子#2 | 队列后半部分（距离 curr_dist+1） |
| 转移操作 | 无需转移——自动发生 |

> 🎯 **优雅之处**：把"分层"这个抽象概念，用队列的 FIFO 性质自然而然地表达出来。不需要显式管理层次，代码更简洁。

### 认知升级

前面讲 BFS 时给出的**队列驱动**代码，其实就是这里"从两个盒子简化为一个队列"的最终产物：

```
经过两次优化：
朴素 O(V²)  →  两个盒子 O(V+E)  →  一个队列 O(V+E)  = 教科书上的 BFS
   ↑                                          ↑
"扫全表找当前层"                          "队列自动管理分层"
```

### 为什么用队列的性质保证了正确性？

**BFS 求最短路径的关键定理**：

> **BFS 中每个顶点被"首次发现"时，它到起点的距离就是最短距离。**

**证明思路**（反证）：
- 假设 v 首次被 BFS 发现时距离为 d，但实际最短距离是 d' < d
- 那 v 应该在距离 d' 的层被发现（BFS 按层扩展）
- 但 v 是"首次发现" → 之前从未在 d' 层被扩展到 → 与 BFS 遍历完 d' 层前不进入 d 层矛盾
- 所以 d 就是最短距离

> ✅ **核心**：**队列的 FIFO 性质 + BFS 的层次扩展 = 保证最短距离**。缺一不可。

### 如果不用队列会怎样？

- 用**栈** → DFS → **不保证最短路径**！  
  DFS 可能先探深路径把某顶点距离设为 5，后来才发现更短的距离 2，但已经"漏掉"了这次更新
- 用**普通数组**（不区分头尾） → 无法保证"距离小的先处理" → 错

> 💡 **一句话**：**BFS 用队列 = 无权最短路 ; DFS 用栈 = 不能求最短路**。这是队列和栈在图算法中最本质的分野。

## 第102页：最终代码——队列版无权最短路径

### 完整代码

```c
void unweighted(TABLE T)
{
    QUEUE Q;
    vertex v, w;
    
/*1*/  Q = create_queue(NUM_VERTEX); make_null(Q);
/*2*/  enqueue(s, Q);
/*3*/  while(!is_empty(Q))
       {
/*4*/      v = dequeue(Q);
/*5*/      T[v].known = TRUE;    /* not really needed anymore */
/*6*/      for each w adjacent to v
/*7*/          if(T[w].dist == INT_MAX)
               {
/*8*/              T[w].dist = T[v].dist + 1;
/*9*/              T[w].path = v;
/*10*/             enqueue(w, Q);
               }
       }
/*11*/ dispose_queue(Q);
}
```

### 逐行精讲

**行 /*1*/：创建并清空队列**

```c
Q = create_queue(NUM_VERTEX); make_null(Q);
```

- 容量 NUM_VERTEX（每个顶点最多入队1次）

**行 /*2*/：起点入队**

```c
enqueue(s, Q);
```

- 队列的第一个元素是起点 s
- 前提：调用前 T 已初始化，T[s].dist = 0，其他顶点 dist = INT_MAX

> 💡 **和拓扑排序 Kahn 算法的区别**：拓扑排序初始入队**所有入度0的顶点**（可能有多个"起点"）；BFS 只入队**一个起点**（单源）。

**行 /*3*/：主循环——队列非空**

```c
while(!is_empty(Q))
```

- 只要还有顶点没处理就继续

**行 /*4*/：出队**

```c
v = dequeue(Q);
```

- FIFO：先入队的先出来
- v 是当前"距离最小的未处理顶点"

**行 /*5*/：标记 known（可选）**

```c
T[v].known = TRUE;   /* not really needed anymore */
```

- **PPT 注释**：`not really needed anymore` —— 其实不需要了！
- **原因**：判断"是否已处理"改成"是否在队列中"或"dist 是否 == ∞"。队列版天然避免重复处理。
- **保留原因**：向后兼容 —— 如果外部代码要读 T[v].known，保留这个赋值让语义完整。

> 💡 **考试要点**：知道 known 字段在队列版中变成"冗余信息"就好。

**行 /*6*/-/*10*/：扩展邻居**

```c
for each w adjacent to v
    if(T[w].dist == INT_MAX)      // 还没被发现
    {
        T[w].dist = T[v].dist + 1; // 距离基于 v 更新
        T[w].path = v;             // 记录前驱
        enqueue(w, Q);             // 加入队列
    }
```

**逐句解析**：

- **行 /*7*/**：`T[w].dist == INT_MAX` 判断 w 是否首次被发现（还是初始 ∞）。
- **行 /*8*/**：`T[w].dist = T[v].dist + 1`。**关键点**：距离基于 v 的 dist 累加，而不是维护外部的 curr_dist 变量。
- **行 /*9*/**：记录前驱 v，为路径重构准备。
- **行 /*10*/**：入队 w，等待后续处理。

> 💥 **反面教材**：如果去掉 `if(T[w].dist == INT_MAX)`：
> - w 可能被反复入队（每次某个邻居扩展到它都入队一次）
> - 队列膨胀 → 内存和时间浪费
> - 更糟：`T[w].dist` 可能被重复覆盖（虽然值相同，但语义混乱）

**行 /*11*/：释放队列**

```c
dispose_queue(Q);
```

- 释放动态分配的队列内存
- C 语言必须手动释放

### 关键设计精髓

**1. `T[w].dist == INT_MAX` 一石三鸟**：

- 判断 w 是否首次被发现
- 替代 `visited[]` 数组
- 保证每个顶点只入队一次

**2. `T[w].dist = T[v].dist + 1` 无需 curr_dist 变量**：

- 距离信息**分布式存储**在每个顶点中
- 不需要显式的"当前层数"变量
- **每个顶点通过它的父节点计算自己的距离**

**3. FIFO 保证正确性**：

- 早入队 = 距离小 = 先处理
- 处理时的距离就是最短距离

### 完整复杂度

| 步骤 | 分析 | 时间 |
|------|------|------|
| 初始化 | 队列创建 + 入队起点 | O(1) |
| 主循环 | 每个顶点出队1次 | O(V) |
| 邻居扫描 | 每条边最多检查1次 | O(E) |
| **总计** | | **O(V + E)** |

> ✅ **从 O(V²) 到 O(V+E)**：稀疏图上巨大提升！

## 总结

```
无权最短路径优化演进史：

朴素版 (p096)                两个盒子版 (p099)              一个队列版 (p102)
─────────────                ─────────────                ─────────────
for curr_dist:               维护 box1, box2:              维护 一个队列 Q:
  for 全部顶点:               取 box1 元素处理              取 队首处理
    找当前层的处理             新邻居放 box2                 新邻居入队尾
                              box2 → box1                  (自动排序)
─────────────                ─────────────                ─────────────
O(|V|²)                       O(|V|+|E|)                  O(|V|+|E|)
    ↑                             ↑                             ↑
浪费在                        显式管理                       利用 FIFO
"扫全表"                      分层                           隐式分层

BFS 求无权最短路的三大保证：
├── 起点距离 0，逐层扩展
├── 首次发现 = 最短距离（FIFO 保证）
├── path 数组记录前驱 → 可回溯完整路径
└── 复杂度 O(|V|+|E|) ✅
```

> 🎯 **核心洞察**：**"两个盒子"和"一个队列"是同一件事**——都是"当前层 + 下一层"的容器。队列因其 FIFO 性质**自动**完成了盒子的"轮转"功能。这是数据结构选择的艺术。

> 💡 **前后呼应**：
> - **回顾**：BFS 章节讲过队列驱动的遍历，现在明白它就是无权最短路的解法
> - **展望**：下一节 Dijkstra —— **带权版 BFS**，把"普通队列"升级为"优先队列"，把"边数分层"升级为"权值累积"
