## 第49页：DFS 驱动函数 `DFSTraverse`（"总调度员"）

### 完整代码

```c
Boolean visit[MAX];              // 访问标志数组
Status(*VisitFunc)(int v);       // 函数变量（函数指针）

void DFSTraverse(Graph G, Status(*Visit)(int v))
{
    // 对图G进行深度优先遍历
    VisitFunc = Visit;           // 函数指针指向一个具体的函数
    for(v=0; v<G.vexnum; ++v)
        visit[v] = FALSE;        // 标志初始化
    for(v=0; v<G.vexnum; ++v)
        if(!visited[v])
            DFS(G, v);           // 对尚未访问的顶点调用函数DFS
}
```

### 逐段精讲

**1. 全局变量声明**

```c
Boolean visit[MAX];
Status(*VisitFunc)(int v);
```

- **`visit[MAX]`**：全局的"访问标记数组"。`visit[i]=TRUE` 表示顶点 i 已访问。
- **`VisitFunc`**：函数指针，指向"访问一个顶点时要执行的操作"（比如打印顶点编号）。这是一种**策略模式**——把"遍历框架"和"访问动作"解耦。

**2. 初始化所有标记为 FALSE**

```c
for(v=0; v<G.vexnum; ++v)
    visit[v] = FALSE;
```

- **动作**：把所有顶点标记为"未访问"。
- **目的**：确保每次遍历从干净状态开始。
- **为什么不能省？** 如果之前做过一次遍历，`visit[]` 里还残留着上次的 TRUE，本次遍历就会跳过那些点。

**3. 遍历所有顶点，对未访问的启动 DFS**

```c
for(v=0; v<G.vexnum; ++v)
    if(!visited[v])
        DFS(G, v);
```

- **动作**：从 0 到 N-1 逐个检查，如果某顶点还没被访问过，就从它启动一次 DFS。
- **目的**：处理**不连通图**！一次 DFS 只能覆盖一个连通分量。如果图有多个连通分量，需要多次启动 DFS。

> 💡 **为什么连通图也要这个循环？** 连通图的话，第一次 `DFS(G, 0)` 就能访问所有顶点，后续循环中 `!visited[v]` 全为 FALSE，不会再进入。所以这个循环是"保险措施"，对连通图无害，对非连通图必需。

> ⚠️ **易错点**：如果你只写 `DFS(G, 0)` 而不加外层循环，非连通图中部分顶点永远不会被访问！

## 第50页：DFS 递归核心函数（抽象版）

### 完整代码

```c
void DFS(Graph G, int v)
{
    // 从顶点v出发递归地深度优先遍历图G
    visited[v] = TRUE;              // 标记v为已访问
    VisitFunc(v);                   // 对v执行访问操作
    for(w=FirstAdjVex(G,v); w>=0; w=NextAdjVex(G,v,w))
        if(!visited[w])
            DFS(G, w);             // 对尚未访问的顶点调用函数DFS
}
```

### 逐段精讲

**1. 标记并访问当前顶点**

```c
visited[v] = TRUE;
VisitFunc(v);
```

- **动作**：先标记再访问。
- **为什么先标记再访问？** 防止在访问过程中（如果有回调等复杂逻辑），其他路径再次进入这个顶点。标记要尽早。

**2. 遍历所有邻居（核心循环）**

```c
for(w=FirstAdjVex(G,v); w>=0; w=NextAdjVex(G,v,w))
    if(!visited[w])
        DFS(G, w);
```

| 函数 | 含义 |
|------|------|
| `FirstAdjVex(G, v)` | 返回 v 的**第一个邻居**的下标（若无邻居返回-1） |
| `NextAdjVex(G, v, w)` | 返回 v 的邻居中，**w 之后的下一个邻居**（若无更多返回-1） |

- **动作**：从 v 的第一个邻居开始，逐个检查。如果邻居 w 未访问，递归深入。
- **目的**：这是一个**抽象接口**——不管底层是邻接矩阵还是邻接表，都能工作。具体实现不同，但框架不变。
- **`w>=0` 的含义**：返回 -1 表示"没有更多邻居了"，循环结束。

> 🎯 **这就是 DFS 的递归本质**：访问自己 → 对每个未访问邻居递归 → 所有邻居处理完则返回（回退）。

> 💡 **递归调用栈 = 隐式的"路径栈"**：每次递归深入，当前状态被压入调用栈；回退时弹出。这就是为什么 DFS 的底层数据结构是**栈**。

## 第51页：邻接矩阵版 DFS（"扫一行找邻居"）

### 完整代码

```c
void DFS(MGraph G, int v)
{
    // 从顶点v出发递归地深度优先遍历图G
    visited[v] = TRUE;
    VisitFunc(v);
    for(w=0; w<G.vexnum; ++w)
        if(G.arcs[v][w] && !visited[w])
            DFS(G, w);           // 对尚未访问的顶点调用函数DFS
}
```

### 逐段精讲

**关键差异：如何找邻居**

```c
for(w=0; w<G.vexnum; ++w)
    if(G.arcs[v][w] && !visited[w])
```

- **动作**：从 w=0 到 w=N-1 逐个检查矩阵第 v 行的每一格。
- **`G.arcs[v][w]`**：如果为 1（或非∞/非0），说明 v→w 有边。
- **`!visited[w]`**：w 还没被访问过。
- 两个条件同时满足 → 递归进入 w。

> ⚠️ **问题**：即使 v 只有2个邻居，这个循环也要扫完整行的 N 个格子。大量时间浪费在检查"不存在的边"上。

### 时间复杂度分析

```
每个顶点被访问恰好1次 → DFS 被调用 N 次
每次调用内部循环 → 扫描一整行 = O(N)

总时间 = N × O(N) = O(N²)
```

> ✅ **结论**：邻接矩阵存储的 DFS 时间复杂度 = **O(N²)**，与边数无关。

## 第52页：邻接表版 DFS（"沿链表走"）

### 完整代码

```c
void DFS(ALGraph G, int v)
{
    // 从顶点v出发递归地深度优先遍历图G
    visited[v] = TRUE;
    VisitFunc(v);
    for(p=G.vertices[v].firstarc; p; p=p->nextarc)
    {
        w = p->adjvex;
        if(!visited[w])
            DFS(G, w);          // 对尚未访问的顶点调用函数DFS
    }
}
```

### 逐段精讲

**关键差异：如何找邻居**

```c
for(p=G.vertices[v].firstarc; p; p=p->nextarc)
{
    w = p->adjvex;
    if(!visited[w])
        DFS(G, w);
}
```

| 表达式 | 含义 |
|--------|------|
| `G.vertices[v].firstarc` | 顶点 v 的邻接链表头指针 |
| `p` | 当前遍历到的边节点 |
| `p->nextarc` | 下一条边 |
| `p->adjvex` | 当前边指向的目标顶点 |
| `!p`（即 `p==NULL`） | 链表遍历完毕，循环结束 |

- **动作**：从链表头开始，沿 `nextarc` 逐个访问每条边，只检查**实际存在的邻居**。
- **目的**：不浪费时间在"不存在的边"上。有几个邻居就检查几次。

> 🎯 **对比邻接矩阵版**：
> - 矩阵版：`for(w=0; w<N; w++) if(arcs[v][w])` → 扫 N 格
> - 邻接表版：`for(p=firstarc; p; p=p->nextarc)` → 只扫 degree(v) 个

### 时间复杂度分析

```
每个顶点被访问1次 → 每个顶点节点检查1次 = O(N)
每条边被检查的次数：
  无向图：每条边在两个链表中各出现1次 → 总检查 2E 次
  有向图：每条弧只在一个链表中 → 总检查 E 次

总时间 = O(N + E)  （无向图 O(N + 2E) = O(N + E)）
```

> ✅ **结论**：邻接表存储的 DFS 时间复杂度 = **O(N + E)**。稀疏图下远优于 O(N²)。

## 第53页：算法复杂度总结

### 核心对比

设图有 **n 个顶点，e 条边**：

| 存储方式 | 找一个顶点的所有邻居 | DFS 总时间 |
|---------|-------------------|-----------|
| **邻接表** | O(degree) | **O(n + e)** |
| **邻接矩阵** | O(n)（扫一整行） | **O(n²)** |

### 为什么邻接表是 O(n+e)？

- 无向图中共有 **2e 个边节点**（每条边两端各一个）
- 扫描所有边节点的时间 = O(e)（PPT原文写O(e)，无向图实际是O(2e)=O(e)）
- 加上访问每个顶点的 O(n) → 总计 **O(n + e)**

### 为什么邻接矩阵是 O(n²)？

- 每个顶点要扫一整行找邻居 → 每行 O(n)
- 共 n 个顶点 → n × O(n) = **O(n²)**
- 无论图有多少条边，矩阵版永远是 O(n²)

> 💡 **核心考点**：
> - 题目说"邻接表存储"→ 遍历复杂度 = O(N+E)
> - 题目说"邻接矩阵存储"→ 遍历复杂度 = O(N²)
> - 这两个结论适用于 DFS 和 BFS（后面会讲 BFS 也是一样的）

> 💥 **常见错误**：写成 O(E) 忘了加 O(N)。即使没有任何边（E=0），DFS 驱动函数还是要扫一遍所有顶点做初始化和检查 `!visited[v]`，所以至少 O(N)。

## 总结

```
DFS 代码架构：

DFSTraverse（驱动函数）
├── 初始化 visited[] = FALSE
└── for 每个顶点 v：if(!visited[v]) DFS(G, v)
        ↓
    DFS（递归核心）
    ├── visited[v] = TRUE
    ├── VisitFunc(v)
    └── for 每个邻居 w of v：
            if(!visited[w]) DFS(G, w)

两种实现：
┌──────────────┬────────────────────────────────┐
│  邻接矩阵版  │ for(w=0;w<N;w++)               │
│              │   if(arcs[v][w] && !visited[w]) │
│              │ → O(N²)                         │
├──────────────┼────────────────────────────────┤
│  邻接表版    │ for(p=firstarc; p; p=p->next)   │
│              │   w=p->adjvex; if(!visited[w])  │
│              │ → O(N+E)                        │
└──────────────┴────────────────────────────────┘
```

> 💡 **延伸思考**：为什么不用迭代（显式栈）代替递归？
> - 递归版代码简洁，但调用栈深度最坏 O(N)（链状图）
> - 实际工程中如果图很大（几百万顶点），递归可能栈溢出，此时改用显式栈迭代版本
> - 面试可能问"非递归 DFS"——用栈模拟即可，逻辑等价
