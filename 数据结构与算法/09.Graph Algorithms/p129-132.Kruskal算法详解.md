# Kruskal 算法详解（p129-132）

MST 的第二种求解方法——Kruskal 算法。和 Prim "从一棵树逐步长大"不同，Kruskal 是"把全部边排好序，从最便宜的开始一条条试，不成环就收"。视角从"点"切换到"边"。

---

## 第129页：Kruskal 算法概述（"逐边筛选"）

### 算法规则（4 条）

1. **每次只加一条边**（one at a time）
2. **按权值从小到大选边**（nondecreasing order of cost）
3. **不成环就加入 MST**（does not form a cycle → accept）
4. **选够 n-1 条边就停**（since G is connected, exactly n-1 edges will be selected）

> 🎯 **生活类比**：想象你在淘宝上买铁路轨道模型来连接小区——
> - 把所有可选的轨道**按价格从低到高排序**
> - 从最便宜的开始买：
>   - 如果这条轨道连接的两个小区**还没被同一条路线覆盖** → 买！
>   - 如果两个小区**已经连通了**（加上会形成环路）→ 跳过，太浪费
> - 买够 n-1 条就收工

### 与 Prim 的对比

| | Prim | Kruskal |
|---|---|---|
| 视角 | 以**顶点**为中心 | 以**边**为中心 |
| 增长方式 | 一棵树逐步长大 | 多棵小树逐步合并 |
| 每步操作 | 选最短的**跨界边** | 选最短的**不成环的边** |
| 初始状态 | 一棵单点树 | N 棵单点树（森林） |
| 终止状态 | 树覆盖所有顶点 | 森林合并为一棵树 |

## 第130页：贪心策略的精确表述

> **"A second greedy strategy is continually to select the edges in order of smallest weight and accept an edge if it does not cause a cycle."**

### 关键判断：加边会不会成环？

**什么时候加边 (u, v) 会成环？**

- 当 u 和 v **已经在同一棵树中**（已经连通了）
- 此时再加一条连接它们的边 → 形成环路

**什么时候不会成环？**

- u 和 v 在**不同的树中**（还没连通）
- 加上这条边 → 把两棵树合并为一棵

> 💡 **成环判断 = 连通性判断**：两个端点在同一个连通分量里 → 成环；不在 → 不成环。

> 🎯 **这就引出了并查集（Union-Find / Disjoint Set）**——专门用来高效判断"两个元素是否在同一集合"并执行"合并两个集合"的数据结构。

## 第131页：森林的合并过程

### 形式化描述

> **"Formally, Kruskal's algorithm maintains a forest -- a collection of trees."**
>
> **"Initially, there are |V| single-node trees. Adding an edge merges two trees into one."**
>
> **"When the algorithm terminates, there is only one tree, and this is the minimum spanning tree."**

### 演进过程

```
初始：|V| 个顶点，每个都是独立的"单点树"
     {v1}, {v2}, {v3}, {v4}, {v5}, {v6}, {v7}     ← 7 棵树

加第1条边 (v1,v4,权1)：
     {v1,v4}, {v2}, {v3}, {v5}, {v6}, {v7}         ← 6 棵树

加第2条边 (v6,v7,权1)：
     {v1,v4}, {v2}, {v3}, {v5}, {v6,v7}             ← 5 棵树

加第3条边 (v1,v2,权2)：
     {v1,v4,v2}, {v3}, {v5}, {v6,v7}                ← 4 棵树

...

加第6条边（第n-1=6条）：
     {v1,v2,v3,v4,v5,v6,v7}                         ← 1 棵树 = MST！
```

> 💡 **本质**：从 |V| 棵独立的树开始，每次加一条边就**合并两棵树为一棵**。n-1 次合并后，|V| 棵树变成 1 棵树 = MST。

> ✅ **从"森林"到"树"的过程** = 从"完全不连通"到"完全连通"的过程。

## 第132页：终止条件与判环方法

### 终止条件

> **"The algorithm terminates when enough edges are accepted."**

- **"够了"的含义**：接受了 **n-1 条**边
- 提前终止：如果还没接受够 n-1 条但所有边都处理完了 → 图不连通（不存在 MST）

### 判环方法

> **"It turns out to be simple to decide whether edge (u,v) should be accepted or rejected."**

使用**并查集（Union-Find）**：

```
对于待选边 (u, v)：
    if Find(u) == Find(v):     // u 和 v 在同一集合（同一棵树）
        reject (u,v)            // 加了会成环！
    else:
        accept (u,v)            // 不会成环，加入 MST
        Union(u, v)             // 合并两棵树
```

### 并查集的两个操作

| 操作 | 含义 | 时间复杂度（路径压缩+按秩合并） |
|------|------|--------------------------|
| **Find(x)** | x 属于哪棵树？（返回树的根/代表元） | ≈ O(α(n)) ≈ O(1) |
| **Union(x,y)** | 把 x 所在的树和 y 所在的树合并 | ≈ O(α(n)) ≈ O(1) |

> 💡 **α(n) 是反阿克曼函数**——增长极其缓慢，对所有实际 n 都 ≤ 4。所以每次 Find/Union 视为 O(1)。

### Kruskal 完整伪代码

```
Kruskal(G):
    将所有边按权值排序                          // O(E log E)
    初始化并查集：每个顶点独立                    // O(V)
    MST = {}
    
    for each edge (u,v) in sorted order:       // 最多 E 次
        if Find(u) ≠ Find(v):                  // 不在同一棵树
            MST = MST ∪ {(u,v)}                // 加入 MST
            Union(u, v)                        // 合并
            if |MST| == n-1: break             // 够了就停
    
    return MST
```

### 时间复杂度

| 步骤 | 操作 | 时间 |
|------|------|------|
| 排序所有边 | 对 E 条边按权值排序 | **O(E log E)** |
| 处理边 | 最多 E 次 Find + n-1 次 Union | O(E·α(V)) ≈ O(E) |
| **总计** | 瓶颈在排序 | **O(E log E) = O(E log V)** |

> 💡 为什么 O(E log E) = O(E log V)？因为 E ≤ V²，所以 log E ≤ 2 log V = O(log V)。

### Kruskal vs Prim 的选择

| 场景 | 推荐算法 | 原因 |
|------|---------|------|
| **稀疏图**（E ≈ V） | **Kruskal** | O(E log E) ≈ O(V log V)，比 Prim O(V²) 好 |
| **稠密图**（E ≈ V²） | **Prim**（数组版） | O(V²) vs Kruskal O(V² log V)，Prim 更好 |
| 边已经排好序 | **Kruskal** | 排序这步免费，直接 O(E) |

## 总结

```
Kruskal 算法核心：

思想："把所有边按价格排序，从最便宜的开始逐条尝试——不成环就要，成环就跳"

数据结构：
├── 边的排序结果（优先队列 or 预排序数组）
└── 并查集（Union-Find）：判断两端点是否已连通

算法流程：
  排序 → 遍历边 → Find判断 → 不同集合则Union并加入MST → 够n-1条停

正确性：贪心 + 切割性质（每次选的全局最轻不成环边=某切割的最轻跨界边）

复杂度：O(E log E)（瓶颈=排序）

Kruskal vs Prim：
  Kruskal = 以边为中心，适合稀疏图
  Prim = 以顶点为中心，适合稠密图
  两者都是贪心，都正确，输出相同的 MST（权值和相同）
```

> 🎯 **考试口诀**：
> - **Prim = "树长大"**：从一个点开始，每次伸出最短触手
> - **Kruskal = "森林合并"**：先排序，逐边选，不环就收

> 💡 **前后呼应**：
> - Kruskal 用的**并查集**是第 8 章（或独立章节）学过的数据结构
> - Kruskal 的"排序+贪心选择"模式在后面的**贪心算法**章节还会反复出现
> - 如果要手算 Kruskal，关键是画一个边的排序表，逐条标记"accept/reject"
