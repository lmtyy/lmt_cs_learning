# 广度优先搜索 BFS（p054-057）

DFS 是"一条路走到黑"，BFS 是"逐层扩张"。就像水波一样从起点向外一圈圈铺开——先近后远。BFS 与 DFS 一起构成图遍历的双子星。

---

## 第54页：BFS 核心思想（"水波扩散"）

### 算法描述

从起始顶点 vᵢ 出发（未访问）：

1. **访问 vᵢ**（第0层）
2. **访问 vᵢ 的所有邻居** w₁, w₂, ..., wₖ（第1层，一次访问一整层）
3. **依次从这些顶点出发**，访问它们各自的未访问邻居（第2层）
4. 循环，直到所有顶点访问完毕

### 图解（8顶点无向图）

```
         V0             ← 第0层：起点
        /  \
      V1    V2          ← 第1层：V0的邻居
      / \   / \
    V3  V4 V5  V6       ← 第2层：V1和V2的邻居
     \  /
      V7                ← 第3层：V3或V4的邻居
```

**访问顺序**：V0 → (V1, V2) → (V3, V4, V5, V6) → (V7)

> 🎯 **类比一**：往湖里丢一颗石子——涟漪从中心一圈一圈向外扩散。第1圈涟漪先到达，第2圈稍后，第3圈更后。BFS 就是这种"按距离由近到远"的访问顺序。

> 🎯 **类比二**：新冠传播链——第1天你感染，第2天你的密切接触者感染，第3天他们的密切接触者感染... "第k天感染的人" = "距离你 k 步的人"。这就是 BFS 的**分层性**。

### BFS vs DFS 的本质差异

| | DFS | BFS |
|---|---|---|
| 策略 | 深挖到底再回头 | 逐层横向扩张 |
| 底层结构 | **栈**（LIFO） | **队列**（FIFO） |
| 天然实现 | 递归 | 迭代 |
| 是否回溯 | ✅ 有 | ❌ 无 |
| 找最短路径（无权图） | ❌ 不保证 | ✅ **保证最短**！ |

> ✅ **BFS 的杀手锏**：在无权图中，BFS 从起点访问到任意顶点的**路径长度就是最短路径**。这是因为它按层遍历——第 k 层的顶点距离起点恰好 k 步。

## 第55页：BFS 序列示例（不唯一 + 具体路径）

### 同一个图的两种 BFS 序列

从 V0 出发：

- 序列1：V₀, **V₁, V₂**, V₃, V₄, V₅, V₆, V₇
- 序列2：V₀, **V₂, V₁**, V₅, V₆, V₃, V₄, V₇

区别在于：先访问 V₁ 还是 V₂？取决于 V₀ 邻接链表中哪个在前。

### BFS 生成树

上图中的**绿色边**标注了 BFS 生成树的结构：

```
              V0
             /  \
           V1    V2
          / \   /  \
        V3  V4 V5   V6
         \
         V7
```

- **黑色/橙色边**（图中）：BFS 访问过的原始边
- **绿色边**（BFS 树的边）：真正扩展新顶点用的边
- **紫色边**（V3-V7 或 V4-V7）：交叉边——发现时目标已访问

> 💡 **关键性质**：BFS 生成树是一棵**最短路径树**——从起点到任意顶点的路径长度就是它们在树中的深度。

### 关键认知

> ✅ 与 DFS 一样，**BFS 序列本身不唯一**，但如果**存储结构确定，则序列唯一**（因为邻接链表的顺序固定）。

## 第56页：BFS 完整执行示例（9顶点网格图）

### 图结构

```
A --- B --- E
|    |     |
D    C     G
|    |     |
F --- H --- I
```

### BFS 从 A 出发的执行过程

访问顺序：A(1) → B(2) → C(3) → D(4) → E(5) → F(6) → G(7) → H(8) → I(9)

| 步骤 | 访问 | 加入队列的新邻居 | 距离A |
|------|------|----------------|-------|
| 1 | A | B, D, C | 0 |
| 2 | B | E | 1 |
| 3 | C | F | 1 |
| 4 | D | (无新邻居) | 1 |
| 5 | E | G | 2 |
| 6 | F | H | 2 |
| 7 | G | (无新邻居) | 3 |
| 8 | H | I | 3 |
| 9 | I | (无新邻居) | 4 |

### BFS 生成树（右图）

```
     A(1)
    / | \
   B  D  C
   |     |
   E     F
   |     |
   G     H
         |
         I
```

> 💡 **观察**：BFS 生成树的**每一层深度就是该顶点到起点的最短路径长度**。
> - A 到 E 的最短距离 = E 在树中的深度 = 2 步（A→B→E）
> - A 到 I 的最短距离 = I 在树中的深度 = 4 步（A→C→F→H→I）

> 🎯 **和 DFS 生成树对比**：DFS 生成树倾向于"细长"（深度大），BFS 生成树倾向于"矮胖"（深度=最短距离）。

## 第57页：BFS 实现要点

### 两个核心事实

**1. BFS 每步访问多个顶点，且无回溯**

- DFS：访问一个 → 深入 → 回退 → 换方向
- BFS：**访问一整层** → **换到下一层** → 无回头

> **推论**：BFS **不是递归过程**！它是纯粹的迭代过程。

### 为什么 BFS 用队列，DFS 用栈？

| 数据结构 | 特性 | 适合谁 | 原因 |
|---------|------|-------|------|
| 栈（LIFO） | 后进先出 | DFS | 最新发现的顶点最先被处理 → "深入" |
| **队列（FIFO）** | **先进先出** | **BFS** | 先发现的顶点先被处理 → "按层扩展" |

> 💡 **底层原理**：想让"距离近的先处理"→ 先加入的先出来 → FIFO → 队列。这不是巧合，是"分层"这个需求推导出来的必然选择。

### 防重复访问机制

**2. 用 `visited[]` 数组标记访问状态**

和 DFS 一样，BFS 也需要 `visited[]` 数组，理由同样是——图有环，可能反复访问同一顶点导致死循环。

### BFS 伪代码框架

```
BFS(G, v):
    创建队列 Q
    visited[v] = TRUE
    访问 v
    Q.enqueue(v)
    
    while Q 非空:
        u = Q.dequeue()
        for u 的每个邻居 w:
            if !visited[w]:
                visited[w] = TRUE
                访问 w
                Q.enqueue(w)
```

> ⚠️ **易错点1**：**入队时就要标记**，不能等出队时才标记！
> - 错误做法：只在出队时标记 `visited`
> - 后果：同一个顶点可能被多次加入队列（因为多个邻居都发现它未标记）→ 队列爆炸 + 重复访问

> ⚠️ **易错点2**：BFS 也需要外层循环处理不连通图，和 DFS 驱动函数一样：
> ```c
> for(v=0; v<N; v++)
>     if(!visited[v])
>         BFS(G, v);
> ```

### BFS 与 DFS 的类比总结

| | DFS | BFS |
|---|---|---|
| 数据结构 | 栈（含递归调用栈） | 队列 |
| 实现方式 | 递归 or 迭代+栈 | **只能迭代+队列** |
| 空间复杂度 | O(N)（递归深度） | O(N)（队列最大宽度） |
| 时间复杂度 | O(N+E) 邻接表 / O(N²) 矩阵 | O(N+E) 邻接表 / O(N²) 矩阵 |
| 无权图最短路 | ❌ | ✅ |
| 应用 | 拓扑排序、连通分量、回溯 | 最短路径、层序遍历、状态搜索 |

## 总结

```
BFS 核心机制：
  队列 + visited数组 + 逐层扩展

BFS 生成树：
  ├── 每层深度 = 到起点的最短距离（无权图）
  ├── 是一棵"最短路径树"
  └── 存储结构决定唯一形状

时间复杂度：
  邻接表：O(N + E)  ← 每个顶点入队1次出队1次，每条边检查1次（或2次）
  邻接矩阵：O(N²)   ← 每个顶点出队后扫一整行

BFS vs DFS：
  同：都遍历所有顶点，都需要 visited[]
  异：BFS按层扩张（队列），DFS深度探索（栈/递归）
      BFS天然给出无权最短路，DFS天然给出拓扑序列
```

> 💡 **前后呼应**：BFS 的"分层"性质是后面 **Dijkstra 最短路径算法**的雏形——Dijkstra 可以看作带权版的 BFS，用**优先队列**代替普通队列，"距离最近的先出队"。

> 🎯 **一句话记忆**：**DFS 用栈爬得深，BFS 用队铺得远**。
