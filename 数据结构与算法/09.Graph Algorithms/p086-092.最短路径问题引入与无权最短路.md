## 第86页：最短路径问题的形式化定义

### 问题描述

**输入**：
- 有向图 G = (V, E)
- 每条边（arc）都有一个**非负权值**（nonnegative weight）
- 一个**源点** source vertex
- 一个**目标点** target vertex

**输出**：
- 从源点到目标点的**总代价最小**的路径

### 三个关键限定

| 限定 | 含义 | 为什么重要 |
|------|------|-----------|
| **有向图** | 边有方向 | 单行道场景 |
| **非负权值** | weight ≥ 0 | Dijkstra 只在非负权下正确！ |
| **有源点** | 从某个特定顶点出发 | 单源最短路，不是全源 |

> 🎯 **生活类比**：
> - 顶点 = 城市，边 = 公路，权值 = 里程数 → **最短里程**问题
> - 顶点 = 状态，边 = 状态转移，权值 = 转移代价 → **最优策略**问题
> - 顶点 = 网页，边 = 链接，权值 = 加载时间 → **最快加载路径**问题

### 为什么强调"非负权值"？

- **Dijkstra 算法要求非负权**——它假设"一旦确定最短路，就不可能被更优的路径更新"，这个假设在负权下不成立
- 有负权时要用 **Bellman-Ford 算法**（复杂度更高 O(VE)）
- 有负环（环的总权值 < 0）时：最短路径**没有下界**（一直绕环就能无限减小），问题无解

> 💡 **本课程范围**：只讨论非负权，所以放心用 Dijkstra。

### 最短路径的多种变体

| 问题 | 输入 | 算法 |
|------|------|------|
| **单源最短路（无权）** | 一个源点 s | BFS，O(V+E) |
| **单源最短路（非负权）** | 一个源点 s，权≥0 | **Dijkstra**，O(V²) 或 O((V+E)logV) |
| 单源最短路（有负权无负环） | 一个源点 s，可负权 | Bellman-Ford，O(VE) |
| 全源最短路 | 任意两点之间 | **Floyd-Warshall**，O(V³) |

## 第87页：一个关键的观察——权重路径 vs 无权路径

### 示例图（7顶点带权有向图）

```
         v1 ──2──→ v2
       ↗ ↓  ↘ 3         ↓ 10
    4/  1↓  ↘         ↓
      v3 ──2──→ v4 ──2──→ v5
       ↘  5           4↓   ↑ 6
         ↘             ↓   ↑
          v6 ←──1──── v7
```

（近似结构，具体见 PPT）

### 观察结论

> **从 v1 到 v6 的最短带权路径**：`v1 → v4 → v7 → v6`，总权 = **1 + 4 + 1 = 6**

> **从 v1 到 v6 的最短无权路径**：只有 **2 条边**（比如 v1 → v4 → v6，权=1+8=9 但边数只有2）

### 关键认知

> ✅ **"最少的边数"和"最小的权值和"是两个完全不同的目标！**

- 无权最短路径 = **BFS 就够了**（按层扩展 = 边数最少）
- 带权最短路径 = **需要 Dijkstra**（考虑权值，可能绕远路但总代价更小）

> 🎯 **类比**：
> - 无权 = "换乘次数最少的地铁路线"（每段边等价）
> - 带权 = "总耗时最短的路线"（要考虑每段用时）
> - 有时"多换一次乘反而更快"——这就是无权和带权差异的直觉

## 第88页：无权最短路径问题

### 问题描述

给定**无权图** G 和一个源点 s，求 **s 到所有其他顶点的最短路径**（按边数计）。

### "无权"= 每条边权值视为1

- 所有边权重都相等（默认1）
- 最短路径长度 = 边数
- BFS 天生按边数分层 → **BFS 就是无权最短路的天然解法**

### 为什么讨论"到所有顶点"？

- 求"s 到某个特定顶点 t"的最短路径 = 求"s 到所有顶点"，然后取出 t 的结果
- 反之，"求 s 到所有点"和"求 s 到 t"的复杂度是一样的（都要遍历图）
- 所以**单源最短路径**问题通常一次性求出到所有点的距离

> 💡 **实际算法**：BFS 一遍就能得到 s 到所有可达顶点的最短距离，一次 O(V+E)。

## 第89页：BFS 求无权最短路——起点（距离 0）

### 从 s = v3 出发

**初始化**：
- v3 是起点，距离自己 = **0**
- 其他所有顶点距离 = **∞**（未知/不可达）

**图上标注**（PPT）：只有 v3 标注了 0，其他顶点未标注。

### 距离数组 dist[]

```
dist[v1] = ∞
dist[v2] = ∞
dist[v3] = 0     ← 起点
dist[v4] = ∞
dist[v5] = ∞
dist[v6] = ∞
dist[v7] = ∞
```

> 💡 **∞ 的实现**：通常用 `INT_MAX` 或 `0x3f3f3f3f` 表示。

### 认知起点：距离 0 的顶点集合

第 0 层（距离 = 0）：**{v3}**

**下一步**：找所有从 v3 出发**一步能到达**的顶点。这些顶点的距离就是 1。

---

## 第90页：BFS 扩展第一层（距离 1）

### 从 v3 出发能一步到达的顶点

> **v1 和 v6** 是从 v3 出发一条边可到的顶点（v3 → v1 和 v3 → v6）。

### 更新距离数组

```
dist[v1] = 1     ← 从∞更新为1
dist[v3] = 0
dist[v6] = 1     ← 从∞更新为1
其他 = ∞
```

**图上标注**：v3 旁标 0，v1 旁标 1，v6 旁标 1。

### 认知第 1 层

第 1 层（距离 = 1）：**{v1, v6}**

**下一步**：从 v1 和 v6 出发一步能到的顶点（且这些顶点还未确定距离）。

> ⚠️ **重点**：只考虑**未确定距离**的顶点！如果某个邻居已经有距离了，说明前面已经通过更短的路径找到它——**跳过**。这是 BFS 保证"最短"的关键。

---

## 第91页：BFS 扩展第二层（距离 2）

### 从 v1, v6 出发能到达的新顶点

**v1 的邻居**：v2, v4（可能还有其他）
- v2 未确定 → `dist[v2] = 2`
- v4 未确定 → `dist[v4] = 2`

**v6 的邻居**：（观察图）可能是死胡同或者能到 v4/v7
- 如果 v6 → v4，但 v4 已被 v1 更新为 2 → 跳过
- 如果 v6 无未访问邻居 → 无贡献

### 更新距离数组

```
dist[v1] = 1
dist[v2] = 2     ← 新
dist[v3] = 0
dist[v4] = 2     ← 新
dist[v6] = 1
其他 = ∞
```

**图上标注**：v3=0, v1=1, v6=1, v2=2, v4=2

### 关键判断

> **PPT 原文**：*"...whose shortest paths are not already known."*

翻译：只更新那些**距离还未知**（=∞）的顶点。已经有值的说明前面已经用更短的路径到达过，不需要更新。

> 💡 **这就是 BFS 保证正确性的核心**：**先访问的距离 ≤ 后访问的距离**。因为 BFS 按层扩展，越早访问的顶点距离越小。

## 第92页：BFS 扩展第三层 + 最终结果

### 从 v2, v4 出发能到达的新顶点

- **v2 的邻居**：v5（或其他）→ `dist[v5] = 3`
- **v4 的邻居**：v5(已=3或未), v7 → `dist[v7] = 3`

（细节取决于图的实际边，但结论是 v5 和 v7 距离为 3）

### 最终距离数组

```
dist[v1] = 1
dist[v2] = 2
dist[v3] = 0     ← 起点
dist[v4] = 2
dist[v5] = 3
dist[v6] = 1
dist[v7] = 3
```

### 分层可视化

| 层 | 距离 | 顶点集合 |
|----|------|---------|
| 0 | 0 | {v3}（起点） |
| 1 | 1 | {v1, v6} |
| 2 | 2 | {v2, v4} |
| 3 | 3 | {v5, v7} |

> ✅ **这就是从 v3 出发的完整"距离分布图"**。

### BFS 求最短路的算法框架

```c
BFS_shortest_path(G, s):
    for each v in V:
        dist[v] = ∞
    dist[s] = 0
    Queue Q
    Q.enqueue(s)
    
    while Q not empty:
        u = Q.dequeue()
        for each neighbor w of u:
            if dist[w] == ∞:                // 未访问
                dist[w] = dist[u] + 1        // 距离 + 1
                Q.enqueue(w)
```

### 关键实现细节

**1. 用 `dist[w] == ∞` 代替 `visited[]` 数组**：
- 传统 BFS 用 `visited[]` 布尔数组
- 无权最短路 BFS 直接用 `dist[]` 是否为 ∞ 判断
- **一石二鸟**：既标记访问又记录距离

**2. `dist[w] = dist[u] + 1`**：
- w 是通过 u 发现的，所以距离比 u 多 1
- 因为 BFS 保证按层扩展，第一次遇到 w 时的距离就是最短的

> 💡 **为什么第一次遇到就是最短的？** BFS 是"距离小的先出队"，所以处理 u 时，dist[u] 是当前所有已知顶点中最小的（除已处理外）。u 更新的邻居 w 得到 dist[u]+1，这是从"距离已知的顶点"能得到 w 的最小值。

### 复杂度

- 每个顶点入队/出队 1 次 → O(V)
- 每条边被检查 1 次（有向图）→ O(E)
- **总计 O(V + E)**

### 局限性

> ⚠️ **BFS 只对无权图有效！**

带权图上直接跑 BFS 会得到错误答案，因为：
- BFS 按边数分层，忽略权值
- 一条 5 步的路径可能比 2 步的路径**总权更小**（权值差异大时）
- 需要更复杂的算法（Dijkstra）才能正确处理

## 总结

```
最短路径问题体系：

问题变体：
├── 单源无权 → BFS，O(V+E)                    ← 本节
├── 单源非负权 → Dijkstra，O(V²) 或 O((V+E)log V)
├── 单源可负权 → Bellman-Ford，O(VE)
└── 全源 → Floyd-Warshall，O(V³)

BFS 求无权最短路：
├── 起点 dist=0，其他 dist=∞
├── 按层扩展：距离k的顶点释放距离k+1的邻居
├── 每个顶点只在"第一次被发现"时更新（保证最短）
└── 复杂度 O(V+E)

核心洞察：
  BFS "按边数分层" ⇔ "边数=距离"（无权前提）
  加权后这个等价关系失效 → 需要 Dijkstra
```

> 💡 **前后呼应**：
> - **之前**：BFS 作为遍历算法讲过
> - **现在**：BFS 是无权最短路的天然解法
> - **接下来**：Dijkstra = "带权版 BFS"——用**优先队列**代替普通队列，"当前距离最小的先出队"，把 BFS 的"边数分层"升级为"权值分层"

> 🎯 **一句话记忆**：**"BFS 从起点像水波扩散，第 k 圈涟漪就是距离 k 的顶点。"**
