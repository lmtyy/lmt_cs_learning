# 最小生成树 MST 引入（p112-115）

进入本章最后一大主题：最小生成树（Minimum Spanning Tree）。从"用最少的电线把所有房间连起来"引入，给出 MST 的精确定义，然后预告两种经典贪心算法。

---

## 第112页：MST 的基本前提

### 两条基础事实

> **1. 对连通图 G，从任意顶点启动 DFS 或 BFS 都能访问所有顶点。**

这是遍历章节的结论——连通意味着所有点互达，遍历能完全覆盖。

> **2. 生成树 = 仅由 G 中的边组成，且包含 G 所有顶点的树。**

回顾第 17 页的定义：生成树 ⊂ G（边是 G 的子集），覆盖所有顶点，连通且无环。

> 💡 **MST 要解决的核心问题**：在所有可能的生成树中，找**边权之和最小**的那棵。

---

## 第113页：MST 的三个特征

### 形式化定义

**最小生成树（Minimum Spanning Tree）**满足：

| 特征 | 含义 |
|------|------|
| **n 个顶点** | 覆盖图的全部顶点 |
| **n-1 条边** | 树的固有性质 |
| **无环（acyclic）** | 是树，不是一般子图 |
| **✅ 总代价最低（lowest total cost）** | 所有生成树中，边权之和最小 |

> 🎯 **类比**：一个城市里有多条公路连接各个小区。修所有路太贵，但必须保证每个小区都能互相到达。MST = "花最少钱修路，让所有小区连通"。

### MST vs 最短路径

| | MST | 最短路径 |
|---|---|---|
| 目标 | 全局边权和最小 | 源到目标的路径权和最小 |
| 输出 | 一棵树（n-1 条边） | 一条路径 |
| 图类型 | 无向连通图 | 有向或无向 |
| 算法 | Prim / Kruskal | Dijkstra / BFS |

> ⚠️ **常见混淆**：MST 中两点间的路径**不一定是最短路径**！MST 优化的是"全局总代价"，不是任何一对顶点之间的距离。

## 第114页：MST 具体示例（7顶点带权图）

### 原图（左图）

```
        v1 ──2── v2
       / |  \  3    |10
     4/  1  \     |
    v3──2── v4 ──7── v5
     | \    | \      |
    5|  8   4  6     6
     |   \  |        |
    v6 ──1── v7
```

边权列表：v1-v2(2), v1-v3(4), v1-v4(1), v2-v4(3), v2-v5(10), v3-v4(2), v3-v6(5), v4-v5(7), v4-v6(8), v4-v7(4), v5-v7(6), v6-v7(1)

### 最小生成树（右图）

选中的边：**v1-v2(2), v1-v4(1), v3-v4(2), v4-v7(4), v5-v7(6), v6-v7(1)**

总代价 = 2 + 1 + 2 + 4 + 6 + 1 = **16**

```
    v1 ──2── v2
     |
     1
     |
    v4 ──2── v3
     |
     4
     |
    v7 ──6── v5
     |
     1
     |
    v6
```

> 💡 **PPT 指出**：这棵 MST 恰好是**唯一的**。但这不是一般情况——如果有多条权值相同的边，MST 可能不唯一（比如两条权值为3的边互相替代）。

> ✅ **MST 唯一性条件**：如果所有边的权值都**互不相同**，则 MST 一定唯一。

## 第115页：MST 的应用场景 + 两种算法预告

### 真实应用

> **"If we need to wire a house with a minimum of cable, how to solve it?"**

| 应用场景 | 顶点 | 边 | 权值 |
|---------|------|---|------|
| 房屋布线 | 房间/设备 | 可能的布线路径 | 电缆长度 |
| 通信网络 | 基站 | 光纤连接 | 铺设费用 |
| 城市交通 | 城市 | 公路 | 修建成本 |
| 电路板设计 | 引脚 | 走线 | 线长 |

### 示例图（6顶点）

```
      V0
    6/ | \5
   V1  1  V3
    5\ | /5
      V2
    6/ | \4
   V4  2  V5
      \ /
       6
```

### 两种经典算法

> **"There are two basic algorithms to solve this problem; both are greedy."**
>
> **Prim's Algorithm and Kruskal's Algorithm.**

| | Prim | Kruskal |
|---|---|---|
| 策略 | 从一个顶点开始，每次加入**最短的跨切割边** | 从全局出发，每次加入**不成环的最短边** |
| 增长方式 | "逐点扩展"（树逐步生长） | "逐边合并"（森林逐步连通） |
| 底层结构 | 优先队列 | 并查集（Union-Find） |
| 适合 | 稠密图 | 稀疏图 |
| 复杂度 | O(V²) 或 O((V+E)log V) | O(E log E) |
| 类似 | Dijkstra（同为贪心+优先队列） | 拓扑排序（按边排序处理） |

> 🎯 **共同点**：两者都是**贪心算法（Greedy）**——每一步都选"当前看起来最优"的边，最终得到全局最优解。

> 💡 **贪心为什么正确？** MST 的关键定理——**切割性质（Cut Property）**：对于图的任意一个切割（把顶点分成两组），跨越切割的最轻边**一定属于某棵 MST**。Prim 和 Kruskal 都在不同的切割上反复应用这个性质。

> 💡 **前后呼应**：
> - Prim 算法结构与 Dijkstra **几乎相同**——只是更新规则从"dist[v]+w(v,w)"变成"w(v,w)"
> - Kruskal 算法需要**并查集**——高效判断"加这条边会不会成环"

## 总结

```
最小生成树 MST：
├── 定义：连通图的边权和最小的生成树
├── 三个特征：n顶点 + n-1边 + 无环 + 权和最小
├── 前提：无向连通带权图
├── 唯一性：边权全不同 → 唯一；否则可能多棵
└── 两种贪心算法：
    ├── Prim：逐点扩展（像 Dijkstra）
    └── Kruskal：逐边合并（像排序+并查集）

MST ≠ 最短路径：
  MST 优化全局总代价
  最短路径优化单条路径代价
```

> 🎯 **一句话总结 MST**：**"用最便宜的方式把所有点连起来——不求任何两点之间最短，只求整体花费最少。"**
