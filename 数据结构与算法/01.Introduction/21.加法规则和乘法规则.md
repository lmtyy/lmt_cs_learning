这张图片展示了算法复杂度分析中两个最基础、也是最重要的运算规则：**加法规则**和**乘法规则**。

简单来说，这就是教你**当代码变复杂（有多个部分或嵌套）时，如何计算总的复杂度**。

下面我为你详细拆解这两个规则的含义、实际例子以及数学证明。

### 一、 规则解读与直观理解

已知条件：

- 算法1的时间复杂度是 $T_1(N) = O(f(N))$
- 算法2的时间复杂度是 $T_2(N) = O(g(N))$

#### 1. 加法规则 (Rule a) —— “谁大听谁的”

**公式**：

$$T_1(N) + T_2(N) = O(\max(f(N), g(N)))$$

- **含义**：当两个算法**顺序执行**（一个做完，再做另一个）时，总的时间复杂度由**耗时最长**的那部分决定。

- **代码例子**：

  C++

  ```c++
  // 第一部分：复杂度 O(N)
  for (int i = 0; i < N; i++) { printf("%d", i); }
  
  // 第二部分：复杂度 O(N^2)
  for (int i = 0; i < N; i++) {
      for (int j = 0; j < N; j++) { printf("%d", i+j); }
  }
  ```

  - 总时间 = $N + N^2$。
  - 因为 $N^2$ 远大于 $N$，所以我们只保留 $N^2$。
  - **结果**：$O(N^2)$。

#### 2. 乘法规则 (Rule b) —— “层层相乘”

**公式**：

$$T_1(N) \times T_2(N) = O(f(N) \times g(N)))$$

- **含义**：当两个算法**嵌套执行**（比如循环里套循环）时，总的时间复杂度是它们复杂度的**乘积**。

- **代码例子**：

  C++

  ```c++
  // 外层：复杂度 O(N)
  for (int i = 0; i < N; i++) {
      // 内层：复杂度 O(N)
      // 这里的操作每一轮外层循环都要做一次
      for (int j = 0; j < N; j++) {
          printf("*");
      }
  }
  ```

  - 总时间 = $N \times N$。
  - **结果**：$O(N^2)$。

### 二、 严格的数学证明 (Proof)

这是算法考试中经常出现的考点。我们需要利用 **Big O 的定义** 来证明。

**回顾定义**：

$T(N) = O(h(N))$ 意味着存在常数 $c$ 和 $n_0$，使得当 $N \ge n_0$ 时，$T(N) \le c \cdot h(N)$。

**准备工作（翻译已知条件）**：

1. 因为 $T_1(N) = O(f(N))$，所以存在 $c_1, n_1$，使得 $T_1(N) \le c_1 \cdot f(N)$ (当 $N \ge n_1$)。
2. 因为 $T_2(N) = O(g(N))$，所以存在 $c_2, n_2$，使得 $T_2(N) \le c_2 \cdot g(N)$ (当 $N \ge n_2$)。

为了让两个不等式同时成立，我们需要取一个公共的起点 $n_0$。

**设 $n_0 = \max(n_1, n_2)$**。

#### 证明 (a)：加法规则

**目标**：证明 $T_1(N) + T_2(N) \le C \cdot \max(f(N), g(N))$。

**推导过程**：

当 $N \ge n_0$ 时，我们将上面两个不等式相加：

$$T_1(N) + T_2(N) \le c_1 \cdot f(N) + c_2 \cdot g(N)$$

这里使用一个关键的**放缩技巧**：

因为 $f(N)$ 和 $g(N)$ 肯定都小于等于它们俩之中的最大值，即：

- $f(N) \le \max(f(N), g(N))$
- $g(N) \le \max(f(N), g(N))$

所以，我们可以把右边的式子替换成：

$$T_1(N) + T_2(N) \le c_1 \cdot \max(f(N), g(N)) + c_2 \cdot \max(f(N), g(N))$$

合并同类项（提取公因式）：

$$T_1(N) + T_2(N) \le (c_1 + c_2) \cdot \max(f(N), g(N))$$

**结论**：

我们找到了一个新的常数 $C = c_1 + c_2$，使得总时间小于 $C$ 乘以最大值。

所以 $T_1(N) + T_2(N) = O(\max(f(N), g(N)))$。

**证毕。**

#### 证明 (b)：乘法规则

**目标**：证明 $T_1(N) \times T_2(N) \le C \cdot (f(N) \times g(N))$。

**推导过程**：

当 $N \ge n_0$ 时，直接将两个不等式相乘：

$$T_1(N) \times T_2(N) \le (c_1 \cdot f(N)) \times (c_2 \cdot g(N))$$

利用乘法结合律整理一下常数：

$$T_1(N) \times T_2(N) \le (c_1 \cdot c_2) \cdot (f(N) \times g(N))$$

**结论**：

我们找到了一个新的常数 $C = c_1 \cdot c_2$，使得总时间小于 $C \cdot (f(N) \times g(N))$。

**证毕。**

### 总结

这两个规则其实就是告诉你：在分析代码时，**看到并列的代码块就取最大值，看到嵌套的代码块就做乘法**。证明过程的核心在于利用不等式的放缩性质和构造新的常数 $C$。
