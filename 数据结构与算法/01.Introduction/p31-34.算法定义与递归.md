这四页PPT系统地给出了算法（Algorithm）的正式定义，并引入了算法设计中极具威力的核心思想：**递归（Recursion）**。

以下是每一页PPT知识点的详细解析：

### 第一页：什么是算法？(What is an algorithm?)

这页PPT从最基础的角度定义了算法的作用和基本模型。

- **计算过程 (Computational procedure)**：算法本质上是一套处理计算的步骤。它接收某个值作为**输入 (input)**，经过处理后，产生某个值作为**输出 (output)**。
- **解决特定问题 (Solving a well-specified computational problem)**：算法不是漫无目的的代码，它是为了解决某个具体定义的数学或逻辑问题而存在的工具。
- **实例：排序问题 (Sorting problem)**：
  - **输入**：一个包含 $n$ 个数字的序列 $<a_1, a_2, ..., a_n>$。
  - **输出**：对输入序列进行重新排列（permutation），得到一个新的序列 $<a_1', a_2', ..., a_n'>$。并且这个新序列必须满足一个特定的条件：从小到大排列，即 $a_1' \le a_2' \le ... \le a_n'$。

### 第二页：算法必须满足的严苛条件 (Criteria of an algorithm)

一段代码能跑不代表它配叫“算法”。在计算机科学中，真正的算法必须严格满足以下几个特征：

- **输入 (Input)**：包含零个或多个外部提供的量。
- **输出 (Output)**：**至少产生一个量**。一个不产生任何结果的程序是毫无意义的。
- **确定性与有限性 (Definiteness Finiteness)**：
  - 确定性意味着算法的每一步都必须精确无误，不能有歧义。
  - 有限性（Finiteness）极其重要：算法**必须在有限的步骤后终止 (Terminates)**。如果你写了一个死循环（比如一直打印1），那叫程序，但不叫算法。
- **有效性/可行性 (Effectiveness)**：算法中的每一个指令都必须足够基础（basic enough），原则上哪怕是一个人拿着纸和笔，只要花足够的时间，也能一步步执行出结果。

### 第三页：递归算法基础 (Recursive Algorithms)

这页PPT引入了编程中非常重要且常常让人头疼的概念——递归。简单来说，递归就是“套娃”。

- **递归函数 (Recursive functions) 的两种形式**：
  - **直接递归 (Direct recursion)**：函数在自己的代码里直接调用自己。
  - **间接递归 (Indirect recursion)**：函数A调用了函数B，而函数B在执行过程中又反过来调用了函数A，形成了一个循环调用链。
- **终止条件 (Terminating condition)**：
  - 这是递归的生命线。如果没有终止条件，递归就会变成死循环（无限套娃），最终导致内存撑爆（Stack Overflow）。
  - 当满足这个条件时，函数会**直接计算并返回结果，而不再调用自己**，从而开始一层层向外“拆套娃”。

### 第四页：递归设计的两大黄金法则 (Characteristics of a recursive definition)

这是这四页中最重要的一页，教你如何写出正确的递归，并通过一个反面教材进行了说明。

写递归必须满足两个条件：

1. **基线条件 / 出口 (Base case)**：必须存在至少一个可以**不需要递归就能直接求解**的基础情况。（对应上一页的终止条件）。
2. **不断推进 (Making progress)**：每一次递归调用，传入的参数都必须让你**更接近基线条件**。

- **案例分析：为什么绿框里的代码是一个“Bad（糟糕）”的递归？**

  - 代码逻辑：

    - `if (N == 0) return 0;` （这是 Base case，出口没问题）。
    - `else return Bad(N / 3 + 1) + N - 1;` （这是递归调用）。

  - **致命缺陷在于没有“Making progress（不断逼近出口）”**：

    假设在C语言或Java中（整数除法会向下取整），如果我们传入 `N = 1`：

    - `N == 0` 为假，进入 `else`。
    - 计算传给下一次递归的值：`1 / 3` 的整数结果是 `0`。然后 `0 + 1 = 1`。
    - 于是，代码执行了 `return Bad(1) + 1 - 1;`
    - 发现了没？你传入了 `1`，它内部又调用了 `Bad(1)`。这就是一个永远也解不开的死循环。传入 `N = 2`同理（`2/3 + 1 = 1`，也会无限调用 `Bad(1)`）。

**总结**：这四页PPT清晰地勾勒了算法的边界，并指出了编写递归函数时最容易犯的致命错误——忘记确保每一次递归都在实打实地朝着“出口”迈进。
