# 如何评价算法的好坏：复杂度分析

评价一个算法的核心指标有两个：**时间复杂度**（快不快）和 **空间复杂度**（省不省内存）。我们通常使用 **大O表示法 (Big O Notation)** 来描述。

## 一、 大O表示法 (Big O Notation)

$O(f(n))$ 描述的是算法运行时间的**增长趋势**（Growth Rate），而不是具体的时间。

- **$n$**：代表输入数据的规模（比如数组里有多少个元素）。
- **核心思想**：我们只关注**最高阶项**，忽略常数和低阶项。
  - 例如：$2n^2 + 10n + 1000$ $\rightarrow$ 简化为 **$O(n^2)$**。
  - 因为当 $n$ 变得极大时，10n 和 1000 带来的影响微乎其微。

------

## 二、 时间复杂度 (Time Complexity)

这是衡量算法执行效率的主要标准。以下按效率**从高到低**排列：

### 1. $O(1)$ - 常数阶 (Constant Time)

**最快。** 无论数据量 $n$ 多大，消耗的时间固定。

- **场景**：访问数组中的某个特定索引（`arr[5]`）、哈希表查找。

Python

```python
def check_first(arr):
    print(arr[0])  # 只做一次操作，与 arr 长度无关
```

### 2. $O(\log n)$ - 对数阶 (Logarithmic Time)

**极快。** 每次操作都把问题规模砍掉一半。

- **场景**：二分查找 (Binary Search)、二叉树搜索。
- **理解**：100万个数据，只要找20次左右就能找到。

Python

```python
# 二分查找逻辑
while low <= high:
    mid = (low + high) // 2
    # 每次循环，搜索范围减半
```

### 3. $O(n)$ - 线性阶 (Linear Time)

**一般。** 耗时与数据量成正比。数据翻倍，时间翻倍。

- **场景**：遍历数组、简单的线性查找。

Python

```python
def print_all(arr):
    for i in arr:  # 数组有多长，循环跑多少次
        print(i)
```

### 4. $O(n \log n)$ - 线性对数阶

**稍慢。** 它是 $n \times \log n$。

- **场景**：大多数高效排序算法（归并排序、快速排序、堆排序）。
- **注意**：这是基于比较排序的算法能达到的理论最快速度。

### 5. $O(n^2)$ - 平方阶 (Quadratic Time)

**慢。** 数据量稍微增大，耗时爆炸式增长。

- **场景**：双层嵌套循环（冒泡排序、选择排序）。

Python

```python
for i in range(n):       # 外层 n 次
    for j in range(n):   # 内层 n 次
        print(i, j)      # 总共 n * n 次
```

### 6. $O(2^n)$ - 指数阶 (Exponential Time)

**极慢（不可接受）。**

- **场景**：未优化的递归（如斐波那契数列的暴力递归解法）。
- **后果**：$n$ 稍微大一点（比如 n=30），电脑就算不过来了。

------

## 三、 空间复杂度 (Space Complexity)

衡量算法在运行过程中需要占用多少**额外内存**。

1. **$O(1)$**：原地操作，只需要几个临时变量，不随 $n$ 增加而占用更多内存。（如：冒泡排序）。
2. **$O(n)$**：需要开辟一个和输入一样大的辅助空间，或者递归深度为 $n$。（如：归并排序需要临时数组）。

------

## 四、 最好、最坏与平均情况

同一个算法，在不同数据排列下表现可能不同。

1. **最好情况 (Best Case)**：数据已经排好序了，你只需要检查一遍。
2. **最坏情况 (Worst Case)**：**最重要指标**。数据完全是反的。我们在工程中通常以最坏情况作为算法的保障底线。
3. **平均情况 (Average Case)**：综合评估，更能反映实际性能。
