# 方法论：如何分析一个算法 (How to Analyze an Algorithm)

分析一个算法，主要考察两个维度：

1. **正确性 (Correctness)**：它真的能算出对的结果吗？
2. **效率 (Efficiency)**：它需要消耗多少资源（时间 & 空间）？

## 一、 核心前提：RAM 计算模型

在分析之前，我们必须约定一个“虚拟计算机”，即 **Random Access Machine (RAM) 模型**。

- **单处理器**：指令一条接一条顺序执行。
- **基本操作**：加减乘除、赋值、逻辑判断、数组索引访问，耗时均为 **常数时间 $O(1)$**。
- **内存访问**：访问任意内存地址耗时相同。

> **注意**：我们分析的是“步数 (Steps)”，而不是“秒数 (Seconds)”。

------

## 二、 维度一：正确性分析 (Correctness)

写完代码，如何证明它是对的？靠跑测试用例（Test Case）是不够的，通常使用 **循环不变式 (Loop Invariant)** 来证明。

### 什么是循环不变式？

它是一个在循环的**开始前**、**进行中**、**结束后**都始终成立的**逻辑性质**。类似于数学归纳法。

### 案例：插入排序 (Insertion Sort)

假设我们要把数组 `A` 排序。

- **不变式**：`A[0...i-1]` 这个子数组始终是有序的。
- **初始化**：当 `i=1` 时，子数组只有 `A[0]` 一个元素，显然是有序的（成立）。
- **保持**：每次循环，我们把 `A[i]` 插入到 `A[0...i-1]` 的正确位置，使得 `A[0...i]` 变得有序。
- **终止**：当循环结束（`i=n`），意味着 `A[0...n-1]`（整个数组）是有序的。**证毕。**

------

## 三、 维度二：渐进效率分析 (Asymptotic Analysis)

这是工程中最常用的部分。我们关心的是：**当输入规模 $N \to \infty$ 时，运行时间的增长率。**

### 1. 三种符号体系

- **$O$ (Big-O)**：**上界 (Upper Bound)**。
  - 含义：算法**最坏**也就这么慢，不会更差了。
  - *工程意义：这是我们给客户的“性能承诺书”。*
- **$\Omega$ (Big-Omega)**：**下界 (Lower Bound)**。
  - 含义：算法**最好**也就这么快，不可能更好了。
- **$\Theta$ (Big-Theta)**：**紧确界 (Tight Bound)**。
  - 含义：既是上界又是下界，精准地描述了性能。

### 2. 三种情况 (Cases)

- **最坏情况 (Worst Case)**：**（重点）** 输入数据完全逆序。我们通常以这个作为算法的最终评价指标。
- **平均情况 (Average Case)**：输入数据随机分布。需要用到概率论（期望值），通常较难计算。
- **最好情况 (Best Case)**：输入数据已经排好序。通常没有参考价值（除非是专门针对有序数据的优化算法）。

------

## 四、 高阶技巧：摊还分析 (Amortized Analysis)

有些算法（如动态数组 `std::vector`），大部分操作很快，偶尔会发生一次极慢的操作（扩容）。

如果只看“最坏情况”，它很慢；但如果把总耗时**平摊**到每次操作上，它其实很快。

### 案例：动态数组 (Vector) 的 `push_back`

- **普通情况**：数组没满，直接放进去。耗时 $O(1)$。
- **扩容情况**：数组满了，开辟 2 倍新空间，把 $N$ 个旧元素搬过去。耗时 $O(N)$。
- **分析**：
  - 虽然最坏是 $O(N)$，但扩容发生的频率极低（指数级间隔）。
  - **结论**：`push_back` 的**摊还复杂度**是 **$O(1)$**。

## 五、 实战：如何计算复杂代码的时间复杂度？

我们通过一段 C++ 代码来演示具体的计算步骤。

**代码逻辑**：双层循环，但内层循环次数取决于外层变量 `i`。

C++

```c++
void analyzeMe(int n) {
    int counter = 0;              // cost: c1 (执行1次)
    
    for (int i = 0; i < n; i++) { // cost: c2 (执行 n+1 次)
        
        // 注意：j < i，意味着内层循环次数在增加
        for (int j = 0; j < i; j++) { // cost: c3
            counter++;                // cost: c4
        }
    }
}
```

### 步骤 1：识别基本操作 (Identify Basic Operation)

代码中最核心、执行最频繁的语句是 `counter++`。

### 步骤 2：建立求和公式 (Summation)

内层循环执行次数取决于 `i`：

- 当 `i=0`，内层跑 0 次。
- 当 `i=1`，内层跑 1 次。
- ...
- 当 `i=n-1`，内层跑 $n-1$ 次。

总执行次数 $T(n)$ 是一个等差数列求和：

$$T(n) = 0 + 1 + 2 + ... + (n-1)$$

### 步骤 3：求解与化简 (Solve & Simplify)

根据高斯求和公式：

$$T(n) = \frac{(n-1) \times n}{2} = \frac{n^2 - n}{2} = \frac{1}{2}n^2 - \frac{1}{2}n$$

### 步骤 4：确定渐进复杂度 (Asymptotic Notation)

1. 保留最高阶项：$\frac{1}{2}n^2$
2. 去除常数系数：$n^2$

**最终结果：** 该算法的时间复杂度为 **$O(n^2)$**。

------

## 六、 总结：分析算法的标准化流程

1. **定义输入规模**：什么是 $N$？（数组长度、矩阵的维数、树的节点数等）。
2. **选取核心语句**：哪一行代码运行次数最多？
3. **计算执行次数**：建立关于 $N$ 的数学表达式（通常涉及数列求和或递归树）。
4. **保留最高阶**：忽略低阶项和常数，得出 $O$。
5. **考虑特殊情况**：检查是否有最好/最坏/摊还场景的差异。
