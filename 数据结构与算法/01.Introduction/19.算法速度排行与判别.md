这一部分的内容展示了计算机科学中最重要的基石之一：**典型增长率（Typical Growth Rates）与数学背景**。它本质上是一张“算法速度排行榜”，告诉我们在面对不同规模的数据（$N$）时，哪些算法跑得快，哪些跑得慢。

以下是关于这些复杂度的详细解析，以及如何用“肉眼”和逻辑去判断代码属于哪一种复杂度。

### 一、 算法界的“速度阶梯”

根据增长率从慢到快（即效率从高到低）的排序如下：

1. **$O(1)$ - 常数阶 (Constant)**：**最快**。无论数据多少，时间不变。
2. **$O(\log N)$ - 对数阶 (Logarithmic)**：**非常快**。随着数据量爆炸式增长，时间只增加一点点。
3. **$O(N)$ - 线性阶 (Linear)**：**中规中矩**。数据翻倍，时间翻倍。
4. **$O(N \log N)$ - 线性对数阶**：**稍慢**。通常是高效排序算法的极限。
5. **$O(N^2)$ - 平方阶 (Quadratic)**：**慢**。通常是双层循环。
6. **$O(N^3)$ - 立方阶 (Cubic)**：**很慢**。三层循环。
7. **$O(2^N)$ - 指数阶 (Exponential)**：**极慢**。数据稍微多一点（比如30个）就会跑死机器。
8. **$O(N!)$ - 阶乘阶**：**灾难**。通常用于解决全排列问题，只能处理极小数据。

### 二、 如何精准判断复杂度？（判别指南）

很多时候我们不仅要背诵这个顺序，更要在看到代码或思路时能立刻反应过来。以下是核心判别技巧：

#### 1. 怎么判断是 $O(\log N)$？（关键词：二分、砍半）

只要你看到代码中有“**每次循环，变量都除以 2**”或者“**每次操作都把范围缩小一半**”的逻辑，它就是 $O(\log N)$。

- **特征**：并没有遍历所有数据，而是跳着走。

- **经典场景**：二分查找（Binary Search）、在字典里查字（你不会一页页翻，而是直接翻中间）。

- **代码长相**：

  C++

  ```c++
  // 每次 i 变成了 i*2，或者 i/2
  while (i < N) {
      i = i * 2; 
  }
  ```

  问：2的多少次方等于N？答案就是 $\log_2 N$。

#### 2. 怎么判断是 $O(N)$？（关键词：遍历、一遍）

这是最常见的。凡是把数据**从头到尾看一遍**，且每个数据处理时间差不多，就是 $O(N)$。

- **特征**：一个简单的 `for` 循环。

- **代码长相**：

  C++

  ```c++
  for (int i = 0; i < N; i++) {
      print(i);
  }
  ```

#### 3. 怎么判断是 $O(N \log N)$？（关键词：分治、排序、循环套二分）

这是你问的重点。$N \log N$ 实际上是 $N \times \log N$。这意味着：**你执行了 $N$ 次任务，但这每次任务的复杂度是 $\log N$**（或者反过来）。

- **核心区别**：

  - $O(N)$ 是把所有人点一次名。
  - $O(\log N)$ 是在一群人里用二分法找一个人。
  - $O(N \log N)$ 是**对每个人（$N$）都做一次二分查找（$\log N$）**。

- **经典场景**：

  - **高效排序**：归并排序（Merge Sort）、快速排序（Quick Sort）、堆排序（Heap Sort）。
  - **分治法**：把一个大问题拆成两半（$\log N$层），每一层都要把所有数据过一遍（$N$）。

- **代码长相**：

  C++

  ```c++
  // 外层循环 N 次
  for (int i = 0; i < N; i++) {
      // 内层是一个二分查找，或者内部变量每次乘2
      int j = 1;
      while (j < N) {
          j = j * 2;
      }
  }
  ```

#### 4. 怎么判断是 $O(N^2)$？（关键词：双重循环、两两比较）

这个最好判断。通常是**两层嵌套的循环**，且每一层都跟 $N$ 有关。

- **特征**：外层跑 $N$ 圈，内层也跑 $N$ 圈。

- **经典场景**：冒泡排序、选择排序、两个数组的两两匹配。

- **代码长相**：

  C++

  ```c++
  for (int i = 0; i < N; i++) {
      for (int j = 0; j < N; j++) {
          // ...
      }
  }
  ```

#### 5. 怎么判断是 $O(2^N)$？（关键词：递归裂变）

这种复杂度通常出现在**递归**中，且每次递归会**分裂**成两个或更多新的递归调用。

- **特征**：像细胞分裂一样，1变2，2变4，4变8。

- **经典场景**：求斐波那契数列的暴力递归解法、汉诺塔问题。

- **代码长相**：

  C++

  ```c++
  int fib(int n) {
      if (n <= 1) return n;
      return fib(n-1) + fib(n-2); // 一次调用变成了两次
  }
  ```

------

### 三、 总结：一眼定乾坤的口诀

1. **没有循环**，或者循环次数固定（如100次） $\rightarrow$ **$O(1)$**
2. **一个循环**，变量每次`+1` $\rightarrow$ **$O(N)$**
3. **一个循环**，变量每次`×2` 或 `/2` $\rightarrow$ **$O(\log N)$**
4. **两个循环嵌套**，都是`+1` $\rightarrow$ **$O(N^2)$**
5. **一个循环（N）嵌套一个折半循环（log N）** $\rightarrow$ **$O(N \log N)$**
6. **递归**，每次函数里调用自己**2次** $\rightarrow$ **$O(2^N)$**

掌握这几个核心模式，你就能分析出绝大多数算法的时间复杂度了。
