计算时间复杂度并不是真的去拿着秒表测几秒，也不是死板地数每一行代码到底运行了 3 次还是 4 次。

核心在于：**找规律**。我们要计算的是**随着输入规模 $N$ 的增大，代码执行次数的增长趋势**。

下面是具体的计算步骤和原则：

### 1. 基础单位：什么是“1次”？

在计算机看来，所有**简单的、不依赖输入规模的操作**，都算作 **$O(1)$**（常数级）。

- **赋值**：`int a = 1;`
- **运算**：`a + b`, `a * 100`, `i++`
- **比较**：`i < n`, `if (a == b)`
- **数组访问**：`arr[5]`
- **函数返回**：`return`

**注意**：无论这些操作写了多少行，只要它们跟 $N$ 没关系（比如你写了 1000 行 `print("hello")`），它们的复杂度统统是 $O(1)$。

------

### 2. 常见结构的计算方法

#### A. 顺序结构 (加法法则)

代码从上往下执行，把各个部分的复杂度加起来。

C++

```C++
x = 1;        // 执行 1 次
y = 2;        // 执行 1 次
z = x + y;    // 执行 1 次
```

总次数 = 3。

**复杂度**：$O(1)$。

#### B. 循环结构 (乘法法则) —— 最常见的考点

循环的复杂度 = **循环体的复杂度 $\times$ 循环次数**。

**案例 1：单层循环**

C++

```C++
for (int i = 0; i < n; i++) {  // 循环 n 次
    printf("%d", i);           // 循环体是 O(1)
}
```

总次数 $\approx n$。

**复杂度**：$O(n)$。

**案例 2：双层嵌套循环**

C++

```C++
for (int i = 0; i < n; i++) {      // 外层 n 次
    for (int j = 0; j < n; j++) {  // 内层 n 次
        printf("%d", i + j);       // O(1)
    }
}
```

总次数 $= n \times n = n^2$。

**复杂度**：$O(n^2)$。

**案例 3：对数循环 (Logarithmic)**

如果是“每次乘 2”或者“每次除 2”，而不是“每次加 1”，那么它就是对数级。

C++

```c++
int i = 1;
while (i < n) {
    i = i * 2;  // i 的变化：1, 2, 4, 8, 16...
}
```

问：$2$ 的多少次方等于 $n$？答案是 $\log_2 n$。

**复杂度**：$O(\log n)$。

------

### 3. 如何简化？（大O的精髓）

当你数完次数得到一个公式（比如 $3n^2 + 5n + 100$）时，我们不需要保留这么详细的信息，只需要保留**最高阶项**，并**去掉系数**。

**原则：**

1. **去常数**：$O(2n)$ $\rightarrow$ $O(n)$。因为当 $N$ 是一亿时，2 倍还是 1 倍区别不大了，都是线性增长。
2. **去低阶项**：$O(n^2 + n)$ $\rightarrow$ $O(n^2)$。因为当 $N$ 很大时，$N^2$ 起决定性作用，$N$ 相比之下微不足道。

**练习：**

- 计算结果：$100$ 次 $\rightarrow$ **$O(1)$**
- 计算结果：$3n + 5$ 次 $\rightarrow$ **$O(n)$**
- 计算结果：$2n^2 + 100n$ 次 $\rightarrow$ **$O(n^2)$**

------

### 4. 实战演示：一行行数（不用太精确）

假设有这样一段代码：

C++

```c++
void func(int n) {
    int a = 0;                         // 执行 1 次
    
    for (int i = 0; i < n; i++) {      // 判定 n+1 次，自增 n 次 (近似看作 n)
        a = a + 1;                     // 执行 n 次
        
        for (int j = 0; j < n; j++) {  // 内层循环整体跑了 n * n 次
            printf("hello");           // 执行 n * n 次
        }
    }
}
```

**精确计算**：

$1 + (n+1) + n + n + n(n+1) + n^2 \approx 2n^2 + 3n + 2$

（注意：为了简单，我们通常直接看最深的那层循环）

**大O计算（考试/面试用法）**：

1. 直接找**嵌套最深**的那个语句。
2. 这里是 `printf("hello")`。
3. 它被包了两层 $0$ 到 $n$ 的循环。
4. 所以是 $n \times n$。
5. **复杂度：$O(n^2)$。**

### 总结口诀

1. **单条语句**看作 1。
2. **一层循环**就是 $n$。
3. **两层循环**是 $n^2$。
4. **折半查找**是 $\log n$。
5. **最后忽略系数和常数**。
