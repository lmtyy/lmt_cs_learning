# 方法论：如何设计一个数据结构 (How to Design a Data Structure)

设计数据结构不是凭空创造，而是为了解决**特定场景**下的**效率瓶颈**。通常遵循以下 5 个标准步骤：

## 第一步：需求分析 (Requirement Analysis)

**核心问题：** 这个数据结构是用来解决什么痛点的？

- **读写频率：** 是读多写少（如字典、配置），还是写多读少（如日志流）？
- **数据特性：** 数据是按顺序来的吗？有重复吗？需要排序吗？数据量大概是多少？
- **关键痛点：** 现有的结构哪里不够好？
  - *例如：* 普通数组的插入太慢 ($O(N)$)，我需要 $O(1)$ 的插入。
  - *例如：* 普通链表的查找太慢 ($O(N)$)，我需要 $O(\log N)$ 的查找。

## 第二步：定义接口 (Define Operations / API)

**核心问题：** 用户怎么使用它？

先定义“黑盒”外部的按钮，再考虑内部实现。

- **基础操作 (CRUD)：**
  - `insert(data)`: 增
  - `delete(key)`: 删
  - `update(key, new_val)`: 改
  - `search(key)`: 查
- **特殊操作：**
  - `getMax()` / `getMin()`: 获取极值
  - `successor()`: 获取后继节点

## 第三步：选择底层存储模型 (Choose Underlying Storage)

**核心问题：** 数据在内存里到底怎么摆放？

所有高级数据结构，底层基本只有两种选择，或者是它们的组合：

1. **连续存储 (Contiguous Allocation) —— 数组 (Array)**
   - **优点**：省内存（无指针开销）、缓存友好 (Cache Friendly)、支持随机访问。
   - **缺点**：大小固定（扩容成本高）、中间插入/删除慢。
2. **链式存储 (Linked Allocation) —— 链表 (Linked List)**
   - **优点**：动态扩容方便、插入/删除快。
   - **缺点**：费内存（存指针）、缓存不友好、只能顺序访问。

## 第四步：设计内部算法与维护逻辑 (Internal Logic)

**核心问题：** 为了保证效率，每次操作内部要怎么变？

- **维护有序性：** 插入新数据时，是直接扔在最后（$O(1)$），还是找位置插进去保持有序（$O(N)$）？
- **辅助结构 (Auxiliary Structures)：** 是否需要额外的索引或元数据来加速？
  - *例如：* 跳表 (SkipList) 在链表之上增加了多级索引。
  - *例如：* 最小栈 (MinStack) 增加了一个辅助栈来存最小值。

## 第五步：复杂度分析与权衡 (Trade-off)

**核心问题：** 真的变快了吗？代价是什么？

- **没有任何数据结构是完美的。设计就是权衡 (Trade-off)。**
- **空间换时间 (Space for Time)：** 多费点内存存索引，换取查找速度（如：哈希表、线段树）。
- **时间换空间 (Time for Space)：** 数据压缩存储，用的时候再解压（如：BitMap）。

# 实战案例：设计“最小栈” (MinStack)

**场景需求**：我们需要一个栈，但需要增加一个功能 `getMin()`，能瞬间 ($O(1)$) 得到当前栈里的最小值。

### 1. 逻辑设计：双栈法 (Auxiliary Stack Strategy)

如果只用一个栈，弹出最小值后，我就不知道“第二小”的是谁了。

- **Data Stack (主栈)**：存所有数据。
- **Min Stack (辅栈)**：存“当前的最小值序列”。

### 2. 代码实现 (C++)

C++

```c++
#include <stack>
#include <stdexcept>
using namespace std;

class MinStack {
private:
    stack<int> s_data; // 主数据栈
    stack<int> s_min;  // 辅助栈：存当前最小值

public:
    MinStack() {}

    // 入栈：O(1)
    void push(int val) {
        s_data.push(val);
        // 如果新来的值比当前最小值还小（或相等），就入辅助栈
        if (s_min.empty() || val <= s_min.top()) {
            s_min.push(val);
        }
    }

    // 出栈：O(1)
    void pop() {
        if (s_data.empty()) return;
        int val = s_data.top();
        s_data.pop();
        // 如果弹出的正是最小值，辅助栈也要同步弹出
        if (val == s_min.top()) {
            s_min.pop();
        }
    }

    // 获取栈顶：O(1)
    int top() {
        if (s_data.empty()) throw runtime_error("Stack is empty");
        return s_data.top();
    }

    // 获取最小值：O(1)
    int getMin() {
        if (s_min.empty()) throw runtime_error("Stack is empty");
        return s_min.top(); // 直接看辅助栈顶
    }
};
```

### 3. 最终评价

- **时间复杂度**：所有操作均为 $O(1)$。
- **空间复杂度**：$O(N)$（最坏情况辅栈和主栈一样大）。
- **结论**：通过消耗一倍的空间，换取了极致的时间效率（典型的**空间换时间**策略）。
