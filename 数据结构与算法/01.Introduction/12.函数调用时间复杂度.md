这是一个非常好的问题！很多初学者最容易在这里栽跟头：**把一行函数调用当成“1次”操作**。

**结论是：函数调用本身算 $O(1)$，但必须加上函数“肚子”里执行的所有代码。**

你可以把函数调用想象成一个**“压缩包”**。计算复杂度时，你必须把这个包“解压”开来看。

以下是三种常见情况的计算方法：

### 1. 情况一：普通函数（里面没有循环）

如果函数内部只是简单的加减乘除、赋值或判断，那么它就是 **$O(1)$**。

C++

```C++
// 这个函数是 O(1)
void printHello(int n) {
    printf("Hello %d", n); // 1次
}

// 主程序
for (int i = 0; i < n; i++) {
    printHello(i); // 这里算作 1 次
}
```

**总复杂度**：$n \times 1 = O(n)$。

- **解释**：这种情况下，你把函数调用当成普通的一行语句（比如 `a++`）算就行了。

------

### 2. 情况二：陷阱函数（里面藏着循环）

这是考试和面试中最坑的地方。虽然 `heavyFunc()` 只写了一行，但它内部可能跑了 $n$ 次。

C++

```C++
// 这个函数内部是 O(n)
void heavyFunc(int n) {
    for (int k = 0; k < n; k++) { // 跑 n 次
        printf("*");
    }
}

// 主程序
for (int i = 0; i < n; i++) { // 外层跑 n 次
    heavyFunc(n);             // 注意！这一行不是 1，而是 n！
}
```

**错误算法**：外层 $n$ $\times$ 内层 $1$ (误以为函数调用是1) = $O(n)$ ❌

**正确算法**：外层 $n$ $\times$ 内层 $n$ (把函数展开看) = $O(n^2)$ ✅

**核心原则**：遇到函数调用，**不要看它名字多短，要看它定义里写了什么**。如果是 $O(n)$ 的函数放在 $O(n)$ 的循环里，那就是嵌套循环，得乘起来。

------

### 3. 情况三：递归函数（自己调用自己）

递归函数不能简单地“数行数”。你需要画图（递归树）或者用公式算。

**公式：总次数 = 递归深度 $\times$ 每次递归内部的操作次数**

**例子 1：简单的递归（比如阶乘）**

C++

```C++
void recursive(int n) {
    if (n <= 0) return;
    print(n);         // 内部操作是 O(1)
    recursive(n - 1); // 递归深度是 n (从 n 减到 0)
}
```

**复杂度**：深度 $n \times 1 = O(n)$。

**例子 2：斐波那契数列（二叉树形递归）**

C++

```C++
int fib(int n) {
    if (n <= 1) return n;
    return fib(n-1) + fib(n-2); // 每次分裂成 2 个分支
}
```

这就像细胞分裂：1变2，2变4，4变8...

**复杂度**：$O(2^n)$ （指数级爆炸）。

### 总结

计算函数调用时，请遵循**“代入法”**：

1. 先算出那个函数单独运行的复杂度是多少（比如是 $T_{sub}$）。
2. 回到调用它的地方，把那行 `func()` 代码替换成 $T_{sub}$。
3. 再结合外面的循环进行加法或乘法运算。

**一句话口诀：函数不是黑盒子，拆开看看再下笔。**
