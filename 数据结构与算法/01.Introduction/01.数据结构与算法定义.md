# 数据结构与算法 (Data Structure & Algorithm)

## 一、 核心概念：程序 = 数据结构 + 算法

如果把编程比作**烹饪**：

- **数据结构**：是**食材的组织方式**。你是把土豆切丝、切块，还是磨成泥？是堆在篮子里，还是整齐码在架子上？
- **算法**：是**菜谱**。是先炒肉还是先放菜？火候多大？每一步的具体逻辑是什么？

------

## 二、 数据结构 (Data Structure) —— 静态的组织

数据结构研究的是数据在计算机内存中是如何“排位”的。

### 1. 线性结构 (Linear Structure)

数据元素之间是一对一的关系。

- **数组 (Array)**：一块连续的内存。
  - *特点*：查询快（有索引），增删慢（要挪动位置）。
- **链表 (Linked List)**：像寻宝游戏，每个节点存储数据和指向下一个点的指针。
  - *特点*：增删快，查询慢（得从头找）。
- **栈 (Stack)**：先进后出 (LIFO)。
  - *应用*：函数的调用栈、浏览器的后退功能。
- **队列 (Queue)**：先进先出 (FIFO)。
  - *应用*：排队系统、任务调度。

### 2. 非线性结构 (Non-Linear Structure)

数据元素之间是一对多或多对多的关系。

- **树 (Tree)**：分层结构。
  - *典型*：二叉搜索树、红黑树（用于数据库索引、文件系统）。
- **图 (Graph)**：复杂的网状关系。
  - *应用*：社交网络、地图导航、推荐算法。

------

## 三、 算法 (Algorithm) —— 动态的逻辑

算法是解决问题的具体步骤。衡量算法好坏有两个核心维度：**复杂度分析**。

### 1. 时间复杂度 ($O$)

衡量随着数据量 $n$ 的增大，运行时间增长的趋势。

- $O(1)$：常数级（最快）。
- $O(\log n)$：对数级（如：二分查找）。
- $O(n)$：线性级（如：简单遍历）。
- $O(n^2)$：平方级（如：冒泡排序）。

### 2. 空间复杂度

解决问题需要额外占用多少内存。

## 四、 为什么要一起学？

| **维度** | **数据结构**            | **算法**               |
| -------- | ----------------------- | ---------------------- |
| **角色** | 存储载体 (Data Carrier) | 处理逻辑 (Logic)       |
| **状态** | 静态的 (Static)         | 动态的 (Dynamic)       |
| **关系** | 算法建立在数据结构之上  | 数据结构为算法提供支撑 |

**总结：** 只有选择了最合适的数据结构（货架设计），配合最优的算法（搬运路线），程序才能运行得既快又省资源。
