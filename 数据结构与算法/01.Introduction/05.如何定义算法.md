# 什么是算法 (What is an Algorithm)

## 一、 定义 (Definition)

**算法 (Algorithm)** 是解决特定问题的一系列**清晰、有限**的指令步骤。

简单来说，算法就是计算机解题的“**操作手册**”或“**菜谱**”。

- **输入 (Input)**：这就是“食材”。
- **算法 (Algorithm)**：这就是“菜谱”和“烹饪过程”。
- **输出 (Output)**：这就是做好的“菜”。

> **公式**：算法 + 数据结构 = 程序

## 二、 算法的五大特性 (The 5 Key Characteristics)

一个合格的算法**必须**同时满足以下 5 个特征（考试/面试常考点）：

1. **有穷性 (Finiteness)**
   - 算法必须在执行有限步之后结束，不能无限循环（死循环不是算法）。
2. **确定性 (Definiteness)**
   - 每一步指令必须清晰明确，不能有歧义。
   - *反例*：“加少许盐”是不确定的；“加 5 克盐”是确定的。
3. **可行性 (Effectiveness)**
   - 每一步都是计算机能通过有限次基本运算完成的。
4. **输入 (Input)**
   - 算法有 **0 个**或**多个**输入（算法可以没有输入，比如直接输出 "Hello World"）。
5. **输出 (Output)**
   - 算法至少要有 **1 个**输出（没有输出的算法是没有意义的）。

## 三、 算法 vs 程序 (Algorithm vs. Program)

虽然两者常混用，但在计算机科学中通过以下点区分：

| **维度**   | **算法 (Algorithm)**     | **程序 (Program)**                          |
| ---------- | ------------------------ | ------------------------------------------- |
| **形态**   | 解决问题的逻辑思维       | 算法的代码实现                              |
| **语言**   | 伪代码、自然语言、流程图 | C++, Java, Python 等                        |
| **终止性** | **必须停止**             | **可以不停止** (如操作系统、服务器守护进程) |

## 四、 算法的表示方法

1. **自然语言**：用人类语言描述（容易有歧义，不推荐）。
2. **流程图 (Flowchart)**：使用图形符号表示步骤（如菱形代表判断，矩形代表操作）。
3. **伪代码 (Pseudocode)**：介于自然语言和编程语言之间，只关注逻辑，不关注语法细节（通用性最强）。
4. **编程语言**：具体的代码实现（如 C++）。

## 五、 举个例子：高斯求和

**问题**：计算 $1 + 2 + 3 + ... + 100$

- **笨算法 (累加法)**：

  1. 设置 sum = 0, i = 1
  2. sum = sum + i
  3. i 增加 1
  4. 如果 i <= 100，回到步骤 2；否则输出 sum。

  - *评价*：需要算 100 次，**$O(N)$**。

- **好算法 (高斯公式)**：

  1. 直接计算 $(1 + 100) \times 100 / 2$
  2. 输出结果。

  - *评价*：只需要算 1 次，**$O(1)$**。

------

**总结**：设计算法的终极目标，就是找到**步骤最少、占用内存最小**的那个“菜谱”。
