# 最优BST算法代码与复杂度（p111-116）

本节给出最优 BST 的完整算法代码（OptimalBinarySearchTree），以及构造最优解和复杂度分析。

---

## 第111页：具体求解过程

### 填表顺序

**1）递归出口**：没有内部节点时，构造空树

T\[1]\[0], T\[2]\[1], T\[3]\[2], ..., T\[n+1]\[n]

**2）按子树大小 r（起止下标的差）递增**，构造具有 1 个、2 个、...、n 个内部结点的最优二叉搜索树：

| r（大小-1） | 计算的子树 |
|------|---------|
| 0 | T\[1]\[1], T\[2]\[2], ..., T\[n]\[n] |
| 1 | T\[1]\[2], T\[2]\[3], ..., T\[n-1]\[n] |
| 2 | T\[1]\[3], T\[2]\[4], ..., T\[n-2]\[n] |
| ... | ... |
| r | T\[1]\[r+1], T\[2]\[r+2], ..., T\[i]\[i+r], ..., T\[n-r]\[n] |
| n-1 | T\[1]\[n]（最终答案） |

## 第112页：OBST 框架代码

```c
void OBST(int *a, int *b, int n, int **m, int **s, int **w)
{
    // 初始化：构造没有内部节点时的情况
    for (int i = 0; i <= n; i++)
    {
        w[i+1][i] = a[i];    // 空树的权重 = qᵢ
        m[i+1][i] = 0;       // 空树的代价 = 0
    }
    
    // 按子树大小递增填表
    for (int r = 0; r < n; r++)
        for (int i = 1; i <= n - r; i++)
        {
            int j = i + r;
            构造T[i][j]，填写w[i][j], m[i][j], s[i][j]
        }
}
```

### 代码对照

| 行 | 作用 | 对应递推 |
|---|------|---------|
| `w[i+1][i] = a[i]` | 空树权重初始化 | w_{i,i-1} = q_{i-1} |
| `m[i+1][i] = 0` | 空树代价初始化 | m(i,i-1) = 0 |
| `for r` | 按大小递增 | 保证依赖的子问题已算好 |
| `j = i + r` | 终止下标 | T\[i]\[j] 含 r+1 个内部结点 |

## 第113页：构造 T\[i]\[j] 的过程

T\[i]\[j] 表示用第 i 到第 j 个内部节点构造的树，做根的结点可以是第 i, i+1, ..., j 中任意一个。

### 步骤

**1）首选 i 作为根**，其左子树空，右子树为结点 i+1, i+2, ..., j 构成即 T\[i+1]\[j]：

```c
m[i][j] = w[i][j] + 0 + m[i+1][j];    // m(i,i-1)=0
s[i][j] = i;
```

**2）不选 i 做根**，设 k 为其根，则 k = i+1, ..., j。左子树为结点 i, i+1, ..., k-1，右子树为 k+1, k+2, ..., j：

```c
t = w[i][j] + m[i][k-1] + m[k+1][j];
if (t < m[i][j])
{
    m[i][j] = t;
    s[i][j] = k;
}
```

**3）k = k+1，跳回 2**，直到 k > j 为止。

## 第114页：OptimalBinarySearchTree 完整代码

```c
void OptimalBinarySearchTree(int *a, int *b, int n, int **m, int **s, int **w)
{
    // 初始化对角线赋值
    for (int i = 0; i <= n; i++) {
        w[i+1][i] = a[i];
        m[i+1][i] = 0;
    }
    
    for (int r = 0; r < n; r++)
        for (int i = 1; i <= n - r; i++) {
            int j = i + r;
            
            // 加第 j 个结点后，权值 w 改变
            w[i][j] = w[i][j-1] + a[j] + b[j];
            
            // 如第 i 个结点作根的值（初始假设）
            m[i][j] = m[i+1][j];
            s[i][j] = i;
            
            // 取第 k 个结点作根（k从i+1到j）
            for (int k = i + 1; k <= j; k++) {
                int t = m[i][k-1] + m[k+1][j];
                if (t < m[i][j]) {
                    m[i][j] = t;
                    s[i][j] = k;
                }
            }
            
            // 加上 wᵢⱼ（提到循环外面，因为与k无关）
            m[i][j] += w[i][j];
        }
}
```

### 代码逐行对照递推公式

$$
    m(i, j) = w_{ij} + \min_{i \le k \le j}\{m(i, k-1) + m(k+1, j)\}
$$

| 代码行 | 对应公式部分 | 说明 |
|--------|-----------|------|
| `w[i][j] = w[i][j-1] + a[j] + b[j]` | wᵢⱼ 递推计算 | 利用 wᵢⱼ = w_{i,j-1} + aⱼ + bⱼ |
| `m[i][j] = m[i+1][j]` | k=i 时的初始值 | m(i,i-1) + m(i+1,j) = 0 + m(i+1,j) |
| `s[i][j] = i` | 初始假设根为 i | |
| `t = m[i][k-1] + m[k+1][j]` | 枚举 k 时的代价 | 左子树 + 右子树 |
| `if (t < m[i][j])` | 取 min | 找更优的根 |
| `m[i][j] += w[i][j]` | 加上 wᵢⱼ | wᵢⱼ 与 k 无关，最后统一加 |

> 🔑 代码技巧：先在内层循环中只比较 m\[i]\[k-1]+m\[k+1]\[j]（不含 wᵢⱼ），最后统一加 w\[i]\[j]。因为 wᵢⱼ 对所有 k 相同，不影响 min 的结果。

## 第115页：6. 构造最优解

### 利用 s 表回溯

算法 OptimalBinarySearchTree 中用 **s\[i]\[j]** 保存最优子树的根顶点中元素。

- 当 s\[1]\[n] = k 时，xₖ 为所求二叉搜索树的根顶点元素
- 其左子树为 T(1, k-1)，因此 l = s\[1]\[k-1] 即表示 T(1, k-1) 的根顶点为 xₗ
- 依次类推，容易由 s 记录的信息在 **O(n)** 时间内构造出所求的最优二叉搜索树

> 与矩阵连乘、0-1 背包的回溯方式完全类似：递归查 s 表即可还原整棵树。

## 第116页：7. 计算复杂性

### 空间复杂度

算法中用到三个数组 m、s、w，故所需的空间为 **O(n²)**。

### 时间复杂度

算法的主要计算量在于计算：

$$
    w_{ij} + \min_{i \le k \le j}\{m(i, k-1) + m(k+1, j)\}
$$

对于固定的差 r = j - i，需要计算时间 O(j - i + 1) = O(r + 1)。

因此算法所耗费的总时间为：

$$
    \sum_{r=0}^{n-1} \sum_{i=1}^{n-r} O(r+1) = O(n^3)
$$

### 复杂度汇总

| 项目 | 复杂度 |
|------|--------|
| 时间 | **O(n³)** |
| 空间 | **O(n²)** |
| 构造最优解 | O(n) |

### 与其他 DP 问题对比

| 问题 | 时间 | 空间 | 递推结构 |
|------|------|------|---------|
| 矩阵连乘 | O(n³) | O(n²) | 区间 DP，枚举断点 |
| 最优BST | O(n³) | O(n²) | 区间 DP，枚举根 |
| 0-1背包 | O(nc) | O(nc) | 线性 DP，枚举物品 |

> ✅ 矩阵连乘和最优 BST 都是"区间 DP"模型——子问题是连续区间 \[i,j]，外层按区间长度递增，内层枚举分割点。结构非常相似。

## 总结

```
最优 BST 算法完整流程：

OptimalBinarySearchTree:
  初始化: w[i+1][i]=a[i], m[i+1][i]=0
  外层循环: r = 0 → n-1（子树大小）
    中层循环: i = 1 → n-r（起始位置）
      j = i + r（终止位置）
      w[i][j] = w[i][j-1] + a[j] + b[j]（递推计算权重）
      内层循环: k = i → j（枚举根）
        取 min{m[i][k-1] + m[k+1][j]}
      m[i][j] += w[i][j]（加上权重）

构造最优解:
  s[1][n] = k → 根为 xₖ
  递归查 s 表构造左右子树

复杂度: O(n³) 时间, O(n²) 空间

注: Knuth 优化可将时间降为 O(n²)
  利用 s[i][j-1] ≤ s[i][j] ≤ s[i+1][j] 缩小 k 的搜索范围
```
