# 最优BST问题描述与最优子结构（p093-097）

本节正式描述最优 BST 问题，并证明其最优子结构性质：最优 BST 的左右子树也必须是各自关键字集合的最优 BST。

---

## 第93页：3. 最优二叉搜索树问题描述

### 问题定义

对于有序集 S 及其存取概率分布 (a₀, b₁, a₁, ..., bₙ, aₙ)，在所有表示有序集 S 的二叉搜索树中找出一棵具有**最小平均路长**的二叉搜索树。

### 直觉原则

结点在二叉搜索树中的层次越深，需要比较的次数就越多，因此要构造一棵最小二叉树，一般尽量把**搜索概率较高的结点放在较高的层次**。

> ⚠️ 但不能简单地贪心——仅把概率最大的放在根未必全局最优，因为 BST 的结构约束（左<根<右）限制了结点的位置。需要 DP 全局优化。 如1：0.35，2：0.25，3：0.25，4：0.15，贪心算法会把1当根，左结点为外叶子，树倾斜，而DP结果可能会选2当根，左右结点都是内节点，树均衡

---

## 第94页：4. 最优子结构性质

### 选择根结点 k 后的划分

假设选择 k 为树根，则：
- **左子树 L**：包含内部结点 1, 2, ..., k-1 和外部结点 a₀, a₁, ..., a_{k-1}
- **右子树 R**：包含内部结点 k+1, ..., n 和外部结点 aₖ, a_{k+1}, ..., aₙ

```
         (k)
        /   \
      [L]   [R]
   1,...,k-1  k+1,...,n
   a₀,...,a_{k-1}  aₖ,...,aₙ
```

> 🔑 选定根 k 后，左右子树的关键字集合就完全确定了（由 BST 的有序性决定）。

---

## 第95页：最优子结构图示（整棵树）

### 示例

```
              51
             /  \
          14      72
         /  \    /  \
       06   33  53   97
       /\  /\   /\   /\
      13 25 43 64 84 99
```

- 根 = 51
- 左子树（粉色区域）：{06, 13, 14, 25, 33, 43}
- 右子树（蓝色区域）：{53, 64, 72, 84, 97, 99}

---

## 第96页：子树继续递归分解

右子树以 72 为根，进一步分解：
- 72 的左子树（粉色）：{53, 64}
- 72 的右子树（蓝色）：{84, 97, 99}

> 最优子结构递归地应用在每一层：如果整棵树是最优的，那么每棵子树也必须是各自关键字集合上的最优 BST。

## 第97页：最优子结构性质的表述

### 代价分解

设 **COST(L)** 和 **COST(R)** 分别是二分检索树 T 的左子树和右子树的成本。

则检索树 T 的成本是：

$$
    \text{COST}(T) = P(k) + \text{COST}(L) + \text{COST}(R) + \ldots
$$

（省略号部分是因为左右子树嵌入到更深一层后代价增加的部分，下一节递推时会精确给出）

### 最优子结构

> ✅ 若 T 是最优的，则上式及 **COST(L)** 和 **COST(R)** 必定都取最小值。

即：
- 左子树 L 必须是关键字 {1, ..., k-1} 在概率 (a₀, b₁, a₁, ..., b_{k-1}, a_{k-1}) 下的最优 BST
- 右子树 R 必须是关键字 {k+1, ..., n} 在概率 (aₖ, b_{k+1}, a_{k+1}, ..., bₙ, aₙ) 下的最优 BST

### 反证法（与之前类似）

如果左子树不是最优的，换成更优的左子树后整棵树代价更小 → 与 T 最优矛盾。

> 类比：和矩阵连乘一样——最优解在某个 k 处"断开"（选根），左右两部分也必须各自最优。

---

## 总结

```
最优 BST 的最优子结构：

选择根 k → 树被分为左右两棵子树
  左子树：关键字 1,...,k-1 + 外部结点 a₀,...,a_{k-1}
  右子树：关键字 k+1,...,n + 外部结点 aₖ,...,aₙ

最优子结构性质：
  最优BST的左子树 = 左边关键字集合的最优BST
  最优BST的右子树 = 右边关键字集合的最优BST

         ┌───────────────────────┐
         │ COST(T) = P(k)       │
         │         + COST(L)    │
         │         + COST(R)    │
         │         + (深度增加项) │
         └───────────────────────┘
              ↓           ↓
         L必须最优    R必须最优

→ 可以用 DP 枚举所有可能的根 k，取代价最小的
```
