# 矩阵连乘DP建模与递归分析（p012-018）

本节展示动态规划解矩阵连乘问题的前两步：分析最优子结构、建立递归关系，并通过递归实现说明朴素递归的指数级复杂度，引出DP填表的必要性。

---

## 第12页：步骤1 — 分析最优解的结构

### 符号约定

设 **A[i:j]** 为矩阵连乘积 AᵢAᵢ₊₁···Aⱼ

### 最优划分

计算 A[1:n] 的最优次序，设该次序在矩阵 Aₖ 和 Aₖ₊₁ 之间断开（1 ≤ k < n），则完全加括号方式为：

$$((A_1 \cdots A_k)(A_{k+1} \cdots A_n))$$

### 总计算量分解

总计算量 = A[1:k] 的计算量 + A[k+1:n] 的计算量 + 两个结果矩阵相乘的计算量

> 🔑 这个分解结构是建立递推关系的基础。

---

## 第13页：最优子结构特征

### 核心定理

计算 A[1:n] 的一个最优次序所包含的计算矩阵子链 **A[1:k]** 和 **A[k+1:n]** 的次序**也是最优的**。

> ✅ 矩阵连乘积计算次序问题的最优解包含着其子问题的最优解，即具有**最优子结构性质**（Optimal Substructure）。

### 两个结果矩阵相乘的代价

- A[1:k] 的结果为 r₁ × rₖ 的矩阵
- A[k+1:n] 的结果为 rₖ₊₁ × rₙ 的矩阵（实际上就是 rₖ × rₙ，因为相邻矩阵维度匹配）
- 它们相乘的计算量为 **p₁ × pₖ × pₙ**（即 r₁ × rₖ × rₙ）

> 类比：最优子结构就像"最短路径的子路径也是最短的"——如果北京到广州的最短路经过武汉，那北京到武汉这段也必须是最短的。

## 第14页：步骤2 — 建立递归关系

### 定义

设计算 A\[i:j] 所需的**最少乘法次数**为 ** m\[i]\[j]** ，原问题的最优值为**m\[1][n]**。

### 基准情况

当 i = j 时：A[i:j] = Aᵢ（单个矩阵，无需计算）

$$
m[i][i] = 0, \quad i = 1, 2, \ldots, n
$$

### 递推关系（i < j）

$$
m[i][j] = m[i][k] + m[k+1][j] + p_{i-1} \cdot p_k \cdot p_j
$$

其中 k ∈ {i, i+1, ..., j-1}  A\[i][k]代表第i到第k个矩阵相乘，根据连乘法则，最终矩阵的行数为第一个矩阵行数(p\_{i-1})，列数为最后一个矩阵列数(p\_{k})即A\[i]\[k]为p\_{i-1}行p\_{k}列的矩阵，同理A\[k+1][j]为k行j列矩阵。计算这两矩阵相乘需要p\_{i-1}\*p\_{k}\*p\_{j}

### 最优递推公式

$$
m[i][j] = \begin{cases} 0 & i = j \\ \displaystyle\min_{i \le k < j} \{m[i][k] + m[k+1][j] + p_{i-1} p_k p_j\} & i < j \end{cases}
$$

> ✅ 这是矩阵连乘问题的**核心递推方程**，考试必须会写。 p1既是A0的列数也是A1的行数

| 符号 | 含义 |
|------|------|
| m[i][j] | A[i:j] 的最优（最少）乘法次数 |
| p[] | 维度数组，Aᵢ 的维度为 pᵢ₋₁ × pᵢ |
| k | 断开位置，枚举所有可能取最小值 |

## 第15页：采用递归方法计算

### RecurMatrixChain 代码

```c
int RecurMatrixChain(int i, int j, int p[])
{
    if (i == j) return 0;// 一个矩阵连乘不需要任何计算
    // 取第一个断开位置(k=i)时的计算量作为初始值
    int u = RecurMatrixChain(i, i, p) + RecurMatrixChain(i+1, j, p)// 原图漏传p
            + p[i-1] * p[i] * p[j];
    s[i][j] = i;  // 记录i到j-1矩阵连乘，最优分割点是s[i][j],初始化为在第一个矩阵后分隔
    
    // 循环枚举 k 的所有可取断开位置
    for (int k = i+1; k < j; k++)
    {
        int t = RecurMatrixChain(i, k, p) + RecurMatrixChain(k+1, j, p)
                + p[i-1] * p[k] * p[j];
        if (t < u)
        {
            u = t;
            s[i][j] = k;  // 更新最优断开位置
        }
    }
    return u;
}
```

**关键变量说明**：
| 变量 | 作用 |
|------|------|
| i, j | 矩阵链的起始和终止位置 |
| p[] | 维度数组 |
| u | 当前找到的最小计算量 |
| s[i][j] | 记录 A[i:j] 的最优断开位置 k |

> 问题：这个递归有大量重复计算！

## 第16页：递归树（n=4 的情况）

```
                        [1......4]
           /        /        |        \        \        \
      [1..1]    [2..4]    [1..2]    [3..4]    [1..3]    [4..4]
               / | \      / \       / \       / | \
          [2..2][3..4][2..3][4..4] [3..3][4..4] [1..1][2..3][1..2][3..3]
                ...    ...                        ...    ...
```

> 🔑 同一子问题被反复计算（如 [2..3]、[3..4] 出现多次），这正是**重叠子问题**的体现。

> 递归树展开后节点数为指数级，与 Fibonacci 的递归树问题如出一辙。

## 第17页：递归算法的时间复杂度分析

### T(n) 的递归不等式

T(n)表示n个矩阵连乘的时间复杂度

设判断语句和赋值语句花费常数时间（下称必要花销），则：
$$
T(n) \ge \begin{cases} O(1) & n = 1 \\ \displaystyle 1 + \sum_{k=1}^{n-1}(T(k) + T(n-k) + 1) & n > 1 \end{cases}
$$

其中n=1代表1个矩阵相乘，不需要计算。只需要必要花销即可

先说Σ，这表示遍历所有分界点k的可能，每一个分界点k都代表吧原本的n个矩阵连乘转换为前k个矩阵连乘T(k)和后n-k个矩阵连乘T(n-k)，同时加上诸如计算p[i-1] * p[k] * p[j]等的必要花销。最前面的1代表代码中初始化k=i即分界点默认为1的花销。k的下标从1开始取，意味着哪怕兜兜转转最优解就是初始假设的情况（k=i）也需要进一遍Σ

当 n > 1 时：

$$T(n) \ge 1 + (n-1) + \sum_{k=1}^{n-1}T(k) + \sum_{k=1}^{n-1}T(n-k) = n + 2\sum_{k=1}^{n-1}T(k)$$

### 结论

用数学归纳法可证明：

$$T(n) \ge 2^{n-1} = \Omega(2^n)$$

> ✅ 朴素递归的时间复杂度为**指数级** Ω(2ⁿ)，与穷举一样不可接受。

## 第18页：子问题数量分析 → DP 的可行性

### 不同子问题的个数

对于 1 ≤ i ≤ j ≤ n，不同的有序对 (i,j) 对应于不同的子问题 m\[i][j]。

不同子问题个数最多只有：

$$
\binom{n}{2} + n = \frac{n(n-1)}{2} + n = \Theta(n^2)
$$

### DP 的核心思路

用动态规划解此问题时，在计算过程中**保存已解决的子问题答案**，每个子问题只计算一次，而在后面用到时简单地查一下，从而避免了大量的重复计算。

> ✅ 子问题总数 = O(n²)，远小于递归树的指数级节点数。这就是 DP 能把指数级降到多项式级的原因！

| 方法 | 子问题计算次数 | 总时间 |
|------|-------------|--------|
| 朴素递归 | 指数级重复 | Ω(2ⁿ) |
| 动态规划 | 每个只算1次 | O(n³)（后面会证明） |

## 总结

```
矩阵连乘 DP 建模逻辑线：

Step 1: 分析最优子结构
  └── 最优解在某个 k 处断开，左右子链也必须是最优的

Step 2: 建立递归关系
  └── m[i][j] = min{ m[i][k] + m[k+1][j] + p_{i-1}·p_k·p_j }
                k∈[i, j-1]

朴素递归实现 → 递归树大量重叠 → T(n) = Ω(2ⁿ)

但子问题总数只有 Θ(n²) 个
  → 用表存储 → 每个子问题只算一次 → 多项式时间

        递归(指数级)
             │
     发现重叠子问题 + 子问题数有限
             │
             ▼
        DP 填表(多项式级)
```
