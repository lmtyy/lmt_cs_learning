## 第3页：例1 — Fibonacci Numbers（递归树）

### 递推定义

$$
F(n) = \begin{cases} 1 & n=0 \\ 1 & n=1 \\ F(n-1) + F(n-2) & n>1 \end{cases}
$$

### 递归树（以 F(4) 为例）

```
           F(4)
          /    \
       F(3)    F(2)
      /   \    /   \
   F(2)  F(1) F(1) F(0)
   / \
F(1) F(0)
```

> 🔑 关键观察：F(2) 被计算了 **2次**，F(1) 被计算了 **3次**。随着 n 增大，重复计算呈指数级增长。

> ✅ 这就是动态规划要解决的核心问题 — **重叠子问题（Overlapping Subproblems）**。

---

## 第4页：自底向上迭代计算 Fibonacci

### Bottom-up Iteration

```
f(0) = 0
f(1) = 1
f(2) = 0 + 1 = 1
f(3) = 1 + 1 = 2
f(4) = 1 + 2 = 3
f(5) = 2 + 3 = 5
...
f(n-2) = ...
f(n-1) = ...
f(n) = f(n-1) + f(n-2)
```

> 类比理解：就像爬楼梯，不是每次都从1楼重新爬，而是把每层的结果记下来，一步步往上推。

| 方法 | 时间复杂度 | 空间复杂度 | 重复计算 |
|------|-----------|-----------|---------|
| 递归（自顶向下） | O(2ⁿ) | O(n) 栈 | 大量重复 |
| 迭代（自底向上） | O(n) | O(1)* | 无重复 |

> *只需保存前两个值即可。

> ✅ 自底向上是动态规划的典型实现方式：先算小问题，逐步推出大问题的解。

## 第5页：动态规划算法思想

### 核心思想

```
┌─────────────────────────────────────────────────┐
│  将待求解的问题分解成若干个子问题，              │
│  先求解子问题，并存储子问题的解，                │
│  而 避免计算重复的子问题，                       │
│  再由子问题的解得到原问题的解。                  │
└─────────────────────────────────────────────────┘
```

> ✅ 三个关键词：**分解** → **存储** → **避免重复**

动态规划的本质就是：**用空间换时间**，通过记录已解决的子问题的答案来消除冗余计算。

---

## 第6页：动态规划与分治的联系区别

### 联系

- 都是将问题**分解成子问题**，由子问题的解得到原问题的解

### 区别

| | 分治法 | 动态规划 |
|--|--------|---------|
| 子问题关系 | 相互**独立** | 互相**有联系** |
| 重复计算 | 无 | 存在**重叠子问题** |
| 典型实现 | 递归 | 填表（自底向上） |

> 类比：分治像拼拼图（每块独立拼好再合并），动态规划像织毛衣（每一针都要参考前面织好的部分）。

> ✅ 判断用DP还是分治的关键：子问题之间是否有重叠。有重叠 → DP，无重叠 → 分治。

---

## 总结

```
动态规划 引入逻辑线：

Fibonacci递归 ──→ 发现重复子问题 ──→ 自底向上迭代消除重复
     │                                       │
     ▼                                       ▼
  指数时间 O(2ⁿ)                        线性时间 O(n)

                    ┌──────────────┐
                    │ DP 核心思想  │
                    ├──────────────┤
                    │ 1. 分解子问题│
                    │ 2. 存储解    │
                    │ 3. 避免重复  │
                    └──────────────┘

DP vs 分治：
  分治 → 子问题独立 → 递归即可
  DP   → 子问题重叠 → 必须存储（填表）
```
