# 矩阵连乘问题提出与穷举分析（p007-011）

本节引入矩阵连乘问题（Matrix-Chain Multiplication），说明不同加括号方式对计算量的巨大影响，给出穷举法的复杂度分析，为后续DP解法做铺垫。

---

## 第7页：矩阵连乘问题 — 问题提出

### 问题定义

给定 n 个矩阵 {A₁, A₂, ..., Aₙ}，其中 Aᵢ 是一个 **rᵢ₋₁ × rᵢ** 的矩阵（1 ≤ i ≤ n），即 Aᵢ × Aᵢ₊₁ 是可乘的。

**目标**：求出 n 个矩阵相乘的**最优计算次序**，使得计算量（标量乘法次数）最小。

### 引例

设三个矩阵 A₁, A₂, A₃ 大小分别为：
- A₁: 10 × 100
- A₂: 100 × 5
- A₃: 5 × 50

两种加括号方式：
- (A₁(A₂A₃))
- ((A₁A₂)A₃)

> 🔑 相同的矩阵、不同的计算顺序，乘法次数可能相差**数量级**。

---

## 第8页：矩阵乘法基础与计算量对比

### 矩阵乘法性质

- 矩阵相乘满足**结合律**，连乘积可以有许多不同的次序
- 不同次序用**加括号**方式确定

### 两个矩阵相乘的计算量

若 A 是 p×r 矩阵，B 是 r×q 矩阵，则 C = AB 是 p×q 矩阵：

$$c_{ij} = \sum_{k=1}^{r} a_{ik} b_{kj}$$

> ✅ 两矩阵相乘的标量乘法次数 = **p × q × r**

### 引例计算

A₁(10×100), A₂(100×5), A₃(5×50)：

| 加括号方式 | 计算过程 | 乘法次数 |
|-----------|---------|---------|
| (A₁(A₂A₃)) | A₂A₃: 100×5×50=25000, 再×A₁: 10×100×50=50000 | **75000** |
| ((A₁A₂)A₃) | A₁A₂: 10×100×5=5000, 再×A₃: 10×5×50=2500 | **7500** |

> ✅ 相差 10 倍！最优加括号方式非常重要。

---

## 第9页：完全加括号的矩阵连乘积

### 完全加括号的定义

- 单个矩阵是完全加括号的
- 若矩阵连乘积 A 是完全加括号的，则 A 可表示为 2 个完全加括号的矩阵连乘积 B 和 C 的乘积并加括号，即 **A = (BC)**

### n=4 的例子

矩阵运算 A\*B\*C\*D 的所有完全加括号方式：
1. A \* ((B\*C) \* D)
2. A \* (B \* (C\*D))
3. (A\*B) \* (C\*D)
4. (A \* (B\*C)) \* D
5. ((A\*B) \* C) \* D

> 共 5 种方式。随着 n 增大，加括号方式数量急剧增长。

## 第10页：两个矩阵的乘法代码

### MatrixMultiply 函数

```c
void MatrixMultiply(int **a, int **b, int **c, int ra, int ca,
                    int rb, int cb)
{
    if (ca != rb) error("矩阵不可乘");
    for (int i = 0; i < ra; i++)
        for (int j = 0; j < cb; j++)
        {
            int sum = a[i][0] * b[0][j];
            for (int k = 1; k < ca; k++)
                sum += a[i][k] * b[k][j];
            c[i][j] = sum;
        }
}
```

**参数说明**：
| 参数 | 含义 |
|------|------|
| ra | A 的行数 |
| ca | A 的列数（= B 的行数 rb） |
| rb | B 的行数 |
| cb | B 的列数 |

> 三重循环 → 时间复杂度 O(ra × ca × cb) = O(p × r × q)

## 第11页：穷举搜索法 — 复杂度分析

### 加括号方案数 P(n)

对于 n 个矩阵的连乘积，设有 P(n) 个计算次序。可以在第 k 个和第 k+1 个矩阵之间将原矩阵划分为两个子矩阵序列，然后分别对两个子序列完全加括号：

$$
P(n) = \begin{cases} 1 & n=1 \\ \displaystyle\sum_{k=1}^{n-1} P(k) \cdot P(n-k) & n>1 \end{cases}
$$

### Catalan 数

P(n) = C(n-1)，其中：

$$C(n) = \frac{1}{n+1}\binom{2n}{n} = \Omega\left(\frac{4^n}{n^{3/2}}\right)$$

> ✅ P(n) 是指数级增长的（Catalan 数），因此**穷举法不可行**。

| n | P(n) |
|---|------|
| 2 | 1 |
| 3 | 2 |
| 4 | 5 |
| 5 | 14 |
| 10 | 4862 |
| 15 | 2,674,440 |

> 类比：穷举所有括号方式就像暴力枚举所有排列一样不现实，这正是需要 DP 的原因。

## 总结

```
矩阵连乘问题 逻辑线：

问题提出：n个矩阵相乘，求最优加括号使乘法次数最少
    │
    ▼
关键观察：不同括号方式计算量差距巨大（75000 vs 7500）
    │
    ▼
穷举分析：方案数 P(n) = Catalan数 = Ω(4ⁿ/n^(3/2))
    │
    ▼
结论：穷举不可行 → 需要动态规划！

核心公式：
  两矩阵相乘代价 = p × r × q
  方案数递推：P(n) = Σ P(k)·P(n-k)
```
