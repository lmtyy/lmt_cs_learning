# 流水作业调度问题建模与递推（p039-045）

本节介绍流水作业调度问题（Flow Shop Scheduling）：问题描述、最优子结构证明、递推方程的推导过程。

---

## 第39页：3 流水作业调度（Scheduling to Maximize Profit）

### 问题描述

已知 n 个作业 {1, 2, ..., n} 要在由两台机器 M₁ 和 M₂ 组成的**流水线**上完成加工。每个作业的加工顺序都是先在 M₁ 上加工，然后在 M₂ 上加工。M₁ 和 M₂ 加工作业 i 所需的时间分别为 **aᵢ** 和 **bᵢ**，1 ≤ i ≤ n。

**目标**：确定这 n 个作业的**最优加工次序**，使得从第一个作业在机器 M₁ 上开始加工，到最后一个作业在机器 M₂ 上加工完成所需的时间**最少**。

### 约束条件

- 每个作业必须**先 M₁ 后 M₂**
- M₁ 上的作业是连续的（无空闲）
- M₂ 必须等 M₁ 做完该作业后才能开始加工该作业
- M₂ 上的作业也必须按序逐个加工（但可能有等待空闲）

---

## 第40页：例1 — 两个任务的顺序比较

设有两个任务 i 和 j，它们在 M₁ 和 M₂ 上执行的时间分别为 aᵢ, bᵢ 和 aⱼ, bⱼ。假设 M₁ 和 M₂ 都处于空闲状态，比较不同顺序：

### 顺序1：先 i 后 j

```
M₁: |——aᵢ——|——aⱼ——|
M₂:         |——bᵢ——|——bⱼ——|
```

总时间 = aᵢ + aⱼ + bⱼ（M₂在aᵢ结束后才能开始bᵢ，bᵢ结束后才做bⱼ）

但如果 M₂ 上有"作业积压"（bᵢ还没做完，aⱼ已经做完了），则：
- 总完工时间 = **aᵢ + bᵢ + bⱼ**

### 顺序2：先 j 后 i

```
M₁: |——aⱼ——|——aᵢ——|
M₂:              |——bⱼ——|   等待...   |——bᵢ——|
```

如果 M₂ 上有空闲（bⱼ做完了但aᵢ还没做完），则：
- 总完工时间 = **aⱼ + aᵢ + bᵢ**

### 关键观察

```
M₁ 总耗时 h1 = Σaᵢ 是固定的（与顺序无关）
```

> ✅ 要缩短完工时间，就要**缩短 M₂ 的等待时间 t**（即 M₂ 空闲等 M₁ 的时间）。

---

## 第41页：定义 T(S, t)

### 符号定义

- 记 N = {1, 2, ..., n}，S 为 N 的子集
- 一般情况下，当机器 M₁ 开始加工 S 中的作业时，机器 M₂ 可能正在加工其它的作业，要等待时间 **t** 后才可用来加工 S 中的作业
- 将这种情况下流水线完成 S 中的作业所需的**最短时间**记为 **T(S, t)**

### 原问题

> ✅ 流水作业调度问题的最优值即是 **T(N, 0)**（全部作业，M₂初始等待时间为0）

### 含义图解

```
M₁: |——aᵢ——| ...
M₂:   ←t→  |——bᵢ——| ...

t = M₂需要等待的时间（可能在加工之前的作业）
```

当第一个作业只有一个（如 A₁）时：
- M₁ 加工 a₁ 时间
- M₂ 需等 max(t, a₁) 后再加工 b₁

---

## 第42页：1. 流水作业调度问题具有最优子结构性质

### 设置

设 π 是 n 个流水作业的一个**最优调度**（实际上是作业的一个排列），π(1), ..., π(n) 表示 π 中各个作业。

该最优调度的加工时间为：
$$
	a_{\pi(1)} + T'
$$

其中 T' 是在机器 M₂ 的等待时间为 b_{π(1)} 时，安排作业 π(2), ..., π(n) 所需的时间。

### 要证明的结论

若 π 是最优调度，那么 π 中除去 π(1)，在 M₂ 上等待时间为 b_{π(1)} 时，按照 π 中顺序安排 π(2), ..., π(n) 的顺序**也是最优的**。

即：T' = T(S, b_{π(1)})，其中 S = N - {π(1)}

---

## 第43页：最优子结构的反证法证明

### 证明过程

记 S = N - {π(1)}，则 T(S, b_{π(1)}) 为在 M₂ 上等待时间为 b_{π(1)} 时 S 的最优调度时间。

**要证**：T' = T(S, b_{π(1)})

1. 由 T' 的定义知：**T' ≥ T(S, b_{π(1)})**（T(S, b_{π(1)}) 是最优值，T' 不可能比它更小）

2. 假设 **T' > T(S, b_{π(1)})**：
   - 设 π' 是作业集 S 在机器 M₂ 等待时间 b_{π(1)} 时的一个最优调度
   - 则 π(1), π'(2), ..., π'(n) 是 N 的一个调度
   - 该调度所需的时间为 a_{π(1)} + T(S, b_{π(1)}) < a_{π(1)} + T'
   - 这与 **π 是 N 的一个最优调度矛盾**！

3. 所以 T' = T(S, b_{π(1)})

> ✅ 说明流水作业调度问题具有**最优子结构性质**。

> 类比：和矩阵连乘的证明一样，都是"假设子问题的解不是最优的 → 用更优的替换 → 原问题得到更好的解 → 矛盾"。

---

## 第44页：2. 递归计算最优值

### 原始递推方程

由流水作业调度问题的最优子结构性质可知：

$$T(N, 0) = \min_{1 \le i \le n} \{a_i + T(N \setminus \{i\}, b_i)\}$$

**对照代码理解**：
- 枚举第一个加工的作业 i（1 ≤ i ≤ n）
- 第一个作业在 M₁ 上花费 aᵢ 时间
- 之后剩余作业集 N\{i} 需在 M₂ 等待时间为 bᵢ 的条件下完成
- 取所有选择中的最小值

### 图解

```
M₁: |——aᵢ——|——aⱼ——| ─ ─ ─
M₂:         |——bᵢ——|——bⱼ——| ─ ─ ─
```

第一个作业 i 做完后，M₂ 的等待时间变为 bᵢ（M₂需要bᵢ时间来加工作业i，这段时间后续作业需要等待）。

---

## 第45页：一般递推公式 T(S, t)

### M₂ 等待时间 t 的传递

上述关系式可以推广到一般情形计算 T(S, t)：

t代表M2还差多久能干完前置任务，才会空闲下来能干S里的任务

t'代表M2还差多久能干完前置任务，才会空闲下来能干S-{i}里的任务（即干完作业i），t与t'的计算起点一直

**情况1**：t > aᵢ（M₂还在忙，M₁先做完了aᵢ，M₂做完之前的任务后还需处理bᵢ）
- 新的等待时间 t' = t - aᵢ + bᵢ

```
M₁: |——aᵢ——|——aⱼ——| ...
M₂: |═══t═══|——bᵢ——|——bⱼ——| ...
```

**情况2**：t ≤ aᵢ（M₁还没做完aᵢ，M₂已经空闲了）
- 新的等待时间 t' = 0 + bᵢ = bᵢ

```
M₁: |————aᵢ————|——aⱼ——| ...
M₂:   ←t→      |——bᵢ——|——bⱼ——| ...
```

### 统一公式

两种情况合并：新等待时间 = bᵢ + max{t - aᵢ, 0}
$$
T(S, t) = \min_{i \in S} \{a_i + T(S \setminus \{i\},\ b_i + \max\{t - a_i,\ 0\})\}
$$
表示遍历S，取出一个作业，使得剩下的最优序列与总最优序列一致。体现在剩下作业调度算法中，作业集合为S\{i}，等待时间为作业i在M2的工作时间bi+max{t-ai,0}

### 复杂度

通过该式可给出流水调度的动态规划算法，但其时间复杂度将是 **O(2ⁿ)**（因为状态是集合 S 的所有子集，共 2ⁿ 个）。

> ✅ 可采用下述 **Johnson 算法**来得到 O(n log n) 的高效解法。

## 总结

```
流水作业调度 DP 建模逻辑线：

问题：n 个作业在 M₁→M₂ 两台机器上的最优排序
  │
  ▼
关键定义：T(S, t) = 集合S中作业在M₂等待t时的最短完工时间
原问题 = T(N, 0)
  │
  ▼
最优子结构（反证法）：
  最优调度π中，去掉第一个作业后，剩余作业的排序也是最优的
  │
  ▼
递推方程：
  T(N, 0) = min{ aᵢ + T(N\{i}, bᵢ) }      （选第一个作业）
  T(S, t) = min{ aᵢ + T(S\{i}, bᵢ+max{t-aᵢ,0}) }  （一般情况）
  │
  ▼
直接 DP 复杂度 O(2ⁿ)（子集状态空间） → 需要 Johnson 算法优化
```
