# 最优BST代价计算实例（p087-092）

本节通过具体实例计算不同 BST 的平均搜索代价（平均路长），明确代价公式，为后续 DP 建模做准备。

---

## 第87页：例 — 3个标识符的所有 BST

### 标识符集

{1, 2, 3} = {do, if, stop}

### 5 种可能的 BST

```
(a)  1        (b) 1       (c)  2      (d)   3      (e)  3
      \            \          / \         /            /
       2            3        1   3       1            2
        \          /                      \          /
         3        2                        2        1
```

### 两组概率设定

1. **等概率**：pᵢ = qᵢ = 1/7（每个内、外结点检索概率相同）
2. **非等概率**：P₁=0.5, P₂=0.1, P₃=0.05, q₀=0.15, q₁=0.1, q₂=0.05, q₃=0.05

---

## 第88页：比较次数的定义

### 搜索代价规则

在检索过程中，每进行一次比较，就进入下面一层：

| 搜索类型 | 落在哪里 | 比较次数 |
|---------|---------|---------|
| 成功检索 | 内部结点（第 k 层） | **层数 + 1** |
| 不成功检索 | 外部结点（第 k 层） | **该外部结点的层数** |

> 🔑 注意区别：内部结点比较次数 = 深度+1（因为要跟该结点比一次），外部结点比较次数 = 深度（比较到其父结点后发现走到空，不需要额外比较）。

## 第89页：计算树 a 的平均搜索代价

### 概率设定

P₁=0.5, P₂=0.1, P₃=0.05, q₀=0.15, q₁=0.1, q₂=0.05, q₃=0.05

### 树 a 的结构与各结点深度

```
       1 (深度0)
      / \
    q₀   2 (深度1)
        / \
      q₁   3 (深度2)
           / \
         q₂   q₃
```

### 代价计算（原ppt下部计算过程中把q0误带成0.05）

落入q0只需一次比较：x?1若x<1则落入q0结束，x=1则落入1结束，x>1则落入右边继续
$$
    P_a(n) = 1 \times p_1 + 2 \times p_2 + 3 \times p_3 + 1 \times q_0 + 2 \times q_1 + 3 \times (q_2 + q_3)
$$

$$
= 1 \times 0.5 + 2 \times 0.1 + 3 \times 0.05 + 1 \times 0.15 + 2 \times 0.1 + 3 \times (0.05 + 0.05)
$$

$$
    = 0.5 + 0.2 + 0.15 + 0.15 + 0.2 + 0.3 = \textbf{1.5}
$$

## 第90页：计算树 b ~ e 的平均搜索代价

### 树 b

$$
    P_b(n) = 1 \times p_1 + 2 \times p_3 + 3 \times p_2 + 1 \times q_0 + 2 \times q_3 + 3 \times (q_1 + q_2)
$$

$$
    = 0.5 + 0.1 + 0.3 + 0.15 + 0.1 + 0.45 = \textbf{1.6}
$$

### 树 c

$$
    P_c(n) = 1 \times p_2 + 2 \times (p_1 + p_3) + 2 \times (q_0 + q_1 + q_2 + q_3)
$$

$$
    = 0.1 + 2 \times 0.55 + 2 \times 0.35 = 0.1 + 1.1 + 0.7 = \textbf{1.9}
$$

### 树 d

$$
    P_d(n) = 1 \times p_3 + 2 \times p_1 + 3 \times p_2 + 1 \times q_3 + 2 \times q_0 + 3 \times (q_1 + q_2)
$$

$$
    = 0.05 + 1.0 + 0.3 + 0.05 + 0.3 + 0.45 = \textbf{2.15}
$$

### 树 e

$$
    P_e(n) = 1 \times p_3 + 2 \times p_1 + 3 \times p_2 + 1 \times q_3 + 2 \times q_0 + 3 \times (q_1 + q_2)
$$

$$
    = 0.05 + 1.0 + 0.3 + 0.05 + 0.3 + 0.45 = \textbf{2.15}
$$

### 五棵树代价汇总

| 树 | 根 | 平均搜索代价 |
|---|---|-----------|
| a | 1 (do) | **1.5** ← 最优 |
| b | 1 (do) | 1.6 |
| c | 2 (if) | 1.9 |
| d | 3 (stop) | 2.15 |
| e | 3 (stop) | 2.15 |

> ✅ 树 a 的平均搜索代价最小（1.5），是这组概率下的最优 BST。因为 P₁=0.5 最大，把它放在根部（深度最浅）使得搜索 "do" 的代价最低。

## 第91页：形式化符号定义

### 概率符号（本PPT后续使用的记号）

- **bⱼ**：找到元素 x = xⱼ 的概率（成功搜索），1 ≤ j ≤ n
- **aᵢ**：确定 x ∈ (xᵢ, x_{i+1}) 的概率（不成功搜索），0 ≤ i ≤ n

其中约定 x₀ = -∞，x_{n+1} = +∞。

### 约束条件

$$
    a_i \ge 0,\ 0 \le i \le n; \quad b_j \ge 0,\ 1 \le j \le n; \quad \sum_{i=0}^{n} a_i + \sum_{j=1}^{n} b_j = 1
$$

### 存取概率分布

称集合 {a₀, b₁, a₁, b₂, ..., bₙ, aₙ} 为有序集 S 的**存取概率分布**。

> 🔑 注意符号变化：这里用 **bⱼ 表示成功概率**（对应之前的 P(j)），**aᵢ 表示不成功概率**（对应之前的 Q(i)）。后续递推公式将使用这套符号。

## 第92页：平均路长公式

### 定义

在一个表示 S 的二叉树 T 中：
- 设存储元素 xᵢ 的结点深度为 **cᵢ**
- 叶结点 (xⱼ, x_{j+1}) 的结点深度为 **dⱼ**

### 平均路长（平均搜索代价）

$$
    p = \sum_{i=1}^{n} b_i(1 + c_i) + \sum_{j=0}^{n} a_j d_j
$$

| 部分 | 含义 |
|------|------|
| bᵢ(1 + cᵢ) | 成功搜索 xᵢ 的代价 × 概率 |
| aⱼdⱼ | 不成功搜索落入间隙 j 的代价 × 概率 |

> ✅ **p** 表示在二叉搜索树 T 中作一次搜索所需的**平均比较次数**，也称为二叉搜索树 T 的**平均路长**。

> 在一般情况下，不同的二叉搜索树的平均路长是不同的。**最优 BST 就是使 p 最小的那棵树。**

---

## 总结

```
最优 BST 代价计算逻辑线：

代价公式：p = Σ bᵢ(1+cᵢ) + Σ aⱼdⱼ
  其中 bᵢ=成功概率, aⱼ=不成功概率
       cᵢ=内部结点深度, dⱼ=外部结点深度

实例（n=3, 5棵树）：
  树a: p=1.5 ← 最优（高频关键字在根部）
  树b: p=1.6
  树c: p=1.9
  树d: p=2.15
  树e: p=2.15

关键观察：
  搜索频率高的关键字应该在浅层
  但不能只看单个概率，要综合考虑子树的所有概率
  → 需要 DP 来全局优化
```
