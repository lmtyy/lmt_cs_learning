# 最优BST子结构证明与代价递推（p098-100）

本节给出最优子结构性质的严格证明，推导子树代价与整棵树代价之间的关系式，为建立递推方程做准备。

---

## 第98页：子树的定义与权重

### 子树的形式化

二叉搜索树 T 的一棵含有顶点 xᵢ, ..., xⱼ 和叶顶点 (x_{i-1}, xᵢ), ..., (xⱼ, x_{j+1}) 的子树可以看作是有序集 {xᵢ, ..., xⱼ} 关于全集为 {x_{i-1}, x_{j+1}} 的一棵二叉搜索树（T 自身可以看作是有序集）。

### 子树的权重 wᵢⱼ

根据 S 的存取分布概率，在子树的顶点处被搜索到的概率是：

$$
    w_{ij} = \sum_{k=i-1}^{j+1} a_k + \sum_{k=i}^{j} b_k
$$

> 🔑 wᵢⱼ 就是子树中所有结点（内部+外部）的概率之和。即搜索落入这棵子树的总概率。

### 条件概率

{xᵢ, ..., xⱼ} 的存储概率分布为 {ā_{i-1}, b̄ᵢ, ..., b̄ⱼ, āⱼ}，其中：

$$
    \bar{b}_k = b_k / w_{ij}, \quad i \le k \le j
$$

$$
    \bar{a}_h = a_h / w_{ij}, \quad i-1 \le h \le j
$$

> 即在"已知搜索落入该子树"的条件下，各结点的条件概率。

## 第99页：代价关系推导

### 设置

设 Tᵢⱼ 是有序集 {xᵢ, ..., xⱼ} 关于存储概率分布 {a_{i-1}, bᵢ, ..., bⱼ, aⱼ} 的一棵**最优二叉搜索树**：
- 平均路长为 **pᵢⱼ**
- 根顶点存储的元素为 **xₘ**
- 左子树 Tₗ 的平均路长为 **pₗ**
- 右子树 Tᵣ 的平均路长为 **pᵣ**

### 关键观察：深度变化

由于 Tₗ 和 Tᵣ 中顶点深度是它们在 Tᵢⱼ 中的深度**减1**，所以得到：

$$
    w_{ij} \cdot p_{ij} = w_{i,m-1}(p_l + 1) + w_{m,m} + w_{m+1,j}(p_r + 1)
$$

展开：

$$
    w_{ij} \cdot p_{ij} = w_{ij} + w_{i,m-1} \cdot p_l + w_{m+1,j} \cdot p_r \quad (i \le j)
$$

### 公式解读

| 项 | 含义 |
|---|------|
| w_{i,m-1}(pₗ + 1) | 左子树的搜索代价（子树内代价 + 每个结点多比较1次因为多了根这一层） |
| w_{m,m} = bₘ | 搜索到根结点 xₘ 的代价（比较1次×概率bₘ） |
| w_{m+1,j}(pᵣ + 1) | 右子树的搜索代价（同理多比较1次） |

> 🔑 子树从独立变成嵌入整棵树后，所有结点深度+1（仅看嵌入点作为根节点的子树，不考虑全局），所以每个结点的代价增加1×其概率 = 增加整棵子树的权重 w（概率是全局的，子树权重即子树所有结点概率和，不为1，因为是全局的）。

## 第100页：最优子结构的反证法证明

### 证明

由于 Tₗ 是有序集 {xᵢ, ..., x_{m-1}} 的一棵二叉搜索树，故 pₗ ≥ p_{i,m-1}（p_{i,m-1} 是该集合的最优平均路长）。

若 pₗ > p\_{i,m-1}，则用 T\_{i,m-1}（最优的左子树）替换 Tₗ，可得到平均路长比 Tᵢⱼ **更小**的二叉搜索树。这与 Tᵢⱼ 是最优二叉搜索树**矛盾**。

同理可证，Tᵣ 也是一棵最优二叉搜索树。

> ✅ 因此，最优二叉搜索树问题具有**最优子结构性质**。

### 构造策略

> 构造最优二叉搜索树时，可以选择先构造其左右子树，使其左右子树最优，然后构造整棵树。

这正是 DP 自底向上构造的思路：
1. 先求所有单个关键字的最优 BST（平凡）
2. 再求两个关键字的最优 BST
3. 逐步扩大，直到求出全部 n 个关键字的最优 BST

## 总结

```
最优 BST 代价关系推导：

子树权重：
  wᵢⱼ = Σa_k (k=i-1..j) + Σb_k (k=i..j)
  = 搜索落入该子树的总概率

代价关系（选 xₘ 为根）：
  wᵢⱼ · pᵢⱼ = wᵢⱼ + w_{i,m-1}·pₗ + w_{m+1,j}·pᵣ

    为什么多了一个 wᵢⱼ？
    → 子树嵌入后所有结点深度+1
    → 每个结点多比较1次
    → 总增加量 = Σ概率×1 = wᵢⱼ

最优子结构（反证法）：
  如果左/右子树不是最优的 → 替换后整体更优 → 矛盾
  ∴ 左右子树必须各自最优
```
