# 动态规划算法的基本要素（p031-033）

本节从理论层面总结动态规划算法能够适用的两个关键性质：最优子结构和重叠子问题。

---

## 第31页：2 动态规划算法的基本要素（Elements of Dynamic Programming）

### DP 适用的两大基本要素

| 编号 | 中文 | 英文 |
|------|------|------|
| 1 | 最优子结构性质 | Optimal Substructure |
| 2 | 重叠子问题性质 | Overlapping Subproblems |

> ✅ 一个问题能用 DP 求解，必须同时满足这两个性质。缺一不可。

---

## 第32页：1. 最优子结构（Optimal Substructure）

### 定义

当问题的最优解**包含了其子问题的最优解**时，称该问题具有**最优子结构性质**。

### 证明方法：反证法（剪切-粘贴法）

分析最优子结构性质时的标准套路：

1. **假设**：由问题的最优解导出的子问题不是最优的
2. **构造矛盾**：设法说明在这个假设下可以构造出一个比原问题更优解更好的解
3. **结论**：与"原解是最优的"矛盾 → 假设不成立 → 子问题的解必须也是最优的

> 类比：假如北京→广州的最短路经过武汉，而北京→武汉这段不是最短路，那我把北京→武汉这段换成更短的路径，就得到了北京→广州的更短路径——矛盾！所以北京→武汉这段也必须是最短的。

### 在矩阵连乘中的体现

- 原问题：A[1:n] 的最优计算次序
- 子问题：A[1:k] 和 A[k+1:n] 的计算次序
- 如果 A[1:k] 的次序不是最优的 → 替换成更优的 → 原问题的总代价减少 → 与原解最优矛盾 ✓

## 第33页：2. 重叠子问题（Overlapping Subproblems）

### 现象

在用递归算法**自顶向下**解此问题时，每次产生的子问题并不总是新问题，有些子问题被**反复计算多次**。

### DP 的应对策略

- 动态规划算法对**每个问题只解一次**
- 将其解保存在一个**表格**中
- 当再次需要解此问题时，用**常数时间**查看一下结果

### 结论

> ✅ 因此，用动态规划算法通常只需要**多项式时间**。

### 对比

| | 朴素递归 | 动态规划 |
|--|---------|---------|
| 同一子问题 | 每次重新算 | 只算一次，存表 |
| 再次用到时 | 重新递归展开 | O(1) 查表 |
| 总时间 | 指数级 | 多项式级 |

> 类比：递归像考试时每道题都从头推导，DP像把公式抄在草稿纸上随时查。一个费时，一个省时。

---

## 总结

```
动态规划适用的两个基本要素：

┌─────────────────────────────────────────┐
│  1. 最优子结构 (Optimal Substructure)   │
│     最优解包含子问题的最优解            │
│     证明方法：反证法（剪切-粘贴）       │
├─────────────────────────────────────────┤
│  2. 重叠子问题 (Overlapping Subproblems)│
│     子问题被重复求解                    │
│     解决方法：存表，每个只算一次        │
└─────────────────────────────────────────┘

两者的关系：
  最优子结构 → 保证可以从子问题的最优解推出原问题最优解
  重叠子问题 → 保证用表存储能显著减少计算量

  缺少最优子结构 → 子问题最优 ≠ 原问题最优，DP 不适用
  缺少重叠子问题 → 直接分治即可，不需要存表
```
