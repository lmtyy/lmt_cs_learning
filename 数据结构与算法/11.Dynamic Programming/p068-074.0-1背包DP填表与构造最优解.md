# 0-1背包DP填表、构造最优解与复杂度（p068-074）

本节给出 0-1 背包问题的完整 DP 实现：填表代码、实例演示、构造最优解（Traceback）以及复杂度分析。

## 第68页：填表实例

### 题目

四个物品，背包容积为 5：
- w\[4] = {2, 1, 3, 2}
- v\[4] = {12, 10, 20, 15}

求最大价值 m\[1]\[c] 及选取的物品编号。

### 递推公式回顾

$$
m(i, j) = \begin{cases} \max\{m(i+1, j),\ m(i+1, j-w_i) + v_i\} & j \ge w_i \\ m(i+1, j) & 0 \le j < w_i \end{cases}
$$

$$
m(n, j) = \begin{cases} v_n & j \ge w_n \\ 0 & 0 \le j < w_n \end{cases}
$$

### DP 表 m\[i]\[j]

| i\j | 0 | 1 | 2 | 3 | 4 | 5 |
|-----|---|---|---|---|---|---|
| 4 | 0 | 0 | 15 | 15 | 15 | 15 |
| 3 | 0 | 0 | 15 | 20 | 20 | 35 |
| 2 | 0 | 10 | 15 | 25 | 30 | 35 |
| 1 | | | | | | **37** |

> ✅ 最大价值 m\[1]\[5] = **37**

### 填表代码（对照递推公式）

```c
// 初始化最后一行 m[n][j]（边界条件）
for (int j = 0; j < w[n]; j++)     // j < w[n] 时放不下
    m[n][j] = 0;
for (int j = w[n]; j <= c; j++)    // j >= w[n] 时放得下
    m[n][j] = v[n];

// 从 i=n-1 到 i=2，逐行填表
for (int i = n - 1; i > 1; i--) {
    for (int j = 0; j < w[i]; j++)           // 放不下第 i 个物品
        m[i][j] = m[i+1][j];
    for (int j = w[i]; j <= c; j++)          // 放得下，取 max
        m[i][j] = max(m[i+1][j], m[i+1][j - w[i]] + v[i]);
}

// 单独处理第 1 个物品（原问题答案）
m[1][c] = m[2][c];
if (c >= w[1])
    m[1][c] = max(m[1][c], m[2][c - w[1]] + v[1]);
```

### 代码与递推公式的对照

| 代码行 | 对应公式 | 说明 |
|--------|---------|------|
| `m[n][j] = 0` (j < w\[n]) | m(n,j) = 0 当 j < wₙ | 最后一个物品放不下 |
| `m[n][j] = v[n]` (j ≥ w\[n]) | m(n,j) = vₙ 当 j ≥ wₙ | 最后一个物品放得下 |
| `m[i][j] = m[i+1][j]` (j < w\[i]) | m(i,j) = m(i+1,j) 当 j < wᵢ | 第 i 个放不下 |
| `m[i][j] = max(...)` (j ≥ w\[i]) | m(i,j) = max{...} 当 j ≥ wᵢ | 选或不选取较大值 |

## 第69页：3. 算法描述（Knapsack 完整代码）

```c
template<class Type>
void Knapsack(Type p, int w, int c, int n, Type **m)
{
    // 初始化只考虑最后一个物品的行（即最后一行）
    int jMax = min(w[n] - 1, c);
    for (int j = 0; j <= jMax; j++)
        m[n][j] = 0;
    for (int j = w[n]; j <= c; j++)
        m[n][j] = v[n];
    
    // 从 i=n-1 到 i=2 逐行填表
    for (int i = n - 1; i > 1; i--)
    {
        jMax = min(w[i] - 1, c); //jMax为重量-1与背包总容量的较小值
        for (int j = 0; j <= jMax; j++) //若jMax里的w[i]不-1则这里要去掉等号；若不与c取小值则可能会使j>c下面的m矩阵越界
            m[i][j] = m[i+1][j];
        for (int j = w[i]; j <= c; j++)
            m[i][j] = max(m[i+1][j], m[i+1][j - w[i]] + v[i]);
    }
    
    // 单独处理第1个物品
    m[1][c] = m[2][c];
    if (c >= w[1])
        m[1][c] = max(m[1][c], m[2][c - w[1]] + v[1]);
}
```

### 为什么第1个物品要单独处理？

只剩下第 1 个物品时，如果剩余背包容积大于 w₁，要进行选择。代码中只计算 m\[1]\[c]（原问题答案），不需要填满整行。（m\[1][c-1]等等不需要填，而前面的逻辑是j从0遍历到jMax，可能会填到m\[1][c-1])

## 第70页：构造最优解（回溯过程图解）

### 实例回溯

利用 m 表从 m\[1]\[5] 出发，逐步确定每个物品是否被选中：

```
从 m[1][5]=37 开始，c=5

Step 1: m[1][5] vs m[2][5]
  m[1][5]=37 ≠ m[2][5]=35 → x₁=1（选了物品1）
  c = 5 - w[1] = 5 - 2 = 3

Step 2: m[2][3] vs m[3][3]
  m[2][3]=25 ≠ m[3][3]=20 → x₂=1（选了物品2）
  c = 3 - w[2] = 3 - 1 = 2

Step 3: m[3][2] vs m[4][2]
  m[3][2]=15 = m[4][2]=15 → x₃=0（没选物品3）
  c 不变 = 2

Step 4: m[4][2] = 15 > 0 → x₄=1（选了物品4）
```

> ✅ 最优解：x = {1, 1, 0, 1}，选物品1、2、4，总价值 = 12+10+15 = **37**

## 第71页：构造最优解（判断逻辑）

### 回溯判断规则

| 比较 | 结论 | 操作 |
|------|------|------|
| m\[i]\[c] ≠ m\[i+1]\[c] | xᵢ = 1（选了第 i 个物品） | c = c - w\[i] |
| m\[i]\[c] = m\[i+1]\[c] | xᵢ = 0（没选第 i 个物品） | c 不变 |
| m\[n]\[c] > 0 | xₙ = 1 | 最后一个物品被选 |
| m\[n]\[c] = 0 | xₙ = 0 | 最后一个物品没被选 |

### 实例验证

| 步骤 | 比较 | 结果 | 剩余容量 |
|------|------|------|---------|
| i=1 | m\[1]\[5]=37 ≠ m\[2]\[5]=35 | x\[1]=1 | c=5-2=3 |
| i=2 | m\[2]\[3]=25 ≠ m\[3]\[3]=20 | x\[2]=1 | c=3-1=2 |
| i=3 | m\[3]\[2]=15 = m\[4]\[2]=15 | x\[3]=0 | c=2 |
| i=4 | m\[4]\[2]=15 > 0 | x\[4]=1 | — |

## 第72页：Traceback 代码

```c
template<class Type>
void Traceback(Type **m, int *w, int c, int n, int *x)
{
    for (int i = 1; i < n; i++)
        if (m[i][c] == m[i+1][c])
            x[i] = 0;           // 没选第 i 个物品
        else
        {
            x[i] = 1;           // 选了第 i 个物品
            c -= w[i];          // 背包容量减少
        }
    x[n] = (m[n][c]) ? 1 : 0;  // 最后一个物品：有值就选，即上页i最大的行j=（总容量-已经确定选了的东西的总重量）的列不是0就代表选了最后一个物品，上页例中，选了物品2、4，w2+w4=3，被c减得2，j=2的列为15>0，代表选了物品1
}
```

### 代码逐行对照

| 行 | 作用 | 判断依据 |
|---|------|---------|
| `if (m[i][c] == m[i+1][c])` | 不选第 i 个物品 | 选与不选的最优值相同 → 说明没选 |
| `else { x[i]=1; c-=w[i]; }` | 选了第 i 个物品 | 最优值来自"选了"的分支 |
| `x[n] = (m[n][c]) ? 1 : 0` | 最后一个物品的判断 | m\[n]\[c] > 0 说明能放且放了 |

> 🔑 回溯的核心逻辑：如果 m\[i]\[c] = m\[i+1]\[c]，说明"不选 i"就已经达到最优了，所以 i 没被选中。反之说明最优值一定来自"选了 i"的那个分支。

## 第73页：练习

### 题目

n=3, (w₁, w₂, w₃) = (2, 3, 4), (v₁, v₂, v₃) = (1, 2, 5), c=6，填表 m\[i]\[j]。

### 解答

**边界 i=3**：w₃=4, v₃=5
- j < 4: m\[3]\[j] = 0
- j ≥ 4: m\[3]\[j] = 5

| j | 0 | 1 | 2 | 3 | 4 | 5 | 6 |
|---|---|---|---|---|---|---|---|
| m\[3]\[j] | 0 | 0 | 0 | 0 | 5 | 5 | 5 |

**i=2**：w₂=3, v₂=2
- j < 3: m\[2]\[j] = m\[3]\[j]
- j ≥ 3: m\[2]\[j] = max{m\[3]\[j], m\[3]\[j-3]+2}

| j | 0 | 1 | 2 | 3 | 4 | 5 | 6 |
|---|---|---|---|---|---|---|---|
| m\[2]\[j] | 0 | 0 | 0 | 2 | 5 | 5 | 5 |

计算过程：
- m\[2]\[3] = max{m\[3]\[3], m\[3]\[0]+2} = max{0, 2} = 2
- m\[2]\[4] = max{m\[3]\[4], m\[3]\[1]+2} = max{5, 2} = 5
- m\[2]\[5] = max{m\[3]\[5], m\[3]\[2]+2} = max{5, 2} = 5
- m\[2]\[6] = max{m\[3]\[6], m\[3]\[3]+2} = max{5, 2} = 5

**i=1**：只算 m\[1]\[6]，w₁=2, v₁=1
- m\[1]\[6] = max{m\[2]\[6], m\[2]\[4]+1} = max{5, 6} = **6**

> ✅ 最大价值 = 6

**回溯**：
- m\[1]\[6]=6 ≠ m\[2]\[6]=5 → x₁=1, c=6-2=4
- m\[2]\[4]=5 = m\[3]\[4]=5 → x₂=0, c=4
- m\[3]\[4]=5 > 0 → x₃=1

> ✅ 最优解：x = {1, 0, 1}，选物品1和物品3，总价值 = 1+5 = **6**（与之前引例一致）

## 第74页：4. 计算复杂性分析

### 时间复杂度

- 上述算法需要 **O(nc)** 计算时间（n 个物品 × c 个容量值×每个格子计算时间O(1)(因为计算某格子需要用到的数字一定在之前被算过，并储存在m例，只需要取出、进行计算即可，无需递归））
- 构造最优结构需要 **O(n)** 的计算时间

### 注意事项

> ⚠️ 缺点：算法要求每个物品的重量为**整数**，当背包容量 c 很大时，算法需要的计算时间较多。

| 参数 | 影响 |
|------|------|
| n（物品数量） | 表的行数 |
| c（背包容量） | 表的列数 |
| n × c | 总计算量 |

> 🔑 O(nc) 是**伪多项式时间**（pseudo-polynomial）：它对数值 c 是多项式的，但对输入规模（c 的二进制位数 log c）是指数的。当 c 非常大时（如 c = 2³⁰），表格巨大，不实用。

### 与其他方法对比

| 方法 | 时间复杂度 | 适用场景 |
|------|-----------|---------|
| 穷举法 | O(2ⁿ) | n 很小 |
| DP 填表 | O(nc) | c 不太大且为整数 |
| 贪心法 | 不适用于 0-1 背包 | 只适用于分数背包 |

## 总结

```
0-1 背包 DP 完整流程：

Step 1: 最优子结构 → 去掉物品1的决策后，剩余是子问题最优解
Step 2: 递推方程 → m(i,j) = max{m(i+1,j), m(i+1,j-wᵢ)+vᵢ}
Step 3: 填表 → 从 i=n 到 i=1，对每个 j 从 0 到 c
Step 4: Traceback → 比较 m[i][c] 与 m[i+1][c] 判断是否选了物品 i

填表方向：
  行：从下往上（i = n → 1）
  列：从左到右（j = 0 → c）

  j→  0  1  2  3  4  5
i↓
n    [边界：能放就放，不能放就0]
n-1  [由第n行推出]
...
2    [由第3行推出]
1    [只算m[1][c]]

复杂度：O(nc) 时间，O(nc) 空间
注意：伪多项式时间，c很大时不实用
```
