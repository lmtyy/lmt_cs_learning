# Johnson算法实现与复杂度（p052-055）

本节给出 Johnson 法则的具体算法步骤、代码实现及复杂度分析。

## 第52页：Johnson 法则的调度

### 定理

对于流水作业调度问题，必存在一个最优调度 π，使得 π(i) 和 π(i+1) 满足 Johnson 不等式：

$$
\min\{b_{\pi(i)},\ a_{\pi(i+1)}\} \ge \min\{b_{\pi(i+1)},\ a_{\pi(i)}\}
$$

推广到任意 i < j：


$$
	\min\{b_{\pi(i)},\ a_{\pi(j)}\} \ge \min\{b_{\pi(j)},\ a_{\pi(i)}\} \quad \forall\ i < j
$$

> ✅ 即：最优调度中，任意两个作业的相对顺序都满足 Johnson 不等式。只要找到一个所有相邻对都满足 Johnson 不等式的排列，就是最优调度。

## 第53页：Johnson 算法（3步）

### 算法步骤

**(1)** 将作业分为两组：

- **AB** = {i | aᵢ < bᵢ}（M₁ 上快的，应该先做）
- **BA** = {i | bᵢ ≤ aᵢ}（M₂ 上快的，应该后做）

**(2)** 排序：
- 将 **AB** 中作业依 **aᵢ 的非减次序**排列（aᵢ 小的排前面）
- 将 **BA** 中作业依 **bᵢ 的非增次序**排列（bᵢ 大的排前面）

**(3)** 拼接：
- AB 中作业接 BA 中作业即构成满足 Johnson 法则的最优调度

### 直觉理解

> 类比：AB 组的作业"M₁上花时间少"，应尽早上 M₁ 让 M₂ 尽早开工；BA 组的作业"M₂上花时间少"，放后面不会让 M₂ 等太久。

> AB 内部按 aᵢ 升序 → M₁ 上短的先做，让 M₂ 尽早有活干
> BA 内部按 bᵢ 降序 → M₂ 上长的先做，让 M₂ 尽量不空闲

### 思考题

> 这样构成的调度一定满足 Johnson 法则吗？

答：是的。可以验证 AB 内部任意两个、BA 内部任意两个、以及 AB 中的接 BA 中的，都满足 min{bᵢ, aⱼ} ≥ min{bⱼ, aᵢ}。

## 第54页：算法描述（FlowShop 代码）

```c
int FlowShop(int n, int* a, int* b, int* c)
{
    // --- 第1步：构造排序关键字 ---
    class Jobtype {
    public:
        int operator <= (Jobtype a) const
        { return (key <= a.key); }
        int key;      // 排序关键字
        int index;    // 原始作业编号
        bool job;     // true=属于AB组, false=属于BA组
    };
    
    Jobtype *d = new Jobtype[n];//存放作业数组，按该作业在a、b的运行时间小值排序
    for (int i = 0; i < n; i++) {
        d[i].key = a[i] > b[i] ? b[i] : a[i];  // key = min(aᵢ, bᵢ)
        d[i].job = a[i] <= b[i];                 // true → AB组 (aᵢ<bᵢ 或 aᵢ=bᵢ)
        d[i].index = i;
    }
    
    // --- 第2步：按 key 排序 ---
    sort(d, n);  // 按 key 非减排序，这样AB即为d[].job=true部分的顺序作业排列，BA为d[].job=false部分的作业逆序排列
    
    // --- 第3步：分组放入结果数组 c[] ---
    int j = 0, k = n - 1;
    for (int i = 0; i < n; i++) {
        if (d[i].job)              // AB组：从前往后放
            c[j++] = d[i].index;
        else                       // BA组：从后往前放
            c[k--] = d[i].index;
    }
    
    // --- 第4步：计算最优完工时间 ---
    j = a[c[0]];              // M₁ 第一个作业完成时间
    k = j + b[c[0]];          // M₂ 第一个作业完成时间
    for (int i = 1; i < n; i++) {
        j += a[c[i]];                        // M₁ 累计时间
        k = (j < k) ? k + b[c[i]] : j + b[c[i]];  // M₂ 完成时间
    }
    
    delete d;
    return k;  // 返回最优完工时间
}
```

### 代码逻辑拆解

| 阶段 | 代码 | 作用 |
|------|------|------|
| 构造 key | `key = a[i] > b[i] ? b[i] : a[i]` | key = min(aᵢ, bᵢ)，统一排序依据 |
| 分组标记 | `job = a[i] <= b[i]` | true=AB组(aᵢ≤bᵢ), false=BA组(bᵢ<aᵢ) |
| 排序 | `sort(d, n)` | 所有作业按 key 非减排序 |
| 分组放置 | AB→`c[j++]`, BA→`c[k--]` | AB 从前往后放，BA 从后往前放 |
| 计算完工时间 | 累加 M₁ 时间，更新 M₂ 完成时间 | 模拟流水线执行过程 |

### 为什么统一排序后分前后放就对了？

排序后所有作业按 min(aᵢ, bᵢ) 从小到大排列：
- AB 组（aᵢ ≤ bᵢ）的 key = aᵢ → 从前往后放 = 按 aᵢ 非减排列 ✓
- BA 组（bᵢ < aᵢ）的 key = bᵢ → 从后往前放 = 按 bᵢ 非增排列 ✓

> ✅ 一次排序 + 双指针分组，巧妙地同时实现了 Johnson 算法的三个步骤。

### 计算完工时间的代码对照

```c
j += a[c[i]];                          // M₁ 累计：j 是 M₁ 做完第 i 个作业的时刻
k = (j < k) ? k + b[c[i]] : j + b[c[i]];  // M₂ 完成时刻
```

- `j < k`：M₁ 做完了但 M₂ 还在忙（之前的作业没做完）→ M₂ 做完上一个才做这个 → k + b\[c\[i]]
- `j >= k`：M₂ 空闲了等 M₁ → M₂ 从 j 时刻开始做 → j + b\[c\[i]]

## 第55页：5. 计算复杂性分析

### 时间复杂度

算法的主要计算时间是**排序**，因此，最坏情况下算法所需的时间复杂度为：

> ✅ **O(n log n)**

### 各步骤时间

| 步骤 | 操作 | 时间 |
|------|------|------|
| 构造 key 和分组标记 | 遍历 n 个作业 | O(n) |
| 排序 | 按 key 排序 | O(n log n) |
| 分组放入 c\[] | 遍历 n 个作业 | O(n) |
| 计算完工时间 | 遍历 n 个作业 | O(n) |
| **总计** | | **O(n log n)** |

### 与直接 DP 对比

| 方法 | 时间复杂度 |
|------|-----------|
| DP（子集状态） | O(2ⁿ) |
| **Johnson 算法** | **O(n log n)** |

> 从指数级 O(2ⁿ) 降到 O(n log n)，这是因为 Johnson 法则把"在所有排列中找最优"变成了"按规则排序"——利用了问题的特殊结构。

## 总结

```
Johnson 算法完整流程：

输入：n 个作业的 aᵢ, bᵢ
  │
  ▼
Step 1: 分组
  AB = {i | aᵢ ≤ bᵢ}    （M₁快的，先做）
  BA = {i | bᵢ < aᵢ}     （M₂快的，后做）
  │
  ▼
Step 2: 排序
  AB 按 aᵢ 非减排列
  BA 按 bᵢ 非增排列
  │
  ▼                          实现技巧：
Step 3: 拼接                  key = min(aᵢ,bᵢ)
  最优调度 = AB + BA          统一排序后双指针分组
  │
  ▼
输出：最优调度顺序 c[] 和最优完工时间

时间复杂度：O(n log n)（瓶颈在排序）
```
