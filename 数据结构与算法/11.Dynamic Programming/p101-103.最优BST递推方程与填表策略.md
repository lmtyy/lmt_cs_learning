# 最优BST递推方程与填表策略（p101-103）

本节建立最优 BST 的核心递推方程 m(i,j)，并说明自底向上的计算顺序。

---

## 第101页：5. 递归计算最优值

### 代价公式展开

最优二叉搜索树 Tᵢⱼ 的平均路长为 pᵢⱼ，则所求的最优值为 p_{1,n}。

由二叉树的花费公式（选 xₖ 为根）：

$$
    w_{ij} p_{ij} = w_{i,k-1}(p_{i,k-1} + 1) + w_{k,k} + w_{k+1,j}(p_{k+1,j} + 1)
$$

w_ij代表选中点对应数值位于结点i数值与结点j数值间的概率（权重），p_ij代表结点i到结点j的区间内（以xk为根的平均路长）。wij\*pij=左子树权重+左子树平均路长（xk为根结点算）+根节点权重\*根节点平均路长（1）+右子树权重\*右子树平均路长（xk为根结点算）。由于结点编号按顺序，故i~k-1为左子树结点的编号。pi,k-1是以左子树平均路长（以左子树根节点为起点算），+1是因为左子树所有节点在以xk为根节点的树中的路长=自身到左子树根节点的路长(pi,k-1)+xk到左子树根节点路长(1)

展开得：
$$
    w_{ij} p_{ij} = w_{ij} + w_{i,k-1} p_{i,k-1} + w_{k+1,j} p_{k+1,j} \quad (i \le j)
$$

### 递推方程（以 wᵢⱼpᵢⱼ 为目标）

根据最优二叉搜索树问题的最优子结构性质，可建立计算 pᵢⱼ 的递归式：

$$
w_{ij} p_{ij} = w_{ij} + \min_{i \le k \le j} \{w_{i,k-1} p_{i,k-1} + w_{k+1,j} p_{k+1,j}\}, \quad i \le j
$$

遍历一遍，考虑所有i到j间某结点作为根节点的情况，取全局最小值

**初始条件**：
$$
    p_{i,i-1} = 0, \quad 1 \le i \le n
$$

> 含义：空树（没有关键字）的平均路长为 0。考虑只有根节点xi的树，其子树按上面的推导，左子树为pi,i-1，右子树为pi+1,i，后面都比前面多1，但所谓的子树应为空树，所以规定pi,i-1=0

## 第102页：定义 m(i,j) 简化递推

### 记号变换

记 wᵢⱼ · pᵢⱼ 为 **m(i, j)**。

则 m(1, n) = w\_{1,n} · p\_{1,n} = p\_{1,n}（因为 w_{1,n} = 1，所有概率之和为1）为所求最优值。

### 最终递推公式

$$
    m(i, j) = w_{ij} + \min_{i \le k \le j} \{m(i, k-1) + m(k+1, j)\}, \quad i \le j
$$

$$
    m(i, i-1) = 0, \quad i = 1, 2, \ldots, n
$$

### 公式解读

| 项 | 含义 |
|---|------|
| wᵢⱼ | 子树权重（所有结点深度+1 带来的代价增量） |
| m(i, k-1) | 左子树（关键字 i ~ k-1）的最优代价 |
| m(k+1, j) | 右子树（关键字 k+1 ~ j）的最优代价 |
| min over k | 枚举所有可能的根 k，取最小值 |

> ✅ 这个递推公式和矩阵连乘非常相似！区别是：
> - 矩阵连乘：m\[i]\[j] = min{m\[i]\[k] + m\[k+1]\[j] + **乘法代价**}
> - 最优BST：m(i,j) = min{m(i,k-1) + m(k+1,j)} + **wᵢⱼ**

## 第103页：自底向上的计算顺序

### 依赖关系

根据该公式，计算树 T\[i]\[j] 的花费只用到了 T\[i]\[k-1] 和 T\[k+1]\[j]，即更小规模的子树。

### 按内部结点个数递增计算

**1）构造只有 1 个内部结点的最优二叉搜索树**

T\[1]\[1], T\[2]\[2], ..., T\[n]\[n]

此时只有一个关键字 xᵢ，根必定是 xᵢ 本身：

$$
    m(i, i) = w_{ii} + m(i, i-1) + m(i+1, i) = w_{ii} + 0 + 0 = w_{ii}
$$

其中 wᵢᵢ = a_{i-1} + bᵢ + aᵢ，wii代表结点i的权重，a\_{i-1}代表未命中且命中点>i-1的概率（即i-1~i），bi代表命中点i的概率，ai代表i~i+1的概率

同时记录根结点：s\[1]\[1]=1, s\[2]\[2]=2, ..., s\[n]\[n]=n

**2）构造具有 2 个内部结点的最优二叉搜索树**

T\[1]\[2], T\[2]\[3], ..., T\[n-1]\[n]

对每个 m(i, i+1)，枚举 k = i 或 k = i+1，取较小者。

**3）逐步扩大...**

直到构造 T\[1]\[n]，即原问题。

### 与矩阵连乘的对比

| | 矩阵连乘 | 最优BST |
|--|---------|--------|
| 子问题 | m\[i]\[j] = A\[i:j] 的最优乘法次数 | m(i,j) = 关键字 i~j 的最优搜索代价 |
| 递推 | min{m\[i]\[k]+m\[k+1]\[j]+p_{i-1}p_kp_j} | min{m(i,k-1)+m(k+1,j)} + wᵢⱼ |
| 断开位置 | k 在 i~j-1 之间 | k 在 i~j 之间（k是根） |
| 填表顺序 | 按链长度递增 | 按内部结点个数递增 |
| 额外代价项 | p_{i-1}·pₖ·pⱼ（随 k 变化） | wᵢⱼ（与 k 无关，提到 min 外面） |

> ✅ wᵢⱼ 与 k 的选择无关，所以可以提到 min 外面。这是与矩阵连乘公式结构上的一个区别。

---

## 总结

```
最优 BST 递推方程：

  m(i, j) = wᵢⱼ + min { m(i, k-1) + m(k+1, j) }
                    i≤k≤j

  m(i, i-1) = 0（空树代价为0）

填表顺序（按子树大小递增）：
  大小1: m(1,1), m(2,2), ..., m(n,n)
  大小2: m(1,2), m(2,3), ..., m(n-1,n)
  大小3: m(1,3), m(2,4), ..., m(n-2,n)
  ...
  大小n: m(1,n) ← 最终答案

同时记录 s[i][j] = 最优根 k 的位置（用于构造最优BST）
```
