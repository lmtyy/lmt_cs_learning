## 第77页：1. 二叉搜索树（BST）定义

### 性质

二叉搜索树（Binary Search Tree）是一棵空树或者满足以下性质：

- 每个结点作为搜索对象，它的关键字是**互不相同**的
- 对于树上的所有结点，如果它有**左子树**，那么左子树上所有结点的关键字都**小于**该结点的关键字
- 对于树上的所有结点，如果它有**右子树**，那么右子树上所有结点的关键字都**大于**该结点的关键字

> ✅ 简记：**左 < 根 < 右**，且对所有子树递归成立。

## 第78页：BST 示例与搜索过程

### 示例（英文单词按字典序）

```
            xal
           /   \
        wan     zol
       /   \    /  \
     wen   wil yo  zom
          /    /  \
        wim  xul  yum
        /    /    /  \
      wul  xem  yon   zi
```

### 搜索过程

从根结点开始：
1. 如果根为空，则搜索**不成功**
2. 否则使用待搜索值与根结点比较：
   - 等于根结点关键字 → 搜索**成功**返回
   - 小于根结点 → 向**左子树**搜索
   - 大于根结点 → 向**右子树**搜索

> 每次比较排除一棵子树，搜索代价 = 该结点在树中的**深度 + 1**（即从根到该结点的路径长度 + 1）。

## 第79页：同一关键字集合的不同 BST

### 关键字集合

Name: 1=for, 2=if, 3=loop, 4=repeat, 5=while

### 两棵不同的 BST

**树 a：**
```
        if
       /  \
     for  while
          /
       repeat
       /
     loop
```

**树 b：**
```
        if
       /  \
     for  repeat
          /    \
        loop  while
```

> 🔑 对于同一个给定的关键字集合，可能有若干不同的二分搜索树。不同的树结构导致不同的搜索效率。

## 第80页：不同 BST 的性能对比

### 最坏情况比较次数

| 树 | 最大深度 | 最坏比较次数 |
|---|--------|-----------|
| 树 a | 4（loop在第4层） | **4 次** |
| 树 b | 3（loop/while在第3层） | **3 次** |

### 平均比较次数（等概率搜索）

假设只做成功的检索，且检索每个标识符的概率相同（各 1/5）：

**树 a**：各结点深度
- if: 深度0 → 1次
- for: 深度1 → 2次
- while: 深度1 → 2次
- repeat: 深度2 → 3次
- loop: 深度3 → 4次
- 平均 = (1+2+2+3+4)/5 = **12/5 = 2.4 次**

**树 b**：各结点深度
- if: 深度0 → 1次
- for: 深度1 → 2次
- repeat: 深度1 → 2次
- loop: 深度2 → 3次
- while: 深度2 → 3次
- 平均 = (1+2+2+3+3)/5 = **11/5 = 2.2 次**

> ✅ 构造不同的二叉搜索树就有不同的性能特征。树 b 无论最坏还是平均都优于树 a。

> 类比：BST 就像字典的索引结构。同一本字典的索引可以有不同的分层方式，好的分层让你翻更少的页就能找到目标词条。

### 引出问题

> 如果我们知道每个关键字被搜索的概率（不一定相等），如何构造一棵**平均搜索代价最小**的 BST？这就是最优二叉搜索树问题。
