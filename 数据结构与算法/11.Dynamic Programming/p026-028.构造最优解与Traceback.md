# 构造最优解与Traceback（p026-028）

本节是矩阵连乘DP的第4步：利用填表过程中记录的 s[i][j] 数组，递归地还原出最优加括号方案。

---

## 第26页：步骤4 — 构造最优解

### 问题

上述 MatrixChain 算法只算出了最优连乘次序所用的数乘次数 **m\[1][n]**，并未直接给出最优连乘次序（即完全加括号方法）。

### 关键信息：s\[i][j]

但二维数组 **s\[i][j]** 已经记录了足够的信息：

> ✅ s\[i][j] = k 说明：计算连乘积 A[i:j] 的最佳方式应该在矩阵 Aₖ 和 Aₖ₊₁ 之间断开，即最优加括号方式为 **(A[i:k])(A[k+1:j])**

> 类比：s[][] 就像一本"决策日志"，填表时每走一步都记下"在哪里切最好"，回头只要翻日志就能还原整个最优方案。

---

## 第27页：s[][] 表实例（6个矩阵）

### s\[i][j] 表

| i\j | 1 | 2 | 3 | 4 | 5 | 6 |
|-----|---|---|---|---|---|---|
| 1 | 0 | 1 | 1 | 3 | 3 | 3 |
| 2 | | 0 | 2 | 3 | 3 | 3 |
| 3 | | | 0 | 3 | 3 | 3 |
| 4 | | | | 0 | 4 | 5 |
| 5 | | | | | 0 | 5 |
| 6 | | | | | | 0 |

### 回溯过程（以 A[1:6] 为例）

```
s[1][6] = 3  →  (A[1:3])(A[4:6])

左半 A[1:3]:
  s[1][3] = 1  →  (A₁)(A[2:3])
  s[2][3] = 2  →  (A₂)(A₃)
  ∴ A[1:3] = A₁(A₂A₃)

右半 A[4:6]:
  s[4][6] = 5  →  (A[4:5])(A₆)
  s[4][5] = 4  →  (A₄)(A₅)
  ∴ A[4:6] = (A₄A₅)A₆

最终结果：(A₁(A₂A₃))((A₄A₅)A₆)
```

> ✅ 整个回溯就是一棵二叉树的前序遍历：每次查 s\[i][j] 确定根（断开点），然后递归左子树和右子树。

## 第28页：Traceback 代码

```c
void Traceback(int i, int j, int **s)
{
    if (i == j) return;
    Traceback(i, s[i][j], s);         // 递归左半：A[i:k]
    Traceback(s[i][j]+1, j, s);       // 递归右半：A[k+1:j]
    cout << "Multiply A" << i << "," << s[i][j];
    cout << " and A" << (s[i][j]+1) << "," << j << endl;
}
```

调用 **Traceback(1, n, s)** 即可得到 A[1:n] 的最优计算次序。

### 执行流程（n=6）

```
Traceback(1,6)          s[1][6]=3
├─ Traceback(1,3)       s[1][3]=1
│  ├─ Traceback(1,1)    return
│  └─ Traceback(2,3)    s[2][3]=2
│     ├─ Traceback(2,2) return
│     └─ Traceback(3,3) return
│     输出: Multiply A2,2 and A3,3
│  输出: Multiply A1,1 and A2,3
├─ Traceback(4,6)       s[4][6]=5
│  ├─ Traceback(4,5)    s[4][5]=4
│  │  ├─ Traceback(4,4) return
│  │  └─ Traceback(5,5) return
│  │  输出: Multiply A4,4 and A5,5
│  └─ Traceback(6,6)    return
│  输出: Multiply A4,5 and A6,6
输出: Multiply A1,3 and A4,6
```

> 输出顺序正好是**实际的乘法执行顺序**（后序遍历：先算子问题，再合并）。

## 总结

```
矩阵连乘 DP 完整四步：

Step 1: 分析最优子结构 → 最优解包含子问题的最优解
Step 2: 建立递归关系   → m[i][j] = min{m[i][k]+m[k+1][j]+p_{i-1}p_k p_j}
Step 3: 自底向上填表   → O(n³) 时间，O(n²) 空间
Step 4: 构造最优解     → 利用 s[i][j] 递归回溯，Traceback

          m[][] 表           s[][] 表
        ┌──────────┐       ┌──────────┐
        │ 最优代价  │       │ 断开位置  │
        │ (数值)    │       │ (决策)    │
        └──────────┘       └──────────┘
              ↓                   ↓
         答案是多少？         怎么做到的？
```
