# 备忘录方法Memoization（p034-038）

本节介绍动态规划的第三个基本要素——备忘录方法（Memoization），它是介于朴素递归和自底向上DP之间的折中方案：保留递归的自顶向下结构，同时用表存储避免重复计算。

---

## 第34页：3. 备忘录方法（Memoization）

### 核心思想

- 用一个表格来保存已解决的子问题的答案，用的时候查表即可
- 采用的递归方式是**自顶向下**，动态规划是**自底向上**
- 初始化为每个子问题的记录存入一个**特殊的值**（如 0），表示并未求解。在求解过程中，查看相应记录：如果是特殊值，表示未求解；否则只要取出该子问题的解答即可

### 与标准DP的对比

| | 自底向上 DP | 备忘录（Memoization） |
|--|-----------|-------------------|
| 方向 | 从小问题到大问题 | 从大问题递归到小问题 |
| 计算顺序 | 预先确定（按链长度等） | 按需计算（递归调用时才算） |
| 是否存表 | ✅ | ✅ |
| 避免重复 | ✅ | ✅ |

> 类比：自底向上 DP 像按课程表从第一节课开始上到最后一节；备忘录像做卷子时遇到不会的先跳过，需要用到某个知识点时再去翻书学，学完记下来下次直接用。

## 第35页：MemoizedMatrixChain 代码（初始化）

```c
int MemoizedMatrixChain(int n, int **m, int **s)
{
    for (int i = 1; i <= n; i++)
        for (int j = i; j <= n; j++)
            m[i][j] = 0;       // 初始化为0，表示未求解
    return LookupChain(1, n);   // 自顶向下从原问题开始
}
```

### 代码逐行对照

| 行 | 作用 |
|---|------|
| 双重循环 `m[i][j] = 0` | 将所有子问题标记为"未求解"（0是特殊值，因为真实最优值不可能为0，至少需要一次乘法） |
| `return LookupChain(1, n)` | 从原问题 A\[1:n] 开始自顶向下递归求解 |

> 🔑 注意：这里用 0 作为"未求解"标记，是因为矩阵连乘中只要 i < j，m\[i]\[j] 必然大于 0。对于其他问题可能需要用 -1 或 ∞ 作为标记。

## 第36页：LookupChain 代码（核心递归）

```c
int LookupChain(int i, int j)
{
    if (m[i][j] > 0) return m[i][j];   // 已求解，直接返回
    if (i == j) return 0;               // 基准：单个矩阵
    
    // 递归求解断开位置为 i 时的计算量（作为初始值）
    int u = LookupChain(i, i) + LookupChain(i+1, j)
            + p[i-1] * p[i] * p[j];
    s[i][j] = i;
    
    // 循环每个可取断开位置，求计算量
    for (int k = i+1; k < j; k++)
    {
        int t = LookupChain(i, k) + LookupChain(k+1, j)
                + p[i-1] * p[k] * p[j];
        if (t < u) {
            u = t;
            s[i][j] = k;
        }
    }
    m[i][j] = u;   // 存入表中
    return u;
}
```

### 代码逐行对照

| 代码行 | 作用 | 对应思想 |
|--------|------|---------|
| `if (m[i][j] > 0) return m[i][j]` | 查表：如果已经算过，直接返回 | **避免重复计算的核心** |
| `if (i == j) return 0` | 基准情况 | 单个矩阵无需乘法 |
| `u = LookupChain(i,i) + LookupChain(i+1,j) + ...` | 取 k=i 时的值作为初始最优值 | 默认断开位置 |
| `for (k = i+1; k < j; k++)` | 枚举所有其他断开位置 | 穷尽所有可能 |
| `m[i][j] = u` | 存表：将结果记录下来 | 供后续查表使用 |

> 🔑 与朴素递归 RecurMatrixChain 的**唯一区别**：开头多了一行 `if (m[i][j] > 0) return m[i][j]`，以及结尾多了一行 `m[i][j] = u`。就这两行把指数级变成了多项式级。

### 作业：利用备忘录方法计算 A\[1:4] 时都计算了哪些子问题？

调用 `LookupChain(1, 4)` 的递归展开：

```
LookupChain(1,4)  ← 未求解，需计算
├─ k=1: LookupChain(1,1)=0, LookupChain(2,4) ← 未求解
│        └─ LookupChain(2,4)
│           ├─ k=2: LookupChain(2,2)=0, LookupChain(3,4) ← 未求解
│           │        └─ LookupChain(3,4)
│           │           ├─ k=3: LookupChain(3,3)=0, LookupChain(4,4)=0
│           │           └─ 存入 m[3][4]，返回
│           ├─ k=3: LookupChain(2,3) ← 未求解
│           │        └─ LookupChain(2,3)
│           │           ├─ k=2: LookupChain(2,2)=0, LookupChain(3,3)=0
│           │           └─ 存入 m[2][3]，返回
│           └─ 存入 m[2][4]，返回
├─ k=2: LookupChain(1,2) ← 未求解
│        └─ LookupChain(1,2)
│           ├─ k=1: LookupChain(1,1)=0, LookupChain(2,2)=0
│           └─ 存入 m[1][2]，返回
│        LookupChain(3,4) ← 已求解！直接返回 m[3][4]
├─ k=3: LookupChain(1,3) ← 未求解
│        └─ LookupChain(1,3)
│           ├─ k=1: LookupChain(1,1)=0, LookupChain(2,3) ← 已求解！
│           ├─ k=2: LookupChain(1,2) ← 已求解！LookupChain(3,3)=0
│           └─ 存入 m[1][3]，返回
│        LookupChain(4,4)=0
└─ 存入 m[1][4]，返回
```

实际计算的子问题：m\[3]\[4], m\[2]\[3], m\[2]\[4], m\[1]\[2], m\[1]\[3], m\[1]\[4]（共6个非平凡子问题）

> ✅ 关键观察：m\[3]\[4] 和 m\[2]\[3] 第二次被用到时直接查表返回，没有重复计算。

## 第37页：备忘录方法的复杂度

### 时间复杂度分析

- 备忘录采用**自顶向下**的计算，将计算结果存入矩阵后返回
- 每个子问题仅在**第一次被调用时**计算
- m\[i]\[j] 共有 **O(n²)** 个备忘记录项 //m为n\*n矩阵共n^2个元素
- 初始化需要 O(n²) 时间
- 每个记录项只填入一次，每次填入时需枚举 k，共耗费 **O(n)** 时间 //n个矩阵连乘，最浅的一层分界点要遍历n-1次

> ✅ 所以备忘录方法耗时 **O(n³)**，与自底向上 DP 相同。

详细证明：
### 📐 精确的数学推导（证明总枚举次数是 \(O(n^3)\)）

假设总共有 \(n\) 个矩阵。
在 DP 填表过程中：
*   对于一个**长度为 \(L\)** 的区间（包含 \(L\) 个矩阵，即 \(j-i+1 = L\)），在二维表中有 **\(n-L+1\)** 个这样的状态。
*   每一个长度为 \(L\) 的区间，为了求它的最优解，枚举分界点 \(k\) 的次数是 **\(L-1\)** 次。

我们把所有长度区间的枚举次数加起来，计算**总循环次数 \(S\)**：

$$
 S = \sum_{L=2}^{n} \underbrace{(n - L + 1)}_{\text{长度为L的区间个数}} \times \underbrace{(L - 1)}_{\text{该区间枚举k的次数}} 
$$
为了方便计算，令 \(t = L-1\)，那么 \(t\) 的取值范围从 \(1\) 到 \(n-1\)。
此时 \(n-L+1 = n - t\)。代入得：

$$
 S = \sum_{t=1}^{n-1} (n - t) \times t 
$$
将其拆开：

$$
S = \sum_{t=1}^{n-1} (nt - t^2) = n \sum_{t=1}^{n-1} t - \sum_{t=1}^{n-1} t^2
$$

代入经典的求和公式
$$
\sum_{t=1}^{n-1} t = \frac{n(n-1)}{2},
	\sum_{t=1}^{n-1} t^2 = \frac{(n-1)n(2n-1)}{6}
$$

：

$$
 S = n \times \frac{n(n-1)}{2} - \frac{(n-1)n(2n-1)}{6} 
$$

提取公因式 \(\frac{n(n-1)}{6}\)：

$$
 S = \frac{n(n-1)}{6} \times [3n - (2n-1)] 
$$

$$
 S = \frac{n(n-1)}{6} \times (n+1) 
$$

最终得出严格精确的枚举次数：
$$
 S = \frac{n(n-1)(n+1)}{6} = \frac{n^3 - n}{6}
$$

### 💡 为什么最终时间复杂度依然严格是 \(O(n^3)\)？
因为 \(S = \frac{n^3 - n}{6}\)，这个式子展开后，**最高阶项是 \(\frac{n^3}{6}\)**。
在计算机科学的大 \(O\) 复杂度分析中，**常数系数 \(1/6\) 被直接忽略，低阶项 \(-n\) 也**被忽略。
因此，虽然是 \(\frac{n^3}{6}\)，但最终的结论是 **\(\Theta(n^3)\)**。

## 第38页：DP 与备忘录的适用场景

### 选 DP（自底向上）的情况

> 当一个问题的所有子问题都**至少要解一次**时，则用动态规划算法比备忘录方法好。

原因：DP 没有递归调用的开销，循环的常数因子更小。

### 选备忘录（自顶向下）的情况

> 当子问题空间中的**部分子问题可不必求解**时，用备忘录方法则较有利，因为从其控制结构可以看出，该方法只解哪些确实需要求解的子问题。

| 场景 | 推荐方法 | 原因 |
|------|---------|------|
| 所有子问题都要算 | 自底向上 DP | 无递归开销，常数更优 |
| 部分子问题不需要算 | 备忘录 | 按需计算，跳过不必要的子问题 |

> 类比：如果考试范围覆盖全部章节 → 老老实实从头学到尾（DP）。如果考试只考重点 → 先看大题考什么，缺什么补什么（备忘录）。

```
DP 三种实现策略对比：

┌───────────────┬───────────────┬───────────────┐

  所有子问题都要解 → DP（省去递归开销）

```
