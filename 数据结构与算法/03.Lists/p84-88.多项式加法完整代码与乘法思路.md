## 第一页（P84）—— 加法函数的初始化部分

### 函数签名

```c
void AddPolyn(polynomial &Pa, polynomial &Pb)
```

- `Pa`、`Pb` 是两个多项式链表的引用
- 结果存入 Pa（就地修改），Pb 被消耗掉

### 初始化代码

```c
ha = GetHead(Pa);       // ha = Pa 的头节点
hb = GetHead(Pb);       // hb = Pb 的头节点
qa = NextPos(Pa, ha);   // qa = Pa 的第一个数据节点
qb = NextPos(Pb, hb);   // qb = Pb 的第一个数据节点
while (qa && qb)        // 当两个链表都还有未处理的项时
{
    // Polynomial qa and qb are both not empty
    a = GetCurElem(qa);  // a = qa 节点的数据（包含 coef 和 expn）
    b = GetCurElem(qb);  // b = qb 节点的数据
```

### 辅助函数说明

| 函数               | 含义                                        |
| ------------------ | ------------------------------------------- |
| `GetHead(P)`       | 获取链表 P 的头节点                         |
| `NextPos(P, node)` | 获取 node 在链表 P 中的下一个节点           |
| `GetCurElem(node)` | 获取 node 节点存储的数据（termtype 结构体） |

### 变量角色

- `ha`：Pa 链表中当前处理位置的前驱（用于插入/删除操作）
- `hb`：Pb 链表中当前处理位置的前驱
- `qa`：Pa 中当前正在比较的节点
- `qb`：Pb 中当前正在比较的节点
- `a`、`b`：当前两个节点的数据（系数+指数）

### while 循环的含义

`while(qa && qb)` —— 只要两个多项式都还有未处理的项，就继续比较。当任一方遍历完毕时退出循环，剩余部分单独处理。

## 第二页（P85）—— case -1 和 case 0（指数比较）

### switch 语句

```c
switch (*cmp(a, b))
```

`cmp(a, b)` 比较 a 和 b 的指数：
- 返回 -1：a 的指数 < b 的指数
- 返回 0：a 的指数 == b 的指数
- 返回 1：a 的指数 > b 的指数

### case -1：Pa 的指数更小

```c
case -1:
    ha = qa;              // ha 前进到 qa（qa 保留在结果中）
    qa = NextPos(Pa, qa); // qa 移动到下一个节点
    break;
```

**含义：** Pa 当前项的指数比 Pb 小，说明这一项在 Pb 中没有同类项，直接保留在结果链表中不做任何修改。ha 和 qa 都前进一步。

**示例：** Pa 有 (7,0)，Pb 当前是 (8,1)。指数 0 < 1，所以 (7,0) 直接保留。

### case 0：指数相同（同类项）

```c
case 0:
    sum = a.coef + b.coef;           // 系数相加
    if (abs(sum - 0.0) >= 1E-6)      // 判断 sum 是否不为零
    {
        SetCurElem(qa, sum);         // 系数不为零，更新 qa 的系数
    }
```

**含义：** 两项指数相同，合并同类项。系数相加后：
- 如果结果不为零（`abs(sum) >= 1E-6`），更新 Pa 中该节点的系数
- 如果结果为零，需要删除该节点（见下一页）

**为什么用 `abs(sum - 0.0) >= 1E-6` 而不是 `sum != 0`？**

因为 `coef` 是 `float` 类型，浮点数比较不能直接用 `==`。由于浮点精度问题，两个浮点数相加的结果可能是一个极小的数（如 0.0000000001）而不是精确的 0。所以用一个很小的阈值（1E-6）来判断"是否足够接近零"。

## 第三页（P86）—— case 0 的 else 分支（系数为零时删除）

### 代码

```c
    else    // sum == 0（系数相加为零）
    {
        DelFirst(ha, qa);     // 从 Pa 中删除 qa 节点
        FreeNode(qa);         // 释放 qa 的内存
        qa = NextPos(Pa, qa); // qa 移动到下一个（注意：这里应该是 NextPos(Pa, ha)）
    }
    DelFirst(hb, qb);        // 从 Pb 中删除 qb 节点
    FreeNode(qb);            // 释放 qb 的内存
    qb = NextPos(Pb, hb);    // qb 移动到 Pb 的下一个节点
    break;
```

### 逐行解析

**系数为零时（else 分支）：**

- `DelFirst(ha, qa)`：从 Pa 链表中把 qa 节点摘除（ha 是 qa 的前驱，ha->next 跳过 qa）
- `FreeNode(qa)`：释放 qa 节点的内存
- `qa = NextPos(Pa, ha)`：qa 更新为 ha 的下一个节点（即原来 qa 的后继）

**无论系数是否为零，Pb 的当前节点都要处理：**

- `DelFirst(hb, qb)`：从 Pb 中摘除 qb
- `FreeNode(qb)`：释放 qb
- `qb = NextPos(Pb, hb)`：qb 前进

**为什么 Pb 的节点总是要删除？**

因为结果存在 Pa 中。Pb 的节点要么已经合并到 Pa 的对应节点中（系数相加），要么两者抵消为零。无论哪种情况，Pb 的这个节点都不再需要了。

### 示例

Pa 有 (9,8)，Pb 有 (-9,8)：
- 指数相同，case 0
- sum = 9 + (-9) = 0
- 进入 else：从 Pa 删除 (9,8)，释放；从 Pb 删除 (-9,8)，释放
- 结果链表中不保留指数为 8 的项

## 第四页（P87）—— case 1（Pb 的指数更小）+ 循环结束处理

### case 1：Pb 的指数更小

```c
case 1:
    DelFirst(hb, qb);       // 从 Pb 中摘除 qb
    InsFirst(ha, qb);       // 把 qb 插入到 Pa 中 ha 之后
    qb = NextPos(Pb, hb);   // qb 前进到 Pb 的下一个节点
    ha = NextPos(Pa, ha);   // ha 前进（因为刚插入了一个节点）
    break;
```

**含义：** Pb 当前项的指数比 Pa 小，说明这一项在 Pa 中没有对应项，需要把它从 Pb 中取出插入到 Pa 的正确位置。

**操作过程：**
1. 从 Pb 中摘下 qb 节点
2. 把 qb 插入到 Pa 中 ha 的后面（ha 是当前位置的前驱，插入后保持指数有序）
3. qb 前进到 Pb 的下一个待处理节点
4. ha 前进（因为新插入的节点现在在 ha 后面）

**示例：** Pa 当前是 (9,8)，Pb 当前是 (22,7)。指数 8 > 7，case 1。把 (22,7) 从 Pb 摘下，插入到 Pa 中 (3,1) 和 (9,8) 之间。

### 循环结束后的处理

```c
} // end of while

if (ListEmpty(Pa))       // 如果 Pa 已经遍历完（没有剩余项了）
    Append(Pa, qb);      // 把 Pb 剩余的所有项接到 Pa 尾部

FreeNode(hb);            // 释放 Pb 的头节点
}
```

**`if(ListEmpty(Pa))`：**

当 while 循环因为 `qa == NULL` 退出时，说明 Pa 的所有项都已处理完，但 Pb 可能还有剩余项。这些剩余项的指数都比 Pa 中所有项大，直接追加到 Pa 尾部即可。

注意：如果是 `qb == NULL` 导致退出（Pb 先遍历完），Pa 的剩余项本来就在 Pa 中，不需要额外操作。

**`FreeNode(hb)`：**

Pb 的所有数据节点要么已经被插入到 Pa 中，要么已经被释放。最后释放 Pb 的头节点，完成清理。

## 第五页（P88）—— 多项式乘法演示

### 核心难点

PPT指出：多项式乘法的唯一难点是——**乘法结果中会产生同类项，需要合并。**

### 两个多项式

**ah：** `7x⁰ + 3x¹ + 9x⁸ + 5x¹⁷`

**bh：** `8x¹ + 22x⁷ + (-9)x⁸`

### 乘法过程

每一项乘以另一个多项式的每一项：

ah 的第一项 (7,0) × bh 的所有项：
- 7×8 = 56，指数 0+1 = 1 → (56, 1)
- 7×22 = 154，指数 0+7 = 7 → (154, 7)（PPT中写的是(154,7)但图中可能有误差）
- 7×(-9) = -63，指数 0+8 = 8 → 但图中显示不同的值

ah 的第二项 (3,1) × bh 的所有项：
- 3×8 = 24，指数 1+1 = 2 → (24, 2)
- 3×22 = 66，指数 1+7 = 8
- ...

### 图示解读

PPT中的 **ch** 链表展示了部分乘法结果（合并同类项之前或之后的中间状态）：

```
ch: head(-1) → (56,1) → (24,2) → (154,7) → (3,8) → ...
                → (45,9) → (198,15) → ... → (-45,25) → NULL
```

这些项是 ah 的各项分别与 bh 的各项相乘后，按指数排序并合并同类项的结果。

### 乘法算法思路

```
对 ah 的每一项 ai：
    对 bh 的每一项 bj：
        新项 = (ai.coef × bj.coef, ai.expn + bj.expn)
        将新项插入结果链表 ch（按指数有序插入，遇到同类项则合并）
```

### 时间复杂度

设 ah 有 m 项，bh 有 n 项：
- 产生 m×n 个乘积项
- 每个乘积项需要插入到结果链表（最坏 O(m×n) 次比较找插入位置）
- 总体：O(m²×n²) 最坏情况

优化方法：先生成所有乘积项，排序后再合并同类项，可以降到 O(mn × log(mn))。

### 乘法与加法的关系

乘法可以分解为多次加法：

```
ah × bh = ah × (bh的第1项) + ah × (bh的第2项) + ah × (bh的第3项) + ...
```

每个 `ah × (bh的第i项)` 就是把 ah 的每一项系数乘以 bh 第 i 项的系数，指数加上 bh 第 i 项的指数，得到一个新多项式。然后把这些新多项式逐个相加（用前面的 AddPolyn）。

### 加法算法的完整流程图

```
开始
  ↓
初始化 ha, hb, qa, qb
  ↓
while (qa && qb)
  ↓
比较 qa 和 qb 的指数
  ├── qa.expn < qb.expn (case -1)
  │     → 保留 qa，ha=qa，qa前进
  │
  ├── qa.expn == qb.expn (case 0)
  │     → sum = qa.coef + qb.coef
  │     ├── sum ≠ 0 → 更新 qa 的系数，ha=qa，qa前进
  │     └── sum == 0 → 删除 qa，释放，qa=ha的下一个
  │     → 删除 qb，释放，qb前进
  │
  └── qa.expn > qb.expn (case 1)
        → 从 Pb 摘下 qb，插入 Pa 中 ha 之后
        → qb前进，ha前进
  ↓
循环结束
  ↓
Pa 空？→ 是：Append(Pa, qb 剩余部分)
  ↓
释放 Pb 头节点
  ↓
结束（结果在 Pa 中）
```

### 考试重点

1. **手动模拟加法过程**：给定两个多项式链表，逐步写出每次比较的结果、指针的移动、节点的插入/删除。这是最常见的大题。

2. **三种 case 的处理逻辑**：
   - case -1（Pa 指数小）：保留，前进
   - case 0（指数相同）：合并，可能删除
   - case 1（Pb 指数小）：从 Pb 摘下插入 Pa

3. **浮点数判零**：不能用 `sum == 0`，要用 `abs(sum) < 1E-6`。这是编程细节题。

4. **内存管理**：Pb 的节点要么被转移到 Pa，要么被释放。最后 Pb 的头节点也要释放。不能有内存泄漏。

5. **乘法的思路**：
   - 系数相乘，指数相加
   - 结果需要合并同类项
   - 可以转化为"多次加法"来实现

6. **时间复杂度**：
   - 加法：O(m + n)，m 和 n 是两个多项式的项数
   - 乘法：O(m × n × (m+n))（朴素方法），每个乘积项插入有序链表需要遍历
