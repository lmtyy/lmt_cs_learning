这两页PPT详细讲解了单链表（Single Linked List, SLL）中的另一个核心操作：**遍历（Traversing）**。遍历是指从链表的头到尾，依次访问每一个节点的过程。

以下是这两页PPT知识点的详细拆解：

### 第一页：遍历的概念与图解思考 (Traversing a SLL)

这页PPT通过图示和提问，引导思考遍历操作的本质和效率。

#### 1. 核心问题提出

- **What would be the code for traversing a SLL? (遍历单链表的代码该怎么写？)**

  这是为了引出下一页的具体代码实现。它提示我们需要一种机制，能够让一个指针从头结点开始，一步步向后移动，直到链表末尾。

- **What would be the runtime of this method? (这个方法的运行时间是多少？)**

  - **答案解析：** 遍历操作的时间复杂度是 **$O(N)$**。因为要访问链表中的每一个节点，如果链表有 $N$ 个数据节点，我们就必须执行 $N$ 次移动和访问操作。所需的时间与链表的长度成正比。

#### 2. 图解分析 (核心逻辑的可视化)

图示展示了一个带有头结点（Header）和三个数据节点（分别存有 "one", "two", "three"）的单链表，以及一个用于遍历的游标指针 **`curr`** (current 的缩写，代表“当前节点”)。

- **`curr` 指针的作用：** 它就像一个探照灯，照到哪个节点，我们就能读取哪个节点的数据。
- **移动轨迹（黑色实线箭头与橙色虚线箭头）：**
  1. **初始状态：** `curr` 首先需要指向第一个真正的数据节点。注意，橙色虚线显示了从 `Header` 的指针域指向 `curr`，意味着 `curr` 初始被赋值为 `Header->next`（即指向存有 "one" 的节点）。
  2. **向后移动：** 当读取完 "one" 后，`curr` 需要移动到下一个节点。图中的黑色实线箭头从 `curr` 指向 "one" 节点的指针域，然后橙色虚线再从那里指回 `curr`。这图解表达的核心代码逻辑是 **`curr = curr->next`**。即：把当前节点的“下一个节点的地址”赋给 `curr` 自己，这样 `curr` 就成功跳到了下一个节点（"two"）。
  3. **循环往复：** `curr` 会继续用同样的逻辑，跳到 "three"，直到最后跳出链表末尾（变成 `NULL`）。

### 第二页：遍历的代码实现与常见错误分析 (traversing a SLL - Implementation & Errors)

这页PPT给出了遍历链表打印数据的标准C语言代码，并非常贴心地列出了初学者最容易犯的两个逻辑错误。

#### 1. 标准正确代码实现 (左侧)

C

```c
void DisplayList_L(LinkList L)
/* traverse a linked list */
{
    p = L->next;  // 关键步骤 1：初始化游标指针
    while( p != NULL) // 关键步骤 2：设定循环终止条件
    {
        printf("%d", p->data); // 步骤 3：访问当前节点的数据
        p = p->next;           // 关键步骤 4：游标指针向后移动一步
    }
}
```

- **`p = L->next;` (红色字体强调)：** 为什么不是 `p = L`？因为 `L` 是头结点（Header），里面没有有效数据。我们需要从第一个**真实数据节点**开始遍历，所以 `p` 必须初始化为 `L->next`。
- **`while( p != NULL)`：** 只要 `p` 没有指空（说明当前还在链表的数据节点上），循环就继续。当 `p` 指向最后一个节点的 `next`（即 `NULL`）时，循环结束。
- **`p = p->next;`：** 这是链表遍历的灵魂语句。它让指针 `p` 顺着链条往下走。

#### 2. 常见错误分析 (Common error - 右侧绿色框)

PPT列出了两种经典的错误写法，这两者都会导致严重的运行时Bug。

**错误写法一：使用 `do-while` 循环且没有前置判空**

C

```c
do {
    printf("%d", p->data);
    p = p->next;
} while(p != NULL);
```

- **致命缺陷：** `do-while` 循环的特点是“先执行一次循环体，再判断条件”。如果传入的链表是一个**空链表**（即只有头结点，`L->next` 本身就是 `NULL`），那么 `p` 初始就是 `NULL`。程序会直接进入 `do` 代码块，尝试执行 `printf("%d", p->data)`。此时 `p` 是空指针，访问 `p->data` 会立即导致**段错误（Segmentation Fault）**，程序崩溃。

**错误写法二：在 `while` 循环中顺序写反**

C

```c
while(p != NULL) {
    p = p->next;
    printf("%d", p->data);
}
```

- **逻辑缺陷（Off-by-one error / 越界访问）：**

  假设 `p` 一开始正确指向了第一个数据节点（比如存的是 "1"）。

  进入循环后，**第一步发生的是 `p = p->next;`**，`p` 直接跳到了第二个节点（比如 "2"）。

  紧接着打印 `p->data`，屏幕上出现的是 "2"。

  **结果 1：** 第一个节点的数据 "1" 被完美错过了，永远没有被打印。

  **结果 2（更严重的崩溃）：** 当遍历到最后一个节点时，进入循环，执行 `p = p->next;` 后，`p` 变成了 `NULL`。紧接着执行 `printf` 时，又试图访问空指针 `p->data`，程序再次崩溃。

**总结：** 遍历链表的标准范式必须是：**先判断是否为空 -> 如果不空则访问当前节点数据 -> 然后让指针指向下一个节点**。左侧的代码完美地遵循了这个范式。
