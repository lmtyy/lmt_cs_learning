## 第一页（P81）—— 多项式的两种存储方式对比

### 背景：什么是多项式？

多项式如：`7x⁰ + 3x¹ + 9x⁸ + 5x¹⁷` = 7 + 3x + 9x⁸ + 5x¹⁷

每一项由两个信息组成：
- **系数（coefficient, coef）**：如 7、3、9、5
- **指数（exponent, expn）**：如 0、1、8、17

### 方式一：数组实现（右侧黄色框）

```c
#define MAX_TERMS 100

typedef struct {
    float coef;    // 系数
    int expn;      // 指数
} termtype;

typedef struct {
    int noelem;                  // 当前项数
    termType elem[MAX_TERMS];   // 固定大小的数组存储各项
} polynomial;
```

**特点：**
- 预分配固定大小的数组（MAX_TERMS = 100）
- 如果多项式只有 3 项，剩下 97 个位置浪费
- 如果多项式超过 100 项，数组不够用
- 插入/删除某一项需要移动元素

### 方式二：链表实现（左侧蓝色框）

```c
typedef struct termtype {
    float coef;              // 系数
    int expn;                // 指数
    struct termtype *next;   // 指向下一项的指针
} termtype;

typedef termtype *polynomial;  // 多项式就是一个链表的头指针
```

**特点：**
- **Unbounded（无界）**：项数不受限制，需要多少就分配多少
- **Convenient to insert or delete（插入删除方便）**：加减项只需改指针，不需要移动元素

### 为什么链表更适合多项式？

多项式运算（加法、乘法）经常需要：
- 动态增加新项（两个多项式相乘可能产生很多新项）
- 合并同类项后删除系数为 0 的项
- 项数不确定

这些特点正好匹配链表的优势。

## 第二页（P82）—— 链表实现的最终声明

### 代码

```c
typedef struct termtype {
    float coef;              // 系数
    int expn;                // 指数
    struct termtype *next;   // 指向下一项
} termtype;

typedef termtype *polynomial;  // polynomial 是指向 termtype 的指针
```

### 解读

- 每个节点代表多项式的一项
- `coef` 存系数（浮点数，因为系数可能是小数）
- `expn` 存指数（整数）
- `next` 指向下一项
- `polynomial` 类型就是链表头指针

### 多项式的链表表示约定

通常约定：
- 链表按**指数递增**（或递减）排列
- 只存储系数不为 0 的项（稀疏表示）
- 有一个头节点（expn 设为 -1 或其他特殊值作为标记）

例如多项式 `7 + 3x + 9x⁸ + 5x¹⁷`：

```
head(-1) → (7,0) → (3,1) → (9,8) → (5,17) → NULL
```

只有 4 个节点，而不是 18 个（如果用数组按下标存，需要 a[0]~a[17] 共 18 个位置，大部分为 0）。

### 稀疏多项式的优势

考虑多项式 `x¹⁰⁰⁰ + 1`：
- 数组实现：需要 1001 个位置，其中 999 个是 0（极度浪费）
- 链表实现：只需要 2 个节点 `(1,1000) → (1,0) → NULL`

## 第三页（P83）—— 多项式加法的链表演示

### 场景

两个多项式相加：

**ha（多项式A）：** `7x⁰ + 3x¹ + 9x⁸ + 5x¹⁷`

链表表示（按指数递增）：
```
ha: head(-1) → (7,0) → (3,1) → (9,8) → (5,17) → NULL
```

 `8x¹ + 22x⁷ + (-9)x⁸`

链表表示（按指数递增）：
```
hb: head(-1) → (8,1) → (22,7) → (-9,8) → NULL
```

### 加法结果

**ha + hb = ?**

逐项比较指数，合并同类项：

| ha 的项 | hb 的项 | 比较指数    | 操作                 | 结果项            |
| ------- | ------- | ----------- | -------------------- | ----------------- |
| (7, 0)  | (8, 1)  | 0 < 1       | ha 的项直接保留      | (7, 0)            |
| (3, 1)  | (8, 1)  | 1 == 1      | 同类项合并：3+8=11   | (11, 1)           |
| (9, 8)  | (22, 7) | 8 > 7       | hb 的项插入          | (22, 7)           |
| (9, 8)  | (-9, 8) | 8 == 8      | 同类项合并：9+(-9)=0 | 系数为0，删除该项 |
| (5, 17) | —       | hb 已遍历完 | ha 剩余项直接保留    | (5, 17)           |

最终结果：`7x⁰ + 11x¹ + 22x⁷ + 5x¹⁷`

```
ha(结果): head(-1) → (7,0) → (11,1) → (22,7) → (5,17) → NULL
```

### 图示解读

PPT中的第三行图展示了加法过程中的中间状态：

- 结果链表复用了 ha 的链表结构（就地修改，不额外分配新链表）
- `(7,0)` 保留不变
- `(3,1)` 被修改为 `(11,1)`（系数 3+8=11）
- `(22,7)` 从 hb 中"摘下来"插入到结果链表中
- `(9,8)` 和 `(-9,8)` 合并后系数为 0，两个节点都被删除（释放）
- `(5,17)` 保留不变

红色箭头表示指针的重新连接过程。

### 加法算法的核心逻辑

```c
// 伪代码：多项式加法 ha = ha + hb
void PolyAdd(polynomial ha, polynomial hb) {
    pa = ha->next;  // ha 的第一个数据项
    pb = hb->next;  // hb 的第一个数据项
    pre = ha;       // 结果链表的当前尾部

    while (pa != NULL && pb != NULL) {
        if (pa->expn < pb->expn) {
            // ha 的指数小，ha 的项直接保留
            pre = pa;
            pa = pa->next;
        }
        else if (pa->expn > pb->expn) {
            // hb 的指数小，把 hb 的项插入到结果链表
            temp = pb->next;
            pb->next = pa;
            pre->next = pb;
            pre = pb;
            pb = temp;
        }
        else {
            // 指数相同，合并同类项
            pa->coef += pb->coef;
            if (pa->coef == 0) {
                // 系数为0，删除该项
                pre->next = pa->next;
                free(pa);
                pa = pre->next;
            } else {
                pre = pa;
                pa = pa->next;
            }
            // 释放 hb 中的对应节点
            temp = pb->next;
            free(pb);
            pb = temp;
        }
    }

    // 如果 hb 还有剩余项，接到结果链表尾部
    if (pb != NULL) {
        pre->next = pb;
    }

    free(hb);  // 释放 hb 的头节点
}
```

### 算法要点

1. **双指针同步遍历**：pa 遍历 ha，pb 遍历 hb，根据指数大小决定操作
2. **三种情况**：
   - `pa->expn < pb->expn`：ha 的项指数小，直接保留，pa 前进
   - `pa->expn > pb->expn`：hb 的项指数小，从 hb 摘下插入结果链表，pb 前进
   - `pa->expn == pb->expn`：同类项合并，系数相加；若结果为 0 则删除
3. **就地修改**：结果存在 ha 中，hb 的节点要么被插入 ha，要么被释放
4. **时间复杂度**：O(m + n)，m 和 n 分别是两个多项式的项数

### 为什么链表适合这个操作？

- 合并时需要频繁插入和删除节点 → 链表 O(1)
- 两个多项式项数可能差异很大 → 链表不浪费空间
- 结果多项式项数不确定 → 链表动态增长

如果用数组实现，合并同类项后删除某一项需要移动后续所有元素，效率低下。
