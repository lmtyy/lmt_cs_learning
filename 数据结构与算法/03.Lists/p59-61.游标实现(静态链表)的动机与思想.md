这三页PPT引入了一个新主题：**静态链表 / 游标实现（Cursor Implementation）**。这是在没有指针的编程语言中模拟链表行为的方法。

---

## 第一页（P59）—— 为什么需要游标实现？

### 背景问题

很多早期编程语言（如 BASIC、FORTRAN）**不支持指针**。

- 没有 `struct Node *next` 这种语法
- 没有 `malloc` / `free` 这种动态内存分配机制
- 变量只能是基本类型（整数、浮点数）或数组

但这些语言的程序员有时也需要链表的功能（动态插入删除、不需要移动元素）。怎么办？

### 解决方案：游标实现（Cursor Implementation）

用**数组 + 整数下标**来模拟指针和动态内存分配。

核心思想：
- 用数组代替堆内存
- 用数组下标（整数）代替指针
- 用一个"空闲链表"代替 malloc/free

这种方法也叫**静态链表**，因为底层存储是静态分配的数组，但逻辑上表现得像链表。

### 现代意义

虽然现代语言（C、C++、Java、Python）都支持指针或引用，但游标实现在某些场景下仍然有用：

- **嵌入式系统**：内存有限，不想用动态分配（malloc 有碎片化问题）
- **实时系统**：malloc/free 的执行时间不确定，不满足实时性要求
- **内存池技术**：预分配一大块内存，自己管理分配和回收，本质就是游标实现的思想
- **数据库存储引擎**：用固定大小的页面管理记录，类似静态链表

## 第二页（P60）—— 动态链表的问题与游标实现的思路

### 动态链表的问题

普通链表的 `InsertList_L` 和 `DeleteList_L` 函数使用 `malloc()` 和 `free()` 来获取和释放内存空间。

这属于**动态内存管理（dynamic management）**，存在以下问题：

1. **频繁调用 malloc/free 开销大**：每次 malloc 都要在堆上搜索合适的空闲块，free 要合并相邻空闲块，这些操作有时间开销
2. **内存碎片化**：反复 malloc/free 后，堆上会出现很多小的不连续空闲块，导致即使总空闲内存足够，也无法分配大块连续内存
3. **某些语言根本没有这个机制**（如上一页提到的 BASIC、FORTRAN）

### 两种解决方案

PPT给出了两种解决思路：

**方案一：游标实现（Cursor Implementation）—— 用静态内存模拟链表**

- 预先分配一个大数组
- 数组的每个元素就是一个"节点"
- 用数组下标（整数）代替指针
- 自己管理哪些数组位置是"空闲"的（用一个空闲链表串起来）

**方案二：预分配大量节点到空闲链表（spare lists）**

- 初始时一次性 malloc 很多节点，串成一个空闲链表
- 需要新节点时从空闲链表取，不需要时还回空闲链表
- 避免频繁调用系统的 malloc/free

这两种方案本质思想相同：**自己管理内存池，避免依赖系统的动态分配。**

### 类比理解

可以把游标实现想象成一个"停车场"：

- 数组 = 停车场（固定大小，预先建好）
- 数组下标 = 车位编号
- 空闲链表 = 空车位登记表
- malloc = 从登记表上找一个空车位
- free = 把车位归还到登记表

## 第三页（P61）—— 链表的两个核心特征（为游标实现做铺垫）

### 为什么要先回顾链表的特征？

在设计游标实现之前，需要明确：普通链表到底依赖哪些底层机制？只有搞清楚这些，才能知道游标实现需要"模拟"什么。

### 特征一：数据存储在结构体的集合中

> "The data are stored in a collection of structures, each structure contains data and a pointer to the next structure."

普通链表的每个节点是一个结构体：

```c
struct Node {
    int data;           // 数据
    struct Node *next;  // 指向下一个结构体的指针
};
```

**游标实现如何模拟？**

用一个结构体数组：

```c
#define MAXSIZE 1000

struct CursorNode {
    int data;    // 数据
    int next;    // 下一个节点的数组下标（用整数代替指针）
};

struct CursorNode CursorSpace[MAXSIZE];  // 预分配的"内存池"
```

- `CursorSpace[i].data` 存数据
- `CursorSpace[i].next` 存下一个节点的下标（不是指针，是整数）
- `next = 0` 表示 NULL（约定下标 0 为空，类似指针的 NULL）

### 特征二：新节点通过 malloc 从系统全局内存获取，通过 free 释放

> "A new structure can be obtained from the system's global memory by a call to malloc and released by a call to free."

普通链表：
```c
struct Node *p = (struct Node *)malloc(sizeof(struct Node));  // 从堆获取
free(p);  // 归还给堆
```

**游标实现如何模拟？**

自己实现 CursorAlloc 和 CursorFree：

```c
// 模拟 malloc：从空闲链表取一个节点
int CursorAlloc(void) {
    int p = CursorSpace[0].next;       // 空闲链表的第一个可用位置
    CursorSpace[0].next = CursorSpace[p].next;  // 空闲链表头后移
    return p;                           // 返回分配到的下标
}

// 模拟 free：把节点归还到空闲链表
void CursorFree(int p) {
    CursorSpace[p].next = CursorSpace[0].next;  // p 的 next 指向原空闲链表头
    CursorSpace[0].next = p;                     // 空闲链表头变为 p
}
```

- `CursorSpace[0]` 是空闲链表的头节点（哨兵），它的 `next` 指向第一个可用的空闲位置
- CursorAlloc 相当于从空闲链表头部"摘下"一个节点
- CursorFree 相当于把节点"挂回"空闲链表头部

### 完整对应关系

| 普通链表               | 游标实现              |
| ---------------------- | --------------------- |
| 指针（`Node *`）       | 数组下标（`int`）     |
| NULL                   | 0                     |
| `malloc(sizeof(Node))` | `CursorAlloc()`       |
| `free(p)`              | `CursorFree(p)`       |
| `p->next`              | `CursorSpace[p].next` |
| `p->data`              | `CursorSpace[p].data` |
| 堆内存                 | `CursorSpace[]` 数组  |
