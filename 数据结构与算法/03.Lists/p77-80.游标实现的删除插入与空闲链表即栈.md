## 第一页（P77）—— 删除操作的图示演示

### 场景：从链表中删除元素 'e'

假设用户链表头在下标 1，内容为 c, b, e, g, f, d。

**左图（删除前）：**

| Slot | Element  | Next |
| ---- | -------- | ---- |
| 0    | freelist | 8    |
| 1    | header   | 2    |
| 2    | c        | 3    |
| 3    | b        | 4    |
| 4    | e        | 5    |
| 5    | g        | 6    |
| 6    | f        | 7    |
| 7    | d        | 0    |
| 8    |          | 9    |
| 9    |          | 10   |
| 10   |          | 0    |

追踪用户链表（蓝色）：`1(header) → 2(c) → 3(b) → 4(e) → 5(g) → 6(f) → 7(d) → NULL`

追踪空闲链表（绿色）：`0 → 8 → 9 → 10 → NULL`

**右图（删除 'e' 后）：**

| Slot | Element  | Next  |
| ---- | -------- | ----- |
| 0    | freelist | **4** |
| 1    | header   | 2     |
| 2    | c        | 3     |
| 3    | b        | **5** |
| 4    | e        | **8** |
| 5    | g        | 6     |
| 6    | f        | 7     |
| 7    | d        | 0     |
| 8    |          | 9     |
| 9    |          | 10    |
| 10   |          | 0     |

### 发生了什么？

两步操作：

**第一步：修改前驱的 next（让链表绕过被删节点）**
- 节点 3（'b'）是节点 4（'e'）的前驱
- `CursorSpace[3].next` 从 4 改为 5（跳过了 'e'，直接连到 'g'）

**第二步：CursorFree(4)（把下标4归还空闲链表）**
- `CursorSpace[4].next = CursorSpace[0].next = 8`（下标4的next指向原空闲链表第一个节点）
- `CursorSpace[0].next = 4`（空闲链表头指向4）

删除后：
- 用户链表：`1(header) → 2(c) → 3(b) → 5(g) → 6(f) → 7(d) → NULL`
- 空闲链表：`0 → 4 → 8 → 9 → 10 → NULL`

## 第二页（P78）—— 删除操作的代码

### 完整代码

```c
void delete(ElementType x, List L)
{
    position p, tmp_cell;
    p = find_previous(x, L);
    if (!is_last(p, L))
    {
        tmp_cell = CursorSpace[p].next;
        CursorSpace[p].next = CursorSpace[tmp_cell].next;
        CursorFree(tmp_cell);
    }
}
```

### 逐行解析

**`p = find_previous(x, L);`**

- 找到值为 x 的节点的**前驱**节点
- `find_previous` 返回的是 x 前面那个节点的下标
- 如果 x 不存在，`find_previous` 会返回链表最后一个节点的下标

**`if (!is_last(p, L))`**

- 检查 p 是否是最后一个节点
- 如果 p 是最后一个节点，说明 p 后面没有节点了，即 x 不存在于链表中
- 只有 p 不是最后一个节点时（即 x 确实存在），才执行删除

**`tmp_cell = CursorSpace[p].next;`**

- `tmp_cell` 就是要删除的节点（x 所在的节点）
- 它是 p 的下一个节点

**`CursorSpace[p].next = CursorSpace[tmp_cell].next;`**

- 前驱 p 的 next 跳过 tmp_cell，直接指向 tmp_cell 的下一个
- 链表中 x 被绕过了

**`CursorFree(tmp_cell);`**

- 把被删除的节点归还空闲链表

### 与指针实现的对比

```c
// 指针实现
void delete(ElementType x, Node *L) {
    Node *p = find_previous(x, L);
    if (!is_last(p, L)) {
        Node *tmp = p->next;
        p->next = tmp->next;
        free(tmp);
    }
}

// 游标实现
void delete(ElementType x, int L) {
    int p = find_previous(x, L);
    if (!is_last(p, L)) {
        int tmp_cell = CursorSpace[p].next;
        CursorSpace[p].next = CursorSpace[tmp_cell].next;
        CursorFree(tmp_cell);
    }
}
```

结构完全一致，只是语法不同。

### 时间复杂度

- `find_previous`：O(n)（需要遍历链表找到 x 的前驱）
- 删除操作本身：O(1)
- 总体：O(n)

## 第三页（P79）—— 插入操作的代码

### 顶部提示：判断尾节点

```
Check if the node is tail node
    CursorSpace[r].next != 0
```

这是一个辅助判断：如果 `CursorSpace[r].next != 0`，说明 r 不是尾节点（后面还有节点）。

### 插入函数代码

```c
status Insert(elementType e, List L, int p)  // insert after p
{
    int Tmpcell;
    Tmpcell = CursorAlloc();
    if (Tmpcell == 0)
        return ERROR;
    CursorSpace[Tmpcell].data = e;
    CursorSpace[Tmpcell].next = CursorSpace[p].next;
    CursorSpace[p].next = Tmpcell;
    return OK;
}
```

### 逐行解析

**`Tmpcell = CursorAlloc();`**

- 从空闲链表分配一个新单元格
- 如果返回 0，说明空间已满

**`if (Tmpcell == 0) return ERROR;`**

- 分配失败的防御性检查
- 等价于指针实现中 `if (newNode == NULL) return ERROR;`

**`CursorSpace[Tmpcell].data = e;`**

- 把数据 e 存入新节点

**`CursorSpace[Tmpcell].next = CursorSpace[p].next;`**

- 新节点的 next 指向 p 原来的后继
- 这一步必须在下一步之前，否则 `CursorSpace[p].next` 会被覆盖

**`CursorSpace[p].next = Tmpcell;`**

- p 的 next 指向新节点
- 新节点被插入到 p 之后

### 图解

```
插入前：... → p → q → ...
插入后：... → p → Tmpcell → q → ...
```

---

### 与指针实现的对比（续）

```c
// 指针实现：在 p 后插入
Node *newNode = (Node *)malloc(sizeof(Node));
if (newNode == NULL) return ERROR;
newNode->data = e;
newNode->next = p->next;
p->next = newNode;

// 游标实现：在 p 后插入
int Tmpcell = CursorAlloc();
if (Tmpcell == 0) return ERROR;
CursorSpace[Tmpcell].data = e;
CursorSpace[Tmpcell].next = CursorSpace[p].next;
CursorSpace[p].next = Tmpcell;
```

逻辑完全对称，翻译规则不变。

### 时间复杂度

O(1)——插入操作本身只需要常数次赋值（前提是已经知道插入位置 p）。

### 执行示例

假设链表为 `1(header) → 2(c) → 3(b) → 5(g) → NULL`，要在下标 3（'b'）后面插入 'x'：

```
Tmpcell = CursorAlloc() = 4（假设分配到下标4）
CursorSpace[4].data = 'x'
CursorSpace[4].next = CursorSpace[3].next = 5
CursorSpace[3].next = 4
```

结果：`1(header) → 2(c) → 3(b) → 4(x) → 5(g) → NULL`

## 第四页（P80）—— 空闲链表的本质是栈

### 核心观察

PPT指出空闲链表（freelist）本身就是一个有趣的数据结构：

**"最后放入空闲链表的单元格，会最先被取出。"**

- CursorFree 把节点**头插**到空闲链表（放在最前面）
- CursorAlloc 从空闲链表**头部取出**第一个节点

这意味着：
- 最近释放的节点 → 下次分配时最先被使用
- 最早释放的节点 → 最后才会被分配

### 这就是 LIFO（Last In, First Out）

**后进先出**——这正是**栈（Stack）**的特性。

| 操作 | 空闲链表              | 栈               |
| ---- | --------------------- | ---------------- |
| 放入 | CursorFree（头插）    | Push（压栈）     |
| 取出 | CursorAlloc（头部取） | Pop（弹栈）      |
| 顺序 | 后放入的先取出        | 后进先出（LIFO） |

### 为什么这个观察重要？

1. **概念联系**：空闲链表不是一个孤立的概念，它本质上就是一个用链表实现的栈。这帮助你理解不同数据结构之间的内在联系。

2. **为下一章做铺垫**：PPT在这里提到栈，是为了自然过渡到下一个主题——栈（Stack）的详细讲解。栈是数据结构课程的下一个重要内容。

3. **实际应用**：操作系统的内存管理、编译器的符号表管理等，都用类似的"空闲链表当栈用"的技术。

### 验证 LIFO 行为

```
初始空闲链表：0 → 1 → 2 → 3 → NULL

CursorAlloc() → 取出 1    空闲链表：0 → 2 → 3 → NULL
CursorAlloc() → 取出 2    空闲链表：0 → 3 → NULL
CursorFree(1)              空闲链表：0 → 1 → 3 → NULL（1被头插回来）
CursorAlloc() → 取出 1    空闲链表：0 → 3 → NULL（最近放回的1最先被取出）
```

确实是后进先出。
