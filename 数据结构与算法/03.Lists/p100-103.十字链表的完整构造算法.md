## 第一页（P100）—— 构造算法的流程概述

### 四个步骤

**(1) 读取矩阵的行数和列数**

输入矩阵的基本信息：m 行、n 列、t 个非零元素。

**(2) 创建矩阵头结构、行头指针数组、列头指针数组**

- 分配矩阵头结构 `CrossList M`
- 分配行头指针数组 `rhead[0..m-1]`（或 `rhead[0..m]`）
- 分配列头指针数组 `chead[0..n-1]`（或 `chead[0..n]`）

**(3) 初始化头指针**

所有行头和列头指针初始化为 NULL，表示每行每列目前都没有非零元素。

**(4) 迭代处理每个非零元素**

对每个非零元素，执行以下操作：
- **读取三元组** `(i, j, e)`：行号、列号、值
- **分配新节点**：malloc 一个 OLNode
- **链入行链表**：把新节点插入到第 i 行的链表中（按列号有序）
- **链入列链表**：把新节点插入到第 j 列的链表中（按行号有序）

### 关键点

每个新节点需要同时插入两个链表（行链表和列链表），这是十字链表构造的核心难点。插入时要保持有序性：
- 行链表中按列号递增排列
- 列链表中按行号递增排列

## 第二页（P101）—— 构造函数的初始化部分

### 代码

```c
Status CreateSMatrix_OL(CrossList &M)
{
    if (M)
        free(M);                          // 如果 M 已存在，先释放旧的

    scanf(&m, &n, &t);                    // 读入行数、列数、非零元素个数
    M.mu = m;
    M.nu = n;
    M.tu = t;

    // 分配行头指针数组（m+1 个指针）
    if (!M.rhead = (OLink*)malloc((m+1) * sizeof(OLink)))
        exit(OVERFLOW);

    // 分配列头指针数组（n+1 个指针）
    if (!M.chead = (OLink*)malloc((n+1) * sizeof(OLink)))
        exit(OVERFLOW);

    // 初始化所有行头和列头为 NULL
    M.rhead[] = M.chead[] = NULL;
}
```

### 逐行解析

**`if(M) free(M);`**

如果矩阵 M 之前已经存在（非空），先释放旧的内存，避免内存泄漏。

**`scanf(&m, &n, &t);`**

从输入读取：
- m = 行数
- n = 列数
- t = 非零元素个数

**`M.mu = m; M.nu = n; M.tu = t;`**

把基本信息存入矩阵头结构。

**分配行头指针数组：**

```c
M.rhead = (OLink*)malloc((m+1) * sizeof(OLink));
```

- 分配 m+1 个指针的空间（多分配一个是为了方便从下标 1 开始使用，或者作为哨兵）
- 每个 `M.rhead[i]` 将指向第 i 行的第一个非零元素

**分配列头指针数组：**

```c
M.chead = (OLink*)malloc((n+1) * sizeof(OLink));
```

- 分配 n+1 个指针
- 每个 `M.chead[j]` 将指向第 j 列的第一个非零元素

**初始化为 NULL：**

```c
// 伪代码，实际需要循环
for (int k = 0; k <= m; k++) M.rhead[k] = NULL;
for (int k = 0; k <= n; k++) M.chead[k] = NULL;
```

表示初始时每行每列都没有非零元素。

## 第三页（P102）—— 插入节点到行链表

### 循环读取并插入每个非零元素

```c
for (scanf(&i, &j, &e); i != 0; scanf(&i, &j, &e))
{
```

循环读取三元组 `(i, j, e)`，当 `i == 0` 时结束（约定输入以 0 结尾）。

### 分配新节点

```c
    if (!p = (OLNode*)malloc(sizeof(OLNode)))
        exit(OVERFLOW);
    p->i = i;  p->j = j;  p->e = e;
```

创建新节点并填入行号、列号、值。

### 插入到行链表（按列号有序）

```c
    // 情况1：该行为空，或新节点的列号比该行第一个节点的列号还小
    if (M.rhead[i] == NULL || M.rhead[i]->j > j)
    {
        p->right = M.rhead[i];   // 新节点的 right 指向原来的第一个
        M.rhead[i] = p;          // 行头指向新节点（头插）
    }
    // 情况2：需要在行链表中间找到正确位置
    else
    {
        // 遍历找到插入位置：找到最后一个列号 < j 的节点 q
        for (q = M.rhead[i]; (q->right) && q->right->j < j; q = q->right);
        p->right = q->right;    // 新节点的 right 指向 q 的原后继
        q->right = p;           // q 的 right 指向新节点
    }
```

### 两种情况的图解

**情况1（头插）：** 新节点列号最小，插到行链表最前面

```
插入前：rhead[i] → (i,5,x) → (i,8,y) → NULL
插入 (i,2,z)：
插入后：rhead[i] → (i,2,z) → (i,5,x) → (i,8,y) → NULL
```

**情况2（中间插入）：** 找到正确位置后插入

```
插入前：rhead[i] → (i,2,z) → (i,8,y) → NULL
插入 (i,5,x)：
  q 停在 (i,2,z)（因为 q->right->j = 8 > 5，停止）
插入后：rhead[i] → (i,2,z) → (i,5,x) → (i,8,y) → NULL
```

## 第四页（P103）—— 插入节点到列链表

### 插入到列链表（按行号有序）

逻辑与行链表完全对称，只是方向不同：

```c
    // 情况1：该列为空，或新节点的行号比该列第一个节点的行号还小
    if (M.chead[j] == NULL || M.chead[j]->i > i)
    {
        p->down = M.chead[j];   // 新节点的 down 指向原来的第一个
        M.chead[j] = p;         // 列头指向新节点（头插）
    }
    // 情况2：需要在列链表中间找到正确位置
    else
    {
        // 遍历找到插入位置：找到最后一个行号 < i 的节点 q
        for (q = M.chead[j]; (q->down) && q->down->i < i; q = q->next);
        p->down = q->down;      // 新节点的 down 指向 q 的原下方节点
        q->down = p;            // q 的 down 指向新节点
    }
}   // end of for loop

return OK;
}
```

### 两种情况的图解

**情况1（头插）：** 新节点行号最小，插到列链表最前面

```
插入前：chead[j] → (3,j,x) → (7,j,y) → NULL
插入 (1,j,z)：
插入后：chead[j] → (1,j,z) → (3,j,x) → (7,j,y) → NULL
```

**情况2（中间插入）：**

```
插入前：chead[j] → (1,j,z) → (7,j,y) → NULL
插入 (3,j,x)：
  q 停在 (1,j,z)（因为 q->down->i = 7 > 3，停止）
插入后：chead[j] → (1,j,z) → (3,j,x) → (7,j,y) → NULL
```

### 完整插入过程总结

每个新节点 p 需要执行两次插入：
1. 插入到第 `p->i` 行的行链表中（按列号有序，通过 right 指针链接）
2. 插入到第 `p->j` 列的列链表中（按行号有序，通过 down 指针链接）

两次插入互不干扰，因为它们操作的是不同的指针域（right vs down）。

### 时间复杂度

对于每个非零元素：
- 行链表插入：最坏 O(该行非零元素数)
- 列链表插入：最坏 O(该列非零元素数)

总体构造时间：O(t × max(行中最大非零数, 列中最大非零数))

如果输入数据已经按行优先、列递增排序，则每次都是尾插，行链表插入退化为 O(1)，总时间 O(t)。
