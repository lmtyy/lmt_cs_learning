这四页PPT展示了游标实现链表的基本操作函数：判空、判断尾节点、查找元素。它们和普通指针链表的逻辑完全一致，只是把指针操作替换成了数组下标操作。

---

## 第一页（P73）—— 函数接口设计原则

### 核心观点

PPT指出：游标实现的链表函数，必须和指针实现的链表函数**传递和返回相同的参数**。

这意味着：
- 对外接口（函数签名）保持一致
- 调用者不需要知道底层是用指针还是用数组实现的
- 只是内部实现从指针操作变成了数组下标操作

### 对比示例

```c
// 指针实现的接口
int is_empty(Node *L);
Node* find(ElementType x, Node *L);

// 游标实现的接口
int is_empty(int L);          // L 是头节点的下标（int 代替了 Node*）
int find(ElementType x, int L);  // 返回值也是 int（下标代替指针）
```

类型上的对应：
- `Node *` → `int`（下标）
- `NULL` → `0`
- `p->next` → `CursorSpace[p].next`
- `p->data` → `CursorSpace[p].element`

### 为什么这样设计？

这体现了**抽象数据类型（ADT）**的思想：
- 链表的逻辑行为（插入、删除、查找、判空）不依赖于具体实现方式
- 无论底层是指针还是数组，对外提供的操作语义相同
- 可以在不改变上层代码的情况下切换实现方式

## 第二页（P74）—— is_empty：判断链表是否为空

### 代码

```c
int is_empty(List L)  /* using a header node */
{
    return (CursorSpace[L].next == 0);
}
```

### 解析

- `L` 是链表头节点的下标
- `CursorSpace[L].next` 是头节点的 next 字段
- 如果 next == 0（等价于 NULL），说明头节点后面没有数据节点，链表为空
- 返回 1（真）表示空，返回 0（假）表示非空

### 与指针实现的对比

```c
// 指针实现
int is_empty(Node *L)
{
    return (L->next == NULL);
}

// 游标实现
int is_empty(int L)
{
    return (CursorSpace[L].next == 0);
}
```

逻辑完全一样，只是：
- `L->next` 变成了 `CursorSpace[L].next`
- `NULL` 变成了 `0`

### 示例

回顾之前的表格，如果链表 L 的头节点在下标 5：
- `CursorSpace[5].next = 10`（非零）→ 链表非空
- 如果 `CursorSpace[5].next = 0` → 链表为空

## 第三页（P75）—— is_last：判断是否为最后一个节点

### 代码

```c
int is_last(position p, List L)  /* using a header node */
{
    return (CursorSpace[p].next == 0);
}
```

### 解析

- `p` 是当前节点的下标
- 如果 `CursorSpace[p].next == 0`，说明 p 后面没有节点了，p 就是链表的最后一个节点
- 返回 1（真）表示是最后一个，返回 0（假）表示不是

### 参数 L 的作用

注意函数签名中有参数 `List L`（链表标识），但函数体中并没有使用它。这是为了保持接口一致性——在某些实现中（比如循环链表），判断是否为最后一个节点可能需要用到 L。这里保留参数是为了接口统一。

### 与指针实现的对比

```c
// 指针实现
int is_last(Node *p, Node *L)
{
    return (p->next == NULL);
}

// 游标实现
int is_last(int p, int L)
{
    return (CursorSpace[p].next == 0);
}
```

### 使用场景

遍历链表时，用 is_last 判断是否到达末尾：

```c
int p = CursorSpace[L].next;  // 第一个数据节点
while (!is_last(p, L)) {
    // 处理 p
    p = CursorSpace[p].next;
}
// 此时 p 是最后一个节点
```

## 第四页（P76）—— find：查找元素

### 代码

```c
position find(ElementType x, List L)
/* using a header node */
{
    position p;
    p = CursorSpace[L].next;
    while (p && CursorSpace[p].element != x)
        p = CursorSpace[p].next;
    return p;
}
```

### 逐行解析

**`p = CursorSpace[L].next;`**

- 从头节点的下一个开始（跳过头节点，因为头节点不存有效数据）
- p 指向第一个数据节点

**`while (p && CursorSpace[p].element != x)`**

两个终止条件（任一满足就退出循环）：
- `p == 0`：已经遍历完整个链表，没找到 x
- `CursorSpace[p].element == x`：找到了 x

注意 `p` 在这里当作布尔值使用：`p != 0` 为真，`p == 0` 为假。这和指针实现中 `p != NULL` 的写法一样。

**`p = CursorSpace[p].next;`**

- 移动到下一个节点

**`return p;`**

- 如果找到了 x，返回 x 所在节点的下标
- 如果没找到，返回 0（等价于返回 NULL）

### 与指针实现的对比

```c
// 指针实现
Node* find(ElementType x, Node *L)
{
    Node *p = L->next;
    while (p != NULL && p->data != x)
        p = p->next;
    return p;
}

// 游标实现
int find(ElementType x, int L)
{
    int p = CursorSpace[L].next;
    while (p && CursorSpace[p].element != x)
        p = CursorSpace[p].next;
    return p;
}
```

结构完全对称，只是语法不同。

### 时间复杂度

O(n)——最坏情况需要遍历整个链表。

### 执行示例

用之前 P68 的表格，链表 L 头在下标 5，内容为 a, b, e：

查找 'b'：
```
p = CursorSpace[5].next = 10    → CursorSpace[10].element = 'a' ≠ 'b'
p = CursorSpace[10].next = 1    → CursorSpace[1].element = 'b' == 'b' → 退出循环
return 1（'b' 在下标 1）
```

查找 'z'（不存在）：
```
p = 10 → 'a' ≠ 'z'
p = 1  → 'b' ≠ 'z'
p = 9  → 'e' ≠ 'z'
p = CursorSpace[9].next = 0 → 退出循环（p为0，条件p&&...为假）
return 0（未找到）
```

## 四页知识脉络总结

```
游标实现的基本操作函数（P73-P76）
│
├── 设计原则（P73）
│   └── 接口与指针实现保持一致，体现 ADT 抽象
│
├── is_empty（P74）
│   ├── CursorSpace[L].next == 0 → 空
│   └── 对应指针版：L->next == NULL
│
├── is_last（P75）
│   ├── CursorSpace[p].next == 0 → 是最后一个
│   └── 对应指针版：p->next == NULL
│
└── find（P76）
    ├── 从头节点的 next 开始顺序查找
    ├── 找到返回下标，没找到返回 0
    ├── 时间复杂度 O(n)
    └── 对应指针版：遍历直到 NULL 或找到目标
```

### 核心规律

所有游标实现的函数，都遵循同一个翻译规则：

| 指针写法      | 游标写法                  |
| ------------- | ------------------------- |
| `p->next`     | `CursorSpace[p].next`     |
| `p->data`     | `CursorSpace[p].element`  |
| `p == NULL`   | `p == 0`                  |
| `p != NULL`   | `p != 0`（或直接写 `p`）  |
| `p = p->next` | `p = CursorSpace[p].next` |

掌握这个翻译规则后，任何指针链表的函数都可以机械地转换为游标实现版本。
