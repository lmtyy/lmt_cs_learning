这三页PPT讲的是**双向链表（Doubly Linked List）的构造和插入操作**，从节点定义、空链表创建，到插入操作的指针修改细节和完整函数实现。

## 第一页（P53）—— 双向链表的节点定义与空链表创建

### 节点结构定义

```c
typedef struct DLNode {
    ElemType data;          // 数据域
    struct DLNode *prior;   // 前驱指针，指向前一个节点
    struct DLNode *next;    // 后继指针，指向后一个节点
} DLNode, *DLinkList;
```

逐字段解释：

- `ElemType data`：存储实际数据，`ElemType` 是一个类型别名（可以是 `int`、`char`、结构体等）
- `struct DLNode *prior`：指向前驱节点的指针。注意这里用的名字是 `prior`（有些教材用 `prev`），含义相同
- `struct DLNode *next`：指向后继节点的指针
- `DLNode`：节点类型的别名
- `*DLinkList`：节点指针类型的别名，用来表示"整个链表"（通过头指针）

与循环链表节点的对比：

```c
// 循环链表节点（上一组PPT）
typedef struct CLNode {
    ElemType data;
    struct CLNode *next;    // 只有一个指针
} CLNode, *CLinkList;

// 双向链表节点（本页）
typedef struct DLNode {
    ElemType data;
    struct DLNode *prior;   // 多了一个前驱指针
    struct DLNode *next;
} DLNode, *DLinkList;
```

### 创建空双向链表

```c
void CreateList_DL(LinkList &DL)
// create an empty list
{
    DL = (DLinkList)malloc(sizeof(DLNode));  // 分配头节点
    DL->next = DL->prior = NULL;             // 头节点的前驱和后继都置为 NULL
}
```

逐行解析：

**`DL = (DLinkList)malloc(sizeof(DLNode));`**

- 在堆上分配一个 `DLNode` 大小的内存块
- 这个节点是**头节点（header node）**，不存有效数据，只作为链表的入口标记
- 强制类型转换为 `DLinkList`（即 `DLNode *`）

**`DL->next = DL->prior = NULL;`**

- 头节点的 `next` 和 `prior` 都设为 NULL
- 表示链表为空：头节点后面没有数据节点，前面也没有

图示（空双向链表）：

```
NULL ← [DL(头节点)] → NULL
```

### 与循环链表空表的对比

|          | 循环链表空表                | 双向链表空表                  |
| -------- | --------------------------- | ----------------------------- |
| 创建方式 | `CL->next = CL`（指向自己） | `DL->next = DL->prior = NULL` |
| 空表判断 | `head->next == head`        | `head->next == NULL`          |
| 设计思想 | 形成环                      | 两端为 NULL                   |

### 为什么要用头节点？

头节点的存在使得插入和删除操作不需要对"链表为空"或"在第一个位置操作"做特殊处理。没有头节点时，在链表头部插入需要修改头指针本身；有头节点时，头部插入和中间插入的代码逻辑完全一致。

## 第二页（P54）—— 双向链表插入操作的指针修改过程

### 场景描述

在节点 `p`（存储数据 `ai`）**之前**插入一个新节点 `q`（存储数据 `x`）。

插入前的状态：

```
... ⇄ [ai-1] ⇄ [ai(p)] ⇄ ...
```

插入后的目标状态：

```
... ⇄ [ai-1] ⇄ [x(q)] ⇄ [ai(p)] ⇄ ...
```

### 四步操作（对应图中的编号①②③④）

```c
q = (DLinkList)malloc(sizeof(DLNode));  // 分配新节点
q->data = e;                             // 存入数据

// ① q->prior = p->prior;    新节点的前驱 = p的原前驱（即ai-1）
// ② p->prior->next = q;     p的原前驱的后继 = 新节点（ai-1的next指向q）
// ③ q->next = p;            新节点的后继 = p
// ④ p->prior = q;           p的前驱 = 新节点
```

### 逐步图解

**初始状态：**

```
[ai-1] ⇄ [ai(p)]
  ↑prior    ↑
  p->prior  p
```

**第①步：`q->prior = p->prior`**

新节点 q 的前驱指向 ai-1：

```
[ai-1] ⇄ [ai(p)]
  ↑
  q->prior ← [x(q)]
```

这一步必须在第④步之前执行，因为第④步会覆盖 `p->prior`。

**第②步：`p->prior->next = q`**

ai-1 的后继从 p 改为指向 q：

```
[ai-1] → [x(q)]     [ai(p)]
         ↑
    ai-1的next现在指向q
```

**第③步：`q->next = p`**

新节点 q 的后继指向 p：

```
[ai-1] → [x(q)] → [ai(p)]
```

**第④步：`p->prior = q`**

p 的前驱从 ai-1 改为指向 q：

```
[ai-1] ⇄ [x(q)] ⇄ [ai(p)]
```

插入完成，所有四个指针都正确连接。

### 指针修改顺序的重要性

这四步的顺序不是随意的，核心原则是：**在覆盖某个指针之前，先把它的旧值保存或使用完毕。**

- 第①步用到了 `p->prior`（旧值），所以必须在第④步（修改 `p->prior`）之前
- 第②步用到了 `p->prior->next`，也依赖 `p->prior` 的旧值，所以也必须在第④步之前

实际上，只要满足以下约束，顺序可以有多种合法排列：
- ① 必须在 ④ 之前（因为④覆盖了 `p->prior`）
- ② 必须在 ④ 之前（同理）
- ③ 和 ④ 的相对顺序无所谓

所以合法顺序包括：①②③④、①②④③、②①③④ 等。

### 常见错误

如果先执行第④步：

```c
p->prior = q;           // ④ 先执行，p->prior 被覆盖为 q
q->prior = p->prior;   // ① 此时 p->prior 已经是 q，所以 q->prior = q（自己指向自己！）
```

这就导致了逻辑错误，链表断裂。考试中这是常见的扣分点。

## 第三页（P55）—— 双向链表插入的完整函数实现

### 函数签名

```c
Status ListInsert_DL(DLinkList &L, int i, ElemType e)
```

- `DLinkList &L`：双向链表的引用（C++风格），传入链表头指针
- `int i`：插入位置，表示在第 i 个位置之前插入
- `ElemType e`：要插入的数据值
- 返回值 `Status`：操作状态（OK 或 ERROR）

### 函数功能

在双向链表 L 的第 i 个位置之前插入一个值为 e 的新节点。合法的 i 范围是 `1 ≤ i ≤ length+1`。

### 代码逐行解析

```c
Status ListInsert_DL(DLinkList &L, int i, ElemType e)
{
    // 第1步：定位到第 i 个节点
    if (!(p = GetElemP_DL(L, i)))
        return ERROR;
```

- `GetElemP_DL(L, i)` 是一个辅助函数，返回链表 L 中第 i 个节点的指针
- 如果 i 不合法（超出范围），返回 NULL，此时函数返回 ERROR
- `p` 现在指向第 i 个节点（即新节点要插入在 p 之前）

```c
    // 第2步：分配新节点
    if (!(s = (DLinkList)malloc(sizeof(DLNode))))
        return ERROR;
```

- 分配一个新节点 `s`
- 如果 `malloc` 失败（内存不足），返回 NULL，函数返回 ERROR
- 这是防御性编程的好习惯

```c
    // 第3步：设置新节点的数据
    s->data = e;
```

```c
    // 第4步：修改指针（对应上一页的四步操作）
    // （PPT中这里有一个空白框，应该填入上一页的四条指针操作语句）
    s->prior = p->prior;      // 新节点的前驱 = p的原前驱
    p->prior->next = s;       // p的原前驱的后继 = 新节点
    s->next = p;              // 新节点的后继 = p
    p->prior = s;             // p的前驱 = 新节点

    return OK;
}
```

### 完整代码（补全空白框）

```c
Status ListInsert_DL(DLinkList &L, int i, ElemType e)
{
    // 定位第 i 个节点
    if (!(p = GetElemP_DL(L, i)))
        return ERROR;
    
    // 分配新节点
    if (!(s = (DLinkList)malloc(sizeof(DLNode))))
        return ERROR;
    
    // 赋值
    s->data = e;
    
    // 修改四个指针
    s->prior = p->prior;
    p->prior->next = s;
    s->next = p;
    p->prior = s;
    
    return OK;
} // ListInsert_DL
```

### 时间复杂度分析

- `GetElemP_DL(L, i)`：从头节点遍历到第 i 个节点，时间复杂度 O(n)
- 指针修改操作：O(1)
- 总体时间复杂度：**O(n)**（瓶颈在定位）

如果你已经持有目标位置的指针（不需要调用 `GetElemP_DL`），则插入操作本身是 O(1)。

### 边界情况考虑

**在链表头部插入（i = 1）：**

此时 `p` 是第一个数据节点，`p->prior` 是头节点。操作正常，因为头节点的 `next` 会被正确修改。

**在链表尾部之后插入（i = length + 1）：**

此时 `GetElemP_DL` 需要能返回一个"尾部之后"的位置。具体实现取决于 `GetElemP_DL` 的设计。有些实现会让 i = length+1 时返回 NULL，然后用不同的逻辑处理尾部插入（因为此时 `p->prior->next` 中的 `p` 是 NULL，会段错误）。

这是一个需要注意的细节：PPT中的代码假设 `p` 不为 NULL，所以严格来说只适用于 `1 ≤ i ≤ length` 的情况。尾部追加可能需要单独处理。

### 与单链表插入的对比

```c
// 单链表插入（在 prev 后插入）—— 2条指针操作
newNode->next = prev->next;
prev->next = newNode;

// 双向链表插入（在 p 前插入）—— 4条指针操作
s->prior = p->prior;
p->prior->next = s;
s->next = p;
p->prior = s;
```

双向链表操作量翻倍，但好处是：你可以直接在某个节点"前面"插入，而单链表只能在某个节点"后面"插入（因为单链表没有前驱信息）。

## 三页知识脉络总结

```
双向链表构造（P53）
│  节点结构：prior | data | next
│  空表创建：头节点的 prior 和 next 都为 NULL
│
插入操作的指针修改（P54）
│  在 p 之前插入 q：
│  ① q->prior = p->prior     （新节点前驱指向p的原前驱）
│  ② p->prior->next = q      （原前驱的后继改为新节点）
│  ③ q->next = p             （新节点后继指向p）
│  ④ p->prior = q            （p的前驱改为新节点）
│  关键：①②必须在④之前，否则 p->prior 被覆盖导致错误
│
完整插入函数（P55）
   步骤：定位 → 分配 → 赋值 → 改指针
   时间复杂度：O(n)（定位） + O(1)（改指针）
   注意边界：尾部插入可能需要特殊处理
```

### 考试重点提醒

1. **手写指针修改代码**：给你一个场景（在某节点前/后插入），要求写出正确的指针操作语句，并且顺序不能错。
2. **判断顺序是否合法**：给你一组打乱的指针操作，判断哪些顺序会导致链表断裂。
3. **对比单链表和双向链表的插入/删除**：单链表改2个指针，双向链表改4个指针；单链表只能"后插"，双向链表可以"前插"也可以"后插"。
