## 第一页（P49）—— 循环链表的遍历代码

### 完整代码

```c
void DisplayList_CL(LinkList CL)
/* traverse a linked list */
{
    p = head->next;       // p 指向第一个数据节点
    while (p != head)     // 当 p 没有回到头节点时，继续遍历
    {
        printf("%d", p->data);  // 输出当前节点的数据
        p = p->next;            // 移动到下一个节点
    }
}
```

### 逐行解析

**`p = head->next;`**

- `head` 是头节点（哨兵节点，不存有效数据）
- `head->next` 是第一个真正的数据节点
- 把 `p` 初始化为第一个数据节点，准备开始遍历

**`while (p != head)`**

- 这是循环链表遍历的核心终止条件
- 因为循环链表的最后一个节点的 `next` 指回 `head`，所以当 `p` 再次等于 `head` 时，说明已经走完一圈
- 如果链表为空（`head->next == head`），则 `p` 一开始就等于 `head`，循环体不执行，直接结束

**循环体内：**
- `printf("%d", p->data);` —— 打印当前节点数据
- `p = p->next;` —— 移动到下一个节点

### 与普通单链表遍历的对比

PPT右上角黄色框标注了普通单链表的终止条件：`while(p->next != NULL)`

|                     | 普通单链表                       | 循环链表             |
| ------------------- | -------------------------------- | -------------------- |
| 终止条件            | `p->next != NULL` 或 `p != NULL` | `p != head`          |
| 最后一个节点的 next | NULL                             | head（头节点）       |
| 空链表判断          | `head->next == NULL`             | `head->next == head` |

### 为什么这里用 while 而不是 do...while？

上一组PPT我提到过循环链表遍历通常用 `do...while`。但这里的代码用的是 `while`，原因是：

- `p` 的初始值是 `head->next`（第一个数据节点），而不是 `head` 本身
- 所以初始时 `p != head` 是成立的（除非链表为空）
- 用 `while` 可以自然处理空链表的情况（直接不进入循环）

如果写成 `do...while`，空链表时会错误地执行一次循环体，访问到头节点的无效数据。

### 执行过程示例

假设循环链表为：`head → 10 → 20 → 30 → (回到 head)`

```
初始：p = head->next = 节点10
第1次循环：p(10) != head ✓ → 打印10 → p = 节点20
第2次循环：p(20) != head ✓ → 打印20 → p = 节点30
第3次循环：p(30) != head ✓ → 打印30 → p = head
第4次判断：p(head) != head ✗ → 退出循环
输出：10 20 30
```

## 第二页（P50）—— 引出双向链表的动机

### 问题：有时需要反向遍历

PPT指出：有时候我们需要反向遍历链表（从后往前走），但标准的单链表实现无法做到这一点。

**为什么单链表不能反向遍历？**

单链表每个节点只有一个 `next` 指针，指向后继。当你站在某个节点上，你完全不知道前一个节点在哪里：

```
A → B → C → D
        ↑
      你在这里，想回到B？做不到。
```

即使是循环单链表，虽然能从尾部"绕回"头部，但要找到某个节点的前驱，仍然需要从头遍历一圈，时间复杂度 O(n)。

### 解决方案：非常简单

PPT说"solution is simple"——只需要在节点结构中**增加一个额外的指针域**，指向前一个节点（前驱节点）。

```c
// 单链表节点
struct SNode {
    int data;
    struct SNode *next;      // 只有后继指针
};

// 双向链表节点
struct DNode {
    int data;
    struct DNode *prev;      // 新增：前驱指针
    struct DNode *next;      // 后继指针
};
```

这样每个节点同时知道"前面是谁"和"后面是谁"，就可以双向移动了。

### 实际需求场景

什么时候需要反向遍历？

- **浏览器的前进/后退**：浏览历史就是一个双向链表，后退就是走 `prev`，前进就是走 `next`
- **文本编辑器的撤销/重做**：undo 走 prev，redo 走 next
- **LRU 缓存**：需要快速删除任意节点，双向链表可以 O(1) 找到前驱并删除
- **操作系统的进程调度队列**：需要从两端操作

## 第三页（P51）—— 双向链表的代价与收益

### 代价（Cost）

**1. 空间开销增加**

每个节点多了一个指针：

```c
// 64位系统上
struct DNode {
    int data;           // 4 字节
    struct DNode *prev; // 8 字节（新增）
    struct DNode *next; // 8 字节
};
// 总计：20 字节（可能因对齐变为 24 字节）
// 对比单链表节点：12 字节（可能对齐为 16 字节）
```

存储 1000 个 int 的空间对比：
- 单链表：约 16000 字节
- 双向链表：约 24000 字节
- 多了 50% 的空间开销

**2. 插入和删除的操作量翻倍**

单链表插入只需要修改 1-2 个指针，双向链表需要修改 4 个指针：

```c
// 单链表：在 prev 后插入 newNode
newNode->next = prev->next;
prev->next = newNode;
// 2 条指针操作

// 双向链表：在 prev 后插入 newNode
newNode->next = prev->next;
newNode->prev = prev;
prev->next->prev = newNode;  // 原后继的 prev 指向新节点
prev->next = newNode;
// 4 条指针操作
```

虽然时间复杂度仍然是 O(1)，但常数因子翻倍了，代码也更容易写错。

### 收益（Benefit）

**删除操作被大大简化。**

在单链表中，要删除某个节点，你必须知道它的前驱节点：

```c
// 单链表删除节点 target
// 问题：你必须先找到 target 的前驱 prev
prev->next = target->next;
free(target);
```

但如果你只有 `target` 的指针，没有 `prev`，你就必须从头遍历找到 `prev`，这是 O(n) 的。

在双向链表中，每个节点自带 `prev` 指针，删除任意节点只需要：

```c
// 双向链表删除节点 target（已知 target 指针）
target->prev->next = target->next;   // 前驱的 next 跳过 target
target->next->prev = target->prev;   // 后继的 prev 跳过 target
free(target);
```

不需要遍历，直接 O(1) 完成。这在很多算法中是关键优势（比如 LRU 缓存淘汰）。

### 总结权衡

|              | 单链表           | 双向链表         |
| ------------ | ---------------- | ---------------- |
| 每节点空间   | data + 1个指针   | data + 2个指针   |
| 插入操作     | 改 2 个指针      | 改 4 个指针      |
| 已知节点删除 | O(n)（需找前驱） | O(1)（自带前驱） |
| 反向遍历     | 不支持           | 支持             |
| 代码复杂度   | 简单             | 稍复杂           |

## 第四页（P52）—— 双向链表的结构定义与图示

### 三个核心特征

**1. 每个节点同时指向后继和前驱**

```c
struct DNode {
    int data;
    struct DNode *prev;  // 指向前一个节点
    struct DNode *next;  // 指向后一个节点
};
```

每个节点有三个域：`prev | data | next`

**2. 有两个 NULL**

- 第一个节点的 `prev` 指向 NULL（它前面没有节点了）
- 最后一个节点的 `next` 指向 NULL（它后面没有节点了）

```
NULL ← [节点1] ⇄ [节点2] ⇄ [A] ⇄ [B] → NULL
```

**3. 优势：给定任意节点，可以直接访问其前驱，方便反向遍历**

### 图示解读

PPT中的图展示了一个带头节点的双向链表：

```
Head
  ↓
[∅|数据|•] ⇄ [•|数据|•] ⇄ [•|A|•] ⇄ [•|B|∅]
```

- `Head` 指针指向第一个节点
- 第一个节点的 `prev`（左边的域）是 ∅（NULL）
- 最后一个节点 B 的 `next`（右边的域）是 ∅（NULL）
- 中间节点的 `prev` 和 `next` 都指向相邻节点
- `⇄` 表示双向连接

### 双向链表的遍历

**正向遍历（从前往后）：**

```c
void TraverseForward(DNode *head) {
    DNode *p = head;
    while (p != NULL) {
        printf("%d ", p->data);
        p = p->next;
    }
}
```

**反向遍历（从后往前）：**

```c
void TraverseBackward(DNode *tail) {
    DNode *p = tail;  // 从尾节点开始
    while (p != NULL) {
        printf("%d ", p->data);
        p = p->prev;  // 往前走
    }
}
```

### 双向链表的插入操作（完整版）

在节点 `p` 之后插入新节点 `s`：

```c
void InsertAfter(DNode *p, DNode *s) {
    // 第1步：新节点的 next 指向 p 的原后继
    s->next = p->next;
    // 第2步：新节点的 prev 指向 p
    s->prev = p;
    // 第3步：p 的原后继的 prev 指向新节点（注意判空）
    if (p->next != NULL) {
        p->next->prev = s;
    }
    // 第4步：p 的 next 指向新节点
    p->next = s;
}
```

指针修改顺序很重要，如果先执行第4步，`p->next` 就被覆盖了，第1步和第3步就找不到原来的后继了。

### 双向链表的删除操作

删除节点 `p`：

```c
void DeleteNode(DNode *p) {
    // 修改前驱的 next
    if (p->prev != NULL) {
        p->prev->next = p->next;
    }
    // 修改后继的 prev
    if (p->next != NULL) {
        p->next->prev = p->prev;
    }
    free(p);
}
```

注意边界情况：如果 `p` 是第一个节点，`p->prev` 为 NULL；如果是最后一个节点，`p->next` 为 NULL。
