这四页PPT的主题是**链表的变体（Variations of Linked Lists）**，从单链表的缺陷出发，引出循环链表和双向链表的设计思想，并给出了循环链表的代码实现。

---

## 第一页（P45）—— 单链表的两个问题

### 问题一：无法从尾部回到头部

普通单链表的结构是这样的：

```
Head → A → B → C → D → NULL
```

最后一个节点的 `next` 指向 `NULL`，表示链表结束。一旦你遍历到了尾部（D节点），想回到头部怎么办？没办法，除非你一直保存着 `head` 指针。如果你在某个函数中只拿到了中间或尾部的某个节点指针，你就"回不去了"。

### 问题二：无法反向遍历

单链表每个节点只有一个 `next` 指针，指向后继节点。这意味着你只能从前往后走，不能从后往前走。

```c
struct Node {
    int data;
    struct Node *next;  // 只能往后走，没有往前的指针
};
```

比如你当前在节点C，想访问它前面的节点B，你做不到——因为C里没有任何信息告诉你B在哪里。唯一的办法是从 `head` 重新遍历到B，时间复杂度 O(n)。

### 解决方案

为了解决这两个问题，人们发明了两种链表变体：

- **循环链表（Circular Linked List）**：解决"无法从尾部回到头部"的问题
- **双向链表（Doubly Linked List）**：解决"无法反向遍历"的问题

当然，这两种思想可以组合使用（循环双向链表），同时解决两个问题。

## 第二页（P46）—— 循环链表的设计约定

### 核心思想

一种流行的约定是：**让最后一个节点的指针指回第一个节点**（而不是指向 NULL）。

```
普通单链表：Head → A → B → C → D → NULL
循环链表：  Head → A → B → C → D → (回到 Head 或 A)
```

这样从任何节点出发，沿着 `next` 一直走，最终都能回到起点，形成一个"环"。

### 有无头节点（Header）的两种形式

PPT提到循环链表可以有两种实现方式：

**方式一：无头节点（没有 header）**

最后一个节点的 `next` 直接指回第一个数据节点：

```
→ A → B → C → D ─┐
↑                  │
└──────────────────┘
```

**方式二：有头节点（有 header）**

头节点是一个不存数据的哨兵节点，最后一个数据节点的 `next` 指回头节点：

```
→ Header → A → B → C → D ─┐
↑                            │
└────────────────────────────┘
```

头节点的好处是：统一了空链表和非空链表的操作逻辑，不需要对"链表为空"做特殊判断。

### 与双向链表的结合

PPT还提到循环的思想也可以用在双向链表上：第一个节点的 `prev` 指针指向最后一个节点。这样就形成了一个双向的环：

```
┌────────────────────────────────────┐
↓                                    │
A ⇄ B ⇄ C ⇄ D
│                                    ↑
└────────────────────────────────────┘
```

从任何节点出发，无论往前还是往后，都能遍历整个链表。

## 第三页（P47）—— 循环链表的结构图与遍历终止条件

### 结构图解读

PPT中的图展示了一个带头节点的循环链表：

```
Head → [节点1] → [节点2] → [A] → [B] ─┐
↑                                       │
└───────────────────────────────────────┘
```

- `Head` 是头指针，指向第一个节点
- 最后一个节点 B 的 `next` 不是 NULL，而是指回 Head（第一个节点）
- 整个链表形成一个闭合的环

### 关键问题：如何判断遍历结束？

在普通单链表中，遍历终止条件很简单：

```c
// 普通单链表遍历
while (current != NULL) {
    // 处理 current
    current = current->next;
}
```

但循环链表没有 NULL，如果你用同样的条件，会陷入死循环，永远转圈。

**解决方法：检查当前节点的 next 是否等于 head。**

```c
// 循环链表遍历
struct Node *current = head->next;  // 从第一个数据节点开始（如果有头节点）

if (current != head) {  // 链表非空
    do {
        // 处理 current
        current = current->next;
    } while (current != head);  // 回到 head 就说明转了一圈
}
```

或者另一种写法（无头节点的情况）：

```c
struct Node *current = head;
do {
    // 处理 current
    current = current->next;
} while (current != head);  // 回到起点，遍历结束
```

核心逻辑：**起点就是终点**。从 head 出发，沿着 next 走，当再次遇到 head 时，说明已经遍历了一整圈。

### 注意事项

- 必须用 `do...while` 而不是 `while`，因为起始时 `current == head` 就是成立的，如果用 `while(current != head)` 会直接跳过循环体。
- 空链表需要单独判断（如果有头节点，空链表就是 `head->next == head`）。

## 第四页（P48）—— 循环链表的代码实现

### 节点结构定义

```c
typedef struct CLNode {
    ElemType data;          // 数据域，ElemType 是数据类型的别名（可以是 int、char 等）
    struct CLNode *next;    // 指针域，指向下一个节点
} CLNode, *CLinkList;       // CLNode 是节点类型，CLinkList 是指向节点的指针类型
```

这里 `CLinkList` 等价于 `CLNode *`，是为了语义清晰——当你声明一个链表时用 `CLinkList`，强调它代表"整个链表"（通过头指针）；当你声明一个节点时用 `CLNode *`，强调它是"某个节点"。

### 创建空循环链表

```c
void CreateList_CL(LinkList &CL)  // CL 是引用参数（C++ 风格），用于返回创建的链表
// create an empty list
{
    CL = (CLinkList)malloc(sizeof(CLNode));  // 分配一个头节点
    CL->next = CL;                           // 头节点的 next 指向自己
}
```

逐行解析：

**第一行：`CL = (CLinkList)malloc(sizeof(CLNode));`**

- `malloc(sizeof(CLNode))`：在堆上分配一个节点大小的内存
- `(CLinkList)`：将 `void *` 强制转换为 `CLNode *` 类型
- 这个节点就是**头节点（header node）**，它不存有效数据，只是作为链表的"入口标记"

**第二行：`CL->next = CL;`**

- 让头节点的 `next` 指向自己
- 这就是空循环链表的标志：只有一个头节点，它的 next 指回自己

图示：

```
    ┌───┐
    │   ↓
  [ CL ] ──→ 自己
    ↑   │
    └───┘
```

### 为什么空链表是 `CL->next = CL`？

这是循环链表的核心约定：

- 空链表：`head->next == head`（头节点指向自己，没有数据节点）
- 非空链表：`head->next` 指向第一个数据节点，最后一个数据节点的 `next` 指回 `head`

这样判断链表是否为空就很简单：

```c
int IsEmpty(CLinkList CL) {
    return CL->next == CL;  // 头节点的 next 指向自己就是空
}
```

### 补充：如果要往空循环链表中插入第一个元素

```c
void InsertFirst_CL(CLinkList CL, ElemType value) {
    CLNode *newNode = (CLNode *)malloc(sizeof(CLNode));
    newNode->data = value;
    newNode->next = CL->next;  // 新节点的 next 指向 CL 的下一个（此时是 CL 自己）
    CL->next = newNode;        // 头节点的 next 指向新节点
}
```

插入后的状态：

```
CL(头节点) → newNode → CL(头节点)
```

形成了一个包含一个数据节点的环。

## 总结：这四页的知识脉络

```
单链表的缺陷
├── 不能从尾部回到头部 ──→ 循环链表（尾节点 next 指回头部）
└── 不能反向遍历 ──→ 双向链表（每个节点多一个 prev 指针）

循环链表要点：
├── 最后一个节点的 next 指向第一个节点（或头节点）
├── 遍历终止条件：current->next == head
├── 空链表标志：head->next == head
└── 可以和双向链表结合使用
```
