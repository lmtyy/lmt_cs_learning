## 第一页（P65）—— CursorFree 操作与空间耗尽处理

### CursorFree：模拟 free

PPT说："执行 free 时，把该单元格放到空闲链表的头部。"

```c
void CursorFree(int p) {
    CursorSpace[p].next = CursorSpace[0].next;  // p 的 next 指向原来的第一个空闲节点
    CursorSpace[0].next = p;                     // 空闲链表头指向 p
}
```

这就是一个标准的**头插法**：把释放的节点插到空闲链表最前面。

**为什么用头插法？**
- O(1) 时间复杂度，不需要遍历
- 不需要维护尾指针
- 释放顺序和分配顺序无关，空闲链表不需要保持有序

**示例：** 假设当前空闲链表为 `0 → 3 → 4 → 5 → ... → 0(NULL)`，现在释放下标 7：

```
执行前：CursorSpace[0].next = 3
执行后：
  CursorSpace[7].next = 3    （7 指向原来的第一个空闲节点 3）
  CursorSpace[0].next = 7    （空闲链表头指向 7）
空闲链表变为：0 → 7 → 3 → 4 → 5 → ...
```

### 空间耗尽时的处理

PPT指出一个重要细节：**如果没有可用空间了，CursorAlloc 会正确地返回 0。**

回顾 CursorAlloc 的代码：

```c
int CursorAlloc(void) {
    int p;
    p = CursorSpace[0].next;                    // 取第一个空闲节点
    CursorSpace[0].next = CursorSpace[p].next;  // 空闲链表头后移
    return p;
}
```

当空闲链表为空时：
- `CursorSpace[0].next = 0`（空闲链表头的 next 是 0，表示没有空闲节点了）
- `p = CursorSpace[0].next = 0`
- 返回 0

**返回 0 的含义：**
- 0 在游标实现中等价于 NULL
- 调用者收到 0 就知道"分配失败，没有可用空间了"
- 类似于 `malloc` 返回 NULL 表示内存不足

**第二行变成 no-op（空操作）：**

当 `p = 0` 时，第二行 `CursorSpace[0].next = CursorSpace[0].next` 就是把自己赋值给自己，没有任何效果。这是一个巧妙的设计——不需要额外的 `if` 判断就能正确处理空间耗尽的情况。

| 7    | 8     |

| 9    | 10    |
| 10   | **0** |

- `free list head` 标注在下标 0 的位置，表示 `CursorSpace[0]` 是空闲链表的头
- 最后一个节点（下标 10）的 next 为 0，表示链表结束（NULL）
- 所有 11 个位置都在空闲链表中（因为还没有创建任何用户链表）

### 左侧绿色框的三个要点总结

**1. Freelist：不属于任何链表的单元格组成的空闲链表**

CursorSpace 中的每个单元格，在任何时刻，要么属于某个用户链表，要么属于空闲链表。两者互不重叠，合起来覆盖整个数组。

**2. 使用 cell 0 作为空闲链表的头节点**

- `CursorSpace[0]` 永远是空闲链表的头（哨兵）
- 它不存有效数据，只通过 `next` 字段指向第一个可用的空闲单元格
- 这意味着下标 0 永远不会被分配给用户使用

**3. next 值为 0 等价于 NULL**

- 因为下标 0 被保留为空闲链表头，不可能是有效数据节点
- 所以任何节点的 `next = 0` 就表示"没有下一个节点"
- 等价于普通链表中的 `next = NULL`

### 空闲链表的逻辑视图

```
CursorSpace[0] → CursorSpace[1] → CursorSpace[2] → ... → CursorSpace[10] → NULL(0)
     (头)           (空闲)           (空闲)                    (空闲)
```

## 第三页（P67）—— 用户链表使用头节点

### 核心约定

PPT说："有了上述基础，游标实现链表就很直接了。为了一致性，我们的链表将使用头节点（header node）来实现。"

### 什么意思？

当用户要创建一个链表时，需要先从空闲链表中分配一个节点作为该链表的**头节点**：

```c
int CreateList(void) {
    int header = CursorAlloc();       // 从空闲链表取一个节点作为头节点
    if (header == 0) return 0;        // 分配失败
    CursorSpace[header].next = 0;     // 头节点的 next 为 0（空链表）
    return header;                     // 返回头节点的下标
}
```

- 头节点不存有效数据，只是链表的入口标记
- 头节点的 `next` 指向第一个数据节点（如果有的话）
- 空链表：`CursorSpace[header].next == 0`

### 为什么要用头节点？

和普通链表使用头节点的理由一样：

1. **统一操作逻辑**：在头部插入和在中间插入的代码完全一致，不需要特殊处理
2. **空链表不需要特殊处理**：空链表就是头节点的 next 为 0
3. **链表的"身份标识"**：头节点的下标就代表了整个链表（类似普通链表中 head 指针代表整个链表）

### 多个链表共存

游标实现的一个重要特点是：**多个用户链表和空闲链表共享同一个 CursorSpace 数组。**

```
CursorSpace 数组（共 1000 个位置）
├── 空闲链表（从 CursorSpace[0] 开始）
├── 用户链表 A（头节点在某个下标，比如 3）
├── 用户链表 B（头节点在某个下标，比如 7）
└── 用户链表 C（头节点在某个下标，比如 5）
```

每个链表的节点散落在数组的各个位置，通过 `next` 字段串联起来。逻辑上是独立的链表，物理上共享同一块数组空间。

### 示例：创建链表并插入元素

```c
// 创建一个空链表
int myList = CreateList();  // 假设分配到下标 1

// 在头部插入元素 42
int newNode = CursorAlloc();  // 假设分配到下标 2
CursorSpace[newNode].data = 42;
CursorSpace[newNode].next = CursorSpace[myList].next;  // 新节点的 next = 0（原来是空的）
CursorSpace[myList].next = newNode;  // 头节点的 next 指向新节点

// 再插入元素 99
int newNode2 = CursorAlloc();  // 假设分配到下标 3
CursorSpace[newNode2].data = 99;
CursorSpace[newNode2].next = CursorSpace[myList].next;  // 新节点的 next = 2
CursorSpace[myList].next = newNode2;  // 头节点的 next 指向新节点
```

此时链表逻辑结构：`myList(1) → 99(3) → 42(2) → NULL(0)`

CursorSpace 状态：

| Slot | Data        | Next                                |
| ---- | ----------- | ----------------------------------- |
| 0    | —           | 4（空闲链表头，指向第一个空闲位置） |
| 1    | —（头节点） | 3                                   |
| 2    | 42          | 0                                   |
| 3    | 99          | 2                                   |
| 4    | —（空闲）   | 5                                   |
| ...  | ...         | ...                                 |

### 考试重点

1. **CursorAlloc 返回 0 的含义**：空间耗尽，等价于 malloc 返回 NULL
2. **CursorFree 的实现**：头插法归还空闲链表，O(1) 时间
3. **区分两种"链表"**：
   - 空闲链表（freelist）：管理未使用的单元格，头在 CursorSpace[0]
   - 用户链表：存储实际数据，头节点从空闲链表分配
4. **给定操作序列画出 CursorSpace 状态**：这是常见考题，比如"依次 alloc 3 次，free 1 次后，画出数组状态"
5. **多链表共存**：同一个 CursorSpace 数组中可以同时存在多个独立的用户链表 + 一个空闲链表，它们通过各自的 next 字段互不干扰

### 完整代码汇总（CursorAlloc + CursorFree）

```c
#define SpaceSize 1000

typedef struct Node {
    ElementType data;
    int next;
} Node;

Node CursorSpace[SpaceSize];

// 初始化：所有单元格串成空闲链表
void CursorInit(void) {
    for (int i = 0; i < SpaceSize - 1; i++) {
        CursorSpace[i].next = i + 1;
    }
    CursorSpace[SpaceSize - 1].next = 0;  // 最后一个指向 0（NULL）
}

// 模拟 malloc：从空闲链表头部取一个节点
int CursorAlloc(void) {
    int p = CursorSpace[0].next;
    CursorSpace[0].next = CursorSpace[p].next;
    return p;  // 返回 0 表示分配失败
}

// 模拟 free：头插法归还到空闲链表
void CursorFree(int p) {
    CursorSpace[p].next = CursorSpace[0].next;
    CursorSpace[0].next = p;
}

// 创建一个带头节点的空链表
int CreateList(void) {
    int header = CursorAlloc();
    if (header == 0) return 0;       // 空间不足
    CursorSpace[header].next = 0;    // 空链表
    return header;
}

// 判断链表是否为空
int IsEmpty(int header) {
    return CursorSpace[header].next == 0;
}
```

### 与普通链表的完整对照表

| 概念         | 普通链表             | 游标实现                      |
| ------------ | -------------------- | ----------------------------- |
| 存储空间     | 堆（heap）           | 全局数组 CursorSpace[]        |
| 节点地址     | 内存地址（指针）     | 数组下标（int）               |
| 空指针       | NULL                 | 0                             |
| 分配节点     | malloc(sizeof(Node)) | CursorAlloc()                 |
| 释放节点     | free(p)              | CursorFree(p)                 |
| 访问数据     | p->data              | CursorSpace[p].data           |
| 访问后继     | p->next              | CursorSpace[p].next           |
| 移动到下一个 | p = p->next          | p = CursorSpace[p].next       |
| 空闲空间管理 | 操作系统/运行时库    | 自己维护的 freelist           |
| 链表标识     | head 指针            | header 节点的下标             |
| 空链表判断   | head->next == NULL   | CursorSpace[header].next == 0 |
