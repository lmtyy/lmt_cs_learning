这页PPT是对**双向链表（DLL, Doubly Linked List）与单链表（SLL, Singly Linked List）**的优缺点总结，属于前面几页内容的收尾对比。

---

## 优势（Advantages）

### 1. 可以双向遍历

双向链表每个节点有 `prior` 和 `next` 两个指针，所以从任意节点出发，既可以往后走也可以往前走。

```c
// 正向遍历
while (p != NULL) { p = p->next; }

// 反向遍历
while (p != NULL) { p = p->prior; }
```

单链表只能从前往后，如果需要反向访问，必须从头重新遍历。

**什么场景下"双向遍历"是必须的？**

- 浏览器前进/后退
- 文本编辑器中光标的左移/右移
- 音乐播放器的上一首/下一首
- 操作系统中双向调度队列

### 2. 删除和前插操作更简单

**删除操作：**

单链表要删除节点 p，必须先找到它的前驱（从头遍历 O(n)）：

```c
// 单链表：已知 p，要删除它
// 必须先找到 prev 使得 prev->next == p
prev->next = p->next;
free(p);
```

双向链表直接通过 `p->prior` 拿到前驱，O(1)：

```c
// 双向链表：已知 p，直接删除
p->prior->next = p->next;
p->next->prior = p->prior;
free(p);
```

**在某节点前面插入：**

单链表只能做"后插"（在某节点后面插入），要做"前插"同样需要找前驱。双向链表可以直接前插，因为 `p->prior` 就是前驱。

---

## 劣势（Disadvantages）

### 1. 需要更多空间

每个节点多了一个指针域：

|                       | 单链表节点      | 双向链表节点        |
| --------------------- | --------------- | ------------------- |
| 结构                  | data + next     | data + prior + next |
| 64位系统大小（存int） | 4 + 8 = 12 字节 | 4 + 8 + 8 = 20 字节 |
| 对齐后                | 16 字节         | 24 字节             |

空间开销增加了约 50%。如果数据本身很小（比如存 char），指针的额外开销占比就非常大。

### 2. 链表操作更慢

虽然时间复杂度相同（都是 O(1) 的指针操作），但双向链表每次插入需要修改 4 个指针，删除需要修改 2 个指针；而单链表插入只改 2 个，删除只改 1 个。

常数因子翻倍，在大量操作时会有可感知的性能差异。

### 3. 更容易出 bug

这是工程实践中很现实的问题。双向链表需要维护的指针更多，写代码时：

- 插入要改 4 个指针，顺序还有约束（不能先覆盖 `p->prior`）
- 删除要改 2 个指针，还要注意边界（尾节点的 `next` 为 NULL）
- 每次操作都要同时维护正向和反向两个方向的一致性

一旦某个指针忘了改或者改错了顺序，链表就会断裂或形成环，而且这类 bug 往往不会立即崩溃，而是在后续遍历时才暴露，很难调试。

---

## 总结：如何选择？

| 场景                                       | 选择                   |
| ------------------------------------------ | ---------------------- |
| 只需要正向遍历，插入删除都在已知前驱之后   | 单链表（简单、省空间） |
| 需要双向遍历，或需要 O(1) 删除任意已知节点 | 双向链表               |
| 实现栈（只在一端操作）                     | 单链表足够             |
| 实现 LRU 缓存（需要快速删除+移动节点）     | 双向链表               |
| 内存极度紧张、节点数量巨大                 | 单链表                 |

核心权衡就是一句话：**用空间和代码复杂度换取操作的灵活性。**
