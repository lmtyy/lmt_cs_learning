## 第一页（P104）—— 练习题1：多项式加法

### 题目

> 写一个函数将两个多项式相加。不要破坏输入。使用链表实现。如果两个多项式分别有 M 和 N 项，你的程序的时间复杂度是多少？

### 关键要求

- **不要破坏输入**：之前PPT中的 AddPolyn 是就地修改 Pa、消耗 Pb 的。这道题要求保留原始输入不变，所以需要**创建一个新链表**存储结果。

### 思路

```c
polynomial AddPoly(polynomial Pa, polynomial Pb) {
    // 创建结果链表 Pc
    polynomial Pc = CreateEmptyList();
    position qa = FirstPos(Pa);  // Pa 的第一个数据节点
    position qb = FirstPos(Pb);  // Pb 的第一个数据节点
    position tail = GetHead(Pc); // Pc 的尾部（用于尾插）

    while (qa && qb) {
        if (qa->expn < qb->expn) {
            // 复制 Pa 的当前项到 Pc
            InsertAfter(tail, NewNode(qa->coef, qa->expn));
            tail = tail->next;
            qa = qa->next;
        }
        else if (qa->expn > qb->expn) {
            // 复制 Pb 的当前项到 Pc
            InsertAfter(tail, NewNode(qb->coef, qb->expn));
            tail = tail->next;
            qb = qb->next;
        }
        else {  // 指数相同
            float sum = qa->coef + qb->coef;
            if (abs(sum) >= 1E-6) {  // 系数不为零
                InsertAfter(tail, NewNode(sum, qa->expn));
                tail = tail->next;
            }
            qa = qa->next;
            qb = qb->next;
        }
    }
    // 复制剩余项
    while (qa) { InsertAfter(tail, NewNode(qa->coef, qa->expn)); tail = tail->next; qa = qa->next; }
    while (qb) { InsertAfter(tail, NewNode(qb->coef, qb->expn)); tail = tail->next; qb = qb->next; }

    return Pc;
}
```

### 时间复杂度

**O(M + N)**

- 两个指针各自最多遍历一遍自己的链表
- 每个节点最多被访问一次
- 总操作次数不超过 M + N

## 第二页（P105）—— 练习题2和3

### 题目2：在单链表中查找特定元素（递归 + 非递归）

> 写一个程序在单链表中查找特定元素。分别用递归和非递归方式实现，比较运行时间。链表要多大递归版本才会崩溃？

### 非递归版本

```c
Node* find_iterative(Node *head, int target) {
    Node *p = head;
    while (p != NULL) {
        if (p->data == target)
            return p;
        p = p->next;
    }
    return NULL;  // 没找到
}
```

时间复杂度：O(n)，空间复杂度：O(1)

### 递归版本

```c
Node* find_recursive(Node *p, int target) {
    if (p == NULL)
        return NULL;           // 基准情况：到达链表末尾
    if (p->data == target)
        return p;              // 找到了
    return find_recursive(p->next, target);  // 递归查找下一个
}
```

时间复杂度：O(n)，空间复杂度：O(n)（递归调用栈）

### 递归版本什么时候崩溃？

递归每深入一层就在调用栈上压入一个栈帧。当链表长度超过系统栈的最大深度时，会发生**栈溢出（Stack Overflow）**。

- 典型系统栈大小：1MB ~ 8MB
- 每个栈帧大小：约 32~64 字节（取决于局部变量和参数）
- 大约 **10,000 ~ 100,000** 个节点时递归版本就会崩溃

具体数值取决于操作系统和编译器设置。可以通过实验测试。

### 运行时间对比

虽然两者时间复杂度都是 O(n)，但递归版本更慢：
- 每次递归调用有函数调用开销（保存寄存器、压栈、跳转）
- 递归深度大时缓存命中率低
- 非递归版本只有一个简单的 while 循环，CPU 流水线效率高

### 题目3：非递归反转单链表，O(N) 时间，O(1) 额外空间

> 写一个非递归过程，在 O(N) 时间内使用常数额外空间反转单链表。

### 思路：三指针法

用三个指针 `prev`、`curr`、`next` 逐个翻转每个节点的 next 指向：

```c
Node* reverse(Node *head) {
    Node *prev = NULL;
    Node *curr = head;
    Node *next = NULL;

    while (curr != NULL) {
        next = curr->next;    // 保存下一个节点
        curr->next = prev;    // 当前节点的 next 反转，指向前一个
        prev = curr;          // prev 前进
        curr = next;          // curr 前进
    }
    return prev;  // prev 现在是新的头节
}
```

### 执行过程示例

原始链表：`1 → 2 → 3 → 4 → NULL`

```
初始：prev=NULL, curr=1, next=NULL

第1次循环：
  next = 2
  1->next = NULL (指向prev)
  prev = 1, curr = 2
  状态：NULL ← 1   2 → 3 → 4 → NULL

第2次循环：
  next = 3
  2->next = 1 (指向prev)
  prev = 2, curr = 3
  状态：NULL ← 1 ← 2   3 → 4 → NULL

第3次循环：
  next = 4
  3->next = 2 (指向prev)
  prev = 3, curr = 4
  状态：NULL ← 1 ← 2 ← 3   4 → NULL

第4次循环：
  next = NULL
  4->next = 3 (指向prev)
  prev = 4, curr = NULL
  状态：NULL ← 1 ← 2 ← 3 ← 4

循环结束，返回 prev = 4
结果：4 → 3 → 2 → 1 → NULL
```

### 复杂度分析

- 时间：O(N)，每个节点恰好访问一次
- 空间：O(1)，只用了三个额外指针变量

## 第三页（P106）—— 练习题4：懒惰删除（Lazy Deletion）

### 题目

> 一种替代删除策略是使用懒惰删除：删除元素时，仅用一个额外的 bit 字段将其标记为"已删除"。数据结构中维护已删除和未删除元素的计数。当已删除元素数量等于未删除元素数量时，遍历整个链表，对所有标记节点执行真正的删除。
>
> a. 列出懒惰删除的优缺点。
> b. 写出使用懒惰删除的标准链表操作的实现。

### 懒惰删除的节点结构

```c
struct Node {
    int data;
    int deleted;         // 0 = 有效，1 = 已标记删除
    struct Node *next;
};

struct LazyList {
    Node *head;
    int activeCount;     // 未删除的元素数
    int deletedCount;    // 已标记删除的元素数
};
```

### a. 优缺点分析

**优点：**

1. **删除操作 O(1)**：不需要找前驱，不需要修改指针，只需设置标记位
2. **支持"撤销删除"**：如果需要恢复，只需把标记位改回来
3. **避免频繁的内存分配/释放**：减少 malloc/free 的调用次数，降低内存碎片
4. **批量删除效率高**：积累到一定数量后一次性清理，均摊代价低

**缺点：**

1. **空间浪费**：已标记删除的节点仍然占用内存，直到真正清理
2. **遍历变慢**：查找、遍历时需要跳过已删除节点，实际遍历长度可能是有效元素的两倍
3. **每个节点多一个 bit/字段**：额外空间开销
4. **逻辑复杂**：所有操作（查找、插入、计数）都需要考虑"跳过已删除节点"

### b. 核心操作实现

**删除操作（懒惰）：**

```c
void LazyDelete(LazyList *L, int target) {
    Node *p = L->head->next;
    while (p != NULL) {
        if (p->data == target && !p->deleted) {
            p->deleted = 1;       // 标记删除
            L->deletedCount++;
            L->activeCount--;
            break;
        }
        p = p->next;
    }
    // 检查是否需要真正清理
    if (L->deletedCount >= L->activeCount) {
        RealDelete(L);            // 执行真正的批量删除
    }
}
```

**真正的批量删除：**

```c
void RealDelete(LazyList *L) {
    Node *prev = L->head;
    Node *curr = L->head->next;
    while (curr != NULL) {
        if (curr->deleted) {
            prev->next = curr->next;  // 绕过已删除节点
            Node *temp = curr;
            curr = curr->next;
            free(temp);               // 真正释放内存
        } else {
            prev = curr;
            curr = curr->next;
        }
    }
    L->deletedCount = 0;  // 清零计数
}
```

**查找操作（跳过已删除节点）：**

```c
Node* LazyFind(LazyList *L, int target) {
    Node *p = L->head->next;
    while (p != NULL) {
        if (!p->deleted && p->data == target)
            return p;
        p = p->next;
    }
    return NULL;
}
```

**判空操作：**

```c
int LazyIsEmpty(LazyList *L) {
    return L->activeCount == 0;
}
```

### 触发真正删除的条件

> "当已删除元素数量等于未删除元素数量时"

即 `deletedCount >= activeCount` 时触发清理。此时链表中至少一半的节点是无效的，遍历效率已经很低，值得花 O(n) 时间做一次彻底清理。

### 均摊分析

- 每次懒惰删除：O(n)（查找目标）+ O(1)（标记）
- 真正删除触发频率：每删除 n/2 次触发一次，代价 O(n)
- 均摊到每次删除：O(n) / (n/2) = O(1) 的额外清理代价
