## 第一页（P56）—— 删除操作的指针修改过程

### 场景描述

要删除节点 `p`（存储数据 `ai`），它的前驱是 `ai-1`，后继是 `ai+1`。

删除前的状态：

```
... ⇄ [ai-1] ⇄ [ai(p)] ⇄ [ai+1] ⇄ ...
```

删除后的目标状态：

```
... ⇄ [ai-1] ⇄ [ai+1] ⇄ ...
```

本质就是让 `ai-1` 和 `ai+1` 直接相连，跳过 `p`，然后释放 `p` 的内存。

### 三步操作

```c
p->prior->next = p->next;   // ① p的前驱的后继 = p的后继
p->next->prior = p->prior;  // ② p的后继的前驱 = p的前驱
free(p);                     // ③ 释放p的内存
```

### 逐步图解

**初始状态：**

```
[ai-1] ⇄ [ai(p)] ⇄ [ai+1]

ai-1->next = p
p->prior = ai-1
p->next = ai+1
ai+1->prior = p
```

**第①步：`p->prior->next = p->next`**

含义：p 的前驱节点（ai-1）的 next 指针，从指向 p 改为指向 p 的后继（ai+1）。

```
[ai-1] ──next──→ [ai+1]
         （跳过了p）
[ai(p)] 的 prior 和 next 暂时没变，但 ai-1 已经不指向它了
```

对应图中的虚线 ①：ai-1 的 next 直接连到 ai+1。

**第②步：`p->next->prior = p->prior`**

含义：p 的后继节点（ai+1）的 prior 指针，从指向 p 改为指向 p 的前驱（ai-1）。

```
[ai-1] ⇄ [ai+1]
         （双向连接完成）
[ai(p)] 现在完全被孤立了，没有任何节点指向它
```

对应图中的虚线 ②：ai+1 的 prior 直接连到 ai-1。

**第③步：`free(p)`**

释放节点 p 占用的堆内存，防止内存泄漏。

### 与插入操作的对比

|            | 插入（在 p 前插入 q）      | 删除（删除 p）             |
| ---------- | -------------------------- | -------------------------- |
| 指针操作数 | 4 条                       | 2 条（+ 1 条 free）        |
| 复杂度     | 需要注意顺序               | 顺序无所谓                 |
| 核心思想   | 把新节点"嵌入"两个节点之间 | 让前驱和后继"绕过"当前节点 |

### 为什么删除操作的两条指针语句顺序无所谓？

```c
p->prior->next = p->next;   // ①
p->next->prior = p->prior;  // ②
```

- 第①步读取 `p->prior` 和 `p->next`
- 第②步也读取 `p->next` 和 `p->prior`
- 这两步都只修改了 p 的邻居节点的指针，没有修改 p 本身的 `prior` 和 `next`
- 所以无论先执行①还是先执行②，读取到的 `p->prior` 和 `p->next` 都是正确的旧值

交换顺序完全合法：

```c
// 这样写也对
p->next->prior = p->prior;  // ② 先执行
p->prior->next = p->next;   // ① 后执行
free(p);
```

这一点和插入操作不同——插入时如果顺序错了会覆盖关键指针。

### 双向链表删除 vs 单链表删除

**单链表删除节点 p 的困境：**

```c
// 单链表：要删除 p，必须知道 p 的前驱 prev
prev->next = p->next;
free(p);
```

问题是：如果你只有 `p` 的指针，你不知道 `prev` 是谁。必须从头遍历找到 `prev`，时间 O(n)。

**双向链表删除节点 p：**

```c
// 双向链表：p 自带前驱信息，直接操作
p->prior->next = p->next;
p->next->prior = p->prior;
free(p);
```

不需要遍历，直接 O(1) 完成。这就是双向链表多花一个指针空间换来的核心收益。

## 第二页（P57）—— 删除操作的完整函数实现

### 函数签名

```c
Status ListDelete_DL(DLinkList &L, int i, ElemType &e)
```

- `DLinkList &L`：双向链表的引用
- `int i`：要删除的位置（第 i 个节点）
- `ElemType &e`：引用参数，用于返回被删除节点的数据值（让调用者知道删了什么）
- 返回值 `Status`：OK 或 ERROR

好的，接着上面继续：

---

### 代码逐行解析

```c
Status ListDelete_DL(DLinkList &L, int i, ElemType &e)
{
    // 第1步：定位第 i 个节点
    if (!(p = GetElemP_DL(L, i)))
        // if p=NULL the i'th node is not existent
        return ERROR;
```

- `GetElemP_DL(L, i)` 从头节点开始遍历，返回第 i 个节点的指针
- 如果 i 超出范围（比如 i < 1 或 i > length），返回 NULL
- 此时函数返回 ERROR，表示删除失败

```c
    // 第2步：保存被删除节点的数据
    e = p->data;
```

- 把即将被删除的节点的数据值赋给 `e`
- 因为 `e` 是引用参数，调用者可以拿到这个值
- 这是一种常见的设计模式：删除的同时"取出"数据，类似栈的 pop 操作

```c
    // 第3步：修改指针，绕过节点 p
    p->prior->next = p->next;
    p->next->prior = p->prior;
```

- 第一条：p 的前驱的 next 跳过 p，直接指向 p 的后继
- 第二条：p 的后继的 prior 跳过 p，直接指向 p 的前驱
- 执行完后，p 被完全孤立，链表中没有任何节点再指向它

```c
    // 第4步：释放内存
    free(p);
    
    return OK;
} // ListDelete_DL
```

- 释放 p 占用的堆内存
- 返回 OK 表示删除成功

### 完整代码汇总

```c
Status ListDelete_DL(DLinkList &L, int i, ElemType &e)
{
    if (!(p = GetElemP_DL(L, i)))
        return ERROR;
    
    e = p->data;
    
    p->prior->next = p->next;
    p->next->prior = p->prior;
    free(p);
    
    return OK;
}
```

### 时间复杂度

- `GetElemP_DL(L, i)`：遍历到第 i 个节点，O(n)
- 指针修改 + free：O(1)
- 总体：**O(n)**

同样，如果你已经持有目标节点的指针（不需要定位），删除本身是 O(1)。

### 边界情况分析

**删除第一个数据节点（i = 1）：**

```
Head(头节点) ⇄ [a1(p)] ⇄ [a2] ⇄ ...
```

- `p->prior` = Head（头节点）
- `p->prior->next = p->next` → Head->next = a2 ✓
- `p->next->prior = p->prior` → a2->prior = Head ✓
- 正常工作，不需要特殊处理（这就是头节点的好处）

**删除最后一个数据节点（i = length）：**

```
... ⇄ [an-1] ⇄ [an(p)]
                  p->next = NULL
```

这里有个问题：`p->next->prior = p->prior` 中，`p->next` 是 NULL，对 NULL 解引用会导致**段错误（Segmentation Fault）**。

所以严格来说，代码应该加一个判断：

```c
p->prior->next = p->next;          // 这条总是安全的（前驱一定存在，至少是头节点）
if (p->next != NULL)                // 后继可能为 NULL（删除尾节点时）
    p->next->prior = p->prior;
free(p);
```

PPT中的代码省略了这个边界检查，考试时如果要求严谨，建议加上。

**删除唯一的数据节点（链表只有一个元素）：**

```
Head ⇄ [a1(p)]    p->next = NULL
```

- `p->prior->next = p->next` → Head->next = NULL ✓
- `p->next->prior = p->prior` → NULL->prior → 段错误！

同样需要上面的 NULL 检查。

### 与插入函数的对称性

|          | ListInsert_DL       | ListDelete_DL       |
| -------- | ------------------- | ------------------- |
| 定位     | GetElemP_DL(L, i)   | GetElemP_DL(L, i)   |
| 内存操作 | malloc（分配）      | free（释放）        |
| 数据操作 | s->data = e（写入） | e = p->data（取出） |
| 指针操作 | 4条                 | 2条                 |
| 返回     | OK / ERROR          | OK / ERROR          |

## 两页知识脉络总结

```
双向链表删除（P56-P57）
│
├── 核心操作（只需2条指针语句 + 1条free）
│   ① p->prior->next = p->next   （前驱绕过p）
│   ② p->next->prior = p->prior  （后继绕过p）
│   ③ free(p)                     （释放内存）
│
├── 顺序无关性
│   ①②可以互换，因为都只读取 p 的指针域，不修改 p 本身
│
├── 与单链表对比
│   单链表删除需要先找前驱 O(n)，双向链表自带前驱 O(1)
│
├── 完整函数流程
│   定位(O(n)) → 保存数据 → 改指针(O(1)) → 释放内存
│
└── 边界注意
    删除尾节点时 p->next 为 NULL，需要判空防止段错误
```

### 考试重点

1. **手写删除代码**：给定节点 p，写出删除它的指针操作。只有两条核心语句，比插入简单。
2. **边界处理**：面试或笔试中如果要求写"健壮"的代码，记得对 `p->next` 判空。
3. **free 的位置**：必须在指针修改之后。如果先 free(p)，再访问 `p->prior` 和 `p->next` 就是访问已释放内存（use-after-free），属于未定义行为。
