## 第12页：用随机投点法计算π值——原理

### 几何模型

设有一个半径为 r 的圆及其**外切正方形**：
- 正方形面积 = (2r)² = 4r²
- 圆面积 = πr²

### 概率推导

向正方形随机投掷 n 个点，设落入圆内的点数为 k。

由于投入的点在正方形上**均匀分布**，点落入圆内的概率为：

```
P = 圆面积 / 正方形面积 = πr² / 4r² = π/4
```

当 n 足够大时，k/n 就逼近这一概率 π/4，从而：

```
π ≈ 4k/n
```

> 💡 直觉理解：想象你蒙着眼睛往一块正方形靶子上扔飞镖，数一下有多少落在了内切圆里。扔的次数越多，"落圆内的比例"就越接近 π/4。

> ✅ 核心公式：**π ≈ 4k/n**（k = 圆内点数，n = 总投点数）

### 为什么只用第一象限？

PPT图示中(b)展示了只取四分之一圆（第一象限）的情况：
- 正方形变为 r×r，面积 = r²
- 四分之一圆面积 = πr²/4
- 概率仍然是 π/4，公式不变

## 第13-15页：代码实现

### 辅助函数：生成 [start, end) 范围的随机实数

```cpp
#include <iostream.h>
#include <stdlib.h>
#include <time.h>

double random(double start, double end)
{
    return start + (end - start) * rand() / (RAND_MAX + 1.0);
}
```

> 💡 `rand()/(RAND_MAX+1.0)` 产生 [0,1) 的均匀随机数，再线性映射到 [start, end)。不加1.0，rand可能随机到RAND_MAX导致结果取到上限end(end应是无限逼近但取不到的)，加了1.0后最大值变为RAND_MAX/(RAND_MAX+1.0)，可视为一个非常趋近1但取不到1的数字）

---

### 核心函数：计算π

```cpp
double pi(int n)
{
    // 用随机投点法计算π
    int k = 0;
    for (int i = 1; i <= n; i++)
    {
        double x = random(0, 1);
        double y = random(0, 1);
        if ((x * x + y * y) <= 1)  // 判断是否落在四分之一圆内
            k++;
    }
    return 4 * k / (double)n;
}
```

**逐行拆解：**

| 行 | 作用 |
|----|------|
| `int k = 0` | 计数器：落在圆内的点数 |
| `random(0,1)` × 2 | 在单位正方形 [0,1)×[0,1) 内随机取点 (x,y) |
| `x*x + y*y <= 1` | 判断点是否在以原点为圆心、半径为1的四分之一圆内 |
| `k++` | 在圆内则计数+1 |
| `4*k/(double)n` | 用公式 π ≈ 4k/n 计算估计值 |

> ✅ 判断条件 `x²+y² ≤ 1` 就是圆的方程（单位圆），无需算距离再比较。

## 精度与投点数的关系

| 投点数 n | 估计精度（大约） |
|----------|----------------|
| 100 | ±0.3 |
| 10,000 | ±0.03 |
| 1,000,000 | ±0.003 |
| 10⁸ | ±0.0003 |

> 💡 收敛速度：误差大约与 1/√n 成正比。要多一位精度，投点数需要增加 100 倍。这就是为什么数值概率算法说"精度随时间增加而提高"——但提高得比较慢。

## 总结

```
随机投点法计算π
├── 原理：均匀随机投点 → 落圆内的比例 ≈ π/4
├── 公式：π ≈ 4k/n
├── 只用第一象限：x²+y² ≤ 1 判断是否在圆内
├── 代码结构
│   ├── random(0,1) 产生均匀随机坐标
│   ├── 判断 x²+y² ≤ 1
│   └── 统计比例 × 4
└── 精度：误差 ∝ 1/√n（收敛较慢）
```

> ✅ 这是数值概率算法的典型特征：结果是近似解，精度随投入计算量（n）增大而提高，但收敛速度有限。
