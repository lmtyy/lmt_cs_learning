## 第25页：蒙特卡罗算法——基本概念

### 应用场景

在实际应用中常会遇到一些问题，不论采用确定性算法或概率算法都**无法保证每次都能得到正确的解答**。

### 蒙特卡罗算法的特点

- 在一般情况下可以保证对问题的所有实例都以**高概率**给出正确解
- 但是通常**无法判定一个具体解是否正确**，并且只适用于二元判定题。对多元输出如彩票中奖号码无能为力

> 💡 与拉斯维加斯算法的关键区别：拉斯维加斯"要么对要么没有"，蒙特卡罗"一定给答案但可能错"。

---

## 第26页：蒙特卡罗算法——p正确、优势、一致性

### p正确的蒙特卡罗算法

设 p 是一个实数，且 **1/2 < p < 1**。

如果一个蒙特卡罗算法对于问题的任一实例得到正确解的概率**不小于 p**，则称该蒙特卡罗算法是 **p 正确的**。

**优势（advantage）** = p - 1/2

> 💡 为什么要求 p > 1/2？因为如果 p ≤ 1/2，还不如抛硬币瞎猜。优势就是"比瞎猜好多少"。

### 一致的蒙特卡罗算法

如果对于同一实例，蒙特卡罗算法**不会给出2个不同的正确解答**，则称该算法是**一致的**。

> 💡 "一致"意味着：如果算法给出正确答案，那答案是唯一确定的（不会这次对了输出A，下次对了输出B）。

### 提高正确率的方法

对于一个一致的 p 正确蒙特卡罗算法，要提高获得正确解的概率：
- 只要执行该算法**若干次**
- 并选择**出现频次最高的解**即可

## 第27页：素数测试——Wilson定理

### Wilson定理

对于给定的正整数 n，判定 n 是素数的**充要条件**是：

```
(n-1)! ≡ -1 (mod n)
```

### 缺点

计算量太大，**无法实现对较大素数的测试**。

> 💡 计算 (n-1)! 需要 O(n) 次乘法，对于大整数（如几百位）完全不可行。

---

## 第28页：素数测试——费尔马小定理

### 费尔马小定理

如果 p 是一个素数，且 0 < a < p，则：

```
a^(p-1) ≡ 1 (mod p)
```

### 示例

67 是素数，2⁶⁶ mod 67 = 1 ✅

### 缺点

费尔马小定理只是素数判定的一个**必要条件**（不是充分条件）。

满足费尔马小定理条件的整数 n **未必全是素数**。

> ⚠️ 反例：**561**（= 3 × 11 × 17）是合数，但对所有与561互素的 a 都满足 a⁵⁶⁰ ≡ 1 (mod 561)。这类数叫 **Carmichael 数**。

## 第29页：素数测试——二次探测定理

### 二次探测定理

如果 p 是素数，且 0 < x < p，则方程：

```
x² ≡ 1 (mod p)
```

的解为 **x = 1** 或 **x = p - 1**。

### 证明思路

```
x² ≡ 1 (mod p)
⟹ x² - 1 ≡ 0 (mod p)
⟹ (x+1)(x-1) ≡ 0 (mod p)
⟹ p 必整除 x-1 或 x+1
⟹ 由 p 是素数且 0 < x < p，得 x = 1 或 x = p-1
```

### 应用方法

利用二次探测定理，可以在利用费尔马小定理计算 a^(n-1) mod n 的过程中增加对整数 n 的**二次探测**。

一旦发现违背二次探测条件，即可得出 **n 不是素数**的结论。反之，如果通过了二次探测，就认为是素数，即使可能出错。

> ✅ 关键思想：在计算 a^(n-1) mod n 的快速幂过程中，每次平方时顺便检查"x² mod n = 1 但 x ≠ 1 且 x ≠ n-1"，如果出现则 n 一定是合数。

## 第30页：素数测试——代码实现

### power 函数：计算 aᵖ mod n 并实施二次探测

```java
private static int power(int a, int p, int n)
{   // 计算 a^p mod n，并实施对n的二次探测
    int x, result;
    if (p == 0)
        result = 1;
    else
    {
        x = power(a, p/2, n);       // 递归计算
        result = (x * x) % n;       // 二次探测
        if ((result == 1) && (x != 1) && (x != n-1))
            composite = true;       // 违背二次探测 → 合数
        if ((p % 2) == 1)           // p是奇数
            result = (result * a) % n;
    }
    return result;
}
```

数学上保证：如果 n是合数，且a 是“见证者（Witness）”，那么在计算 a^{n−1} 的平方链中，必然存在某一层 x 使得 x^2≡1(modn)但 x≢±1(modn)。

二次探测的关键判断：**

- `result = x² mod n`
- 如果 `result == 1`（即 x² ≡ 1 mod n）
- 但 `x != 1` 且 `x != n-1`
- → 违背二次探测定理 → n 一定不是素数

### prime 函数：素数测试的蒙特卡罗算法

```java
public static boolean prime(int n)
{   // 素数测试的蒙特卡罗算法
    rnd = new Random();
    int a, result;
    composite = false;
    a = rnd.random(n - 3) + 2;      // 随机选取 a ∈ [2, n-2]
    result = power(a, n-1, n);       // 费尔马小定理: a^(n-1) mod n
    if (composite || (result != 1))
        return false;                // 合数
    else
        return true;                 // 可能是素数
}
```

**执行逻辑：**

1. 随机选取 a ∈ [2, n-2]
2. 计算 a^(n-1) mod n，过程中实施二次探测
3. 如果发现违背二次探测（composite = true）→ 返回 false（合数）
4. 如果 a^(n-1) mod n ≠ 1（违背费尔马小定理）→ 返回 false（合数）
5. 否则返回 true（可能是素数）

> ✅ 这就是 **Miller-Rabin 素数测试**的核心思想：费尔马小定理 + 二次探测。

## 第31页：算法性能分析

### 偏假性质

算法 prime 是一个**偏假 3/4 正确**的蒙特卡罗算法。

> "偏假"含义：如果算法说"是合数"，则**一定正确**；如果说"是素数"，则**可能错误**（错误概率 ≤ 1/4）。

### 重复调用降低错误率

通过多次重复调用，错误概率不超过 **(1/4)ᵏ**。

| 调用次数 k | 错误概率上界 |
|-----------|------------|
| 1 | 1/4 = 25% |
| 5 | (1/4)⁵ ≈ 0.1% |
| 10 | (1/4)¹⁰ ≈ 10⁻⁶ |
| 20 | (1/4)²⁰ ≈ 10⁻¹² |

> ✅ 这是一个很保守的估计，实际使用的效果要好得多。

> 💡 实际中调用 20~30 次就足以让错误概率低于宇宙射线导致计算机出错的概率，所以可以认为"实际上是正确的"。

## 总结

```
蒙特卡罗算法 & 素数测试
├── 蒙特卡罗算法概念
│   ├── p正确：正确概率 ≥ p（p > 1/2）
│   ├── 优势 = p - 1/2
│   ├── 一致性：不会给出多个不同正确解
│   └── 提高正确率：多次执行 + 取众数
├── 素数测试
│   ├── Wilson定理：(n-1)! ≡ -1 (mod n) —— 充要但不实用
│   ├── 费尔马小定理：a^(p-1) ≡ 1 (mod p) —— 必要非充分
│   │   └── 反例：Carmichael数（561）
│   ├── 二次探测定理：x² ≡ 1 (mod p) → x=1 或 x=p-1
│   └── Miller-Rabin算法 = 费尔马 + 二次探测
│       ├── power(): 快速幂 + 每次平方时二次探测
│       └── prime(): 随机选a，检查两个条件
└── 性能：偏假3/4正确，k次调用错误率 ≤ (1/4)^k
```

> ✅ 考试重点：

> 4. Miller-Rabin 代码中二次探测的位置和条件
> 5. 偏假 3/4 正确 + (1/4)ᵏ 错误率
