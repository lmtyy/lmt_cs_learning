# 随机数与线性同余法（p05-10）

这几页讲解了伪随机数的概念、线性同余法的原理，以及一个完整的 `RandomNumber` 类实现。

---

## 第5页：随机数

### 核心概念

- 随机数在概率算法设计中扮演着**十分重要的角色**
- 现实计算机上**无法产生真正的随机数**，因此概率算法中使用的随机数都是一定程度上随机的，即**伪随机数（Pseudo-random Number）**

> 💡 类比：真随机数就像掷骰子的物理过程，伪随机数则是用数学公式"模拟"出的看起来随机的序列。虽然本质上是确定的，但统计性质足够好，可以当作随机数来用。

---

## 第6页：线性同余法（Linear Congruential Method）

### 定义

**线性同余法**是产生伪随机数的最常用方法。

### 数学公式

由线性同余法产生的随机序列 a₀, a₁, ..., aₙ 满足：

```
a₀ = d
aₙ = (b × aₙ₋₁ + c) mod m    n = 1, 2, ...
```

### 参数约束

| 参数 | 约束 | 含义 |
|------|------|------|
| b | b ≥ 0 | 乘数（multiplier） |
| c | c ≥ 0 | 增量（adder） |
| d | d ≤ m | 种子（seed），序列的起始值 |
| m | 尽量大 | 模数，决定了序列的周期上限 |

### 参数选取经验

- **m 应取得充分大**，可取 m = 65536
- **gcd(m, b) = 1**，因此 b 应取一个素数

> ✅ 关键结论：b、c 和 m 的选取直接关系到所产生的随机序列的随机性能。选得不好，序列会快速出现周期性重复。

> 💡 直觉理解：线性同余法本质上就是一个"搅拌器"——把上一次的数乘以一个大数再加一个偏移，然后取余数。参数选得好，搅拌效果就好，序列就"看起来更随机"。

## 第7-10页：随机数类 RandomNumber 实现

### 类声明

```cpp
const unsigned long maxshort = 65536L;
const unsigned long multiplier = 1194211693L;
const unsigned long adder = 12345L;

class RandomNumber
{
private:
    unsigned long randSeed;  // 当前种子
public:
    // 构造函数，缺省值0表示由系统自动产生种子
    RandomNumber(unsigned long s = 0);
    // 产生0：n-1之间的随机整数
    unsigned short Random(unsigned long n);
    // 产生[0：1)之间的随机实数
    double fRandom(void);
};
```

### 参数对应关系

| 代码常量 | 对应线性同余法参数 | 值 |
|----------|-------------------|------|
| `maxshort` | m（模数） | 65536 |
| `multiplier` | b（乘数） | 1194211693 |
| `adder` | c（增量） | 12345 |
| `randSeed` | d / aₙ（当前种子） | 动态变化 |

---

### 构造函数：产生种子

```cpp
RandomNumber::RandomNumber(unsigned long s)
{
    if (s == 0)
        randSeed = time(0);  // 用当前时间作为种子
    else
        randSeed = s;        // 用户指定种子
}
```

> 💡 `time(0)` 返回当前时间的秒数，每次运行程序时不同，因此可以产生不同的随机序列。如果指定了固定种子，则每次运行结果相同（可复现）。

---

### Random：产生 0 ~ n-1 之间的随机整数

```cpp
unsigned short RandomNumber::Random(unsigned long n)
{
    randSeed = multiplier * randSeed + adder;
    return (unsigned short)((randSeed >> 16) % n);
}
```

**执行步骤拆解：**

1. **线性同余更新种子**：`randSeed = multiplier * randSeed + adder`
   - 对应公式 aₙ = b × aₙ₋₁ + c（这里利用 unsigned long 的自然溢出来隐式取模）
2. **右移16位**：`randSeed >> 16`
   - 取高位比特，得到 0 ~ 65535 之间的整数
   - 为什么右移？因为线性同余法**低位的随机性较差**，高位更随机
3. **映射到目标范围**：`% n`
   - 将 0~65535 映射到 0~(n-1)

> ✅ 关键技巧：右移16位取高位，是因为线性同余法的低位比特周期短、随机性差。

---

### fRandom：产生 [0, 1) 之间的随机实数

```cpp
double RandomNumber::fRandom(void)
{
    return Random(maxshort) / double(maxshort);
}
```

**执行步骤拆解：**

1. 调用 `Random(maxshort)` 产生一个 0 ~ (maxshort-1) 即 0 ~ 65535 之间的整数
2. 除以 `maxshort`（65536.0），得到 [0, 1) 区间的浮点数

> 💡 精度：产生的随机实数精度为 1/65536 ≈ 0.0000153，对于大多数概率算法已经足够。

## 总结

```
伪随机数生成
├── 为什么是"伪"？ ── 计算机无法产生真随机数
├── 线性同余法公式 ── aₙ = (b·aₙ₋₁ + c) mod m
│   ├── m 取大值（65536）
│   ├── gcd(m,b) = 1 → b 取素数
│   └── d 为种子（起始值）
└── RandomNumber 类实现
    ├── 构造函数 ── time(0) 或用户指定种子
    ├── Random(n) ── 线性同余 + 右移16位 + %n → [0, n-1]
    └── fRandom() ── Random(65536) / 65536.0 → [0, 1)
```

> ✅ 考试重点：线性同余法公式 + 参数选取原则 + 为什么要右移16位取高位
