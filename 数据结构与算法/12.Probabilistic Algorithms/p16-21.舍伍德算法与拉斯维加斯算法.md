# 舍伍德算法与拉斯维加斯算法（p16-21）

这几页介绍了舍伍德（Sherwood）算法的设计思想、随机洗牌技术，以及拉斯维加斯（Las Vegas）算法的基本框架和时间分析。

---

## 第16页：舍伍德算法——设计思想

### 问题背景

设 A 是一个确定性算法，当输入实例为 x 时所需的计算时间记为 t_A(x)。

设 Xₙ 是算法 A 的输入规模为 n 的实例全体，则平均时间为：

```
t̄_A(n) = Σ t_A(x) / |Xₙ|    （对所有 x ∈ Xₙ 求t_A(x)的和）
```

### 问题所在

这显然不能排除存在某个 x ∈ Xₙ 使得 t_A(x) >> t̄_A(n) 的可能性。

> 💡 就是说：确定性算法在**某些特定输入**上可能表现极差（如快排遇到已排序数组）。

### 舍伍德算法的目标

希望获得一个概率算法 B，使得对问题的输入规模为 n 的**每一个实例**均有：

```
t_B(x) = t̄_A(n) + s(n)
```

> ✅ 核心思想：**消除最坏情况**，让所有输入的运行时间都接近平均时间，只多付出一个小的随机化开销 s(n)。

当 s(n) 与 t̄_A(n) 相比可忽略时，舍伍德算法可获得很好的平均性能。

## 第17页：舍伍德算法——随机预处理

### 无法直接改造的情况

有时确定性算法无法直接改造成舍伍德型算法。此时可借助**随机预处理技术**：

- 不改变原有的确定性算法
- 仅对其输入进行**随机洗牌（shuffle）**
- 同样可收到舍伍德算法的效果

### 舍伍德算法的优点

> ✅ 计算时间复杂性对**所有实例**而言**相对均匀**。

> 💡 类比：确定性快排怕遇到有序数组（最坏 O(n²)），但如果排序前先把数组随机打乱，任何输入都会变成"随机情况"，期望时间就稳定在 O(n log n)。

## 第18页：快速排序算法——Partition

### Partition 伪代码

```
Algorithm Partition(A[l..r])
// 以第一个元素作为 pivot 对子数组进行划分
// Input: A[0..n-1] 的子数组 A[l..r]，l < r
// Output: A[l..r] 的划分，返回分割位置 j

p ← A[l]
i ← l;  j ← r + 1
repeat
    repeat i ← i + 1 until A[i] ≥ p
    repeat j ← j - 1 until A[j] ≤ p
    swap(A[i], A[j])
until i ≥ j
swap(A[i], A[j])    // undo last swap when i ≥ j
swap(A[l], A[j])
return j
```

> 💡 这就是标准快排的 Partition 过程。问题在于：如果 pivot 选得不好（如总是最小/最大元素），划分极不均匀，退化为 O(n²)。

## 第19页：舍伍德算法——随机洗牌 shuffle

### 思路

对于确定性选择算法，可以用**洗牌算法 shuffle** 将数组 a 中元素随机排列，然后用确定性选择算法求解。效果与舍伍德型算法一样。

### 洗牌算法实现

```java
public static void shuffle(Comparable[] a, int n)
{
    // 随机洗牌算法，左边是洗过一次的，右边是没洗的
    rnd = new Random();
    for (int i = 1; i < n; i++)// i从1开始，因为第0各元素作pivot
    {
        int j = rnd.random(n - i + 1) + i; // 在右边没交换的地方找个数交换
        MyMath.swap(a, i, j);
    }
}
```

**逐步拆解：**

| 步骤 | 操作 | 效果 |
|------|------|------|
| i = 1 | 从 [1, n] 随机选 j，swap(a[1], a[j]) | 第1个位置放了一个随机元素 |
| i = 2 | 从 [2, n] 随机选 j，swap(a[2], a[j]) | 第2个位置放了剩余中的随机元素 |
| ... | ... | ... |
| i = n-1 | 从 [n-1, n] 随机选 j | 最后两个位置随机排列 |

> ✅ 这是 **Fisher-Yates 洗牌算法**，能保证产生均匀的随机排列（每种排列出现的概率相同 = 1/n!）。

> 💡 舍伍德算法的套路：**先 shuffle 再用原算法**，就能把最坏情况的概率"摊平"到所有输入上。

## 第20页：拉斯维加斯算法——基本框架

### 显著特征

拉斯维加斯算法所作的**随机性决策有可能导致算法找不到所需的解**。

### obstinate 算法框架

```java
public static void obstinate(Object x, Object y)
{
    // 反复调用拉斯维加斯算法 LV(x, y)
    // 直到找到问题的一个解 y
    boolean success = false;
    while (!success)
        success = lv(x, y);
}
```

> 💡 思路很简单：既然单次可能找不到解，那就**反复尝试**，直到成功为止。因为每次成功概率 > 0，所以最终一定会成功。

## 第21页：拉斯维加斯算法——时间分析

### 符号定义

| 符号 | 含义 |
|------|------|
| p(x) | 对输入 x 调用 LV 算法获得一个解的概率 |
| t(x) | obstinate 找到解的期望时间 |
| s(x) | 求解成功所需的平均时间 |
| e(x) | 求解失败所需的平均时间 |

### 正确性要求

一个正确的拉斯维加斯算法应该对所有输入 x 均有 **p(x) > 0**。

### 期望时间递推方程

```
t(x) = p(x)·s(x) + (1 - p(x))·(e(x) + t(x))
```

含义：
- 以概率 p(x) 一次成功，花费 s(x)
- 以概率 1-p(x) 失败，花费 e(x) 后还需要继续尝试（再花 t(x)）
- e(x)+t(x)代表从某次失败开始算起，求解成功的时间期望与一开始相同，即具有无记忆性
- 单次求解期望时间t0(x)=p(x)s(x)+(1-p(x))e(x)，即t(x)=t0(x)/p(x)，平均需要求解1/p(x)次才能成功

### 解方程得：

```
t(x) = s(x) + [(1 - p(x)) / p(x)] · e(x)
```

> ✅ 核心公式：**t(x) = s(x) + (1-p(x))/p(x) · e(x)**

> 💡 直觉：如果成功概率 p(x) 很高（接近1），则 (1-p)/p 很小，总时间接近一次成功时间 s(x)。如果 p(x) 很小，则需要反复尝试很多次，e(x) 的影响被放大。
