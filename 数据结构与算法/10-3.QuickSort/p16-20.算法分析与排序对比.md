# QuickSort — 算法分析与排序对比（第16-20页）

QuickSort 的最好/最坏/平均时间复杂度分析，优化方法，以及各排序算法复杂度对比。

---

## 第16页：QuickSort 效率总结

| 情况 | 复杂度 | 发生条件 |
|------|--------|---------|
| **最好** | Θ(n log n) | 每次划分恰好在中间（等分） |
| **最坏** | Θ(n²) | 已排好序的数组！ |
| **平均** | Θ(n log n) | 随机数组 |

### 优化方法

| 优化 | 效果 |
|------|------|
| 更好的 pivot 选择：**三数取中（median of three）** | 避免已排序数组的最坏情况 |
| 小子数组切换为**插入排序** | 减少递归开销 |
| 消除递归（尾递归优化） | 减少栈空间 |

> 这些优化组合起来可以带来 **20-25%** 的性能提升。

### 适用场景

> QuickSort 被认为是**大规模内部排序**（n ≥ 10000）的首选方法。

---

## 第17页：最坏情况分析 — 为什么已排序数组最坏？

### 原因

如果用 A[0] 作为中轴，而数组已经排好序：
- 从左到右的扫描会停在 A[1]（因为 A[1] > A[0] 对于严格递增不成立...实际上对于已排序数组，pivot 是最小的）
- 从右到左的扫描一直处理到 A[0]
- 导致分裂点出现在位置 0

结果：每次划分产生的两个区域分别包含 **n-1 个元素**和 **1 个元素**。

### 比较次数

```
第1次 Partition：n+1 次比较 → 分成 (0) 和 (n-1)
第2次 Partition：n 次比较 → 分成 (0) 和 (n-2)
第3次 Partition：n-1 次比较 → 分成 (0) 和 (n-3)
……
总计：(n+1) + n + (n-1) + … + 3 = O(n²)
```

> 类比：每次只"排好"一个元素（最小的），跟选择排序一样慢了！

---

## 第18页：最坏情况递推关系

划分的两个区域分别包含 **n-1** 个元素和 **1** 个元素，如果每一步都出现这种情况：

```
         ┌ O(1)           n ≤ 1
T(n) =   │
         └ T(n-1) + O(n)  n > 1
```

### 求解

```
T(n) = T(n-1) + O(n)
     = T(n-2) + O(n-1) + O(n)
     = T(1) + O(2) + O(3) + … + O(n)
     = O(n²)
```

> ✅ 解得：**T(n) = O(n²)**

---

## 第19页：最好情况递推关系

每次划分所取的基准都恰好为**中值**，即每次划分都产生两个大小为 **n/2** 的区域：

```
         ┌ O(1)              n ≤ 1
T(n) =   │
         └ 2T(n/2) + O(n)   n > 1 //无论怎样，i和j都会完整扫过一遍数组，所以扫的这部分是O(n)
```

> ✅ 解得：**T(n) = O(n log n)**

这和 MergeSort 的递推关系完全一样！

## 第20页：各排序算法时间复杂度对比

| 算法 | 最坏复杂度 | 平均复杂度 |
|------|---------|---------|
| 冒泡排序 | n² | n² |
| 计数排序 | n² | n² |
| 插入排序 | n² | n² |
| 选择排序 | n² | n² |
| **快速排序** | **n²** | **n log n** |
| **归并排序** | **n log n** | **n log n** |

### 关键对比

| | QuickSort | MergeSort |
|---|-----------|-----------|
| 最坏 | O(n²) ❌ | O(n log n) ✅ |
| 平均 | O(n log n) | O(n log n) |
| 空间 | O(log n) ✅（原地） | O(n) ❌ |
| 稳定性 | ❌ 不稳定 | ✅ 稳定 |
| 实际速度 | 通常**最快** | 常数因子较大 |

> ✅ 实践中的选择：
> - 需要**最坏保证** → MergeSort 或 HeapSort
> - 追求**平均最快** → QuickSort（配合三数取中等优化）
> - 需要**稳定性** → MergeSort
> - **外部排序**（磁盘） → MergeSort

## 总结

```
QuickSort 复杂度全景：

         最好          平均          最坏
时间：  O(n log n)   O(n log n)    O(n²)
空间：  O(log n)     O(log n)      O(n)（递归栈退化）

最坏情况：已排序数组 + 选第一个做 pivot
  → 每次只切出1个元素
  → T(n) = T(n-1) + O(n) = O(n²)

最好情况：每次恰好等分
  → T(n) = 2T(n/2) + O(n) = O(n log n)

避免最坏情况的方法：
  ① 三数取中选 pivot
  ② 随机化选 pivot
  ③ 小规模切换插入排序

排序算法选择指南：
  ┌── 大规模、内部排序 → QuickSort（实际最快）
  ├── 需要最坏保证   → MergeSort / HeapSort
  ├── 需要稳定性     → MergeSort
  ├── 小规模         → InsertionSort
  └── 外部排序       → MergeSort
```
