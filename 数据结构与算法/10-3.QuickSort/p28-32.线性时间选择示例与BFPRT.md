# 线性时间选择 — 示例与确定性算法（第28-32页）

RandomizedSelect 的执行示例，以及确定性最坏 O(n) 的 BFPRT 算法（中位数的中位数）。

---

## 第28页：RandomizedSelect 执行示例

### 问题

在数组 **4, 1, 10, 9, 7, 12, 8, 2, 15** 中寻找**第5小**的元素（下标从1到9）

### 执行过程

**第一次调用** RandomizedPartition(a, 1, 9) = 3

```
划分后：2  1  [4]  9  7  12  8  10  15
                ↑ pivot=4 归位在第3位
```

5 > 3 → 第5小不在左半，调用 RandomizedSelect(a, 4, 9, 2)
（在右半中找第 5-3=2 小）

**第二次调用** RandomizedPartition(a, 4, 9) = 6

```
划分后：2  1  4  8  7  [9]  12  10  15
                         ↑ pivot=9 归位在第6位
j = 6-4+1 = 3
```

2 < 3（即 k=2 < j=3）→ 调用 RandomizedSelect(a, 4, 6, 2)
（在左半中找第2小）

**第三次调用** RandomizedPartition(a, 4, 6) = 5

```
划分后：2  1  4  7  [8]  9  12  10  15
                      ↑ pivot=8 归位在第5位
j = 5-4+1 = 2
```

k=2 = j=2 → **找到了！第5小元素 = 8** ✅

---

## 第29页：算法复杂度

- 在**最坏情况**下，总是在最大元素处划分，算法 RandomizedSelect 需要 **O(n²)** 计算时间
- 但可以证明，算法 RandomizedSelect 可以在 **O(n) 平均时间**内找出 n 个输入元素中的第 k 小元素

## 第30页：确定性 O(n) 的思路

### 核心思想

> 如果能在线性时间内找到一个划分基准，使得按这个基准所划分出的 2 个子数组的长度都至少为原数组长度的 **ε 倍**（0 < ε < 1 是某个正常数），那么就可以在最坏情况下用 O(n) 时间完成选择任务。

### 例子

若 ε = 9/10，算法递归调用所产生的子数组的长度至少缩短 1/10。所以，在最坏情况下，算法所需的计算时间 T(n) 满足递归式：

```
T(n) ≤ T(9n/10) + O(n)
```

由此可得 **T(n) = O(n)**。

> 关键：只要每次都能保证砍掉常数比例，几何级数就收敛到 O(n)。

## 第31页：BFPRT 算法 — 中位数的中位数

### 步骤

1. 将 n 个输入元素划分成 ⌈n/5⌉ 个组，每组 5 个元素（最后一组可能不足5个）
2. 用任意一种排序算法，将每组中的元素排好序，并取出每组的**中位数**，共 ⌈n/5⌉ 个
3. **递归调用 Select** 来找出这 ⌈n/5⌉ 个中位数的中位数（如果 ⌈n/5⌉ 是偶数，就找它的 2 个中位数中较大的一个）。以这个元素作为划分基准 x

### 为什么能保证 ε？

设所有元素互不相同。在这种情况下：
- 找出的基准 x 至少比 **3(n-5)/10** 个元素大（因为每一组中有 2 (=floor((5-1)/2))个元素小于本组的中位数，而 n/5 个中位数中又有 (n-5)/10 (=floor((n/5-1)/2))个小于基准 x）
- 同理，基准 x 也至少比 **3(n-5)/10** 个元素小（两个比自己小的中位数+自己共三个中位数乘上每个中位数有 (n-5)/10比x小
- 而当 n ≥ 75 时，3(n-5)/10 ≥ n/4

> ✅ 所以按此基准划分所得的 2 个子数组的长度都至少缩短 **1/4**

## 第32页：确定性 Select 代码与复杂度

### 代码

```java
private static Comparable select(int p, int r, int k)
{
    if (r - p < 5) {
        // 用某个简单排序算法对数组 a[p:r] 排序
        bubbleSort(p, r);
        return a[p + k - 1];
    }

    // 将 a[p+5*i] 至 a[p+5*i+4] 的第3小元素
    // 与 a[p+i] 交换位置
    // 找中位数的中位数，r-p-4 即上面所说的 n-5
    for (int i = 0; i <= (r-p-4)/5; i++)
    {
        int s = p + 5*i;
        int t = s + 4;
        for (int j = 0; j < 3; j++) bubble(s, t-j);
        MyMath.swap(a, p+i, s+2);
    }

    Comparable x = select(p, p+(r-p-4)/5, (r-p-4)/10);  // 递归找中位数的中位数
    int i = partition(p, r, x);                           // 用 x 做 pivot 划分
    j = i - p + 1;

    if (k <= j) return select(p, i, k);
    else return select(i+1, r, k-j);
}
```

### 复杂度分析

```
         ┌ C₁                 n < 75
T(n) ≤   │
         └ C₂n + T(n/5) + T(3n/4)    n ≥ 75
```

> ✅ **T(n) = O(n)**

### 关键点解释

| 设计点 | 说明 |
|--------|------|
| 每组大小定为 **5** | 保证每组排序是常数时间 |
| 判断阈值 **75** | 保证 3(n-5)/10 ≥ n/4 成立 |
| 递归式中两个子问题 | T(n/5)：找中位数的中位数；T(3n/4)：选择子问题 |
| n/5 + 3n/4 = 19n/20 = εn | 0 < ε < 1，这是使 T(n)=O(n) 的关键 |

> 当然，除了 5 和 75 之外，还有其他选择（如每组 7 个元素等），但 5 是最小的能保证线性的分组大小。
