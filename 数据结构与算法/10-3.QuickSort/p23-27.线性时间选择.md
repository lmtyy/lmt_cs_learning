# 线性时间选择 Selection in Linear Time（第23-27页）

利用 Partition 思想解决选择问题（找第 k 小元素），只需递归一侧，期望 O(n)。

---

## 第23页：问题引入

### 问题

> 数组中存储 n 个数，要求将数组中全部值等于 x 的元素移动到数组的右侧。

更一般地：

> 已知 n 元数组 A[1:n]，试确定其中**第 k 小**的元素。

- k = 1 → 找最小的元素
- k = n → 找最大的元素

---

## 第24页：利用快速分类思想解决

根据 Partition 过程：如果划分元素 v 确定在 A(j) 的位置上，则有 **j-1** 个元素小于或等于 A(j)，且有 **n-j** 个元素大于或等于 A(j)。

---

## 第25页：选择算法的逻辑

假设在一次划分中，划分元素 v 处于第 j 个位置：

| 情况 | 判断 | 操作 |
|------|------|------|
| **k = j** | 第 k 小就是划分元素 v | 直接返回 A(j) |
| **k < j** | 第 k 小在新数组 A[1:j-1] 中 | 递归在 A[1:j-1] 中找第 **k** 小元素 |
| **k > j** | 第 k 小在新数组 A[j+1:n] 中 | 递归在 A[j+1:n] 中找第 **k-j** 小元素 |

> ✅ 与 QuickSort 的关键区别：每次只递归**一侧**（不是两侧都递归）

---

## 第26页：采用划分的选择算法 — 图解

```
划分后：
┌──────┬──────┬─────┬────────┬───┬──────────┬─────┬──────┐
│ A(1) │ A(2) │ … │ A(j-1) │A(j)│ A(j+1) │ …  │ A(n) │
└──────┴──────┴─────┴────────┴───┴──────────┴─────┴──────┘
  ←── 全部 ≤ v ──→    v    ←── 全部 ≥ v ──→

k<j时：第k小          k=j时：          k>j时：第k小
在左侧集合中          就是v           在右侧集合中
                                    （找第k-j小）
```

## 第27页：RandomizedSelect 代码

```cpp
template<class Type>
Type RandomizedSelect(Type a[], int p, int r, int k)
{
    if (p == r)
        return a[p];                         // base case：只剩一个元素

    int i = RandomizedPartition(a, p, r);    // 随机化划分，返回 pivot 位置
    int j = i - p + 1;                       // pivot 是当前子数组中第 j 小的

    if (k <= j)
        return RandomizedSelect(a, p, i, k);       // 在左半找第 k 小
    else
        return RandomizedSelect(a, i+1, r, k-j);   // 在右半找第 k-j 小
}
```

### 代码逻辑

| 行 | 作用 |
|---|------|
| `if (p == r)` | 只剩一个元素，直接返回 |
| `RandomizedPartition(a, p, r)` | 随机选 pivot 做划分，返回 pivot 位置 i |
| `j = i - p + 1` | 计算 pivot 在当前子数组中是第几小 |
| `if (k <= j)` | 第 k 小在左半（包含 pivot） |
| `else` | 第 k 小在右半，注意找的是第 **k-j** 小 |

### 复杂度

| 情况 | 复杂度 | 说明 |
|------|--------|------|
| 期望（平均） | **O(n)** | 每次期望砍掉常数比例，几何级数收敛 |
| 最坏 | O(n²) | 与 QuickSort 类似的退化（概率极低） |
| 确定性最坏 O(n) | BFPRT 算法 | 用"中位数的中位数"选 pivot |

### 为什么是 O(n)？

```
Select 只递归一侧：
  T(n) = T(n/2) + O(n)
       = O(n) + O(n/2) + O(n/4) + …
       = O(n) · (1 + 1/2 + 1/4 + …)
       = O(n) · 2
       = O(n)
```
