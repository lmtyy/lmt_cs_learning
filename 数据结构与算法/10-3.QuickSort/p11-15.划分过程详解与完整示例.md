# QuickSort — 划分过程详解与完整示例（第11-15页）

详细讲解双指针扫描的逻辑、Partition 代码实现，以及完整的排序示例追踪。

---

## 第11页：划分的过程 — 双指针扫描规则

### 方法

使用一种**两次扫描**子数组的方法：一次是**从左到右**，另一次是**从右到左**，每次都把子数组的元素和中轴进行比较。

### 左指针 i 的规则

- 从**第二个元素**开始向右扫描
- 希望小于中轴的元素位于子数组的第一部分
- 扫描会**忽略**小于中轴的元素
- 直到遇到第一个**大于等于**中轴的元素才会停止

### 右指针 j 的规则

- 从**最后一个元素**开始向左扫描
- 希望大于中轴的元素位于子数组的第二部分
- 扫描会**忽略**大于中轴的元素
- 直到遇到第一个**小于等于**中轴的元素才会停止

## 第12页：指针相遇的三种情况

### 情况一：i < j（指针未相遇）

```
         → i                    j ←
┌───┬──────────┬──────┬──────┬──────────┬──────────┐
│ P │ 全部≤p   │ ≥p  │ …… │  ≤p     │ 全部≥p   │
└───┴──────────┴──────┴──────┴──────────┴──────────┘
```

> 简单交换 A[i] 和 A[j]，再分别对 i 加 1、j 减 1，然后继续扫描。

### 情况二：i > j（指针交叉）

```
         j ←    → i
┌───┬──────────┬──────┬──────┬──────────┐
│ P │ 全部≤p   │ ≤p  │ ≥p  │ 全部≥p   │
└───┴──────────┴──────┴──────┴──────────┘
```

> 如果扫描指针 i 和 j 相交，i > j，把**中轴和 A[j] 交换**，得到该数组的一个分区。

### 情况三：i = j（指针指向同一元素）

```
              → i=j ←
┌───┬──────────┬──────┬──────────┐
│ P │ 全部≤p   │  =p │ 全部≥p   │
└───┴──────────┴──────┴──────────┘
```

> 如果扫描指针 i 指向同一元素，i=j，被指向元素的值**一定等于 p**。同样把中轴和 A[j] 交换即可。

## 第13页：Partition 代码实现

```cpp
template <class Type>
int Partition(Type a[], int p, int r)
{
    int i = p;          // 左侧扫描指针起始
    int j = r + 1;      // 右侧扫描指针起始
    Type x = a[p];      // 中轴元素

    while (true)
    {
        while (a[++i] < x);   // 移动左侧扫描指针（找 ≥x 的）
        while (a[--j] > x);   // 移动右侧扫描指针（找 ≤x 的）
        if (i >= j) break;    // 指针相遇或交叉，退出
        Swap(a[i], a[j]);     // 交换"不该在左边的"和"不该在右边的"
    }

    a[p] = a[j];    // 将 a[j] 放到原来 pivot 的位置
    a[j] = x;       // 将 pivot 放到 j 的位置（pivot 归位）
    return j;        // 返回 pivot 的最终位置
}
```
Q:为什么j=r+1而不是r？
A:因为内层while()里是--j，一进去就自减一次，所以a[r+1]根本不会影响任何东西，也就是说，如果改成j=r,则a[r]被算法抛弃了
Q:那改成j--配套j=r行不行？
A:不行，因为j--是先比较再减，则会导致a[j]不再是<=x的分界点（分界点变成a[j+1]了)，除非后面代码全部修改否则就错了

### 代码注释

| 行 | 作用 | 说明 |
|---|------|------|
| `i = p` | 左指针初始化 | 从 pivot 位置开始（++i 后变为 p+1） |
| `j = r + 1` | 右指针初始化 | 从 r+1 开始（--j 后变为 r） |
| `x = a[p]` | 保存 pivot 值 | 中轴元素 |
| `while(a[++i] < x)` | i 右移 | 跳过所有 < pivot 的，停在 ≥ pivot 处 |
| `while(a[--j] > x)` | j 左移 | 跳过所有 > pivot 的，停在 ≤ pivot 处 |
| `if(i >= j) break` | 判断相遇 | 指针交叉则退出循环 |
| `Swap(a[i], a[j])` | 交换 | 把错位的两个元素互换 |
| `a[p] = a[j]; a[j] = x` | pivot 归位 | 最后把 pivot 放到正确位置 |

### ⚠️ 该算法是否有问题？

> 需要注意 i 指针的越界问题：如果数组中所有元素都 < pivot，`while(a[++i] < x)` 会一直右移直到越界。解决方法：在数组末尾放**哨兵**（如上一页的 +∞），或者加边界判断 `i < r`。

## 第14页：完整排序示例 — 排列 [5, 3, 1, 9, 8, 2, 4, 7]

### 第一次 Partition（整个数组，pivot=5）

```
下标：  0   1   2   3   4   5   6   7
初始：  5   3   1   9   8   2   4   7
       i=0                         j=8(r+1)

第1轮：i→停在下标3(9≥5)，j←停在下标6(4≤5)
       交换：5  3  1  4  8  2  9  7

第2轮：i→停在下标4(8≥5)，j←停在下标5(2≤5)
       交换：5  3  1  4  2  8  9  7

第3轮：i→停在下标5(8≥5)，j←停在下标4(2≤5)
       i>j → break！

pivot归位：a[0]=a[4]=2, a[4]=5
结果：  2  3  1  4  [5]  8  9  7
              ↑ pivot归位在下标4
```

### 递归左半 [2, 3, 1, 4]（pivot=2）

```
初始：  2   3   1   4
       i                j

i→停在下标1(3≥2)，j←停在下标2(1≤2)
交换：  2  1  3  4

i→停在下标2(3≥2)，j←停在下标1(1≤2)
i>j → break！

pivot归位：a[0]=a[1]=1, a[1]=2
结果：  1  [2]  3  4
```

继续递归 [1]（base case）和 [3, 4]：

```
[3, 4]：pivot=3
i→停在下标1(4≥3)，j←停在下标0(3≤3) → i=j
pivot归位：[3]  4 → 已经有序
```

### 递归右半 [8, 9, 7]（pivot=8）

```
初始：  8   9   7
i→停在下标1(9≥8)，j←停在下标2(7≤8)
交换：  8  7  9

i→停在下标2(9≥8)，j←停在下标1(7≤8)
i>j → break！

pivot归位：a[0]=a[1]=7, a[1]=8
结果：  7  [8]  9
```

### 最终排序结果

```
1  2  3  4  5  7  8  9  ✅
```

## 第15页：思考问题

### 问题 1：扫描停止时 i 和 j 有几种关系？

| 关系 | 情况 | 处理 |
|------|------|------|
| i < j | 正常交换 | swap(A[i], A[j])，继续扫描 |
| i > j | 指针交叉 | break，将 pivot 与 A[j] 交换 |
| i = j | 指向同一元素 | 该元素必等于 pivot，break 后照常处理 |

### 问题 2：下标 i 和 j 会不会超出 a 的下标界？

- **j 不会越界**：因为 a[p] = pivot，j 向左扫描到 a[p] 时一定会停（a[p] ≤ pivot）
- **i 可能越界**：如果 pivot 是数组中最大的元素，i 会一直右移。解决方案：
  - 加哨兵 a[r+1] = +∞
  - 或在 while 循环中加条件 `i < r`

### 问题 3：该排序算法是否稳定？

> ❌ **QuickSort 不是稳定排序**。

例如：[5, 5*, 3]，pivot=5 时，5* 可能被交换到 5 的前面，相等元素的相对顺序会改变。

（相等元素顺序改变就叫做不稳定）

## 总结

```
Partition 双指针扫描逻辑：

  pivot = a[p]
  i 从左向右：跳过 < pivot 的，停在 ≥ pivot 的
  j 从右向左：跳过 > pivot 的，停在 ≤ pivot 的

  i < j → swap(a[i], a[j])，继续
  i ≥ j → 停止，swap(a[p], a[j])，pivot 归位

完整排序 [5,3,1,9,8,2,4,7] 的递归过程：

QuickSort([5,3,1,9,8,2,4,7])
  Partition → pivot=5 归位在[4]
  ├── QuickSort([2,3,1,4])
  │     Partition → pivot=2 归位在[1]
  │     ├── QuickSort([1]) → base case
  │     └── QuickSort([3,4])
  │           Partition → pivot=3 归位
  │           └── base case
  └── QuickSort([8,9,7])
        Partition → pivot=8 归位在[1]
        ├── QuickSort([7]) → base case
        └── QuickSort([9]) → base case

结果：[1, 2, 3, 4, 5, 7, 8, 9]
```
