# QuickSort — Partition 算法与代码（第6-10页）

Partition（划分）是快速排序的核心操作。本段介绍 Partition 算法、QuickSort 主程序代码，以及详细的划分示例。

---

## 第6页：Partition 算法（Hoare 划分）

### 伪代码

```
Algorithm Partition(A[l..r])
// 以第一个元素 A[l] 为 pivot 划分子数组
// 输入：A[0..n-1] 的子数组 A[l..r]，由左右下标 l 和 r 定义（l < r）
// 输出：A[l..r] 的一个划分，返回分割位置

p ← A[l]              // 选第一个元素为 pivot
i ← l;  j ← r + 1    // i 从左往右，j 从右往左

repeat
    repeat i ← i + 1 until A[i] ≥ p    // i 右移，找到 ≥ pivot 的
    repeat j ← j - 1 until A[j] ≤ p    // j 左移，找到 ≤ pivot 的（原文写 A[j]·p 应为 ≤p）
    swap(A[i], A[j])                     // 交换
until i ≥ j                              // 直到指针相遇

swap(A[i], A[j])       // 撤销最后一次多余的交换（i≥j时的那次）
swap(A[l], A[j])       // 将 pivot 放到正确位置
return j               // 返回 pivot 的最终位置
```
这里最后必须与j进行交换并返回j，不能与i交换返回i，这是由于pivot选取的是数组第一位，而数组第一位这个位置理应由<=pivot的数占据，所以与pivot交换的数必须<=pivot，而A[i]>=pivot(i只会停在>=pivot的位置，即交换前），A[j]<=pivot，故与j交换。若选取数组最后一位为pivot，则应与i交换
### 双指针扫描过程

```
pivot = A[l]

    i →→→→→→                    ←←←←←← j
┌───┬───┬───┬───┬───┬───┬───┬───┐
│ p │   │   │   │   │   │   │   │
└───┴───┴───┴───┴───┴───┴───┴───┘

i 向右找 ≥ pivot 的元素（"不该在左边的"）
j 向左找 ≤ pivot 的元素（"不该在右边的"）
找到后交换，继续扫描
直到 i ≥ j（指针相遇），把 pivot 放到 j 的位置
```

## 第7页：QuickSort 的分治三步

| 步骤 | 名称 | 操作 |
|------|------|------|
| **分解(Divide)** | Partition | 以 a[p] 为基准元素将 a[p:r] 划分成 3 段：a[p:q-1]、a[q]、a[q+1:r]，使得 a[p:q-1] 中任何一个元素小于或等于 a[q] |
| **递归求解(Conquer)** | 递归 | 通过递归调用快速排序算法分别对 a[p:q-1] 和 a[q+1:r] 进行排序 |
| **合并(Merge)** | 无需操作 | 由于对 a[p:q-1] 和 a[q+1:r] 的排序是就地进行的，所以在 a[p:q-1] 和 a[q+1:r] 都已排好序后步需执行任何计算，a[p:r] 就已排好序 |

> ✅ 合并步骤是**空的**！这是 QuickSort 比 MergeSort 常数更小的原因之一。

## 第8页：QuickSort 主程序代码

```cpp
template<class Type>
void QuickSort(Type a[], int p, int r)
{                           // p = a的起始位置，r = a的终止位置
    if (p < r)
    {
        int q = Partition(a, p, r);  // Partition 负责将 a 进行一次ij双指针排序，
                                      // 返回分割元素的位置，这时q左边的数<=a[p],右边>=a[p]
        QuickSort(a, p, q - 1);       // 对左半段排序
        QuickSort(a, q + 1, r);       // 对右半段排序
    }
}
```

### 调用方式

```cpp
QuickSort(a, 0, n - 1);  // 对整个数组排序
```

### 与 MergeSort 的代码对比

| | MergeSort | QuickSort |
|---|-----------|-----------|
| 分割 | `mid = (l+r)/2`（简单取中） | `q = Partition(a, p, r)`（复杂划分） |
| 递归 | 两次递归 | 两次递归 |
| 合并 | `Merge(a, b, l, mid, r)` | **无** |

## 第9页：划分过程 — pivot 的选择

- Partition 的过程中，首先要选择一个元素，根据其值划分数组。称该元素为**中轴（pivot）**
- 选择中轴有许多不同的策略，我们使用最简单的策略，选择**第一个元素**：**s = a[p]**

### 常见的 pivot 选择策略

| 策略 | 说明 | 优缺点 |
|------|------|--------|
| 取第一个元素 | s = a[p] | 简单，但对已排序数组退化为 O(n²) |
| 取最后一个元素 | s = a[r] | 同上 |
| 取中间元素 | s = a[(p+r)/2] | 稍好 |
| **三数取中** | median(a[p], a[(p+r)/2], a[r]) | 实践中最常用，避免最坏情况 |
| 随机选择 | s = a[random(p,r)] | 期望 O(n log n)，但不确定性 |

## 第10页：划分举例 — Partition 执行过程

### 数组：[65, 70, 75, 80, 85, 60, 55, 50, 45, +∞]

pivot = 65（第一个元素），+∞ 是哨兵（防止 i 越界）

### 逐步执行

| 步骤 | 数组状态 | i | j(p) | 操作 |
|------|---------|---|------|------|
| 初始 | **65** 70 75 80 85 60 55 50 45 +∞ | 2 | 9 | i 停在 70(≥65)，j 停在 45(≤65) |
| 交换1 | 65 **45** 75 80 85 60 55 50 **70** +∞ | 3 | 8 | 交换 70↔45 |
| 交换2 | 65 45 **50** 80 85 60 55 **75** 70 +∞ | 4 | 7 | 交换 75↔50 |
| 交换3 | 65 45 50 **55** 85 60 **80** 75 70 +∞ | 5 | 6 | 交换 80↔55 |
| 交换4 | 65 45 50 55 **60** **85** 80 75 70 +∞ | 6 | 5 | 交换 85↔60 |
| i≥j | — | 6 | 5 | i≥j，循环停止！ |
| 撤销 | 65 45 50 55 **60** 85 80 75 70 +∞ | — | — | 撤销最后一次交换(swap back) |
| pivot归位 | **60** 45 50 55 **[65]** 85 80 75 70 +∞ | — | — | swap(A[1], A[5])：pivot 65 归位 |

### 最终结果

```
┌────────────────┬──────┬────────────────────┐
│ 60  45  50  55 │  65  │ 85  80  75  70  +∞ │
│   全部 < 65    │pivot │    全部 > 65        │
└────────────────┴──────┴────────────────────┘
```

> ✅ pivot 65 已在最终位置（下标5），左边全 < 65，右边全 > 65。

### ⚠️ 关于哨兵 +∞

数组末尾放 +∞ 是为了防止 i 指针右移时**越界**。实际实现中可以用边界判断代替。

## 总结

```
Partition 算法核心逻辑：

1. 选 pivot = A[l]（第一个元素）
2. i 从左往右找 ≥ pivot 的
3. j 从右往左找 ≤ pivot 的
4. 交换 A[i] 和 A[j]
5. 重复直到 i ≥ j
6. 撤销最后一次交换
7. 将 pivot 与 A[j] 交换 → pivot 归位
8. 返回 j（pivot 的最终位置）

QuickSort 主程序：
  if (p < r):
    q = Partition(a, p, r)   ← 划分（O(n)）
    QuickSort(a, p, q-1)     ← 递归左半
    QuickSort(a, q+1, r)     ← 递归右半
    （无需合并！）
```
