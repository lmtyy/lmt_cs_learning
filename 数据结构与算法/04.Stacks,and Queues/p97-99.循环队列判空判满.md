### 第一页 (第97页)：核心思想 —— 利用取模运算实现“闭环”
**核心目标：** 展示循环队列在物理上依然是一个数组，但在逻辑上通过数学运算将其变成了一个“圆”。

*   **解决“假溢出”的关键：** `When an element moves past the end of a circular array, it wraps around to the beginning.`（当元素越过数组的末尾时，它会**绕回到开头**。）
*   **💡 核心数学工具：取模运算符 `%` (Modulo)**
    *   这一页明确给出了入队时 `rear` 指针更新的终极公式。`After Enqueue(3), the rear index moves from 5 to 0. By operator %`
    *   假设数组最大长度 `MAXQSIZE = 6`，当前 `rear` 在索引 5。入队一个元素后，计算 `rear = (rear + 1) % 6`，得到 `(5 + 1) % 6 = 0`。指针成功从尾部跳回了头部 0。
*   **💡 右侧的“圆盘”图解：**
    *   `front` 箭头从起点开始走，`rear` 箭头在圆圈中绕圈。如果 `rear` 跨过了下标 0（右边画着新元素 4），它就会自然地绕到下标 0 的位置存放。

### 第二页 (第98页)：方法一 —— 使用计数器 `counter` 判定状态
**核心目标：** 介绍最直观、最不容易出错的判空/判满方法。

*   **问题提出：** `How to detect an empty or full queue, using a circular array algorithm?` 在之前的线性队列中，`front == rear` 代表空。**但在循环队列中，随着不断的入队和出队，你会发现队满和队空时，`front` 和 `rear` 都有可能碰在一起。** 如果是 `front == rear`，你根本不知道是空的还是满的。
*   **方法一（绿字）：** `- Use a counter of the number of elements in the queue.`（使用一个记录队列中元素**数量**的计数器）。
    *   **逻辑：**
        *   当 `counter == 0` 时，队列为空（Empty queue）。
        *   当 `counter == MAXQSIZE`（数组最大容量）时，队列为满（Full queue）。
*   **💡 图解验证：**
    *   **Full queue 图：** `front` 和 `rear` 箭头确实碰到了同一个位置，但因为 `counter` 已经达到了最大值，所以系统知道这是“已满”。
    *   **Empty queue 图：** 同样 `front` 和 `rear` 碰在一起，但因为 `counter` 为 0，系统知道这是“空的”。
*   **优点：** 这种写法极其简单、逻辑清晰，不容易出 Bug。
*   **缺点：** 需要多维护一个 `counter` 变量。在多线程并发环境下，维护这个计数器的同步成本可能会稍微高一点。

---

### 第三页 (第99页)：方法二 —— 留空一格法 (牺牲一个存储空间)
**核心目标：** 介绍教科书和考试中最喜欢考察的“经典做法”：不浪费变量，但浪费一个数组空间来区分状态。

*   **核心规则：** `A queue is considered full when one space is left.` 当队列中**只剩下一个空位**时，就认为队列已满。
*   **判满公式（必考！）：** `front == (rear + 1) % MAXQSIZE`
    *   **解读：** 如果队尾指针 `rear` 再往前走一步（取模后）就要撞上队头指针 `front` 了，此时坚决不让入队，认为队列已满。
*   **判空公式：** `front == rear`
    *   **解读：** 如果队头和队尾指针完全重合，队列为空。
*   **💡 图解验证：**
    *   **Full queue 图：** 圆盘里存放了 J4, J5, J6, J7, J8 共5个数据（总容量是6），还剩一个 J9 的空位。此时 `rear` 指向 J9 这个空位，`front` 指向 J4。如果此时再入队一个数据，`rear` 就会绕到 J4 的位置，这时 `rear` 就等于 `front` 了，但我们无法区分那是“满”还是“空”。所以**提前一步**，在还剩一个空位时就喊“满”。
    *   **Empty queue 图：** 没有任何数据时，`front` 和 `rear` 指向同一个位置，判定为空。
*   **优点：** 不需要额外的 `counter` 变量，物理结构更纯粹。
*   **缺点：** 浪费了一个数组元素的存储空间（容量为 N 的数组，实际上只能存 N-1 个元素）。
*   **⚠️ 考试常客：** 如果你在做数据结构的考试卷子，遇到“循环队列判空判满”的选择题或代码题，**大概率要你写这个留空一格的公式**，而不是用计数器。

### 💡 核心学习总结（面对考试你该怎么记？）

关于循环队列，你只需要记住这三句话就能拿满分：

1.  **怎么让指针绕圈？**
    *   记住取模公式：入队 `rear = (rear + 1) % maxSize;`，出队 `front = (front + 1) % maxSize;`。
2.  **判空和判满的两种方法：**
    *   **方案一（简单直接，推荐工程用）：** 设置一个 `size` 计数器。`size==0` 为空，`size==maxSize` 为满。
    *   **方案二（经典理论，考试高频）：** 牺牲一个空间。
        *   判空：`front == rear`。
        *   判满：`front == (rear + 1) % maxSize`。
3.  **谁算长度？**
    *   在采用“留空一格”法的循环队列中，当前队列的实际元素个数计算公式是：`(rear - front + maxSize) % maxSize`。（这个公式在求平均、求长度时经常用到，注意括号里的 `+ maxSize` 是为了防止负数的出现）。
