### 第一页 (第100页)：循环队列的结构体定义
**核心目标：** 明确在 C 语言中，如何用代码描述一个循环队列。

*   **宏定义：** `#define MAXQSIZE 100` 设定队列的最大容量为 100。
*   **结构体 `SqQueue`（顺序队列）：**
    *   **`QElemType *base;`**：**基地址指针**。它指向一块连续内存的起始位置（也就是通过 `malloc` 申请的动态数组的起点）。在底部的图解中，它对应 `base` 框并指向蓝色圆盘的入口。
    *   **`int front;`**：**队头下标（索引）**。注意这里不是指针，而是一个整数！它代表当前队列第一个元素在数组中的下标。对应图解中橙色的 `front` 框，指向了包含 J1 的格子。
    *   **`int rear;`**：**队尾下标（索引）**。同样是一个整数。它代表**下一个即将存放元素的空位**下标。对应图解中橙色的 `rear` 框，指向了包含 J6 的格子。
*   **💡 图解理解：**
    *   下方的图解非常清晰地展示了“物理是数组，逻辑是圆”的概念。橙色区块是控制块，保存着基地址和两个整数下标。蓝色圆圈是内存中的数组，`front` 和 `rear` 两个整数就像两根手指，在圆圈（底层数组）上标注着位置。

---

### 第二页 (第101页)：初始化队列 `InitQueue`
**核心目标：** 在内存中开辟空间，并将队列置于“初始空”状态。

*   **1. 分配内存空间：**
    *   `Q.base = (QElemType *) malloc(MAXQSIZE * sizeof(QElemType));`
    *   动作：向操作系统申请一块大小为 100 个元素单位的连续内存空间。基地址保存在 `Q.base` 中。
*   **2. 内存分配失败处理：**
    *   `if (!Q.base) exit(OVERFLOW);` 如果系统内存耗尽导致 `malloc` 返回空指针，则直接报错崩溃。
*   **3. 确立初始状态（核心）：**
    *   `Q.front = Q.rear = 0;`
    *   **关键理解：** 循环队列初始化为空时，队头和队尾下标**都设为 0**（都指向数组的第一个格子）。根据我们之前学习的“留空一格”判空法，`front == rear` 意味着此时队列是**空的**。
*   **💡 图解验证：**
    *   右侧的蓝色圆盘图上，`front` 和 `rear` 两个箭头（对应两个整数下标）**重叠在一起**，指向了圆盘上的同一个位置（下标 0）。这就是标准的空队列状态。

### 第三页 (第102页)：入队操作 `Enqueue`
**核心目标：** 实现入队逻辑，包含“留空一格”的判满检查，以及取模运算。

*   **图例演示：**
    *   **Before Adding（入队前）：** `front` 在包含 J1 的格子，`rear` 在包含 J3 的格子（表示 J4 将要放在这里）。
    *   **After Adding（入队后）：** 存放了一个新元素（示意图中红色的 X 表示放入了新数据）。**最关键的变化是：`rear` 指针顺时针移动了一格**。
*   **算法伪代码（下方黄框）：**
    *   **第一步：判满（核心！）**
        *   `If queue q is full: (q.rear+1) MOD MAXQSIZE == q.front`
        *   这个公式完美应用了我们上一页的“留空一格”法。如果 `rear` 再往前走一步（取模后）就会撞上 `front`，说明队列满了（实际上还空着一个格子，但我们认为它满了）。
    *   **第二步：入队与指针更新**
        *   `else: q.rear = (q.rear + 1) MOD MAXQSIZE`
        *   如果没满，就把数据存进 `q.base[q.rear]`（PPT伪代码里省略了这一行，但逻辑上必然存在），然后**更新 `rear` 下标**。利用**取模运算 `%`** 确保 `rear` 从数组末尾 5 加 1 后，能自动绕回到起点 0。
*   **💡 这段代码的工程意义：**
    *   这段伪代码**非常精炼**。它在入队之前，**仅仅使用了简单的加法、取模和比较**，就同时完成了“防溢出”和“自动绕回”两大功能。这是计算机科学中使用数学手段解决物理限制的绝佳范例。

### 💡 总结与进阶练习（如果考手写代码，怎么写？）

如果你需要在期末考试中手写循环队列的代码，下面这个完整的 `Enqueue` 函数是标准答案的变体。请注意**数据存入和指针更新的顺序**：

```c
Status EnQueue(SqQueue &Q, QElemType e) {
    // 1. 判满 (留空一格法)
    if ((Q.rear + 1) % MAXQSIZE == Q.front) {
        return ERROR; // 队满，入队失败
    }
    
    // 2. 存入数据
    Q.base[Q.rear] = e;
    
    // 3. 更新队尾指针 (取模绕回)
    Q.rear = (Q.rear + 1) % MAXQSIZE;
    
    return OK;
}
```

**下一幕剧情预告：**
当前 PPT 只讲了 `Enqueue` 和初始化。接下来一定会出现 **`Dequeue`（出队）** 的图解和代码。出队的逻辑与入队非常对称：
1.  先判空：`if (Q.front == Q.rear) return ERROR;`
2.  然后取数据：`e = Q.base[Q.front];`
3.  最后更新队头指针：`Q.front = (Q.front + 1) % MAXQSIZE;`
