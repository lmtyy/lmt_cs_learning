### 第一页 (第83页)：示例 2 —— 消除尾递归后的正确代码（迭代法）
**核心目标：** 用现实代码展示如何将上一页那个会发生“栈溢出”的递归函数，改写成绝对安全的版本。

*   **代码的变化（对比上一页的递归版本）：**
    *   在上一页（第80页）中，我们使用的是递归调用：`print_list(L->next);`。
    *   在这一页中，这段代码被完全去掉了，改成了两行代码：
        1.  `L = L->next;` （更新当前指针，指向下一个节点）
        2.  `goto top;` （跳转回代码开头的标签 `top:`）
*   **💡 发生了什么？** 
    *   这实际上就是**把递归变成循环**。这里是利用了C语言中略受诟病，但非常底层的 `goto` 语句。
    *   **为何这样做是安全的？** 因为这里**没有发生新的函数调用**。整个程序始终在同一个 `print_list` 函数的同一个栈帧（栈空间）里运行。不管链表中包含 20,000 个节点还是 200 万个节点，系统分配的那个栈帧永远只有一层，**绝对不会发生“栈溢出（Stack Overflow）”**。
*   **💡 额外提示（工程最佳实践）：** 
    *   在实际工程开发中，我们通常不直接写 `goto`（因为会破坏代码结构的清晰度，被许多现代编程规范列为禁忌）。更优雅的写法是直接替换为一个标准的 `while` 循环，效果是一模一样的。
    *   *对应的 `while` 循环写法如下：*
        ```c
        void print_list(LIST L) {
            while(L != NULL) {
                print_element(L->element);
                L = L->next;
            }
        }
        ```

### 第二页 (第84页)：理论升华 —— 尾递归消除的“机械原则”
**核心目标：** 总结出一套通用的法则。只要满足“尾递归”条件，任何人都可以通过这套机械步骤，把递归改写为循环。

*   **PPT原文提炼：** `"Tail recursion can be mechanically eliminated by changing the recursive call to a goto preceded by one assignment per function argument."`
*   **翻译与拆解：** “尾递归可以通过**机械地**将递归调用替换为 `goto` 语句（并且在 `goto` 之前，为每个函数参数执行一次赋值）来消除。”
*   **💡 通用“消除尾递归”三步法（考试/面试必背）：**
    假设你有一个尾递归函数 `void func(int a, int b) { ... return func(x, y); }`。消除它的步骤如下：
    1.  **加标签：** 在函数体的第一行前面加一个标签，例如 `start:`。
    2.  **参数重赋值（关键步骤）：** 找到原来的递归调用处。在调用之前，必须**按顺序把原来的参数，替换为递归调用时传入的新参数**。例如：`a = x; b = y;`。
    3.  **跳转：** 将最后的递归调用替换为 `goto start;`。
*   **💡 现代编译器的视角：**
    *   如果编译器开启了**尾递归优化（Tail Call Optimization, TCO）**，程序员其实根本不需要手动去写 `goto`。编译器在底层自动生成的汇编代码，**本质上就是按照这个“机械原则”操作的**。它不会生成 `CALL` 指令（函数调用），而是直接复用当前的栈空间，重置参数，并用 `JMP` 指令（跳转）回到函数开头。

### 💡 全章核心知识总结（避免爆栈的必杀技）

看完这两页，你可以把“栈与函数调用”这个章节的终极知识点总结为两句话：

1.  **递归不总是好的：** 虽然递归代码极其简洁、逻辑优美（适合处理树、图），但在工程中，**深度过深的递归（尤其是尾递归）是导致程序“栈溢出（Stack Overflow）”崩溃的头号杀手**。因为系统调用栈的空间非常有限（通常只有几 MB）。
2.  **解决爆栈的两种武器：**
    *   **武器一（手工优化）：** 遇到尾递归，直接手工改写为 `while` 循环（或者使用 `goto` 跳转）。这是最稳健、安全的做法。
    *   **武器二（编译器优化）：** 现代语言（如 C++、Java、JS 等）如果支持尾递归优化，写递归也没关系（由编译器在底层帮你复用栈帧）。但**千万不要把程序的安危完全寄托在“编译器是否开启了尾递归优化”上**，最靠谱的办法还是自己在写代码时手动改为循环。

如果你在接下来的课程中，看到**“汉诺塔”**或**“树的前中后序遍历”**等算法，你会发现它们有的非常适合递归（因为很难用循环写，且深度通常是树的高度，不会太深），有的则需要警觉尾递归。
