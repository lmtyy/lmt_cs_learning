这四张PPT是“中缀转后缀”章节的**终极代码实现（C语言版）**。

前面几十页PPT都在讲算法的**概念和动态推演**，而这四页给出了**真正的函数代码**。这段代码在数据结构课程中极具代表性，也是很多计算机专业期末考试的**高频考点**。

我将这4页PPT整合为四个部分为你详细拆解：**辅助工具 -> 初始化准备 -> 核心转换逻辑 -> 收尾工作**。

---

### 第一页 (第72页)：定义“优先级”比较函数 (The Tool)
**核心目标：** 实现一个判断运算符“谁大谁小”的函数，这是整个转换算法的基础。

*   **代码功能：** `int priority(char op)` 返回传入字符的优先级数字。
    *   `*` 和 `/`：返回 **2**（最高）。
    *   `+` 和 `-`：返回 **1**（中等）。
    *   `(` 和 `#`：返回 **0**（最低）。*注意：在这里，左括号 `(` 在栈中作为阻挡墙，优先级被设定为最低，以确保它能被压在栈底，防止被外部符号弹出。*
    *   其他字符（`default`）：返回 **-1**（通常代表这是操作数，如 `a, b, 1, 2` 等）。

---

### 第二页 (第73页)：主函数初始化与处理“左括号”
**核心目标：** 建立操作符栈，并处理最容易的左括号。

*   **函数签名：** `void translate(char *S1, char *S2)`
    *   `S1`：存放**中缀表达式**的源字符串。
    *   `S2`：存放即将生成的**后缀表达式**的目标字符串。
*   **初始化准备：**
    *   `InitStack(OPTR);`：初始化一个**操作符栈（Operator Stack）**。
    *   `Push(OPTR, '#');`：**引入哨兵！** 在栈底放入 `#` 符号。这样做的好处是，当栈为空时，`GetTop` 取出的会是 `#`，其优先级为0，可以防止空栈取顶导致程序崩溃，也方便后续循环判断。
    *   `i = j = 0;`：i 是 S1 的扫描下标，j 是 S2 的输出写入下标。
    *   外层 `while (ch != '#')`：只要从 S1 读到的字符不是 `#`，就一直循环。（通常在 S1 末尾手动加一个 `#`）。
*   **`switch (ch)` 分支：**
    *   **`case '(':`**
        *   **动作：** `Push(OPTR, ch);` 遇到左括号，**直接压入栈中**。
        *   **循环推进：** `ch = S1[++i];` 读取下一个字符。

---

### 第三页 (第74页)：处理“右括号”与“普通运算符”（核心难点）
**核心目标：** 实现遇到 `)` 时的“疯狂弹栈”，以及遇到常规运算符时的“优先级比较”。

*   **`case ')':` (遇到右括号，对应PPT 67页的推演)**
    *   **动作：** 建立一个内部 `while` 循环。用 `GetTop` 查看栈顶，只要栈顶不是左括号 `(`，就把栈顶元素**弹出（Pop）并直接存入输出数组 S2**。这个过程一直持续到遇见左括号。
    *   **清理现场：** 循环结束后，执行 `pop(OPTR, e);` 把栈里的那个左括号 `(` **弹出来扔掉（不存入输出）**，完成括号的闭合。
    *   **循环推进：** 读取下一个字符。
*   **`case '+':` / `case '-':` / `case '*':` / `case '/':` (遇到运算符)**
    *   **动作（“高平等弹出”逻辑）：**
        *   `GetTop(OPTR, e);` 先看一眼当前栈顶是什么。
        *   `while (priority(e) >= priority(ch))`：**核心判断！** 只要栈顶符号的优先级 **大于等于** 当前扫描到的符号 `ch` 的优先级，就执行循环体：
            *   `S2[j++] = e;`：把栈顶符号存入输出数组。
            *   `pop(OPTR, e);`：将其正式从栈里移除。
            *   `GetTop(OPTR, e);`：再继续看新的栈顶，循环比较。
    *   **循环结束后：** `Push(OPTR, ch);` 将当前的运算符 `ch` **压入栈中**。
    *   **循环推进：** 读取下一个字符。

---

### 第四页 (第75页)：处理操作数与最终收尾
**核心目标：** 处理数字/字母，以及整个表达式扫描完毕后的清栈工作。

*   **代码片段补齐：** `{ S2[j++] = e; pop(OPTR,e); GetTop(OPTR,e); }` (这是第三页 while 循环的中间部分，表示弹栈并存入S2的过程)。
*   **`default:` (遇到其它字符，即操作数如 a, b, 1, 2)**
    *   **动作：** `S2[j++] = ch;` **直接输出**到后缀结果数组 S2。
    *   **循环推进：** `ch = S1[++i];`
*   **最外层 `while` 循环结束后的操作：**
    *   `while (!StackEmpty(OPTR)) { pop(OPTR,e); S2[j++] = e; }`：此时中缀字符串已经全部读完了，但栈里可能还残留着之前没来得及输出的运算符（比如最外层的 `+`）。这一步会**把栈里的所有残余符号弹出，并追加到 S2 的末尾**。
    *   `S2[j] = '#';`：在后缀字符串的末尾也加上 `#` 哨兵，表示字符串结束。
    *   `DestroyStack(S);`：**释放内存（好习惯）**，销毁临时使用的栈，防止内存泄漏。

### 💡 综合学习总结与考试/面试默写指南

如果你想写出这段代码，只需要记住**外层的 switch 结构**，以及**内层的 priority 比较**即可。你可以把这段核心逻辑缩写成一段口诀：

```c
while(ch != '#') {
    switch(ch) {
        case '(': Push(OPTR, ch); break;
        case ')': while(top != '(') { pop; output; } pop '('; break;
        case '+': case '-': case '*': case '/':
            while(priority(top) >= priority(ch)) { pop; output; }
            Push(OPTR, ch); break;
        default: output(ch); break; // 操作数直接输出
    }
    ch = next_char;
}
// 最后清空栈：
while(!StackEmpty(OPTR)) { pop; output; }
```

**⚠️ 提示：** 这几页PPT展示的代码严格遵循了我们在之前PPT中推演的**“左括号是墙”、“右括号清空”、“高平等弹出”**三大原则。如果你在期末考试中需要手写“中缀转后缀”，直接默写第72页的`priority`函数和第73-75页的`translate`函数框架，就能拿到高分。
