这三张PPT是**“循环队列（Circular Queue）”**核心操作的**完整代码实现与理论总结**。它们涵盖了入队、求长度、出队以及时间复杂度分析，是数据结构考试中**必考的高频考点**。

这三页的逻辑非常严密：**入队代码（包含判满） -> 求长度公式 -> 出队图解（判空与绕回） -> 出队代码 -> 算法效率总结**。

### 第一页 (第103页)：入队 `EnQueue` 与 求队列长度 `QueueLength`
**核心目标：** 给出入队的标准 C 语言实现，以及“留空一格”法下队列实际长度的计算公式。

*   **1. 入队函数 `EnQueue(SqQueue &Q, QElemType e)`：**
    *   **第1步：判满（直接应用公式）。**
        *   `if (Q.front == (Q.rear+1) % MAXQSIZE) return ERROR;`
        *   如果 `rear` 再走一步就撞上 `front`，说明队列已满（还剩最后一个格子没用），返回失败。
    *   **第2步：存入数据。**
        *   `Q.base[Q.rear] = e;` 把新元素放入当前 `rear` 指向的空位中。
    *   **第3步：队尾指针绕回（最关键的一步）。**
        *   `Q.rear = (Q.rear+1) % MAXQSIZE;` 使用**取模运算 `%`** 让指针加1，如果越界就自动绕回 0。
*   **2. 计算队列长度 `QueueLength(SqQueue Q)`：**
    *   这是一个**必背公式**！
    *   `return (Q.rear - Q.front + MAXQSIZE) % MAXQSIZE;`
    *   **💡 为什么是这么复杂的公式？（核心理解）：**
        *   因为 `rear` 和 `front` 都在绕着圆圈跑。如果 `rear >= front`，直接用 `rear - front` 即可（如 5 - 2 = 3）。
        *   但是，如果 `rear` 已经绕了一圈，比如 `rear = 1`，`front = 5`。此时 `rear - front = -4`，负数会导致逻辑错误。
        *   为了防止负数，我们在括号里强制 **`+ MAXQSIZE`**。哪怕 `rear < front`，加上数组最大容量后依然会得到一个正数。最后再对 `MAXQSIZE` **取模 `%`**，得到最终的实际元素个数。

### 第二页 (第104页)：出队的图解演示
**核心目标：** 以图形方式展示出队时 `front` 指针是如何“擦除”元素并绕回的。

*   **图例解析：**
    *   **Before deleting（出队前）：** `front` 指向 J1，`rear` 指向 J3（后面的空位）。
    *   **After deleting（出队后）：** `front` 顺时针移动了一格，指向了 J2（原本包含 J2 的格子）。J1 虽然物理上还在内存里，但已经被队列逻辑上“删除”了。
*   **下方的伪代码流程：**
    *   **判空：** `If queue q is empty: q.rear == q.front`。如果队尾和队头重合，队列为空。
    *   **出队与指针更新：** `Else : q.front = (q.front + 1) MOD MAXQSIZE`。如果没有空，就把队头指针加 1 并取模绕回。注意这里**不需要清除原来的数据**，只需要让 `front` 跨过它即可。

### 第三页 (第105页)：出队 `DeQueue` 的代码实现与时间复杂度分析
**核心目标：** 将出队的图解转化为 C 代码，并给出整个循环队列的时间复杂度结论。

*   **1. 出队函数 `DeQueue(SqQueue &Q, QElemType &e)`：**
    *   **第1步：判空。**
        *   `if (Q.rear == Q.front) return ERROR;` 如果队头和队尾重合，说明队列里没有元素，直接报错。
    *   **第2步：取出数据。**
        *   `e = Q.base[Q.front];` 把 `front` 当前指向位置的数据取出来，交给传出参数 `e` 带回给调用者。
    *   **第3步：队头指针绕回（同样使用取模）。**
        *   `Q.front = (Q.front+1) % MAXQSIZE;` 队头往前走一步，逻辑上删除了刚才取出的那个元素。
*   **2. 底部黄框：时间复杂度分析（红字强调）**
    *   `"Time consideration: constant time for inserting and deleting."`
    *   **结论：** 入队（Inserting）和出队（Deleting）操作的时间复杂度都是 **O(1)**（常数时间）。
    *   **为什么？** 对比之前“固定队头”的朴素队列，那时的出队需要把后面所有元素往前搬移，时间复杂度是 O(n)。而循环队列仅仅是通过**修改两个整数（`front` 和 `rear` 的下标）**就完成了入队和出队，没有任何数据搬移操作。这体现了循环队列在算法效率上的巨大优势。

### 💡 必背考点总结（考试满分必备）

如果你要在期末考试或面试中展示你对循环队列的扎实掌握，请务必记住这三个“黄金公式”：

1.  **入队判满：** `(Q.rear + 1) % MAXQSIZE == Q.front`  **(留一格法)**
2.  **出队判空：** `Q.front == Q.rear`
3.  **求队列长度：** `(Q.rear - Q.front + MAXQSIZE) % MAXQSIZE`

**⚠️ 避坑指南：**
*   很多同学在写 **出队判空** 时会错误地写成 `if (Q.front == 0)` 或者 `if(Q.rear == 0)`。**这是绝对错误的！** 因为 `front` 和 `rear` 是绕着圆圈跑的，不能看它的绝对下标，只能看它们**两个是否重合**。
*   在 C 语言中，如果使用了 **引用 `&Q`**（如 `SqQueue &Q`），里面操作的是原队列。如果你在用 C 语言（非 C++）考试，记得把 `&Q` 换成指针 `*Q`，用 `Q->front` 的形式去访问。
