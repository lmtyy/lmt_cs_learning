# Fibonacci 数列递归与非递归表达（第7-11页）

本段通过 Fibonacci 数列深入理解递归的执行过程（栈帧变化），并引出非递归的闭合公式。

---

## 第7页：Fibonacci 数列定义

### 递归定义

```
         ┌ 1                n = 0
F(n) =   │ 1                n = 1
         └ F(n-1) + F(n-2)  n > 1
```

### 展开示例

```
F₄ = F₃ + F₂
   = (F₂ + F₁) + (F₁ + F₀)
   = F₂ + F₁ + F₁ + F₀
   = 3F₁ + 2F₀
   = 3×1 + 2×1
   = 5
```

> ✅ 注意这里的定义是 F(0)=1, F(1)=1（有些教材用 F(0)=0, F(1)=1，本课用前者）

## 第8页：Fibonacci 递归代码

```cpp
int Fibonacci(int n)
{
    int f1, f2;
    1   if (n <= 1)  return 1;       // base case
    2   f1 = Fibonacci(n - 1);       // 递归求前一项
    3   f2 = Fibonacci(n - 2);       // 递归求前前一项
    4   return f1 + f2;              // 合并结果
}
```

> 行号标注（1-4）是为了配合后面的栈帧演示。

## 第9页：递归调用栈帧演示 — Fibonacci(3)

PPT 展示了 `Fibonacci(3)` 从调用到返回的完整栈帧变化过程：

### 完整执行流程

```
步骤  当前栈                    执行动作                    说明
────  ──────────────────────    ─────────────────────────   ──────────────
 1    [main] [fib(3)]          Line1: n=3>1, 不返回        进入递归
 2    [main] [fib(3)] [fib(2)] Line2: fib(3)调用fib(2)     继续向下
 3    [main] [fib(3)] [fib(2)] Line1: n=2>1, 不返回        继续
 4    [...] [fib(2)] [fib(1)]  Line2: fib(2)调用fib(1)     再向下
 5    [...] [fib(2)] [fib(1)]  Line1: n=1≤1, return 1      触底！
 6    [main] [fib(3)] [fib(2)] fib(2).f1 = 1               回到fib(2)
 7    [...] [fib(2)] [fib(0)]  Line3: fib(2)调用fib(0)     求f2
 8    [...] [fib(2)] [fib(0)]  Line1: n=0≤1, return 1      触底！
 9    [main] [fib(3)] [fib(2)] fib(2).f2 = 1               ←注意PPT显示f:0
10    [main] [fib(3)] [fib(2)] Line4: return 1+1=2?        ←但PPT按f2=0算
                                                            得return 1
11    [main] [fib(3)]          fib(3).f1 = 1               回到fib(3)
12    [main] [fib(3)] [fib(1)] Line3: fib(3)调用fib(1)     求f2
13    [main] [fib(3)] [fib(1)] Line1: n=1≤1, return 1      触底！
14    [main] [fib(3)]          fib(3).f2 = 1               
15    [main] [fib(3)]          Line4: return 1+1=2         
16    [main]                   main中 x = 2                最终结果
```

### 可视化递归树

```
              fib(3)
             /      \
         fib(2)     fib(1)
         /    \        ↓
     fib(1)  fib(0)  return 1
       ↓       ↓
    return 1  return 1
```

> 类比：递归就像"俄罗斯套娃"式的函数调用，每个"娃"（栈帧）都有自己的局部变量（n, f1, f2），互不干扰。调用时入栈，返回时出栈。

### ⚠️ 重复计算问题

即使 n=3 这么小的规模，`fib(1)` 就被计算了 **2次**。当 n 更大时：

| n | 调用次数 | 重复率 |
|---|---------|--------|
| 5 | 15 | 大量 F(2)、F(1) 重复 |
| 10 | 177 | 严重 |
| 30 | ~270万 | 灾难性 |

> 这正是递归 Fibonacci 的致命缺陷——**指数级时间复杂度 O(2ⁿ)**

## 第10-11页：非递归方式表达（闭合公式）

### 推导思路

从递推关系出发，用特征方程法求解：

**Step 1**：写出递推关系
```
F(n) = F(n-1) + F(n-2)
```

**Step 2**：改写为齐次线性递推
```
F(n) - F(n-1) - F(n-2) = 0
```

**Step 3**：设 F(n) = xⁿ，代入得特征方程
```
x² - x - 1 = 0
```

**Step 4**：求解特征方程
```
x = (1 ± √5) / 2

x₁ = (1 + √5) / 2 ≈ 1.618  （黄金比例 φ）
x₂ = (1 - √5) / 2 ≈ -0.618
```

**Step 5**：通解形式
```
F(n) = c₁·x₁^(n+1) + c₂·x₂^(n+1)
```

**Step 6**：代入初始条件确定常数，得到 Fibonacci 闭合公式

### ✅ Fibonacci 闭合公式（Binet公式）

```
              1      ┌   1 + √5  n+1       1 - √5  n+1  ┐
F(n) = ────────── × │ (────────)    -  (────────)      │
             √5      └      2                 2          ┘
```

### 意义

| 方面 | 说明 |
|------|------|
| 理论价值 | 证明了 F(n) 的增长率是 **Θ(φⁿ)**，其中 φ ≈ 1.618 |
| 实际意义 | 递归算法的时间复杂度确实是指数级的 |
| 计算用途 | 可以 O(1) 直接计算第 n 项（但有浮点精度问题） |

> 类比：特征方程法就像把"递推的锁链"拆解成"独立的公式"，不用一步步推，直接跳到答案。

## 总结

```
Fibonacci 递归的教训：

递归代码：简洁 → 但 O(2ⁿ) 指数爆炸
  │
  ├─ 原因：大量重复子问题（overlapping subproblems）
  │
  ├─ 解法1：迭代（自底向上，O(n)）
  ├─ 解法2：记忆化递归（备忘录，O(n)）
  ├─ 解法3：矩阵快速幂（O(log n)）
  └─ 解法4：Binet闭合公式（O(1)，但有精度问题）

特征方程法：
  递推关系 → 特征方程 x²-x-1=0 → 通解 → 代入初值 → 闭合公式
```
