# 递归 Recursion — 概念与阶乘示例（第3-6页）

本段回顾递归的基本概念和条件，以阶乘函数作为入门示例，为后续分治法奠定基础。

---

## 第3页：递归（Recursion）定义

### 什么是递归？

- **直接**调用自己，或**间接**通过一些语句调用自己
- 一个直接或间接地调用自身的算法称为**递归算法**
- 一个使用函数自身给出定义的函数称为**递归函数**

### 优缺点

| | 说明 |
|------|------|
| ✅ 优点 | 描述某些数学问题非常自然，证明算法很容易 |
| ❌ 缺点 | 时间、空间消耗多（函数调用栈开销） |

### 递归的两个阶段

```
        回推（从未知到已知方向推导）
  ┌──────────────────────────────────┐
  │          ←←←←←←←←←←←           │
  ▼                                  │
┌──────┐                         ┌──────┐
│ 未知  │                         │ 已知  │
└──────┘                         └──────┘
  │           →→→→→→→→→→→           ▲
  │                                  │
  └──────────────────────────────────┘
        递推（从已知逐步返回结果）
```

> 类比：问路 → 你不知道终点怎么走（未知），就一直问下去（回推），直到问到一个确切知道答案的人（已知/base case），然后答案逐层传回来（递推）。

## 第4页：例 — 阶乘函数 Factorial

### 数学定义

```
        ┌ 1            n = 0
n! =    │
        └ n·(n-1)!     n > 0
```

- `n = 0` 时直接给出结果 → **非递归部分，终止条件**

### 代码实现

```cpp
int Factorial(int n)
{
    if (n == 0) return 1;       // ← 非递归部分（base case）
    return n * Factorial(n-1);  // ← 递归部分
}
```

### 递归调用展开：Factorial(4)

```
调用过程（回推）：                返回过程（递推）：

Factorial(4)                     → 4 × 6  = 24  ← 最终结果
  └→ 4 * Factorial(3)           → 3 × 2  = 6
       └→ 3 * Factorial(2)      → 2 × 1  = 2
            └→ 2 * Factorial(1)  → 1 × 1  = 1
                 └→ 1 * Factorial(0)
                      └→ return 1  ← 触底（base case）
```

> ✅ 两个阶段清晰可见：先一路"递"到 base case，再一路"归"回来算结果。

## 第5-6页：递归必须满足的条件

### 条件一：必须有基本部分（base）

> 定义中必须包含一个**基本部分(base)**，其中对于 n 的一个或多个值，f(n) 必须是直接定义的（递归出口）。

### 条件二：递归部分必须趋向 base

> 在递归部分 **(recursive component)** 中，右侧所出现的所有 f 的参数都必须有一个比 n 小，以便重复运用递归部分来改变右侧出现的 f，直至出现 f 的基本部分。

### 总结为四条铁律

| # | 条件 | Factorial 中的体现 |
|---|------|-------------------|
| 1 | 递归定义必须有**确切的含义** | n! = n × (n-1)!，数学上严格 |
| 2 | 一步比一步简单，最后有**终结**，不能无限循环 | n 每次减 1，最终到 0 |
| 3 | "递归出口"本身**不再使用递归定义** | `n==0` 时直接 `return 1` |
| 4 | 每个递归函数都必须有**非递归定义的初始值** | Factorial(0) = 1 |

> 类比：递归就像一面镜子对着另一面镜子，如果没有终止条件，就会"无限走廊"——程序表现为**栈溢出(Stack Overflow)**。

---

## 总结

```
递归三要素检查清单：

  ┌─ ① Base Case（终止条件）
  │     → 直接给出结果，不调用自身
  │
  ├─ ② Recursive Step（递归步骤）
  │     → 调用自身，但参数必须更"小"
  │
  └─ ③ Convergence（收敛性）
        → 保证经过有限步一定能到达 base case

阶乘 Factorial 验证：
  ① base: n==0 → return 1           ✓
  ② step: n*Factorial(n-1)          ✓
  ③ 收敛: n 每次减1，必达0            ✓
```
