# 分治法框架与时间复杂度分析（第28-34页）

正式定义分治法的通用框架、伪代码模板，以及分治递推关系的一般求解公式。

---

## 第28页：Divide and Conquer — 定义

> The most well known algorithm design strategy:
> （最著名的算法设计策略）

三步走：

1. **Divide** — 将问题实例拆分为两个或更多个更小的实例
2. **Conquer** — 递归地解决这些更小的实例
3. **Combine** — 将这些解合并，得到原始（更大）问题的解

---

## 第29页：分治法的基本思想（中文表述）

分而治之方法与软件设计的**模块化方法**非常相似。为了解决一个大的问题，可以：

1. 把它分成两个或多个更小的问题；
2. 分别解决每个小问题；
3. 把各小问题的解答组合起来，即可得到原问题的解答。

> ✅ **关键特征**：小问题通常与原问题**相似**（同类型），可以递归地使用分而治之策略来解决。

## 第30页：分治法递归结构图

```
              ┌────────┐
              │ 原问题  │
              └───┬────┘
         ┌────────┼────────┐
         ▼        ▼        ▼
    ┌────────┐ ┌────────┐ ┌────────┐
    │子问题1  │ │子问题2  │ │子问题k  │  ← 与原问题 相同类型
    └───┬────┘ └───┬────┘ └───┬────┘
        ▼          ▼          ▼
   ┌────────┐ ┌────────┐ ┌────────┐
   │子问题1' │ │子问题2' │ │子问题3' │  ← 继续分解
   └───┬────┘ └────────┘ └────────┘
       ▼
  ┌──────────┐
  │ 不可再分，│  ← Base Case
  │ 直接求解  │
  └──────────┘
```

### 核心特征

| 特征 | 说明 |
|------|------|
| 子问题与原问题**同类型** | 才能递归调用同一个函数 |
| 层层分解 | 直到规模足够小（不可再分） |
| 自底向上合并 | 小问题的解组合成大问题的解 |

## 第31页：分治法伪代码框架

用 **P** 表示问题的输入，|P| 表示问题规模：

```
Divide-and-Conquer(P)
{
    if (|P| <= n₀)  Adhoc(P);        // 规模够小，直接求解
    
    divide P into smaller subinstances
        P1, P2, …, Pk;               // 分割成 k 个子问题
    
    for (i = 1; i <= k; i++)
        yi = Divide-and-Conquer(Pi);  // 递归解决每个子问题
    
    return Merge(y1, …, yk);          // 合并得到最后结果
}
```

### 各部分职责

| 部分 | 函数 | 作用 |
|------|------|------|
| 判断是否够小 | `|P| <= n₀` | 判断输入规模 P 是否足够的小 |
| 直接求解 | `Adhoc(P)` | 解该规模问题的函数（base case） |
| 分割 | `divide P into P1…Pk` | 分割函数 |
| 递归求解 | `Divide-and-Conquer(Pi)` | 分治解决 |
| 合并 | `Merge(y1,…,yk)` | 合并得到最后结果 |

## 第32页：设计分治法时的关键问题

> 1. 应把原问题分为**多少个**子问题才较适宜？
> 2. 每个子问题是否**规模相同**或怎样才为适当？

### 经验法则

| 问题 | 常见选择 | 原因 |
|------|---------|------|
| 分几个子问题？ | 通常 **k = 2**（二分） | 分析简单，递归树平衡 |
| 子问题规模？ | 通常**等分**（n/m，m=2） | 保证递归树平衡，效率最优 |

> 不等分的分治也存在（如 QuickSort 最坏情况分成 1 和 n-1），但平衡分割通常效率更高。

## 第33页：分治法计算效率 — 参数定义

| 符号 | 含义 |
|------|------|
| **n** | 原问题规模 |
| **k** | 分成 k 个子问题 |
| **n/m** | 每个子问题的规模（m 为缩减因子） |
| **n₀** | 分解阈值，设 n₀ = 1 |
| **Adhoc** | 解规模为 1 的问题，耗费 **1** 个单位时间 |
| **f(n)** | 分解原问题 + Merge 合并 k 个子问题的解所需时间 |
| **T(n)** | 分治法解规模为 \|P\| = n 的问题所需的计算时间 |

---

## 第34页：计算时间分析 — 通用递推公式

### 递推关系

```
         ┌ O(1)              n = 1
T(n) =   │
         └ kT(n/m) + f(n)   n > 1
```

- `kT(n/m)`：k 个规模为 n/m 的子问题
- `f(n)`：分解 + 合并的代价

### 通用解（展开法）

$$T(n) = n^{\log_m k} + \sum_{j=0}^{\log_m n - 1} k^j \cdot f(n/m^j)$$

### 直觉理解

| 项 | 含义 |
|---|------|
| $n^{\log_m k}$ | 所有叶子节点的代价总和（递归到 base case） |
| $\sum k^j \cdot f(n/m^j)$ | 每一层分解+合并的代价累加 |

### 常见特例（Master Theorem 的雏形）

| k | m | f(n) | T(n) | 典型算法 |
|---|---|------|------|---------|
| 1 | 2 | O(1) | O(log n) | 二分搜索 |
| 2 | 2 | O(n) | O(n log n) | MergeSort |
| 2 | 2 | O(1) | O(n) | 遍历 |
| 2 | 2 | O(n²) | O(n²) | 某些矩阵算法 |

> ✅ 这个公式是后续分析所有分治算法复杂度的基础工具。

## 总结

```
分治法通用模板：

Divide-and-Conquer(P):
  ├── if small enough → Adhoc(P)        [O(1)]
  ├── Divide into P1, P2, …, Pk         [分割]
  ├── for each Pi → recurse              [k × T(n/m)]
  └── Merge(y1, …, yk)                   [f(n)]

时间复杂度递推：
  T(n) = kT(n/m) + f(n)

通用解：
  T(n) = n^(log_m k) + Σ k^j · f(n/m^j)

设计要点：
  • 子问题数 k：通常取 2
  • 缩减因子 m：通常取 2（等分）
  • 关键在于 f(n)（分割+合并代价）的大小
```
