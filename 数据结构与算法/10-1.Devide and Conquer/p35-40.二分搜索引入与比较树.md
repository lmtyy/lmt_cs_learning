# 二分搜索技术 — 问题引入与比较树（第35-40页）

从线性搜索的不足引出二分搜索的思想，用"二元比较树"分析搜索过程。

---

## 第35页：问题提出

### 搜索问题定义

给定**已排好序**的 n 个元素 `a[0:n-1]`，现要在这 n 个元素中找出一特定元素 **x**：
- 找到 → 将其位置赋于变量 j
- 没找到 → j 置 -1

### 约束

> 如果只允许进行元素间的**比较**而不允许对它们进行其它的运算，则所设计的算法称为**以比较为基础的算法**。

> 类比：你只能问"这个数比 x 大还是小？"，不能做加减乘除。

---

## 第36页：线性搜索及其比较树

### 线性搜索（Sequential Search）

从头到尾逐个比较：先比 A[1]，再比 A[2]，……直到找到或遍历完。

### 二元比较树描述

任何以比较为基础的搜索算法的执行过程都可以用一棵**二元比较树**来描述：
- 每次比较 → 一个**内顶点**（圆形节点），代表尚不知道x是否属于A
- 不成功的结果 → 一个**外顶点**（方形节点），代表x一定不属于A

### 线性搜索的比较树

```
           x:A[1]
          /      \
     不成功     x:A[2]
               /      \
          不成功      ...
                        \
                      x:A[n]
                     /   |    \
            A[n-1]<x<A[n]  不成功  X>A[n]
                           不成功   不成功
```

> 线性搜索的比较树是一棵**退化的斜树**——每次只排除一个元素。

---

## 第37页：线性算法复杂度

> 该方法没有很好利用已经**排好序**这个条件，顺序搜索方法平均需要 **O(n)** 次比较。

### 问题

既然数组已排序，能不能每次比较排除更多元素？

→ 答案就是**二分搜索**！

---

## 第38页：二分搜索方法

### 基本思想

取一个**中间元素**做比较元素：
- 如果 x **等于**它 → 结束，找到了
- 如果 x **小于**它 → 去**左半部**查找
- 如果 x **大于**它 → 去**右半部**搜索

### 分治法形式化

将问题表示为：`I = (n, a₁, …, aₙ, x)`

选取一个下标 **k**，可得到三个子问题：

| 子问题 | 表示 | 含义 |
|--------|------|------|
| I₁ | (k-1, a₁, …, a_{k-1}, x) | 左半部分 |
| I₂ | (1, a_k, x) | 中间元素本身 |
| I₃ | (n-k, a_{k+1}, …, aₙ, x) | 右半部分 |

```
┌──────────────────┬────┬──────────────────┐
│    I₁ (左半)      │a[k]│    I₃ (右半)      │
│  a₁ … a_{k-1}    │    │ a_{k+1} … aₙ    │
└──────────────────┴────┴──────────────────┘
                     ↑
                   比较点
```

> ✅ 每次比较后，只需处理三个子问题中的**一个**（而非全部），规模直接砍半！

---

## 第39页：二元比较树 — 9个元素的示例

含 9 个元素（下标 0-8）的二分搜索比较树：

```
                    (4)
                 /       \
              (1)         (6)
            /     \     /     \
          (0)    (2)  (5)    (7)
                   \           \
                   (3)         (8)
```

### 搜索路径示例

| 查找目标在位置 | 比较路径 | 比较次数 |
|-------------|---------|---------|
| a[4] | 4 | 1 |
| a[1] | 4→1 | 2 |
| a[6] | 4→6 | 2 |
| a[0] | 4→1→0 | 3 |
| a[3] | 4→1→2→3 | 4 |
| a[8] | 4→6→7→8 | 4 |

> 最多比较 ⌈log₂(9)⌉=4次

---

## 第40页：二元比较树 — 一般结构

### 二分搜索比较树的通用结构

```
                x:A[⌊(n-1)/2⌋]              ← 根节点（中间元素）
               /                  \
    x:A[⌊(n-3)/4⌋]          x:A[⌊(3n-1)/4⌋]    ← 第二层
      /         \              /          \
  x:A[0]    x:A[⌊(n-1)/2⌋-1]   x:A[⌊(n-1)/2⌋+1]   x:A[n-1]
   / \        / \                 / \              / \
  □  □      □  □               □  □            □  □
```

其中：
- **圆形节点**（内顶点）= 成功比较（找到元素的位置）
- **方形节点**（外顶点）= 不成功（元素不在数组中）

### 关键性质

| 性质 | 说明 |
|------|------|
| 内顶点数 | n（对应 n 个元素） |
| 外顶点数 | n + 1（对应 n+1 个"间隙"） |
| 树高 | ⌈log₂(n+1)⌉ |
| 最多比较次数 | ⌈log₂(n+1)⌉ |

> ✅ 二分搜索每次排除**一半**元素，所以比较树是**平衡的**，高度为 O(log n)。

---

## 总结

```
线性搜索 vs 二分搜索：

线性搜索：
  比较树 = 退化斜树（链状）
  每次只排除 1 个元素
  复杂度：O(n)

二分搜索：
  比较树 = 平衡二叉树
  每次排除一半元素
  复杂度：O(log n)

二分搜索的分治结构：
  ├── 比较 x 和 A[mid]
  ├── 若相等 → 找到
  ├── 若 x < A[mid] → 在左半搜索（规模砍半）
  └── 若 x > A[mid] → 在右半搜索（规模砍半）
```
