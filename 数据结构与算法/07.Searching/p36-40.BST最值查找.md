这5页PPT详细讲解了二叉搜索树（BST）中**查找最小值（Find_min）和最大值（Find_max）**的操作。

这是一组非常经典的教学PPT，它不仅讲了算法逻辑，还特意展示了**同一个功能，用“递归”和“非递归（迭代）”两种不同代码实现**的对比。

我为你逐页梳理：

---

### 🟨 Slide 36：任务定义
*   **核心功能**：`find_min` 和 `find_max` 是两种例程（函数）。
*   **返回值**：返回树中**最小元素**和**最大元素**所在节点的**位置（指针）**。注意这里不是返回具体的数值，而是返回内存中的节点地址。

### 🟨 Slide 37：算法逻辑原理（非常直观）
结合二叉搜索树“左小右大”的严格定义，查找最值不用遍历所有节点，只需要“一条路走到黑”：
*   **找最小（find_min）**：从根节点开始，**只要存在左孩子，就一直往左走**。走到最尽头（没有左孩子了），那个停下来的节点就是全树最小的元素。
*   **找最大（find_max）**：逻辑完全相反，**一直往右走**。走到最尽头（没有右孩子了），那个停下来的节点就是全树最大的元素。

### 🟨 Slide 38：教学意图（为什么给两种写法）
*   **第一句话**：“这个逻辑太简单了，以至于很多程序员**根本懒得用递归**（改用循环）。”
*   **讲师安排**：为了让学生同时掌握这两种编程思维，PPT里决定给大家展示双拼方案：
    *   `find_min` 使用 **递归** 方式实现。
    *   `find_max` 使用 **非递归（迭代/循环）** 方式实现。
*   *延伸意义*：递归代码简洁，但会消耗函数调用栈空间（O(log n)）；非递归代码稍复杂，但只消耗常数空间（O(1)），在实际工程中尤其适合处理树很深的情况。

### 🟨 Slide 39：`find_min` 的递归代码实现
*   **三个边界/分支情况**（这段代码写得非常标准）：
    1.  **出口 1（空树）**：`if( T == NULL ) return NULL;` —— 如果根节点就是空的，说明树不存在，直接返回 NULL。
    2.  **出口 2（找到了）**：`else if( T->left == NULL ) return( T );` —— 如果当前节点的**左指针是空的**，说明往左走不动了。**当前这个节点 `T` 就是最小值**，直接返回 `T`。
    3.  **递归动作**：`else return( find_min ( T->left ) );` —— 如果上面都不满足（有左孩子），那就递归调用 `find_min`，去**左孩子**那边继续找。

### 🟨 Slide 40：`find_max` 的非递归（迭代）代码实现
这段代码采用 `while` 循环来代替函数的自我调用：
*   **安全判断**：`if( T != NULL )` —— 先确保这棵树不是空的。
*   **循环核心**：`while( T->right != NULL ) { T = T->right; }`
    *   只要当前节点的右指针 `right` 不为空，就把 `T` 指针向右移一下（`T = T->right`）。这就相当于沿着右侧“单行道”一直往下走。
*   **返回结果**：循环结束时（此时 `T->right` 已经是 NULL 了，走到头了），`return T;` 把找到的最右侧节点返回。

**💡 进阶知识点（与下一章 Delete 操作紧密相关）：**
为什么 PPT 要专门花几页讲怎么找最小和最大？
因为**在二叉搜索树中删除一个节点时，如果有两个子节点（左右都有），核心算法就是用“左子树的最大值”或者“右子树的最小值”来代替被删节点**！
所以，你现在学习的这两个 `find_min` 和 `find_max`，是为未来最复杂的 BST **“删除节点（Delete）”** 操作打基础的。一定要掌握这两种写法。
