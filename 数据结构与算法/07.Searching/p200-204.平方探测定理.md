这5张PPT（200-204）是平方探测（Quadratic Probing）中非常硬核的理论证明与工程实践警戒线。

它们接续之前的定理，用严谨的数学推导证明了**“为什么满足两个条件（表大小是质数、装填因子<0.5）就一定能插入成功”**，随后指出了如果在实际代码中**违反这两个条件，或者试图做标准删除**，会引发怎样的灾难。

我为你把这5页PPT拆解为三个递进层次来详细讲解：

---

### 🟨 第一层：数学定理的严谨证明（Slide 200 - 202）
*   **Slide 200（推导冲突的前提）**：
    PPT假设一种最坏情况：在平方探测的过程中，两次不同的探测（分别为第 `i` 次和第 `j` 次，且 `i ≠ j`）**计算出了同一个数组位置**。
    公式推导：`h(x) + i² ≡ h(x) + j² (mod H_SIZE)`，把相同的哈希值消去后，得到 `i² ≡ j² (mod H_SIZE)`，最终通过因式分解得到 **(i - j)(i + j) ≡ 0 (mod H_SIZE)**。
*   **Slide 201（反证法论证）**：
    *   由于表的大小 `H_SIZE` 是**质数**，根据数学性质，如果 `(i - j)(i + j)` 能被 `H_SIZE` 整除，那么要么 `(i - j)` 是 `H_SIZE` 的倍数，要么 `(i + j)` 是 `H_SIZE` 的倍数。
    *   **推导矛盾**：因为我们限制探测的步数 `i` 和 `j` 都小于 `H_SIZE/2`，那么：
        *   `(i - j)` 的绝对值肯定小于 `H_SIZE/2`，绝对不可能是 `H_SIZE` 的倍数。
        *   `(i + j)` 的和肯定小于 `H_SIZE`，也绝对不可能是 `H_SIZE` 的倍数。
    *   由于这两种情况都“不可能”，所以前提（出现重复位置）彻底不成立！
*   **Slide 202（最终结论）**：
    *   证明了在探测的前 `H_SIZE/2` 次内，**所有探测到的替补位置绝对不会重复**。
    *   因为加上原位置，每一个键至少有 `H_SIZE/2 + 1` 个互不相同的落脚点可以选。
    *   **实战结论**：只要哈希表里**现有元素不超过一半**，**必然**能在这前一半的探测序列中找到至少一个空位，插入绝不会死循环。

### 🟨 第二层：实战工程禁忌——表大小必须为质数（Slide 203）
这一页用具体的反例，警告了在代码中忽视“质数”条件的可怕后果。
*   **反面教材**：假设你的哈希表大小设定为 **16**（不是质数）。
*   **后果**：发生冲突后，平方探测的步长分别是 1, 4, 9, 16, 25...
    *   `16 % 16 = 0`（原地踏步）
    *   `25 % 16 = 9`（回到了之前探测过的位置）
*   **结论**：在非质数的哈希表中，平方探测的可选位置会**急剧减少**，可能很快陷入死循环，根本找不到空位。**因此，在实现平方探测时，哈希表的大小必须被强制约束为质数！**

### 🟨 第三层：开放寻址的致命局限——无法物理删除（Slide 204）
这页PPT展示了一个非常反直觉的结论：**在开放寻址的哈希表中，你无法直接删掉（置为NULL）一个元素。**
*   **为什么不能直接置空？**
    *   因为当初插入元素时，我们用“线性探测”或“平方探测”规避了冲突，留下了一条探测的**“锁链”**（比如：A本来该去位置1，但被占了，一路探到了位置5）。
    *   如果你**直接清空**了中间的一个元素（比如位置3），这条锁链就断了。
*   **灾难性后果（PPT示例）**：
    *   如果有元素 B、C 当初是因为遭遇冲突，**跨过位置3** 才插到后面的。你一旦把位置3置空，当程序再来查找 B 或 C 时，计算初始哈希后，一路顺着探测走到位置3，发现这里是空的。**程序会误以为“数据不存在”，直接返回查找失败！**（PPT明确指出 `virtually all of the remaining finds will fail`）。
*   **唯一的解药：懒删除（Lazy Deletion）**
    *   要删除一个节点时，**绝对不能把它设为NULL**。而是应该把它标记为**“已被逻辑删除（标记位）”**。
    *   在查找时：遇到“已删除”标记，**当成“已占用”继续往下探测**。
    *   在插入时：遇到“已删除”标记，**可以覆盖写回**（成功复用这块内存）。
    *   *（PPT最后一句幽默地总结：虽然叫“懒删除”，但其实一点也不懒，因为这是强制必须遵守的规则，否则程序就会出Bug。）*

**💡 综合总结：**
这5页PPT把平方探测从“纸上谈兵”拉回到了**“工程落地必须考虑的边界条件”**。如果你要手写一个平方探测的哈希表，**必须死死记住这三个硬性法则**：
1. 表大小选**质数**（避免探测循环死锁）。
2. 装填因子**小于 0.5** 时必须立即扩容（保证能找到空位）。
3. 删除时必须使用**懒删除**标记（防止探测链断裂）。
