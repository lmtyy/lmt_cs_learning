### 🟨 Slide 83 & 84：AVL 树的“最坏形态”长什么样？
*   **PPT的目的**：这两页展示了一棵高度为 **9**、但节点数量**最少（仅 143 个）**的 AVL 树。
*   **为什么这么画？**
    *   为了在数学上证明 AVL 树性能的“下限”。我们需要知道，**在 AVL 规则允许的“最歪”情况下，这棵树最多能有多高。**
*   **结构分析（Slide 84）**：
    *   左子树是一棵高度为 `7` 的最小规模 AVL 树。
    *   右子树是一棵高度为 `8` 的最小规模 AVL 树。
    *   *原理*：在 AVL 规则下（高度差 ≤ 1），为了让根节点的高度尽可能大，我们可以让一边比另一边刚好高出 1 层。这种**“左矮右高”**的组合，能够用最少的节点，撑出最高的树。

### 🟨 Slide 85：AVL 树的“灵魂公式”（与斐波那契数列的关系）
这是证明 AVL 树性能优秀的关键数学推导：
*   **公式**：`N(h) = N(h-1) + N(h-2) + 1`
    *   **解释**：要构造一个高度为 `h`、但节点数最少的 AVL 树，它的组成是：**1个根节点 + 左子树(高度h-1的最少节点) + 右子树(高度h-2的最少节点)**。
*   **基础情况**：
    *   当 `h=0` 时（只有一个根节点），`N(0)=1`。
    *   当 `h=1` 时（根节点带一个或两个孩子），`N(1)=2`。
*   **惊人的结论**：
    *   PPT明确指出：这个公式 `N(h)` 的数列和**斐波那契数列（Fibonacci）**极度相似！
    *   数学推导表明，当节点数为 `n` 时，AVL 树的最大高度 `h ≈ 1.44 * log₂(n)`。
    *   虽然比完美平衡树的 `log₂(n)` 稍微高一点（多了 0.44 倍），但在大 O 表示法中，**它依然严格属于 `O(log n)` 级别**。这就保证了 AVL 树的查找、插入、删除永远是极快的。

### 🟨 Slide 86：实战判定——高度与平衡因子（BF）
在写 AVL 树的代码之前，必须先定义好如何“衡量”树。
*   **三个核心定义**（PPT下方文字）：
    1.  **`height of node = h`**（节点的高度）。
    2.  **`balance factor = h_left - h_right`**（**平衡因子 = 左子树高度 - 右子树高度**）。
    3.  **`empty height = -1`**（**空节点的高度约定为 -1**，这是非常关键的基准线）。
*   **图上实例验证（Tree A）**：
    *   PPT图中节点旁边的红色数字代表**平衡因子（BF）**。
    *   **根节点 `6`**：左子树是节点 `4`（高度1），右子树是节点 `9`（高度0）。它的 `BF = 1 - 0 = 1`。
    *   **节点 `4`**：左子树是 `1`（高度0），右子树是 `5`（高度0）。它的 `BF = 0 - 0 = 0`。
    *   *(注：PPT图中节点4旁边标的数字是1，这应该是课件原图的笔误。按照下方的公式定义，**BF = 左高 - 右高**，所以节点4的平衡因子严格等于 **0**。)*
*   **学完这页的终极结论**：一棵树是 AVL 树，当且仅当这棵树**所有节点的平衡因子（BF）都是 -1、0 或 1**。如果出现 -2 或者 2，就说明树“歪”了，触发了危险的平衡条件。
