这页PPT是一份关于**数据结构（特别是“查找”算法）**章节的课程大纲（Overview）。它像一张地图，展示了接下来要学习的五个核心模块。

我为你逐一详细讲解这五个部分的具体内容以及它们之间的逻辑关系：

### 1. Searching Problem（查找问题）
*   **核心概念**：这是本章的总起点。在数据结构中，“查找”指的是在一个数据集合（查找表）中，找出满足特定条件的数据元素。
*   **关键要素**：通常涉及**关键字（Key）**——用来标识一个数据元素的某个数据项（如学号、身份证号）。
*   **目标**：如何设计高效的数据存储结构，以便快速找到我们需要的数据。

### 2. Static Searching Problem（静态查找问题）
*   **核心概念**：“静态”意味着**数据集合在查找过程中不会发生改变**（即没有插入和删除操作，只进行查找）。
*   **常见算法**：
    *   **顺序查找**：从头到尾挨个找，效率低（O(n)），但不需要数据有序。
    *   **二分查找（折半查找）**：要求数据必须是有序的，利用分治思想，效率高（O(log n)）。
*   **物理结构**：这部分通常使用**数组**来实现。

### 3. Binary Search Tree（二叉搜索树 / 二叉排序树 - BST）
*   **核心概念**：进入**动态查找**领域。它是一棵二叉树，特点是**左子树所有节点值 < 根节点值 < 右子树所有节点值**。
*   **优点**：不仅支持查找，还支持高效的**插入**和**删除**操作。
*   **缺点（引出下一节）**：如果输入的数据是本来就有序的（如 1, 2, 3, 4, 5），二叉搜索树会退化成一条“链表”，查找效率会降回 O(n)。

### 4. AVL Tree（AVL树 / 平衡二叉搜索树）
*   **核心概念**：为了解决二叉搜索树退化的问题而诞生。AVL树是一种**高度平衡**的二叉搜索树。
*   **平衡条件**：任意节点的左子树和右子树的高度差（平衡因子）绝对值不超过 1。
*   **机制**：当插入或删除节点导致树失去平衡时，它会通过**旋转操作**（左旋、右旋等）自动调整树的形状，保持树的高度为 O(log n)，从而保证查找效率始终稳定在 O(log n)。

### 5. Hashing（哈希 / 散列查找）
*   **核心概念**：这是一种完全不同的查找思路。前面四种方法都是基于“比较”来查找（比较值的大小），而哈希是通过**计算**来查找。
*   **原理**：利用一个**哈希函数**，将关键字直接映射到存储空间的地址（数组下标）。理想情况下，查找时间复杂度为 **O(1)**（常数时间，极快）。
*   **难点（冲突处理）**：不同的关键字可能映射到同一个地址，这称为**哈希冲突**。这部分会讲解如何处理冲突（如开放地址法、链地址法等）。

---

**💡 总结：这页PPT的逻辑递进关系**
1.  **定义问题**：什么是查找？
2.  **简单情况**：如果不怎么变动的数据，怎么找？（静态查找：二分法）。
3.  **动态情况**：如果数据要频繁变动，怎么找？（BST树）。
4.  **优化情况**：BST树会退化，怎么解决？（AVL树）。
5.  **另一种思路**：能不能不比较，直接算出位置？（哈希表）。
