### 第二页：本章概览 (Overview)

这页列出了本章将要详细讲解的知识点目录。从这些条目可以推断，这门课将从理论（ADT）到语言实现（Struct/Union），再到具体应用（Polynomial/Sparse Array）来全方位讲解数组。

以下是各个条目的具体含义：

**1. ADT of Array (数组的抽象数据类型)**

- **含义：** ADT 代表 Abstract Data Type。这里不会先谈具体的内存地址，而是先定义数组在逻辑上是什么。
- **核心：** 定义数组的数据对象（一组具有相同类型的元素）和数据关系（线性关系，通过索引映射），以及对数组可以进行的操作（如初始化、销毁、取值、赋值等）。

**2. Structure and Unions (结构体与联合体)**

- **含义：** 这是 C/C++ 语言中的概念。
- **Structure (结构体)：** 允许你将不同类型的数据组合在一起（例如，一个“学生”结构体可能包含姓名字符串、年龄整数和分浮点数）。
- **Union (联合体)：** 一种特殊的结构，允许在相同的内存位置存储不同的数据类型（但在同一时间只能存储其中一种）。
- **目的：** 在数据结构课程中，这两者通常用于定义复杂的节点或处理多态数据。

**3. ADT of List (线性表的抽象数据类型)**

- **含义：** 定义什么是“线性表”（Linear List）。
- **核心：** 这是一个更有序的序列概念（$a_1, a_2, ..., a_n$）。它定义了插入、删除、查找等逻辑操作。数组是实现线性表的一种方式。

**4. Sequence List (顺序表)**

- **含义：** 这是线性表（List）的一种具体**物理实现**。
- **核心：** 使用一段**地址连续**的存储单元（即数组）依次存储线性表的数据元素。这是本章的重点，即如何用“数组”来实现“线性表”。

**5. ADT of Polynomial (多项式的抽象数据类型)**

- **含义：** 这是一个经典的数组/线性表**应用案例**。
- **核心：** 讲解如何用数组或线性表来表示数学中的多项式（例如 $P(x) = 3x^4 + 2x^2 + 5$），以及如何通过计算机程序实现多项式的加减乘除运算。

**6. Special Array and Sparse Array (特殊矩阵与稀疏矩阵)**

- **含义：** 涉及二维或多维数组的高级存储技术。
- **Special Array (特殊矩阵)：** 如对称矩阵、三角矩阵，可以通过特殊映射压缩存储空间。
- **Sparse Array (稀疏矩阵)：** 指大部分元素为 0 的大矩阵。为了节省空间，不会直接存成巨大的二维数组，而是使用“三元组”等方式只存储非零元素。

**7. String (字符串)**

- **含义：** 字符串在底层本质上就是字符数组（Array of characters）。
- **核心：** 可能会讲解字符串的存储结构以及模式匹配算法（如 KMP 算法）。
