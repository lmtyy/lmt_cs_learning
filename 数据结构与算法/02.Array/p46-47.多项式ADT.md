这两页 PPT 展示的是**“一元多项式（Polynomial）的抽象数据类型（ADT）定义”**。

在数据结构课程中，当你设计好数据的**存储结构**（比如上一组 PPT 提到的“稀疏结构体数组”或“稠密数组”）之后，下一步就是要定义**在这个结构上能进行哪些操作**。这两页 PPT 就是给多项式这个对象写的一份**“使用说明书”**。

为了让你更清晰地理解，我把这些函数按照“生命周期管理”、“核心运算”和“辅助工具”三个维度进行归类详细讲解。

---

### 第一部分：生命周期的管理（图1）
这部分函数负责“创造”一个多项式，以及“销毁”它，是资源管理的基石。

1.  **`void CreatePolyn(polynomial &P, int m)`**
    *   **功能**：构造并初始化一个多项式。
    *   **参数解读**：
        *   `&P`：引用传递，表示这个函数会直接修改传进来的多项式 P。
        *   `int m`：表示这个多项式由 `m` 个**非零项**组成。
    *   **背后的执行流程**：函数内部会接收用户输入的 m 个（系数，指数）对，并将它们按某种顺序（通常要求按照指数从小到大的顺序排列）存入 `P.elem` 数组中。

2.  **`void DestroyPolyn(polynomial &P)`**
    *   **功能**：销毁多项式 P。
    *   **为什么需要这个？** 如果你的多项式结构体内部的 `elem` 数组是使用 `malloc` **动态分配**的（在 C 语言中），这个函数内部必须包含 `free(P.elem)` 来释放掉申请的内存。如果没有这一步，就会造成**内存泄漏**。

3.  **`int PolynLength(polynomial P)`**
    *   **功能**：返回多项式的**非零项个数**。
    *   **返回结果**：直接返回结构体内部的 `P.noelem` 值。

4.  **`void PrintPolyn(polynomial P)`**
    *   **功能**：将多项式的数学公式打印在屏幕上。
    *   **实现逻辑**：遍历结构体数组，按照 `系数x^指数` 的格式输出。

### 第二部分：核心数学运算（图2）
这部分是多项式 ADT 的精华所在，定义了多项式的数学运算接口。

1.  **`void AddPolyn(polynomial &Pa, polynomial &Pb)`**
    *   **功能**：多项式**加法**。
    *   **注释解析**：`//Addition, Pa=Pa+Pb, and then destroy Pb`。
    *   **解读（非常重要）**：算法执行完毕后，**Pa 被修改**，变成了 `Pa` 和 `Pb` 相加的结果；而 **Pb 会被直接销毁（释放内存）**。这种设计（将一个运算结果覆盖到其中一个操作数上，并销毁另一个）是 C 语言数据结构算法为了节省内存空间的常见做法。

2.  **`void SubPolyn(polynomial &Pa, polynomial &Pb)`**
    *   **功能**：多项式**减法**。
    *   **注释解析**：`//Subtraction, Pa=Pa-Pb, and then destroy Pb`。
    *   **逻辑**：和加法类似，它会在原地更新 `Pa`，并清理掉 `Pb`。

3.  **`void MultiplyPolyn(polynomial &Pa, polynomial &Pb)`**
    *   **功能**：多项式**乘法**。
    *   **注释解析**：`//Multiplication, Pa=Pa * Pb, and then destroy Pb`。
    *   **算法难度**：乘法比加法复杂得多（因为每一项都要和另一边的每一项相乘，然后还要合并同类项）。在这里，它被封装成了一个黑盒，你只需要调用它，不用管内部怎么实现。

### 第三部分：比较工具函数（图2）
4.  **`int cmp(term a, term b)`**
    *   **功能**：比较两个单项式 `term` 的大小。
    *   **注释解析**：`//compare two polynomial, return -1, 0, 1`。
    *   **为什么需要这个工具函数？**
        *   在上一组 PPT 末尾我提到了：如果用“稀疏结构体数组”表示多项式，为了高效完成加法，**必须要求数组按照指数（`expn`）从小到大有序排列**。
        *   那么，在 `CreatePolyn`（创建）和 `AddPolyn`（合并）的过程中，我们需要一个比较规则来决定谁排前面。
        *   这个 `cmp` 函数通常会在内部比较 `a.expn` 和 `b.expn`。如果 `a` 的指数小返回 `-1`，相等返回 `0`，大则返回 `1`。它就像是一个“齿轮”，驱动着归并排序算法的运转。

### 💡 深度总结与编程实践建议

1.  **面向对象的思想**：虽然这是纯 C 语言（或 C++ 语法），但这组 PPT 完美体现了**面向对象（OOP）**的封装思想。我们把数据和操作数据的函数打包在一起。外面的程序员（调用者）不需要知道多项式底层是用什么数组、怎么开辟内存的，只需要调用 `AddPolyn(P1, P2)` 就能完成运算。
2.  **关于“销毁 Pb”的争议提醒**：
    *   教材里（如严蔚敏老师的经典教材）常常采用这种**破坏性加法**（计算完把加数销毁）。
    *   在**实际商业开发**中，**极不推荐**这种设计。通常我们会定义一个函数 `Polynomial AddPolyn(Polynomial Pa, Polynomial Pb)`，它返回一个**全新的**多项式结果，而保留 `Pa` 和 `Pb` 不变（这叫做“函数式编程”思想，没有副作用）。因为在实际程序中，`Pa` 和 `Pb` 可能还要被拿去参与别的运算。
3.  **后续课程预告**：
    如果你以后学 C++，这些操作会被重载成运算符（比如直接写 `Polynomial P3 = P1 + P2;`）。而在数据结构课里，你极有可能面临一个**手写作业**：**请利用前面讲过的“归并算法”，用 C 语言实现 `AddPolyn` 这个函数。** 你需要拿出 `pa` 和 `pb` 的指针，比较它们的指数，相等就系数相加，不等就谁小谁先放入结果数组。
