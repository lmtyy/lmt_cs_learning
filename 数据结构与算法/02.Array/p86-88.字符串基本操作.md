这三页 PPT 是数据结构课程中关于**“字符串（String）抽象数据类型（ADT）”的标准操作接口定义**。

如果你把字符串看作一个“黑盒”，这三页 PPT 就是**这个黑盒的使用说明书**。它定义了我们在代码中可以对字符串做哪些操作，而不用管底层是用数组还是链表实现的。

为了方便你记忆，我将这 8 个操作按照**“基础查询”、“增删改拼接”和“高级模式匹配”**三个维度来分类讲解。

---

### 第一部分：基础查询与判断（图1）

这些是最基础的属性读取操作。

**1. `StrEmpty(S)` （判空）**
*   **功能**：判断字符串是否为空串。
*   **前置条件**：字符串 S 必须存在（已经初始化）。
*   **返回结果**：如果是空串（长度=0，即 `""`），返回 `TRUE`；否则返回 `FALSE`。

**2. `StrCompare(S, T)` （比较大小）**
*   **功能**：比较两个字符串的大小（通常用于字符串排序）。
*   **比较规则（核心）**：
    *   逐个字符比较它们的 ASCII 码。
    *   如果在某个位置字符不同，ASCII 码大的字符串判定为“大”。
    *   如果一个字符串是另一个的前缀（如 `"abc"` 和 `"abcd"`），则**短的字符串判定为“小”**。
*   **返回结果**：
    *   \(>0\)：表示 S 大于 T。
    *   \(= 0\)：表示 S 等于 T（内容完全相同）。
    *   \(< 0\)：表示 S 小于 T。

**3. `StrLength(S)` （求长度）**
*   **功能**：返回字符串 S 中包含的**字符个数**。
*   *注意区分*：在 C 语言原生字符串中，长度不包括末尾的 `\0`；而在 ADT 的定义中，长度就是实实在在的用户可见的字符数。

---

### 第二部分：增删改拼接（图2）

这些操作会**改变字符串的内容或结构**，所以参数通常带有 **`&`**（在 C/C++ 中表示引用，在纯 C 中通常意味着传入指向结构体的指针，因为要修改原数据）。

**1. `ClearString(&S)` （清空）**
*   **功能**：清空字符串 S，把它变成一个空串。
*   **操作结果**：原字符串的数据会被清除，或者长度设置为 0。底层内存是否释放取决于具体实现。

**2. `Concat(&T, S1, S2)` （拼接）**
*   **功能**：将字符串 S1 和 S2 拼接在一起，存入新字符串 T 中。
*   *举例*：S1=`"Hello "`，S2=`"World"`，执行后 T 变成了 `"Hello World"`。
*   *C语言对照*：相当于标准库中的 `strcat`，或者 `sprintf` 的组合。

**3. `SubString(&Sub, S, pos, len)` （求子串 - 重点）**
*   **功能**：从字符串 S 中，截取一段字符生成一个新串 `Sub`。
*   **前置条件（非常重要，考试常考判断）**：
    1.  起始位置 `pos` 必须合法：\(0 < pos \le \text{StrLength}(S)\)。（注意这里的 \(pos\) 是从 **1** 开始计数的物理/逻辑位置，而不是 C 语言的 0 下标）。
    2.  截取长度 `len` 必须合法：\(0 \le len \le \text{StrLength}(S) - pos + 1\)。（不能越界）。
*   **结果**：从 S 的第 `pos` 个字符开始，往后取 `len` 个字符，存入 `Sub`。

### 第三部分：高级模式匹配（图3）

这两个操作是字符串特有的高级操作，也是后续讲解**KMP算法**的前置基础。

**1. `Index(S, T, pos)` （定位/模式匹配）**
*   **功能**：**寻找子串**。在字符串 S 中，从第 `pos` 个位置开始，查找是否存在子串 T。
*   **结果**：
    *   如果找到了，返回 **T 在 S 中第一次出现的位置**（注意，返回的位置也是从 **1** 开始计数的）。
    *   如果找不到，返回 **0**。（这是字符串 ADT 的规范，因为从 1 开始计数，0 被用来表示“无此位置”）。
*   *未来学习方向*：标准库的 `strstr()` 函数就是做这个的。但在数据结构中，我们不仅要找，还要关注怎么找最快。这引出了之后**“朴素的模式匹配算法（BF算法）”**和**“KMP算法”**的诞生。

**2. `Replace(&S, T, V)` （替换）**
*   **功能**：在字符串 S 中，**将所有**等于子串 T 的部分，替换成字符串 V。
*   **前置条件**：T 必须是非空串（不能替换一个“空”）。
*   *举例*：`S = "abaab"，T = "ab"，V = "X"`。执行替换后，S 会变成 `"XaX"`。
*   *实现逻辑*：这个操作通常是基于 `Index` 和 `Concat`/`SubString` 组合实现的。先找到 T，截取 T 前后的内容，用 V 拼接起来。直到 S 中没有 T 为止。

### 💡 深度总结与学习建议

1.  **区分“位置”和“下标”**：这三页 PPT 中，`SubString` 和 `Index` 的 `pos` 参数，以及返回的结果，**都是基于 1 开始计数的逻辑位置**，**而不是 C 语言中基于 0 的数组下标**。考试做题时一定要看清题目要求，这是一个高频陷阱。
2.  **数学上的完美主义**：标准的 ADT 定义中，为了逻辑的严密性，通常将结果位置定义为 1 到 Length（找不到返回 0），这和真实编程语言（C/Java 返回 -1，或者返回 NULL 指针）在实现上会略有不同。
3.  **接下来的重头戏**：这三页中最重要的是 **`Index(S, T, pos)`**。接下来的课程，你将会花费大量精力学习 **“如果 S 非常长，T 也非常长，怎么让 `Index` 的速度成倍提升”**。这就是大名鼎鼎的 **KMP 算法**的起点。请务必记住这个函数的定义和返回值的含义，后续所有的推导都将建立在“查找子串位置”这个需求之上。
