这是一个非常经典且触及 C/C++ 语言核心机制的问题。

简单直接的结论是：**数组名本质上是一个“地址常量”（Constant Address），而指针是一个“地址变量”。**

你可以把数组名 `arr` 理解为 **`int \* const arr`**（一个指向固定的内存位置的指针），你不能改变这个指针指向哪里，但你可以利用它去计算别的地址。

以下是详细的层层拆解：

------

### 1. 为什么 `arr++` 也就是自增是不合法的？

当我们写 `a++` 时，它的实际含义是 `a = a + 1`。这包含了两个步骤：

1. 计算 `a + 1` 的值。
2. **将结果重新赋值给 `a`。**

关键就在这第 2 步。在 C 语言中，要能被赋值，变量必须是一个 **“可修改的左值” (Modifiable L-value)**。

- **数组名的本质：** 数组名 `arr` 代表了该数组在内存中首元素的固定地址。一旦程序编译链接完成，这个数组在内存中的位置就固定了。
- **无法赋值：** 你不能告诉编译器：“把这块数组的起始位置往后挪一格”。如果允许 `arr++`，那么原本 `arr[0]` 的地址就丢失了，程序将无法再正确访问整个数组原本的内存块。

**比喻：**

数组名就像你家的**门牌号**（比如“南京路1号”）。

- `arr++` 相当于你想把“南京路1号”这个门牌变成“南京路2号”。这是不可能的，因为房子本身没有移动，门牌是固定钉在墙上的。

### 2. 为什么 `arr + 1` 是合法的？

`arr + 1` 并没有试图改变 `arr` 本身。

- **计算行为：** 它只是执行了一次**指针运算**。它读取 `arr` 的地址值，加上一个单位的偏移量（`sizeof(int)`），然后产生一个新的、临时的地址值（右值）。
- **不涉及赋值：** 这个操作并没有把结果写回给 `arr`，所以不违反“常量”的规则。

**比喻：**

- `arr + 1` 相当于你说：“去南京路1号的**隔壁**看看”。
- 你只是计算出了隔壁的地址，并没有试图把自家的门牌号改成隔壁的。

------

### 3. 代码对比与内存视角

为了更清晰地理解，我们可以对比“数组”和“指针变量”：

C

```
int arr[5] = {1, 2, 3, 4, 5};
int *p = arr; // p 是一个指针变量，初始化指向 arr 的首地址

// 场景 1: 自增
p++;   // ✅ 合法。p 是变量，是一个“小纸条”，上面写的地址可以擦掉重写。
arr++; // ❌ 错误。arr 是“门牌”，是固定死的，不能改写。

// 场景 2: 加常数
*(p + 1) = 10;   // ✅ 合法。计算 p 下一个位置的地址。
*(arr + 1) = 10; // ✅ 合法。计算 arr 下一个位置的地址。
```

### 4. 底层原理：符号表 (Symbol Table)

从编译器的角度来看：

- **指针变量 `p`：** 编译器会为 `p` 分配一个独立的内存空间（比如 4 字节或 8 字节）来存放地址。`p` 既有自己的地址，也有存储的值。
- **数组名 `arr`：** 编译器在符号表中将 `arr` 这个名字直接映射到数组首元素的地址。**`arr` 本身在内存中并没有像 `p`那样有一个专门存储“地址值”的变量空间。** `arr` 就是那个地址本身。既然没有地方存这个“指针变量”，自然也就无法对它进行“修改并保存”的操作。

### 总结

| **操作**   | **表达式** | **是否合法** | **原因**                                             |
| ---------- | ---------- | ------------ | ---------------------------------------------------- |
| **自增**   | `arr++`    | **❌ 非法**   | `arr` 是不可修改的左值 (常量指针)，不能被重新赋值。  |
| **加常数** | `arr + 1`  | **✅ 合法**   | 仅生成一个新的临时地址值 (右值)，未修改 `arr` 本身。 |
