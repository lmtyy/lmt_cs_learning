这页 PPT 的核心内容是展示**如何通过指针算术（Pointer Arithmetic）而不是数组下标（Subscripts）来遍历和打印数组**。

它揭示了 C 语言中“数组即指针”的底层操作逻辑。以下是详细的代码解析和知识点拓展：

### 1. 代码功能解析

这段代码定义了一个函数 `print1`，其功能是打印出一维数组中每个元素的**内存地址**和对应的**数值**。

#### **函数签名**

C

```c
void print1(int *ptr, int rows)
```

- **`int \*ptr`**：
  - 这里接收的是数组的首地址。当你在 `main` 函数中调用 `print1(list, 5)` 时，数组名 `list` 退化为首地址传给 `ptr`。
  - **知识点**：这是“传址调用”。函数内部没有复制数组，而是直接操作原数组的内存。
- **`int rows`**：
  - 必须显式传递数组长度，因为指针 `ptr` 只是一个地址，它不知道后面连着多少个有效数据。

#### **核心打印语句**

C

```c
printf("%8u%5d\n", ptr + i, *(ptr+i));
```

这一行是整页 PPT 的精华，包含了两个关键操作：

1. **地址计算：`ptr + i`**
   - 这是打印数组元素的**物理内存地址**。
   - **重点知识（指针步长）**：
     - 这里的 `+ i` 不是数学上的加 $i$，而是加 $i \times \text{sizeof(int)}$。
     - 如果 `ptr` 的地址是 `1000`，且 `int` 占 4 字节：
       - `i=0` -> 地址 `1000`
       - `i=1` -> 地址 `1004` (自动跳过 4 字节)
       - `i=2` -> 地址 `1008`
2. **取值操作：`\*(ptr + i)`**
   - 这是打印该地址里存放的**数值**。
   - **重点知识（解引用）**：
     - `*` 是解引用操作符（Dereference Operator）。
     - `*(ptr + i)` 的含义是：“去 `ptr + i` 这个地址看看，把里面的东西取出来”。
     - **等价性**：这完全等同于写法 `ptr[i]`。`ptr[i]` 只是 `*(ptr + i)` 的语法糖，编译器底层执行的正是 PPT 中的这种指针运算。
3. **格式控制符**
   - `%8u`：以**无符号十进制整数**（Unsigned Decimal）形式打印地址。`8` 表示输出宽度至少占 8 位（用于对齐）。
     - *注：在现代标准 C 中，打印地址通常推荐用 `%p`（以十六进制显示），但教学中为了直观展示地址的线性增加，常用 `%u`。*
   - `%5d`：以有符号十进制整数形式打印数值。

### 2. 回答 PPT 底部的问题

**Question:** *"What Address contents would be?"* (地址内容会是什么？)

**Answer:**

输出的地址会呈现**等差数列**的规律，公差为 `sizeof(int)`（通常是 4）。

**模拟输出示例：**

假设数组起始地址为 `2000`，存储的数据是 `{10, 20, 30}`。程序的输出将会是：

| **循环变量 i** | **ptr + i (地址)** | ***(ptr + i) (值)** | **备注**      |
| -------------- | ------------------ | ------------------- | ------------- |
| 0              | `2000`             | `10`                | 基地址        |
| 1              | `2004`             | `20`                | 增加 4 字节   |
| 2              | `2008`             | `30`                | 再增加 4 字节 |

### 3. 知识点总结与拓展

1. **访问数组的两种方式：**
   - **下标法（Subscripting）：** `array[i]` —— 写法简单，适合初学者。
   - **指针法（Pointer Arithmetic）：** `*(ptr + i)` —— 执行效率在老式编译器上可能更高（现代编译器已优化，两者差异不大），但它是理解数据结构（如链表、树）的基础。
2. **指针类型的意义：**
   - 为什么定义指针要写 `int *` 而不是 `void *`？
   - 因为只有知道类型是 `int`，计算机才知道 `ptr + 1` 应该跳过 **4** 个字节。如果是 `double *`，它就会跳过 **8** 个字节。
3. **野指针风险：**
   - 如果不小心访问了 `*(ptr + rows)`（即越界访问），C 语言不会报错，但你会读取到内存中的垃圾数据，甚至导致程序崩溃（Segmentation Fault）。
