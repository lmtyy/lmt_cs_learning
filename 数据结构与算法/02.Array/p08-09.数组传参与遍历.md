### 第一页 PPT：数组基础与求和函数

**核心知识点：**

1.  **宏定义与数组声明 (`#define MAX_SIZE 100`)**
    *   使用 `#define` 定义常量，而不是直接在代码里写“100”。这是一种良好的编程习惯（**Magic Number 避免**）。如果以后想把数组改大，只需改这一处即可。
    *   `int input[MAX_SIZE]` 声明了一个包含 100 个整型元素的数组。

2.  **数组元素的初始化 (赋值)**
    *   代码：`for(i=0; i<MAX_SIZE; i++) input[i] = i;`
    *   这里使用了 `for` 循环遍历数组。**注意数组下标是从 0 开始的**，到 `MAX_SIZE - 1` 结束（这里是 0 到 99）。

3.  **数组作为函数参数传递 (`int sum(int list[], int n)`)**
    *   **声明方式的陷阱**：在 C 语言中，当你把数组名作为参数传递给函数时，**数组会退化为指针**。`int list[]` 在形参里其实等价于 `int *list`。函数接收到的并不是整个数组的拷贝（那样效率太低），而是数组首元素的**内存地址**。
    *   **为什么需要 `int n` 参数？**因为函数内部只知道收到了一个地址，**不知道数组有多长**。所以必须额外传递一个 `n` 表示元素个数，以防越界访问。

4.  **累加算法**
    *   初始化一个累加器 `itemsum = 0`。
    *   循环读取 `list[i]`，并使用 `+=` 操作符进行累加。

5.  **程序执行逻辑**
    *   `main` 函数中：给数组填满值 -> 调用 `sum` 函数 -> 将返回值赋给 `answer` -> 使用 `printf` 打印结果。这里打印的结果是 \(0+1+2+...+99 = 4950\)。

### 第二页 PPT：指针访问数组与地址计算

**核心知识点：**

这一页展示了 **“数组名即是指针”** 这一 C 语言的核心特性，并演示了如何通过指针算术运算（Pointer Arithmetic）来遍历数组。

1.  **函数声明 `void print1(int *ptr, int rows)`**
    *   这里直接使用了指针 `int *ptr` 来接收数组。这明确告诉程序员，我们处理的是一个指向整型的内存地址。

2.  **指针算术运算 (`ptr + i`)**
    *   **重点概念**：当 `ptr` 是一个 `int *` 类型的指针时，`ptr + 1` 并不只是地址值加 1。
    *   在 C 语言中，`ptr + i` 的**实际内存地址** = `ptr` 的基地址 + `i * sizeof(int)`。
    *   如果你的电脑上 `int` 占 4 个字节，那么 `ptr + 1` 就会跳过 4 个字节，直接指向下一个整数。这使得我们可以方便地遍历数组。

3.  **通过指针访问数据 (`*(ptr + i)`)**
    *   **解引用（Dereferencing）**：`*` 号获取地址里的值。
    *   `*(ptr + i)` 就等价于我们熟悉的数组写法 `ptr[i]`。在 C 语言底层，`array[i]` 实际上是 `*(array + i)` 的语法糖。

4.  **`printf` 格式化输出 (`%8u%5d`)**
    *   `%8u`：打印无符号整数（内存地址通常以无符号十六进制或十进制表示），占用 8 个字符宽度的右对齐。
    *   `%5d`：打印数组元素的值，占用 5 个字符宽度的右对齐。
    *   这样打印出来的结果会非常整齐，便于观察规律。

5.  **底部的思考题：`What Address contents would be ?`**
    *   如果去执行这个代码，输出的每一行会类似：
        `16064192    0`
        `16064196    1`
        `16064200    2`
        ...
    *   **规律观察**：
        *   第一列（地址）每次都会增加 **4**（在你的机器上，通常意味着 `sizeof(int) == 4`）。
        *   第二列（值）每次递增 **1**。
    *   这直观地展示了**数组在内存中是连续存放的**，并且印证了指针算术中的“步长”概念。

### 💡 总结与易错点提醒

1.  **数组传参必带长度**：图1里 `sum` 函数必须带 `n` 参数。如果函数内部像图2一样去算 `size`（如 `sizeof(ptr)/sizeof(int)`）是**绝对错误**的，因为 `ptr` 只是个指针，那个算式永远得不到数组的真实长度。
2.  **下标与指针的互换**：要熟练掌握 `array[i]` 等价于 `*(array + i)`。这在处理多维度数组或者进行底层内存操作时非常重要。
3.  **地址增长的单位**：记住 `ptr + i` 跳过的字节数取决于指针的类型（`int*` 跳 4 字节，`char*` 跳 1 字节）。

如果你实际运行一下第二页的代码，就会清晰看到那些地址以固定步长递增的过程，那是理解内存模型最直观的方式。
