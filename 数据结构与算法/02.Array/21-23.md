这三页 PPT 展示了如何使用 **C 语言结构体**实现一个功能完备的**动态顺序表（Sequence List）**。它不仅定义了数据结构，还给出了初始化逻辑及其内存模型。

------

### 第一页：类型定义 (Type Definitions)

这一页定义了顺序表的“骨架”，为动态扩容做好了准备。

- **预处理宏定义**：
  - `#define LIST_INIT_SIZE 100`：定义了顺序表的**初始最大容量**为 100。
  - `#define LISTINCREMENT 10`：定义了当空间不足时，每次**增加的存储量**。这体现了动态顺序表的灵活性。
- **结构体设计 (`SqList`)**：
  - `ElemType *elem`：存储空间的**基地址指针**。它指向动态分配的数组首元素。
  - `int length`：表示当前顺序表中**实际包含的元素个数**。
  - `int listsize`：表示当前已分配的**总存储容量**（以 `ElemType` 为单位）。
- **知识点丰富**：
  - **解耦设计**：通过 `typedef struct ElemType`，你可以根据需要随时更改表中存储的数据类型（如改为 `int`, `float` 或自定义结构体），而无需修改顺序表的逻辑代码。

------

### 第二页：初始化函数 (`InitList`)

这一页展示了如何在堆（Heap）上动态开辟空间。

- **内存分配核心**：
  - 使用 `malloc` 函数按照 `LIST_INIT_SIZE` 申请初始空间。
  - `L.elem = (ElemType *) malloc(...)`：将申请到的 `void *` 转换为具体的元素指针。
- **健壮性检查**：
  - `if(!L.elem) Exit(OVERFLOW);`：如果内存申请失败（如系统内存耗尽），程序必须报错并退出，防止操作空指针。
- **状态初始化**：
  - `L.Length = 0;`：初始时表中没有数据，逻辑长度设为 0。
  - `L.Listsize = LIST_INIT_SIZE;`：记录当前最大的物理承载量为 100。

------

### 第三页：内存模型示意图 (Visual Model)

这一页将抽象的代码映射到了直观的物理内存中。

- **逻辑位序 vs. 物理下标**：
  - **逻辑位序**：$a_1, a_2, \dots, a_n$（从 1 开始计数的元素）。
  - **数组下标**：$0, 1, \dots, n-1$（从 0 开始计数的索引）。
  - 图中明确显示 $a_1$ 存放在下标 0 的位置，而 $a_i$ 存放在下标 $i-1$。
- **变量的含义**：
  - **`L.elem`**：图中左侧的红色箭头，指向数组的起始位置（下标 0）。
  - **`L.length`**：记录当前存到了哪个位置（图中为 $n$）。
  - **`L.listsize`**：表示整个蓝色方块的总长度（图中为 100）。
- **动态数组（Dynamic Array）**：
  - 图中下标 0 到 99 的连续空间即为“动态数组”，它的位置和大小在程序运行时确定。

------

### 核心知识点总结表

| **概念**     | **说明**                             | **对应 PPT 变量/宏** |
| ------------ | ------------------------------------ | -------------------- |
| **基地址**   | 顺序表在内存中开始存放的位置         | `L.elem`             |
| **逻辑长度** | 表中当前实际有多少个有效数据         | `L.length`           |
| **物理容量** | 当前这块内存最多能存多少个数据       | `L.listsize`         |
| **扩容步长** | 当 `length == listsize` 时增加的空间 | `LISTINCREMENT`      |
