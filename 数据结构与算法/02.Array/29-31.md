这三页 PPT 详细展示了**顺序表删除操作（Deletion）**的逻辑定义、核心步骤以及 C 语言代码实现。删除操作是维护顺序表连续存储特性的关键，其核心在于数据的“前移”。

------

### 1. 删除操作的逻辑定义

这页内容明确了删除操作的数学模型和预期结果：

- **基本操作**：删除顺序表中的第 $i$ 个元素，并通过变量 $e$ 返回被删除的值。
- **操作前的状态**：线性表为 $(a_1, a_2, \dots, a_{i-1}, a_i, a_{i+1}, \dots, a_n)$。
- **操作后的状态**：表长度减少至 $n-1$。原本在 $a_i$ 之后的元素依次前移，新的序列变为 $(a_1, a_2, \dots, a_{i-1}, a_{i+1}, \dots, a_n)$。

------

### 2. 算法处理的关键点

这页总结了实现删除算法时必须遵循的逻辑：

1. **合法性检查**：首先判断删除位置 $i$ 是否合法，以及列表 $L$ 是否为空。若不合法或表空，则返回错误代码 0。
2. **元素前移**：将第 $i$ 个元素之后的所有元素依次向**前**移动一个位置。
3. **状态更新**：完成移动后，将顺序表 $L$ 的长度减 1。

------

### 3. C 语言代码实现解析

这页展示了利用指针操作实现删除的底层代码：

- **函数签名**：`Status Delete(SqList &L, int i, ElemType &e)`。
- **边界判定**：`if (i <= 0 || i > L.Length) return ERROR;`。由于下标从 0 开始，第 $i$ 个元素的物理索引是 $i-1$。
- **定位与备份**：
  - `p = &(L.elem[i-1]);`：通过指针 `p` 指向待删除元素的内存地址。
  - `e = *p;`：将被删元素的值存入 `e`。
- **前移循环**：
  - `q = &(L.elem[L.Length-1]);`：定位顺序表最后一个元素的地址。
  - `for (++p; p <= q; ++p) *(p-1) = *p;`：从被删元素的后一个位置开始，依次将元素值覆盖到前一个位置。
- **收尾工作**：`L.length--;` 减少逻辑长度，并返回 `OK`。

------

### 💡 丰富知识点拓展

- **数据移动方向**：
  - **插入操作**：必须**从后往前**移，防止数据被覆盖。
  - **删除操作**：必须**从前往后**移（即从被删位置的后一个元素开始依次前移），将后方元素覆盖到前方。
- **时间复杂度分析**：
  - **最好情况**：删除表尾元素（$i=n$），不需要移动任何元素，$T(n) = O(1)$。
  - **最坏情况**：删除表头元素（$i=1$），需要移动后续所有 $n-1$ 个元素，$T(n) = O(n)$。
  - **平均情况**：平均需要移动 $\frac{n-1}{2}$ 个元素，因此平均时间复杂度为 **$O(n)$**。
- **空间复杂度**：
  - 操作在原数组上进行，仅需常数级的辅助空间（如指针 `p`, `q`），因此空间复杂度为 **$O(1)$**。
- **返回被删值的意义**：
  - 在很多应用场景中（如任务队列或缓冲区），删除操作往往伴随着对数据的处理。通过引用参数 `&e` 返回被删值，可以方便调用者继续使用该数据，而函数本身的返回值（Status）则专用于表示操作是否成功。
