在C语言中，**联合体 (Union)**，有时也被称为“共用体”，是一种特殊的数据类型。

简单来说，联合体允许你在**同一块内存空间**中存储不同类型的数据。这就像是一个“多功能盒子”，你在同一时间只能在这个盒子里放一样东西，但这个盒子的大小足以放下你规定范围内最大的那样东西。

以下是关于联合体的详细解析：

------

### 1. 核心概念：内存共享

联合体与结构体（Struct）最大的区别在于内存的分配方式：

- **结构体 (Struct)**：每个成员都有自己独立的内存空间。总大小 $\approx$ 所有成员大小之和。
- **联合体 (Union)**：所有成员**共用**同一块内存地址。总大小 $\ge$ 最大成员的大小。

**这意味着：**

1. 联合体的所有成员的**起始地址是相同**的。
2. 在任何特定时刻，你**只能有效使用其中一个成员**。
3. 当你给一个成员赋值时，**会覆盖掉**其他成员的值（因为它们用的是同一块地盘）。

### 2. 语法定义与使用

语法和结构体非常相似，只是关键字变成了 `union`。

C

```c
#include <stdio.h>
#include <string.h>

// 定义一个联合体
union Data {
    int i;
    float f;
    char str[20];
};

int main() {
    union Data data;

    // 1. 联合体的大小
    // 大小等于最大成员的大小 (这里是str[20]，即20字节)
    // 注意：实际大小可能会因为内存对齐而略大（例如变为24字节）
    printf("Memory size occupied by data : %lu\n", sizeof(data));

    // 2. 赋值与覆盖
    data.i = 10;
    printf("data.i : %d\n", data.i);

    data.f = 220.5; // 这里写入float，会覆盖之前的int数据
    printf("data.f : %f\n", data.f);
    
    // 此时如果我们再尝试读取 i，得到的就是被 float 覆盖后的乱码（或者说是float的二进制表示被当做int读）
    printf("data.i (corrupted) : %d\n", data.i); 

    strcpy(data.str, "C Programming"); // 写入字符串，覆盖之前的 float
    printf("data.str : %s\n", data.str);

    return 0;
}
```

### 3. 为什么要使用联合体？（应用场景）

你可能会问：“既然会互相覆盖，为什么不直接用结构体？” 联合体主要用于以下场景：

#### A. 节省内存空间

在资源受限的环境（如嵌入式系统）中，如果你有一组数据，但在程序的某个时刻**只需要用到其中一种**，使用联合体可以大大节省内存。

**例子：** 假设你要存储一个“输入事件”，可能是鼠标点击（坐标 x, y），也可能是键盘按键（keycode）。

C

```c
struct Mouse { int x; int y; };
struct Keyboard { int keycode; };

struct Event {
    int type; // 0 for mouse, 1 for keyboard
    union {
        struct Mouse mouse;       // 8 bytes
        struct Keyboard keyboard; // 4 bytes
    } input; // 这个 union 只占 8 bytes，而不是 12 bytes
};
```

#### B. 硬件寄存器操作 (位域 + 联合体)

这是 C 语言底层开发中最经典的用法。通过联合体，我们可以用一种方式（如整个整数）写入数据，用另一种方式（如位域）读取具体的标志位。

C

```c
typedef union {
    unsigned int value; // 可以一次性操作整个32位寄存器
    struct {
        unsigned int flagA : 1; // 第0位
        unsigned int flagB : 1; // 第1位
        unsigned int mode  : 2; // 第2-3位
        unsigned int reserved : 28;
    } bits;
} Register;

Register reg;
reg.value = 0;   // 清空寄存器
reg.bits.flagA = 1; // 只设置某一位
```

#### C. 判断大小端模式 (Endianness)

不同计算机存储多字节数据（如 int）的顺序不同。联合体可以用来检测当前机器是**大端**还是**小端**。

C

```c
int check_endian() {
    union {
        int i;
        char c;
    } u;
    
    u.i = 1; // 十六进制是 0x00000001
    
    // 如果是小端，1 存在低地址，c 读取低地址，结果为 1
    // 如果是大端，1 存在高地址，c 读取低地址，结果为 0
    return (u.c == 1); 
}
```

### 4. 联合体 vs 结构体：总结对比

| **特性**     | **结构体 (Struct)**                           | **联合体 (Union)**                |
| ------------ | --------------------------------------------- | --------------------------------- |
| **内存分配** | 每个成员有独立空间                            | 所有成员共用一块空间              |
| **总大小**   | $\ge$ 所有成员大小之和                        | $\ge$ 最大成员的大小              |
| **同时访问** | 可以同时存储和访问所有成员                    | 同一时间只能存储/有效访问一个成员 |
| **地址**     | 成员地址是连续递增的                          | 所有成员的首地址是相同的          |
| **主要用途** | 组合不同属性的数据（如 Student 有 name, age） | 多种格式解释同一段内存 / 节省空间 |

------

### 总结

联合体是 C 语言赋予程序员直接操作内存布局的强大工具。它就像一个“变形金刚”，外表（内存地址）不变，但内在形态（数据类型解释）可以随意切换。

**记住最关键的一点：** 写入联合体的一个成员，会改变其他所有成员的值，因为它们本质上是在看同一个内存地址里的比特流。
