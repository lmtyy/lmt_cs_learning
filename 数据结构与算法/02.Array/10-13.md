这四页 PPT 系统地讲解了 C 语言中用于构建复杂数据类型的核心机制：**结构体 (Structure)** 和 **联合体 (Union)**，以及**自引用结构 (Self-Referential Structure)**。这是 C 语言实现面向对象思想（如类）和高级数据结构（如链表、树）的基石。

以下是针对这四页内容的深度解析及核心知识点拓展：

------

### 第一页：结构体的定义 (Defining Structures)

这页展示了如何定义一个包含不同数据类型的“包裹”。

#### 1. 核心概念

- **异构容器：** 数组 (Array) 只能存**同一种**类型的数据，而结构体 (Structure) 可以将**不同**类型的数据（字符、整数、浮点数）组合在一起。

- **代码解读：**

  C

  ```c
  struct {
      char name[20];
      int age;
      float salary;
  } person;
  ```

  这里定义了一个匿名结构体类型，并直接声明了一个全局变量 `person`。

#### 2. 知识点拓展（内存对齐 - Padding）

这是 PPT 没写但极其重要的面试考点：**结构体的大小不是简单的成员大小相加。**

- **计算：** `name`(20) + `age`(4) + `salary`(4) = 28 字节？
  - 在这个特定例子中可能是 28，因为 20 正好是 4 的倍数。
- **对齐规则：** 为了 CPU 读取效率，编译器会在成员之间插入“填充字节” (Padding)。
  - 假设结构体是 `struct { char c; int i; };`
  - 大小不是 1+4=5，而是 **8** 字节（`char` 后面填充 3 个字节，让 `int` 从 4 的倍数地址开始）。

------

### 第二页：结构体成员的访问 (Accessing Structure)

这页讲解了如何操作结构体内部的数据。

#### 1. 成员运算符 (.)

- **语法：** `结构体变量名.成员名`
- **代码重点：** `person.age = 20;` 是标准的赋值。

#### 2. 知识点拓展（字符串赋值的陷阱）

PPT 中特意使用了 `strcpy(person.name, "james");` 而不是 `person.name = "james";`。

- **为什么不能直接赋值？**
  - 在 C 语言中，数组名 `name` 是一个**常量指针**（指向数组首地址）。你不能改变数组“指向”哪里，只能改变数组“里面”的内容。
  - `person.name = "james"` 会导致编译错误（试图给常量赋值）。
- **解决方案：** 必须使用 `strcpy` (String Copy) 函数，将字符串的内容逐个字节复制到 `name` 数组的内存空间中。

#### 3. 关联知识（指针访问 ->）

如果 `person` 是一个指针而不是变量（例如 `struct person *p`），访问方式变为**箭头运算符**：

- `p->age = 20;` 等价于 `(*p).age = 20;`

### 第三页：结构体与联合体的嵌套 (Structures and Unions)

这页展示了更高级的用法：`typedef`、`enum` 和 `union` 的混合使用。这是操作系统和底层协议栈中常见的设计模式。

#### 1. 联合体 (Union) 的本质

- **定义：** 代码中的 `union { float grade_n; char grade_s; } u;`
- **核心特征（互斥共享）：**
  - `grade_n` 和 `grade_s` **共用同一块内存地址**。
  - 同一时刻，你只能使用其中**一个**。
  - **修改其中一个，会覆盖另一个的值。**
- **大小计算：** Union 的大小等于**最大成员**的大小（这里是 `float`，4 字节）。

#### 2. 标记联合 (Tagged Union)

- 代码中引入了 `enum tag_course {exam, other} coursetype;`。
- **设计模式：** 这是一个标准的 **Tagged Union** 设计。
  - 由于 Union 里的数据是混在一起的，计算机不知道当前存的是 `float` 还是 `char`。
  - 程序员利用 `coursetype` 这个“标签”来记录当前 Union 里有效的数据类型：
    - 如果 `coursetype == exam`，程序就去读 `u.grade_n`。
    - 如果 `coursetype == other`，程序就去读 `u.grade_s`。

#### 3. `typedef` 的作用

- `typedef struct coursegrade { ... }` 使得后面可以直接使用 `struct coursegrade` 作为类型名，甚至可以进一步简化为只写 `CourseGrade`（取决于 typedef 的具体写法，PPT 里的写法实际上还需要加 struct 关键字，除非写成 `typedef struct { ... } name;`）。

### 第四页：自引用结构 (Self-Referential Structures)

这页是数据结构（链表、树、图）的起跑线。

#### 1. 链表节点定义

C

```
typedef struct list {
    char data;
    struct list *next; // 重点在这里
};
```

- **自引用：** 结构体内部包含了一个指向**同类型结构体**的指针。这是构建**链表 (Linked List)** 的基础。

#### 2. 为什么必须是指针？

- 如果在结构体里写 `struct list next;` (没有 `*`)，编译器会报错。
- **原因：** 结构体包含自己，自己又包含自己……会导致**无限递归**，编译器无法计算该结构体需要多少内存（Size infinite）。
- **指针的解法：** 指针的大小是固定的（32位系统是4字节，64位是8字节），编译器可以确定大小，因此合法。

#### 3. 灵魂拷问：`sizeof(struct list) = ?`

PPT 里的这个气泡问题非常有水平。

- **表面计算：** `char` (1 byte) + `pointer` (4 bytes, 假设32位系统) = 5 bytes?
- **实际答案（内存对齐）：**
  - 为了让指针 `next` 存放在 4 的倍数地址上（提高访问速度），编译器会在 `char data` 后面填充 3 个字节 (Padding)。
  - **结果：** 1 (char) + **3 (padding)** + 4 (pointer) = **8 bytes**。
  - *注：在 64 位系统中，指针是 8 字节，对齐通常也是 8 字节，所以可能是 1 + 7 + 8 = 16 bytes。*

#### 4. 代码逻辑

- `node1.next = NULL;`：链表尾部的标志。`NULL` 通常定义为 `(void *)0`。
- `node2.next = &node1;`：将 `node2` 链接到 `node1`，形成链式结构 `node2 -> node1 -> NULL`。

### 总结

这四页 PPT 循序渐进地构建了 C 语言的高级数据处理能力：

1. **Struct**：数据的打包（就像一个文件夹）。
2. **Union**：内存的复用（就像一个多功能插座，一次只能插一个）。
3. **Self-Reference**：动态数据结构的连接器（链表的锁链）。
4. **底层原理**：贯穿始终的是**内存管理**（对齐、指针大小、地址操作）。
