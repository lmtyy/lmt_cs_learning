这两页 PPT 讲的是数据结构中非常基础且重要的概念：**顺序表（Sequence List）**。

简单来说，顺序表就是**“用数组（内存中连续的空间）来实现的线性表”**。

这两页 PPT 分别从**“定义与物理实现”**和**“核心数学特性（随机存取）”**两个角度进行了讲解。我为你详细拆解如下：

### 第一页 PPT：什么是顺序表？（物理实现）

这一页介绍了顺序表的核心概念，并对比了 C 语言中两种在内存里开辟连续空间的方法。

**1. 核心定义**
*   **原文**：`Using consecutive set of memory locations (Array) to save the list...`
*   **解读**：顺序表的本质，就是**在计算机内存中开辟一段连续的存储空间**，把线性表的元素 \(a_1, a_2, ..., a_n\) 一个接一个地紧挨着存入进去。
*   **右侧图示**：蓝色的方块代表了连续的内存格子，`a1` 到 `an` 正好对应占据了这些格子。

**2. 两种开辟连续内存的方法（考研/面试常见考点）**

PPT 列举了在 C 语言中实现顺序表存储的两种截然不同的方式：

*   **方式一：静态分配 (Static allocated storage)**
    *   **代码**：`int a[100];`
    *   **特点**：这是在栈（Stack）或全局数据区上分配内存。大小（这里是100）必须在**编译时**确定。
    *   **缺点**：**定长，不灵活。** 如果你预估错了，存了 101 个元素就会导致数组越界崩溃；如果只存了 2 个，那浪费了 98 个元素的宝贵内存空间。

*   **方式二：动态分配 (Dynamically allocated storage)**
    *   **代码**：
        ```c
        int *pa;
        pa = (int *)malloc(100 * sizeof(int));
        ```
    *   **特点**：这是在**堆（Heap）**上申请内存。利用标准库函数 `malloc`（Memory Allocation）在程序**运行时**向操作系统申请空间。
    *   **优点**：**动态，灵活。** 如果一开始不知道要用多大的数组，就可以先申请小一点，后续如果满了，还可以配合 `realloc` 函数进行扩容。
    *   **核心注意**：使用 `malloc` 动态分配的内存**不会自动回收**，在 C 语言中必须手动使用 `free(pa);` 释放掉，否则会造成**内存泄漏**。

**3. 顺序映射 (Sequential mapping)**
*   图下方的橙色箭头指向了连续的内存结构，表达了一个核心物理特性：**逻辑上相邻的元素（如 \(a_i\) 和 \(a_{i+1}\)），在物理内存地址上也是相邻的。**

### 第二页 PPT：顺序表的核心性能优势（随机存取）

这一页通过数学公式揭示了顺序表为什么比链表存取更快。

**1. 地址计算公式（核心重点！）**
*   **公式**：\(Loc(a_i) = Loc(a_1) + (i - 1) \times L\)
*   **字母含义**：
    *   \(Loc(a_i)\)：第 \(i\) 个元素的**内存地址**。
    *   \(Loc(a_1)\)：**首地址**（也就是整个数组第一个元素 \(a_1\) 在内存中的地址）。
    *   \(i\)：要查找的元素**下标**（序号）。
    *   \(L\)：**每个元素占用的内存大小**（比如在 32 位系统中，`int` 占 4 个字节，那么 \(L=4\)）。
*   **左侧图示**：图中用了大括号 `Units of L` 标记了每个元素占用的字节数，非常直观。

**2. 什么叫“随机存取” (Random-access storage)？**
*   **推导过程**：由于第 \(i\) 个元素的地址可以通过**一步数学计算**（加法和乘法）直接得到，这意味着：
    *   **无论你要找第 1 个元素，还是第 10000 个元素，计算地址所花的时间是一样的**。
*   **结论**：这就是为什么顺序表被称为具有 **“随机存取”** 的特性。读取任意元素的时间复杂度是 **\(O(1)\)**（常数时间）。
*   **对比理解**：如果是链表，你要找第 10000 个元素，必须从第一个节点开始，依次顺着指针找 10000 次（时间复杂度 \(O(n)\)）。**在读取数据方面，顺序表完胜链表。**

### 💡 总结与复习要点

这两页 PPT 学完后，你应该能回答以下核心问题：

1.  **顺序表用什么底层物理结构？**
    *   **数组**（计算机内存中一段连续的空间）。
2.  **静态数组 vs 动态指针的区别是什么？**
    *   静态数组（`int a[100]`）在编译时确定大小，定长；动态指针（`int *pa = malloc(...)`）在运行时申请，可以扩容。**（注意动态分配的要手动 `free`）**。
3.  **顺序表最大的优势是什么？**
    *   **随机存取（Random Access）**。因为地址可以用公式直接计算出，查找任意元素的时间都是 \(O(1)\)，非常快。
4.  **那顺序表的缺点是什么？（PPT 没写，但后续课程必讲）**
    *   因为内存是连续的，**插入和删除**元素时，需要把插入点**后面所有的元素**整体向后或向前移动，这非常耗时（时间复杂度 \(O(n)\)）。
    *   **动态扩容代价大**：如果数组装满了需要扩容，往往需要重新申请一个更大的内存块，并**把原来的所有数据搬运过去**，非常消耗性能。

**学习建议**：把这两页讲的是**“顺序表（基于数组）”**，和你之前用指针链表实现的**“单链表”**在心里形成一个强烈的对比，数据结构课程的核心思想就在于此（**以空间换时间，还是以时间换空间**）。
