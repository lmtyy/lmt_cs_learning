这三个工具都是程序中进行**动态内存分配（Dynamic Memory Allocation）**的核心。虽然它们的目的都是在堆（Heap）上申请空间，但在功能、语言所属和底层机制上有显著区别。

------

### 1. `malloc` (C 语言标准库函数)

`malloc` 代表 **m**emory **alloc**ation，是 C 语言中最基础的分配方式。

- **语法：** `void* malloc(size_t size);`
- **核心特性：**
  - **按字节分配：** 你必须手动计算需要的字节数（通常配合 `sizeof` 使用）。
  - **返回类型：** 返回一个 `void*` 指针，代表“纯粹的内存地址”，在使用前通常需要**强制类型转换**。
  - **不初始化：** 申请到的内存空间里存放的是随机的“垃圾值”。
  - **不调用构造函数：** 它只负责在堆上划出一块地，并不关心这块地上要盖什么“建筑”（对象）。
- **配套使用：** 必须使用 `free()` 来释放内存。

------

### 2. `new` (C++ 运算符)

`new` 是 C++ 的关键字/运算符，它不仅是内存分配，更是**对象的创建**。

- **语法：** `Type *ptr = new Type;` 或 `Type *ptr = new Type[n];`
- **核心特性：**
  - **类型安全：** 它自动返回对应类型的指针，不需要强转。
  - **自动计算大小：** 你告诉它要创建一个 `int` 或一个结构体，它会自动计算需要的字节数。
  - **调用构造函数：** 这是与 `malloc` 的本质区别。`new` 会在分配空间后，自动执行该类的**构造函数**来初始化对象。
  - **异常处理：** 如果内存分配失败，`new` 默认会抛出 `std::bad_alloc` 异常，而 `malloc` 只是简单地返回 `NULL`。
- **配套使用：** 必须使用 `delete`（或 `delete[]` 用于数组）来释放，它会自动调用**析构函数**。

------

### 3. `realloc` (C 语言重新分配函数)

`realloc` 代表 **re**-**alloc**ation，专门用于调整已经分配好的内存块的大小。

- **语法：** `void* realloc(void* ptr, size_t new_size);`
- **核心特性：**
  - **原地扩容（最佳情况）：** 如果当前内存块后面有足够的空闲空间，直接向后延伸。
  - **异地搬迁（一般情况）：** 如果后面没空位了，它会找一块足够大的新地方，**自动将旧数据拷贝过去**，然后释放旧内存，并返回新地址。
  - **收缩空间：** 它也可以用来缩小内存块，释放多余的部分。
- **危险点：** 如果分配失败会返回 `NULL`，此时如果直接赋值给原指针（如 `p = realloc(p, size)`），会导致原内存地址丢失，造成**内存泄漏**。

------

### 核心对比表

| **特性**             | **malloc**        | **new**               | **realloc**       |
| -------------------- | ----------------- | --------------------- | ----------------- |
| **所属语言**         | C (C++可兼容)     | C++ 专有              | C (C++可兼容)     |
| **分配目的**         | 申请原始内存块    | 实例化对象            | 调整已有内存大小  |
| **是否调用构造函数** | 否                | **是**                | 否                |
| **返回类型**         | `void *` (需强转) | 具体类型指针 (自动)   | `void *` (需强转) |
| **失败表现**         | 返回 `NULL`       | 抛出异常 (Exception)  | 返回 `NULL`       |
| **所需参数**         | 字节总数          | 类型/元素个数         | 原指针 + 新字节数 |
| **释放方式**         | `free`            | `delete` / `delete[]` | `free`            |

------

### 技术细节拓展：内存扩容的代价

在数据结构中（例如你正在学习的**顺序表**），当数组满了需要扩容时，`realloc` 的机制非常关键。

1. **就地扩容**：时间复杂度 $O(1)$。
2. **异地搬迁**：时间复杂度 $O(n)$，因为涉及全量数据的内存拷贝（`memcpy`）。

> **注意：** 在 C++ 中，**绝对不要**混合使用这些工具。比如用 `malloc` 分配的内存千万不能用 `delete` 释放，反之亦然。这会导致未定义行为（Undefined Behavior），通常表现为程序直接崩溃。
