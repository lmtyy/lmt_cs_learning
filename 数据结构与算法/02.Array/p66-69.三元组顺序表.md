### 第一步：从“图像”到“列表”的转化（图1）

**核心知识点：三元组顺序表的存储逻辑。**

*   **左侧矩阵 A**：这是一个 \(6\) 行 \(7\) 列的稀疏矩阵。绝大部分是 0，非零元素用红色标出。
*   **右侧蓝表**：这就是对应的**“三元组顺序表”**。
    *   **列标**：`row`（行号）、`col`（列号）、`value`（值）。
    *   **存储规则（极其重要！）**：仔细观察蓝表中的 `row` 列（0, 0, 2, 2, 3, 4, 5, 5），你会发现它是**严格按照“行优先”的顺序排列的**。
    *   它先存第 0 行的两个非零元素；第 1 行全是 0，跳过；再存第 2 行的...以此类推。
    *   **目的**：虽然我们抛弃了矩阵的大网格结构，但通过这种**“有序的三元组列表”**，我们依然保留了矩阵的**逻辑顺序**。

### 第二步：如何在 C 语言中定义这种结构？（图2 & 图3）

**核心知识点：将链表/列表的抽象思维落地为 C 语言代码。**（这也是你在考试或上机时必写的代码）

1.  **图2：定义数据结构（Type definitions）**
    *   `#define MAX_TERMS 101`：定义一个最大容量。因为我们这里用的是**静态数组**，必须要预先设一个上限。`101` 代表最多能存 101 个非零项。
    *   **单项结构体 `term`**：
        ```c
        typedef struct {
            int col;    // 列号
            int row;    // 行号
            int value;  // 数值
        } term;
        ```
        它描述了三元组中的一个具体对象。
    *   **整体矩阵结构体 `Sparse_Matrix`**：
        ```c
        typedef struct {
            int rows;       // 原矩阵的总行数（宏观信息）
            int cols;       // 原矩阵的总列数（宏观信息）
            int terms;      // 实际存储的非零项总数（计数器）
            term elem[MAX_TERMS]; // 存放所有三元组的数组
        } Sparse_Matrix;
        ```
        这就是标准教材（如严蔚敏版）里定义的**三元组顺序表**。

2.  **图3：实际映射（数据填充）**
    *   图3就是把图1的数据填入图2定义的数据结构中去。
    *   **控制块（右上角灰框）**：
        *   `A.rows = 6` （行数）
        *   `A.columns = 7` （列数）
        *   `A.Terms = 8` （非零项数量：正好对应右下角蓝表的 8 行数据）。
    *   **指针连接**：红色箭头 `A.elem` 指向了数组的首地址。这正是动态/静态数组发挥作用的时刻。控制块通过指针掌握了所有元素的信息。

### 第三步：一个经典的算法问题——矩阵转置（图4）

**核心知识点：引入算法需求，为接下来的代码实现做铺垫。**

*   **什么是矩阵转置？**
    *   数学上，将原矩阵 \(A_{m \times n}\) 的行变成列，列变成行，得到新矩阵 \(B_{n \times m}\)。即 \(B_{ji} = A_{ij}\)。
*   **图4 示例**：
    *   **上方的 \(A_{6\times7}\)**：比如第 0 行第 3 列是 22。
    *   **下方的 \(B_{7\times6}\)**：转置后，22 就变成了第 3 行第 0 列。
*   **💡 对于“三元组顺序表”，转置不仅仅是“行、列互换”这么简单，它隐藏了一个巨大的难题！**

**思考题（请务必体会）：**
如果现在给你图1右侧的三元组列表，让你把它变成转置后的三元组列表：
1.  把所有行号和列号交换（这很容易，比如 `row:0, col:1, value:12` 变成 `row:1, col:0, value:12`）。
2.  **灾难出现了**：交换完之后，列表变成了 `row: 1, 2, 0, 5, 2, 1, 0, 3`。**它不再是按行优先排序的了！**
3.  但为了后续能高效地继续使用这个三元组，我们**必须在转置的过程中（或转置后），重新让它们按行优先排序**。

**接下来的课程重点（必考算法）**：
老师很快就会讲**“如何在三元组顺序表上进行转置”**。
通常会有两个算法：
1.  **普通转置（朴素转置）**：时间复杂度 \(O(cols \times terms)\)。也就是外层循环遍历列数，内层循环遍历所有项去找。效率低。
2.  **快速转置（Fast Transpose）**：时间复杂度 \(O(cols + terms)\)。利用**辅助数组**（`num[]` 和 `cpot[]`）预先计算出每一列在结果中的起始位置，直接把三元组“一步到位”放到正确的位置，无需二次排序。

**学习建议**：你现在不需要死记硬背代码，但必须清楚理解图3中 `rows`, `cols`, `terms`, `elem[]` 这四个元素的关系，以及图4所揭示的“转置会打乱行优先顺序”这个难点。掌握了这些，后面听“快速转置算法”就会非常顺利。
