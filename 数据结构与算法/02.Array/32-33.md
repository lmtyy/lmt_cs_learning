### 1. 顺序表的基本查找问题

在顺序表的应用中，最常面临两个核心操作问题：

- **按位序查找（Get the $i$th element）**：已知逻辑位置 $i$，获取该位置对应的数据元素内容。
- **按值查找（Find an element equal to value $e$）**：已知一个目标值 $e$，在表中寻找是否存在与之相等的元素，并返回其物理下标或逻辑位序。

------

### 2. `Find` 函数实现（按值查找）

第二页通过 C 语言代码展示了如何在一个 `SqList` 结构中定位特定元素。

#### 代码逻辑解析：

- **函数原型**：`int Find(SqList L, ElemType e, int Compare())`
  - `L`：被查找的顺序表。
  - `e`：目标查找值。
  - `Compare()`：**函数指针**。这允许调用者自定义比较规则（例如：如果是结构体，是比较 ID 还是比较姓名）。
- **循环机制**：使用 `for(i=0; i<L.length; i++)` 遍历整个表。
  - 这是典型的**线性查找（Linear Search）**，从物理下标 $0$ 开始，直到最后一个元素 $L.length-1$。
- **比较与返回**：
  - 如果 `Compare(L.elem[i], e)` 返回真，则认为找到了匹配项，立即返回当前的物理下标 `i`。
  - 如果循环结束仍未找到（即 `i >= L.length`），则返回 `-1` 表示查找失败。

### 💡 丰富知识点拓展

#### ① 随机访问（Random Access）与按位查找

顺序表实现“按位查找”极其高效。由于顺序表的物理存储是连续的，查找第 $i$ 个元素不需要遍历，直接通过基地址偏移即可计算：

$$Address(a_i) = Base + (i-1) \times sizeof(ElemType)$$

因此，**按位查找的时间复杂度为 $O(1)$**。

#### ② 线性查找的时间复杂度分析

对于 PPT 中的 `Find` 函数（按值查找）：

- **最好情况**：目标在第一个位置，比较 $1$ 次，复杂度 $O(1)$。
- **最坏情况**：目标在最后一个位置或不存在，比较 $n$ 次，复杂度 $O(n)$。
- **平均情况**：假设目标出现在每个位置的概率相等，平均比较次数为 $\frac{n+1}{2}$，因此**平均时间复杂度为 $O(n)$**。

#### ③ 函数指针 `Compare` 的妙用

在工程实践中，`ElemType` 可能是一个复杂的结构体（如 `Student`）。通过传入 `Compare` 函数，我们可以在不修改 `Find` 函数源码的情况下，灵活切换查找逻辑（如按学号找、按身份证号找等），这体现了程序的**高内聚低耦合**。

#### ④ 逻辑位序与物理下标的转换

- **逻辑位序**：人类习惯从 $1$ 开始计数（第 $1, 2, \dots, n$ 个）。
- **物理下标**：C 语言数组从 $0$ 开始计数（$0, 1, \dots, n-1$）。
- 在代码实现时，必须时刻注意这个 **$-1$** 的偏移，否则会导致**越界错误**。
