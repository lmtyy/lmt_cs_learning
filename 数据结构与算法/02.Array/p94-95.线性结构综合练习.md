### 第一题：多项式加法（图1）

**【题目分析】**
*   **已知数据结构**：使用“稀疏表示法”存储多项式，结构体包含：
    *   `termType`：单项式（包含系数 `coef` 和指数 `expn`）。
    *   `polynomial`：多项式（包含非零项数量 `noelem` 和存单项式的数组 `elem`）。
    *   数组大小固定为 `MAX_TERMS = 100`（静态数组）。
*   **目标函数**：`void AddPolyn(polynomial &Pa, polynomial &Pb);`
*   **要求**：实现两个多项式的加法，结果存入 `Pa`，并销毁（清空） `Pb`。

**【核心考点与解题思路】**

**1. 关键前提假设（PPT 没写，但必须默认成立）**
要让多项式相加变得高效，**`elem` 数组中的单项式必须按指数（`expn`）从小到大排序**。如果没排序，这道题的代码将极其复杂（需要双重循环排查）。我们假设它是排好序的。

**2. 算法逻辑：双指针归并法（类似于合并两个有序表）**
既然两个数组都是按指数排好序的，我们就可以使用**两个指针（`i` 和 `j`）**分别遍历 `Pa` 和 `Pb`。
*   **情况 A（指数相等）**：`Pa.elem[i].expn == Pb.elem[j].expn`。
    *   系数相加：`new_coef = Pa.elem[i].coef + Pb.elem[j].coef`。
    *   **如果 `new_coef != 0`**：将新项存入结果数组，`i++`, `j++`。
    *   **如果 `new_coef == 0`**：该项抵消了，不存入结果，只需 `i++`, `j++`。
*   **情况 B（Pa 当前指数更小）**：`Pa.elem[i].expn < Pb.elem[j].expn`。
    *   直接把 `Pa.elem[i]` 复制到结果数组，`i++`。
*   **情况 C（Pb 当前指数更小）**：`Pa.elem[i].expn > Pb.elem[j].expn`。
    *   直接把 `Pb.elem[j]` 复制到结果数组（因为结果要存入 Pa），`j++`。
*   **收尾**：当一个指针走到尽头后，把另一个多项式剩余的所有项直接追加到结果数组末尾。

**3. 空间扩容隐患（陷阱）**
题目中定义的是 `elem[MAX_TERMS]` 静态数组。
*   **致命问题**：如果 `Pa` 有 60 项，`Pb` 有 50 项，且没有可以相互抵消的项，相加的结果将会有 110 项，**超出了 MAX_TERMS (100) 的限制**！
*   **代码实现时必须处理**：在向结果数组中存入新项之前，**必须判断当前的 `noelem` 是否已经等于 `MAX_TERMS`**。如果已满，要么报错，要么需要改用动态分配的数组（如前面学过的 `realloc`）。

### 第二题：求两个有序顺序表的并集（图2）

**【题目分析】**
*   **已知数据结构**：动态顺序表 `SqList`（含动态数组指针 `elem`、当前长度 `length`、分配容量 `listsize`）。
*   **已知条件**：两个列表 **`L1` 和 `L2` 已经是排好序的（Sorted）**。
*   **目标**：编写程序计算 `L1 ∪ L2`（并集）。
*   **隐含要求**：求出的并集也必须是有序的，且去除重复元素。

**【核心考点与解题思路】**

**1. 算法选择：依然是双指针归并法**
因为我们拿到的本身就是**两个有序表**，绝对不能使用之前讲过的“暴力嵌套循环”（时间复杂度 \(O(n \times m)\)）。必须利用有序特性，使用双指针归并，时间复杂度降为 **\(O(m + n)\)**。

**2. 算法执行流程**
*   **准备结果容器**：创建一个新的空顺序表 `L3`（可以用 `InitList` 初始化，初始大小设为 `L1.length + L2.length`）。
*   **双指针遍历**：`i` 指向 `L1`，`j` 指向 `L2`。
*   **循环比较**：当 `i < L1.length` 且 `j < L2.length` 时，比较 `L1.elem[i]` 和 `L2.elem[j]`：
    *   **如果 `L1.elem[i] < L2.elem[j]`**：说明 `L1` 当前的数比较小，且 `L2` 中还没有出现。将 `L1.elem[i]` **追加**到 `L3` 末尾，`i++`。
    *   **如果 `L1.elem[i] > L2.elem[j]`**：将 `L2.elem[j]` **追加**到 `L3` 末尾，`j++`。
    *   **如果 `L1.elem[i] == L2.elem[j]`**：说明遇到了相同元素（并集要去重）。只需将其中一个（比如 `L1.elem[i]`）**追加**到 `L3`，然后**同时让 `i++` 和 `j++`**（跳过重复项）。
*   **收尾**：将剩余未遍历完的列表元素，按顺序全部**追加**到 `L3` 末尾。
*   **善后**：题目没要求结果存在新表，可以将 `L3` 的值覆盖回 `L1`（注意释放 `L1` 旧内存），或者直接返回 `L3`。

### 💡 总结与学习建议（这两道题的核心套路）

如果你看懂了这两道题的考点，你会发现**它们的解法在本质上是一模一样的！**

1.  **核心套路**：凡是遇到**“两个已经排好序的数据结构（多项式、顺序表）”**要进行合并、相加、求并集时，**首选算法一定是“双指针归并（类似于拉链）”**。
2.  **标准模板**：
    *   初始化两个下标指针 `i=0`, `j=0`。
    *   `while(i < 长度1 && j < 长度2)`：比较 `A[i]` 和 `B[j]` -> 谁小谁先放入结果 -> 对应指针后移 -> 相等则去重/相加处理。
    *   两个 `while` 循环处理剩下的尾巴。
3.  **深度思考**：图1 的多项式加法和图2 的并集，其算法的时间复杂度都是 **\(O(m+n)\)**。你可以对比一下前面学过的**“矩阵求并集”**（如果无序，需要 \(O(m \times n)\)）。这就是 **“数据结构的有序性”** 带来的巨大性能优势！考试时，写出这种高效的 \(O(m+n)\) 算法，是拿高分的关键。
