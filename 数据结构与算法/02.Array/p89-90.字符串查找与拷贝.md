这两页 PPT 展示了 C 语言中两个极其经典的**字符串底层操作函数**：一个是字符串查找（`Index`），一个是字符串拷贝（`strcpy`）。

你可以把它们看作是**“C语言指针和内存操作的实战演习”**。在理解了之前的“ADT 抽象定义”后，这两页给你展示了**“在计算机底层，代码到底是怎么跑起来的”**。

我们逐页进行详细拆解：

---

### 第一页：字符串子串定位（图1 - `Index` 函数）

**核心目标：** 在主字符串 `s` 中，找到子字符串 `t` **第一次出现的位置**。

#### 1. 算法逻辑分析（指针运算）
*   **函数签名**：`int Index(char *s, char* t)`
*   **核心循环**：`for(k=0; s[k]!='\0'; k++)`
    *   这是一个从主串 `s` 开头向后移动的遍历循环。`k` 是当前尝试匹配的**起始下标**（注意：这里是基于 **0** 的数组下标，不是 ADT 定义里的 1）。
*   **关键代码**：`if(strncmp(s+k, t, strlen(t))) break;`
    *   `s+k`：这是一个非常精妙的指针运算，表示从主串 `s` 的第 `k` 个下标开始往后看。
    *   `strncmp(s+k, t, strlen(t))`：C 标准库函数。**把从 `s+k` 开始的 `strlen(t)` 个字符，与子串 `t` 进行比对**。
    *   **为什么是 `strncmp` 而不是 `strcmp`？** 因为 `strcmp` 会比较直到遇到 `\0`，如果主串 `s` 后面还有很长内容，就会比较错误。用 `strncmp` 限制比较长度（限定为子串长度）保证了只在子串范围内进行比较。
*   **返回值处理（潜在的陷阱）**：
    *   `if (s[k] != '\0') return k+1;`
    *   **特别注意：** 如果找到了，它返回的是 `k+1`。因为 `k` 是 C 语言的数组下标（从 0 开始），而我们在 ADT 定义中规定，**逻辑位置通常从 1 开始**（找不到返回 0）。这个 `+1` 就是为了将底层下标转换为外部使用的逻辑位置。

#### 2. 算法复杂度分析（引申考点）
虽然这段代码写得很短，但它实际上是 **“朴素模式匹配算法（Brute-Force，简称 BF 算法）”**。
*   **最坏情况时间复杂度**：**\(O(m \times n)\)**（其中 \(m\) 是主串长度，\(n\) 是子串长度）。
*   **举例**：如果主串是 `"AAAAAAAAB"`，子串是 `"AAAAB"`。每次比对到最后一步才发现不匹配，`k` 指针每次必须回溯，导致效率极低。这也是为什么后面课程要引入 **KMP 算法** 的原因。

### 第二页：字符串拷贝（图2 - `strcpy` 函数）

**核心目标：** 将源字符串 `strSrc` 的内容，完整复制到目标内存 `strDest` 中。这是 C 语言指针操作最经典的代码。

#### 1. 代码逐行解析
*   **防御性编程（排雷）**：`assert((strDest!=NULL) && (strSrc !=NULL));`
    *   在操作内存前，必须确保传入的指针不是空指针（`NULL`）。如果传入空指针，程序直接崩溃报错，防止后续发生严重的“段错误（Segmentation Fault）”。
*   **保存首地址**：`char *address = strDest;`
    *   由于后面的代码会让 `strDest` 指针一直向后移动，为了最后能返回目标内存的起始地址，必须提前把这个“头”保存下来。
*   **核心搬运（单行魔咒）**：`while( (*strDest++ = *strSrc++) != '\0' );`
    *   这是一个**极简、极高效的 C 语言写法**。用中文翻译执行顺序是：
        1.  `*strSrc`：取出源指针指向的字符。
        2.  `*strDest = *strSrc`：把取出的字符，赋值给目标指针指向的位置。
        3.  `strSrc++` 和 `strDest++`：两个指针都向后移动一位，指向下一个字符。
        4.  判断刚才赋值的**字符**（也就是第 2 步的结果）是不是字符串结束符 `\0`。
    *   如果是 `\0`，循环结束；如果不是，继续循环。
*   **返回结果**：`return address;`
    *   返回目标字符串的起始首地址，这是标准库 `strcpy` 的规范，方便调用者进行链式操作（如 `printf("%s", strcpy(dest, src));`）。

#### 2. 为什么在这个实现里，最后没有给字符串加 `\0`？
这是一个非常敏锐的问题！
*   **答案是：不需要！**
*   在这个 `while` 循环的**最后一次执行**时，当指针指向源字符串的 `\0` 时，它会先把 `\0` **赋值**给目标字符串的对应位置，然后再因为 `\0` 不满足 `!= '\0'` 而跳出循环。所以，**目标字符串的末尾已经天然被赋予了 `\0`，不需要额外写代码去加。**

### 💡 核心总结：这两页 PPT 告诉了我们什么？

1.  **C 语言的核心是内存和指针**：图2 的 `*strDest++ = *strSrc++` 这种写法，只有 C 语言能写得这么优雅高效，这正是底层系统编程的魅力。
2.  **理论与实践的距离**：图1 的 `Index` 实现非常易懂，但它也暴露了朴素匹配算法的性能瓶颈。**数据结构课程不满足于“能用”，还要追求“高效”**。这也是为什么老师在讲完这段简单的 `Index` 后，一定会引出复杂的 **KMP 算法**。
3.  **安全边界**：图2 第一句 `assert` 告诉我们，**在 C 语言中编写内存操作函数，必须把“防空指针”作为第一要务**。未来你写 C 语言工程时，也一定要养成先检查指针是否为 `NULL` 的习惯。
