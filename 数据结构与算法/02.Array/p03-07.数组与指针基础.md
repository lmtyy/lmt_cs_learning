# 五页PPT知识点详细讲解

## 第一页（P3）：What is an array? — 什么是数组

这页给出了数组的两个核心定义：

**定义1：一组连续的内存单元（A consecutive set of memory locations）**

数组在物理内存中是连续存放的，这是数组最本质的特征。正因为连续，才能通过基地址+偏移量直接计算出任意元素的地址，实现 O(1) 的随机访问。

**定义2：一组 <index, value> 对的集合（A set of pairs <index, value>）**

从抽象角度看，数组本质上是一个从下标（index）到值（value）的映射关系。给定一个下标，就能唯一确定一个值。

**图示的含义：**
- 上面一行（✓）：所有格子里放的是同类型的元素（都是笑脸），这是合法的数组——数组要求元素类型相同。
- 下面一行（✗）：格子里放了不同类型的东西（花、忍者、怪物等），这不是合法的数组——数组不允许混合类型。

**要点总结：** 数组的两大特征是"连续存储"和"同类型元素"。

## 第二页（P4）：ADT of Array — 数组的抽象数据类型

这页用ADT的标准格式来形式化定义数组。

**Objects（数据对象）：**

一组 <index, value> 对的集合，其中每个合法的 index 值都对应集合 item 中的一个 value。也就是说，数组是一个"下标→值"的全映射，每个下标位置都必须有值。

**Functions（操作集合）：**

| 操作                                    | 含义                                                         |
| --------------------------------------- | ------------------------------------------------------------ |
| `InitArray(&A, n, index1, ..., indexn)` | 创建一个 n 维数组，各维的大小分别为 index1 到 indexn         |
| `DestroyArray(&A)`                      | 销毁数组，释放内存                                           |
| `ItemValue(A, &e, index1, ..., indexn)` | 取值操作——根据给定的下标组合，返回数组 A 中对应位置的元素值赋给 e |
| `Assign(&A, e, index1, ..., indexn)`    | 赋值操作——将值 e 存入数组 A 中指定下标位置                   |

**深入理解：**
- `&A` 表示传引用，函数内部可以修改数组本身
- 这里定义的是 n 维数组的通用ADT，一维数组只是 n=1 的特例
- ADT只定义"做什么"，不关心"怎么做"，具体实现可以是顺序存储、也可以是其他方式
- 注意 Assign 的语义：返回一个"除了新插入的 <index, e> 对之外与原数组相同"的数组，这是函数式的描述方式

## 第三页（P5）：Array in C — C语言中的一维数组声明

这页介绍C语言中一维数组的两种声明方式：

**1. 普通整型数组：**
```c
int list[5];
```
- 声明了一个包含5个整数的数组
- 元素为 `list[0]` 到 `list[4]`（C语言下标从0开始）
- 编译时就分配了 5×sizeof(int) 字节的连续内存空间（栈上分配）

**2. 指针数组：**
```c
int *plist[100];
```
- 声明了一个包含100个元素的数组，每个元素是一个指向 int 的指针
- 每个 `plist[i]` 存放的是一个地址，该地址指向某个 int 变量
- 占用空间为 100×sizeof(int*) 字节

**两者的区别：**
- `int list[5]` 直接存储5个整数值
- `int *plist[100]` 存储100个地址，每个地址指向一个整数

这是理解C语言中数组与指针关系的基础。

## 第四页（P6）：Array in C — 一维数组的内存布局

这页展示了 `int list[5]` 在内存中的具体地址分布：

| 变量      | 内存地址                   |
| --------- | -------------------------- |
| `List[0]` | base address = a（基地址） |
| `List[1]` | a + sizeof(int)            |
| `List[2]` | a + 2*sizeof(int)          |
| `List[3]` | a + 3*sizeof(int)          |
| `List[4]` | a + 4*sizeof(int)          |

**寻址公式（核心）：**

对于一维数组，第 i 个元素的地址为：

```
Address(list[i]) = base_address + i × sizeof(元素类型)
```

**为什么这很重要：**
- 这个公式说明，访问任意元素只需要一次乘法和一次加法，时间复杂度为 O(1)
- 这就是数组"随机访问"能力的本质原因
- 如果 sizeof(int) = 4 字节（32位系统），那么 list[3] 的地址就是 a + 12

**注意：** 下标从0开始不是随意的设计，正是因为 list[0] 的偏移量为 0×sizeof(int) = 0，即它就在基地址处，这样寻址公式最简洁。

## 第五页（P7）：Array in C — 数组名与指针的关系

这页揭示了C语言中一个非常重要的概念：**数组名本质上是指针**。

**声明对比：**
```c
int *list1;      // 指针变量，指向int
int list2[5];    // 数组，包含5个int
```

**核心知识点1：list1 和 list2 都是指向 int 的指针**

- `list1` 是一个指针变量，可以指向任何 int 地址
- `list2` 是数组名，它的值就是数组首元素的地址，类型上等价于 `int*`
- 区别在于：`list2` 是常量指针（不能修改指向），而 `list1` 是变量指针（可以重新赋值）

**核心知识点2：编译器为 list2 预留了5个int的空间**

声明 `int list2[5]` 时，编译器在栈上分配了 5×sizeof(int) 字节的连续空间。而 `int *list1` 只分配了一个指针大小的空间（通常4或8字节），并没有分配实际存储数据的空间。

**核心知识点3：下标访问与指针运算等价**

```c
&list2[2]  等价于  (list2 + 2)    // 取第2个元素的地址
list2[2]   等价于  *(list2 + 2)   // 取第2个元素的值
```

这就是C语言中著名的**数组下标与指针算术的等价关系**：
- `list2 + 2` 不是地址加2个字节，而是加 2×sizeof(int) 个字节（指针算术自动按元素大小缩放）
- `*(list2 + 2)` 是对该地址解引用，得到存储在那里的值
- 所以 `a[i]` 本质上是 `*(a + i)` 的语法糖

**实际意义：** 理解这个等价关系后，你就能明白为什么C语言中数组可以传递给接受指针参数的函数，以及为什么指针可以用下标运算符 `[]` 来访问。
