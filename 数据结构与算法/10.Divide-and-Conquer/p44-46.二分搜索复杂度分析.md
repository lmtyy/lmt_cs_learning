# 二分搜索 — 空间时间分析与比较次数（第44-46页）

分析二分搜索的空间复杂度和各种情况下的时间复杂度（最好/平均/最坏）。

---

## 第44页：二分搜索所需的空间和时间

### 所需空间

存放数组 a 的地址，还有 n、left、right、middle 及 x 的地址需要存储，共 **24 字节**。

> 即额外空间 O(1)，只需几个辅助变量。

### 计算时间——需要讨论的情况

| 情况 | 含义 |
|------|------|
| 成功检索的**最好**情况 | x 恰好是中间元素，1次比较 |
| 不成功检索的**最好**情况 | 最少几次比较能确定不存在 |
| 成功检索的**平均**情况 | 所有成功位置的平均比较次数 |
| 不成功检索的**平均**情况 | 所有失败位置的平均比较次数 |
| 成功检索的**最坏**情况 | 最深的内顶点 |
| 不成功检索的**最坏**情况 | 最深的外顶点 |

---

## 第45页：成功检索与不成功检索的最好情况（n=9 示例）

### 比较树（9个元素，下标 0-8）

```
                    (4)          ← 第1层：比较1次
                 /       \
              (1)         (6)    ← 第2层：比较2次
            /     \     /     \
          (0)    (2)  (5)    (7)  ← 第3层：比较3次
                   \           \
                   (3)         (8)  ← 第4层：比较4次
         □ □ □     □ □   □ □     □ □  ← 外顶点（不成功）
```

### 数组与比较次数

| 下标 a | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 | 8 |
|--------|---|---|---|---|---|---|---|---|---|
| 元素 | 2 | 5 | 6 | 7 | 9 | 23 | 54 | 82 | 101 |
| **成功比较次数** | 3 | 2 | 3 | 4 | **1** | 3 | 2 | 3 | 4 |

- 成功检索共有 **n = 9** 种
- 不成功检索共有 **n + 1 = 10** 种

### 不成功检索的比较次数

| 外顶点位置 | 比较次数 |
|-----------|---------|
| 前3个间隙 | 3 |
| 中间几个间隙 | 3 或 4 |
| 后2个间隙 | 4 |

> 不成功比较次数（每个外顶点）：3, 3, 3, 4, 4, 3, 3, 3, 4, 4

---

## 第46页：最多比较次数的一般结论

### 本例分析

- 当 x 在数组 A 中时，算法在**圆形顶点（内顶点）**结束
- 当 x 不在 A 中时，算法在**方形顶点（外顶点）**结束
- 因为 2³ ≤ 9 < 2⁴，所以比较树的叶顶点都在第 **4** 或第 **5** 级上出现
- 因而元素比较的最多次数为 **4**

### 一般结论

当 n ∈ [2^(k-1), 2^k) 时：

| 情况 | 最多比较次数 |
|------|-----------|
| 一次**成功**的搜索 | 至多 **k** 次 |
| 一次**不成功**的搜索 | **k-1** 次或 **k** 次 |

### 用 log 表示

由于 k = ⌈log₂(n+1)⌉：

| 情况 | 比较次数 |
|------|---------|
| 最好（成功） | **1**（x 恰好在中间） |
| 最坏（成功） | **⌈log₂(n+1)⌉** |
| 最坏（不成功） | **⌈log₂(n+1)⌉** |

> ✅ 无论成功还是失败，二分搜索最多比较 ⌈log₂(n+1)⌉ 次，即 **O(log n)**。

---

## 总结

```
二分搜索复杂度汇总：

┌──────────────────────────────────────────┐
│  空间：O(1)（只需 left, right, middle 等变量）│
│                                            │
│  时间：                                     │
│    最好情况：O(1)  — x 恰在正中间            │
│    最坏情况：O(log n) — 搜到叶子层           │
│    平均情况：O(log n)                       │
│                                            │
│  比较树性质：                                │
│    内顶点（成功）= n 个                      │
│    外顶点（失败）= n+1 个                    │
│    树高 = ⌈log₂(n+1)⌉                      │
└──────────────────────────────────────────┘

对比线性搜索：
  线性搜索：O(n) → 慢
  二分搜索：O(log n) → 快得多（n=10⁶ 时只需约20次比较！）
```
