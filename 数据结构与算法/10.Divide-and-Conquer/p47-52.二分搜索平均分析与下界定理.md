# 二分搜索 — 平均复杂度推导与下界定理（第47-52页）

通过内部/外部路径长度推导平均比较次数，证明二分搜索是最优的比较搜索算法。

---

## 第47页：二分搜索的时间复杂度汇总

| 情况 | 复杂度 |
|------|--------|
| 最坏情况下的成功检索 | **Θ(log n)** |
| 最坏情况下的不成功检索 | **Θ(log n)** |
| 最好情况下的成功检索 | **Θ(1)** |
| 最好情况下的不成功检索 | **Θ(log n)** |
| 每种不成功的检索时间都为 | **Θ(log n)** |

> 注意：不成功检索即使在最好情况下也是 Θ(log n)，因为必须走到叶子才能确认"不存在"。

---

## 第48页：平均情况分析 — 路径长度法

### 关键定义

| 符号 | 定义 |
|------|------|
| **I**（内部路径长度） | 由根到所有**内结点**的距离之和 |
| **E**（外部路径长度） | 由根到所有**外结点**的距离之和 |

### 核心关系

```
E = I + 2n        ……(*)
```

> 这是二叉树的经典性质：外部路径长度 = 内部路径长度 + 2×内结点数

### 引入平均比较次数

| 符号 | 含义 | 公式 |
|------|------|------|
| **U(n)** | 平均情况下**不成功**检索的比较数 | U(n) = E / (n+1) |
| **S(n)** | 平均情况下**成功**检索的比较数 | S(n) = I/n + 1 |

- U(n) = E/(n+1)：外部路径总长 ÷ 外顶点数（n+1个失败位置）
- S(n) = I/n + 1：内部路径总长 ÷ 内顶点数 + 1（每次比较在到达该节点后还要做一次比较）

---

## 第49页：S(n) 与 U(n) 的关系推导

### 公式体系

```
E = I + 2n                    ……(1)
S(n) = I/n + 1                ……(2)
U(n) = E/(n+1) = Θ(log n)    ……(3)
S(n) = (1 + 1/n)U(n) - 1     ……(4)
```

### 推导过程

由 (1)：E = I + 2n

由 (2)：I = n·(S(n) - 1)

代入 (1)：E = n·(S(n) - 1) + 2n = n·S(n) + n

由 (3)：U(n) = E/(n+1) = (n·S(n) + n)/(n+1)

反解 S(n)：

```
S(n) = (n+1)/n · U(n) - 1
     = (1 + 1/n) · U(n) - 1    ……(4)
```

### 结论

当 n → ∞ 时：

```
S(n) = (1 + 1/n)U(n) - 1 ≈ U(n) - 1
```

而 U(n) = Θ(log n)

> ✅ 所以 **S(n) = Θ(log n)**

## 第50页：二分检索在各种情况下的检索时间（总结表）

| 计算时间 | 最好 | 平均 | 最坏 |
|---------|------|------|------|
| **成功的检索** | Θ(1) | Θ(log n) | Θ(log n) |
| **不成功的检索** | Θ(log n) | Θ(log n) | Θ(log n) |

> ✅ 除了"成功检索的最好情况"是 Θ(1)，其余全部是 Θ(log n)。

## 第51页：下界定理 — 二分搜索是最优的

### 定理 1

> 设数组 A[1:n] 的元素满足 A[1] < A[2] < … < A[n]，x 属于 A[1,n]。则以比较为基础去判断的任何算法在最坏情况下所需的最少比较次数 Find(n) 不小于 **⌈log(n+1)⌉**。

### 含义

> 任何一种以比较为基础的算法，其最坏情况时间都**不可能低于 O(log n)**，也就是不可能存在其最坏情况时间比二分检索数量级还低的算法。

### 直觉理解

```
n 个元素 + (n+1) 个间隙 = 2n+1 种可能结果
比较树是二叉树，高度至少为 ⌈log₂(2n+1)⌉ ≈ log₂(n)
每次比较最多排除一半 → 至少需要 log₂(n) 次才能确定结果
```

> ✅ **二分搜索已经达到了比较搜索的理论最优下界！**

## 第52页：Exercises（课后习题）

### 习题 1

> Show that the solution of **T(n) = T(n/2) + 1** is O(lg n).

**提示**：这就是二分搜索的递推关系。展开：
```
T(n) = T(n/2) + 1
     = T(n/4) + 1 + 1
     = T(n/8) + 3
     = ...
     = T(1) + log₂n
     = O(log n)
```
或是根据T(n)=kT(m/n)+f(n) -> T(n)=n^{log_m{k}}+Σ{j=0}{log_m{n}}(k^{j}f(m/(n^{j}))

### 习题 2

> Show that the solution to **T(n) = 2T(n/2 + 17) + n** is O(n lg n).

**提示**：当 n 足够大时，n/2 + 17 ≈ n/2，可忽略常数项 17。本质上等价于 T(n) = 2T(n/2) + n，即 MergeSort 的递推关系，解为 O(n log n)。严格证明可用代入法（substitution method）。
