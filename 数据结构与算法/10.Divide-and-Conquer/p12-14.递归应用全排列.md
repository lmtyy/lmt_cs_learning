# 递归应用 — 排列问题 Permutation（第12-14页）

设计递归算法生成全排列，是理解递归"分解 → 组合"思想的经典例题。

---

## 第12页：排列问题引入

### 问题

设计一个递归算法生成 n 个元素 {r₁, r₂, …, rₙ} 的全排列，记为 `perm({r₁, r₂, …, rₙ})`。

### 示例：{a, b, c} 的全排列

> n 个元素的排列方式共有 **n!** 种。

3个元素 → 3! = 6 种：abc, acb, bac, bca, cab, cba

### 观察规律——按首字母分组

```
abc + acb = a(bc + cb) = a·perm({b,c})
bac + bca = b(ac + ca) = b·perm({a,c})
cab + cba = c(ab + ba) = c·perm({a,b})
```

> 类比：排队拍照——轮流让每个人站第一个，剩下的人在后面自由排列。

---

## 第13页：递归公式化

### 符号定义

| 符号 | 含义 |
|------|------|
| R = {r₁, r₂, …, rₙ} | 要排列的 n 个元素 |
| Rᵢ = R - {rᵢ} | 去掉第 i 个元素后的子集 |
| (rᵢ)perm(X) | 在 perm(X) 的每个排列前面加上 rᵢ |

### 递归定义

```
当 n = 1 时：perm(R) = (r)，其中 r 是集合 R 中唯一的元素

当 n > 1 时：perm(R) = (r₁)perm(R₁), (r₂)perm(R₂), …, (rₙ)perm(Rₙ)
```

### 展开示例：R = {a, b, c}

```
Perm(R) = a·perm({b,c}) + b·perm({a,c}) + c·perm({a,b})
```

逐项展开：

```
perm({b,c}) = b·perm({c}) + c·perm({b})

∴ a·perm({b,c}) = ab·perm({c}) + ac·perm({b})
                 = ab·c + ac·b
                 = (abc, acb)

同理：
b·perm({a,c}) = ba·perm({c}) + bc·perm({a})
              = ba·c + bc·a
              = (bac, bca)

c·perm({a,b}) = ca·perm({b}) + cb·perm({a})
              = ca·b + cb·a
              = (cab, cba)
```

> ✅ 总结果：abc, acb, bac, bca, cab, cba — 恰好 3! = 6 种

## 第14页：排列问题的递归算法（代码）

```cpp
template<class T>
void Perm(T list[], int k, int m)
{ // 生成 list[k:m] 的所有排列方式
    int i;
    if (k == m)
    { // 只剩一个元素，输出一个完整排列
        for (i = 0; i <= m; i++)
            cout << list[i];
        cout << endl;
    }
    else // list[k:m] 有多个排列方式
    { // 递归地产生这些排列方式
        for (i = k; i <= m; i++)
        {
            Swap(list[k], list[i]);   // 把第i个元素换到位置k
            Perm(list, k + 1, m);     // 递归排列 list[k+1:m]
            Swap(list[k], list[i]);   // 换回来（回溯）
        }
    }
}
```

### 调用方式

```cpp
// 输出 list[0:n-1] 的全排列
Perm(list, 0, n - 1);
```

### 算法核心思想：Swap + 递归 + 回溯

```
for i = k to m:
    ┌─ Swap(list[k], list[i])     ← 选择：把第i个放到最前面
    ├─ Perm(list, k+1, m)         ← 探索：递归排列后面的
    └─ Swap(list[k], list[i])     ← 撤销：恢复原状，试下一个
```

### 执行过程示意：list = [a, b, c], Perm(list, 0, 2)

```
k=0, i=0: Swap(a,a)→[a,b,c] → Perm(list,1,2) → Swap(a,a)
    k=1, i=1: Swap(b,b)→[a,b,c] → Perm(list,2,2) → 输出 "abc"
    k=1, i=2: Swap(b,c)→[a,c,b] → Perm(list,2,2) → 输出 "acb"
              Swap(b,c)→[a,b,c] ← 恢复

k=0, i=1: Swap(a,b)→[b,a,c] → Perm(list,1,2) → Swap(a,b)
    k=1, i=1: Swap(a,a)→[b,a,c] → 输出 "bac"
    k=1, i=2: Swap(a,c)→[b,c,a] → 输出 "bca"
              Swap(a,c)→[b,a,c] ← 恢复

k=0, i=2: Swap(a,c)→[c,b,a] → Perm(list,1,2) → Swap(a,c)
    k=1, i=1: Swap(b,b)→[c,b,a] → 输出 "cba"
    k=1, i=2: Swap(b,a)→[c,a,b] → 输出 "cab"
              Swap(b,a)→[c,b,a] ← 恢复
```

### 复杂度

| 指标 | 值 |
|------|-----|
| 排列总数 | n! |
| 每次输出开销 | O(n) |
| 总时间复杂度 | **O(n · n!)** |
| 递归深度（空间） | O(n) |

---

## 总结

```
全排列递归结构：

perm({a,b,c})
├── a + perm({b,c})     ← Swap(a,a)
│       ├── b + perm({c}) → abc
│       └── c + perm({b}) → acb
├── b + perm({a,c})     ← Swap(a,b)
│       ├── a + perm({c}) → bac
│       └── c + perm({a}) → bca
└── c + perm({a,b})     ← Swap(a,c)
        ├── b + perm({a}) → cba
        └── a + perm({b}) → cab

核心模式：选择 → 递归探索 → 回溯撤销（Backtracking 的雏形）
```
