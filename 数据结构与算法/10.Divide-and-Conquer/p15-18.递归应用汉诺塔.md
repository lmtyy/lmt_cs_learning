# 递归应用 — Hanoi 塔问题（第15-18页）

汉诺塔是递归的经典教学案例，完美体现"把大问题拆成相同结构的小问题"的递归本质。

---

## 第15页：问题描述

### 规则

设 A、B、C 是三个塔座。开始时，在塔座 A 上有一叠共 n 个圆盘，这些圆盘自下而上，由大到小地叠在一起，各圆盘从小到大的编号为 1, 2, …, n。

**目标**：将塔座 A 上的这一叠圆盘移到塔座 B 上，并仍按同样顺序叠置。

**移动规则**：

| # | 规则 |
|---|------|
| (1) | 每次只能移动**一个**圆盘 |
| (2) | 任何时刻都不允许将**较大**的圆盘压在**较小**的圆盘之上 |
| (3) | 在满足 (1) 和 (2) 的前提下，可将圆盘移至 A, B, C 中任一塔座上 |

```
初始状态：

     |          |          |
    [1]         |          |
   [ 2 ]       |          |
  [  3  ]      |          |
 [   4   ]     |          |
─────────── ─────────── ───────────
     A          B          C
```

---

## 第16页：递归分解思路

### 函数定义

`hanoi(n, A, B, C)` — 把 n 个盘从 A 移到 B，借助 C

### 分解为三步

| 步骤 | 操作 | 函数调用 | 图示 |
|------|------|---------|------|
| ① | 先把上面 n-1 个盘子从 A 借助 **B** 放到 **C** | `hanoi(n-1, A, C, B)` | 上面的盘搬到C |
| ② | 把第 n 个盘（最大的）从 A 移到 B | `move(n, A, B)` | 最大盘直达目标 |
| ③ | 把 C 上的 n-1 个盘子借助 **A** 移到 **B** | `hanoi(n-1, C, B, A)` | 搬回到B上 |

### 图示（n=4 的情况）

```
        A           B           C

初始：  [1234]      [ ]         [ ]

步骤①：  [  4 ]      [ ]       [123]    ← hanoi(3, A, C, B)
                                          把上面3个从A搬到C

步骤②：  [   ]      [ 4 ]      [123]    ← move(4, A, B)
                                          最大盘直接到B

步骤③：  [   ]      [1234]     [  ]     ← hanoi(3, C, B, A)
                                          把3个从C搬到B
```

> 类比：搬家时先把上面的小箱子搬到临时位置，再搬最重的大箱子到目的地，最后小箱子搬过来摞上去。

### Base Case

- n = 1 时：只要将编号为 1 的圆盘从 A 直接移动到 B 即可

## 第17页：算法代码

```cpp
void Hanoi(int n, int A, int B, int C)
{
    if (n > 0)
    {
        Hanoi(n-1, A, C, B);   // 步骤①：n-1个从A到C（借助B）
        move(n, A, B);          // 步骤②：第n个从A到B
        Hanoi(n-1, C, B, A);   // 步骤③：n-1个从C到B（借助A）
    }
}
```

### 参数含义

| 参数 | 含义 |
|------|------|
| n | 要移动的盘子数 |
| A | 源塔座（盘子当前在这里） |
| B | 目标塔座（盘子要去这里） |
| C | 辅助塔座（中转站） |

> ⚠️ 注意参数顺序：`Hanoi(n, 源, 目标, 辅助)`。两次递归调用中，源/目标/辅助的角色在**轮换**。

## 第18页：汉诺塔移动次数分析

### 递推关系

```
M(1) = 1
M(n) = 2M(n-1) + 1
```

- `2M(n-1)`：两次递归各移动 n-1 个盘
- `+1`：移动最大盘那一步

### 逐步展开

```
M(n) = 2M(n-1) + 1
     = 2[2M(n-2) + 1] + 1
     = 2²M(n-2) + 2 + 1
     = 2³M(n-3) + 2² + 2 + 1
     = ……
     = 2ⁱM(n-i) + 2ⁱ⁻¹ + 2ⁱ⁻² + … + 2 + 1
     = 2ⁱM(n-i) + (2ⁱ - 1)
```

### 代入 i = n-1

```
M(n) = 2^(n-1) · M(1) + 2^(n-1) - 1
     = 2^(n-1) · 1 + 2^(n-1) - 1
     = 2^(n-1) + 2^(n-1) - 1
     = 2 · 2^(n-1) - 1
```

> ✅ **M(n) = 2ⁿ - 1**

### 验证

| n | M(n) = 2ⁿ - 1 | 验证 |
|---|---------------|------|
| 1 | 1 | 只移1次 ✓ |
| 2 | 3 | 小到C，大到B，小到B ✓ |
| 3 | 7 | ✓ |
| 4 | 15 | ✓ |

> 增长极快：n=64 时需要 2⁶⁴ - 1 ≈ 1.8×10¹⁹ 次移动，按每秒1次算需约 **5845亿年**。

## 总结

```
汉诺塔的递归精髓：

问题规模 n → 拆成 2个规模 n-1 的子问题 + 1步操作

hanoi(n, A→B, 辅助C)
  ├── hanoi(n-1, A→C, 辅助B)    ← 搬开上面的
  ├── move(第n个, A→B)           ← 移动最大的
  └── hanoi(n-1, C→B, 辅助A)    ← 搬回来

递推关系：M(n) = 2M(n-1) + 1
闭合解：  M(n) = 2ⁿ - 1
复杂度：  O(2ⁿ) — 指数级，无法优化（已证明是最优解）
```
