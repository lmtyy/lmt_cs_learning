## 作业题 1

> 应用合并排序对序列 **E, X, A, M, P, L, E** 按照字母顺序排序，写出排序的过程。

### 参考解答

```
初始：  E  X  A  M  P  L  E

第1层分解：
  [E, X, A, M] 和 [P, L, E]

第2层分解：
  [E, X] [A, M] | [P, L] [E]

第3层分解：
  [E] [X] [A] [M] | [P] [L] [E]

自底向上合并：

  [E] [X] → 合并 → [E, X]
  [A] [M] → 合并 → [A, M]
  [P] [L] → 合并 → [L, P]
  [E]     → 保持 → [E]

  [E, X] + [A, M] → 合并 → [A, E, M, X]
  [L, P] + [E]    → 合并 → [E, L, P]

  [A, E, M, X] + [E, L, P] → 合并 → [A, E, E, L, M, P, X]
```

> ✅ 最终结果：**A, E, E, L, M, P, X**

## 作业题 2

> 分析 Merge 函数**最好**与**最坏**的键值比较次数。

### 参考解答

设合并两段长度分别为 m 和 n 的有序序列（总长 m+n）。

| 情况 | 比较次数 | 发生条件 |
|------|---------|---------|
| **最好** | min(m, n) | 一段的所有元素都小于另一段的第一个元素 |
| **最坏** | m + n - 1 | 两段元素交错排列 |

- 最好例子：合并 [1,2,3] 和 [4,5,6] → 比较 3 次（1<4, 2<4, 3<4，然后左段用完）
- 最坏例子：合并 [1,3,5] 和 [2,4,6] → 比较 5 次（每次都要比较才能决定下一个）

## 习题 4

> 用数学归纳法证明：当 n 是 2 的方幂（n = 2^k, k > 1）时，递推关系
> T(n) = { 2, if n=2; 2T(n/2)+n, if n=2^k for k>1 }
> 的解为 T(n) = n lg n。

### 参考解答

**基础步骤**：n = 2 时，T(2) = 2 = 2·lg2 = 2·1 = 2 ✅

**归纳假设**：假设对 n/2 有 T(n/2) = (n/2)·lg(n/2)

**归纳步骤**：

```
T(n) = 2T(n/2) + n
     = 2 · (n/2) · lg(n/2) + n      （代入归纳假设）
     = n · lg(n/2) + n
     = n · (lg n - lg 2) + n
     = n · (lg n - 1) + n
     = n·lg n - n + n
     = n·lg n                         ✅
```

> ✅ 证毕：T(n) = n lg n

## 习题 5

> 插入排序可以表示为递归过程：为了排序 A[1..n]，先递归排序 A[1..n-1]，然后将 A[n] 插入到已排好序的 A[1..n-1] 中。写出这个递归版插入排序的运行时间递推关系。

### 参考解答

```
T(1) = O(1)                    （只有1个元素，无需排序）
T(n) = T(n-1) + O(n)           （递归排前n-1个 + 插入第n个）
```

- `T(n-1)`：递归排序前 n-1 个元素
- `O(n)`：将 A[n] 插入到已排好序的 A[1..n-1] 中，最坏需要比较 n-1 次

**求解**：

```
T(n) = T(n-1) + cn
     = T(n-2) + c(n-1) + cn
     = T(1) + c·2 + c·3 + … + cn
     = O(1) + c(2 + 3 + … + n)
     = O(n²)
```

> ✅ 递归版插入排序的时间复杂度仍为 **O(n²)**，与迭代版相同。
