# MergeSort — 分治策略、代码与合并过程（第5-10页）

详解归并排序的分治思路、主程序代码，以及 Merge 合并操作的逐步执行过程。

---

## 第5页：分治策略求解

### 归并排序的分治逻辑

- 若 n 为 1，算法终止（base case）
- 否则，将这一元素集合分割成**两个**或更多个子集合，对每一个子集合分别排序
- 然后将排好序的子集合**归并**为一个集合

> ✅ 性能最佳的方式就是分成两个**元素个数相近**的集合，这就是**2路归并**。

### 为什么要等分？

| 分割方式 | 子问题规模 | 递推关系 | 复杂度 |
|---------|---------|---------|--------|
| 等分（n/2 + n/2） | 均衡 | T(n) = 2T(n/2) + O(n) | **O(n log n)** |
| 极端分割（1 + n-1） | 不均衡 | T(n) = T(n-1) + O(n) | O(n²) ← 退化！ |

## 第6页：归并排序过程（伪代码）

```
void MergeSort(type a[], int left, int right)
{
    if (left < right)  // 至少有两个元素
    {
        找到数组的中间元素下标 mid
        递归调用算法本身对从 left 到 mid 的数组元素排序
        递归调用算法本身对从 mid 到 right 的数组元素排序
        合并两段排好序的数组
    }
}
```

## 第7页：归并排序主程序代码

```cpp
template<class Type>
void MergeSort(Type a[], int left, int right)
{ // A[left:right] 是一个全程数组，
  // 含有 right-left+1 个待排序的元素。
    if (left < right)
    { // 至少有2个元素
        int mid = (left + right) / 2;   // 求当前数组的分割点
        MergeSort(a, left, mid);         // 递归排序左半
        MergeSort(a, mid + 1, right);    // 递归排序右半，此时得到的a按mid分开，左右各是两个有序的数组
        Merge(a, b, left, mid, right);   // 合并两个排好序的子问题，放入数组b，防止数据覆盖
        copy(a, b, left, right);         // 把b复制回a
    }
}
```

### 各行职责

| 行 | 作用 | 分治角色 |
|---|------|---------|
| `int mid = (left+right)/2` | 求分割点 | **Divide** |
| `MergeSort(a, left, mid)` | 递归排序左半 | **Conquer** |
| `MergeSort(a, mid+1, right)` | 递归排序右半 | **Conquer** |
| `Merge(a, b, left, mid, right)` | 合并两段有序数组到 b | **Merge** |
| `copy(a, b, left, right)` | 把结果从 b 复制回 a | 辅助 |

## 第8页：合并过程 Merge — 示例1

### 合并 [2,4,5,8] 和 [1,3,6,7]

**前提**：待合并的两段数据已经排好序。

左半指针 ↑，右半指针 ↑，每次比较两个指针指向的元素，较小的放入数组 b：

| 步骤 | 左半剩余 | 右半剩余 | 比较 | 放入 b |
|------|---------|---------|------|--------|
| 1 | **2**,4,5,8 | **1**,3,6,7 | 2 vs 1 | b ← 1 |
| 2 | **2**,4,5,8 | **3**,6,7 | 2 vs 3 | b ← 2 |
| 3 | **4**,5,8 | **3**,6,7 | 4 vs 3 | b ← 3 |
| 4 | **4**,5,8 | **6**,7 | 4 vs 6 | b ← 4 |
| 5 | **5**,8 | **6**,7 | 5 vs 6 | b ← 5 |
| 6 | **8** | **6**,7 | 8 vs 6 | b ← 6 |
| 7 | **8** | **7** | 8 vs 7 | b ← 7 |
| 8 | **8** | (空) | 剩余直接放入 | b ← 8 |

**结果**：b = [1, 2, 3, 4, 5, 6, 7, 8] ✅

## 第9页：合并过程 Merge — 示例2

### 合并 [1,7,8,9] 和 [2,3,4,5]

| 步骤 | 左半剩余 | 右半剩余 | 比较 | 放入 b | b 当前状态 |
|------|---------|---------|------|--------|-----------|
| 1 | **1**,7,8,9 | **2**,3,4,5 | 1 vs 2 | b ← 1 | [1] |
| 2 | **7**,8,9 | **2**,3,4,5 | 7 vs 2 | b ← 2 | [1,2] |
| 3 | **7**,8,9 | **3**,4,5 | 7 vs 3 | b ← 3 | [1,2,3] |
| 4 | **7**,8,9 | **4**,5 | 7 vs 4 | b ← 4 | [1,2,3,4] |
| 5 | **7**,8,9 | **5** | 7 vs 5 | b ← 5 | [1,2,3,4,5] |
| 6 | **7**,8,9 | (空) | 右半用完！ | — | — |

右半用完后，**停止扫描**，把左半剩余 [7,8,9] 全部放入 b：

**结果**：b = [1, 2, 3, 4, 5, 7, 8, 9] ✅

### ⚠️ 关键细节

> 在移动扫描指针的同时，要判断其是否已经到达数组的尾部。如到达，则停止扫描，把**未参加排序的数全部放入备用数组中**。

## 总结

```
MergeSort 执行流程：

原数组 A[left:right]
  │
  ├── mid = (left+right)/2
  ├── MergeSort(left, mid)     ← 递归排序左半
  ├── MergeSort(mid+1, right)  ← 递归排序右半
  └── Merge(left, mid, right)  ← 合并两个有序段
        │
        ├── 两个指针分别指向左半、右半的头部
        ├── 每次比较，取较小者放入 b
        ├── 某一半用完 → 另一半剩余直接追加
        └── 最后 copy b → a

Merge 操作的比较次数：
  最好：n/2（一半完全小于另一半）
  最坏：n-1（两半交错排列）
  → 合并是 O(n)
```
