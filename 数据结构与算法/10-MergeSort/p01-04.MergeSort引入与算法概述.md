# 合并排序 MergeSort — 引入与算法概述（第1-4页）

从插入排序的不足引出 MergeSort 的动机，介绍合并排序的基本算法框架。

---

## 第1页：问题提出

### 排序问题

> 已知 n 个元素的数组 A[1:n]，将 A 中元素按**不降顺序**排列。

---

## 第2-3页：插入排序 InsertionSort（对比铺垫）

### 代码

```cpp
template <class Type>
INSERTIONSORT(Type A[], int n)
{
    Type item;
    int i;
    for (int j = 1; j < n; j++)
    {
        item = A[j];             // 取出待插入元素
        i = j - 1;				 // i是已排好序的部分的最大角标
        while (item < A[i] && i > -1) do//如果该元素与A[i]逆序
        {
            A[i+1] = A[i];      // 过了判定说明item要插在A[i]前面，所以相当于A[i]要后移一位
            i = i - 1;			// 继续看已排好序的次大角标
        }
        A[i+1] = item;          // 插入排好序的值
    }
}
```

### 时间复杂度分析

- while 循环中的语句可能执行 **0 ~ j** 次（j = 1, 2, …, n-1）
- **最坏情况**（逆序数组）：
  ```
  移动次数 = 1 + 2 + … + (n-1) = n(n+1)/2 - 1 = Θ(n²)
  ```
- **最好情况**（已排好序）：
  while 循环根本不进入，计算时间是 **Ω(n)**

| 情况 | 时间复杂度 | 何时发生 |
|------|----------|---------|
| 最好 | Ω(n) | 输入已经有序 |
| 最坏 | Θ(n²) | 输入完全逆序 |
| 平均 | Θ(n²) | 随机输入 |

> ❌ 插入排序最坏 O(n²)，对大规模数据太慢。能否做到 O(n log n)？→ **MergeSort！**

---

## 第4页：MergeSort 算法框架

### 算法步骤

1. **分割**：将数组 A[1..n] 一分为二，分别复制到：
   - B[1..⌊n/2⌋]（左半）和 C[1..⌈n/2⌉]（右半）

2. **递归排序**：分别对 B 和 C 进行 MergeSort

3. **合并（Merge）**：将已排好序的 B 和 C 合并回数组 A：
   - 重复以下操作，直到其中一个数组的元素全部用完：
     - 比较 B 和 C 中当前未处理部分的**第一个元素**
     - 将**较小者**复制到 A 中，并将该数组的指针后移一位
   - 当某个数组的元素全部处理完后，将另一个数组的**剩余元素**直接追加到 A 中

### 分治三步

| 步骤 | 操作 | 说明 |
|------|------|------|
| **Divide** | 将 A 分成两半 B 和 C | O(n) 复制 |
| **Conquer** | 递归排序 B 和 C | 2T(n/2) |
| **Merge** | 将有序的 B 和 C 合并回 A | O(n) |

### 直觉

```
      [38, 27, 43, 3, 9, 82, 10]
              /              \
    [38, 27, 43, 3]    [9, 82, 10]
      /        \          /      \
  [38, 27]  [43, 3]   [9, 82]  [10]
   /   \     /   \     /   \     |
 [38] [27] [43] [3]  [9] [82]  [10]
   \   /     \   /     \   /     |
  [27, 38]  [3, 43]   [9, 82]  [10]
      \        /          \      /
    [3, 27, 38, 43]    [9, 10, 82]
              \              /
      [3, 9, 10, 27, 38, 43, 82]
```

> 类比：一堆考卷分成两摞，分别排好序后再**交叉合并**——两个有序堆合并成一个有序堆很快（每次只需比较两堆最顶上的那张）。

---

## 总结

```
MergeSort 核心思想：

  ┌── Divide：一分为二
  ├── Conquer：递归排序两半
  └── Merge：O(n) 合并两个有序子数组

递推关系：T(n) = 2T(n/2) + O(n)
解：T(n) = O(n log n) ← 比插入排序快得多！

对比：
  InsertionSort: O(n²) 最坏
  MergeSort:     O(n log n) 始终  ← 稳定高效
```
