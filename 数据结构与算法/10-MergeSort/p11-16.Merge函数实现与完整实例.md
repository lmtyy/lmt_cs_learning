# Merge 函数实现与完整实例（第11-16页）

Merge 函数的代码实现细节，以及一个 10 元素数组的完整归并排序过程追踪。

## 第11页：合并函数 Merge 的实现思路

### 内存结构

```
数组A（原地存储两段已排好序的数据）：

┌──────────────────────────┬──────────────────────────┐
│  已分类序列A（前半）       │  已分类序列B（后半）       │
│ A(0) A(1) … A(⌊(n-1)/2⌋)│ A(⌊(n-1)/2⌋+1) … A(n-1) │
└──────────────────────────┴──────────────────────────┘
       ↓ 比较大小                    ↓
       小值放入 ↘                    剩余已分类元素直接追加
                ↘
辅助数组B：
┌──────┬──────┬──────────────────────────────────────┐
│ A(0) │A(⌊..⌋+1)│  ……                                │
└──────┴──────┴──────────────────────────────────────┘
```

### Merge 函数需要的参数

| 参数 | 含义 |
|------|------|
| `a[]` | 原数组（包含两段已排序的数据） |
| `b[]` | 目标数组（合并结果存放处） |
| `l` | a 前段的起始位置 |
| `m` | a 前段的终止位置（则后段起始为 m+1） |
| `r` | a 后段的终止位置 |

## 第12页：Merge 合并函数代码

```cpp
template<class Type>
void Merge(Type a[], Type b[], int l, int m, int r)
{ // 合并 a[l:m] 和 a[m+1:r] 到 b[l:r]
    int i = l;       // 前半段指针
    int j = m + 1;   // 后半段指针
    int k = l;       // 数组b的下标指针

    // 把 a[l:m] 和 a[m+1:r] 中的数据进行比较，按顺序放入 b[]
    while ((i <= m) && (j <= r))
        if (a[i] <= a[j])  b[k++] = a[i++];
        else               b[k++] = a[j++];

    // 处理剩余元素
    if (i > m)  // 前段用完，把后段剩余直接追加
        for (int q = j; q <= r; q++)
            b[k++] = a[q];
    else        // 后段用完，把前段剩余直接追加
        for (int q = i; q <= m; q++)
            b[k++] = a[q];
}
```

### 代码逻辑图解

```
前半段指针 i → a[l:m]
后半段指针 j → a[m+1:r]
目标指针   k → b[l:r]

while 两段都没用完：
  ├── a[i] <= a[j] → b[k] = a[i], i++, k++
  └── a[i] > a[j]  → b[k] = a[j], j++, k++

某段用完后：
  ├── 前段用完(i>m) → 后段剩余全部追加到 b
  └── 后段用完(j>r) → 前段剩余全部追加到 b
```

> ✅ Merge 的时间复杂度：每个元素恰好被放入 b 一次 → **O(n)**（n = r - l + 1）

## 第13-14页：归并分类算法实例

### 原始数组

```
[310, 285, 179, 652, 351 | 423, 861, 254, 450, 520]
         左半（5个）          右半（5个）
```

### 左半排序过程（递归分解 → 逐步合并）

圈中数字表示分割点位置：

```
初始：  310  285 | 179  652  351
         ④   ③     ②       ①      ← 分割点标记

第1步：  285  310 | 179  652  351     ← 合并(310,285)→[285,310]

第2步：  285  310 | 179 | 652  351    ← 合并(652,351)→[351,652]

第3步：  179  285  310 | 652  351     ← 合并([285,310], 179)→[179,285,310]

第4步：  179  285  310 | 351  652     ← 合并([652],[351])→[351,652]（已在上步）

第5步：  179  285  310  351  652      ← 合并[179,285,310]和[351,652]
```

### 右半排序过程（同理）

```
423  861 | 254  450  520
→ 反复递归调用，合并
→ 254  423  450  520  861
```

### 最终合并

```
左半已排序：  179  285  310  351  652
右半已排序：  254  423  450  520  861
                    ↓ 合并
最终结果：    179  254  285  310  351  423  450  520  652  861  ✅
```

## 总结

```
Merge 函数核心逻辑：

void Merge(a, b, l, m, r):
  i=l, j=m+1, k=l
  ┌── while(i<=m && j<=r): 比较放入 b
  ├── 前段用完 → 后段剩余追加
  └── 后段用完 → 前段剩余追加

MergeSort 递归树（10个元素）：
  深度 = ⌈log₂n⌉ = 4 层
  每层合并总工作量 = O(n)
  总复杂度 = O(n) × O(log n) = O(n log n)

调用顺序：
  先一路向左递归到底（分解）
  再从最小子问题开始逐层向上合并
```
