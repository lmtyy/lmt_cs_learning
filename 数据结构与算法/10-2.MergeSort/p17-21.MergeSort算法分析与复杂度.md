# MergeSort — 算法分析与复杂度（第17-21页）

归并排序的时间复杂度推导、最优性证明，以及优缺点总结。

---

## 第17页：MergeSort 的效率总结

- 所有情况（最好/平均/最坏）效率相同：**Θ(n log n)**
- 空间需求：**Θ(n)**（**不是**原地排序，需要辅助数组）
- 可以不用递归实现（自底向上的迭代版本）

> ✅ MergeSort 是少数"任何输入都是 O(n log n)"的排序算法——不会像 QuickSort 那样在最坏情况退化到 O(n²)。

---

## 第18页：算法分析 — 递推关系

用 T(n) 表示归并排序所用的时间。

Merge 和 Copy 所用时间可在 O(n) 时间内完成，即与 n 成正比：**cn**，其中 c 是一个正数。

### 递推关系

```
         ┌ a                n = 1   （只有1个元素，常数时间）
T(n) =   │
         └ 2T(n/2) + cn    n > 1   （两个子问题 + 合并代价）
```

| 项 | 含义 |
|---|------|
| `a` | base case 的常数时间 |
| `2T(n/2)` | 递归排序两个规模为 n/2 的子数组 |
| `cn` | Merge + Copy 的线性代价 |

---

## 第19页：求解递推关系

设 n 是 2 的方幂：n = 2^k，直接推导：

```
T(n) = 2T(n/2) + cn
     = 2[2T(n/4) + cn/2] + cn
     = 4T(n/4) + cn + cn
     = 4T(n/4) + 2cn
     = 4[2T(n/8) + cn/4] + 2cn
     = 8T(n/8) + 3cn
     = ……
     = 2^k · T(1) + k·cn
     = 2^k · a + k·cn
     = an + cn·log₂n       （因为 k = log₂n）
```

> ✅ **T(n) = an + cn·log n = O(n log n)**

### 各层工作量分析

```
第0层（根）：       1 个问题 × cn      = cn
第1层：           2 个问题 × c(n/2)  = cn
第2层：           4 个问题 × c(n/4)  = cn
  ……
第k层（叶子）：    n 个问题 × c(1)    = cn

共 k+1 = log₂n + 1 层，每层 cn
→ 总计 = cn·(log₂n + 1) = O(n log n)
```

---

## 第20页：MergeSort 是渐进最优的

### 排序问题的时间下界

对于基于比较的排序算法，理论下界为：

```
T(n) ≥ (n/2 - 1) · log(n/2) = Ω(n log n)
```

### 结论

> 所以归并排序算法是**渐进最优算法**（asymptotically optimal）。

即：在基于比较的排序中，不可能存在比 O(n log n) 更快的算法。MergeSort 已经达到了这个理论下界。

| 算法 | 最坏复杂度 | 是否最优 |
|------|---------|---------|
| InsertionSort | O(n²) | ❌ |
| MergeSort | O(n log n) | ✅ 渐进最优 |
| QuickSort | O(n²) 最坏 | ❌（但平均 O(n log n)） |
| HeapSort | O(n log n) | ✅ 渐进最优 |

---

## 第21页：MergeSort 的缺点

> 该算法需要**线性的额外空间**。虽然合并也能做到"在位"（in-place），但会导致算法过于复杂，而且因为它的增长次数具有一个很大的常数系数，所以在位的合并排序算法只有**理论上的意义**。

### 优缺点对比

| 方面 | MergeSort | 说明 |
|------|-----------|------|
| ✅ 时间复杂度 | O(n log n)（所有情况） | 最坏也不退化 |
| ✅ 稳定性 | **稳定排序** | 相等元素不改变相对顺序 |
| ✅ 最优性 | 渐进最优 | 达到比较排序下界 |
| ❌ 空间复杂度 | O(n) | 需要辅助数组 b[] |
| ❌ 不是原地排序 | 需要额外空间 | 在位实现复杂且常数大 |
| ❌ 缓存不友好 | 随机访问两个数组 | 实际性能可能不如 QuickSort |

---

## 总结

```
MergeSort 复杂度全景：

时间：
  递推关系：T(n) = 2T(n/2) + cn
  解：T(n) = O(n log n) — 最好/平均/最坏 都一样

空间：O(n) — 需要辅助数组

最优性证明：
  比较排序下界 = Ω(n log n)
  MergeSort 达到 O(n log n)
  → 渐进最优 ✅

实际应用场景：
  ✅ 适合：外部排序（磁盘数据）、链表排序、需要稳定性的场景
  ❌ 不适合：内存紧张、需要原地排序的场景（这时选 QuickSort/HeapSort）
```
