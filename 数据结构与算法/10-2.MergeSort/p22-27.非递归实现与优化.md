# MergeSort — 非递归实现与优化（第22-27页）

介绍自底向上（迭代）的归并排序实现，以及归并排序的优化改进方向。

---

## 第22页：非递归实现（自底向上）

### 核心思想

不用递归分解，而是**从最小单元开始，逐层向上合并**：

```
初始序列（每个元素自成一段）：
  [5] [2] [4] [6] [1] [3] [2] [6]

第1轮（合并长度 s=1，两两合并）：
  [2,5] [4,6] [1,3] [2,6]

第2轮（合并长度 s=2，四四合并）：
  [2,4,5,6] [1,2,3,6]

第3轮（合并长度 s=4，整体合并）：
  [1,2,2,3,4,5,6,6]  ← sorted!
```

> 从底部单个元素开始，每轮**合并长度翻倍**，直到整个数组有序。

## 第23页：非递归实例 — 排序 {20,25,50,25*,90,60,70,10,30,15,55}

（其中 25* 表示第二个 25，用于观察稳定性）

### 逐轮合并过程

| 轮次 | 合并长度 s | 数组状态 | 备注 |
|------|---------|--------|------|
| 初始 | — | 20 25 50 25* 90 60 70 10 30 15 **55** | 55 是落单元素 |
| s=1 | 1 | 20 25 \| 25* 50 \| 60 90 \| 10 70 \| **15 30** \| **55** | 两两合并，55与前一段合并 |
| s=2 | 2 | 20 25 25* 50 \| 10 60 70 90 \| **15 30 55** | 四四合并，末尾3个落单 |
| s=4 | 4 | 10 20 25 25* 50 60 70 90 \| **15 30 55** | 八八合并，末尾3个落单 |
| s=8 | 8 | 10 20 15 25 25* 30 50 55 60 70 90 | 最终合并 |

### 关键边界情况

| 情况 | 处理方式 |
|------|---------|
| 剩余元素 **< s** 个 | 直接放入下次排序，不参加本轮合并 |
| 剩余元素 **≥ s 且 < 2s** | 与前面的 s 个合并（不等长合并） |
| 合并长度每次**翻倍** | s = 1 → 2 → 4 → 8 → … |

> ⚠️ 图中紫色元素（如 15, 30, 55）就是落单/不等长的情况，需要特殊处理。

## 第24页：消去递归的合并描述（伪代码）

### 外层逻辑

```
输入：待排序数组 a[]，数组长度 n
输出：排好序的数组 a[]

s = 1            // s 为记录每次合并的数组长度
while (s < n)
{
    两两合并 a 中排好序的长度为 s 的数组段，
    并处理不能参加合并的元素；
    s += s;      // 合并长度翻倍
}
```

### 内层逻辑（一趟合并 MergePass）

```
输入：待排序数组 x[]，存储结果数组 y[]，子数组长度 s，总长度 n
输出：数组 y[]

i = 0            // i 为记录合并数组段的起点位置
while (i <= n - 2*s)
{
    找到要合并的两段数组的起止位置：
      前段：(i, i+s-1)
      后段：(i+s, i+2s-1)
    调用 Merge 函数排序两段数组
    i = i + 2s   // 待合并数组段起始位置改变
}

// 处理剩下的小于 2s 的元素
if (i + s < n)
    合并 (i, i+s-1) 和 (i+s, n-1) 的数组  // 剩下的元素大于s小于2s
else
    将未参加合并的元素直接赋给 y         // 剩下的元素不到s个
```

## 第25页：非递归 MergeSort 完整代码

### 主函数

```cpp
template <class Type>
void MergeSort(Type a[], int n)
{
    Type *b = new Type[n];
    int s = 1;
    while (s < n)
    {
        MergePass(a, b, s, n);   // a 合并到 b
        s += s;                   // 长度翻倍
        MergePass(b, a, s, n);   // b 合并回 a
        s += s;                   // 长度再翻倍
    }
}
```

### 一趟合并函数 MergePass

```cpp
template <class Type>
void MergePass(Type x[], Type y[], int s, int n)
{ // 合并大小为 s 的相邻子数组
    int i = 0;
    while (i <= n - 2*s)
    {
        Merge(x, y, i, i+s-1, i+2*s-1);//参数是原数组、辅助数组、左段起点、左段终点、右段终点
        i = i + 2*s;
    }
    
    // 剩下的元素个数大于 s，而小于 2s 时
    if (i + s < n)
        Merge(x, y, i, i+s-1, n-1);
    // 剩下的元素个数不到 s 个时
    else
        for (int j = i; j <= n-1; j++)
            y[j] = x[j];
}
```

### 关键设计

| 设计点 | 说明 |
|--------|------|
| 两次 MergePass 交替 | a→b 再 b→a，避免每轮都 copy 回来 |
| s 翻倍两次 | 因为做了两次 MergePass（a→b 和 b→a） |
| 边界处理 | 末尾不足 2s 的元素需要特殊处理 |

## 第26页：非递归版的 Merge 合并函数

```cpp
template <class Type>
void Merge(Type c[], Type d[], int l, int m, int r)
{ // 合并 c[l:m] 和 c[m+1:r] 到 d[l:r]
    int i = l;
    int j = m + 1;
    int k = l;
    
    while ((i <= m) && (j <= r))
        if (c[i] <= c[j]) d[k++] = c[i++];
        else              d[k++] = c[j++];
    
    // 把 c[l:m] 和 c[m+1:r] 中的数据进行比较，按顺序放入 d[] 中
    if (i > m)
        for (int q = j; q <= r; q++)
            d[k++] = c[q];
    else
        for (int q = i; q <= m; q++)
            d[k++] = c[q];
}
```

> 与递归版的 Merge 完全相同，只是参数名从 (a,b) 变成了 (c,d)。

## 第27页：归并排序的缺点与优化方向

### 两个值得商榷的地方

#### 缺点一：分解直到只有一个元素

> 事实上，当元素比较少时，直接进行排序（比如用**插入排序**），比起进一步分拆、合并排序要快得多。因为在这种情况下，大量的时间都花在调用分解、合并函数上。

**优化方法**：对于归并起点的规模应该有适当的限制，即加 **Small(p,q)** 判断——当子数组长度小于某个阈值时（如 16），直接用插入排序。

```cpp
// 优化：小规模时直接用插入排序
if (right - left < THRESHOLD)
    InsertionSort(a, left, right);
else
    // 正常 MergeSort 流程
```

#### 缺点二：辅助数组 B 的借用

> 虽然不可避免，但可以采用另一种方式，以避免数组 A 中的元素的频繁换位。

**优化方法**：交替使用 a 和 b 作为源/目标（非递归版本已经这样做了：a→b, b→a 交替）。

## 总结

```
非递归 MergeSort vs 递归 MergeSort：

┌─────────────┬──────────────┬──────────────┐
│             │ 递归版        │ 非递归版      │
├─────────────┼──────────────┼──────────────┤
│ 方向        │ 自顶向下      │ 自底向上      │
│ 实现方式    │ 递归调用栈    │ while 循环    │
│ 空间开销    │ O(n)+O(logn)栈│ O(n)         │
│ 合并长度    │ 由递归决定    │ s=1,2,4,8…  │
│ 边界处理    │ 自动          │ 需手动处理    │
│ 性能        │ 相同 O(nlogn) │ 相同 O(nlogn) │
└─────────────┴──────────────┴──────────────┘

优化方向：
  ① 小规模子数组用 InsertionSort（减少递归开销）
  ② a/b 交替使用（减少 copy 次数）
  ③ 检查是否已有序再决定是否合并（跳过无用操作）
```
