本节给出装载问题的完整回溯算法代码（类结构、MaxLoading入口、Backtrack递归），并引入上界函数进一步优化剪枝。

## 第32页：2. 算法描述 — 类结构

```cpp
template<class Type>
class Loading {
    friend Type MaxLoading(Type[], Type, int);
  private:
    void Backtrack(int i);
    int n;        // 集装箱数
    Type *w,      // 集装箱重量数组
         c,       // 第一艘轮船的载重量
         cw,      // 当前载重量（已选集装箱的重量之和）
         bestw;   // 当前最优载重量（搜索过程中找到的最大可行cw）
};
```

### 函数职责

| 函数 | 作用 |
|------|------|
| `MaxLoading` | 负责该类成员的初始化，返回不超过 c 的最大子集和（但未给出达到这个最大子集和的相应子集是哪些） |
| `Backtrack(i)` | 实现回溯搜索。`Backtrack(1)` 实现对整个解空间的回溯搜索；`Backtrack(i)` 搜索子集树中第 i 层子树 |

## 第33页：MaxLoading 入口函数

```cpp
template<class Type>
Type MaxLoading(Type w[], Type c, int n)
{ // 返回最优载重量
    Loading<Type> x;
    x.w = w;
    x.c = c;
    x.n = n;
    x.bestw = 0;
    x.cw = 0;
    // 计算最优载重量
    x.Backtrack(1);
    return x.bestw;
}
```

> ✅ 初始化后从第1层开始回溯搜索，最终返回 bestw（第一艘船能装的最大重量）。

## 第34页：Backtrack 递归逻辑（伪代码说明）

```
Backtrack(int i)
{ // 搜索第i层结点

    // 如果到达叶结点，则判断当前的cw，如果
    // 比前面得到的最优解bestw好，则替换原最优解。

    // 搜索左子树（选第i个物品）
    // 如果当前剩余空间可以放下当前物品，也
    // 就是，cw + w[i] <= c，则把当前载重 cw
    // += w[i]，递归访问其左子树，
    // Backtrack(i+1)，访问结束，回到调用点，
    // cw -= w[i]（恢复现场）

    // 搜索右子树（不选第i个物品）
    // 直接 Backtrack(i+1);
}
```

> ✅ 左子树需要判断约束（装得下才能选），右子树永远可行（不选不会超重）。

## 第35页：Backtrack 完整代码

```cpp
template<class Type>
void Loading<Type>::Backtrack(int i)
{ // 搜索第i层结点
    if (i > n) {  // 到达叶结点
        if (cw > bestw) bestw = cw;
        return;
    }
    // 搜索左子树（x[i]=1，选第i个集装箱）
    if (cw + w[i] <= c) {
        cw += w[i];
        Backtrack(i + 1);
        cw -= w[i];       // 恢复现场
    }
    // 搜索右子树（x[i]=0，不选第i个集装箱）
    Backtrack(i + 1);
}
```

### 关键点

- **左子树**：有约束检查 `cw + w[i] <= c`（装得下才递归进入）
- **右子树**：无条件进入（不选当前物品永远可行，不会导致超重）
- 到达叶结点时更新 bestw

### 搜索树示例（w={16,15,15}, c=30）

```
              i=1     (0)
                   1/     \0
         i=2    (16)       (0)
              1/   \0    1/   \0
       i=3 (31)✗  (16)  (15)  (0)
             剪枝  0/\0  1/\0  1/\0
       i=4      (16)(30)(15)(15)(0)
                       ↑最优解
```

> ✅ **右子树永远可行！** 因为不选物品不会增加重量，所以右分支不需要约束检查。

## 第36页：上界函数（限界剪枝）

### 问题

仅用约束函数剪枝还不够——右子树虽然可行，但如果**即使把剩余物品都装里也不会比当前最优解更好**，那这棵右子树也没必要搜索。

### 上界函数定义

引入一个**上界函数**，用于剪去不含最优解的子树。设 Z 是解空间树第 i 层上的当前扩展结点：

| 变量 | 含义 |
|------|------|
| `cw` | 当前载重量（已选物品的重量和） |
| `bestw` | 当前最优载重量 |
| `r` | 剩余集装箱的重量和（第 i+1 到第 n 个物品的总重） |

**上界函数** = cw + r

> 含义：在以 Z 为根的子树中，任一结点所相应的载重量**均不超过** cw + r（即使剩下的全选也就这么多）。

### 剪枝条件

$$cw + r \le bestw$$

当上界 ≤ 已知最优解时，说明这棵子树里不可能产生更优的解 → **剪去 Z 的右子树**。

### 搜索树示例（带上界剪枝）

```
         i=1     (0)
               1/     \0
      i=2    (16)       (0)
           1/   \0    1/   \0
    i=3  (31)✗  (16)  (15)  (0) ← r=0, cw+r=0≤bestw? 视情况剪
           剪枝  \0   1/\0
    i=4        (16) (30)(15)
                     ↑红色标记=被上界剪掉的结点
```

> ✅ 约束函数剪**左**子树（装不下），上界函数剪**右**子树（没前途）。两者配合可以大幅减少搜索空间。
