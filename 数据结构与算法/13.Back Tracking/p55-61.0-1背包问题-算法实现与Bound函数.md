- w = {1, 2, 3, 5}
- p = {4, 7, 9, 10}

| 层 | 操作 | cw | cp | 上界检查 |

- 右子树上界=19 < bestp=20 → **剪枝**

- 右子树上界=20 = bestp=20（不大于bestp）→ **剪枝**

## 第56页：2 算法描述 — Knap 类结构

```cpp
template<class Typew, class Typep>
class Knap {  // 背包描述
    friend Typep Knapsack(Typep*, Typew*, Type, int);
  private:
    Typep Bound(int i);       // 计算第i层结点的上界
    void Backtrack(int i);
    Typew c;                  // 背包容量
    int n;                    // 物品数
    Typew *w;                 // 物品重量数组
    Typep *p;                 // 物品价值数组
    Typew cw;                 // 当前重量
    Typep cp;                 // 当前价值
    Typep bestp;              // 当前最优价值
};
```

### 函数职责

| 函数 | 作用 |
|------|------|
| `Bound(i)` | 计算第 i 层结点的上界（用贪心松弛法估算右子树中还能获得的最大价值） |
| `Backtrack(i)` | 实现回溯搜索。`Backtrack(1)` 搜索整个解空间；`Backtrack(i)` 搜索子集树中第 i 层子树 |

## 第57页：Object 辅助类（用于排序）

```cpp
class Object
{  // 物品
    friend int Knapsack(int *, int *, int, int);
  public:
    int operator <= (Object a) const
        { return (d >= a.d); }   // 重载≤运算符，使排序为降序
  private:
    int ID;         // 物品原始编号
    float d;        // 单位重量价值（p[i]/w[i]）
};
```

> ✅ Object 类封装了物品的 ID 和单位价值 d = p/w，重载 ≤ 运算符使 Sort 按单位价值**降序**排列。排序后物品顺序变了，但通过 ID 可以还原。

## 第58页：Knapsack 入口函数

```cpp
template<class Typew, class Typep>
Typep Knapsack(Typep p[], Typew w[], Typew c, int n)
{ // 为 Knap::Backtrack 初始化
    Typew W = 0;
    Typep P = 0;
    Object *Q = new Object[n];

    for (int i = 1; i <= n; i++) {
        Q[i-1].ID = i;
        Q[i-1].d = 1.0 * p[i] / w[i];   // 计算单位价值
        P += p[i];
        W += w[i];
    }

    if (W <= c) return P;   // 所有物品都装得下，直接返回总价值

    // 依物品单位重量价值排序（降序）
    Sort(Q, n);

    Knap<Typew, Typep> K;
    K.p = new Typep[n + 1];
    K.w = new Typew[n + 1];

    for (int i = 1; i <= n; i++) {
        K.p[i] = p[Q[i-1].ID];   // 按排序后的顺序重新映射价值
        K.w[i] = w[Q[i-1].ID];   // 按排序后的顺序重新映射重量
    }

    K.cp = 0;
    K.cw = 0;
    K.c = c;
    K.n = n;
    K.bestp = 0;

    // 回溯搜索
    K.Backtrack(1);

    delete[] Q;
    delete[] K.w;
    delete[] K.p;
    return K.bestp;
}
```

### 关键步骤解读

| 步骤 | 说明 |
|------|------|
| 计算 W | 总重量，如果 W≤c 说明全能装下，直接返回 |
| 计算 d = p/w | 单位价值，用于排序 |
| Sort(Q, n) | 按单位价值降序排列（使 Bound 函数的贪心策略最有效） |
| 重新映射 K.p[], K.w[] | 排序后物品顺序变了，搜索时按新顺序进行 |

> ✅ 排序是预处理，只做一次。排序后 Bound 函数可以直接从当前位置往后贪心装入，无需额外排序。

## 第59页：Backtrack 递归代码

```cpp
template<class Typew, class Typep>
Typep Knap<Typew, Typep>::Backtrack(int i)
{
    if (i > n) {          // 到达叶结点
        bestp = cp;
        return;
    }
    if (cw + w[i] <= c) {   // x[i]=1，搜索左子树
        cw += w[i];
        cp += p[i];
        Backtrack(i + 1);
        cw -= w[i];          // 恢复现场
        cp -= p[i];
    }
    if (Bound(i + 1) > bestp)   // x[i]=0，搜索右子树（上界剪枝）
        Backtrack(i + 1);
}
```

### 逻辑要点

- **左子树**（选物品 i）：约束检查 `cw + w[i] <= c`，通过则选入并递归
- **右子树**（不选物品 i）：计算 `Bound(i+1)`（从第 i+1 个物品开始的上界），若上界 > bestp 才值得搜索
- 到达叶结点时直接更新 bestp = cp（因为上界函数保证了能到达的叶子一定优于旧 bestp）

> ✅ 注意 Bound 的参数是 `i+1`，因为物品 i 已经决定不选了，要从第 i+1 个开始估算剩余价值。

## 第60页：Bound 上界函数

```cpp
template<class Typew, class Typep>
Typep Knap<Typew, Typep>::Bound(int i)
{  // 计算上界
    Typew cleft = c - cw;      // 剩余容量
    Typep b = cp;              // 从当前价值开始累加

    // 以物品单位重量价值递减序装入物品（已排好序，直接顺序装）
    while (i <= n && w[i] <= cleft) {
        cleft -= w[i];
        b += p[i];
        i++;
    }

    // 装满背包（最后一个装不下的物品取分数部分）
    if (i <= n) b += p[i] / w[i] * cleft;

    return b;
}
```

### 逻辑解读

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | `cleft = c - cw` | 计算背包剩余容量（总容量 - 已选物品重量） |
| 2 | `b = cp` | 上界从当前已获价值开始（已选物品的价值不会丢失） |
| 3 | while 循环 | 贪心地装入后续物品（因为已按单位价值降序排列，直接顺序装就是最优贪心） |
| 4 | 分数装入 | 最后一个装不下的物品按比例取一部分填满剩余容量 |
| 5 | return b | 返回上界值（分数背包的最优解） |

> ✅ 这就是分数背包的贪心解：整个能装就整个装，最后一个按比例切。由于 0-1 背包不允许切割，分数背包解 ≥ 0-1 背包最优解，所以它是合法上界。

### 示例（延续 p53 的数据）

n=4, c=7, 排序后 w={1,2,3,5}, p={4,7,9,10}

假设当前在根结点（cp=0, cw=0）：
- cleft = 7, b = 0
- 装物品1: cleft=6, b=4
- 装物品2: cleft=4, b=11
- 装物品3: cleft=1, b=20
- 物品4: w[4]=5 > cleft=1 → 分数装入: b += 10/5 × 1 = 2
- 上界 = 22

## 第61页：算法效率

由于计算上界函数 Bound 需要 **O(n)** 时间，在最坏情况下有 **O(2ⁿ)** 个右儿子结点需要计算上界函数，故解 0-1 背包问题的回溯算法 Backtrack 所需的计算时间为 **O(n·2ⁿ)**。

> ✅ 虽然最坏情况是 O(n·2ⁿ)，但实际中由于剪枝效果，搜索的结点数远小于 2ⁿ，算法通常表现良好。

## 总结

```
0-1背包回溯算法完整结构：

┌─────────────────────────────────────────────────────┐
│  预处理（Knapsack 入口）：                           │
│    1. 计算每个物品的单位价值 d = p[i]/w[i]          │
│    2. 按 d 降序排序                                  │
│    3. 重新映射 p[], w[] 数组                         │
├─────────────────────────────────────────────────────┤
│  搜索（Backtrack）：                                 │
│    • 左子树：约束检查 cw+w[i]≤c                     │
│    • 右子树：上界检查 Bound(i+1)>bestp              │
│    • 叶结点：bestp = cp                              │
├─────────────────────────────────────────────────────┤
│  上界计算（Bound）：                                 │
│    1. 从当前状态 (cp, cw) 出发                       │
│    2. 贪心装入后续物品（已排好序）                   │
│    3. 最后一个物品按分数装入                         │
│    4. 返回分数背包最优值作为上界                     │
├─────────────────────────────────────────────────────┤
│  时间复杂度：O(n·2ⁿ)                                │
│  预处理排序：O(n log n)                              │
└─────────────────────────────────────────────────────┘
```
