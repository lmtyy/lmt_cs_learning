# 蒙特卡罗方法估算回溯效率与课后思考（p86-91）

本节介绍使用蒙特卡罗（Monte Carlo）方法估算回溯法将要产生的结点数，并以 8 皇后问题为例演示估算过程。最后附一道课后思考题。

---

## 第86页：蒙特卡罗 (Monte Carlo) 方法

- 回溯法的有效性在于当问题实例的规模 n 很大时，它能够使用很少的时间求出问题的解。
- 使用蒙特卡罗方法**估算回溯法将要产生的节点数**。

### 主要思想

在解空间树上产生一条**随机的路径**，然后沿此路径来**估算**解空间中满足约束条件的节点总数 **m**。

> ✅ 蒙特卡罗法不是用来求解问题的，而是用来**预估**回溯法的搜索代价（搜索多少个结点）。在决定是否用回溯法之前，可以先用蒙特卡罗法估计一下工作量。

---

## 第87页：随机路径的生成过程

设 x 是所产生随机路径上的结点，位于解空间树的第 i 层上。

1. 对于 x 的所有儿子结点，用约束函数检测出**满足约束条件的节点数目 mᵢ**
2. 路径上的下一个结点是从 x 的 mᵢ 个满足约束函数的儿子结点中**随机选取**的
3. 这条路径一直延伸到一个叶结点，或者一个所有儿子结点都不满足约束条件的节点为止
4. 通过这些 mᵢ 的值，就可以估算出解空间树中满足约束条件的节点数 m

> ✅ 每次走到一个结点就数一数"有几个儿子能通过约束检查"，记为 mᵢ。走完一条随机路径后，用这些 mᵢ 来估算整棵树的规模。

---

## 第88页：结点总数估算公式

在静态约束下，如果第1层有 m₀ 个满足约束条件的节点，若解空间树的同一层节点具有相同的出度，则：

- 第1层上每个节点平均有 **m₁** 个儿子节点满足约束条件
- 第2层有 **m₀ · m₁** 个满足约束条件的节点
- 第3层上有 **m₀ · m₁ · m₂** 个

因此，结点总数 m：

$$
m = 1 + m_0 + m_0 m_1 + m_0 m_1 m_2 + \cdots
$$

> ✅ 公式含义：第0层1个根结点 + 第1层 m₀ 个 + 第2层 m₀m₁ 个 + ... 逐层累乘。每条随机路径给出一个 (m₀, m₁, m₂, ...) 的样本，多次采样取平均可以得到较好的估计。

---

## 第89页：8后问题解空间的结点总数

8后问题（不带约束剪枝时）解空间的结点总数：

$$
1 + \sum_{j=0}^{7} \left( \prod_{i=0}^{j} (8-i) \right) = 109601
$$

（即 1 + 8 + 8×7 + 8×7×6 + ... + 8! = 109601 个结点，根节点1+第一层8+第二层8\*7...）

> ✅ 这是完全展开的排列树结点数。加入约束函数后实际搜索的结点数远小于此值，蒙特卡罗法就是用来估算剪枝后实际会访问多少个结点。

## 第90页：例 — 估算8后问题的 m 值

### 随机路径1

在 8×8 棋盘上随机放置皇后，每层检查有多少个合法位置：

```
第1行：8个位置可选 → m₀ = 8
第2行：5个位置可选 → m₁ = 5
第3行：4个位置可选 → m₂ = 4
第4行：3个位置可选 → m₃ = 3
第5行：2个位置可选 → m₄ = 2
（路径在此终止或到达叶子）
```

序列 (8, 5, 4, 3, 2)，估算 m = 1+8+8×5+8×5×4+8×5×4×3+8×5×4×3×2 = **1649**

### 随机路径2

```
第1行：8个位置可选 → m₀ = 8
第2行：5个位置可选 → m₁ = 5
第3行：3个位置可选 → m₂ = 3
第4行：2个位置可选 → m₃ = 2
第5行：2个位置可选 → m₄ = 2
第6行：1个位置可选 → m₅ = 1
第7行：1个位置可选 → m₆ = 1
第8行：1个位置可选 → m₇ = 1
```

序列 (8, 5, 3, 2, 2, 1, 1, 1)，估算 m = **2329**

> ✅ 不同的随机路径给出不同的估算值。多次采样取平均可以得到更准确的估计。实际8皇后问题回溯搜索约访问 114 个结点（远小于 109601），说明剪枝效果非常好。

## 第91页：课后思考题 — 海盗分赃问题

### 题目

6个海盗要分赃300金币。规则是由资格最老的海盗提出各人分到的数量，然后全体投票。如方案得到至少半数同意票，则按该方案执行，否则提出方案的海盗被杀死，再由剩下人中资格最老的继续提出方案。海盗都很聪明，在能生存的前提下会追求获利最大化。

**问最后分赃结果是怎样的？**

### 解题方法：逆向归纳法

从人数最少的情况开始分析，逐步推回6人的情况。设海盗编号为1-6，1号资格最老。

**规则要点：**
- 提案者自己也有投票权
- "至少半数同意"即 ≥ 总人数/2 票即可通过（含提案者自己那票）
- 海盗优先保命，其次追求利益最大化

---

**只剩2人（5号和6号）：**
- 5号提案，需要 ≥ 1 票（自己投自己就够了）
- 5号方案：(300, 0) → 5号拿全部，6号拿0
- 6号无论怎么投都改变不了结果

**只剩3人（4号、5号、6号）：**
- 4号提案，需要 ≥ 2 票（需要再拉1票）
- 如果4号死了 → 回到2人情况，6号拿0
- 所以4号只需给6号1个金币，6号就会同意（1 > 0）
- 4号方案：(299, 0, 1)

**只剩4人（3号、4号、5号、6号）：**
- 3号提案，需要 ≥ 2 票（需要再拉1票）
- 如果3号死了 → 回到3人情况，5号拿0
- 所以3号只需给5号1个金币
- 3号方案：(299, 0, 1, 0)

**只剩5人（2号、3号、4号、5号、6号）：**
- 2号提案，需要 ≥ 3 票（需要再拉2票）
- 如果2号死了 → 回到4人情况，4号拿0，6号拿0
- 所以2号给4号1个金币、6号1个金币，就能拉到2票
- 2号方案：(298, 0, 1, 0, 1)

**6人（1号、2号、3号、4号、5号、6号）：**
- 1号提案，需要 ≥ 3 票（需要再拉2票）
- 如果1号死了 → 回到5人情况，3号拿0，5号拿0
- 所以1号给3号1个金币、5号1个金币，就能拉到2票
- 1号方案：**(298, 0, 1, 0, 1, 0)**

---

### 最终答案

| 海盗（资格老→新） | 1号 | 2号 | 3号 | 4号 | 5号 | 6号 |
|---|---|---|---|---|---|---|
| 分得金币 | **298** | 0 | 1 | 0 | 1 | 0 |

> ✅ 1号（资格最老）提出方案 (298, 0, 1, 0, 1, 0)，获得自己+3号+5号共3票通过（≥3票）。
>
> 核心逻辑：提案者只需要用最小代价"收买"那些在下一轮中拿不到任何好处的海盗。给他们1个金币就够了（1 > 0），因为如果当前提案被否决，他们在下一轮什么都得不到。
