# 装载问题-上界函数实现与构造最优解（p37-44）

本节给出带上界函数的改进回溯算法完整代码，并进一步扩展为能构造最优解（记录选择方案）的版本。

---

## 第37页：3. 上界函数

- 在改进算法中，引入类 Loading 的一个私有变量 **r**，用于计算上界函数。引入上界函数后，在达到一个叶结点时就不必再检查该叶结点是否优于当前最优解。
- 即若遍历到某处时，即使把剩下的物品全部装上（而不考虑载重），也达不到最优，则该枝要被剪掉
- 因为上界函数使算法搜索到的每个叶结点都是迄今为止找到的最优解，时间复杂性没有变化，但在平均情况下改进后的算法检查的结点数较少。

> ✅ 有了上界剪枝，能到达叶子的路径一定比 bestw 更好——所以到达叶子时直接更新 bestw = cw，不用再比较。

## 第38页：改进后的类结构

```cpp
Template<class Type>
class Loading {
    friend Type MaxLoading(Type[], Type, int);
  private:
    void Backtrack(int i);
    int n;        // 集装箱数
    Type *w,      // 集装箱重量数组
         c,       // 第一艘轮船的载重量
         cw,      // 当前载重量
         bestw,   // 当前最优载重量
         r;       // 剩余集装箱重量（第i+1到第n个物品的重量和）
};
```

> ✅ 相比之前的版本，多了一个变量 `r`（剩余重量），用来计算上界 cw + r。

## 第39页：改进后的 MaxLoading

```cpp
template<class Type>
Type MaxLoading(Type w[], Type c, int n)
{ // 返回最优载重量
    Loading<Type> x;
    x.w = w;
    x.c = c;
    x.n = n;
    x.bestw = 0;
    x.cw = 0;
    x.r = 0;
    for (int i = 1; i <= n; i++)
        x.r += w[i];   // 初始时 r 为全体物品的重量和
    // 计算最优载重量
    x.Backtrack(1);
    return x.bestw;
}
```

> ✅ 初始化时 r = 所有集装箱的总重量。随着搜索深入，每进入第 i 层时先 `r -= w[i]`（第 i 个物品不再属于"剩余"），回溯时再 `r += w[i]` 恢复。

## 第40页：改进后的 Backtrack 逻辑（伪代码）

```
Backtrack(int i)
{ // 搜索第i层结点

    // 如果到达叶结点，bestw = cw。
    //（不需要比较，因为上界函数保证了能到达的叶子一定优于旧bestw）

    // 修改剩余重量r
    r = r - w[i]

    // 搜索左子树
    // 如果当前剩余空间可以放下当前物品，也就是 cw + w[i] <= c，
    // 则把当前载重 cw += w[i]，递归访问其左子树 Backtrack(i+1)，
    // 访问结束，回到调用点，cw -= w[i]。

    // 搜索右子树
    // 如果剩余重量加上当前载重量大于最优值（cw + r > bestw），
    // 递归访问右子树 Backtrack(i+1)。
    //（否则说明即使剩余全选也不如bestw，剪掉右子树）

    // 回到上一层调用点
    r = r + w[i]   // 恢复r,恢复r在右子树后，因为同层遍历中，一个物品只会被遍历一次，除非上层变化，即回溯，否则该物品遍历后，若选择不放入，则在该层中不会再次遍历该物品
}
```

## 第41页：改进后的 Backtrack 完整代码 + 搜索示例

```cpp
template<class Type>
void Loading<Type>::Backtrack(int i)
{ // 搜索第i层结点
    if (i > n) {  // 到达叶结点
        bestw = cw;
        return;
    }
    // 搜索子树
    r -= w[i];
    if (cw + w[i] <= c) {   // x[i]=1，搜索左子树
        cw += w[i];
        Backtrack(i + 1);
        cw -= w[i];
    }
    if (cw + r > bestw) {   // x[i]=0，搜索右子树（上界剪枝）
        Backtrack(i + 1);
    }
    r += w[i];              // 恢复r
}
```

### 搜索示例（w={16, 15, 15}, c=30）


## 第42页：构造最优解 — 扩展类结构

为了记录最优解对应的具体方案（哪些集装箱被选中），需要增加变量：

```cpp
Template<class Type>
class Loading {
    friend Type MaxLoading(Type[], Type, int, int[]);
  private:
    void Backtrack(int i);
    int n;        // 集装箱数
    int *x,       // 当前解（x[i]=1表示选第i个，x[i]=0表示不选）
        *bestx;   // 当前最优解（记录最优方案）
    Type *w,      // 集装箱重量数组
         c,       // 第一艘轮船的载重量
         cw,      // 当前载重量
         bestw,   // 当前最优载重量
         r;       // 剩余集装箱重量
};
```

> ✅ 新增 `x[]`（当前搜索路径上的选择）和 `bestx[]`（到达叶子时把最优方案拷贝过来）。

## 第43页：构造最优解的 MaxLoading

```cpp
template<class Type>
Type MaxLoading(Type w[], Type c, int n, int bestx[])
{ // 返回最优载重量及最优解
    Loading<Type> X;
    X.x = new int[n + 1];
    X.w = w;
    X.c = c;
    X.n = n;
    X.bestx = bestx;
    X.bestw = 0;
    X.cw = 0;
    X.r = 0;
    for (int i = 1; i <= n; i++)
        X.r += w[i];   // 初始时 r 为全体物品的重量和
    // 计算最优载重量
    X.Backtrack(1);
    delete[] X.x;
    return x.bestw;
}
```

> ✅ 新增参数 `bestx[]` 用于输出最优方案。函数结束后 bestx[i]=1 表示第 i 个集装箱装上第一艘船。

## 第44页：构造最优解的 Backtrack

```cpp
template<class Type>
void Loading<Type>::Backtrack(int i)
{ // 搜索第i层结点
    if (i > n) {  // 到达叶结点
        for (int j = 1; j <= n; j++)
            bestx[j] = x[j];    // 记录当前最优解方案
        bestw = cw;
        return;
    }
    // 搜索子树
    r -= w[i];
    if (cw + w[i] <= c) {       // 搜索左子树
        x[i] = 1;
        cw += w[i];
        Backtrack(i + 1);
        cw -= w[i];
    }
    if (cw + r > bestw) {       // 搜索右子树（上界剪枝）
        x[i] = 0;
        Backtrack(i + 1);
    }
    r += w[i];
}
```

### 与之前版本的区别

| 之前（只求最优值） | 现在（构造最优解） |
|-------------------|-------------------|
| 到达叶子：`bestw = cw` | 到达叶子：拷贝 `x[]` → `bestx[]`，再 `bestw = cw` |
| 搜索左子树：`cw += w[i]` | 搜索左子树：`x[i] = 1; cw += w[i]` |
| 搜索右子树：直接递归 | 搜索右子树：`x[i] = 0;` 再递归 |

> ✅ 核心思想：在搜索过程中用 x[] 记录当前路径上的每一步决策（选1/不选0），到达叶子且更优时就把整个 x[] 拷贝到 bestx[] 中保存。
