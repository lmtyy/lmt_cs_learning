## 第3页：回溯法概述

有许多问题，当需要找出它的**解集**或者要求回答什么解是满足某些约束条件的**最佳解**时，往往要使用回溯法。

可以系统地搜索一个问题的**所有解或任意解**，既有**系统性**，又有**跳跃性**。

> 类比：回溯法就像走迷宫——沿一条路走到底，走不通就退回岔路口换一条路再试。"系统性"指不会漏掉任何岔路，"跳跃性"指一旦发现死路就立即掉头，不浪费时间。

## 第5页：例 — n后问题

在一个 n×n 的棋盘上放置 n 个皇后，要求每两个之间都不能相互"攻击"，即使得任两个皇后都**不在同一行、同一列及同一条斜线上**。

### 4皇后问题示意（4×4棋盘）

```
┌───┬───┬───┬───┐
│   │ 1 │   │   │    皇后1在第1行第2列
├───┼───┼───┼───┤
│   │   │   │ 2 │    皇后2在第2行第4列
├───┼───┼───┼───┤
│ 3 │   │   │   │    皇后3在第3行第1列
├───┼───┼───┼───┤
│   │   │ 4 │   │    皇后4在第4行第3列
└───┴───┴───┴───┘
```

> ✅ 这是 4 皇后问题的一个合法解：任意两个皇后不同行、不同列、不同对角线。

---

## 第6页：4皇后回溯求解过程（步骤 a-h）

逐步展示如何通过回溯法找到解：

| 步骤 | 操作 | 说明 |
|------|------|------|
| a | 皇后1 → (1,1) | 从第1行第1列开始尝试 |
| b | 皇后2 → (2,3) | 第2行第1、2列被攻击（●），放第3列 |
| c | 第3行全冲突 | 皇后2在(2,3)时，第3行所有列均被攻击 → **死路** |
| d | 回溯，皇后2 → (2,4)；皇后3 → (3,2) | 皇后2换到第4列，皇后3放第2列 |
| e | 第4行全冲突 | 皇后3在(3,2)时，第4行所有列均被攻击 → **死路**，皇后3放后面两格分别会与1、2冲突，故回溯到皇后2，皇后2已到末尾，故回溯到皇后1 |
| f | 回溯到根，皇后1 → (1,2) | 皇后1在第1列的所有分支穷尽，换到第2列 |
| g | 皇后2 → (2,4) | 第2行第1-3列被攻击，放第4列 |
| h | 皇后3 → (3,1)，皇后4 → (4,3) | 逐行放置成功 → **找到解！** |

> ✅ 最终解：(1,2), (2,4), (3,1), (4,3)

## 第7页：状态空间搜索树

以 4 皇后为例，展示完整的回溯搜索树：

```
                        (0) 空棋盘
                       /           \
              (1) Q₁在第1列        (5) Q₁在第2列
              /        \              /          \
         X(col1,2)  (2)Q₂在第3列  X(col1,2,3) (6)Q₂在第4列
                     /       \                      |
                X(col1-4)  (3)Q₂在第4列         (7)Q₃在第1列
                            /       \               |       \
                       X(col1-4) (4)Q₃在第2列   X(col1)  (8)Q₄在第3列
                                    |                      ✓ SOLUTION
                                X(col1-4)
```

### 搜索过程解读

| 节点 | 含义 |
|------|------|
| 0 | 根节点，空棋盘 |
| 1-4 | 皇后1在第1列的搜索分支（全部失败） |
| 5-8 | 皇后1在第2列的搜索分支 |
| X | 剪枝：该位置违反约束，不再往下搜索 |
| ✓ | 节点8：找到合法解 |

> 类比：整棵树有 4⁴ = 256 个叶子节点（暴力枚举），但通过剪枝只访问了 8 个内部节点就找到了解，效率大幅提升。

## 总结

```
回溯法核心思想：

┌─────────────────────────────────────────────────┐
│  1. 解空间：将问题的解表示为向量 (x₁,x₂,...,xₙ) │
│  2. 搜索策略：深度优先遍历状态空间树              │
│  3. 剪枝：遇到不满足约束的节点立即回溯           │
│     - 约束函数：判断当前节点是否可行              │
│     - 限界函数：判断当前节点是否有可能导向最优解  │
└─────────────────────────────────────────────────┘

效果：避免穷举所有可能，大幅减少搜索空间
```
