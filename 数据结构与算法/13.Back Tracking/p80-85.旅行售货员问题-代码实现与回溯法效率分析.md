# 旅行售货员问题-代码实现与回溯法效率分析（p80-85）

本节给出 TSP 回溯算法的完整代码实现（Traveling 类、Backtrack 递归），并总结回溯法的效率分析和重排原理。

---

## 第80页：Traveling 类结构

```cpp
template<class Type>
class Traveling
{
    friend Type TSP(int **, int[], int, Type);
  private:
    void Backtrack(int i);
    int n,          // 图G的顶点数
        *x,         // 当前解（x[i]表示第i个访问的城市编号）
        *bestx;     // 当前最优解
    Type **a,       // 图G的邻接矩阵（a[i][j]=边权，无边则为NoEdge）
         cc,        // 当前费用（已走过的路径总费用）
         bestc,     // 当前最优值（最小总费用）
         NoEdge;    // 无边标记（表示两城市间无直接连接），一般设为很大的值，即若两城市无直接连接，认为两城市的直连距离为无穷（NoEdge）
};
```

## 第81页：Backtrack 代码 — i==n 部分（叶结点的父结点）

```cpp
template <class Type>
void Traveling<Type>::Backtrack(int i)
{
    if (i == n)  // 最后一个节点
    {
        if (a[x[n-1]][x[n]] != NoEdge && a[x[n]][1] != NoEdge &&
            (cc + a[x[n-1]][x[n]] + a[x[n]][1] < bestc || bestc == NoEdge))
        {
            for (int j = 1; j <= n; j++)
                bestx[j] = x[j];
            bestc = cc + a[x[n-1]][x[n]] + a[x[n]][1];
        }
    }
```

### 逻辑解读

| 检查条件 | 含义 |
|---------|------|
| `a[x[n-1]][x[n]] != NoEdge` | 倒数第二个城市到最后一个城市有路（最后一段路存在） |
| `a[x[n]][1] != NoEdge` | 最后一个城市能回到出发城市1（回程路存在） |
| `cc + ... < bestc` | 完整路线费用比当前最优值小（找到更优解） |
| `bestc == NoEdge` | 还没找到过任何可行解（第一次找到解时直接记录） |

> ✅ i==n 时排列已确定，只需检查最后两段路（x[n-1]→x[n] 和 x[n]→x[1]）是否存在且总费用更优。

## 第82页：Backtrack 代码 — i<n 部分（内部结点）

```cpp
    else
    {
        for (int j = i; j <= n; j++)  // 是否可进入x[j]子树
            if (a[x[i-1]][x[j]] != NoEdge &&
                (cc + a[x[i-1]][x[j]] < bestc || bestc == NoEdge))
            {
                // 搜索子树
                Swap(x[i], x[j]);
                cc += a[x[i-1]][x[i]];    // 更新费用
                Backtrack(i + 1);          // 进入下一层
                cc -= a[x[i-1]][x[i]];    // 回溯（恢复费用）
                Swap(x[i], x[j]);         // 恢复排列
            }
    }
}
```

### 逻辑解读

| 步骤 | 操作 | 说明 |
|------|------|------|
| `for j=i to n` | 遍历所有可选的下一个城市 | 排列树中位置 i 可以放 x[i..n] 中的任何一个 |
| `a[x[i-1]][x[j]] != NoEdge` | 约束剪枝 | 上一个城市到候选城市必须有边（有路可走） |
| `cc + a[x[i-1]][x[j]] < bestc` | 限界剪枝 | 部分路径费用已经 ≥ bestc 则没必要继续（不可能更优） |
| `Swap(x[i], x[j])` | 选择：把城市 x[j] 放到位置 i | 排列树的标准操作 |
| `cc += a[x[i-1]][x[i]]` | 累加费用 | 加上从上一城市到当前城市的边权 |
| `Backtrack(i+1)` | 递归深入 | 继续选下一个城市 |
| `cc -= ...` + `Swap(...)` | 恢复现场 | 回溯：撤销费用累加和排列交换 |

> ✅ `bestc == NoEdge` 的判断很重要：第一次搜索时还没有可行解，此时即使 cc 很大也应该继续搜索（否则永远找不到第一个解）。

## 第83页：算法效率

- Backtrack 最坏情况下需要 **O((n-1)!)** 时间
- 更新 bestx 需要 **O(n)** 时间
- 算法计算时间为 **O(n!)**

> ✅ TSP 的回溯法时间复杂度为 O(n!)，因为排列树有 (n-1)! 个叶子，每个叶子处拷贝 bestx 需要 O(n)，合计 O(n·(n-1)!) = O(n!)。

## 第84页：回溯法效率分析（通用总结）

通过前面具体实例的讨论容易看出，回溯算法的效率在很大程度上依赖于以下因素：

| 编号 | 因素 |
|------|------|
| (1) | 产生 x[k] 的时间（生成每个候选值的代价） |
| (2) | 满足显约束的 x[k] 值的个数（每层的分支因子） |
| (3) | 计算约束函数 constraint 的时间 |
| (4) | 计算上界函数 bound 的时间 |
| (5) | 满足约束函数和上界函数约束的所有 x[k] 的个数（通过剪枝后实际保留的分支数） |

### 折衷问题

好的约束函数能**显著地减少**所生成的结点数。但这样的约束函数往往**计算量较大**。因此，在选择约束函数时通常存在**生成结点数**与**约束函数计算量**之间的折衷。

> ✅ 类比：一把锁越精密防盗能力越强，但开锁（检查）也越费时间。回溯法的剪枝函数也是如此——太简单则剪枝少，太复杂则每次检查代价高。关键是找到平衡点。

## 第85页：重排原理

对于许多问题而言，在搜索试探时选取 x[i] 的值顺序是任意的。

**重排原理：** 在其它条件相当的前提下，让**可取值最少的 x[i] 优先**。

### 示例对比

```
图(a): 宽解空间树           图(b): 窄解空间树
○─○─○─○─○─○─○─○─○        ○───○───○
│ │ │ │ │ │ │ │ │        │   │   │
○ ○ ○ ○ ○ ○ ○ ○ ○       ○─○─○ ○─○ ○
│ │ │ ... (很多)          │ │ │ │ │ │
○ ○ ○ ○ ○ ○ ○ ○ ○       ○○○ ○○  ○ ...
```

- 图(a)中，从第1层剪去1棵子树，则从所有应当考虑的3元组中一次消去 **12** 个3元组
- 图(b)中，虽然同样从第1层剪去1棵子树，却只从应当考虑的3元组中消去 **8** 个3元组
- 前者的效果明显比后者好

> ✅ 直觉：越靠近树根处分支越多，在根部剪一刀能砍掉更多叶子。所以应该让分支最多（约束最紧、可选值最少）的层放在上面，剪枝效果最好。这就是"重排原理"的本质——优先处理约束最紧的变量。
