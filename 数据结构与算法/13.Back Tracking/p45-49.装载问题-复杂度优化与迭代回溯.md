# 装载问题-复杂度优化与迭代回溯（p45-49）

本节讨论构造最优解时的复杂度优化策略，并给出装载问题的非递归迭代回溯完整代码及搜索过程演示。

---

## 第45页：构造最优解的复杂度优化

### 问题

bestx 可能被更新 O(2ⁿ) 次，每次拷贝整个数组需 O(n)，算法计算时间复杂性为 **O(n·2ⁿ)**。

### 两种改进策略

**(1) 两阶段法：** 首先运算只计算最优值的算法，计算出最优装载量 W，所需计算时间为 O(2ⁿ)，然后再运行改进后的 backtrack，并在算法中将 bestw 置为 W。

> ✅ 第一遍只求值不记录方案（快），第二遍带着已知最优值搜索时剪枝更狠，最终只到达一个叶子就能记录方案。

**(2) 动态更新 bestx：** 在第 i 层的当前结点处，当前最优解由 x[j] (j < i) 和 bestx[j] (i ≤ j ≤ n) 组成。每当算法回溯一层时，将 x[i] 存入 bestx[i]。这样一来，在每个结点处更新 bestx 只需 **O(1)** 时间，从而算法中跟新 bestx 所需的时间为 **O(2ⁿ)**。

> ✅ 不用到叶子才拷贝整个数组，而是每回溯一步就保存一个元素，均摊 O(1)。

## 第46页：迭代回溯 — 初始化

非递归的方式可以省去 **O(n)** 的递归栈空间。

```cpp
template<class Type>
Type MaxLoading(Type w[], Type c, int n, int bestx[])
{ // 迭代回溯
    int i = 1;          // 当前层
    // x[1:i-1] 为当前路径
    int *x = new int[n + 1];
    Type bestw = 0,     // 当前最优装载重量
         cw = 0,        // 当前载重量
         r = 0;         // 剩余集装箱重量
    for (int j = 1; j <= n; j++)
        r += w[j];      // 初始 r = 全体物品总重
```

## 第47页：迭代回溯 — 主循环（完整代码）

```cpp
    // 搜索子集树
    while (true) {
        while (i <= n && cw + w[i] <= c) {//无脑左子树，碰到叶子再考虑回头
            // 进入左子树（选第i个物品）
            r -= w[i];
            cw += w[i];
            x[i] = 1;
            i++;
        }
        if (i > n) {  // 到达叶结点
            for (int j = 1; j <= n; j++)
                bestx[j] = x[j];
            bestw = cw;
        }
        else {  // 进入右子树（不选第i个物品，因为左子树不可行）
            r -= w[i];
            x[i] = 0;
            i++;
        }

        // 剪枝回溯
        while (cw + r <= bestw) {
            i--;
            while (i > 0 && !x[i]) {
                // 从右子树返回（之前x[i]=0的层直接跳过）
                r += w[i];
                i--;
            }
            if (i == 0) {
                delete[] x;
                return bestw;  // 搜索结束
            }
            // 进入右子树（之前选了x[i]=1，现在改为不选）
            x[i] = 0;
            cw -= w[i];
            i++;
        }
    }
}
```

### 代码逻辑解读

| 阶段 | 操作 | 说明 |
|------|------|------|
| 搜索左子树 | `while(i<=n && cw+w[i]<=c)` 循环 | 能选就一直选（沿左子树一路深入，贪心地往下走） |
| 到达叶结点 | `if(i>n)` | 左子树走到底了，记录解 |
| 进入右子树 | `else` 分支 | 左子树走不通（装不下），不选当前物品 |
| 剪枝回溯 | `while(cw+r<=bestw)` | 上界不够好，需要回溯到能产生更优解的结点 |
| 从右子树返回 | `while(i>0 && !x[i])` | 跳过所有"不选"的层（这些层没有可尝试的替代方案） |
| 终止 | `if(i==0)` | 回溯到根节点之上，整棵树搜索完毕 |

## 第48页：迭代回溯搜索过程演示（上半部分）

w = {16, 15, 15}, c = 30

### 阶段1：沿左子树深入

```
i=1: cw+w[1]=16≤30 ✓ → r=30, cw=16, x[1]=1, i=2
i=2: cw+w[2]=31>30 ✗ → 退出左子树循环
     i=2 ≤ n=3，进入右子树: r=15, x[2]=0, i=3
i=3: cw+w[3]=31>30 ✗ → 退出左子树循环
     i=3 ≤ n=3，进入右子树: r=0, x[3]=0, i=4
i=4 > n → 到达叶结点: bestx={1,0,0}, bestw=16
```

## 第49页：迭代回溯搜索过程演示（下半部分）

### 阶段2：剪枝回溯 + 继续搜索

```
检查: cw+r=16+0=16 ≤ bestw=16 → 需要回溯

i--: i=3, x[3]=0 → 从右子树返回, r+=w[3]=15, i=2
     i=2, x[2]=0 → 从右子树返回, r+=w[2]=30, i=1
     i=1, x[1]=1 → 停止（找到之前选了1的层）
     进入右子树: x[1]=0, cw-=w[1]=0, i=2

检查: cw+r=0+30=30 > bestw=16 → 可以继续搜索

i=2: cw+w[2]=15≤30 ✓ → r=15, cw=15, x[2]=1, i=3
i=3: cw+w[3]=30≤30 ✓ → r=0, cw=30, x[3]=1, i=4
i=4 > n → 到达叶结点: bestx={0,1,1}, bestw=30

检查: cw+r=30+0=30 ≤ bestw=30 → 需要回溯
i--: i=3, x[3]=1 → 停止
     进入右子树: x[3]=0, cw-=w[3]=15, i=4
检查: cw+r=15+0=15 ≤ bestw=30 → 继续回溯
i--: i=3, x[3]=0 → 从右子树返回, r+=15, i=2
     i=2, x[2]=1 → 停止
     进入右子树: x[2]=0, cw-=15=0, i=3
检查: cw+r=0+15=15 ≤ bestw=30 → 继续回溯
i--: i=2, x[2]=0 → 从右子树返回, r+=15=30, i=1
     i=1, x[1]=0 → 从右子树返回, r+=16=46, i=0
     i==0 → 搜索结束！

返回 bestw = 30, bestx = {0, 1, 1}
```

> ✅ 最终最优解：不选第1个集装箱（16），选第2、3个（15+15=30），第一艘船装满。

## 总结

```
装载问题迭代回溯：

┌─────────────────────────────────────────────────────┐
│  复杂度优化（两种策略）：                            │
│    ① 两阶段法：先求值O(2ⁿ)，再带bestw构造解        │
│    ② 动态更新bestx：每回溯一步存一个，O(1)均摊      │
├─────────────────────────────────────────────────────┤
│  迭代回溯结构：                                      │
│    while(true) {                                     │
│      沿左子树深入（能选就选）                        │
│      到达叶结点 → 记录解                            │
│      或进入右子树（选不了就跳过）                    │
│      剪枝回溯 while(cw+r<=bestw) {                  │
│        跳过右子树返回层                              │
│        找到上一个x[i]=1的层，改为x[i]=0             │
│        若i==0则搜索结束                              │
│      }                                               │
│    }                                                 │
├─────────────────────────────────────────────────────┤
│  优势：省去O(n)递归栈空间                            │
│  代价：代码复杂度增加                                │
└─────────────────────────────────────────────────────┘
```
