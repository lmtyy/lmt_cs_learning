# 生成问题状态与回溯法基本思想（p13-17）

本节介绍解空间树中结点的三种状态（扩展结点、活结点、死结点），对比深度优先与宽度优先的问题状态生成法，并给出回溯法的正式定义与基本思想。

---

## 第13页：生成问题状态的基本方法

### 三种结点状态

| 术语 | 定义 |
|------|------|
| **扩展结点** | 一个正在产生儿子的结点称为扩展结点（正在遍历它的儿子） |
| **活结点** | 一个自身已生成但其儿子还没有全部生成的节点称做活结点（自己的儿子尚未全部遍历完，当回溯到自己时会变成扩展结点） |
| **死结点** | 一个所有儿子已经产生的结点称做死结点（它的儿子已经全部遍历完了） |

### 示意图（搜索过程中的某一瞬间）

```
                    ○ ← 活结点
                 1/   \0
         ● ← 扩展结点    C
            1/ \0       1/ \0
     ● ← 死结点  E     F    G
         1/\0  1/\0  1/\0  1/\0
        ●  ● (J) K  L  M  N  O
```

- ● 实心 = 死结点（儿子已全部生成）
- ○ 浅色 = 活结点（部分儿子尚未生成）
- 当前扩展结点正在生成它的下一个儿子

> ✅ 在搜索过程中，同一时刻只有一个扩展结点，但可能有多个活结点。

---

## 第14页：深度优先的问题状态生成法

**深度优先的问题状态生成法：** 如果对一个扩展结点 R，一旦产生了它的一个儿子 C，就把 C 当做新的扩展结点。在完成对子树 C（以 C 为根的子树）的穷尽搜索之后，将 R 重新变成扩展结点，继续生成 R 的下一个儿子（如果存在）。

### 搜索顺序示意（完全二叉树）

```
                    ①
                 1/    \0
              ②          ⑨
           1/   \0     1/   \0
         ③      ⑥    ⑩     ⑬
       1/ \0  1/ \0  1/ \0  1/ \0
      ④  ⑤  ⑦  ⑧  ⑪  ⑫  ⑭  ⑮
```

> 访问顺序：沿左子树一路向下，到底后回溯，再进入右子树 — 即标准的 DFS 前序。

---

## 第15页：宽度优先的问题状态生成法

**宽度优先的问题状态生成法：** 在一个扩展结点变成死结点之前，它一直是扩展结点。

### 搜索顺序示意

```
                    ①
                 1/    \0
              ②          ③
           1/   \0     1/   \0
         ④      ⑤    ⑥     ⑦
       1/ \0  1/ \0  1/ \0  1/ \0
      ⑧  ⑨  ⑩  ⑪  ⑫  ⑬  ⑭  ⑮
```

> 一层一层地生成所有儿子后再进入下一层 — 即标准的 BFS。

### 对比

| | 深度优先 | 宽度优先 |
|--|---------|---------|
| 扩展结点切换 | 一产生儿子就切换 | 当前结点所有儿子生成完才切换 |
| 数据结构 | 栈（递归调用栈） | 队列 |
| 空间开销 | O(树深度) | O(树宽度) |

---

## 第16页：回溯法的正式定义

回溯法：为了避免生成那些不可能产生最佳解的问题状态，要不断地利用**限界函数（bounding function）**来处死那些实际上不可能产生所需解的活结点，以减少问题的计算量。

> ✅ **具有限界函数的深度优先生成法称为回溯法。**

### 剪枝示意

```
                    ○
                 1/   \0
            ●         C
         1/ \0       1/ \0
       ●    E       F    G
      1/\0 1/\0   1/\0  1/\0
     ●  ● (J) K  L  M   N  O
```

- 左侧子树被限界函数剪掉（结点变死），不再继续展开
- 搜索跳到右侧子树继续深度优先

---

## 第17页：2. 回溯法的基本思想

### 三步搜索过程

**1.** 从开始结点（根结点）出发，它成为第一个**活结点**，也成为当前的**扩展结点**。

**2.** 在当前的扩展结点处，搜索向**纵深方向**移至一个新结点。这个结点就成为一个新的活结点，并成为当前扩展结点。

**3.** 如果在当前的扩展结点处不能再向纵深方向移动，则当前的扩展结点就成为**死结点**，此时应往回移动（**回溯**）至最近的一个**活结点**处，并使这个活结点成为当前的扩展结点，用这种方式递归地在解空间中搜索，直至找到所要求的解或解空间中已**无活结点**为止。

> ✅ 终止条件：找到解 OR 无活结点（说明问题无解或已遍历完毕）。

---

## 总结

```
回溯法 = DFS + 限界函数（Bounding Function）

关键概念：
┌──────────────┬────────────────────────────────┐
│  扩展结点     │ 当前正在生成儿子的结点          │
│  活结点       │ 已生成，儿子未全部展开          │
│  死结点       │ 儿子已全部展开，或被限界函数剪掉│
├──────────────┼────────────────────────────────┤
│  深度优先     │ 产生一个儿子就立即深入          │
│  宽度优先     │ 当前层全部展开后再进入下层      │
│  回溯法       │ 深度优先 + 限界函数剪枝         │
└──────────────┴────────────────────────────────┘

基本思想（三步循环）：
  展开 → 纵深 → 不可行则回溯到最近活结点 → 重复
```
