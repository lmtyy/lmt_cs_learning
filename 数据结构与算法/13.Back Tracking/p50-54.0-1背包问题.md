# 0-1背包问题（p50-54）

本节介绍用回溯法求解 0-1 背包问题，重点讲解上界函数的设计（贪心松弛法）及其对右子树的剪枝作用。

---

## 第50页：3 0-1背包问题（章节标题）

---

## 第51页：例 — 搜索树与上界函数

### 问题参数

- n=4, c=7（背包容量）
- w = {3, 5, 2, 1}（重量）
- p = {9, 10, 7, 4}（价值）

### 子集树搜索过程

```
                    (0, 0)  ← (当前重量cw, 当前价值cp)
                  1/       \0
            (3, 9)           (0, 0)
          1/    \0          1/    \0
       ✗超重   (3, 9)    (5, 10)   ✗剪枝
              1/   \0     1/    \0
          (5, 16) (7, 17) (5, 16)  ✗剪枝
          1/  \0   1/ \0
       (6, 20) ✗  ✗   ✗
```

结点标注格式：(当前总重, 当前总价值)

- 粉色结点 = 被剪枝
- 左儿子如果可行（cw + w[i] ≤ c），进入左子树
- 只有右子树中有可能包含最优解时才进入右子树搜索，否则将右子树剪去

### 上界函数

**cp + r ≤ bestp** 时剪去右子树

（当前获得价值 + 剩余价值 ≤ 当前最优价值，说明即使把剩余物品全选也不如已知最优解）

---

## 第52页：更紧的上界函数

### 问题

有时即使满足了 cp + r ≤ bestp，但实际上背包并不能装下所有剩余物品，也就是说上界 cp + r **太大了**（太松了）。

### 改进思路：贪心松弛法

计算右子树中解的上界时，将剩余物品**依其单位重量价值排序**，然后依次装入物品，直至装不下时，再装入该物品的**一部分**而装满背包。由此得到的价值是右子树的上界。

> ✅ 这其实就是分数背包问题的贪心解——允许物品切割时的最优值。由于 0-1 背包不允许切割，分数背包的解一定 ≥ 0-1背包的最优解，所以它是一个合法的上界。

如果这个上界不能达到当前得到的最优值，则不搜索该子树。

## 第53页：上界计算示例

### 预处理

以物品**单位重量价值**的递减序装入物品。

| 物品编号 | 重量 w | 价值 p | 单位价值 p/w |
|---------|--------|--------|------------|
| 1 | 3 | 9 | 3 |
| 3 | 2 | 7 | 3.5 |
| 2 | 5 | 10 | 2 |
| 4 | 1 | 4 | 4 |

按单位价值量排序后：{3, 2, 3.5, 4}（对应物品 4, 3, 1, 2）

### 上界计算过程

- n=4, c=7, p={9, 10, 7, 4}, w={3, 5, 2, 1}
- 单位价值量 {3, 2, 3.5, 4}
- 先装入物品4（w=1），然后物品3（w=2），然后物品1（w=3），已用容量=6
- 剩余容量=1，物品2重量5装不下 → 取物品2的 1/5，价值 = 10×(1/5) = 2
- 总上界 = 4 + 7 + 9 + 2 = **22**
- 对应分数解 {1, 0.2, 1, 1}

> ✅ 上界22意味着：在最理想的情况下（允许切割物品），最多能获得价值22。0-1背包的真实最优解 ≤ 22。

## 第54页：上界函数使用规则

### 三条规则

1. 将物品按**单位价值量**排序（预处理，只排一次）
2. 在解空间树的当前扩展结点处，仅当要**进入右子树**时才计算上界函数 Bound，判断是否可以将右子树剪去
3. **进入左子树时，不需要计算上界**（左子树是"选当前物品"，只需检查约束 cw+w[i]≤c 即可）

> ✅ 左子树只需约束检查（装得下就选），右子树才需要上界检查（不选当前物品后还有没有前途）。这跟装载问题的逻辑完全一致。

## 总结

```
0-1背包问题回溯法：

┌─────────────────────────────────────────────────────┐
│  解空间：子集树（每个物品选/不选），2ⁿ 个叶子       │
├─────────────────────────────────────────────────────┤
│  剪枝策略：                                          │
│                                                      │
│  ┌──────────────────┐    ┌──────────────────┐        │
│  │  约束函数（左子树）│    │  上界函数（右子树）│        │
│  │  cw + w[i] > c   │    │  Bound ≤ bestp   │        │
│  │  装不下 → 剪      │    │  没前途 → 剪      │        │
│  └──────────────────┘    └──────────────────┘        │
├─────────────────────────────────────────────────────┤
│  上界函数 Bound 的计算（贪心松弛法）：               │
│    1. 物品按单位价值 p[i]/w[i] 降序排列              │
│    2. 贪心装入：能整个装就装，最后一个装不下时取分数  │
│    3. 得到的总价值 = 分数背包最优值 ≥ 0-1背包最优值  │
│    4. 若 Bound ≤ bestp → 剪去右子树                  │
├─────────────────────────────────────────────────────┤
│  注意：                                              │
│    • 进入左子树：只需约束检查，不算上界              │
│    • 进入右子树：才计算上界 Bound                    │
│    • 预处理排序只做一次                              │
└─────────────────────────────────────────────────────┘
```
