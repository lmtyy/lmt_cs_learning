# 旅行售货商问题与回溯法基本步骤（p18-22）

本节以旅行售货商问题（TSP）为例展示解空间树的另一种形态（排列树），并总结回溯法的基本步骤、剪枝函数分类和空间复杂度特征。

---

## 第18页：例2 — 旅行售货员问题描述

设某售货员要到若干城市去推销商品，已知各城市之间的路程（或路费）。他要选定一条从驻地出发，经过每个城市一遍，最后回到驻地的路线，使**总的路程最小**。

---

## 第19页：旅行售货商问题（TSP）形式化

给定一个有 **n** 个顶点的带权图 G=(V,E)，旅行售货员问题要找出图 G 的费用（权）最小的**周游路线**。

### 示例：4顶点无向带权图

```
        30
   1 ———————— 2
   |\ 5     /|
  6| \    / |10
   |  \ /   |
   3 ———————— 4
       20
   1-3: 6    2-4: 10
   1-4: 5    3-4: 20
   2-3: 4
```

| 边 | 权值 |
|----|------|
| 1-2 | 30 |
| 1-3 | 6 |
| 1-4 | 5 |
| 2-3 | 4 |
| 2-4 | 10 |
| 3-4 | 20 |

### 三条不同的周游路线

| 路线 | 顺序 | 总费用 |
|------|------|--------|
| 路线1 | 1→2→4→3→1 | 30+10+20+6 = 66 |
| 路线2 | 1→3→2→4→1 | 6+4+10+5 = 25 ✅ |
| 路线3 | 1→4→3→2→1 | 5+20+4+30 = 59 |

## 第20页：TSP 的解空间树（排列树）

该问题的解空间可以组织成一棵树，从树的根结点到任一结点的路径定义了图 G 的一条周游路线。

### 解空间树结构

```
            A
            |1
            B (从城市1出发)
         /  |  \
       2/   |3   \4
      C     D     E
    / \   / \   / \
   3/ 4\ 2/ 4\ 2/ 3\
  F   G  H   I  J   K
  |   |  |   |  |   |
 4|  3| 4|  2| 3|  2|
  L   M  N   O  P   Q
```

### 路径解读

| 根到叶路径 | 对应周游路线 |
|-----------|------------|
| A→B→C→F→L | 1→2→3→4→1 |
| A→B→C→G→M | 1→2→4→3→1 |
| A→B→D→H→N | 1→3→2→4→1 |
| A→B→D→I→O | 1→3→4→2→1 |
| A→B→E→J→P | 1→4→2→3→1 |
| A→B→E→K→Q | 1→4→3→2→1 |

> ✅ 对于 n 个城市的 TSP，排列树有 (n-1)! 个叶子结点（固定出发城市后的全排列）。

## 第21页：回溯法的基本步骤

### 三步框架

**(1)** 针对所给问题，定义问题的**解空间**；

**(2)** 确定易于搜索的**解空间结构**；

**(3)** 以**深度优先**方式搜索解空间，并在搜索过程中用**剪枝函数**避免无效搜索。

### 常用剪枝函数

| 类型 | 作用 |
|------|------|
| **约束函数** | 在扩展结点处剪去**不满足约束**的子树 |
| **限界函数** | 剪去**得不到最优解**的子树 |

> 类比：约束函数是"判断合不合法"（可行性剪枝），限界函数是"判断有没有前途"（最优性剪枝）。

---

## 第22页：回溯法的空间特征

用回溯法解题的一个显著特征是在搜索过程中**动态产生**问题的解空间。

### 空间复杂度

- 在任何时刻，算法只保存从根结点到当前扩展结点的路径
- 如果解空间树中从根结点到叶结点的最长路径的长度为 *h(n)*，则：

| 方法 | 空间需求 |
|------|----------|
| 回溯法 | **O(h(n))** — 只存一条路径 |
| 显式存储整个解空间 | O(2^h(n)) 或 O(h(n)!) |

> ✅ 这是回溯法的核心优势之一：不需要显式构造整棵解空间树，只需 O(树高度) 的空间即可完成搜索。

## 总结

```
回溯法基本步骤：

┌─────────────────────────────────────────────────────┐
│  Step 1: 定义解空间                                  │
│  Step 2: 组织解空间结构（树/图）                     │
│  Step 3: DFS + 剪枝函数搜索                          │
├─────────────────────────────────────────────────────┤
│  两类解空间树：                                      │
│    • 子集树：每个元素选/不选 → 2ⁿ 个叶子（0-1背包） │
│    • 排列树：元素的全排列 → n! 个叶子（TSP）         │
├─────────────────────────────────────────────────────┤
│  两类剪枝函数：                                      │
│    • 约束函数 → 可行性剪枝（不合法就剪）            │
│    • 限界函数 → 最优性剪枝（没前途就剪）            │
├─────────────────────────────────────────────────────┤
│  空间优势：                                          │
│    回溯法只需 O(h(n)) 空间（一条路径）               │
│    而非 O(2^h(n)) 或 O(h(n)!)（整棵树）             │
└─────────────────────────────────────────────────────┘
```
