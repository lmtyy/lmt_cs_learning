# 旅行售货员问题-回溯法求解（p74-79）

本节用回溯法求解旅行售货员问题（TSP），介绍排列树的解空间组织方式以及剪枝策略。

---

## 第74页：5 旅行售货员问题 Traveling Salesman Problem

设某售货员要到若干城市去推销商品，已知各城市之间的路程（或路费）。他要选定一条从驻地出发，经过每个城市一遍，最后回到驻地的路线，使**总的路程最小**。

---

## 第75页：旅行售货商问题 — 形式化定义

给定一个有 **n** 个顶点的带权图 G=(V,E)，旅行售货员问题要找出图 G 的费用（权）最小的**周游路线**。

### 示例：4顶点无向带权图

```
        30
   1 ———————— 2
   |\ 5     /|
  6| \    / |10
   |  \ /   |
   3 ———————— 4
       20
```

| 边 | 权值 |
|----|------|
| 1-2 | 30 |
| 1-3 | 6 |
| 1-4 | 5 |
| 2-3 | 4 |
| 2-4 | 10 |
| 3-4 | 20 |

### 三条不同的周游路线

| 路线 | 顺序 | 总费用 |
|------|------|--------|
| 路线1 | 1→2→4→3→1 | 30+10+20+6 = 66 |
| 路线2 | 1→3→2→4→1 | 6+4+10+5 = 25 |
| 路线3 | 1→4→3→2→1 | 5+20+4+30 = 59 |

---

## 第76页：解空间树（排列树）

该问题的解空间可以组织成一棵树，从树的根结点到任一结点的路径定义了图 G 的一条周游路线。

### 排列树结构

```
            A
            |1
            B (固定从城市1出发)
         /  |  \
       2/   3|   \4        ← 第2个访问的城市
      C     D     E
    / \   / \   / \
   3/ 4\ 2/ 4\ 2/ 3\      ← 第3个访问的城市
  F   G  H   I  J   K
  |   |  |   |  |   |
 4|  3| 4|  2| 3|  2|     ← 第4个访问的城市
  L   M  N   O  P   Q
```

| 根到叶路径 | 对应周游路线 | 总费用 |
|-----------|------------|--------|
| A→B→C→F→L | 1→2→3→4→1 | 30+4+20+5=59 |
| A→B→C→G→M | 1→2→4→3→1 | 30+10+20+6=66 |
| A→B→D→H→N | 1→3→2→4→1 | 6+4+10+5=25 ✅ |
| A→B→D→I→O | 1→3→4→2→1 | 6+20+10+30=66 |
| A→B→E→J→P | 1→4→2→3→1 | 5+10+4+6=25 ✅ |
| A→B→E→K→Q | 1→4→3→2→1 | 5+20+4+30=59 |

---

## 第77页：排列树与 Perm 的关系

- 旅行售货员问题的解空间是一棵**排列树**
- 对于排列树的回溯搜索与生成 1, 2, ..., n 的所有排列的递归算法 **Perm** 类似
- 设开始时 x = [1, 2, ..., n]，则相应的排列树由 x[1:n] 的所有排列构成

> ✅ 固定 x[1]=1（出发城市），对 x[2:n] 做全排列，每个排列对应一条从城市1出发经过所有城市再回到城市1的路线。共 (n-1)! 条。

---

## 第78页：算法描述 — 叶结点判断（i=n）

### Backtrack(i) 搜索解空间中第 i 层子树

- `Backtrack(1)`，即根节点，无实际意义，其各子节点代表游历的第一座城市的不同可能
- `Backtrack(2)` 实现对整个解空间的回溯搜索

### 当 i = n 时（到达排列树的叶结点的父结点）

当前扩展节点是排列树的**叶结点的父节点**（只剩最后一个城市未确定，且路线已完全确定）。

检测是否同时存在：
1. 从 x[n-1] 到 x[n] 的边（倒数第二个城市到最后一个城市有路）
2. 从 x[n] 到 x[1] 的边（最后一个城市能回到出发城市）

如果存在这两条边，则找到一个**可行解**。同时计算该费用是否小于当前最优的费用 bestc。如果是，则更新最优值 bestc 和最优解 bestx。

> ✅ i=n 时只需检查：最后一段路和回程路是否存在且总费用更优。

---

## 第79页：算法描述 — 内部结点剪枝（i<n）

### 当 i < n 时（当前扩展节点位于排列树的第 i-1 层）

如果存在从 x[i-1] 到 x[i] 的边（即上一个城市到当前城市有路），则 x[1:i] 构成一条**部分路径**。

计算 x[1:i] 的费用（已走过的路程总和），如果**小于当前最优值** bestc，则算法进入排列树的第 i 层（继续深入搜索）。

否则**剪去该子树**。

> ✅ 两个剪枝条件：
> 1. **约束剪枝**：x[i-1] 和 x[i] 之间没有边（两个城市之间没有路） → 剪
> 2. **限界剪枝**：当前部分路径费用已经 ≥ bestc（已经不可能比当前最优解更好了） → 剪

---

## 总结

```
旅行售货员问题回溯法：

┌─────────────────────────────────────────────────────┐
│  解空间：排列树                                      │
│    • 固定出发城市 x[1]=1                             │
│    • 对 x[2:n] 做全排列                             │
│    • 叶结点数：(n-1)!                                │
├─────────────────────────────────────────────────────┤
│  搜索策略：类似 Perm 全排列生成                      │
│    • swap(x[t], x[i]) 选择 + 递归 + swap恢复        │
├─────────────────────────────────────────────────────┤
│  剪枝条件：                                          │
│                                                      │
│  i < n（内部结点）：                                 │
│    ① 边不存在：x[i-1]→x[i] 无边 → 剪               │
│    ② 费用超限：当前部分路径费用 ≥ bestc → 剪        │
│                                                      │
│  i = n（叶的父结点）：                               │
│    ① 检查 x[n-1]→x[n] 和 x[n]→x[1] 的边是否存在   │
│    ② 计算总费用，若 < bestc 则更新                   │
├─────────────────────────────────────────────────────┤
│  时间复杂度：最坏 O(n!)                              │
│  实际中剪枝可大幅减少搜索量                          │
└─────────────────────────────────────────────────────┘
```
