# 图的m着色问题-算法设计与实现（p69-73）

本节给出图的 m 着色问题的回溯算法设计思路、完整代码（Color 类、mColoring 入口、Ok 约束函数、Backtrack 递归），以及算法效率分析。

---

## 第69页：算法设计

- 设 `Backtrack(i)` 表示搜索解空间中**第 i 层子树**。
- 当 **i > n** 时，表示算法已搜索至一个叶结点，得到一个新的 m 着色方案，因此当前已找到的可 m 着色方案数 **sum 增1**。
- 当 **i ≤ n** 时，当前扩展结点 Z 是解空间中的一个内部结点。该结点有 x[i] = 1, 2, ..., m 共 **m 个儿子结点**。对当前扩展结点 Z 的每一个儿子结点，检查其可行性，并以深度优先的方式递归地对可行子树进行搜索，或剪去不可行子树。

> ✅ 约束检查的含义：给顶点 i 涂颜色 x[i] 后，检查它的所有已着色邻居是否有同色的。有 → 不可行，剪枝；没有 → 可行，继续深入。

## 第70页：算法描述 — Color 类结构

```cpp
class Color
{
    friend int mColoring(int, int, int **);
  private:
    bool Ok(int k);         // 检查颜色可用性（约束函数）
    void Backtrack(int t);  // 回溯搜索
    int n;                  // 图的顶点数
    int m,                  // 可用颜色数
        **a,               // 图的邻接矩阵
        *x;                // 当前解（x[i]=顶点i的颜色）
    long sum;              // 当前已找到的可m着色方案数
};
```

## 第71页：mColoring 入口函数

```cpp
int mColoring(int n, int m, int **a)
{
    Color X;
    // 初始化X
    X.n = n;
    X.m = m;
    X.a = a;
    X.sum = 0;
    int *p = new int[n + 1];
    for (int i = 0; i <= n; i++)
        p[i] = 0;           // 所有顶点初始颜色为0（未着色）
    X.x = p;
    X.Backtrack(1);          // 从第1个顶点开始搜索
    delete[] p;
    return X.sum;            // 返回可着色方案总数
}
```

> ✅ x[i]=0 表示顶点 i 尚未着色。搜索从 Backtrack(1) 开始，即从第1个顶点开始依次着色。

## 第72页：Ok 约束函数 + Backtrack 递归代码

### Ok 函数（检查颜色可用性）

```cpp
bool Color::Ok(int k)
{  // 检查颜色可用性
    for (int j = 1; j <= k - 1; j++)
        if ((a[k][j] == 1)        // 顶点k和顶点j有边相连
            && (x[j] == x[k]))    // 且它们颜色相同
            return false;          // 冲突！该颜色不可用
    return true;                   // 无冲突，颜色可用
}
```

> ✅ Ok(k) 的逻辑：遍历顶点 k 前面所有已着色的顶点 j（j=1 到 k-1），如果 j 和 k 相邻且同色，就返回 false（这个颜色不行）。只检查前面的顶点是因为后面的还没着色。时间复杂的O(n)

### Backtrack 递归代码

```cpp
void Color::Backtrack(int t)
{
    if (t > n)
    {
        sum++;                        // 找到一个合法着色方案
        for (int i = 1; i <= n; i++)
            cout << x[i] << " ";      // 输出该方案
        cout << endl;
    }
    else
        for (int i = 1; i <= m; i++)  // 尝试m种颜色
        {
            x[t] = i;                 // 给顶点t涂颜色i
            if (Ok(t)) Backtrack(t+1); // 可行则递归到下一个顶点
        }
}
```

### 逻辑解读

| 情况 | 操作 |
|------|------|
| `t > n` | 所有顶点都已合法着色 → 方案数+1，输出方案 |
| `t ≤ n` | 对顶点 t 尝试每种颜色 i=1..m，通过 Ok(t) 检查后递归 |

> ✅ 注意这里没有显式的"恢复现场"操作，因为 for 循环中 `x[t] = i` 会在下一次迭代直接覆盖，相当于自动恢复。

## 第73页：算法效率

### 解空间树内结点个数

图的 m 可着色问题的解空间树中内结点个数是：

$$\sum_{i=0}^{n-1} m^i$$

（第0层1个根结点，第1层m个，第2层m²个，...，第n-1层m^(n-1)个）

### 每个内结点的检查代价

对于每个内结点，Ok 检查每个儿子所相应的颜色可用性需要 **O(mn)** 时间。

（m 个儿子 × 每次 Ok 检查遍历前面最多 n-1 个顶点）

### 总时间复杂度

$$
\sum_{i=0}^{n-1} m^i \cdot (mn) = nm \cdot \frac{m^n - 1}{m - 1} = O(nm^n)
$$

> ✅ 最坏情况时间复杂度为 **O(nm^n)**。实际中由于剪枝，搜索的结点数通常远小于 mⁿ。
