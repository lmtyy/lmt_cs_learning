这两页 PPT（幻灯片 93 和 94）提供了一个非常清晰的、逐步的图解，展示了如何使用**自底向上建堆（Bottom-up Build Heap）\**算法将一个无序的数组转化为一个\**最大堆（Max-Heap）**。

请注意，这与前面的最小堆例子不同，现在的规则是：**父节点的值必须大于或等于其所有子节点的值。**

我们来逐步分解这个过程：

### 📄 幻灯片 93：逐步下沉（Percolate Down）过程

这页 PPT 上的四个子图展示了对内部节点进行逐个检查和下沉调整的过程。绿色箭头指示了当前正在被检查/调整的节点。

**初始状态（左上图）：**

这是一棵基于输入数组直接构建的完全二叉树，节点值是无序的。算法从最后一个非叶子节点开始。由于总共有 8 个节点，最后一个非叶子节点是索引为 4 的节点（假设根节点索引为 1），也就是值为 `97` 的节点。

1. **第一步（左上图）：** 绿色箭头指向 `97`。
   - **检查：** 比较 `97` 与其子节点 `70`。
   - **结果：** 因为 $97 > 70$，这部分满足最大堆性质，所以无需交换。
2. **第二步（右上图）：** 绿色箭头向左移动，指向下一个内部节点 `65`。
   - **检查：** 比较 `65` 与其两个子节点 `13` 和 `27`。
   - **结果：** 因为 $65$ 大于 $13$ 且大于 $27$（它大于其中最大的子节点 $27$），满足最大堆性质，无需交换。
3. **第三步（左下图）：** 绿色箭头继续向左，指向节点 `38`。
   - **检查：** 比较 `38` 与其子节点 `97` 和 `76`。
   - **判断：** 最大的子节点是 `97`。由于 $38 < 97$，违反了最大堆规则，必须进行交换。
   - **动作：** 节点 `38` 下沉，节点 `97` 上浮（如红色双向箭头所示）。
   - **后续检查：** `38` 换到原本 `97` 的位置后，现在它有了一个新的子节点 `70`。再次比较：由于 $38 < 70$，再次违反规则，必须继续交换。`38` 继续下沉，`70` 上浮（第二个红色双向箭头）。
4. **第四步（右下图）：** 绿色箭头移动到根节点，指向 `49`。
   - **检查：** 比较 `49` 与其当前的子节点 `97`（刚才换上来的）和 `65`。
   - **判断：** 最大的子节点是 `97`。由于 $49 < 97$，必须交换。
   - **动作：** 节点 `49` 下沉，节点 `97` 成为新的根节点（红色双向箭头）。
   - **后续检查：** `49` 换下来后，现在的子节点是 `70` 和 `76`。最大的是 `76`。由于 $49 < 76$，必须再次交换。`49`继续下沉，`76` 上浮（第二个红色双向箭头）。`49` 最终到达叶子节点位置，此步调整完成。

### 📄 幻灯片 94：最终结果与底层数组映射

这页 PPT 展示了建堆完成后的最终树形结构，并将这种逻辑上的树形结构映射回底层的物理数组。

- **最终的树（右上图）：** 这是经历了上一页所有下沉操作后的最终形态。你可以验证一下：根节点 `97` 大于 `76`和 `65`；`76` 大于 `70` 和 `49`；`65` 大于 `13` 和 `27`；`70` 大于 `38`。这棵树完美地符合了最大堆的定义。
- **底层数组的转变（下半部分）：**
  - **Arbitrary Storage (任意存储):** 这是建堆前的初始数组状态：`[49, 38, 65, 97, 76, 13, 27, 70]`。这正是幻灯片 93 左上角那棵无序树按层序遍历的结果。
  - **Max-Heap (最大堆):** 这是建堆算法执行完毕后的数组状态：`[97, 76, 65, 70, 49, 13, 27, 38]`。这正好对应了右上角那棵完美的树的层序遍历结果。

**总结：** 这两页 PPT 生动地展示了算法是如何在底层数组上“就地（in-place）”操作，通过从后往前的连续“下沉（percolate down）”，高效地将一个混乱的数组重构成一个有序的最大堆的。
