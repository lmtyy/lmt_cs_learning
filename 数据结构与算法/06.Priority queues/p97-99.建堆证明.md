这三页 PPT 提供了一个非常严密的数学证明，完整推导了为什么“自底向上建堆（Build Heap）”的时间复杂度是线性的 $O(n)$。

整个证明的核心逻辑是：**建堆的总时间上限，等于树中所有节点可能下沉的最大深度（即节点的高度）之和。**只要证明所有节点的高度之和是 $O(n)$，就证明了建堆的时间复杂度是 $O(n)$。

以下是对这三页 PPT 的详细拆解：

### 📄 第一页：构建求和公式（image_eb93a5.png）

这页 PPT 设定了数学证明的初始条件，将一棵高度为 $h$ 的完美二叉树（Perfect Binary Tree）中各层节点的高度进行了量化并求和。

- **节点分布规律**：

  - 高度为 $h$ 的节点有 $1$ 个（也就是根节点，或者写成 $2^0$）。
  - 高度为 $h-1$ 的节点有 $2$ 个（第一层子节点，或者写成 $2^1$）。
  - 高度为 $h-2$ 的节点有 $2^2 = 4$ 个。
  - 推广到一般情况：高度为 $h-i$ 的节点，总共有 $2^i$ 个。

- **高度总和公式**：

  基于上述规律，将“每一层的节点个数 $\times$ 该层节点的高度”全部加起来，得到所有节点的高度之和 $S$的公式：

  $$S = \sum_{i=0}^{h} 2^i(h-i)$$

  这是一个典型的**差比数列求和**（等差数列 $h-i$ 与等比数列 $2^i$ 的乘积）。

### 📄 第二页：错位相减法求解公式（image_eb93c3.jpg）

这页 PPT 展示了如何用高中数学中经典的“错位相减法”来解开上一页得出的求和公式。

- **展开原始公式**：

  把 $S$ 完全展开：

  $$S = h + 2(h-1) + 4(h-2) + 8(h-3) + \dots + 2^{h-1}(1)$$

- **乘 $2$ 构造错位公式**：

  将等式两边同乘 $2$，并把每一项向右错开一位对齐：

  $$2S = 2h + 4(h-1) + 8(h-2) + 16(h-3) + \dots + 2^h(1)$$

- **两式相减**：

  用下式减去上式（$2S - S$），也就是相同系数对应的项相减，例如 $2h - 2(h-1) = 2$：

  $$S = -h + 2 + 4 + 8 + \dots + 2^{h-1} + 2^h$$

- **化简等比数列**：

  观察得出，从 $2$ 到 $2^h$ 是一个标准的等比数列求和。根据等比数列求和公式，$2 + 4 + 8 + \dots + 2^h = 2^{h+1} - 2$。

  代入后，最终化简得到完美的闭式解：

  $$S = (2^{h+1} - 1) - (h + 1)$$

### 📄 第三页：结论与大 O 复杂度映射（image_eb93de.png）

最后这页 PPT 将纯数学的推导结果与算法的时间复杂度（大 O 表示法）联系了起来。

- **上限的确立**：

  上一页求出的 $S = (2^{h+1} - 1) - (h + 1)$ 是**完美二叉树**的高度和。而实际建堆时面对的通常是完全二叉树（Complete Tree），它的节点数少于或等于同高度的完美二叉树，因此这个 $S$ 就是实际建堆操作高度和的**严格上限（Upper Bound）**。

- **变量代换证明 $O(n)$**：

  - 在一棵高度为 $h$ 的完全二叉树中，节点总数 $n$ 介于 $2^h$ 和 $2^{h+1} - 1$ 之间。（为了方便理解，我们可以用最大值代入，即令 $n = 2^{h+1} - 1$）。
  - 把 $n$ 代回刚刚求出的公式中：$S = n - (h + 1)$。
  - 因为 $h$（树高）是正数，所以 $n - (h + 1) < n$。
  - 这说明**所有节点的高度之和不仅与 $n$ 成正比，甚至绝对值比 $n$ 还要小**！

- **最终结论**：

  定理明确得出：因为高度和的上限被节点总数 $n$ 给限制住了，所以建堆过程的总运算次数 $S$ 属于 **$O(n)$** 的时间复杂度。
