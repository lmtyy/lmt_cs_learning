这页PPT是操作系统课程中非常经典的一章——**计算机系统结构**，它主要讲述了操作系统底层所依赖的硬件基础。我们可以把这六个知识点拆分为**硬件组件、数据流转、组织方式**三个层面来深入理解。

### 1. Computer System Operation（计算机系统操作）

这是讲计算机硬件和软件是如何协作启动并运行的。

-   **核心概念**：
    -   **引导程序**：计算机通电后，首先执行存储在ROM（只读存储器，如BIOS/UEFI）中的一小段代码，称为引导程序。它负责定位并加载操作系统内核到内存中。
    -   **中断驱动**：操作系统本质上是“中断驱动”的。CPU不会主动去等待输入输出（I/O）完成，而是通过**中断**来响应外部事件（如鼠标移动、键盘敲击、磁盘数据就绪）。
    -   **事件循环**：CPU在执行完当前指令后，会检查是否有中断信号。如果有，则保存当前状态（保存现场），跳转到**中断向量表**对应的处理程序，执行完毕后再恢复状态（恢复现场）。

**补充知识点**：

-   **陷阱**：这是一种由程序主动触发的同步中断（如除零错误、系统调用）。与硬件中断的异步性不同，陷阱是程序“故意”陷入内核以获得服务。
-   **双重模式**：CPU必须有**用户模式**和**内核模式**。系统启动时处于内核模式，加载完OS后，执行用户程序时切换为用户模式。当发生中断或系统调用时，硬件自动将模式切换回内核模式，这是硬件保护机制的基础。

### 2. I/O Structure（输入输出结构）

现代计算机的I/O极其复杂，操作系统必须高效管理。

-   **核心概念**：
    -   **设备控制器**：每个I/O设备（磁盘、网卡、显示器）都有一个电子部件（控制器）负责处理物理细节和缓存数据。
    -   **同步I/O**：CPU发出I/O指令后，陷入等待（忙等），直到I/O完成。这种效率极低。
    -   **异步I/O**：操作系统将I/O请求提交给设备控制器后，CPU不等待，立即去执行其他进程。当设备完成I/O时，通过**中断**通知CPU取走数据。

**补充知识点**：

-   **直接内存访问**：这是I/O结构中的关键机制。如果没有DMA，CPU需要亲自从设备控制器一个一个字节地搬运数据到内存（PIO模式），这会严重占用CPU。DMA允许设备控制器直接与内存交互，传输完成后只发送一个中断给CPU，大幅提升了效率。
-   **端口映射I/O vs 内存映射I/O**：CPU如何与控制器通信？前者通过专用的IN/OUT指令；后者将设备寄存器映射到内存地址空间中，像访问普通内存一样访问设备（更方便，也是现代主流趋势）。

### 3. Storage Structure（存储结构）

-   **核心概念**：
    -   **主存**：CPU可以直接访问的唯一大容量存储区域。它是易失性的（断电丢失）。
    -   **辅助存储**：如磁盘、固态硬盘。它们是**非易失性**的，用于永久保存数据。
    -   **缓存**：为了解决CPU和内存速度不匹配的问题，在CPU芯片内部设置了高速缓存。

**补充知识点**：

-   **寄存器 vs 缓存 vs 内存**：
    -   **寄存器**：位于CPU内部，存取速度最快（纳秒级），成本高，容量小（KB级）。
    -   **缓存**：通常分为L1、L2、L3。L1速度接近寄存器，L3稍慢但容量更大（MB级）。
    -   **内存**：即DRAM，速度慢于缓存（百纳秒级），容量大（GB级）。
-   **磁盘 vs 固态硬盘**：磁盘是机械结构（寻道时间毫秒级），固态硬盘是电子结构（微秒级）。现代操作系统（如Linux）在I/O调度算法上对二者有完全不同的优化策略。

### 4. Storage Hierarchy（存储层次体系）

这是计算机体系结构设计的核心思想。

-   **核心概念**：
    -   这是一个**金字塔结构**。从上到下：**寄存器 -> 缓存 -> 主存 -> 电子磁盘（SSD） -> 机械磁盘（HDD） -> 磁带/光盘**。
    -   **设计原则**：**速度越快，价格越贵，容量越小**。
    -   **数据移动**：数据在层次之间流动。操作系统负责将数据从慢速存储（磁盘）调度到快速存储（内存）中供CPU处理。

**补充知识点**：

-   **局部性原理**：这是层次结构能够生效的理论基础。
    -   **时间局部性**：如果一个数据被访问，近期很可能再次被访问。
    -   **空间局部性**：如果一个数据被访问，其邻近的数据也很可能被访问。
    -   正是由于局部性，缓存才能达到很高的命中率（通常超过90%）。
-   **虚拟内存**：这是存储层次体系的延伸。操作系统利用磁盘空间模拟内存，使得进程认为拥有连续的、巨大的内存空间，实际上物理内存可能不足。

### 5. Hardware Protection（硬件保护）

操作系统必须依靠硬件来防止用户程序出错或恶意破坏系统。

-   **核心概念**：
    -   **双重模式保护**：防止用户程序执行特权指令。
    -   **I/O保护**：所有I/O指令都是特权指令。用户程序不能直接操作硬件，必须通过系统调用请求操作系统代为执行。
    -   **内存保护**：使用**基址寄存器**和**界限寄存器**。CPU在访问内存时，硬件会自动检查地址是否落在当前进程的合法范围内，防止一个进程访问另一个进程的内存区域。
    -   **CPU保护**：使用**定时器**。操作系统设置定时器，如果某个进程占用CPU时间超过设定值，定时器中断会将控制权交还给操作系统，防止进程死循环或无限占用CPU。

**补充知识点**：

-   **特权指令**：只能在内核模式下执行的指令，例如：修改定时器值、清空内存、执行I/O操作、切换模式位等。
-   **安全边界**：现代CPU引入了更多的保护环，如x86架构的**Ring 0（内核）、Ring 3（用户）**。Ring 1和Ring 2很少被主流操作系统使用。

### 6. General System Architecture（通用系统架构）

现代计算机不再是孤立的单机。

-   **核心概念**：
    -   **单处理器系统**：只有一个通用CPU执行指令流。
    -   **多处理器系统**：包含两个或更多的紧密通信的CPU。又分为**非对称多处理**（一个主处理器控制，其他从属）和**对称多处理**（每个CPU地位平等，独立运行）。
    -   **集群系统**：将多台计算机通过网络连接，协同工作，对外表现为一台机器。主要用于**高可用性**（一台宕机，另一台接管）和**高性能计算**。

**补充知识点**：

-   **多核与超线程**：
    -   **多核**：将多个CPU核心封装在一个芯片上，共享缓存，通信延迟远小于传统的多处理器。
    -   **超线程**：将一个物理核心模拟成两个逻辑核心，通过利用核心空闲资源（如流水线停顿）来提升并行度。操作系统会将其识别为两个CPU。
-   **NUMA架构**：在多处理器系统中，如果所有CPU共享同一块内存，总线会成为瓶颈。现代服务器采用**非一致性内存访问**架构，即每个CPU有自己的本地内存（访问快），也可以访问其他CPU的内存（访问慢）。操作系统在进行进程调度和内存分配时，会尽量让进程运行在内存所在的本地CPU上，以优化性能。

---

### 总结

这六点实际上描绘了现代操作系统的运行逻辑：

1.  **硬件启动**（操作） -> **I/O交互**（中断/DMA） -> **数据存放**（存储结构）
2.  **利用层次**（缓存/虚拟内存）提升性能
3.  **借助硬件**（模式/定时器）提供保护
4.  **最终演变为**（多核/集群）复杂架构

操作系统就是在这套硬件基础上，负责管理资源、协调流程、屏蔽硬件差异的“管家”。
