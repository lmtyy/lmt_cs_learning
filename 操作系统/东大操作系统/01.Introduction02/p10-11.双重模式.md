### 1. 双重模式运行 (Dual Mode Operation)

这一页的核心是介绍操作系统如何通过**硬件级别的权限划分**来实现自我保护和进程隔离。

**核心知识点：**

- **硬件支持的两种模式 (Hardware Modes)：**

  现代计算机 CPU 在硬件层面上至少提供两种运行模式（通过 CPU 内部的**模式位 Mode Bit** 来区分，例如 1 代表用户态，0 代表内核态）：

  - **内核态 (Kernel Mode)：** 也称管态、特权态。拥有最高系统权限，可以执行所有机器指令，访问所有物理内存地址和所有底层硬件设备。操作系统 (OS) 核心代码运行在此模式。
  - **用户态 (User Mode)：** 也称目态、普通态。权限受限的沙盒环境。我们日常运行的普通应用程序（如浏览器、Word、用户自己写的代码）都运行在此模式。

- **特权指令 (Privileged Instructions)：**

  - 为了系统安全，某些具有破坏性或涉及系统核心资源的指令**被禁止在用户态下执行**。
  - **示例：** 修改页表 (Modify page tables)、直接访问 I/O 设备、关闭中断等。
  - **违规处理：** 如果用户程序试图在用户态执行特权指令，CPU 硬件会立即拦截，并产生一个**异常 (Exception)**，操作系统会接管并通常会强制终止该违规进程。

- **模式切换 (Transitions)：从用户态到内核态**

  普通程序如果需要读取文件、分配内存或使用网络，必须向操作系统申请。这个过程会触发模式切换：

  1. **用户态执行：** 进程正常运行代码 (`mode bit = 1`)。
  2. **发起调用：** 程序调用系统 API，触发一个特殊的陷入指令 (Trap)。
  3. **陷入内核：** 硬件立刻将 CPU 模式位切换为内核态 (`mode bit = 0`)，并将控制权安全地转移给操作系统的处理函数。
  4. **执行系统调用：** OS 在内核态下拥有高权限，代为执行读取文件等危险/底层操作。
  5. **返回用户态：** 操作完成后，OS 将结果准备好，将模式位切回用户态 (`mode bit = 1`)，并将控制权交还给用户程序继续执行。

  - *触发切换的主要途径：* 系统调用 (System Calls, 程序主动求助)、中断 (Interrupts, 外部硬件信号)、异常 (Exceptions, 程序运行出错)。

### 2. UNIX 系统结构 (UNIX System Structure)

这一页展示了前一页的“双重模式”硬件理论，是如何在真实的 UNIX 操作系统软件架构中落地的。整个系统被划分为层级分明的结构。

**核心知识点：**

整个系统自上而下可以清晰地划分为三大逻辑层：

- **第一层：用户态 (User Mode) - 应用程序层**

  - **Applications (应用程序)：** 位于最顶层，直接与用户交互的软件。
  - **Standard Libs (标准库)：** 包含 Shell（命令行解释器）、编译器、解释器以及各种系统级 C 库（如 `libc`）。
  - *注意：* 程序员平时写的 `printf()` 或 `malloc()` 等函数实际上是标准库提供的，它们运行在用户态，在底层会自动将任务打包，向内核发起真正的系统调用。

- **第二层：内核态 (Kernel Mode) - 操作系统的核心**

  这是一个受到硬件严格保护的整体区域，包含了操作系统最重要的三大组成部分：

  1. **顶部边界：系统调用接口 (System-call interface to the kernel)**
     - 这是用户态程序进入内核态的**唯一合法大门**。负责接收用户程序的请求，进行权限校验和参数检查，然后将任务分发给具体的内核模块。
  2. **核心功能模块 (Kernel Core)：**
     - **I/O 与终端管理 (左)：** 处理信号、字符设备 I/O 和终端驱动。
     - **文件与存储系统 (中)：** 管理文件系统、块设备 I/O、磁盘数据的读写和交换 (Swapping)。
     - **CPU 与内存管理 (右)：** 负责进程的 CPU 调度算法、虚拟内存管理、页表映射 (配合 MMU) 以及缺页替换。
  3. **底部边界：内核到硬件的接口 (Kernel interface to the hardware)**
     - 包含各种设备的底层驱动程序 (Device Drivers)。这层负责将内核的通用逻辑指令翻译成具体的电子信号，发送给底层的物理硬件控制器。

- **第三层：硬件层 (Hardware)**

  - 最底层的物理实体，包括终端控制器、磁盘、磁带、内存控制器以及物理内存条本身。它们只负责听从来自内核底层接口的机器指令去执行物理动作。
