### 第一页：分布式系统与局域网 (History Phase 4: Distributed Systems)

这一页标志着计算模式从早期的“集中式（大型机）”向“分布式”的重大转变。

**核心知识点：**

- **局域网 (LAN - Local Area Networking):** 随着个人电脑（PC）的普及，将不同物理机器连接起来共享资源成为刚需。局域网使得在一个有限地理范围（如办公室、学校）内的计算机可以互相通信。
- **资源共享 (Resource Sharing):** 连网的最初目的。通过网络，多台计算机可以共享昂贵的硬件（如打印机 Printer）、集中存储的数据（文件服务器 File Server）以及后来出现的网页内容（Web Server）。
- **客户端-服务器模型 (Client-Server Model):** 这是分布式系统中最经典、最重要的架构。 * **客户端 (Client):** 用户直接交互的设备（如图中灰色的框），负责发送请求。
  - **服务器 (Server):** 拥有资源并提供集中服务的强大计算机（如图中蓝色的框），负责响应请求。
  - **图中架构解析:** 图表清晰地展示了多个 Client 通过 Network（网络）连接到一个集中的 Server。无论是计算任务（Computing）还是文件存储（File Storage），都可以由服务器集中处理，客户端只负责“享受”服务。

**补充知识点：**

- **对等网络 (P2P - Peer-to-Peer):** 与 C/S 架构相对的是 P2P 架构（如早期的 BT 下载软件）。在 P2P 中，没有绝对的“服务器”，每台机器既是客户端也是服务器，大家互相提供资源。
- **以太网 (Ethernet):** 虽然 PPT 没提，但局域网的绝对霸主是“以太网”技术（IEEE 802.3 标准），也就是我们今天还在用的网线连接技术。

### 第二页：互联网的诞生与服务 (History Phase 4: Internet)

如果局域网是将一个办公室的电脑连起来，那么互联网就是将全世界的局域网连起来的“巨网”。

**核心知识点：**

- **起源与开放标准:** 互联网最初由科研社区推动（起源于美国军方的 ARPANET）。它的成功归功于**开放标准**，特别是 **IP协议 (Internet Protocol)**。开放意味着任何人都可以基于这个规则制造设备或编写软件，从而实现全球互联。
- **IETF (互联网工程任务组):** 这是一个极其重要的国际组织。互联网没有“绝对的统治者”，但它的技术标准（如 TCP/IP, HTTP 等的 RFC 文档）都是由 IETF 这个工程师社区讨论和制定的。 * **内联网 (Intranet):** 基于互联网技术（IP网络）搭建的企业内部专属网络。它使用了与互联网相同的底层技术，但不向公众开放，保证了企业数据的安全。
- **互联网服务的演进史:**
  - **70年代 - 计算资源共享:** `telnet` 协议允许用户远程登录到另一台大型机上使用其算力。
  - **80年代 - 数据与文件共享:** 出现了 `FTP`（文件传输协议）、`NFS`（网络文件系统，Unix系统常用）、`AFS`。
  - **90年代起 - 互动通讯与信息传播:** 电子邮件（Email）、聊天室（BBS）、即时通讯（IM）爆发；更重要的是 `WWW`（万维网）的出现，彻底改变了人类获取信息的方式。
  - **千禧年后:** 音视频流媒体崛起，逐渐取代传统的广播电视，以及点对点文件分享的盛行。

**补充知识点：**

- **WWW ≠ Internet:** 很多人混淆了这两个概念。Internet（互联网）是底层物理网络和协议基础；而 WWW（万维网，也就是我们常说的“网页”）只是运行在互联网之上的一种**服务**。

### 第三页：网络“云”的概念 (Network "Cloud")

这页展示了一个非常经典的网络拓扑抽象图。

**核心知识点：**

- **作为黑盒的“云” (The Network Cloud):** 在计算机网络拓扑图中，工程师们习惯用一朵“云”来代表复杂的广域网（WAN）或互联网。

- **抽象化 (Abstraction):** 对于边缘的服务器或个人电脑来说，数据只要发进了这朵“云”，云内部复杂的路由器怎么转发、走哪条光缆，对终端用户都是**透明的（不可见的）**。图中环绕的计算机，通过一根线接入云端，只要协议正确，就能与云另一端的任何计算机通信。

  **补充知识点：**

- **现代“云计算”的渊源:** 这页 PPT 里的“云”最初只是指代“复杂的网络传输路线”。但在今天，“云计算”赋予了它新的含义：不仅把“网络路线”隐藏在云里，更是把“计算能力”和“存储空间”（如 AWS, 阿里云）也放进了这端看不见的“云”里，用户按需取用即可。

### 第四页：网络层级结构：骨干网与区域网 (Regional Nets + Backbone)

这页揭示了那朵“云”内部的真实物理和逻辑结构，互联网本质上是一个“网中网”（Network of Networks）。

**核心知识点：**

- **层级架构 (Hierarchical Structure):** 互联网不是把全世界几十亿台电脑直接连在一根线上，而是分层的。
- **骨干网 (Backbone):** 图中中间最粗的黑线。这是互联网的“国家级高速公路”。它由超高带宽的光纤组成，负责跨国、跨大洲的海量数据传输。通常由顶级电信运营商（Tier 1 ISP）掌控。 * **区域网 (Regional Net):** 相当于“省道”或“市级公路”。它们连接到骨干网，负责将一个地区（比如一个州、一个城市）的网络流量汇聚起来，传输到骨干网上。
- **局域网 (LAN):** 位于结构的最底层。你家里的路由器、公司的内部网络就是 LAN。LAN 连接终端用户，并向上接入区域网。

**形象比喻：**

假设你要从北京的家里寄一个快递到纽约的某个公司：

1. 快递员从你家收件（**数据离开你的电脑，进入局域网 LAN**）。
2. 快递送到北京市的分拨中心（**数据进入区域网 Regional Net**）。
3. 包裹被装上跨国飞机，飞往美国（**数据进入跨国骨干网 Backbone**）。
4. 到达纽约后，送到区域分拨中心（**进入对方的区域网**）。
5. 最后送到对方公司的办公室（**到达对方的局域网 LAN 里的目标电脑**）。
