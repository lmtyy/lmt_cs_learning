# 生成树、有向树与图存储引入（p017-020）

从"子图"概念过渡到生成树——图算法中最核心的结构之一，然后引出"图怎么存到计算机里"这个关键问题。

---

## 第17页：生成树 Spanning Tree（图的骨架）

### 核心定义

设 G = (V, E) 是**连通带权图**，每条边 (u,v) 有代价 c(u,v)。

**生成树（Spanning Tree）**= G 的一个子图，满足：
1. **连通**（Connected）——所有顶点都能互达
2. **无环**（Acyclic）——没有回路
3. **包含所有顶点**——一个都不能少

生成树的**代价** = 树中所有边的权值之和。

### 关键性质

| 性质 | 说明 |
|------|------|
| n 个顶点的生成树恰好有 **n-1 条边** | 少一条就不连通，多一条就有环 |
| 往生成树里加任意一条边 → **必产生环** | 因为树中任意两点已经有唯一路径 |

> 🎯 **生活类比**：把城市看作顶点，公路看作边。生成树就是"用最少的路把所有城市串起来，不走回头路"。最小生成树就是"修路总预算最低的方案"。

> 💡 **核心考点**：为什么恰好 n-1 条边？——树的递归定义：1个点是树（0边），每加一个点恰好加一条边连到已有树上，所以 n 个点 = n-1 条边。

## 第18页：生成树 vs 子图 — 图解对比

PPT展示了同一个5顶点连通图G（V0~V4）的三种情况：

```
原图 G（连通图）          最小生成树              子图（非生成树）
V0 — V1                 V0 — V1                V0 — V1
|  × |                  |                           \
V2   |                  V2                      V2
|  \ |                  |                        \
V3 — V4                 V3   V4                  V4
```

| 图 | 连通？ | 无环？ | 含所有顶点？ | 是否生成树？ |
|----|--------|--------|-------------|-------------|
| 原图 G | ✅ | ❌（有环） | ✅ | ❌ |
| 中间图 | ✅ | ✅ | ✅ | ✅ 生成树 |
| 右边图 | ❌（V3孤立）或有环 | — | — | ❌ 仅是子图 |

> ✅ **记忆口诀**：生成树 = 连通 + 无环 + 全覆盖。三个条件缺一不可。

> ⚠️ 一个图的生成树**不唯一**！同一个图可能有多棵不同的生成树，但边数一定都是 n-1。

## 第19页：有向树 Directed Tree 与生成森林

### 有向树定义

**有向树（Directed Tree）**：一棵有向图形式的树，满足：
- **恰好一个节点入度为0**（这就是根 root）
- **其他所有节点入度为1**（每个非根节点恰好有一个父亲）

> 类比：普通树就像"家族族谱"——每人只有一个亲爹（入度1），只有祖宗没有爹（入度0）。

### 有向图的生成森林

对于**有向图**（可能不连通），其**生成森林（Spanning Forest）**由若干棵有向树组成：
- 包含图中**所有顶点**
- 包含图中**部分弧**（边）
- 每棵有向树对应一个"可达分量"

PPT示例：
- 左侧：4顶点有向图 V0→V1, V0→V2, V0→V3, V2→V3 分解为两棵有向树
- 右侧：6顶点有向图 A→B→C, F→E→D 等，分解为两棵有向树

> 💡 **考点**：DFS/BFS遍历有向图时，产生的遍历树就是生成森林的一部分。这是后面"图遍历"章节的基础。

## 第20页：图的存储表示引入 — Representing Graphs

### 核心问题

图存到计算机里，需要存什么？

1. **顶点（Vertices）**
2. **边（Edges）**

### 顶点标识

```
G = <V, E>
V = {v₀, v₁, v₂, ..., vₙ₋₁}
顶点的标签（label）就是它的下标（index）
```

> 这意味着顶点用 0 ~ n-1 的整数编号，后续无论邻接矩阵还是邻接表，都靠这个编号索引。

PPT同时展示了无向图（V0~V4）和有向图（V0~V3）两个例子，暗示：**同一套存储方案要能处理两种图**。

> 🎯 下一步要回答的关键问题：边怎么存？两种经典方案——邻接矩阵（二维数组）和邻接表（链表数组）。各有什么时间/空间权衡？

## 总结

```
本段逻辑线：
子图概念 → 生成树（连通+无环+全覆盖，n-1边）
         → 有向树（入度0的根 + 其余入度1）
         → 生成森林（有向图拆成多棵有向树）
         → 引出问题：图怎么存？顶点用下标编号，边的存储方案待续
```

> 💡 **串联提示**：生成树概念是后面 Prim/Kruskal 最小生成树算法的基础；有向树/生成森林是 DFS 遍历结果的结构；图的存储方式直接决定所有图算法的时间复杂度。
