# 图的遍历与深度优先搜索 DFS（p043-048）

进入图算法的核心操作——遍历。图不像树有唯一根节点和天然的层次结构，它可能有环、可能不连通，所以遍历比树复杂得多。本段从"为什么图遍历需要额外机制"讲起，深入 DFS 的思路、执行过程和关键实现细节。

---

## 第43页：图遍历引入（"探索未知领土"）

### 核心目标

**图遍历（Graph Traversal）= 访问所有顶点，每个顶点恰好访问一次。**

### 图遍历 vs 树遍历：有什么不同？

| | 树遍历 | 图遍历 |
|---|---|---|
| 起点 | 唯一（根节点） | 任意选一个顶点开始 |
| 是否有环 | ❌ 树无环 | ✅ 图可能有环 → 可能死循环！ |
| 是否连通 | ✅ 树一定连通 | ❌ 图可能不连通 → 一次遍历可能覆盖不了所有点 |
| 访问控制 | 不需要（天然不会重复） | **必须标记已访问**，否则死循环 |

> 🎯 **类比**：树遍历像"从族长开始依次见每个家族成员"——结构清晰不会重复。图遍历像"在一座迷宫里探索"——有岔路、有回路，必须做标记才不会走重复的路。

### 两种基本方法

从树遍历直接泛化过来：

| 树遍历方法 | 对应的图遍历 |
|-----------|-----------|
| 前序/中序/后序（深度优先） | **DFS（深度优先搜索）** |
| 层序遍历（广度优先） | **BFS（广度优先搜索）** |

## 第44页：DFS 的核心思想（Recursion！）

### 算法描述

给定无向图 G，所有顶点初始标记为 **unvisited**。

1. 选一个起始顶点 v，标记 v 为 **visited**
2. 对 v 的每个**未访问的邻居**，递归执行 DFS
3. 直到从 v 出发能到达的所有顶点都被访问

### 执行过程图解（8顶点无向图）

```
图结构：
        V0
       / \
     V1    V2
    / \   / \
  V3  V4 V5  V6
   \  /
    V7
```

从 V0 开始 DFS：
- 访问 V0 → 选 V1（第一个邻居）
- 访问 V1 → 选 V3（V1的第一个未访问邻居）
- 访问 V3 → 选 V7（V3的邻居）
- 访问 V7 → 选 V4（V7的未访问邻居）
- 访问 V4 → V4的邻居都访问了（V1已访问、V7已访问），**回退**
- 回退到 V7 → V7邻居都访问了，回退
- 回退到 V3 → 回退到 V1 → V1还有邻居？V4已访问，回退
- 回退到 V0 → V0还有 V2 未访问！
- 访问 V2 → V5 → V6 → 完成

> 💡 **本质**：DFS = "一条路走到黑，走不下去了才回头"。是一种**回溯（Backtracking）**策略。

> ✅ **DFS 天然是递归的**：每次深入一个邻居就是一层递归调用，回退就是递归返回。

## 第45页：DFS 序列不唯一！

### 同一个图的两种 DFS 序列

从 V0 出发：

- 序列1：V₀, V₁, **V₃**, V₇, V₄, V₂, V₅, V₆
- 序列2：V₀, V₁, **V₄**, V₇, V₃, V₂, V₅, V₆

区别在第3步：是先访问 V₃ 还是 V₄？取决于**邻居的访问顺序**。

> ⚠️ **关键认知**：DFS 序列取决于"选哪个邻居先走"。在概念层面，DFS 序列**不唯一**。

### 什么时候序列是唯一的？

> ✅ **如果存储结构确定，DFS 序列就唯一了！**

因为邻接表中，每个顶点的链表顺序是固定的——算法总是从 `firstarc` 开始顺序遍历。

## 第46页：存储结构决定唯一序列

### 完整邻接表（8顶点图）

```
0  V₀ → [1] → [2] → ∧
1  V₁ → [3] → [4] → [0] → ∧
2  V₂ → [5] → [6] → [0] → ∧
3  V₃ → [7] → [1] → ∧
4  V₄ → [7] → [1] → ∧
5  V₅ → [6] → [2] → ∧
6  V₆ → [2] → [5] → ∧
7  V₇ → [3] → [4] → ∧
```

### 按此邻接表执行 DFS（从 V0 出发）

| 步骤 | 当前顶点 | 动作 | 原因 |
|------|---------|------|------|
| 1 | V₀ | 访问V₀，看链表：第一个邻居是1(V₁) | V₁未访问，深入 |
| 2 | V₁ | 访问V₁，看链表：第一个是3(V₃) | V₃未访问，深入 |
| 3 | V₃ | 访问V₃，看链表：第一个是7(V₇) | V₇未访问，深入 |
| 4 | V₇ | 访问V₇，看链表：第一个是3(V₃) | V₃**已访问**，跳过；下一个4(V₄) |
| 5 | V₄ | 访问V₄，看链表：7→已访问，1→已访问 | 全部已访问，**回退** |
| 6 | 回退到V₇→V₃→V₁ | V₁链表：3→已,4→已,0→已 | 全部已访问，回退到V₀ |
| 7 | V₀ | 链表：1→已，下一个2(V₂) | V₂未访问，深入 |
| 8 | V₂ | 访问V₂，链表第一个5(V₅) | 深入 |
| 9 | V₅ | 访问V₅，链表第一个6(V₆) | 深入 |
| 10 | V₆ | 访问V₆，链表：2→已，5→已 | 全部已访问，回退 |

**最终 DFS 序列**：V₀, V₁, V₃, V₇, V₄, V₂, V₅, V₆

> 💡 **考试必考**：给你邻接表，写出 DFS 序列。方法就是"从 firstarc 开始，未访问就深入，已访问就看下一个，都访问了就回退"。

### DFS 生成树

DFS 过程中经过的边形成一棵**生成树**（DFS Tree）：

```
        V0
       /  \
     V1    V2
     |      |
     V3    V5
     |      |
     V7    V6
     |
     V4
```

> 这棵树包含了图中的 7 条边（n-1=7），覆盖了所有 8 个顶点。图中其他边（如 V4-V1、V6-V2 等）是"回边"（back edge），它们在 DFS 中被发现时目标已访问，所以被跳过。

## 第47页：`visited[]` 数组——防重复访问的核心机制

### 问题

图有环 → DFS 可能反复访问同一顶点 → **死循环**。

### 解决方案

开一个布尔数组 `visited[N]`：

```
初始：visited[0..N-1] = 0（全部未访问）

访问顶点 i 时：visited[i] = 1（标记已访问）

检查邻居 j 时：if (visited[j] == 0) 才递归进入
```

> 🎯 **类比**：就像在迷宫墙上画 ✓ 标记——走过的路口不再进入。

> 💡 **为什么不能用"父节点记录"代替？** 因为图不是树——一个节点可能被多个节点连接。仅记住"从谁来的"无法防止从其他路径再次到达它。必须有全局的 `visited[]` 数组。

> ⚠️ **空间代价**：额外 O(N) 空间。这是图遍历相比树遍历的固有开销。

## 第48页：DFS 完整执行示例（9顶点网格图）

### 图结构

```
A --- B --- E
|    |     |
D    C     G
|    |     |
F --- H --- I
```

9个顶点排成 3×3 网格，边连接相邻格子。

### DFS 从 A 出发的执行过程

访问顺序：A(1) → B(2) → E(3) → G(4) → C(5) → F(6) → D(7) → H(8) → I(9)

右侧图展示了 DFS 生成的**搜索树**：

```
A — B — E
         |
    C — G
    |
D   H — I
|
F
```

搜索过程中：
- **实线箭头（搜索 →）**：DFS 深入访问新顶点时走的边
- **虚线箭头（回退 ←）**：所有邻居都已访问，递归返回

> 💡 **关键观察**：DFS 生成树的形状取决于邻接表中邻居的排列顺序。改变邻接表顺序，同一个图会生成不同的 DFS 树。

> ✅ **考试技巧**：手动模拟 DFS 时，画一个表格记录每一步"当前节点→选择的邻居→原因（未访问/已访问跳过/回退）"，不容易出错。

## 总结

```
DFS 核心要素：
├── 数据结构：栈（递归调用栈 / 显式栈）
├── 防重复：visited[] 数组，访问即标记
├── 策略："一条路走到黑，死路才回头"
├── 序列唯一性：概念上不唯一，存储结构确定后唯一
├── 产物：DFS 生成树（n-1 条树边 + 若干回边）
└── 本质：前序遍历的图版本

时间复杂度：
  邻接表：O(N + E)  ← 每个顶点访问1次，每条边检查1次（无向图2次）
  邻接矩阵：O(N²)  ← 每个顶点要扫一整行找邻居
```

> 💡 **前后呼应**：DFS 生成树的概念直接关联到后面的：
> - **拓扑排序**（DFS 后序的逆序）
> - **强连通分量**（Tarjan/Kosaraju 算法基于 DFS）
> - **回溯法**（第13章，本质就是 DFS + 剪枝）
