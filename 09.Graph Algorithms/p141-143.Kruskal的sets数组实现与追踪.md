# Kruskal 算法的 sets[] 实现与动画追踪（p141-143）

PPT 用一种简化的并查集实现（`sets[]` 数组）来演示 Kruskal 的判环与合并过程。这是一种直观的"集合编号"方法——每个顶点存自己所属集合的编号，Find 就是读数组，Union 就是扫全表改编号。

---

## 第141页：sets[] 数组初始化 + 算法框架

### 数据结构

```c
int sets[n];    // sets[i] = 顶点 i 所属的集合编号
```

### 初始化

```
sets[0]=0, sets[1]=1, sets[2]=2, sets[3]=3, sets[4]=4, sets[5]=5
```

**每个顶点自己就是自己的集合**——初始有 n 个独立集合，集合编号 = 顶点编号。

### 示例图（6顶点）

```
      V0
    6/ | \5
   V1  1  V3
    5\ | /5
      V2
    6/ | \4
   V4  2  V5
      \ /
       6
```

边排序后（从小到大）：V0-V2(1), V3-V5(2), V1-V4(3), V2-V5(4), V0-V1(5) or V1-V2(5), V0-V3(5), ...

---

## 第142页：算法步骤描述

### 完整流程（5 步）

**(1) 初始化**：`sets[i] = i`，n 个顶点构成 n 个独立连通分量。

**(2) 依次取出 E 中的边**（按权值递增），设取出的边为 `(xi, xj)`。

**(3) 判断**：
```
若 sets[i] == sets[j]  → xi 和 xj 在同一集合 → Reject，返回(2)
否则 → 转到(4)
```

**(4) 合并**：将边 `(xi, xj)` 加入 MST，同时**把 xj 所在集合的所有顶点改为 xi 所在的集合编号**：
```
v = sets[j]    // xj 的集合编号
u = sets[i]    // xi 的集合编号
扫描 sets[]：所有 sets[k]==v 的，改为 sets[k]=u
```

**(5) 重复 (2)(3)(4)，直到取出 n-1 条边。**

### sets[] 方法 vs 标准并查集

| | sets[] 数组法（PPT） | 标准并查集（路径压缩+按秩） |
|---|---|---|
| Find(x) | O(1)——直接读 `sets[x]` | ≈O(1)——沿父指针到根 |
| Union(x,y) | **O(N)**——扫全表改编号 | ≈O(1)——改根指针 |
| 总体复杂度 | O(N·E) | O(E·α(N)) ≈ O(E) |

> ⚠️ **这种简化实现 Union 是 O(N) 的**——每次合并要把一个集合的**所有成员**的编号都改掉。标准并查集用树结构避免了这个问题。PPT 用这种方式是为了**教学直观**，实际工程用标准并查集。

## 第143页：动画追踪（6顶点图的完整执行）

### 初始状态

```
sets: [0, 1, 2, 3, 4, 5]
MST: {}
```

所有顶点独立，每个顶点的 "集合编号" = 自己的编号。

---

### Step 1：取边 (V0, V2) 权=1

- sets[0]=0, sets[2]=2 → **不同集合** → ✅ Accept
- 合并：把集合2的成员改为集合0的编号
  - sets[2] 原来=2 → 改为 0
- **sets: [0, 1, `0`, 3, 4, 5]**
- **MST: {V0-V2(1)}**

图上 V0-V2 连上红线。

---

### Step 2：取边 (V3, V5) 权=2

- sets[3]=3, sets[5]=5 → **不同集合** → ✅ Accept
- 合并：把集合5改为集合3
  - sets[5] 原来=5 → 改为 3
- **sets: [0, 1, 0, 3, 4, `3`]**
- **MST: {V0-V2(1), V3-V5(2)}**

---

### Step 3：取边 (V1, V4) 权=3

- sets[1]=1, sets[4]=4 → **不同集合** → ✅ Accept
- 合并：把集合4改为集合1
  - sets[4] 原来=4 → 改为 1
- **sets: [0, 1, 0, 3, `1`, 3]**
- **MST: {V0-V2(1), V3-V5(2), V1-V4(3)}**

---

### Step 4：取边 (V2, V5) 权=4

- sets[2]=0, sets[5]=3 → **不同集合** → ✅ Accept
- 合并：把集合3的所有成员改为集合0
  - sets[3]=3 → 改为 0
  - sets[5]=3 → 改为 0
- **sets: [0, 1, 0, `0`, 1, `0`]**
- **MST: {V0-V2(1), V3-V5(2), V1-V4(3), V2-V5(4)}**

> 💡 **注意**：合并时要把集合 3 的**所有成员**（V3 和 V5）都改成 0。这就是 O(N) 扫描的操作。

---

### Step 5：取边 (V1, V2) 权=5

- sets[1]=1, sets[2]=0 → **不同集合** → ✅ Accept
- 合并：把集合1的所有成员改为集合，PPT 显示最终 sets = [1, 1, 1, 1, 1, 1]

实际看 PPT 最后一帧的 sets: **[1, 1, 1, 1, 1, 1]**——说明是把集合0改为集合1（或反之，取决于实现选择）。

最终效果：
- **sets: 某种全部归一的状态**
- **MST: {V0-V2(1), V3-V5(2), V1-V4(3), V2-V5(4), V1-V2(5)}**
- 5 条边 = n-1 = 6-1 ✅ → 算法终止

**总代价 = 1 + 2 + 3 + 4 + 5 = 15**

      V2──5──V1

    V5   V3  V4

## 第143页黄框：复杂度分析

### 算法效率

> **算法效率分析：O(e·log e)**
> - 构造一个边的权值构成的堆：O(e·log e)
> - 选择过程：O(log e)

| 步骤 | 实现 | 时间 |
|------|------|------|
| 边排序（或建堆） | 对 E 条边按权值排序 | O(E log E) |
| 每次取最小边 | 堆顶取出 | O(log E) |
| 每次判断是否成环 | 并查集 Find | ≈O(1) |
| 每次合并 | 并查集 Union | ≈O(1)（标准版） |
| **总计** | | **O(E log E)** |

> 💡 PPT 这里用"堆"来管理边的排序——不是一次性全部排好，而是**边放入最小堆**，每次 `extractMin` 取出最小边。效果等价于排序，但对于"提前终止"（选够 n-1 条就停）的场景可能稍快。

## 🎯 考试手算 Kruskal（sets[] 版本）模板

1. **画表**：sets[0..n-1] 初始为 [0,1,2,...,n-1]
2. **边排序表**：边 | 权 | sets[i] | sets[j] | Action
3. **逐条处理**：
   - 读 sets[i] 和 sets[j]
   - 相等 → Reject
   - 不等 → Accept，扫描 sets 把一个集合的编号全改为另一个
4. **够 n-1 条停**
5. **画出 sets 数组每步变化**（PPT 考题风格）

### 💥 易错点

- **Union 时只改一个元素**：错！必须把被合并集合的**所有成员**都改掉。
  - 比如 V2 和 V5 都在集合 3 中，合并到集合 0 时，V2 和 V5 的 sets 值**都要改成 0**
- **Find 复杂度搞混**：sets[] 版本 Find 是 O(1)，但 Union 是 O(N)；标准并查集两者都是 ≈O(1)

## 总结

```
Kruskal 的 sets[] 实现：

数据结构：sets[i] = i 初始各自独立

Find(x)：直接读 sets[x] → O(1)
Union(i,j)：把 sets[]==sets[j] 的全部改成 sets[i] → O(N)

算法：
  排序/建堆 → 逐条取边 → Find判断 → 不同则Accept+Union → 够n-1停

复杂度：
  排序 O(E log E) + Union总共 (n-1)×O(N) = O(NE)
  瓶颈实际在排序 O(E log E)（N 通常 << E log E）

教学价值 vs 实际工程：
  sets[] 法直观易懂但 Union 慢 → 教学用
  标准并查集（路径压缩+按秩）Union/Find 都≈O(1) → 实际用
```
