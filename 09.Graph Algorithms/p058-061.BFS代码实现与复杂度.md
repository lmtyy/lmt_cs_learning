# BFS 代码实现与复杂度分析（p058-060）

BFS 从"水波扩散"的思想落地为代码——用队列驱动的迭代过程。本段拆解抽象版、邻接矩阵版，并推导复杂度的数学本质。

## 第58页：BFS 抽象版完整代码（"队列驱动的迭代"）

### 完整代码

```c
void BFSTraverse(Graph G, Status(*Visit)(int v))
{
    // 按广度优先遍历图G，使用辅助队列Q
    for(v=0; v<G.vexnum; ++v) visited[v] = FALSE;   // 初始化标记
    InitQueue(Q);                                     // 初始化队列
    
    for(v=0; v<G.vexnum; ++v) //防止图不连通导致只能访问连通块内部
        if(!visited[v])
        {
            visited[v] = TRUE; Visit(v);
            EnQueue(Q, v);
            
            while(!QueueEmpty(Q))//每次循环结束都代表这一连通块遍历完毕
            {
                DeQueue(Q, u);
                for(w=FirstAdjVex(G,u); w>=0; w=NextAdjVex(G,u,w))
                    if(!visited[w])
                    {
                        visited[w] = TRUE; Visit(w); EnQueue(w);
                    }
            }
        }
}
```

### 逐段精讲

**1. 初始化阶段**

```c
for(v=0; v<G.vexnum; ++v) visited[v] = FALSE;
InitQueue(Q);
```

- **动作**：把 `visited` 数组全部置为 FALSE，并初始化一个空队列 Q。
- **目的**：BFS 的两个"记账工具"就位——`visited` 记录"谁访问过"，`Q` 记录"接下来该处理谁"。
- **和 DFS 的差异**：DFS 只用 `visited`，隐式栈藏在递归调用里；BFS 需要**显式队列**，因为不能递归。

**2. 外层顶点循环（处理不连通图）**

```c
for(v=0; v<G.vexnum; ++v)
    if(!visited[v])
    {
        // 一次 BFS 处理一个连通分量
    }
```

- **动作**：从 0 到 N-1 逐个检查，遇到未访问顶点就启动一次 BFS。
- **目的**：和 DFS 驱动函数完全同理——一次 BFS 只能覆盖一个连通分量。
- **常见理解误区**：**不是**"外层循环 = 每次访问一层"！外层循环是**换连通分量**用的。每次进入循环体后，内部的 while 才是真正的一次完整 BFS。

**3. 启动 BFS——将起点标记并入队**

```c
visited[v] = TRUE; Visit(v);
EnQueue(Q, v);
```

- **动作**：三个动作依次执行——标记为已访问、执行访问操作（如打印）、入队。
- **⚠️ 关键顺序**：**标记要在入队之前！**
- **为什么？** 如果先入队再标记，可能在入队后、还没轮到出队之前，v 又被其他路径发现并再次入队，造成重复。

**4. BFS 核心循环——队列驱动**

```c
while(!QueueEmpty(Q))
{
    DeQueue(Q, u);
    for(w=FirstAdjVex(G,u); w>=0; w=NextAdjVex(G,u,w))
        if(!visited[w])
        {
            visited[w] = TRUE; Visit(w); EnQueue(w);
        }
}
```

- **动作**：只要队列非空，就取出队头 u，遍历 u 的所有邻居，把未访问的邻居标记+访问+入队。
- **循环终止条件**：队列空了 → 说明当前连通分量的所有顶点都访问完了。

**5. 处理邻居的三部曲**

```c
visited[w] = TRUE;     // 第1步：立刻标记（防止重复入队）
Visit(w);              // 第2步：执行访问操作
EnQueue(w);            // 第3步：入队等待后续处理
```

> 💥 **反面教材**：常见错误 —— 只入队不标记，等出队时才标记：
> ```c
> // ❌ 错误写法
> while(!QueueEmpty(Q))
> {
>     DeQueue(Q, u);
>     visited[u] = TRUE;   // ← 出队才标记，太晚了！
>     Visit(u);
>     for(w = 邻居)
>         if(!visited[w]) EnQueue(w);  // 未标记就入队
> }
> ```
> **后果推演**：假设 A 有两个邻居 B、C，且 B-C 之间也有边。
> - 处理 A：把 B、C 都入队（此时 B、C 都未标记）
> - 处理 B：看 B 的邻居 C——C 还未标记，**再次入队 C**！
> - 队列里出现两个 C。虽然出队时会标记，第二次到 C 时会通过 `visited[u]==TRUE` 跳过，但队列已经膨胀，浪费空间和时间。
> - 大图中这种膨胀是**指数级**的（每个顶点可能被入队 degree 次）。

> ✅ **正确原则**：**入队时立刻标记**——"进入队列 = 已被计划访问"，不再重复计划。

## 第59-60页：邻接矩阵版 BFS 完整代码

### 完整代码

```c
void BFSTraverse(MGraph G, Status(*Visit)(int v))
{
    // 按广度优先遍历图G，使用辅助队列Q
    for(v=0; v<G.vexnum; ++v) visited[v] = FALSE;
    InitQueue(Q);
    
    for(v=0; v<G.vexnum; ++v)
        if(!visited[v])
        {
            visited[v] = TRUE;
            Visit(v);
            EnQueue(Q, v);
            
            while(!QueueEmpty(Q))
            {
                DeQueue(Q, u);
                for(w=0; w<G.vexnum; ++w)              // ← 关键差异
                    if(G.arcs[u][w] && !visited[w])    // ← 扫矩阵一行
                    {
                        visited[w] = TRUE;
                        Visit(w);
                        EnQueue(w);
                    }
            }
        }
}
// 算法时间复杂度：O(n²)
```

### 关键差异逐句拆解

**邻接矩阵版查邻居**：

```c
for(w=0; w<G.vexnum; ++w)
    if(G.arcs[u][w] && !visited[w])
```

- **动作**：从 w=0 到 w=N-1，逐格检查矩阵第 u 行。
- **`G.arcs[u][w]` 非0/非∞**：表示 u→w 有边。
- **`!visited[w]`**：w 还没被访问过。
- 两条件同时满足 → w 是新发现的邻居，标记 + 访问 + 入队。

### 和邻接表版对比

**邻接表版查邻居**：
```c
for(p=G.vertices[u].firstarc; p; p=p->nextarc) {
    w = p->adjvex;
    if(!visited[w]) { ... }
}
```

| 版本 | 查邻居方式 | 每个顶点耗时 |
|------|-----------|-------------|
| 矩阵版 | 扫一整行 N 个格子 | O(N) |
| 邻接表版 | 沿链表走 degree 个节点 | O(degree) |

## 第61页：BFS 算法复杂度分析（"数学推导"）

### 邻接表版——O(N + E)

**推导过程**：

- 每个顶点入队恰好1次，出队恰好1次 → **顶点处理 O(N)**
- 每个顶点出队后遍历其邻接链表，链表长度 = **degree(v)**
- 所有顶点的度数之和：

$$
\sum_{i=0}^{n-1} d_i = 2E \text{（无向图）} \quad \text{或} \quad E \text{（有向图）}
$$

- **总时间** = 顶点处理 + 边扫描 = **O(N + E)**

> 💡 **无向图度数之和为什么是 2E？**  
> 每条边贡献 2 个度（两端各1个）。所以 Σdᵢ = 2E。这是"握手定理"——一个派对上所有人握手次数之和 = 握手对数 × 2。

### 邻接矩阵版——O(N²)

**推导过程**：

- 每个顶点出队后，要扫描矩阵的一整行来找邻居 → **每个顶点 O(N)**
- 共 N 个顶点 → 总时间 **N × O(N) = O(N²)**
- **和边数无关**！即使图只有1条边，也要 O(N²)

### 完整复杂度对比表

| 遍历 | 存储 | 时间复杂度 | 空间复杂度 |
|------|------|-----------|-----------|
| **DFS** | 邻接表 | O(N + E) | O(N) 递归栈 + O(N) visited |
| **DFS** | 邻接矩阵 | O(N²) | O(N) 递归栈 + O(N) visited |
| **BFS** | 邻接表 | O(N + E) | O(N) 队列 + O(N) visited |
| **BFS** | 邻接矩阵 | O(N²) | O(N) 队列 + O(N) visited |

> ✅ **核心结论**：
> - **算法本身**（DFS/BFS）不影响时间复杂度——两者都是 O(N+E) 或 O(N²)
> - **存储方式**才是决定因素——邻接表 O(N+E) vs 邻接矩阵 O(N²)

> 💡 **实际选择**：
> - 稀疏图（E << N²）：**必用邻接表**，BFS/DFS 都会快得多
> - 稠密图（E ≈ N²）：邻接矩阵的 O(N²) = O(E)，此时两种存储时间相当，但矩阵实现更简单

### BFS 的空间复杂度隐藏细节

- 队列最大宽度出现在"最长的一层"——最坏情况可能有 O(N) 个顶点同时在队列中
- 完全二叉树形图中，队列宽度 = 叶子数 = N/2
- 星形图（一个中心连所有）：BFS 一开始就把 N-1 个顶点全部入队 → 队列瞬间达到 O(N)

> ⚠️ **对比 DFS**：DFS 的空间开销是**递归深度**，最坏情况（链状图）也是 O(N)，但常数因子更小（每层只需保存局部状态）。BFS 需要保存"整层"，空间常数更大。

## 总结

```
BFS 代码架构（对比 DFS）：

BFS 用队列迭代                DFS 用栈递归
├── init visited[] & Queue    ├── init visited[]
├── for v in vertices:        ├── for v in vertices:
│    if !visited[v]:          │    if !visited[v]:
│      mark v; enqueue v      │      DFS(v):
│      while Q not empty:     │        mark v
│        u = dequeue          │        for each unvisited w:
│        for each unvisited w:│          DFS(w)  # 递归
│          mark w; enqueue w  

时间复杂度（DFS/BFS 相同）：
┌──────────────┬─────────────┐
│  邻接表      │   O(N + E)  │
│  邻接矩阵    │   O(N²)     │
└──────────────┴─────────────┘

握手定理：Σ degree = 2E（无向）
     → 这就是"邻接表遍历"能达到 O(E) 的数学基础
```

> 🎯 **一句话总结**：BFS 与 DFS 是同一枚硬币的两面——**同样的时间复杂度**，**不同的探索顺序**，**不同的应用场景**。选 BFS 还是 DFS 取决于问题需要什么样的顺序（最短路径 vs 深度探索）。
