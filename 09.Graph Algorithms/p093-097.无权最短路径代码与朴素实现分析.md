# 无权最短路径的代码实现与朴素版本分析（p093-097）

上一节图示演示了 BFS 求无权最短路的思路。本节把它落地为代码——引入 `Known/dist/path` 三元组表结构，然后剖析一种"朴素的循环实现"及其 O(|V|²) 复杂度的来源。

## 第93页：算法名称与本质回顾

### 明确算法归属

> **"This strategy for searching a graph is known as breadth-first search."**

**上一节图解演示的策略，其实就是 BFS（广度优先搜索）**。这里明确点名让概念对上号。

### BFS 的两个核心特征

> **"It operates by processing vertices in layers: the vertices closest to the start are evaluated first, and the most distant vertices are evaluated last."**

| 特征 | 含义 |
|------|------|
| **按层处理**（in layers） | 距离 0 → 1 → 2 → ... 依次扩展 |
| **由近及远**（closest first） | 起点开始，越近的越先处理 |

> 🎯 **认知递进**：
> - 之前学 BFS 时讲的是"遍历所有顶点"
> - 现在明确 BFS 的**核心价值**：它天然产生**距离信息**——每个顶点被访问的顺序 = 它到起点的距离层次
> - 这个副产品被"包装"成"无权最短路径算法"

## 第94页：追踪表结构 —— 三个字段

### 表结构

对每个顶点 v，维护三个字段：

| 字段 | 含义 | 类型 |
|------|------|------|
| **Known** | 该顶点是否已"最终确定" | bool（0/1） |
| **d_v** | 从起点到 v 的距离（当前最优估计） | int |
| **p_v** | 从起点到 v 的最短路径上，**v 的前驱顶点** | 顶点编号 |

### 初始状态（以 v3 为起点）

| v | Known | d_v | p_v |
|---|-------|-----|-----|
| v1 | 0 | ∞ | 0 |
| v2 | 0 | ∞ | 0 |
| **v3** | **0** | **0** | **0** |
| v4 | 0 | ∞ | 0 |
| v5 | 0 | ∞ | 0 |
| v6 | 0 | ∞ | 0 |
| v7 | 0 | ∞ | 0 |

（注意 PPT 中 v3 的 d_v 也初始化为 0——这是起点特有的）

### 三个字段的作用

**1. `Known`**：**"这个顶点的最短路已经确定了，别再改了"** 的标志。

- 初始全为 0（false）
- 一旦某顶点被"处理"（作为出发点扩展邻居），标记为 1

> 💡 **和 BFS 的 `visited[]` 区别**：`visited` 表示"访问过"（进入队列时就标记），`Known` 表示"已作为处理主体"（出队时才标记）。语义更细致。

**2. `d_v`（distance）**：**从起点到 v 的当前距离**。

- 初始为 ∞（表示未知）
- 起点自己是 0
- 随算法进行不断更新（但无权情况下，第一次赋值就是最终值）

**3. `p_v`（predecessor / parent）**：**最短路径上的前驱**。

- 初始为 0（哨兵值，表示无前驱）
- 顶点 v 通过邻居更新时，记录"是谁把我更新的"
- **这个字段是重构完整路径的关键**！

> 🎯 **类比**：想象你在迷宫里，每格挂个小牌子：
> - Known = "这个格子已经完全探索过了"
> - d_v = "从入口走到这格花了几步"
> - p_v = "你从哪个格子来的"（沿这个信息可以走回入口）

## 第95页：算法主循环 + 路径重构

### 算法主逻辑

> **"The algorithm mimics the diagrams by declaring as known the vertices at distance d = 0, then d = 1, then d = 2, and so on, and setting all the adjacent vertices w that still have d_w = ∞ to a distance d_w = d + 1."**

**翻译**：
1. 从 d=0 开始，依次处理 d=1、d=2、...
2. 对每一层的顶点，遍历它们的邻居 w
3. 如果 w 的 d_w == ∞（还没被赋值），设 d_w = d + 1

### 路径重构：`p_v` 的作用

> **"By tracing back through the p_v variable, the actual path can be printed."**

**回溯（trace back）过程**：

```
想知道从 v3 到 v5 的完整路径？
    1. 查 p_v[v5] = v2（假设）→ v5 之前是 v2
    2. 查 p_v[v2] = v1        → v2 之前是 v1
    3. 查 p_v[v1] = v3        → v1 之前是 v3
    4. 到达起点 v3，停止
反过来打印：v3 → v1 → v2 → v5
```

> 💡 **递归打印**（伪代码）：
> ```c
> void print_path(int v) {
>     if (v == source) {
>         print(v);
>         return;
>     }
>     print_path(p_v[v]);  // 先打印前驱
>     print(" → " + v);    // 再打印自己
> }
> ```

> ✅ **核心考点**：路径重构不需要额外的图数据 —— 只需 `p_v[]` 数组，从终点沿前驱指针回溯到起点即可。

### 为什么用"前驱"而不是"后继"？

- 从**终点开始回溯**比"从起点前进"更简单
- 我们通常已知终点想找路径 → 沿前驱链条一路回到起点自然
- 类似链表的单向遍历——记住"上一个"就能走回去

## 第96页：朴素版代码（Naive Implementation）

### 完整代码

```c
void unweighted(TABLE T)   /* assume T is initialized */
{
    unsigned int curr_dist;
    vertex v, w;
    
/*1*/  for(curr_dist = 0; curr_dist < NUM_VERTEX; curr_dist++)
/*2*/      for each vertex v
/*3*/          if((!T[v].known) && (T[v].dist == curr_dist))
               {
/*4*/              T[v].known = TRUE;
/*5*/              for each w adjacent to v
/*6*/                  if(T[w].dist == INT_MAX)
                       {
/*7*/                      T[w].dist = curr_dist + 1;
/*8*/                      T[w].path = v;
                       }
               }
}
```

### 逐段精讲

**行 /*1*/：外层循环——按距离层次推进**

```c
for(curr_dist = 0; curr_dist < NUM_VERTEX; curr_dist++)
```

- **动作**：curr_dist 从 0 递增到 NUM_VERTEX - 1。
- **目的**：模拟"先处理距离0的层，再处理距离1的层..."。
- **为什么上限是 NUM_VERTEX？** 因为图中最长的最短路径不超过 |V|-1 条边（否则会有环）。用 NUM_VERTEX 作上限是保险取整。

> ⚠️ **性能陷阱**：即使实际图的最大距离只有 3，这个循环也会跑到 NUM_VERTEX 次。这就是浪费的源头之一。

**行 /*2*/：内层循环——扫描所有顶点**

```c
for each vertex v
```

- **动作**：每一层内部扫描**全部 N 个顶点**。
- **目的**：找出所有 `dist == curr_dist` 的顶点。

> 💥 **性能瓶颈**：即使当前距离层只有 1 个顶点，也要扫全表 N 次去找它。这就是 O(|V|²) 的根源。

**行 /*3*/：过滤条件——找未处理且距离对的顶点**

```c
if((!T[v].known) && (T[v].dist == curr_dist))
```

- **动作**：只处理**未标记为 Known** 且 **距离恰好等于当前 curr_dist** 的顶点。
- **两个条件都需要？**
  - `!T[v].known`：这个顶点还没被处理过，需要处理
  - `T[v].dist == curr_dist`：当前轮次只处理这一层的顶点
  - 由于无权图中每个顶点仅会遍历一遍，故保留一项即可

**行 /*4*/：标记为已处理**

```c
T[v].known = TRUE;
```

- **动作**：v 现在开始被"处理"（作为主体扩展邻居），标记为已 Known。
- **保证**：处理过一次就不再处理，避免重复扫邻居。

**行 /*5*/-/*8*/：扩展邻居**

```c
for each w adjacent to v
    if(T[w].dist == INT_MAX)         // 还没被发现
    {
        T[w].dist = curr_dist + 1;   // 距离 +1
        T[w].path = v;               // 记录前驱
    }
```

- **动作**：v 的每个邻居 w，如果 `dist == ∞`（还没被发现），就更新为 `curr_dist + 1`，并记录前驱为 v。
- **`T[w].dist == INT_MAX` 的判断**：等价于"这是第一次到达 w"，因为一旦更新过就不是 ∞ 了。
- **`T[w].path = v`**：为路径重构记录。

> 💡 **和 BFS 的关系**：这段代码本质上就是 BFS——只是用"扫描全表找当前层"代替了"队列"来管理层次。这是 BFS 的一个**低效实现方式**。

### 与队列版 BFS 的对比

| | 朴素版（本代码） | 队列版 BFS |
|---|---|---|
| 找当前层顶点 | 扫描全表 O(V) | 从队列取出 O(1) |
| 每轮复杂度 | O(V) × 每个顶点的邻居扫描 | O(V+E) 总计 |
| 层数上界 | 循环 V 轮（浪费） | 队列空即结束 |
| 总复杂度 | **O(V²)** | **O(V+E)** |

> 🎯 **这就是"数据结构决定算法效率"的最佳例证**：
> - 同样的 BFS 思想
> - 用队列 → O(V+E)
> - 用"扫全表"模拟队列 → O(V²)

## 第97页：复杂度分析 + 最坏情况示例

### 时间复杂度

> **"The running time of the algorithm is O(|V|²), because of the doubly nested for loops."**

**推导**：

```
外层循环：curr_dist 从 0 到 NUM_VERTEX-1 → O(|V|)
内层循环：每一轮扫描全部 |V| 个顶点     → O(|V|)

总时间 = O(|V|) × O(|V|) +O(|E|)= O(|V|²+|E|)
```

（内部扩展邻居的部分不影响这个上界，最坏情况所有边都被检查，加起来 O(|E|)，但 O(|E|) ≤ O(|V|²) 已被主项吸收。）

### 可以稍作优化——但不能改变 O(V²) 上界

> **"Although an extra test could be made to avoid this, it does not affect the worst-case running time."**

**优化尝试**：
- 记录"是否还有顶点未 known"
- 如果当前轮次没找到任何符合条件的顶点，提前退出

**为什么不能达到 O(V+E)？**
- 优化只能减少**平均情况**的常数
- 最坏情况仍然是 O(V²)（下面的链状图就是反例）

### 最坏情况——一条长链

**图示（PPT 底部）**：

```
v9 → v8 → v7 → v6 → v5 → v4 → v3 → v2 → v1
```

**分析**（从 v9 出发的最短路径）：

| curr_dist | 内层扫全表要检查 | 找到的顶点 |
|-----------|-----------------|-----------|
| 0 | 9 个顶点 | v9 |
| 1 | 9 个顶点 | v8 |
| 2 | 9 个顶点 | v7 |
| ... | ... | ... |
| 8 | 9 个顶点 | v1 |

**总操作数** = 9 × 9 = 81 = 9²

**推广到 |V| 顶点的链**：|V|² 次操作。

> 💥 **反面教材**：链状图暴露了朴素算法的弱点——每层只有 1 个顶点，但每次都要扫全表 N 次。**99% 的检查都是无用功**。

### 优化方向 —— 下一节引出

> **改进思路**：不要"扫全表找当前层"，而是**主动维护"当前层的顶点集合"**（用队列！）。

这就是**队列版 BFS**，也就是我们前面 BFS 章节讲过的版本。复杂度直接降到 **O(V+E)**。

> 💡 **实际实现建议**：
> - 教材这里给出朴素版是为了讲解**为什么需要队列优化**
> - 实际写代码时**永远用队列版**——O(V+E) vs O(V²)，稀疏图差距巨大

## 总结

```
无权最短路径实现路线：

数据结构（追踪表）：
  每个顶点 v 存 { Known, d_v, p_v }
  ├── Known：是否已确定最短路
  ├── d_v：到起点的距离
  └── p_v：最短路上的前驱（用于回溯完整路径）

朴素版代码（本节）：
  外层 curr_dist 从 0 到 |V|-1
    内层扫全表找 !known && dist==curr_dist 的顶点
      处理它，扩展未发现邻居
  → 复杂度 O(|V|²) —— 链状图最坏情况

路径重构：
  从终点沿 p_v 指针回溯 → 打印起点到终点的完整路径
  print_path(v):
    if v == source: print v; return
    print_path(p_v[v]); print " → " + v

优化方向：
  用队列维护"当前层" → O(|V|+|E|) （这就是队列版 BFS）
```

> 🎯 **考试要点**：
> 1. 无权最短路 = BFS（这是必考知识点）
> 2. 追踪表三个字段：Known / d_v / p_v，各自作用要清楚
> 3. 朴素版 O(V²) vs 队列版 O(V+E)，能说出优化原理
> 4. 路径重构方法：沿 p_v 数组回溯

> 💡 **前后呼应**：下一节将展示队列优化版的完整代码，从 O(V²) 降到 O(V+E)。这个优化方式与拓扑排序从朴素版到 Kahn 版的优化如出一辙——同样是"用队列避免全表扫描"。
