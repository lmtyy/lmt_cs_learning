# 拓扑排序代码实现与优化（p067-072）

从朴素 O(|V|²) 实现到队列优化的 O(|V|+|E|) —— 这是一个经典的"识别瓶颈→用数据结构消除"的优化范例。

---

## 第67页：朴素版拓扑排序代码

### 完整代码

```c
void topsort(graph G)
{
    unsigned int counter;
    vertex v, w;
    for(counter = 0; counter < NUM_VERTEX; counter++)
    {
        v = find_new_vertex_of_indegree_zero();
        if(v == NOT_A_VERTEX)
        {
            error("Graph has a cycle");
            break;
        }
        top_num[v] = counter;
        for each w adjacent to v
            indegree[w]--;
    }
}
```

### 逐段精讲

**1. 主循环——输出 N 个顶点**

```c
for(counter = 0; counter < NUM_VERTEX; counter++)
```

- **动作**：循环 N 次，每次输出一个顶点。
- **目的**：最终 `top_num[v] = counter` 表示顶点 v 在拓扑序中排第几。counter 从 0 到 N-1 递增。

**2. 核心操作——找入度为 0 的顶点**

```c
v = find_new_vertex_of_indegree_zero();
```

- **动作**：扫描 `indegree[]` 数组，找一个**入度为 0** 且**尚未输出**的顶点。
- **返回值**：找到就返回该顶点；找不到返回 `NOT_A_VERTEX`。

**3. 判环——找不到入度 0 的顶点**

```c
if(v == NOT_A_VERTEX)
{
    error("Graph has a cycle");
    break;
}
```

- **动作**：如果还有顶点没输出，但所有未输出顶点的入度都 > 0，说明**图中有环**。
- **为什么？** 如果剩余子图是 DAG，它一定至少有一个入度为 0 的顶点。找不到 = 不是 DAG = 有环。

> ✅ **副产品**：拓扑排序算法**天然能检测有向图中的环**。如果循环没跑满 N 次就 break，说明有环。

**4. 记录拓扑序号**

```c
top_num[v] = counter;
```

- **动作**：给顶点 v 分配拓扑序号。counter 越小 = 排得越前。
- **目的**：最终 `top_num` 数组就是拓扑排序的结果。

**5. 删除顶点 v 的出边——更新邻居入度**

```c
for each w adjacent to v
    indegree[w]--;
```

- **动作**：遍历 v 的所有邻居 w，将它们的入度减 1。
- **目的**：模拟"从图中删除 v 及其所有出边"。v 指向 w 的边没了，w 的入度自然减 1。
- **效果**：原本入度为 1 的 w，减到 0 后，下一轮就可以被选中输出。

> 🎯 **类比**："某门课的先修课都修完了"——每修完一门先修课，这门课的"剩余先修课计数"减1。减到0 = 所有先修课修完 = 可以选这门课了。

## 第68页：`find_new_vertex_of_indegree_zero` 的细节

### 实现方式

这个函数的工作很简单：

```c
// 伪代码
vertex find_new_vertex_of_indegree_zero() {
    for(v = 0; v < NUM_VERTEX; v++)
        if(indegree[v] == 0 && !already_assigned[v])
            return v;
    return NOT_A_VERTEX;
}
```

- 线性扫描 `indegree[]` 数组
- 找第一个 **入度为0** 且 **还没被分配拓扑序号** 的顶点
- 找不到返回 `NOT_A_VERTEX` → 表示有环

### 返回 NOT_A_VERTEX 的含义

> **如果不存在入度为 0 的未分配顶点，说明图有环。**

**直觉理解**：DAG 中的"剩余子图"（去掉已输出顶点后）仍然是 DAG，一定有入度为 0 的点。如果没有——说明每个剩余顶点都被其他剩余顶点指向——这只有环才能做到。

## 第69页：朴素版的时间复杂度

### 分析

```
find_new_vertex_of_indegree_zero() 每次调用 = O(|V|)（扫描整个数组）
共调用 |V| 次（主循环 N 轮）

总时间 = |V| × O(|V|) = O(|V|²)
```

### 瓶颈在哪？

每一轮都**重新扫描整个 indegree 数组**去找入度为 0 的点。但其实——

- 上一轮已经看过大部分顶点了
- 新变成入度 0 的点就那么几个（只有被减到 0 的邻居）
- 绝大部分顶点的入度没变，白白重复检查

> 💡 **优化方向**：维护一个"入度为 0 的顶点集合"——新增入度 0 的点直接加入，取出一个就少一个，不需要每次全盘扫描。

---

## 第70页：优化思路——"盒子"策略

### 核心思想

> **把所有入度为 0 的顶点放进一个"盒子"（special box），需要时直接从盒子里取。**

### 维护规则

| 时机 | 操作 |
|------|------|
| 初始化 | 遍历一遍 indegree[]，所有初始入度为 0 的顶点放入盒子 |
| 输出 v 后 | 检查 v 的每个邻居 w：`indegree[w]--`。如果变为 0 → 放入盒子 |
| 取下一个 | 从盒子取出一个顶点（如果盒子空了但还有未输出顶点 → 有环） |

---

## 第71页：维护入度为 0 的"盒子"的细节

### 入队时机

> **当我们对邻居入度减 1 时，如果入度变为 0，立刻放入盒子。**

这样就**不需要每轮重新扫描**——新的入度为 0 的顶点总是在"被减"的那一刻自动加入盒子。

### "盒子"用什么数据结构？

PPT说：

> **"We can use either a stack or a queue."**

| 实现 | 特性 | 影响 |
|------|------|------|
| 队列 | FIFO | 拓扑序偏"宽度优先"，编号小的先出 |
| 栈 | LIFO | 拓扑序偏"深度优先"，后发现的先出 |

**两种都能产生合法的拓扑序**——只是具体顺序不同。正确性不受影响。

> 💡 **为什么栈和队列都行？** 因为"盒子"里的顶点**互相没有依赖关系**（都是入度 0），取谁先输出都合法。用栈还是队列只决定"同层顶点之间的相对顺序"。

## 第72页：队列优化版完整算法描述（Kahn 算法）

### 算法步骤

1. **计算所有顶点的入度**
2. **将所有入度为 0 的顶点放入队列**
3. **while 队列非空**：
   - 取出队头顶点 v
   - 输出 v（拓扑序）
   - 遍历 v 的所有邻居 w：
     - `indegree[w]--`
     - 如果 `indegree[w] == 0`：**将 w 入队**
4. 如果输出的顶点数 < N → 有环

### 伪代码

```c
void topsort_optimized(graph G)
{
    Queue Q;
    int counter = 0;
    
    // 步骤1：计算所有入度
    compute_indegrees(G, indegree);
    
    // 步骤2：初始入度为0的入队
    for(v = 0; v < NUM_VERTEX; v++)
        if(indegree[v] == 0)
            EnQueue(Q, v);
    
    // 步骤3：逐个出队处理
    while(!QueueEmpty(Q))
    {
        v = DeQueue(Q);
        top_num[v] = ++counter;    // 记录拓扑序号
        
        for each w adjacent to v
        {
            indegree[w]--;
            if(indegree[w] == 0)
                EnQueue(Q, w);     // 入度变0，入队
        }
    }
    
    // 步骤4：判环，每个顶点至少与一个入度为0的顶点联通，所以所有顶点至少会遍历一次，倘若有顶点被遍历两次，counter就会对应多加1导致最终不等于总点数，也就代表有环
    if(counter != NUM_VERTEX)
        error("Graph has a cycle");
}
```

### 时间复杂度分析

| 步骤 | 操作 | 时间 |
|------|------|------|
| 计算入度 | 遍历所有边 | O(\|E\|) |
| 初始入队 | 扫描所有顶点 | O(\|V\|) |
| 主循环 | 每个顶点入队/出队恰好1次 | O(\|V\|) |
| 更新入度 | 每条边恰好被检查1次 | O(\|E\|) |
| **总计** | | **O(\|V\| + \|E\|)** |

> ✅ **从 O(|V|²) 优化到 O(|V|+|E|)**！对稀疏图提升巨大。

### 为什么每条边只检查一次？

每条边 (u, w) 只在"u 被输出时"用来更新 `indegree[w]`。u 只被输出一次 → 这条边只被处理一次。所有边加起来 = O(|E|)。

> 💡 **和 BFS 对比**：Kahn 算法本质上**就是 BFS**！
> - 初始"种子集" = 所有入度为 0 的顶点（BFS 的起点层）
> - "邻居入度变 0" = "BFS 发现新的可访问顶点"
> - 队列驱动 + 逐层处理
> - 唯一区别是 BFS 用 `visited[]`，Kahn 用 `indegree[]`

> 💥 **对比错误写法（朴素版）的本质区别**：
> - 朴素版：每轮**重新遍历全表**找入度 0 → 重复劳动
> - 优化版：用队列**只在入度变为 0 时**记录一次 → 不重复

## 总结

```
拓扑排序实现对比：

朴素版（第67页）：
  每轮扫全表找入度0 → O(|V|²)
  ├── 简单直观
  └── 瓶颈：重复扫描

优化版 / Kahn 算法（第72页）：
  队列维护入度为0的顶点 → O(|V| + |E|)
  ├── 本质 = 图上的 BFS
  ├── 入度数组 + 队列 → 不再重复扫描
  └── 副产品：counter < N 即为有环

关键优化原理：
  "找入度0" 从 被动搜索 变成 主动通知 → 观察者模式
  每次 indegree[w]-- 后检查 → 变0就入队 → O(1) 发现新候选
```
