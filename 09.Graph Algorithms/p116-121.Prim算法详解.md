# Prim 算法详解（p116-121）

MST 的第一种求解方法——Prim 算法。核心思想"让树一点一点长大"，每次贪心地选最便宜的跨界边。结构上和 Dijkstra 几乎一模一样，区别仅在更新规则。

---

## 第116页：Prim 算法的核心思想（"树的生长"）

### 算法概述

> **"One way to compute a minimum spanning tree is to grow the tree in successive stages."**
> **"In each stage, one node is picked as the root, and we add an edge, and thus an associated vertex, to the tree."**

**翻译**：
- MST 从一个顶点（根）开始，逐步"生长"
- 每一步增加一条边（同时带入一个新顶点）
- 重复直到覆盖所有顶点

> 🎯 **生活类比**：想象你在建设电网——先连好一片区域（起始点），然后**每次只往这片区域的边界上接最便宜的一根电线**，把相邻的一个新区域纳入。一步一步长大，直到覆盖全市。

---

## 第117页：Prim 的贪心选择策略

### 算法执行中的"两个阵营"

> **"At any point in the algorithm, we can see that we have a set of vertices that have already been included in the tree; the rest of the vertices have not."**

| 集合 | 含义 | 类比 |
|------|------|------|
| **U**（已在树中） | 已经被 MST 纳入的顶点 | "已通电的区域" |
| **V - U**（未在树中） | 还没被纳入的顶点 | "还没通电的区域" |

### 贪心选择

> **"The algorithm then finds, at each stage, a new vertex to add to the tree by choosing the edge (u, v) such that the cost of (u, v) is the smallest among all edges where u is in the tree and v is not."**

**核心操作**：在所有"一端在 U 中，另一端在 V-U 中"的边里，选**权值最小**的那条。

```
选择规则：
  找边 (u, v) 使得：
    - u ∈ U      （已在树中）
    - v ∈ V-U    （不在树中）
    - w(u,v) 最小（所有跨界边中最便宜的）
  
  然后：把 v 加入 U，(u,v) 加入 MST
```

> 💡 **这就是"切割性质"的直接应用**：
> - 当前 U 和 V-U 形成一个**切割**
> - 跨越这个切割的最轻边一定属于 MST（Cut Property）
> - Prim 每一步都在选这条最轻的跨界边 → 正确性有数学保证

> ⚠️ **为什么贪心这里有效？** 因为 MST 有"贪心选择性质"——每次选局部最优（最轻跨界边）不会破坏全局最优。这不是所有问题都成立的（如背包问题贪心就不一定对），MST 的特殊结构（树的单调性）保证了它。

## 第118页：Prim 的形式化描述

### 形式化步骤

设 V = {1, 2, ..., n}：

1. 初始化 **U = {1}**（任选一个起点，比如顶点1）
2. 重复以下步骤直到 **U = V**：
   - 找到**最短边 (u, v)**，满足 u ∈ U, v ∈ V-U
   - 将 v 加入 U
   - 将边 (u, v) 加入 MST
3. 结束时 MST 恰好有 n-1 条边

### 存储方式

> **"Represented by adjacency matrix"**

PPT指出 Prim 算法用**邻接矩阵**表示图——因为 Prim 需要频繁查询"两点间的边权"，邻接矩阵 O(1) 查询最适合。

### 逐步执行追踪

**Step 1：U = {V0}**

跨界边（一端在{V0}，一端不在）：
- V0-V1: 6
- V0-V2: **1** ← 最小
- V0-V3: 5

**选 V0-V2(1)**，V2 加入 U。

**Step 2：U = {V0, V2}**

跨界边：
- V0-V1: 6, V0-V3: 5
- V2-V1: 5, V2-V3: 5, V2-V4: 6, V2-V5: 4

**选某条最小跨界边**，依次加入 V5, V3, V1, V4。

### 完整生长顺序（PPT）

| 步骤 | U 集合 | 加入的顶点 | 选用的边 | 权值 |
|------|--------|----------|---------|------|
| 0 | {V0} | — | — | — |
| 1 | {V0, V2} | V2 | V0-V2 | 1 |
| 2 | {V0, V2, V5} | V5 | V2-V5 | 4 → 但PPT写2? |
| 3 | {V0, V2, V5, V3} | V3 | V3-V5 | 2 |
| 4 | {V0, V2, V5, V3, V1} | V1 | V1-V2? | 5 |
| 5 | {V0, V2, V5, V3, V1, V4} | V4 | V1-V4 | 3 |

（具体选边顺序以 PPT 图为准）

### 最终 MST 边集

从 PPT 最后一张子图看，MST 包含：V0-V2(1), V2-V5(某权), V3-V5(2), V1-V2(5或别的), V1-V4(3)

> 💡 **关键观察**：每一步都是"从已连通区域的边界向外伸展最短的一根触手"。新加入的顶点和它连入的边一起成为 MST 的一部分。

## 第120页：Prim 的追踪表结构

### 三个字段（和无权最短路/Dijkstra 相同）

| 字段 | 含义（Prim 版） |
|------|----------------|
| **Known** | 顶点是否已在 MST 中（= 在 U 中） |
| **d_v** | 从 v 到**已在树中的最近顶点**的边权 |
| **p_v** | 最近的那条边连的**树中顶点** |

> ⚠️ **d_v 在 Prim 中的含义与 Dijkstra 不同！**

| | Dijkstra | Prim |
|---|---|---|
| **d_v 表示** | 从**源点**到 v 的**累积路径长** | 从 v 到**树中任意顶点**的**最短单边** |
| **更新规则** | `d_w = min(d_w, d_v + w(v,w))` | `d_w = min(d_w, w(v,w))` |

> 🎯 **核心区别一句话**：Dijkstra 的 d_v 是"路径累积"，Prim 的 d_v 是"连接代价"。

## 第121页：Prim 的更新规则

### Update Rule

> **"After a vertex v is selected, for each unknown w adjacent to v, d_w = min(d_w, c_{w,v})."**

**翻译**：v 被选入 MST 后，看 v 的每个**未在树中**的邻居 w：
- 如果边 (w,v) 的权值比 w 当前记录的 d_w 更小
- 就更新 d_w = c(w,v)，并记 p_w = v

### 与 Dijkstra 更新规则的对比

```c
// Dijkstra 更新
if(d_v + w(v,w) < d_w)
    d_w = d_v + w(v,w);    // 累积距离

// Prim 更新
if(w(v,w) < d_w)
    d_w = w(v,w);           // 仅看边权本身
```

> 💡 **为什么 Prim 不用累积？** 因为 MST 不关心"从起点到 v 走了多远"，只关心"把 v 接到树上**用哪条边最便宜**"。只要找到连接 v 和树的最短单边就够了。

> 💥 **常见错误**：把 Dijkstra 的 `d_v + w(v,w)` 照搬到 Prim —— 完全错误！Prim 更新是 `w(v,w)` 而不是 `d_v + w(v,w)`。这是考试中最常见的混淆点。

### Prim 算法伪代码（完整）

```
Prim(G, start):
    for each v in V:
        d[v] = ∞; known[v] = false; p[v] = null
    d[start] = 0
    
    repeat |V| times:
        v = unknown vertex with smallest d[v]    // 贪心选择
        known[v] = true                           // 加入 MST
        for each unknown w adjacent to v:
            if w(v,w) < d[w]:                     // ← Prim 更新！
                d[w] = w(v,w)
                p[w] = v
```

### 时间复杂度

| 实现方式 | "找最小 d_v" | "更新 d_w" | 总计 |
|---------|-------------|-----------|------|
| 数组扫描 | O(V) × V次 | O(E) 次更新 | **O(V²)** |
| 二叉堆 | O(log V) × V次 | O(E·log V) 次 decrease-key | **O((V+E) log V)** |
| 斐波那契堆 | O(log V) amortized | O(E) 次 O(1) decrease-key | **O(E + V log V)** |

> ✅ **稠密图**（E ≈ V²）：数组扫描 O(V²) 最优（不需要堆的开销）
> ✅ **稀疏图**（E ≈ V）：二叉堆 O((V+E)log V) ≈ O(V log V) 更好

## 总结

```
Prim 算法核心：

思想："从一棵只有起点的小树，每次选最便宜的跨界边，让树长大一步"

数据维护：
├── U：已在 MST 中的顶点集合
├── d_v：v 到 U 中最近顶点的距离（单条边权）
├── p_v：d_v 对应的树中顶点（用于重构 MST）
└── Known：是否已在 U 中

核心操作循环 |V| 次：
  1. 从 unknown 中选 d_v 最小的 v → 加入 MST
  2. 更新 v 的未知邻居：d_w = min(d_w, w(v,w))

正确性：切割性质（每次选的最轻跨界边必属于MST）

Prim vs Dijkstra（代码结构相同，更新规则不同）：
  Dijkstra: d_w = min(d_w, d_v + w(v,w))  ← 累积路径
  Prim:     d_w = min(d_w, w(v,w))         ← 仅边权

复杂度：O(V²)（数组）或 O((V+E)logV)（堆）
```

> 🎯 **考试口诀**："Prim 看边权，Dijkstra 看路径。Prim 长树，Dijkstra 找路。"
