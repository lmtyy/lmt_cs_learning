# 邻接表的代码实现 Graph ADT（p040-042）

把上一节讲的邻接表概念落地为 C 语言结构体——边节点、顶点节点、图结构三层嵌套，最后用带权有向图的实例验证理解。

## 第40页：边节点与顶点节点的结构体定义

### 完整代码

```c
#define MAX_VERTEX_NUM 20

typedef struct ArcNode
{
    int adjvex;              // 邻接顶点位置（下标），即终点在结点集Vnode[]数组中的下标，而起点由链表头Head[i]确定
    struct ArcNode *nextarc; // 指向下一条弧的指针
    InfoType *info;          // 弧信息指针（如权值）
}

typedef Vnode
{
    VertexType data;         // 顶点信息
    ArcNode *firstarc;       // 指向第一条弧的指针（以该点为起点的链表）
} Vnode, AdjList[MAX_VERTEX_NUM];
```

### 逐段精讲

**1. 边节点 `ArcNode`（链表中的"一颗珠子"）**

```c
typedef struct ArcNode
{
    int adjvex;              
    struct ArcNode *nextarc; 
    InfoType *info;          
}
```

| 字段 | 类型 | 作用 | 对应上一节概念 |
|------|------|------|--------------|
| `adjvex` | `int` | 这条边指向的目标顶点**在数组中的下标** | 上一节的 `dest` |
| `nextarc` | `ArcNode*` | 指向**同一起点**的下一条边 | 上一节的 `Link` |
| `info` | `InfoType*` | 边的附加信息（权值/容量等） | 上一节的 `cost` |

- **动作**：每个 `ArcNode` 表示一条边。
- **目的**：多条从同一顶点出发的边，通过 `nextarc` 串成单链表。
- **为什么 `adjvex` 是 `int` 而不是指针？** 因为顶点存在数组里，用下标就能 O(1) 定位到目标顶点。不需要指针间接寻址。

> 🎯 **类比**：`ArcNode` 就像通讯录里的一条记录——"联系人座号(adjvex) + 下一条记录的地址(nextarc) + 备注信息(info)"。

**2. 顶点节点 `Vnode`（数组中的"一格"）**

```c
typedef Vnode
{
    VertexType data;         
    ArcNode *firstarc;       
} Vnode, AdjList[MAX_VERTEX_NUM];
```

| 字段 | 类型 | 作用 |
|------|------|------|
| `data` | `VertexType` | 顶点本身的数据（名字/编号） |
| `firstarc` | `ArcNode*` | 指向该顶点**第一条出边**的指针 |

- **动作**：每个 `Vnode` 是数组 `AdjList` 中的一格，代表一个顶点。
- **目的**：`firstarc` 是这个顶点对应的邻接链表的**链表头**。如果该顶点没有出边，`firstarc = NULL`。

> 💡 **C语言技巧**：`AdjList[MAX_VERTEX_NUM]` 在 `typedef` 末尾定义了一个类型别名——`AdjList` 等价于 `Vnode[20]`。后面用 `AdjList vertices;` 声明就是开了20个 `Vnode` 的数组。

### 内存结构图

```
AdjList 数组（顶点表）
┌─────────┬──────────┐
│  data   │ firstarc─┼──→ [ArcNode] → [ArcNode] → NULL
├─────────┼──────────┤
│  data   │ firstarc─┼──→ [ArcNode] → NULL
├─────────┼──────────┤
│  data   │ firstarc─┼──→ NULL  （没有出边）
├─────────┼──────────┤
│  ...    │  ...     │
└─────────┴──────────┘
```

## 第41页：图的完整结构体 `ALGraph`

### 完整代码

```c
typedef struct
{
    AdjList vertices;    // 顶点数组（含各自的邻接链表头指针）
    int vexnum, arcnum;  // 图中当前顶点数和弧数
    int kind;            // 图的种类标志
} ALGraph;
```

### 逐段精讲

| 字段 | 类型 | 作用 |
|------|------|------|
| `vertices` | `Vnode[20]` | 存所有顶点 + 各自链表头 |
| `vexnum` | `int` | 实际顶点数 |
| `arcnum` | `int` | 实际边/弧数 |
| `kind` | `int` | 图类型（DG/DN/UDG/UDN） |

### 与邻接矩阵版本 `Mgraph` 对比

| | Mgraph（邻接矩阵） | ALGraph（邻接表） |
|---|---|---|
| 存顶点 | `vexs[20]` — 纯数据数组 | `vertices[20]` — 数据 + 链表头 |
| 存边 | `arcs[20][20]` — 二维数组 | 分散在各链表的 ArcNode 中 |
| 查边 (i,j) | `arcs[i][j]` → O(1) | 遍历 `vertices[i]` 的链表 → O(degree) |
| 空间 | 固定 400 格 | N 个头 + E 个 ArcNode |

> ⚠️ **注意**：`ALGraph` 中没有单独的"边数组"——边分散在各顶点的链表里。如果你想统计"这个图一共有几条边"，要么信任 `arcnum` 字段，要么遍历所有链表数节点（代价 O(N+E)）。

## 第42页：带权有向图（网络）的邻接表实例

### 图 G9 结构

4个顶点：A(0), B(1), D(2), C(3)

边与权值：
```
A→B: 1,  A→D: 4
B→D: 2,  B→C: 9  (原文标注为3, 权9)
D→C: 6
C→A: 3,  C→B: 5,  C→D: 8
```

### 邻接表表示

```
下标 | data | 链表 [dest, cost, link]
  0  |  A   | → [1, 1, ●] → [2, 4, ∧]
  1  |  B   | → [2, 2, ●] → [3, 9, ∧]
  2  |  D   | → [3, 6, ∧]
  3  |  C   | → [0, 3, ●] → [1, 5, ●] → [2, 8, ∧]
```

### 如何读这个表

以 C(下标3) 为例：
- C 有3条出边：C→A(权3)、C→B(权5)、C→D(权8)
- 对应链表：`[0,3] → [1,5] → [2,8] → NULL`
- 链表长度 = 3 = C 的出度

> 💡 **权值存在哪？** 存在 `ArcNode` 的 `info` 字段里（或者直接把 `info` 替换为一个 `int weight`）。每个边节点现在是三元组 `{dest, cost, next}`。

> ⚠️ **易混点**：注意 PPT 中顶点编号的映射——A=0, B=1, D=2, C=3（不是按字母顺序 ABCD=0123）。这种乱序在实际问题中很常见，`LocateVex` 的作用就是做这个名字→下标的转换。

### 对比邻接矩阵表示

同一个图用邻接矩阵：

```
    A(0) B(1) D(2) C(3)
A [  0    1    4    ∞  ]
B [  ∞    0    2    9  ]
D [  ∞    ∞    0    6  ]
C [  3    5    8    0  ]
```

- 矩阵：16格，其中8格有效（非∞非对角线）→ 利用率50%
- 邻接表：4个头节点 + 8个ArcNode → 总12个节点

> 这个例子比较稠密（8条边/4个顶点），两种方案差距不大。但如果是1000顶点3000边的图，矩阵需 10⁶ 格而邻接表只需 4000 节点——差距250倍。

## 总结

```
邻接表代码三层结构：

第1层：ArcNode（边节点）
  { adjvex, *nextarc, *info }
  ↓ 多个串成链表

第2层：Vnode（顶点节点）= 数组一格
  { data, *firstarc → 链表头 }
  ↓ 多个组成数组

第3层：ALGraph（图）
  { vertices[20], vexnum, arcnum, kind }

带权图：info字段存权值 → 边节点变成 [dest, cost, next] 三元组
```

> 💡 **对比记忆**：
> - 邻接矩阵版：`Mgraph { vexs[], arcs[][], vexnum, arcnum, kind }`
> - 邻接表版：`ALGraph { vertices[], vexnum, arcnum, kind }`
> - 前者边信息集中在二维数组里，后者边信息分散在链表节点中。
